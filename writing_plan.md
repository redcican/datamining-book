### 1.3 两大平衡原则

**原则一：数学严谨性 ↔ 工程直觉**
- 每个核心算法都必须给出**完整的数学推导**（定理、证明、必要假设），但推导之后必须附"**几何直觉图**"或"**直觉解释段落**"，用自然语言复述数学含义。
- 禁止"直接给出公式，不作推导"的写作方式。

**原则二：理论深度 ↔ 实践落地**
- 每章的代码块均须在真实数据集上运行，不使用玩具数据集（除非作为概念验证的最小示例）。
- 从零实现（scratch implementation）与调库实现（sklearn/PyTorch）对比并存，让学生同时理解原理与工程接口。

### 2.3 代码块规范

```python
# ============================================================
# 代码 N.M：算法名称 + 功能简述
# 数据集：使用的数据集名称与来源
# 依赖库：numpy, pandas, sklearn, ...
# ============================================================
# --- 1. 导入依赖与数据加载 ---
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
plt.rcParams['font.sans-serif'] = ['SimHei']  # 支持中文
# --- 2. 核心逻辑 ---
# ...
# --- 3. 结果输出 ---
# ...
```

**代码书写规范**：
- 变量命名遵循 snake_case，有意义的英文命名
- 每段逻辑用 `# --- n. 说明 ---` 分隔
- 复杂函数须有 docstring（中文说明）
- 代码行宽不超过 88 字符
- 代码块在网页端自动显示语言标签（Atom One Dark 主题，带顶部语言 badge）

**禁止事项（严格遵守）**：
1. **禁止示例输出**：代码块内不得出现任何形式的预期输出——既不允许单独的"示例输出"注释块（如 `# 输出示例：\n# result = 42`），也不允许行内预期值注释（如 `# 预期：0.95`）。运行结果说明须放在代码块**外部**的正文段落中。
2. **禁止多余空白行**：代码块内不得有多余空白行。逻辑分隔用 `# --- N. ---` 节标题，而非空白行堆砌；同一节内相关语句之间不加空白行。
3. **注释代替外部说明**：对代码逻辑的解释性文字须写为代码内的中文注释，而非代码块外的散文。代码和注释合在一起应足以自解释，无需额外"说明段"。
4. **所有导入须被使用**：代码块中 import 的每个库均须在代码中实际调用；禁止出现未使用的导入。
5. **案例代码块须完整可运行**：每个 `{% block type="case" %}` 对应的完整 Python 代码块（含数据加载、处理、模型训练、结果输出各阶段）须端到端可运行，不得仅展示函数定义而省略调用逻辑。

### 2.4 数学公式排版规范

- **行内公式**：用 `$...$`，例如损失函数 $\mathcal{L}$
- **重要公式**：单独成行，编号（章节.公式号），例如 **(3.7)**
- **推导过程**：使用 align 环境，每步附文字说明
- **向量/矩阵**：粗体表示向量 $\mathbf{x}$，大写粗体表示矩阵 $\mathbf{X}$
- 复杂定理须附"定理意义"解读段落

#### 带编号的显示公式（`\tag{}`）——强制规范

**错误写法**（单行 `$$`）：
```
$$y_i(\mathbf{w}^\top \mathbf{x}_i + b) \geq 1 \tag{3.30}$$
```

**正确写法**（多行 `$$`，开闭符各占一行）：
```
$$
y_i(\mathbf{w}^\top \mathbf{x}_i + b) \geq 1 \tag{3.30}
$$
```

**原因**：`micromark-extension-math`（`remark-math` 的底层解析器）在单行 `$$...$$` 中若内容含 `\tag{}`，可能将其按行内数学处理；而 KaTeX 行内模式不支持 `\tag{}`，导致渲染失败。将开闭 `$$` 各单独成行可强制触发块级（display）数学模式，`\tag{}` 才能正确渲染。

**记住**：所有含 `\tag{}` 的编号公式，必须使用多行 `$$` 格式，绝不允许将公式与 `$$` 写在同一行。

### 2.5 图表规范

| 图表类型 | 要求 |

#### 图表生成策略（重要，所有章节须遵守）

**规则一：能用 Python 生成的图，优先用代码生成。**

凡是数据可视化、算法过程图、数学函数图、实验结果图等，均须使用 Python（matplotlib / seaborn / plotly / networkx 等）生成，并将对应脚本放置于 `code/` 目录下，以章节编号命名：

```
code/
  ch02/
    fig2_3_normalization_comparison.py   # 图 2.3 的生成代码
    fig2_5_pca_visualization.py
  ch03/
    fig3_4_decision_boundary.py
    fig3_8_roc_curves.py
  ...
```

**图片统一存储路径**：所有生成的图片同时复制到 `public/figures/chXX/` 目录，使全部 `.md` 文档均可通过绝对路径 `/figures/chXX/figname.png` 引用，无需相对路径跳转。

**命名规范**（强制统一）：`fig{ch}_{sec}_{seq}_{description}.png`
- `ch`：章序号（1~14，不补零）
- `sec`：节序号（如 1.1 → `1_1`，可省略小节数字）
- `seq`：该节内图的流水号（01, 02, …）
- `description`：3~5 个英文单词，下划线分隔

示例：`fig1_1_01_timeline.png`、`fig2_3_01_normalization.png`、`fig3_4_02_roc_curves.png`

每个脚本须满足：
- 可独立运行（`python code/ch03/fig3_4_decision_boundary.py` 直接输出图片）
- 输出文件保存至 `code/chXX/output/` 并**同步复制**至 `public/figures/chXX/`，分辨率 300dpi
- 脚本顶部注释说明：对应图编号、图标题、所用数据集
- matplotlib 字体：优先 SimHei（支持中文），回退 DejaVu Sans
- 坐标轴标签 ≥ 12pt，标题 ≥ 13pt，刻度标签 ≥ 12pt，图例 ≥ 12pt（由 `apply_style()` 全局保证）

#### 可视化脚本编写规范（严格遵守）

**字体大小**：所有 `*fontsize=N` 参数中 N 不得小于 12；`apply_style()` 已将全局默认值设为 12，局部覆盖时同样须 ≥ 12。

**Legend（图例）**：
1. 图例不得遮挡主体数据区域（散点、折线、直方图、热力图等）。
2. 选择 `loc` 时须考虑数据分布：将图例放在数据密度最低的角落；对多子图使用不同 `loc`（在 `transforms` / `panel_data` 等列表中逐项指定，而非全局统一）。
3. 图例过宽时须换行：legend label 中用 `\n` 将长项拆为两行（如 `"正态拟合\nN(μ, σ)"`），避免图例框水平扩张遮挡数据。
4. 需要缩小图例尺寸时使用紧凑参数：`labelspacing=0.3, handlelength=1.5, handletextpad=0.5, borderpad=0.4`。
5. 不需要的图例须彻底删除（包括对应 `label=` 参数），不得保留空白图例框。

**文本标注（annotation / text）**：
1. 同侧多个标注在 y 方向必须梯级错开，禁止所有标注使用同一 `xytext` y 坐标。对左侧组和右侧组分别设计递减 y 列表（如 `[0.026, 0.015]` / `[0.026, 0.016, 0.007]`）并逐一 `zip` 赋值。
2. 文本框（`bbox=dict(...)`）居中放置（`ha="center"`）时，若文本宽度超过子图宽度的 60%，须将 x 坐标右移（如从 `0.50` 改为 `0.58`），防止左边缘覆盖 y 轴刻度。

**多子图布局**：
1. 同侧相邻同行节点（timeline、taxonomy 树等）间距 < 4 数据单位时，须使用分层 y（level 1 / level 2），并相应扩展 `ylim` 和图形高度。
2. 阶段标签（phase label）与分层标注框之间须保留 ≥ 0.5 数据单位的垂直间距。
3. 删除不需要的图例后，须同时删除对应的 `import`（如 `import matplotlib.patches as mpatches`），保持"所有导入须被使用"原则。

**运行方式**：`MPLBACKEND=Agg PYTHONPATH=code python3 code/chXX/figXXX.py`

**规则二：无法用 Python 生成的图，留占位 caption。**

凡是需要外部来源的图（如架构示意图、硬件照片、产品界面截图、论文原图引用等），在正文中以如下格式留下占位说明，待后期美工或人工补充：

### 2.6 学术写作规范（教材区别于博客的核心要求）

教材写作须严格区别于博客、技术文章和讲义。以下规范须在所有章节中一致执行。

#### 语言风格

| 博客/讲义风格（❌禁止） | 教材学术风格（✓ 要求） |
|---|---|
| "让我们来看一个例子…" | "如例 X.X 所示，…" |
| "这很直觉上好理解" | "直觉上，…；严格表述见定义 X.X。" |
| "其实很简单" | "其基本思想可归纳为…" |
| "哦对了，还有一点" | "此外，值得指出的是…" |
| 大量使用第一人称 "我们" | 学术性"我们"（表示作者与读者共同推进）可适度使用 |
| Emoji 作为文章装饰 | Emoji 仅用于专用栏目图标（🚀📐🔬🌏⚠️），禁止散布正文 |

#### 历史与发展脉络的叙述方式

**禁止**在描述历史发展时采用"阶段名 + 分项列表"的形式（博客风格）。

**要求**采用**叙事散文体（narrative prose）**：
- 以时间逻辑为主线，段落自然衔接，每段聚焦一个发展阶段
- 段首可用加粗的阶段名或年代范围作为提示词（如"**基础积累期（1960–1970年代）**　"）
- 在正文中自然引用代表性成果（"Codd（1970）的关系代数模型……"）
- 结合图表（时间线图）辅助视觉理解，正文中应有明确引用（"如图 X.X.X 所示"）

#### 结构与交叉引用

- 所有定义、定理、例题、图、表均须编号，正文中以 "（见定义 X.X）"、"（如图 X.X.X 所示）" 形式显式引用
- 每章节末尾的参考文献须与正文行内引用（Author, Year）对应
- 小节间过渡句是学术写作的基本要求，不得省略

#### `###` 子节与 Block 之间的衔接（强制）

每个 `###` 级子节标题与其后第一个正式块（`{% block %}`、`{% hint %}`、代码块、图片）之间，**必须有至少 1–2 句承接性散文**，用于：
1. 说明本子节在上下文中的位置（"完成了 X，下面……"）；
2. 阐明引入该 block 的动机或问题（"为了……，我们需要……"）；
3. 点明 block 的核心内容是什么（"以下给出……的形式化定义"）。

**禁止**：`###` 标题后紧接 `{% block %}` 或 ```` ``` ```` 而无任何过渡文字，即使上方段落已有铺垫，子节内也须独立承接。此规则对所有子节类型（概念定义、算法、例题、案例）一律适用。

### 2.7 专用文本块系统（Block 规范）——强制执行

#### 甲类：正式编号块（Tier A）

```markdown
{% block type="TYPE" label="章.序号" title="标题（英文名）" %}
...正文（支持 $数学$、**加粗**、代码、表格）...
{% endblock %}
```

| 类型（`type=`） | 徽标文字 | 颜色 | 用途 | 每节频率 |
|---|---|---|---|---|
| `definition` | **定义** | 蓝 `#2563eb` | 精确的数学/技术定义；凡正文中以"（见定义 X.X）"引用的概念，必须有此块 | 2–6 个 |
| `theorem` | **定理** | 绿 `#16a34a` | 已证明的数学结论（收敛性、最优性、KKT 条件等） | 1–4 个 |
| `corollary` | **推论** | 青 `#0891b2` | 直接从定理推出的次级结论 | 按需 |

{% endblock %}

### 2.8 算法类章节特有写作规范（第 3 章起强制执行）

算法类章节（分类、回归、聚类、关联规则等）在结构与深度上与概述章（第 1 章）和预处理章（第 2 章）有**根本性区别**。以下规范在 §2.1–§2.7 通用规范之上叠加执行，所有以"算法"为核心组织逻辑的章节必须遵守。

#### 2.8.1 算法的教学弧（Teaching Arc）——七步必须完整

每个主要算法或算法族，在正文中的展开必须遵循以下七步弧线，**不得跳过任何步骤**：

| 步骤 | 名称 | 内容要求 | 对应块 |
|---|---|---|---|
| 1 | **动机** | 为什么需要这个算法？它解决了更简单方法无法处理的什么问题？须有反例或数据驱动论据（1–2 段） | 正文叙述 |
| 2 | **形式化** | 将直觉翻译为精确的数学问题——通常是目标函数的优化（最小化损失/最大化似然）或统计模型设定 | `definition` 块 |
| 3 | **推导** | 从目标函数逐步推导求解过程，每步附数学理由；**禁止**"可以证明"而省略证明 | 正文 + `proof` hint |
| 4 | **算法块** | 将推导结果转化为语言无关的伪代码，含输入/输出说明、编号步骤、**时间复杂度标注** | `algorithm` 块 |
| 5 | **性质分析** | 收敛性（是否保证？速率？）、计算复杂度（$O()$）、前提假设（违反时发生什么？） | `theorem` 块 |
| 6 | **从零实现** | 仅用 NumPy/基础库实现核心逻辑，与 sklearn 等价接口对比，数值差异 < $10^{-6}$ | 代码块 |
| 7 | **适用场景与局限** | 何时用/何时不用，以精炼表格或 `caution` hint 列出 | `caution` hint |

#### 2.8.2 以目标函数为中心

算法类章节的核心组织逻辑是**目标函数**，而非"步骤列表"。每引入一个算法，必须首先回答：

> **"我们在最小化（或最大化）什么？"**

- 决策树：最大化信息增益（最小化条件熵）
- SVM：最大化几何间隔（等价于 $\min \frac{1}{2}\|\mathbf{w}\|^2$）
- 线性回归：最小化均方误差（等价于最大化高斯似然）
- Lasso：最小化 MSE $+ \lambda\|\boldsymbol{\beta}\|_1$

**如果一节读完，学生无法说出"这个算法在优化什么"，则该节尚未写完。**

#### 2.8.3 偏差–方差框架作为贯穿红线

每个算法引入时，须在**偏差–方差权衡谱**上明确定位，说明哪个超参数控制模型复杂度，正则化手段是什么。每个算法章节须包含一幅"偏差–方差权衡图"（模型复杂度 vs 训练/测试误差），且每个算法的局限性分析中必须明确指出该算法倾向于哪一侧（高偏差还是高方差）。

#### 2.8.4 算法间系统比较（每章必须包含独立比较节）

每个算法章节必须包含一个独立的**算法比较小节**（通常作为本章最后一个正文节），在**同一数据集**上对本章所有算法进行统一评测：

- **分类章**：决策边界网格图（九宫格）+ 混淆矩阵 + ROC-AUC 对比表
- **回归章**：拟合曲线叠加对比图 + 残差分布对比 + 指标对比表（$R^2$、RMSE、MAE）
- **聚类章**：多种形状数据集（月牙/同心圆/各向异性）上的聚类结果对比图 + 轮廓系数

比较须体现"同等机会"：所有模型经过相同代价的超参数搜索（如相同的 CV 轮次）。

#### 2.8.5 超参数分析须独立可视化

每个具有关键超参数的算法，须包含**超参数敏感性图**：

- 决策树：最大深度 vs 训练/测试准确率（U 形曲线）
- SVM：$C$ 和 $\gamma$ 的网格搜索热力图
- 线性回归：$\lambda$ vs 系数路径（Ridge）/ 非零系数数目（Lasso）
- KNN：$k$ vs 误差率

**禁止**仅给出"选用 $k=5$"而不解释选择依据。

#### 2.8.6 "从零实现"编写规范（算法章 D 级题强制要求）

1. **纯 NumPy 实现**：不得调用 sklearn 等高层 API；允许 `numpy`、`scipy.linalg`
2. **接口规范**：实现 `fit(X, y)` / `predict(X)` / `score(X, y)` 接口，与 sklearn 保持一致
3. **数值验证**：自实现结果与 sklearn 差异 < $10^{-6}$（确定性算法）或统计等价（随机算法）
4. **注释密度**：每个关键步骤须有中文注释，说明对应推导中的哪一步（"对应正规方程第三步"）

#### 2.8.7 与预处理章节的写作差异对照

| 维度 | 预处理章（第 2 章） | 算法章（第 3 章起） |
|---|---|---|
| 组织逻辑 | 数据问题驱动（缺失/异常/量纲） | **目标函数驱动**（最小化/最大化什么） |
| 数学推导 | 辅助理解，可适度略去 | **核心内容，必须完整**；收敛性/复杂度须有定理 |
| 算法块 | 可选（描述"如何操作"即可） | **必须**（每个算法须有正式伪代码块） |
| 从零实现 | 鼓励，非强制 | **强制**（至少每章主算法 1 个） |
| 方法间比较 | 按单一指标对比 | 须在同一数据集上统一对比，含可视化 |
| 超参数分析 | 无 | **须有敏感性图** |
| 偏差–方差分析 | 不适用 | **贯穿全章** |
| 统计推断 | 不需要 | 回归章须含置信区间、假设检验 |
| 诊断工具 | 缺失值热图、分布对比 | 残差图/学习曲线/决策边界图 |

#### 数学背景深度（本章数学要求最高之一）

**缺失值机制的概率模型**：
- MCAR（完全随机缺失）：$P(\mathbf{R} | \mathbf{X}_{obs}, \mathbf{X}_{mis}) = P(\mathbf{R})$
- MAR（随机缺失）：$P(\mathbf{R} | \mathbf{X}_{obs}, \mathbf{X}_{mis}) = P(\mathbf{R} | \mathbf{X}_{obs})$
- MNAR（非随机缺失）：条件概率依赖缺失值本身

**归一化与标准化**：
$$x_{norm} = \frac{x - x_{min}}{x_{max} - x_{min}}, \quad x_{std} = \frac{x - \mu}{\sigma}$$
分析两者对距离度量的不同影响。

**Mahalanobis 距离**（异常值检测）：
$$D_M(\mathbf{x}) = \sqrt{(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})}$$
证明：当 $\boldsymbol{\Sigma} = \mathbf{I}$ 时退化为欧氏距离。

**PCA 的数学推导**（作为规约方法引入）：
- 协方差矩阵 $\mathbf{C} = \frac{1}{n}\mathbf{X}^T\mathbf{X}$
- 特征分解 $\mathbf{C}\mathbf{v}_k = \lambda_k \mathbf{v}_k$
- 主成分的方差解释比 $\frac{\lambda_k}{\sum_i \lambda_i}$

**特征选择指标**：
- 互信息：$I(X;Y) = \sum_{x,y} p(x,y) \log \frac{p(x,y)}{p(x)p(y)}$
- F 统计量：方差分析推导

#### 数学背景深度（本章数学最为密集）

**信息论基础**（决策树）：
$$H(D) = -\sum_{k=1}^{K} \frac{|C_k|}{|D|} \log_2 \frac{|C_k|}{|D|}$$
$$\text{Gain}(D, A) = H(D) - \sum_v \frac{|D^v|}{|D|} H(D^v)$$
$$\text{Gain\_ratio}(D, A) = \frac{\text{Gain}(D,A)}{H_A(D)}, \quad H_A(D) = -\sum_v \frac{|D^v|}{|D|}\log_2\frac{|D^v|}{|D|}$$
$$\text{Gini}(D) = 1 - \sum_k p_k^2$$

**SVM 完整推导链**（核心数学，须逐步推导）：
1. 原始问题：$\min \frac{1}{2}\|\mathbf{w}\|^2$, s.t. $y_i(\mathbf{w}^T\mathbf{x}_i + b) \geq 1$
2. 软间隔（松弛变量）：$\min \frac{1}{2}\|\mathbf{w}\|^2 + C\sum_i \xi_i$
3. Lagrangian 函数与 KKT 条件
4. 对偶问题：$\max \sum_i \alpha_i - \frac{1}{2}\sum_{i,j}\alpha_i\alpha_j y_i y_j \mathbf{x}_i^T\mathbf{x}_j$
5. 核函数（Mercer 定理）：$K(\mathbf{x}_i, \mathbf{x}_j) = \phi(\mathbf{x}_i)^T\phi(\mathbf{x}_j)$

**反向传播**（神经网络）：
- 链式法则：$\frac{\partial \mathcal{L}}{\partial w_{ij}^{(l)}} = \frac{\partial \mathcal{L}}{\partial a_j^{(l)}} \cdot \frac{\partial a_j^{(l)}}{\partial w_{ij}^{(l)}}$
- $\delta$ 误差项定义与递推关系

**AdaBoost 指数损失**：
$$\mathcal{L}(H) = \mathbb{E}[e^{-yH(x)}]$$
证明 AdaBoost 每轮权重更新最小化指数损失。

**偏差-方差分解**（必须推导）：
$$\mathbb{E}[(y - \hat{f})^2] = \text{Bias}^2 + \text{Variance} + \text{Noise}$$

### 第四章 回归算法

> 本章建议分配 10~12 学时；§4.1–§4.5 适合本科课堂，§4.6（广义线性/样条/GAM）可作为选讲内容。本章须严格遵循 §2.8 算法章节写作规范。

```

    ↓
简单线性回归：OLS 解析解（正规方程）→ 梯度下降解
    ↓           → 几何解释（列空间投影）→ Gauss-Markov 定理
    ↓           → 假设检验（t/F 检验）→ 预测区间 vs 置信区间
多元线性回归 → 多重共线性（VIF）→ 正则化的必要性
    ↓
正则化：岭回归（L2，解析解）→ Lasso（L1，坐标下降）→ 弹性网络
    ↓    → 正则化路径图 → 最优 λ 交叉验证选择
非线性回归：多项式 → 自然样条 → 广义加法模型（GAM）
    ↓

    ↓

    ↓

```

#### 数学背景深度（本章必须完整推导，不得以"可以证明"省略）

**普通最小二乘（OLS）**：
- 目标函数：$\min_{\boldsymbol{\beta}} \|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2$
- 解析解（正规方程）：$\hat{\boldsymbol{\beta}} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$（须证明 $\mathbf{X}^T\mathbf{X}$ 可逆的充要条件：列满秩）
- 几何解释：帽子矩阵 $\mathbf{H} = \mathbf{X}(\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T$ 是 $\mathbf{y}$ 到 $\text{col}(\mathbf{X})$ 的正交投影
- Gauss-Markov 定理：在线性无偏估计量中，OLS 方差最小（BLUE）

**统计推断**（区分预测和推断两种使用场景）：
- $\hat{\boldsymbol{\beta}}$ 的分布：$\hat{\boldsymbol{\beta}} \sim \mathcal{N}(\boldsymbol{\beta},\, \sigma^2(\mathbf{X}^T\mathbf{X})^{-1})$
- 系数显著性 t 检验：$t_j = \hat{\beta}_j / \mathrm{se}(\hat{\beta}_j)$，自由度 $n-d-1$
- 总体显著性 F 检验：$F = \frac{\mathrm{MSR}}{\mathrm{MSE}} \sim F(d,\, n-d-1)$
- 置信区间（均值响应）vs 预测区间（单个新观测）：须推导两者宽度之差来自 $\mathbf{x}_0^T(\mathbf{X}^T\mathbf{X})^{-1}\mathbf{x}_0$ 与额外 $\sigma^2$ 项

**岭回归**（L2 正则化）：
- 目标：$\min_{\boldsymbol{\beta}}\|\mathbf{y}-\mathbf{X}\boldsymbol{\beta}\|_2^2 + \lambda\|\boldsymbol{\beta}\|_2^2$
- 解析解：$\hat{\boldsymbol{\beta}}_{\text{ridge}} = (\mathbf{X}^T\mathbf{X}+\lambda\mathbf{I})^{-1}\mathbf{X}^T\mathbf{y}$（注意：即使 $\mathbf{X}^T\mathbf{X}$ 奇异，ridge 始终可逆）
- 贝叶斯解释：等价于 $\boldsymbol{\beta}\sim\mathcal{N}(\mathbf{0},\,\sigma^2/\lambda\cdot\mathbf{I})$ 先验下的 MAP 估计
- 岭迹图：系数随 $\lambda$ 增大趋于 0 但不为 0

**Lasso 回归**（L1 正则化）：
- 目标：$\min_{\boldsymbol{\beta}}\|\mathbf{y}-\mathbf{X}\boldsymbol{\beta}\|_2^2 + \lambda\|\boldsymbol{\beta}\|_1$
- 稀疏性几何解释：L1 球（菱形）的角点与损失函数等高线的切点必然落在坐标轴上
- 坐标下降更新：软阈值算子 $\hat{\beta}_j \leftarrow \mathrm{sign}(z_j)(|z_j|-\lambda/2)_+$（须完整推导次梯度 KKT 条件）
- Ridge vs Lasso：Ridge 收缩至小值但不置零；Lasso 执行变量选择（硬置零）

**回归树（CART 回归版）**：
- 分裂准则：最小化加权 MSE：$\sum_{v\in\{L,R\}}\frac{|S_v|}{|S|}\cdot\mathrm{MSE}(S_v)$
- 叶节点预测：$\hat{y}=\mathrm{mean}\{y_i:\mathbf{x}_i\in\text{leaf}\}$（最优常数预测器的推导）
- 代价复杂度剪枝：$R_\alpha(T)=R(T)+\alpha|T|$，$\alpha$ 的交叉验证选择

**偏差–方差分解**（连续输出，须从均方预测误差推导）：
$$\mathbb{E}_{D,\varepsilon}\!\left[(y-\hat{f}(\mathbf{x}))^2\right] = \underbrace{\mathrm{Bias}^2[\hat{f}(\mathbf{x})]}_{\text{模型系统偏差}} + \underbrace{\mathrm{Var}[\hat{f}(\mathbf{x})]}_{\text{估计随机波动}} + \underbrace{\sigma^2}_{\text{不可约噪声}}$$
正则化的作用：$\lambda$ 增大 → 偏差↑ 方差↓；须绘制偏差²、方差、总误差随 $\lambda$ 的三条曲线。

### 第十三章 数据隐私与伦理

> 本章是全书**课程思政核心章**，须在技术严谨性之上融入价值判断。

#### 数学背景深度（本章数学重点：差分隐私）

**差分隐私正式定义**：
$$\Pr[\mathcal{M}(D) \in S] \leq e^\varepsilon \cdot \Pr[\mathcal{M}(D') \in S] + \delta$$
- $\varepsilon$-DP 的组合性（串行组合/并行组合定理）
- 全局敏感度：$\Delta f = \max_{D,D':\|D-D'\|_1=1} \|f(D) - f(D')\|_1$
- 拉普拉斯机制：$\mathcal{M}(D) = f(D) + \text{Lap}\left(\frac{\Delta f}{\varepsilon}\right)$
- 高斯机制（$(\varepsilon,\delta)$-DP 版本）
- Moments Accountant（深度学习训练中的隐私预算追踪）

**k-匿名性**：
- 定义：每条记录至少与 $k-1$ 条其他记录在准标识符上不可区分
- 局限性：同质性攻击，背景知识攻击

**联邦学习（FedAvg）**：
$$\mathbf{w}^{t+1} = \sum_{k=1}^K \frac{n_k}{n} \mathbf{w}_k^t$$
梯度聚合的隐私风险分析（梯度反演攻击）

#### 必含图表清单
1. 隐私保护技术谱图（从 k-匿名到差分隐私到联邦学习）
2. ε 值对隐私-效用权衡的影响曲线（多个函数的噪声分布）
3. 联邦学习架构图（客户端/服务器/聚合）
4. 中国隐私法规时间轴（《数据安全法》《个人信息保护法》）
