# 6.4 关联规则的兴趣度量

§6.2–6.3 介绍了两种高效的频繁项集挖掘算法——Apriori 和 FP-Growth。然而，频繁项集挖掘只是关联规则分析的第一步。从频繁项集生成的关联规则数量可能非常庞大，其中大量规则是**平凡的**（trivially true）或**具有误导性的**（misleading）。例如，"购买了任意商品的顾客也购买了塑料袋"这条规则虽然支持度和置信度都很高，但并不包含有价值的信息。

问题的根源在于：§6.1 中定义的**支持度-置信度框架**（support-confidence framework）存在固有缺陷——置信度无法区分"因果相关"与"统计巧合"。当规则后件本身具有很高的支持度时，即使前件和后件完全独立，置信度也可能很高。

为了从大量规则中筛选出真正有价值的关联关系，研究者提出了多种**兴趣度量**（interestingness measures）。本节将系统介绍这些度量的数学定义、统计背景和实践选择指南。

## 6.4.1 支持度-置信度框架的局限性

回顾 §6.1 中的定义：关联规则 $$A \Rightarrow B$$ 的置信度为 $$\text{conf}(A \Rightarrow B) = P(B | A)$$。考虑以下经典反例。

**例 6.4**　某便利店的 10000 笔交易数据中：
- 购买咖啡的交易：6000 笔
- 购买茶的交易：7500 笔
- 同时购买咖啡和茶的交易：4500 笔

规则"咖啡 → 茶"的置信度为 $$4500 / 6000 = 0.75$$，看起来是一条强规则。但茶的整体购买率已经是 $$7500 / 10000 = 0.75$$！知道顾客购买了咖啡，对预测其是否购买茶**毫无帮助**——因为 $$P(\text{茶}|\text{咖啡}) = P(\text{茶}) = 0.75$$。在这种情况下，咖啡和茶的购买行为是**统计独立**的，高置信度完全是因为茶本身的高支持度造成的假象。

更糟糕的情况是：如果同时购买咖啡和茶的交易只有 4000 笔，则 $$P(\text{茶}|\text{咖啡}) = 4000/6000 = 0.667 < 0.75 = P(\text{茶})$$——购买咖啡实际上**降低了**购买茶的概率！这表明咖啡和茶之间存在**负关联**，但置信度 0.667 仍然给人"强规则"的错觉。

这个例子揭示了置信度的根本问题：它**不考虑规则后件的基础概率**。为了修正这一缺陷，我们需要引入新的兴趣度量。

---

## 6.4.2 列联表：统一的分析框架

在引入各种度量之前，先建立一个统一的分析工具。

{% block type="definition" title="定义 6.12　$$2 \times 2$$ 列联表（Contingency Table）" %}
给定事务数据库 $$\mathcal{D}$$（共 $$n$$ 条事务）和两个项集 $$A$$、$$B$$，定义列联表：

|  | $$B$$ | $$\overline{B}$$ | 合计 |
|--|-----|-----------------|------|
| $$A$$ | $$f_{11}$$ | $$f_{10}$$ | $$f_{1+}$$ |
| $$\overline{A}$$ | $$f_{01}$$ | $$f_{00}$$ | $$f_{0+}$$ |
| 合计 | $$f_{+1}$$ | $$f_{+0}$$ | $$n$$ |

其中：
- $$f_{11} = n \cdot \text{supp}(A \cup B)$$：同时包含 $$A$$ 和 $$B$$ 的事务数
- $$f_{10} = n \cdot \text{supp}(A) - f_{11}$$：包含 $$A$$ 但不包含 $$B$$ 的事务数
- $$f_{01} = n \cdot \text{supp}(B) - f_{11}$$：包含 $$B$$ 但不包含 $$A$$ 的事务数
- $$f_{00} = n - f_{11} - f_{10} - f_{01}$$：既不包含 $$A$$ 也不包含 $$B$$ 的事务数
{% endblock %}

列联表将关联规则分析转化为经典的双变量频率分析问题。所有兴趣度量都可以用 $$f_{11}, f_{10}, f_{01}, f_{00}$$ 这四个计数来表示。

为简化符号，定义以下概率量：

$$
p_A = \frac{f_{1+}}{n},\quad p_B = \frac{f_{+1}}{n},\quad p_{AB} = \frac{f_{11}}{n} \tag{6.26}
$$

其中 $$p_A = \text{supp}(A)$$，$$p_B = \text{supp}(B)$$，$$p_{AB} = \text{supp}(A \cup B)$$。在此基础上，支持度和置信度可统一表示为：

$$
\text{supp}(A \Rightarrow B) = p_{AB}, \quad \text{conf}(A \Rightarrow B) = \frac{p_{AB}}{p_A} \tag{6.27}
$$

下面逐一引入各种兴趣度量。

## 6.4.3 基于独立性的度量

判断关联规则是否有趣的核心思想是：**检验 $$A$$ 和 $$B$$ 是否统计独立**。若 $$A$$ 和 $$B$$ 独立，则 $$P(A \cap B) = P(A) \cdot P(B)$$，即 $$p_{AB} = p_A \cdot p_B$$。任何偏离独立假设的程度都可以作为"兴趣"的度量。

{% block type="definition" title="定义 6.13　提升度（Lift）" %}
规则 $$A \Rightarrow B$$ 的**提升度**定义为：

$$
\text{lift}(A \Rightarrow B) = \frac{P(A \cap B)}{P(A) \cdot P(B)} = \frac{p_{AB}}{p_A \cdot p_B} = \frac{\text{conf}(A \Rightarrow B)}{P(B)} \tag{6.28}
$$

- $$\text{lift} = 1$$：$$A$$ 和 $$B$$ 统计独立
- $$\text{lift} > 1$$：$$A$$ 和 $$B$$ 正关联（co-occurrence）
- $$\text{lift} < 1$$：$$A$$ 和 $$B$$ 负关联（avoidance）
{% endblock %}

提升度是最直观的独立性偏离度量，但它有一个重要缺陷——对**稀有项**敏感。当 $$p_A$$ 和 $$p_B$$ 都很小时，即使 $$f_{11}$$ 只是随机波动，提升度也可能非常大。

提升度与经典统计学中的 $$\chi^2$$（卡方）独立性检验有深刻的联系。

{% block type="theorem" title="定理 6.9　提升度与 $$\chi^2$$ 检验的关系" %}
给定列联表（定义 6.12），$$\chi^2$$ 统计量为：

$$
\chi^2 = \sum_{i \in \{0,1\}} \sum_{j \in \{0,1\}} \frac{(f_{ij} - e_{ij})^2}{e_{ij}} \tag{6.29}
$$

其中 $$e_{ij} = f_{i+} \cdot f_{+j} / n$$ 是独立假设下的期望频次。则：

$$
\chi^2 = 0 \iff \text{lift}(A \Rightarrow B) = 1 \tag{6.30}
$$

即 $$\chi^2 = 0$$ 当且仅当 $$A$$ 和 $$B$$ 统计独立。
{% endblock %}

**证明**　$$\chi^2 = 0$$ 当且仅当 $$f_{ij} = e_{ij}$$ 对所有 $$i, j$$ 成立。特别地，$$f_{11} = e_{11} = f_{1+} \cdot f_{+1} / n$$，即 $$p_{AB} = f_{11}/n = (f_{1+}/n) \cdot (f_{+1}/n) = p_A \cdot p_B$$，这正是 $$\text{lift} = p_{AB} / (p_A \cdot p_B) = 1$$。反方向同理。$$\blacksquare$$

定理 6.9 告诉我们，提升度偏离 1 的程度等价于 $$\chi^2$$ 统计量偏离 0 的程度。在实践中，可以对 $$\chi^2$$ 进行**显著性检验**（自由度为 1 的 $$\chi^2$$ 分布，$$p$$-value < 0.05）来判断关联是否具有统计显著性，而非仅凭提升度的绝对值。

## 6.4.4 基于相关性的度量

提升度虽然直观，但在某些场景下不够稳健。以下几种度量从不同角度衡量 $$A$$ 和 $$B$$ 的关联强度。

{% block type="definition" title="定义 6.14　全置信度（All-Confidence）" %}
项集 $$A \cup B$$ 的**全置信度**定义为：

$$
\text{all\_conf}(A, B) = \frac{\text{supp}(A \cup B)}{\max(\text{supp}(A), \text{supp}(B))} = \frac{p_{AB}}{\max(p_A, p_B)} \tag{6.31}
$$

全置信度等于规则 $$A \Rightarrow B$$ 和 $$B \Rightarrow A$$ 中置信度较小的那个。
{% endblock %}

全置信度的优势在于它同时考虑了两个方向的置信度，避免了方向性偏差。例如，"啤酒 → 尿布"的置信度可能很高，但"尿布 → 啤酒"的置信度很低——全置信度取较小值，给出更保守的评估。

{% block type="definition" title="定义 6.15　余弦相似度（Cosine / IS Measure）" %}
项集 $$A$$ 和 $$B$$ 之间的**余弦相似度**（也称 IS 度量）定义为：

$$
\cos(A, B) = \frac{p_{AB}}{\sqrt{p_A \cdot p_B}} = \sqrt{\text{conf}(A \Rightarrow B) \cdot \text{conf}(B \Rightarrow A)} \tag{6.32}
$$
{% endblock %}

余弦相似度是两个方向置信度的**几何平均值**，取值范围为 $$[0, 1]$$。与提升度不同，余弦相似度不会因分母中的小概率乘积而产生极端大值。

{% block type="definition" title="定义 6.16　Kulczynski 度量" %}
项集 $$A$$ 和 $$B$$ 之间的 **Kulczynski 度量**定义为：

$$
\text{kulc}(A, B) = \frac{1}{2}\left(\frac{p_{AB}}{p_A} + \frac{p_{AB}}{p_B}\right) = \frac{1}{2}\left(\text{conf}(A \Rightarrow B) + \text{conf}(B \Rightarrow A)\right) \tag{6.33}
$$
{% endblock %}

Kulczynski 度量是两个方向置信度的**算术平均值**，取值范围同样为 $$[0, 1]$$。它与余弦相似度的区别在于使用算术平均而非几何平均，因此对两个方向置信度差异更敏感。

为了量化两个方向置信度的**不对称程度**，还可以引入不平衡比。

{% block type="definition" title="定义 6.17　不平衡比（Imbalance Ratio）" %}
项集 $$A$$ 和 $$B$$ 之间的**不平衡比**定义为：

$$
\text{IR}(A, B) = \frac{|p_A - p_B|}{p_A + p_B - p_{AB}} \tag{6.34}
$$

不平衡比衡量 $$A$$ 和 $$B$$ 的支持度差异程度，取值范围为 $$[0, 1]$$。当 $$\text{IR} = 0$$ 时，$$p_A = p_B$$，两个方向的置信度相等。
{% endblock %}

在实践中，Kulczynski 度量和不平衡比常常**联合使用**：Kulczynski 度量衡量关联强度，不平衡比衡量关联的对称性。一条理想的关联规则应当同时具有高 Kulczynski 值和低不平衡比。

## 6.4.5 零不变性：稀疏数据的试金石

在购物篮分析中，事务数据库通常是**极度稀疏**的——大多数事务不包含大多数商品，因此 $$f_{00}$$（既不包含 $$A$$ 也不包含 $$B$$ 的事务数）远大于其他三个计数。一个好的兴趣度量不应该受到 $$f_{00}$$ 的影响。

{% block type="definition" title="定义 6.18　零不变性（Null-Invariance）" %}
兴趣度量 $$M$$ 是**零不变的**（null-invariant），如果 $$M$$ 的值不随 $$f_{00}$$ 的变化而变化，即 $$M$$ 仅依赖于 $$f_{11}, f_{10}, f_{01}$$。
{% endblock %}

零不变性是衡量兴趣度量是否适用于稀疏数据的关键性质。

{% block type="theorem" title="定理 6.10　常见度量的零不变性" %}
在以下常见兴趣度量中：

| 度量 | 公式 | 零不变？ |
|------|------|---------|
| 支持度 | $$p_{AB}$$ | 是 |
| 置信度 | $$p_{AB} / p_A$$ | 是 |
| 提升度 | $$p_{AB} / (p_A \cdot p_B)$$ | **否** |
| 全置信度 | $$p_{AB} / \max(p_A, p_B)$$ | 是 |
| 余弦相似度 | $$p_{AB} / \sqrt{p_A \cdot p_B}$$ | 是 |
| Kulczynski | $$\frac{1}{2}(p_{AB}/p_A + p_{AB}/p_B)$$ | 是 |
| $$\chi^2$$ | 见式 (6.29) | **否** |

提升度和 $$\chi^2$$ 不是零不变的；其余度量是零不变的。
{% endblock %}

**证明**　对于提升度，$$\text{lift} = p_{AB} / (p_A \cdot p_B)$$。注意 $$p_B = (f_{11} + f_{01}) / n$$，而 $$n = f_{11} + f_{10} + f_{01} + f_{00}$$。当 $$f_{00}$$ 增大时，$$n$$ 增大，$$p_B$$ 减小，因此提升度增大。故提升度不是零不变的。

对于余弦相似度，$$\cos(A,B) = f_{11} / \sqrt{f_{1+} \cdot f_{+1}}$$。展开 $$f_{1+} = f_{11} + f_{10}$$，$$f_{+1} = f_{11} + f_{01}$$，均不含 $$f_{00}$$，故余弦相似度是零不变的。

其余度量的证明类似。$$\blacksquare$$

定理 6.10 的实际意义在于：当数据库非常稀疏（$$f_{00} \gg f_{11}$$）时，提升度和 $$\chi^2$$ 可能给出**虚假的高值**——仅仅因为大量"双缺席"事务推高了统计量。这时应优先使用零不变的度量（如 Kulczynski 或余弦相似度）。

## 6.4.6 度量的选择指南

各种兴趣度量各有优缺点，下面给出一个实用的选择框架：

| 场景 | 推荐度量 | 理由 |
|------|---------|------|
| 快速筛选 | 提升度 > 1 | 直观易解释，快速排除独立项对 |
| 稀疏数据 | Kulczynski + IR | 零不变，不受 $$f_{00}$$ 干扰 |
| 对称关联 | 余弦相似度 | 几何平均，对称且零不变 |
| 统计显著性 | $$\chi^2$$ + $$p$$-value | 严格的假设检验框架 |
| 综合评估 | 多度量联合 | 不同度量可能给出不同排序 |

实践中最常用的策略是**多度量联合筛选**：先用支持度和置信度做初步过滤（§6.1–6.2），再用 Kulczynski、余弦相似度等度量做精细排序。

## 6.4.7 算法实现

```python
# ============================================================
# 代码 6.5：关联规则兴趣度量的计算与对比
# 功能：从事务数据库计算多种兴趣度量
# 依赖库：numpy
# ============================================================
import numpy as np
# --- 1. 列联表计算 ---
def contingency_table(transactions, A, B):
    """计算项集 A 和 B 的 2×2 列联表"""
    n = len(transactions)
    f11 = sum(1 for t in transactions if A.issubset(t) and B.issubset(t))
    f10 = sum(1 for t in transactions if A.issubset(t) and not B.issubset(t))
    f01 = sum(1 for t in transactions if not A.issubset(t) and B.issubset(t))
    f00 = n - f11 - f10 - f01
    return f11, f10, f01, f00, n
# --- 2. 兴趣度量计算 ---
def compute_measures(f11, f10, f01, f00, n):
    """从列联表计算所有兴趣度量"""
    pA = (f11 + f10) / n
    pB = (f11 + f01) / n
    pAB = f11 / n
    # 支持度、置信度
    support = pAB
    conf_AB = pAB / pA if pA > 0 else 0
    conf_BA = pAB / pB if pB > 0 else 0
    # 提升度
    lift = pAB / (pA * pB) if pA * pB > 0 else 0
    # 全置信度
    all_conf = pAB / max(pA, pB) if max(pA, pB) > 0 else 0
    # 余弦相似度
    cosine = pAB / np.sqrt(pA * pB) if pA * pB > 0 else 0
    # Kulczynski
    kulc = 0.5 * (conf_AB + conf_BA)
    # 不平衡比
    denom_ir = pA + pB - pAB
    ir = abs(pA - pB) / denom_ir if denom_ir > 0 else 0
    # 卡方统计量
    e11 = (f11 + f10) * (f11 + f01) / n
    e10 = (f11 + f10) * (f10 + f00) / n
    e01 = (f01 + f00) * (f11 + f01) / n
    e00 = (f01 + f00) * (f10 + f00) / n
    chi2 = sum(
        (f - e) ** 2 / e if e > 0 else 0
        for f, e in [(f11, e11), (f10, e10), (f01, e01), (f00, e00)]
    )
    return {
        'support': support, 'conf(A→B)': conf_AB,
        'conf(B→A)': conf_BA, 'lift': lift,
        'all_conf': all_conf, 'cosine': cosine,
        'kulc': kulc, 'IR': ir, 'chi2': chi2,
    }
# --- 3. 测试：例 6.4 的反例 ---
n = 10000
# 咖啡和茶独立的情况
f11, f10, f01, f00 = 4500, 1500, 3000, 1000
measures = compute_measures(f11, f10, f01, f00, n)
print("=== 咖啡-茶（独立） ===")
for k, v in measures.items():
    print(f"  {k}: {v:.4f}")
# 正关联的情况（啤酒-薯片）
f11, f10, f01, f00 = 800, 200, 400, 8600
measures2 = compute_measures(f11, f10, f01, f00, n)
print("\n=== 啤酒-薯片（正关联） ===")
for k, v in measures2.items():
    print(f"  {k}: {v:.4f}")
```

上述代码实现了列联表计算和全部 8 种兴趣度量。运行后可以清晰看到：在咖啡-茶的独立情况下，提升度接近 1，Kulczynski 值为 0.5（无偏好方向）；在啤酒-薯片的正关联情况下，各度量一致给出高分。

## 6.4.8 案例分析

{% block type="case" title="案例 6.4　多种兴趣度量在购物篮数据上的对比" %}
**背景**：在购物篮分析中，不同兴趣度量可能对规则给出不同的排序。本案例在合成数据上生成关联规则，系统对比各度量的筛选效果，揭示度量选择对结论的影响。

```python
# ============================================================
# 案例 6.4：兴趣度量对比实验
# 数据集：合成购物篮数据（1000 笔交易，12 种商品）
# 依赖库：numpy, pandas
# ============================================================
import numpy as np
import pandas as pd
from itertools import combinations
# --- 1. 生成包含已知模式的合成数据 ---
np.random.seed(42)
items = ['面包', '牛奶', '鸡蛋', '啤酒', '薯片', '可乐',
         '酸奶', '水果', '蔬菜', '肉类', '调料', '饮料']
n_trans = 1000
# 植入的关联模式
known_patterns = {
    '强正关联': (['啤酒', '薯片'], 0.25),
    '弱正关联': (['面包', '牛奶'], 0.40),
    '独立对':   (['鸡蛋', '可乐'], 0.0),  # 无特殊关联
    '高频独立': (['水果', '蔬菜'], 0.0),   # 各自高频但独立
}
transactions = []
for _ in range(n_trans):
    basket = set()
    # 强正关联：啤酒和薯片同时出现
    if np.random.random() < 0.25:
        basket.update(['啤酒', '薯片'])
    # 弱正关联：面包和牛奶常同时出现
    if np.random.random() < 0.40:
        basket.update(['面包', '牛奶'])
        if np.random.random() < 0.5:
            basket.add('鸡蛋')
    # 独立的高频项
    for item, prob in [('水果', 0.6), ('蔬菜', 0.55),
                       ('鸡蛋', 0.3), ('可乐', 0.25)]:
        if np.random.random() < prob:
            basket.add(item)
    # 随机噪声
    for item in items:
        if item not in basket and np.random.random() < 0.08:
            basket.add(item)
    if basket:
        transactions.append(basket)
# --- 2. 计算所有 2-项集的兴趣度量 ---
n = len(transactions)
item_sup = {}
for item in items:
    item_sup[item] = sum(1 for t in transactions if item in t) / n
results = []
for a, b in combinations(items, 2):
    A, B = frozenset({a}), frozenset({b})
    f11 = sum(1 for t in transactions if a in t and b in t)
    f10 = sum(1 for t in transactions if a in t and b not in t)
    f01 = sum(1 for t in transactions if a not in t and b in t)
    f00 = n - f11 - f10 - f01
    pA, pB, pAB = (f11+f10)/n, (f11+f01)/n, f11/n
    if pA == 0 or pB == 0:
        continue
    lift = pAB / (pA * pB)
    cosine = pAB / np.sqrt(pA * pB)
    kulc = 0.5 * (pAB/pA + pAB/pB)
    ir = abs(pA - pB) / (pA + pB - pAB) if (pA+pB-pAB) > 0 else 0
    results.append({
        '项对': f'{a}-{b}', 'supp': pAB,
        'conf(A→B)': pAB/pA, 'conf(B→A)': pAB/pB,
        'lift': lift, 'cosine': cosine,
        'kulc': kulc, 'IR': ir,
    })
df = pd.DataFrame(results)
# --- 3. 各度量的 Top-5 规则 ---
for metric in ['lift', 'cosine', 'kulc']:
    top5 = df.nlargest(5, metric)[['项对', 'supp', metric, 'IR']]
    print(f"\n=== Top-5 by {metric} ===")
    print(top5.to_string(index=False))
```

**实验分析**：

1. **提升度 Top-5**：可能包含低支持度的稀有项对（如"调料-饮料"），因为稀有项的 $$p_A \cdot p_B$$ 很小，容易产生虚高的提升度。
2. **余弦/Kulczynski Top-5**：更倾向于选出真正的强关联模式（如"啤酒-薯片"），因为这些度量是零不变的，不受稀疏性影响。
3. **不平衡比**：帮助区分对称关联（如"啤酒-薯片"，IR 低）和不对称关联（如"面包-牛奶"，若支持度差异大则 IR 高）。

结论：单一度量不足以全面评价关联规则的质量。**推荐使用 Kulczynski + IR 的组合**：Kulczynski 衡量关联强度，IR 衡量对称性。

## 本节小结

| 关键概念 | 说明 |
|----------|------|
| 列联表（定义 6.12） | $$2 \times 2$$ 频次表，统一所有度量的计算基础 |
| 提升度（定义 6.13） | $$p_{AB} / (p_A p_B)$$，衡量偏离独立性的程度 |
| 全置信度（定义 6.14） | 两个方向置信度的最小值，保守评估 |
| 余弦相似度（定义 6.15） | 两个方向置信度的几何平均，对称且零不变 |
| Kulczynski（定义 6.16） | 两个方向置信度的算术平均，零不变 |
| 不平衡比 IR（定义 6.17） | 衡量两项支持度差异程度 |
| 零不变性（定义 6.18） | 度量不受 $$f_{00}$$ 影响的性质，稀疏数据关键 |
| $$\chi^2$$ 与提升度（定理 6.9） | $$\chi^2 = 0 \iff \text{lift} = 1$$（统计独立） |
| 零不变性分类（定理 6.10） | 提升度和 $$\chi^2$$ 非零不变，其余为零不变 |

兴趣度量为关联规则的质量评估提供了多维视角。没有一种度量在所有场景下都是最优的——选择取决于数据的稀疏性、分析的目标（探索 vs 决策）和对误报/漏报的容忍度。在下一节（§6.5）中，我们将扩展关联规则的层次，讨论多层次和多维关联规则挖掘。

**B1.** 证明余弦相似度 $$\cos(A, B) \leq 1$$，等号成立当且仅当 $$\text{supp}(A) = \text{supp}(B) = \text{supp}(A \cup B)$$（即 $$A$$ 和 $$B$$ 总是同时出现）。

**B2.** 证明以下不等式链：$$\text{all\_conf}(A, B) \leq \cos(A, B) \leq \text{kulc}(A, B)$$。（提示：利用 $$\min \leq$$ 几何平均 $$\leq$$ 算术平均。）

**B3.** 证明：对于 $$2 \times 2$$ 列联表，$$\chi^2$$ 统计量可简化为 $$\chi^2 = n(f_{11} f_{00} - f_{10} f_{01})^2 / (f_{1+} f_{0+} f_{+1} f_{+0})$$。

**B4.** 设 $$\text{lift}(A \Rightarrow B) = L$$。证明 $$\text{kulc}(A, B) = \frac{L}{2}\left(p_B + p_A\right) \cdot \frac{1}{1}$$ 不成立——即 Kulczynski 度量不能仅用提升度和边际概率的简单函数表示。给出 Kulczynski 度量关于 lift、$$p_A$$、$$p_B$$ 的精确表达式。

### F 级（开放探讨）

**F1.** **前沿视野·兴趣度量的公理化研究**：Piatetsky-Shapiro（1991）提出兴趣度量应满足三条公理：(1) 当 $$P(AB) = P(A)P(B)$$ 时度量为 0；(2) 其他条件不变时，$$P(AB)$$ 增大则度量增大；(3) 其他条件不变时，$$P(A)$$ 或 $$P(B)$$ 增大则度量减小。调研哪些常见度量满足全部三条公理，哪些只满足部分，并讨论这些公理是否足以刻画"好的"兴趣度量。

**F2.** **课程思政·数据分析中的"幸存者偏差"与"虚假关联"**：高置信度规则可能是统计巧合的产物——这与日常生活中的"虚假关联"（spurious correlation）现象异曲同工。讨论：(1) 在商业决策中，如果仅依据置信度来制定营销策略，可能导致什么后果？(2) "相关不等于因果"这一原则如何在关联规则分析中体现？(3) 作为数据分析师，如何向非技术背景的决策者解释"这条规则的置信度很高但实际上没有意义"？
