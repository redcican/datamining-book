# 6.1 频繁项集与关联规则基本概念

**关联规则挖掘**（Association Rule Mining）是数据挖掘中最具商业影响力的任务之一。它源于一个看似简单的问题：**超市中哪些商品经常被一起购买？** 1993 年，Agrawal 等人在 IBM Almaden 研究中心提出了这一问题的形式化定义和第一个高效算法 Apriori，开创了关联规则挖掘领域。此后，"啤酒与尿布"的故事（虽然可能是商业传说）成为数据挖掘最广为人知的案例。

关联规则的应用远不止零售：推荐系统（"购买了 A 的用户也购买了 B"）、医疗诊断（"同时出现症状 X 和 Y 时高概率患有疾病 Z"）、网络安全（"事件 A 之后常伴随事件 B"）、基因组学（"基因 A 和 B 倾向于共同表达"）——任何涉及**项目共现模式**的场景都可以建模为关联规则问题。

本节建立关联规则挖掘的**完整数学框架**，为后续 Apriori（§6.2）和 FP-Growth（§6.3）算法奠定理论基础。

## 6.1.1 从购物篮到数学模型

### 6.1.1.1 购物篮分析的直觉

考虑一家超市的交易记录：

| 交易 ID | 购买商品 |
|---------|---------|
| T1 | {面包, 牛奶, 鸡蛋} |
| T2 | {面包, 牛奶, 啤酒, 尿布} |
| T3 | {牛奶, 鸡蛋, 啤酒} |
| T4 | {面包, 牛奶, 鸡蛋, 啤酒} |
| T5 | {面包, 鸡蛋} |

直觉上，我们想回答以下问题：
- **频繁项集**：哪些商品组合经常一起被购买？（如 {面包, 牛奶}）
- **关联规则**：购买了 A 的顾客是否倾向于同时购买 B？（如 面包 → 牛奶）
- **规则强度**：这种关联有多强？是偶然巧合还是真正的模式？

### 6.1.1.2 关联规则挖掘的两步框架

关联规则挖掘可以分解为两个子问题：

1. **频繁项集发现**（Frequent Itemset Mining）：找出所有支持度不低于阈值的项集——这是计算瓶颈；
2. **关联规则生成**（Association Rule Generation）：从频繁项集中提取满足置信度阈值的规则——相对容易。

本节先建立数学框架，§6.2 和 §6.3 分别介绍两种高效的频繁项集发现算法。

---

## 6.1.2 事务数据库与项集

{% block type="definition" title="定义 6.1　事务数据库（Transaction Database）" %}
设 $$\mathcal{I} = \{i_1, i_2, \ldots, i_m\}$$ 为所有可能**项**（item）的有限集合，称为**项集域**（item universe）。

**事务**（transaction）是 $$\mathcal{I}$$ 的一个非空子集 $$T \subseteq \mathcal{I}$$，每个事务有唯一标识 $$\text{tid}$$。

**事务数据库** $$\mathcal{D} = \{T_1, T_2, \ldots, T_n\}$$ 是 $$n$$ 个事务的集合。

**项集**（itemset）是 $$\mathcal{I}$$ 的任意子集 $$X \subseteq \mathcal{I}$$。包含 $$k$$ 个项的项集称为 **$$k$$-项集**。项集域 $$\mathcal{I}$$ 的所有子集构成的**幂集** $$2^{\mathcal{I}}$$ 包含 $$2^m$$ 个项集。
{% endblock %}

以上面的超市数据为例：$$\mathcal{I} = \{\text{面包, 牛奶, 鸡蛋, 啤酒, 尿布}\}$$（$$m = 5$$），$$n = 5$$ 笔交易，$$2^5 = 32$$ 个可能的项集。

---

## 6.1.3 支持度与频繁项集

{% block type="definition" title="定义 6.2　支持度（Support）" %}
项集 $$X$$ 在事务数据库 $$\mathcal{D}$$ 中的**支持度**定义为包含 $$X$$ 的事务比例：

$$
\text{supp}(X) = \frac{|\{T \in \mathcal{D} : X \subseteq T\}|}{|\mathcal{D}|} = \frac{\sigma(X)}{n} \tag{6.1}
$$

其中 $$\sigma(X) = |\{T \in \mathcal{D} : X \subseteq T\}|$$ 为 $$X$$ 的**支持度计数**（support count），$$n = |\mathcal{D}|$$ 为事务总数。

支持度的概率解读：$$\text{supp}(X) = P(X)$$，即从 $$\mathcal{D}$$ 中随机抽取一个事务包含 $$X$$ 的概率。
{% endblock %}

{% block type="definition" title="定义 6.3　频繁项集（Frequent Itemset）" %}
给定**最小支持度阈值** $$\text{minsup} \in (0, 1]$$，若项集 $$X$$ 满足：

$$
\text{supp}(X) \geq \text{minsup} \tag{6.2}
$$

则称 $$X$$ 为**频繁项集**。所有频繁项集的集合记为：

$$
\mathcal{F} = \{X \subseteq \mathcal{I} : \text{supp}(X) \geq \text{minsup}\} \tag{6.3}
$$
{% endblock %}

**手工计算示例**（以 $$\text{minsup} = 0.4$$，即至少出现 2 次为例）：

| 项集 | 支持度计数 $$\sigma$$ | 支持度 | 频繁？ |
|------|------------------|--------|--------|
| {面包} | 4 | 0.80 | 是 |
| {牛奶} | 4 | 0.80 | 是 |
| {鸡蛋} | 4 | 0.80 | 是 |
| {啤酒} | 3 | 0.60 | 是 |
| {尿布} | 1 | 0.20 | 否 |
| {面包, 牛奶} | 3 | 0.60 | 是 |
| {面包, 鸡蛋} | 3 | 0.60 | 是 |
| {牛奶, 鸡蛋} | 3 | 0.60 | 是 |
| {牛奶, 啤酒} | 3 | 0.60 | 是 |
| {面包, 牛奶, 鸡蛋} | 2 | 0.40 | 是 |
| ... | | | |

## 6.1.4 关联规则及其度量

{% block type="definition" title="定义 6.4　关联规则（Association Rule）" %}
**关联规则**是形如 $$X \Rightarrow Y$$ 的蕴含式，其中 $$X, Y \subseteq \mathcal{I}$$，$$X \neq \emptyset$$，$$Y \neq \emptyset$$，且 $$X \cap Y = \emptyset$$。

- $$X$$ 称为规则的**前件**（antecedent / left-hand side）；
- $$Y$$ 称为规则的**后件**（consequent / right-hand side）；
- $$X \cup Y$$ 称为规则涉及的**项集**。

注意：$$X \Rightarrow Y$$ 不是逻辑蕴含（不具有"若 X 则 Y"的必然性），而是**统计关联**。
{% endblock %}

{% block type="definition" title="定义 6.5　关联规则的支持度与置信度" %}
关联规则 $$X \Rightarrow Y$$ 的**支持度**和**置信度**分别为：

$$
\text{supp}(X \Rightarrow Y) = \text{supp}(X \cup Y) = P(X \cup Y) \tag{6.4}
$$

$$
\text{conf}(X \Rightarrow Y) = \frac{\text{supp}(X \cup Y)}{\text{supp}(X)} = P(Y \mid X) \tag{6.5}
$$

- **支持度**度量规则在所有事务中的**普遍性**（representativeness）：规则涉及的项集出现得多频繁；
- **置信度**度量规则的**可靠性**（reliability）：在包含 $$X$$ 的事务中，同时包含 $$Y$$ 的条件概率。

一条**强关联规则**同时满足 $$\text{supp} \geq \text{minsup}$$ 且 $$\text{conf} \geq \text{minconf}$$。
{% endblock %}

### 6.1.4.1 置信度的局限与提升度

仅凭支持度和置信度不足以判断关联的真实性。考虑以下情形：如果 $$Y$$ 本身就是非常常见的项（如"购买牛奶"），即使 $$X$$ 和 $$Y$$ 完全独立，$$\text{conf}(X \Rightarrow Y)$$ 也会很高——因为无论是否购买 $$X$$，大部分顾客都会购买 $$Y$$。

{% block type="definition" title="定义 6.6　提升度（Lift）" %}
关联规则 $$X \Rightarrow Y$$ 的**提升度**定义为：

$$
\text{lift}(X \Rightarrow Y) = \frac{\text{conf}(X \Rightarrow Y)}{\text{supp}(Y)} = \frac{P(X \cup Y)}{P(X) \cdot P(Y)} \tag{6.6}
$$

提升度的解读：
- $$\text{lift} = 1$$：$$X$$ 和 $$Y$$ 统计独立（无关联）；
- $$\text{lift} > 1$$：$$X$$ 的出现**促进**了 $$Y$$ 的出现（正关联）；
- $$\text{lift} < 1$$：$$X$$ 的出现**抑制**了 $$Y$$ 的出现（负关联）。

提升度是对称的：$$\text{lift}(X \Rightarrow Y) = \text{lift}(Y \Rightarrow X)$$。
{% endblock %}

### 6.1.4.2 三种度量的关系

| 度量 | 公式 | 衡量维度 | 取值范围 | 对称性 |
|------|------|---------|---------|--------|
| 支持度 | $$P(X \cup Y)$$ | 普遍性 | $$[0, 1]$$ | 对称 |
| 置信度 | $$P(Y \mid X)$$ | 可靠性 | $$[0, 1]$$ | 不对称 |
| 提升度 | $$P(X \cup Y) / [P(X)P(Y)]$$ | 独立性偏离 | $$(0, +\infty)$$ | 对称 |

**注意**：$$\text{supp}(X \Rightarrow Y) = \text{supp}(Y \Rightarrow X)$$（对称），但 $$\text{conf}(X \Rightarrow Y) \neq \text{conf}(Y \Rightarrow X)$$（不对称）——这意味着"买面包的人常买牛奶"和"买牛奶的人常买面包"是两条**不同**的规则，强度可能不同。

## 6.1.5 频繁项集的反单调性（Apriori 性质）

频繁项集发现的最大挑战是**搜索空间巨大**：$$m$$ 个项的幂集有 $$2^m$$ 个子集。当 $$m = 100$$（超市有 100 种商品）时，$$2^{100} \approx 10^{30}$$——暴力枚举不可行。

Apriori 性质提供了关键的剪枝策略。

{% block type="theorem" title="定理 6.1　频繁项集的反单调性（Apriori 性质 / Downward Closure Property）" %}
若项集 $$X$$ 是频繁的，则 $$X$$ 的**所有非空子集**都是频繁的。等价地：

$$
\forall Y \subseteq X : \text{supp}(Y) \geq \text{supp}(X) \tag{6.7}
$$

**逆否命题**：若项集 $$Y$$ 是非频繁的，则包含 $$Y$$ 的**所有超集**都是非频繁的。

**证明**：

设 $$Y \subseteq X$$。对任意事务 $$T$$，若 $$X \subseteq T$$（$$T$$ 包含 $$X$$），则必有 $$Y \subseteq T$$（因为 $$Y \subseteq X \subseteq T$$）。因此：

$$
\{T \in \mathcal{D} : X \subseteq T\} \subseteq \{T \in \mathcal{D} : Y \subseteq T\} \tag{6.8}
$$

取集合大小并除以 $$n$$：

$$
\sigma(X) \leq \sigma(Y) \implies \text{supp}(X) \leq \text{supp}(Y) \tag{6.9}
$$

若 $$X$$ 频繁（$$\text{supp}(X) \geq \text{minsup}$$），则 $$\text{supp}(Y) \geq \text{supp}(X) \geq \text{minsup}$$，即 $$Y$$ 也频繁。$$\square$$
{% endblock %}

### 6.1.5.1 Apriori 性质的剪枝威力

Apriori 性质允许**逐层**（level-wise）搜索：先找所有频繁 1-项集，再用频繁 1-项集生成候选 2-项集（非频繁 1-项集的所有超集直接剪掉），验证后得到频繁 2-项集，依此类推。

**剪枝示例**：若 {尿布} 是非频繁的（$$\text{supp} = 0.20 < 0.40$$），则包含"尿布"的所有项集——{面包, 尿布}、{牛奶, 尿布}、{面包, 牛奶, 尿布}、……——**全部无需检查**，一次性剪掉 $$2^{m-1}$$ 个候选。

在实际数据中，Apriori 性质通常能将搜索空间缩减数个数量级。这一性质是 Apriori 算法（§6.2）和 FP-Growth 算法（§6.3）的共同理论基础。

## 6.1.6 关联规则的数量分析

{% block type="theorem" title="定理 6.2　关联规则数量的组合爆炸" %}
对一个含 $$m$$ 个项的项集域 $$\mathcal{I}$$，可能的关联规则总数为：

$$
R(m) = \sum_{k=2}^{m} \binom{m}{k} \cdot (2^k - 2) = 3^m - 2^{m+1} + 1 \tag{6.10}
$$

**证明**：

对任意 $$k$$-项集 $$S$$（$$k \geq 2$$），可以将 $$S$$ 分为非空前件 $$X$$ 和非空后件 $$Y = S \setminus X$$，分法数为 $$S$$ 的非空真子集数 $$2^k - 2$$（排除 $$\emptyset$$ 和 $$S$$ 本身）。因此：

$$
R(m) = \sum_{k=2}^{m} \binom{m}{k}(2^k - 2) = \sum_{k=0}^{m} \binom{m}{k} 2^k - 2 \sum_{k=0}^{m} \binom{m}{k} - \left[\binom{m}{0}(2^0-2) + \binom{m}{1}(2^1-2)\right]
$$

利用二项式定理 $$\sum_{k=0}^m \binom{m}{k} 2^k = 3^m$$ 和 $$\sum_{k=0}^m \binom{m}{k} = 2^m$$：

$$
R(m) = 3^m - 2 \cdot 2^m - (-1 + 0) = 3^m - 2^{m+1} + 1 \tag{6.11}
$$

当 $$m = 10$$：$$R(10) = 57002$$ 条规则；当 $$m = 20$$：$$R(20) \approx 3.5 \times 10^9$$——即使中等规模的项集域也会产生天文数字的候选规则。$$\square$$
{% endblock %}

这个结果表明，**即使频繁项集已经找到**，暴力枚举所有可能的规则也不可行。实践中，最小支持度和最小置信度阈值共同将规则数量削减到可处理的范围。

## 6.1.7 从频繁项集到关联规则

给定频繁项集 $$L$$ 和最小置信度 $$\text{minconf}$$，规则生成的过程如下：

```text
算法：关联规则生成(L, minconf)
──────────────────────────────────────────
输入：频繁项集 L（|L| ≥ 2），最小置信度阈值
输出：满足条件的关联规则集合

1. for 每个非空真子集 X ⊂ L:
2.     Y ← L \ X
3.     conf ← supp(L) / supp(X)
4.     if conf ≥ minconf:
5.         输出规则 X ⇒ Y（supp=supp(L), conf=conf）
```

### 6.1.7.1 Python 实现

```python
import numpy as np
from itertools import combinations

def find_frequent_itemsets(transactions, minsup):
    """暴力枚举频繁项集（适用于小数据集演示）"""
    n = len(transactions)
    # 统计所有项
    items = set()
    for t in transactions:
        items.update(t)
    items = sorted(items)
    # 枚举所有项集，计算支持度
    frequent = {}
    for k in range(1, len(items) + 1):
        for combo in combinations(items, k):
            itemset = frozenset(combo)
            count = sum(1 for t in transactions if itemset.issubset(t))
            sup = count / n
            if sup >= minsup:
                frequent[itemset] = sup
    return frequent

def generate_rules(frequent, minconf):
    """从频繁项集生成关联规则"""
    rules = []
    for itemset, sup in frequent.items():
        if len(itemset) < 2:
            continue
        items = list(itemset)
        for k in range(1, len(items)):
            for ante in combinations(items, k):
                antecedent = frozenset(ante)
                consequent = itemset - antecedent
                if antecedent in frequent:
                    conf = sup / frequent[antecedent]
                    lift = sup / (frequent[antecedent] * frequent[consequent])
                    if conf >= minconf:
                        rules.append({
                            'antecedent': antecedent,
                            'consequent': consequent,
                            'support': sup,
                            'confidence': conf,
                            'lift': lift,
                        })
    return rules

# ── 示例运行 ─────────────────────────────────────────
transactions = [
    {'面包', '牛奶', '鸡蛋'},
    {'面包', '牛奶', '啤酒', '尿布'},
    {'牛奶', '鸡蛋', '啤酒'},
    {'面包', '牛奶', '鸡蛋', '啤酒'},
    {'面包', '鸡蛋'},
]
frequent = find_frequent_itemsets(transactions, minsup=0.4)
rules = generate_rules(frequent, minconf=0.6)

print("频繁项集：")
for itemset, sup in sorted(frequent.items(), key=lambda x: (-len(x[0]), -x[1])):
    print(f"  {set(itemset)}: supp={sup:.2f}")

print(f"\n关联规则（minconf≥0.6）：")
for r in sorted(rules, key=lambda x: -x['lift']):
    print(f"  {set(r['antecedent'])} → {set(r['consequent'])}: "
          f"supp={r['support']:.2f}, conf={r['confidence']:.2f}, lift={r['lift']:.2f}")
```

## 案例 6.1：超市购物篮分析

**问题背景**：某连锁超市拥有 1000 笔交易记录（合成数据，模拟真实购物模式），涉及 15 种常见商品。目标是发现频繁购物模式和关联规则，为货架摆放和捆绑促销提供数据支持。

```python
import numpy as np
from itertools import combinations

np.random.seed(42)

# ── 1. 生成模拟购物篮数据 ─────────────────────────────────
items = ['面包', '牛奶', '鸡蛋', '啤酒', '尿布', '可乐', '薯片',
         '酸奶', '水果', '蔬菜', '肉类', '鱼类', '调味品', '零食', '饮用水']

# 定义一些购物模式（概率相关组）
patterns = {
    '早餐组': (['面包', '牛奶', '鸡蛋'], 0.4),
    '零食组': (['啤酒', '薯片', '可乐'], 0.25),
    '健康组': (['水果', '蔬菜', '酸奶'], 0.2),
    '主食组': (['肉类', '蔬菜', '调味品'], 0.3),
}

n_transactions = 1000
transactions = []
for _ in range(n_transactions):
    basket = set()
    # 按模式组添加商品
    for name, (group_items, prob) in patterns.items():
        if np.random.random() < prob:
            # 组中每个商品有较高概率被选中
            for item in group_items:
                if np.random.random() < 0.8:
                    basket.add(item)
    # 添加随机商品
    n_random = np.random.poisson(1.5)
    basket.update(np.random.choice(items, size=min(n_random, 5), replace=False))
    if len(basket) > 0:
        transactions.append(basket)

print(f"事务数：{len(transactions)}")
print(f"平均事务长度：{np.mean([len(t) for t in transactions]):.1f}")
print(f"商品种类：{len(items)}")

# ── 2. 计算 1-项集和 2-项集的支持度 ─────────────────────
n = len(transactions)
item_support = {}
for item in items:
    count = sum(1 for t in transactions if item in t)
    item_support[item] = count / n

print("\n各商品支持度：")
for item, sup in sorted(item_support.items(), key=lambda x: -x[1]):
    print(f"  {item}: {sup:.3f}")

# ── 3. 频繁 2-项集与关联规则 ──────────────────────────────
minsup = 0.08
minconf = 0.5
frequent_2 = {}
for i1, i2 in combinations(items, 2):
    count = sum(1 for t in transactions if {i1, i2}.issubset(t))
    sup = count / n
    if sup >= minsup:
        frequent_2[frozenset({i1, i2})] = sup

print(f"\n频繁 2-项集（minsup≥{minsup}）：{len(frequent_2)} 个")

# 生成规则
rules = []
for pair, sup in frequent_2.items():
    items_pair = list(pair)
    for i in range(2):
        ante = frozenset({items_pair[i]})
        cons = frozenset({items_pair[1-i]})
        conf = sup / item_support[items_pair[i]]
        lift = conf / item_support[items_pair[1-i]]
        if conf >= minconf:
            rules.append((items_pair[i], items_pair[1-i], sup, conf, lift))

print(f"关联规则（minconf≥{minconf}）：{len(rules)} 条")
print("\n前 10 条规则（按提升度降序）：")
rules.sort(key=lambda x: -x[4])
for ante, cons, sup, conf, lift in rules[:10]:
    print(f"  {ante} → {cons}: supp={sup:.3f}, conf={conf:.3f}, lift={lift:.2f}")
```

**业务解读**：

1. **早餐组关联**：面包 → 牛奶（高置信度、高提升度）——建议将面包和牛奶摆放在相邻货架，或推出"早餐套装"捆绑促销；
2. **零食组关联**：啤酒 → 薯片（提升度显著 > 1）——周末促销时可将啤酒和薯片联合打折；
3. **跨组关联**：蔬菜同时出现在"健康组"和"主食组"中——蔬菜是**枢纽商品**，应放在店铺中心位置以引导人流；
4. **高置信度陷阱**：牛奶 → X 的置信度可能偏高（牛奶支持度很高），但提升度接近 1——说明这不是真正的关联，而是牛奶太常见。**必须同时检查提升度**。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 事务数据库（定义 6.1） | $$\mathcal{D} = \{T_1, \ldots, T_n\}$$，$$T_i \subseteq \mathcal{I}$$ |
| 支持度（定义 6.2） | $$\text{supp}(X) = \sigma(X)/n = P(X)$$，度量项集出现频率 |
| 频繁项集（定义 6.3） | $$\text{supp}(X) \geq \text{minsup}$$ |
| 关联规则（定义 6.4） | $$X \Rightarrow Y$$，统计关联而非逻辑蕴含 |
| 置信度（定义 6.5） | $$\text{conf}(X \Rightarrow Y) = P(Y \mid X)$$，度量规则可靠性 |
| 提升度（定义 6.6） | $$\text{lift} = P(X \cup Y)/[P(X)P(Y)]$$，$$>1$$ 为正关联 |
| Apriori 性质（定理 6.1） | $$Y \subseteq X \implies \text{supp}(Y) \geq \text{supp}(X)$$，非频繁项集的超集全部非频繁 |
| 规则数量爆炸（定理 6.2） | $$R(m) = 3^m - 2^{m+1} + 1$$，需要阈值过滤 |
| 两步框架 | 先找频繁项集（计算瓶颈），再生成规则（相对容易） |

**B2.** 对给定频繁项集 $$L = \{A, B, C\}$$，列出所有可能的关联规则（共 6 条）。证明：若 $$\text{conf}(A \Rightarrow BC) \geq \text{minconf}$$，则 $$\text{conf}(AB \Rightarrow C) \geq \text{minconf}$$。（这是关联规则置信度的"反单调性"。）

**B3.** 证明提升度的对称性：$$\text{lift}(X \Rightarrow Y) = \text{lift}(Y \Rightarrow X)$$。进一步证明：$$\text{lift}(X \Rightarrow Y) = 1$$ 当且仅当 $$X$$ 和 $$Y$$ 统计独立。

**B4.** 设事务数据库有 $$n$$ 个事务，项集域大小为 $$m$$。计算暴力枚举所有候选项集所需的支持度计数次数 $$C(m, n)$$。与使用 Apriori 性质（假设只有 $$m/2$$ 个频繁 1-项集，非频繁项集的超集全部剪掉）后的计数次数对比。
