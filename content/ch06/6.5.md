# 6.5 多层次与多维关联规则挖掘

§6.1–6.4 讨论的关联规则都具有相同的形式：项集之间的布尔关联——"购买了 $A$ 的顾客也倾向于购买 $B$"。但在实际应用中，数据往往具有更丰富的结构：

1. **概念层次**（concept hierarchy）：商品之间存在"类别—子类别—具体商品"的层次关系。例如，"2% 牛奶"属于"牛奶"，"牛奶"属于"乳制品"。在不同粒度上挖掘关联规则可能揭示不同层次的业务洞察。
2. **多维属性**：事务除了包含布尔型的"是否购买"信息外，还可能附带数值型属性（如购买数量、价格、顾客年龄等）。

本节将关联规则挖掘从"单层单维"扩展到"多层次多维"，讨论相应的形式化定义、挖掘策略和实现方法。

## 6.5.1 概念层次与多层次关联规则

在超市购物篮分析中，"2% 牛奶→全麦面包"这条规则可能因支持度不足而无法被发现。但如果将商品抽象到更高层次——"牛奶→面包"——支持度可能显著提升。反之，过高的抽象层次可能产生过于笼统的规则（如"食品→饮品"），失去实用价值。

{% block type="definition" title="定义 6.19　概念层次与多层次事务数据库" %}
**概念层次**（concept hierarchy）是项集 $\mathcal{I}$ 上的一棵有根树 $\mathcal{H} = (\mathcal{V}, \mathcal{E})$，其中：

- 叶节点对应原始项（具体商品），内部节点对应抽象概念（类别）；
- 根节点表示最高抽象层（如"所有商品"）；
- 若节点 $v$ 是 $u$ 的祖先，记为 $u \preceq v$，表示 $u$ "是一种" $v$。

给定概念层次 $\mathcal{H}$ 和原始事务数据库 $\mathcal{D}$，**$k$ 层事务数据库** $\mathcal{D}^{(k)}$ 通过将每条事务中的项替换为其在第 $k$ 层的祖先得到：

$$
\mathcal{D}^{(k)} = \{ t^{(k)} : t \in \mathcal{D} \}, \quad t^{(k)} = \{ \text{ancestor}_k(i) : i \in t \} \tag{6.35}
$$

其中 $\text{ancestor}_k(i)$ 是项 $i$ 在层次 $\mathcal{H}$ 的第 $k$ 层祖先。
{% endblock %}

**例 6.5**　超市商品的三层概念层次：

```
Level 0（根）: 所有商品
Level 1（大类）: 乳制品, 烘焙品, 饮料, 零食
Level 2（小类）: 牛奶, 酸奶, 面包, 蛋糕, 果汁, 碳酸饮料, 薯片, 饼干
Level 3（单品）: 2%牛奶, 脱脂牛奶, 全麦面包, 白面包, 橙汁, 可乐, ...
```

在这个层次中，规则"牛奶→面包"（Level 2）比"2% 牛奶→全麦面包"（Level 3）具有更高的支持度，但粒度更粗。

不同层次上的挖掘面临一个关键问题：如何设置最小支持度？

{% block type="definition" title="定义 6.20　多层次支持度策略" %}
**策略一：统一最小支持度**（Uniform minsup）

在所有层次上使用相同的最小支持度阈值 $\sigma_{\min}$。

$$
\mathcal{F}^{(k)} = \{ X \subseteq \mathcal{I}^{(k)} : \text{supp}_{\mathcal{D}^{(k)}}(X) \geq \sigma_{\min} \} \tag{6.36}
$$

**策略二：递减最小支持度**（Reduced minsup）

不同层次使用不同的最小支持度阈值，且高层的阈值高于低层：

$$
\sigma_{\min}^{(1)} > \sigma_{\min}^{(2)} > \cdots > \sigma_{\min}^{(K)} \tag{6.37}
$$

其中 $K$ 是层次深度。
{% endblock %}

统一策略的优势在于简单且可以利用跨层剪枝（见定理 6.11），但其缺点是：若 $\sigma_{\min}$ 设得太高，低层的有趣规则会被遗漏；若设得太低，高层会产生大量冗余规则。递减策略更灵活，但失去了跨层剪枝的能力——高层的非频繁项集在低层不一定也非频繁。

以下定理为统一策略提供了重要的剪枝保证。

{% block type="theorem" title="定理 6.11　层次间支持度的单调性" %}
给定概念层次 $\mathcal{H}$，对任意项 $i$ 和其祖先 $\text{ancestor}_k(i)$：

$$
\text{supp}(i) \leq \text{supp}(\text{ancestor}_k(i)) \tag{6.38}
$$

更一般地，对任意项集 $X$ 和其在第 $k$ 层的抽象 $X^{(k)}$：

$$
\text{supp}(X) \leq \text{supp}(X^{(k)}) \tag{6.39}
$$
{% endblock %}

**证明**　设 $i$ 是 Level $l$（$l > k$）的项，$a = \text{ancestor}_k(i)$ 是其在 Level $k$ 的祖先。每条包含 $i$ 的事务在 $\mathcal{D}^{(k)}$ 中都包含 $a$（因为 $a$ 是 $i$ 的祖先），但包含 $a$ 的事务不一定包含 $i$（$a$ 可能还有其他后代）。因此 $\text{supp}(i) \leq \text{supp}(a)$。对项集的情况，逐项应用此结论即得。$\blacksquare$

定理 6.11 的意义在于：在统一 minsup 策略下，若某个高层概念（如"乳制品"）不频繁，则其所有后代（如"牛奶"、"2% 牛奶"）也不频繁，可以直接剪枝——这正是 Apriori 性质在层次维度上的推广。

## 6.5.2 多层次挖掘算法

多层次挖掘可以采用**自顶向下**（top-down）或**自底向上**（bottom-up）两种策略。

**自顶向下策略**：先在最高层挖掘频繁项集，然后逐层向下细化。只有高层频繁的项才会在低层继续探索。

**算法 6.4　自顶向下多层次挖掘**

```
输入：事务数据库 D，概念层次 H，各层最小支持度 σ(1),...,σ(K)
输出：各层频繁项集 F(1),...,F(K)

1.  对 k = 1, 2, ..., K:
2.      生成第 k 层事务 D(k)（将每项替换为第 k 层祖先）
3.      若 k = 1:
4.          在 D(1) 上运行 Apriori/FP-Growth，得到 F(1)
5.      否则:
6.          候选项集 ← F(k-1) 中频繁项集的所有子项展开
7.          在 D(k) 上统计候选的支持度
8.          F(k) ← {X : supp(X) ≥ σ(k)}
9.  返回 F(1), ..., F(K)
```

**自底向上策略**：先在最底层挖掘频繁项集，然后向上聚合。这种策略保证不会遗漏低层的有趣模式，但计算代价较大。

在实践中，自顶向下策略更常用，因为它利用了定理 6.11 的剪枝能力，避免在低层的巨大搜索空间中做无效探索。

## 6.5.3 多维关联规则

传统关联规则只涉及一个维度（"是否购买"），我们称之为**布尔关联规则**。将维度扩展后，可以发现更丰富的模式。

{% block type="definition" title="定义 6.21　多维关联规则" %}
**多维关联规则**（multidimensional association rule）涉及两个或更多维度（属性/谓词）。其一般形式为：

$$
P_1(X, v_1) \wedge P_2(X, v_2) \wedge \cdots \Rightarrow P_k(X, v_k) \tag{6.40}
$$

其中 $P_i$ 是谓词（属性），$X$ 是变量（如顾客），$v_i$ 是属性值。
{% endblock %}

**例 6.6**　以下是多维关联规则的典型示例：

- **维间规则**（inter-dimension）：$\text{年龄}(X, \text{"20–29"}) \wedge \text{职业}(X, \text{"学生"}) \Rightarrow \text{购买}(X, \text{"笔记本电脑"})$
- **混合规则**（hybrid）：$\text{购买}(X, \text{"打印纸"}) \wedge \text{购买}(X, \text{"墨盒"}) \Rightarrow \text{购买}(X, \text{"打印机"})$（后者是传统单维规则）

多维关联规则的属性可分为两类。

{% block type="definition" title="定义 6.22　布尔维与量化维" %}
- **布尔维**（boolean dimension）：属性值为类别型（如"职业=学生"、"性别=男"），等价于传统的项是否存在。
- **量化维**（quantitative dimension）：属性值为连续型数值（如"年龄=25"、"收入=50000"），需要**离散化**后才能用于关联规则挖掘。
{% endblock %}

量化维的处理是多维关联规则挖掘的核心挑战。直接将每个不同的数值视为一个项会导致严重的稀疏性问题——每个精确值的支持度都极低。

---

## 6.5.4 量化属性的离散化

将连续属性离散化为区间是处理量化维的标准方法。

**策略一：等宽离散化**

将属性值域 $[a, b]$ 均分为 $k$ 个等宽区间：

$$
\text{bin}_j = \left[a + (j-1) \cdot \frac{b-a}{k}, \; a + j \cdot \frac{b-a}{k}\right), \quad j = 1, \ldots, k \tag{6.41}
$$

优点是简单快速，缺点是对倾斜分布效果差——密集区间内的项被过度合并，稀疏区间内几乎没有数据。

**策略二：等频离散化**

每个区间包含大致相同数量的样本。设排序后的属性值为 $v_1 \leq v_2 \leq \cdots \leq v_n$，则第 $j$ 个区间的边界为 $v_{\lfloor (j-1)n/k \rfloor + 1}$ 到 $v_{\lfloor jn/k \rfloor}$。

等频离散化保证了每个区间有足够的支持度，但可能将相近的值分到不同区间。

**策略三：基于聚类的离散化**

对属性值进行 1-D 聚类（如 K-means），将每个簇对应一个区间。这种方法能更好地反映数据的自然分布，但需要额外的聚类步骤。

离散化粒度（区间数 $k$）的选择是一个权衡：

$$
k \uparrow \implies \text{规则更精细，但支持度降低} \tag{6.42}
$$

$$
k \downarrow \implies \text{支持度提升，但规则过于笼统} \tag{6.43}
$$

实践中通常用**交叉验证**或**信息增益**来选择最优的 $k$ 值。

## 6.5.5 算法实现

```python
# ============================================================
# 代码 6.6：多层次关联规则挖掘
# 功能：在带概念层次的购物篮数据上挖掘不同层次的关联规则
# 依赖库：numpy
# ============================================================
import numpy as np
from itertools import combinations
# --- 1. 定义概念层次 ---
hierarchy = {
    # Level 3 (单品) → Level 2 (小类) → Level 1 (大类)
    '2%牛奶':   ('牛奶', '乳制品'),
    '脱脂牛奶': ('牛奶', '乳制品'),
    '酸奶':     ('酸奶', '乳制品'),
    '全麦面包': ('面包', '烘焙品'),
    '白面包':   ('面包', '烘焙品'),
    '蛋糕':     ('蛋糕', '烘焙品'),
    '橙汁':     ('果汁', '饮料'),
    '苹果汁':   ('果汁', '饮料'),
    '可乐':     ('碳酸饮料', '饮料'),
    '薯片':     ('薯片', '零食'),
    '饼干':     ('饼干', '零食'),
}
def lift_to_level(item, level):
    """将单品提升到指定层次 (1=大类, 2=小类, 3=单品)"""
    if level == 3:
        return item
    if item in hierarchy:
        return hierarchy[item][level - 1]
    return item
def transform_transactions(transactions, level):
    """将事务数据库提升到指定层次"""
    return [
        set(lift_to_level(item, level) for item in t)
        for t in transactions
    ]
# --- 2. 简单频繁项集挖掘 ---
def find_frequent(transactions, minsup_ratio):
    n = len(transactions)
    minsup_count = minsup_ratio * n
    # 1-项集
    item_counts = {}
    for t in transactions:
        for item in t:
            item_counts[item] = item_counts.get(item, 0) + 1
    freq_1 = {
        frozenset({item}): c / n
        for item, c in item_counts.items()
        if c >= minsup_count
    }
    all_freq = dict(freq_1)
    freq_items = set(freq_1.keys())
    # 2-项集
    pair_counts = {}
    for t in transactions:
        freq_in_t = [item for item in t
                     if frozenset({item}) in freq_items]
        for a, b in combinations(freq_in_t, 2):
            pair = frozenset({a, b})
            pair_counts[pair] = pair_counts.get(pair, 0) + 1
    for pair, c in pair_counts.items():
        if c >= minsup_count:
            all_freq[pair] = c / n
    return all_freq
# --- 3. 多层次挖掘 ---
np.random.seed(42)
single_items = list(hierarchy.keys())
transactions = []
for _ in range(500):
    basket = set()
    # 模式：牛奶 + 面包
    if np.random.random() < 0.35:
        basket.add(np.random.choice(['2%牛奶', '脱脂牛奶']))
        basket.add(np.random.choice(['全麦面包', '白面包']))
    # 模式：果汁 + 薯片
    if np.random.random() < 0.20:
        basket.add(np.random.choice(['橙汁', '苹果汁']))
        basket.add('薯片')
    # 随机商品
    n_rand = np.random.poisson(2)
    if n_rand > 0:
        basket.update(
            np.random.choice(single_items,
                             size=min(n_rand, 3), replace=False)
        )
    if basket:
        transactions.append(basket)
# 各层次挖掘
for level, minsup in [(1, 0.15), (2, 0.10), (3, 0.05)]:
    trans_level = transform_transactions(transactions, level)
    freq = find_frequent(trans_level, minsup)
    pairs = {k: v for k, v in freq.items() if len(k) == 2}
    print(f"\n=== Level {level} (minsup={minsup}) ===")
    print(f"  频繁 1-项集: {sum(1 for k in freq if len(k)==1)}")
    print(f"  频繁 2-项集: {len(pairs)}")
    for pair in sorted(pairs, key=lambda x: -pairs[x])[:5]:
        print(f"    {set(pair)}: supp={pairs[pair]:.3f}")
```

下面给出多维关联规则（含量化属性离散化）的实现。

```python
# ============================================================
# 代码 6.7：多维关联规则挖掘（含量化属性离散化）
# 功能：将数值属性离散化后发现多维关联规则
# 依赖库：numpy, pandas
# ============================================================
import numpy as np
import pandas as pd
# --- 1. 生成带属性的合成数据 ---
np.random.seed(42)
n = 800
data = pd.DataFrame({
    '年龄': np.random.normal(35, 12, n).clip(18, 70).astype(int),
    '收入': np.random.lognormal(10.5, 0.5, n).astype(int),
    '购买_笔记本': np.random.binomial(1, 0.15, n),
    '购买_手机': np.random.binomial(1, 0.25, n),
    '购买_耳机': np.random.binomial(1, 0.20, n),
})
# 注入关联模式：年轻高收入群体更可能购买笔记本
mask = (data['年龄'] < 30) & (data['收入'] > 40000)
data.loc[mask, '购买_笔记本'] = np.random.binomial(
    1, 0.55, mask.sum()
)
# --- 2. 量化属性离散化 ---
data['年龄段'] = pd.cut(
    data['年龄'],
    bins=[17, 25, 35, 50, 71],
    labels=['18-25', '26-35', '36-50', '51-70']
)
data['收入层'] = pd.qcut(
    data['收入'], q=3,
    labels=['低收入', '中收入', '高收入']
)
# --- 3. 转换为事务 ---
transactions = []
for _, row in data.iterrows():
    t = set()
    t.add(f"年龄={row['年龄段']}")
    t.add(f"收入={row['收入层']}")
    if row['购买_笔记本']:
        t.add('购买=笔记本')
    if row['购买_手机']:
        t.add('购买=手机')
    if row['购买_耳机']:
        t.add('购买=耳机')
    transactions.append(t)
# --- 4. 挖掘多维关联规则 ---
freq = find_frequent(transactions, minsup_ratio=0.03)
# 筛选含"购买"的规则
rules = []
for itemset, supp in freq.items():
    if len(itemset) != 2:
        continue
    items = list(itemset)
    for ante_idx in [0, 1]:
        ante = items[ante_idx]
        cons = items[1 - ante_idx]
        if not cons.startswith('购买='):
            continue
        ante_supp = freq.get(frozenset({ante}), 0)
        if ante_supp == 0:
            continue
        conf = supp / ante_supp
        cons_supp = freq.get(frozenset({cons}), 0)
        lift = conf / cons_supp if cons_supp > 0 else 0
        rules.append({
            'rule': f'{ante} → {cons}',
            'supp': supp, 'conf': conf, 'lift': lift,
        })
rules_df = pd.DataFrame(rules)
top_rules = rules_df.nlargest(10, 'lift')
print("\n=== 多维关联规则 Top-10 (by lift) ===")
for _, r in top_rules.iterrows():
    print(f"  {r['rule']}: supp={r['supp']:.3f}, "
          f"conf={r['conf']:.3f}, lift={r['lift']:.2f}")
```

## 6.5.6 案例分析

{% block type="case" title="案例 6.5　超市商品多层次关联规则挖掘" %}
**背景**：某超市拥有 1000 笔交易记录和三层商品概念层次（大类→小类→单品）。目标是在不同抽象层次上发现关联规则，为不同粒度的营销决策提供支持。

```python
# ============================================================
# 案例 6.5：多层次关联规则系统挖掘
# 数据集：合成超市数据（1000 笔，三层概念层次）
# 依赖库：numpy, pandas
# ============================================================
import numpy as np
import pandas as pd
from itertools import combinations
# --- 1. 扩展概念层次 ---
hierarchy_ext = {
    '2%牛奶': ('牛奶', '乳制品'), '脱脂牛奶': ('牛奶', '乳制品'),
    '全脂牛奶': ('牛奶', '乳制品'), '草莓酸奶': ('酸奶', '乳制品'),
    '原味酸奶': ('酸奶', '乳制品'), '全麦面包': ('面包', '烘焙品'),
    '白面包': ('面包', '烘焙品'), '法棍': ('面包', '烘焙品'),
    '蛋糕': ('蛋糕', '烘焙品'), '橙汁': ('果汁', '饮料'),
    '苹果汁': ('果汁', '饮料'), '可乐': ('碳酸饮料', '饮料'),
    '雪碧': ('碳酸饮料', '饮料'), '原味薯片': ('薯片', '零食'),
    '辣味薯片': ('薯片', '零食'), '饼干': ('饼干', '零食'),
}
np.random.seed(42)
all_items = list(hierarchy_ext.keys())
n_trans = 1000
transactions = []
for _ in range(n_trans):
    basket = set()
    # 关联模式1：牛奶 + 面包（强关联）
    if np.random.random() < 0.35:
        basket.add(np.random.choice(['2%牛奶', '脱脂牛奶', '全脂牛奶']))
        basket.add(np.random.choice(['全麦面包', '白面包', '法棍']))
    # 关联模式2：果汁 + 薯片
    if np.random.random() < 0.20:
        basket.add(np.random.choice(['橙汁', '苹果汁']))
        basket.add(np.random.choice(['原味薯片', '辣味薯片']))
    # 关联模式3：酸奶 + 饼干
    if np.random.random() < 0.15:
        basket.add(np.random.choice(['草莓酸奶', '原味酸奶']))
        basket.add('饼干')
    # 关联模式4：可乐 + 薯片
    if np.random.random() < 0.18:
        basket.add(np.random.choice(['可乐', '雪碧']))
        basket.add(np.random.choice(['原味薯片', '辣味薯片']))
    # 随机商品
    nr = np.random.poisson(1.5)
    if nr > 0:
        basket.update(np.random.choice(all_items, min(nr, 3), replace=False))
    if basket:
        transactions.append(basket)
# --- 2. 各层次挖掘 ---
def lift_item(item, level, hier):
    if level == 3:
        return item
    if item in hier:
        return hier[item][level - 1]
    return item
def trans_level(transactions, level, hier):
    return [set(lift_item(i, level, hier) for i in t) for t in transactions]
print("=" * 60)
for level, minsup in [(1, 0.15), (2, 0.08), (3, 0.04)]:
    tl = trans_level(transactions, level, hierarchy_ext)
    freq = find_frequent(tl, minsup)
    n_1 = sum(1 for k in freq if len(k) == 1)
    pairs = {k: v for k, v in freq.items() if len(k) == 2}
    print(f"\nLevel {level} (minsup={minsup}): "
          f"{n_1} 个频繁单项, {len(pairs)} 个频繁对")
    for pair in sorted(pairs, key=lambda x: -pairs[x])[:5]:
        a, b = list(pair)
        pA = freq.get(frozenset({a}), 0)
        pB = freq.get(frozenset({b}), 0)
        pAB = pairs[pair]
        lift = pAB / (pA * pB) if pA * pB > 0 else 0
        print(f"    {{{a}, {b}}}: "
              f"supp={pAB:.3f}, lift={lift:.2f}")
```

1. **Level 1（大类）**：发现"乳制品→烘焙品"（supp≈0.35, lift≈1.3）等粗粒度规则，适合品类陈列策略。

3. **Level 3（单品）**：发现"2% 牛奶→全麦面包"等细粒度规则，但支持度较低，适合精准促销。

随着层次深入，支持度递减（验证定理 6.11），但规则的具体性和可操作性增强。

## 本节小结

| 关键概念 | 说明 |
|----------|------|
| 概念层次（定义 6.19） | 项的有根树层次结构，叶=单品，内部节点=类别 |
| 支持度策略（定义 6.20） | 统一 minsup（简单可剪枝） vs 递减 minsup（灵活但无法跨层剪枝） |
| 层次支持度单调性（定理 6.11） | 子项支持度 $\leq$ 祖先支持度，跨层剪枝的理论基础 |
| 多维关联规则（定义 6.21） | 涉及两个以上维度/属性的规则形式 |
| 布尔维与量化维（定义 6.22） | 类别型维度 vs 数值型维度，后者需离散化 |
| 离散化策略 | 等宽（简单）、等频（均衡）、聚类（自然分布） |
| 粒度权衡 | 层次/区间越细→规则越精准但支持度越低 |

多层次和多维扩展使关联规则挖掘能够处理更丰富的数据结构，但也带来了新的挑战：层次深度、离散化粒度、维度选择等超参数都需要根据具体业务需求来调整。在下一节（§6.6）中，我们将讨论如何可视化和解释关联规则，帮助业务决策者理解和利用挖掘结果。

**B2.** 设概念层次有 $L$ 层，每个 Level $k$（$k > 1$）的节点有 $b$ 个子节点。在 Level $L$（叶层）有 $m = b^{L-1}$ 个项。证明自顶向下策略在最好情况下（高层大量剪枝）只需检查 $O(b \cdot L)$ 个候选，而自底向上策略需要检查 $O(m^2) = O(b^{2(L-1)})$ 个候选。
