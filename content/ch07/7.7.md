# 7.7 异常检测在欺诈识别中的应用

前面六节分别介绍了异常检测的基本概念（§7.1）和五大类方法——统计方法（§7.2）、距离方法（§7.3）、密度方法（§7.4）、聚类方法（§7.5）和分类方法（§7.6）。每种方法都有其独特的优势和适用场景。本节将这些方法融合到一个完整的工程案例中：**金融欺诈识别**。

金融欺诈是异常检测最重要的应用场景之一。根据美国联邦贸易委员会（FTC）的报告，2023 年全球因欺诈造成的经济损失超过 100 亿美元。在中国，中国人民银行的数据显示，2022 年全国银行卡交易中欺诈交易金额达数十亿元人民币。这些欺诈行为具有典型的异常特征——它们在海量正常交易中占极少数（通常不到 0.1%），且其行为模式与正常交易显著不同。

本节不仅是技术的综合应用，更是从"算法"到"系统"的跨越。真实的欺诈检测系统远不止一个算法——它涉及特征工程、多模型融合、实时推理、反馈闭环和法律合规等多个维度。

## 7.7.1 欺诈检测的问题建模

### 7.7.1.1 欺诈的异常本质

金融欺诈是一个典型的异常检测问题——欺诈交易在整体交易中占极少比例，且其行为模式偏离正常交易。然而，欺诈检测比一般的异常检测更具挑战性：

- **极端不平衡**：欺诈交易通常不到总交易量的 0.1%，远低于一般异常检测场景（1%–5%）；
- **异常非静态**：欺诈者会不断调整策略以规避检测，导致欺诈模式随时间演化；
- **实时性要求**：信用卡交易需要在毫秒级做出"放行/拦截"的决策；
- **标签噪声**：已知的欺诈标签可能不完整——未被发现的欺诈会混在正常交易中。

{% block type="definition" title="定义 7.32　多层欺诈检测框架" %}
一个完整的欺诈检测系统采用**多层防线**架构：

**第一层（规则引擎）**：基于专家规则的即时过滤：

$$
R(\mathbf{x}) = \bigvee_{j=1}^{m} r_j(\mathbf{x}) \tag{7.92}
$$

其中 $$r_j$$ 为业务规则（如"单笔金额超过 50,000 元"、"30 分钟内连续 5 笔交易"），满足任意规则即触发警报。

**第二层（异常评分）**：基于机器学习的异常检测模型，为每笔交易计算综合异常评分：

$$
s_{\text{fusion}}(\mathbf{x}) = g\left(s_1(\mathbf{x}), s_2(\mathbf{x}), \ldots, s_K(\mathbf{x})\right) \tag{7.93}
$$

其中 $$s_k$$ 为第 $$k$$ 个检测模型的评分，$$g$$ 为融合函数。

**第三层（人工审核）**：第二层评分超过阈值但低于"直接拦截"阈值的交易，进入人工审核队列。

完整的判别逻辑为：

$$
\text{decision}(\mathbf{x}) = \begin{cases} \text{拦截} & \text{if } R(\mathbf{x}) = 1 \text{ or } s_{\text{fusion}}(\mathbf{x}) > \tau_{\text{high}} \\ \text{审核} & \text{if } \tau_{\text{low}} < s_{\text{fusion}}(\mathbf{x}) \leq \tau_{\text{high}} \\ \text{放行} & \text{otherwise} \end{cases} \tag{7.94}
$$
{% endblock %}

这一多层架构将**覆盖率**和**精确性**分布在不同层次：规则引擎覆盖已知模式（精确但覆盖率有限），异常检测模型发现未知模式（覆盖率高但可能有误报），人工审核处理不确定案例（精确但成本高）。

### 7.7.1.2 特征工程

原始交易数据（金额、时间、商户类别等）通常不足以直接用于异常检测。有效的特征工程是欺诈检测成功的关键。

{% block type="definition" title="定义 7.33　欺诈检测的交易特征体系" %}
给定交易 $$\mathbf{x}$$（包含金额 $$a$$、时间戳 $$t$$、商户类别 $$m$$、地理位置 $$\ell$$ 等原始字段），构造以下特征类别：

**（1）时间窗口聚合特征**：对持卡人在时间窗口 $$[t - \Delta, t]$$ 内的历史交易进行聚合：

$$
\mathbf{f}_{\text{window}}(\mathbf{x}) = \left[\text{count}_\Delta,\; \text{sum}_\Delta,\; \text{mean}_\Delta,\; \text{std}_\Delta,\; \text{max}_\Delta\right] \tag{7.95}
$$

其中 $$\Delta \in \{1\text{h}, 6\text{h}, 24\text{h}, 7\text{d}, 30\text{d}\}$$，分别捕捉不同时间尺度的行为模式。

**（2）频率与速率特征**：刻画交易的时间间隔模式：

$$
\mathbf{f}_{\text{freq}}(\mathbf{x}) = \left[\Delta t_{\text{prev}},\; \text{rate}_{1\text{h}},\; \text{unique\_merchants}_{24\text{h}},\; \text{unique\_countries}_{24\text{h}}\right] \tag{7.96}
$$

**（3）金额偏离特征**：当前交易金额相对于持卡人历史行为的偏离程度：

$$
z_a = \frac{a - \mu_a^{(\text{hist})}}{\sigma_a^{(\text{hist})}} \tag{7.97}
$$

其中 $$\mu_a^{(\text{hist})}$$ 和 $$\sigma_a^{(\text{hist})}$$ 为持卡人历史交易金额的均值和标准差。
{% endblock %}

特征工程的核心理念是：**将"上下文"信息编码到特征中**。原始金额 $$a = 500$$ 本身不是异常——但如果持卡人历史平均消费仅 50 元，那么 $$z_a = 9.0$$（9 个标准差）就是一个强烈的异常信号。这正是上下文异常（§7.1）在金融场景中的具体体现。

## 7.7.2 类别不平衡的挑战

### 7.7.2.1 不平衡对评估的影响

欺诈检测面临的最大挑战之一是极端的类别不平衡。

{% block type="theorem" title="定理 7.15　极端不平衡下 ROC 曲线的乐观偏差" %}
设数据集中正常样本数为 $$N$$、异常样本数为 $$P$$，不平衡比 $$r = N / P \gg 1$$。对于一个将所有样本标记为正常的"无用分类器"：

$$
\text{FPR} = 0, \quad \text{TPR} = 0 \tag{7.98}
$$

对于一个随机选取 $$k$$ 个样本为异常的分类器：

$$
\text{FPR} = \frac{k \cdot N / (N+P)}{N} = \frac{k}{N+P}, \quad \text{Precision} = \frac{k \cdot P / (N+P)}{k} = \frac{P}{N+P} = \frac{1}{r+1} \tag{7.99}
$$

当 $$r = 999$$（0.1% 异常率）时，随机分类器的精确率仅为 $$0.1\%$$，但其 ROC-AUC 仍为 0.5——**ROC 曲线无法区分"好模型"和"随机猜测"**在极端不平衡下的表现差异。
{% endblock %}

{% hint style="proof" %}
**证明**。ROC 曲线以 FPR 为横轴。FPR $$= \text{FP} / N$$——当 $$N$$ 很大时，即使有大量假阳性，FPR 仍可能很小。具体地，设模型输出 $$k$$ 个报警，其中 $$\text{TP}$$ 个真阳性、$$\text{FP} = k - \text{TP}$$ 个假阳性。则：

$$
\text{FPR} = \frac{k - \text{TP}}{N} \tag{7.100}
$$

当 $$N = 99,900, P = 100$$（$$r = 999$$），即使 $$k = 100$$ 个报警中只有 10 个真阳性（$$\text{Precision} = 10\%$$），$$\text{FPR} = 90 / 99900 = 0.09\%$$——ROC 曲线上的这个点看起来"非常好"（低 FPR，中等 TPR），但实际上 90% 的报警都是误报。

相比之下，PR 曲线直接展示精确率（$$\text{Precision} = \text{TP} / k$$），能真实反映"报警中有多少比例是真正的欺诈"，对业务更有参考价值。 $$\square$$
{% endhint %}

{% hint style="warning" %}
⚠️ **易错点：不要在欺诈检测中只看 ROC-AUC**

ROC-AUC 是最常用的评估指标，但在欺诈检测中容易产生**虚假的安全感**。一个 ROC-AUC 为 0.98 的模型，在 0.1% 异常率下，其精确率可能仅有 5%——意味着每 20 个报警中只有 1 个是真正的欺诈。应使用 **PR-AUC**（精确率-召回率曲线下面积）作为主要指标，辅以 **F1@k**（Top-k 报警中的 F1 值）评估实际运营性能。
{% endhint %}

### 7.7.2.2 应对策略

处理类别不平衡的常用策略包括：

1. **评估指标调整**：使用 PR-AUC、F1@k、Precision@Recall=0.8 等对不平衡友好的指标；
2. **采样策略**：对训练数据进行欠采样（减少正常样本）或过采样（增加异常样本，如 SMOTE），但需注意不可对测试集采样；
3. **代价敏感学习**：为异常样本赋予更高的误分类代价——漏检一笔欺诈的代价远高于误报一笔正常交易；
4. **异常检测方法天然适合**：本章的方法（特别是隔离森林和 One-Class SVM）不需要异常标签，天然适用于极度不平衡的场景。

## 7.7.3 多模型融合

### 7.7.3.1 融合策略

单一的异常检测模型难以覆盖所有类型的欺诈。实际系统通常融合多个模型的评分。

{% block type="definition" title="定义 7.34　基于排名的异常评分融合" %}
给定 $$K$$ 个异常检测模型的评分 $$s_1(\mathbf{x}), \ldots, s_K(\mathbf{x})$$，**排名融合**（Rank Aggregation）策略如下：

**步骤 1（归一化为排名）**：将每个模型的评分转化为排名百分位：

$$
r_k(\mathbf{x}) = \frac{\text{rank}_k(\mathbf{x})}{n}, \quad k = 1, \ldots, K \tag{7.101}
$$

其中 $$\text{rank}_k(\mathbf{x})$$ 为 $$\mathbf{x}$$ 在第 $$k$$ 个模型中的排名（评分越高排名越前）。

**步骤 2（排名聚合）**：取平均排名作为融合评分：

$$
s_{\text{fusion}}(\mathbf{x}) = \frac{1}{K} \sum_{k=1}^{K} r_k(\mathbf{x}) \tag{7.102}
$$

或取最大排名（保守策略）：

$$
s_{\text{fusion}}(\mathbf{x}) = \max_{k=1,\ldots,K} r_k(\mathbf{x}) \tag{7.103}
$$
{% endblock %}

排名融合的优势在于它自动解决了不同模型评分尺度不可比的问题——LOF 的评分通常在 1 附近，隔离森林的评分在 0–1 之间，KNN 距离可能在任意范围——转化为排名后，所有模型的"语言"统一为百分位数。

{% hint style="think" %}
🤔 **思考：为什么融合能提升性能？**

回顾本章各方法的特点：统计方法擅长检测分布尾部的异常，距离方法擅长全局离群点，LOF 擅长局部密度异常，聚类方法能发现群体异常，隔离森林对高维数据高效。这些方法的"盲区"不同——统计方法可能漏检多维联合异常，距离方法在异质密度下表现差，聚类方法在无簇结构数据上失效。融合将各方法的优势互补：某个模型漏检的异常，另一个模型可能给出高排名。排名融合的"平均"操作平滑了各模型的偶然失误，同时保留了一致的强信号。
{% endhint %}

### 7.7.3.2 模型选择与组合

并非所有模型都需要加入融合——相关性高的模型提供冗余信息。实践中，推荐选择来自不同"方法族"的模型：

| 方法族 | 推荐模型 | 独特贡献 |
|--------|---------|---------|
| 统计 | Z-score / 马氏距离 | 分布尾部异常 |
| 距离 | KNN（$$k = 20$$） | 全局离群点 |
| 密度 | LOF（$$k = 20$$） | 局部密度异常 |
| 聚类 | K-means + CBLOF | 群体异常 |
| 分类 | 隔离森林 | 高维孤立点 |

## 7.7.4 实战：信用卡欺诈检测

{% block type="case" title="案例 7.7　信用卡欺诈检测端到端实战" %}

### 业务背景

某银行的信用卡中心每天处理数百万笔交易，其中欺诈交易约占 0.17%。风控团队需要一个自动化系统，在交易发生后实时判断是否为欺诈。本案例使用公开的 **Kaggle Credit Card Fraud Detection** 数据集（经 PCA 降维后的 28 个特征 + 交易金额 + 时间），综合运用本章五种异常检测方法。

### 数据集说明

- 284,807 笔交易，其中 492 笔欺诈（0.17%）
- 特征 V1–V28（PCA 降维后），Amount（金额），Time（时间）
- 由于数据集较大，代码中使用下采样以加速演示

### 完整代码

```python
# ============================================================
# 代码 7.7：信用卡欺诈检测——五种方法融合
# 数据集：合成模拟 Kaggle Credit Card Fraud（30维，10000样本）
# 依赖库：numpy, sklearn, matplotlib
# ============================================================
# --- 1. 生成模拟信用卡交易数据 ---
import numpy as np
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import (roc_auc_score, precision_recall_curve,
                              auc, f1_score)
from sklearn.neighbors import NearestNeighbors, LocalOutlierFactor
from sklearn.ensemble import IsolationForest
from sklearn.svm import OneClassSVM
from sklearn.cluster import KMeans
from scipy.stats import rankdata
np.random.seed(42)
n_normal, n_fraud = 9800, 200
d = 28
# 正常交易：多模态分布（模拟不同消费习惯）
centers = [np.random.randn(d) * 0.5 for _ in range(5)]
X_normal = np.vstack([
    np.random.randn(n_normal // 5, d) * 0.8 + c for c in centers
])
# 欺诈交易：三种模式
# 模式A：极端值（大额欺诈）
X_fraud_a = np.random.randn(80, d) * 0.8 + centers[0]
X_fraud_a[:, :5] += np.random.uniform(3, 6, (80, 5))
# 模式B：分布偏移（盗刷在异常地点）
X_fraud_b = np.random.randn(60, d) * 1.2 + np.random.randn(d) * 3
# 模式C：微量异常（试探性小额交易）
X_fraud_c = np.random.randn(60, d) * 0.3 + centers[2]
X_fraud_c[:, 0] += 2.5
X_fraud = np.vstack([X_fraud_a, X_fraud_b, X_fraud_c])
X = np.vstack([X_normal, X_fraud])
y_true = np.array([0] * n_normal + [1] * n_fraud)
# 添加金额和时间特征
amounts = np.abs(np.random.lognormal(3, 1.5, len(X)))
amounts[n_normal:n_normal+80] *= 5  # 大额欺诈
times = np.sort(np.random.uniform(0, 172800, len(X)))
X = np.column_stack([X, amounts, times])
# --- 2. 标准化 ---
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)
# --- 3. 五种异常检测方法 ---
# 方法1：Z-score（统计方法）
z_scores = np.max(np.abs(X_scaled), axis=1)
# 方法2：KNN 距离（距离方法）
knn = NearestNeighbors(n_neighbors=20)
knn.fit(X_scaled)
knn_dist, _ = knn.kneighbors(X_scaled)
knn_scores = knn_dist[:, -1]
# 方法3：LOF（密度方法）
lof = LocalOutlierFactor(n_neighbors=20, novelty=False)
lof_labels = lof.fit_predict(X_scaled)
lof_scores = -lof.negative_outlier_factor_
# 方法4：K-means 距离（聚类方法）
kmeans = KMeans(n_clusters=5, random_state=42, n_init=10)
kmeans.fit(X_scaled)
km_dist = np.min(
    np.linalg.norm(X_scaled[:, np.newaxis] - kmeans.cluster_centers_, axis=2),
    axis=1
)
# 方法5：隔离森林（分类方法）
iforest = IsolationForest(n_estimators=100, max_samples=256, random_state=42)
iforest.fit(X_scaled)
if_scores = -iforest.decision_function(X_scaled)
# --- 4. 排名融合 ---
all_scores = {
    'Z-score': z_scores,
    'KNN': knn_scores,
    'LOF': lof_scores,
    'K-means': km_dist,
    'iForest': if_scores,
}
ranks = {}
for name, scores in all_scores.items():
    ranks[name] = rankdata(scores) / len(scores)
rank_matrix = np.column_stack(list(ranks.values()))
fusion_avg = rank_matrix.mean(axis=1)
fusion_max = rank_matrix.max(axis=1)
# --- 5. 评估各方法 ---
print("=" * 55)
print(f"{'方法':<15} {'ROC-AUC':>10} {'PR-AUC':>10} {'F1@200':>10}")
print("-" * 55)
for name, scores in all_scores.items():
    roc = roc_auc_score(y_true, scores)
    prec, rec, _ = precision_recall_curve(y_true, scores)
    pr_auc = auc(rec, prec)
    top200 = np.argsort(scores)[-200:]
    y_pred_200 = np.zeros(len(y_true))
    y_pred_200[top200] = 1
    f1 = f1_score(y_true, y_pred_200)
    print(f"{name:<15} {roc:>10.4f} {pr_auc:>10.4f} {f1:>10.4f}")
# 融合结果
for fname, fscores in [('融合(平均)', fusion_avg), ('融合(最大)', fusion_max)]:
    roc = roc_auc_score(y_true, fscores)
    prec, rec, _ = precision_recall_curve(y_true, fscores)
    pr_auc = auc(rec, prec)
    top200 = np.argsort(fscores)[-200:]
    y_pred_200 = np.zeros(len(y_true))
    y_pred_200[top200] = 1
    f1 = f1_score(y_true, y_pred_200)
    print(f"{fname:<15} {roc:>10.4f} {pr_auc:>10.4f} {f1:>10.4f}")
print("=" * 55)
# --- 6. 按欺诈类型分析 ---
print("\n按欺诈类型分析（融合平均排名 Top-200 报警）:")
top200_fusion = set(np.argsort(fusion_avg)[-200:])
type_info = [
    ('大额欺诈(A)', range(n_normal, n_normal + 80)),
    ('分布偏移(B)', range(n_normal + 80, n_normal + 140)),
    ('微量异常(C)', range(n_normal + 140, n_normal + 200)),
]
for name, idx_range in type_info:
    detected = sum(1 for i in idx_range if i in top200_fusion)
    total = len(idx_range)
    print(f"  {name}: {detected}/{total} ({detected/total*100:.0f}%)")
```

### 结果分析

1. **单一方法的表现差异**：
   - **隔离森林** 和 **KNN** 在 ROC-AUC 上表现最优——它们对全局离群点（大额欺诈、分布偏移）的检测能力强；
   - **LOF** 对微量异常（模式 C）的检测优于其他方法——因为这些交易在局部密度上偏低；
   - **Z-score** 对大额欺诈（极端值）非常敏感，但对多维联合异常的检测能力有限；
   - **K-means** 能有效检测偏离簇心的交易，但对簇内的微妙异常不敏感。

2. **融合的优势**：排名融合（特别是平均融合）的 PR-AUC 高于所有单一方法——它综合了各方法的优势，弥补了各方法的盲区。平均融合在整体性能上优于最大融合，因为后者倾向于过度报警。

3. **按欺诈类型的检出率**：大额欺诈（模式 A）和分布偏移欺诈（模式 B）的检出率接近 100%，微量异常欺诈（模式 C）的检出率较低——这符合预期，因为微量试探性交易与正常交易的差异最小。
{% endblock %}

## 7.7.5 工程部署挑战

{% hint style="tip" %}
🔧 **工程实践：从实验到生产的关键挑战**

将异常检测模型部署到真实的欺诈检测系统中，需要解决以下工程挑战：

1. **延迟与吞吐量**：信用卡交易要求毫秒级响应。隔离森林的推理速度（$$O(t \log \psi)$$）远优于 LOF（$$O(n)$$）和 KNN（$$O(n)$$）——后两者的推理需要遍历所有训练数据。实际部署中通常只选择推理速度满足要求的模型。

2. **模型更新**：欺诈模式持续演化（"概念漂移"），模型需要定期重训练。隔离森林的重训练开销极低（$$O(t\psi\log\psi)$$），适合每日或每周更新。One-Class SVM 的重训练开销较高（$$O(n^2)$$），通常使用增量学习策略。

3. **反馈闭环**：人工审核的结果（"确认欺诈"或"误报"）应反馈给模型，形成持续学习循环。这将纯无监督的异常检测逐步过渡为半监督方法。

4. **对抗攻击**：高级欺诈者会研究检测系统的行为，通过"试探"交易了解系统阈值，然后设计"刚好不触发报警"的欺诈策略。应对方法包括：随机化阈值、定期更换模型、对抗训练。

5. **合规与可解释性**：金融监管要求模型决策可解释。隔离森林可通过路径分析给出"哪些特征导致该交易被标记为异常"；One-Class SVM 的核空间决策较难解释——这在受监管行业是一个实际限制。
{% endhint %}

{% hint style="info" %}
🚀 **前沿视野：基于图神经网络的欺诈检测**

近年来，图神经网络（GNN）在欺诈检测领域取得了显著进展。传统方法将每笔交易视为独立样本，忽略了交易之间的关联——但欺诈往往涉及多个账户的协同操作（如洗钱网络、虚假商户链）。

**图构建**：将持卡人、商户和交易构建为异构图——持卡人节点通过"共同商户"或"资金流转"连接，交易节点连接到对应的持卡人和商户。

**GNN 检测**：GraphSAGE（Hamilton 等，2017）和 GAT（Veličković 等，2018）等模型可以聚合邻域信息，自动学习"社交异常"——一个持卡人如果与多个已知欺诈账户有资金往来，即使其单笔交易看起来正常，图模型也能捕捉到这种关联异常。

**标杆成果**：Weber 等人（2019）在 Elliptic 比特币数据集上证明，GNN 方法的 F1 值比传统异常检测方法高出 10–15 个百分点。Liu 等人（2021）提出的 CARE-GNN 显式建模了欺诈者的伪装行为（"交朋友以看起来正常"），进一步提升了检测精度。
{% endhint %}

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 多层防线（定义 7.32） | 规则引擎 → 异常评分模型 → 人工审核的三层架构 |
| 特征工程（定义 7.33） | 时间窗口聚合、频率/速率、金额偏离——将上下文编码为特征 |
| 不平衡偏差（定理 7.15） | 极端不平衡下 ROC 曲线过于乐观，应使用 PR-AUC |
| 排名融合（定义 7.34） | 各模型评分转排名百分位后取平均/最大，解决尺度不可比问题 |
| 融合优势 | 不同方法的盲区互补——融合 PR-AUC 优于任何单一方法 |
| 工程挑战 | 延迟、模型更新、反馈闭环、对抗攻击、合规可解释性 |

本节以欺诈识别为载体，完成了从理论到实践的完整闭环。异常检测不是一个"选择最佳算法"的问题，而是一个"构建检测系统"的工程挑战——正确的特征工程、合理的模型选择与融合、恰当的评估指标、以及对业务需求的深刻理解，缺一不可。

回顾整章内容：从异常的形式化定义（§7.1）出发，我们依次建立了统计（§7.2）、距离（§7.3）、密度（§7.4）、聚类（§7.5）和分类（§7.6）五大方法体系，最终在欺诈识别场景（本节）中将它们融合为一个完整的检测系统。这一从"基本概念→单一方法→方法融合→系统构建"的知识递进结构，反映了数据挖掘从理论到实践的一般路径。

### B 级（数学推导）

**B1.** 证明：在均匀随机分类器下，PR-AUC 等于正类比例 $$P / (N + P)$$，而 ROC-AUC 等于 0.5。讨论当 $$P \ll N$$ 时两者的区别。

**B2.** 推导排名融合的一个统计性质：当 $$K$$ 个模型的排名相互独立且各模型对异常点的排名期望为 $$\mu_r$$（$$\mu_r > 0.5$$）时，平均排名融合的期望值为 $$\mu_r$$，但方差为 $$\sigma_r^2 / K$$。解释为什么模型越多，融合评分越稳定。

**B3.** 设 $$K$$ 个模型中有 $$K_1$$ 个擅长检测 A 类异常、$$K_2$$ 个擅长检测 B 类异常（$$K_1 + K_2 = K$$）。分析"平均融合"和"最大融合"在检测 A 类和 B 类异常上的差异。

### C 级（数值计算）

**C1.** 一个数据集有 10,000 笔正常交易和 10 笔欺诈交易。某模型输出 50 个报警，其中 8 个是真正的欺诈。计算 Precision、Recall、F1、FPR。在 ROC 空间和 PR 空间中分别标出这个模型的位置。

**C2.** 三个模型对 5 个数据点的异常评分为：模型 A: [0.1, 0.3, 0.9, 0.2, 0.8]；模型 B: [10, 30, 50, 20, 45]；模型 C: [1.01, 1.05, 1.50, 1.02, 1.40]。计算排名融合（平均）后的综合评分，确认排名融合消除了尺度差异。

**C3.** 计算 C2 中"最大融合"的结果，并与"平均融合"对比。哪种策略给出的排名差异更大？
