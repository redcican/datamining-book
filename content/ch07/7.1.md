# 7.1 异常检测的基本概念

**异常检测**（Anomaly Detection）是数据挖掘中最具实际价值却最具挑战性的任务之一。与分类、聚类等"正面学习"不同，异常检测关注的是**极少数偏离正常模式的数据点**——它们可能是信用卡欺诈交易、网络入侵攻击、设备故障前兆，也可能是基因突变或天文新发现。

Hawkins（1980）给出了异常的经典定义："**异常是一个观测值，它与其他观测值差异如此之大，以至于令人怀疑它是由不同机制产生的。**"这一直觉性描述揭示了异常检测的核心困难：什么是"差异如此之大"？如何在缺乏标签的情况下量化这种差异？

异常检测的研究可追溯至统计学中的离群值检验。**萌芽期（1960–1980年代）**，Grubbs（1969）提出了正态分布下的离群值检验，Beckman 和 Cook（1983）系统梳理了单变量离群值判别准则。**形成期（1990–2000年代）**，随着数据挖掘兴起，Knorr 和 Ng（1998）提出基于距离的异常定义，Breunig 等人（2000）提出局部离群因子（LOF），将异常检测从全局视角拓展到局部密度视角。**成熟期（2010年代至今）**，隔离森林（Liu et al., 2008/2012）和深度自编码器等方法将异常检测推向高维、大规模数据场景，成为工业界反欺诈、设备预测性维护、网络安全的核心技术。

本节建立异常检测的**完整数学框架**，为后续统计方法（§7.2）、基于距离的方法（§7.3）、基于密度的方法（§7.4）、基于聚类的方法（§7.5）和基于分类的方法（§7.6）奠定理论基础。

## 7.1.1 什么是异常

### 7.1.1.1 异常与噪声的区别

在第二章（§2.7）中，我们曾初步讨论了数据预处理阶段的异常值检测与处理。那里的目标是**清除噪声**，使后续的分类、回归模型不受干扰。本章的视角截然不同：异常本身**就是分析目标**——我们不是要去除它们，而是要**发现**它们。

这一视角转变反映了一个深刻的认识论区别：**噪声**（noise）是随机误差，不包含有用信息，应当过滤；而**异常**（anomaly）是真实但罕见的模式，包含关键业务信息（如欺诈信号、设备故障前兆），应当识别并响应。以下定义将这一区别形式化。

{% block type="definition" title="定义 7.1　异常（Anomaly / Outlier）" %}
设数据集 $$\mathcal{D} = \{\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_n\}$$，$$\mathbf{x}_i \in \mathbb{R}^d$$，由某未知概率分布 $$p(\mathbf{x})$$ 生成。

**异常**是指那些与数据集中大部分观测值存在显著偏离的数据点，即：

$$
\mathcal{O} = \{\mathbf{x}_i \in \mathcal{D} : p(\mathbf{x}_i) < \tau\} \tag{7.1}
$$

其中 $$\tau > 0$$ 为概率密度阈值。等价地，若将 $$\mathcal{D}$$ 视为从混合分布 $$p(\mathbf{x}) = (1-\pi)\,p_0(\mathbf{x}) + \pi\,p_1(\mathbf{x})$$ 中采样，其中 $$p_0$$ 为正常分布，$$p_1$$ 为异常分布，$$\pi \ll 1$$ 为异常先验概率，则异常是由 $$p_1$$ 生成的样本。

**异常**与**噪声**的根本区别在于：
- **噪声**：$$\mathbf{x}_i = \mathbf{x}_i^* + \boldsymbol{\epsilon}_i$$，其中 $$\mathbf{x}_i^*$$ 为真实值，$$\boldsymbol{\epsilon}_i$$ 为随机扰动——噪声叠加在正常数据之上；
- **异常**：$$\mathbf{x}_i \sim p_1(\mathbf{x})$$——异常由完全不同的生成机制产生。
{% endblock %}

定义 7.1 的概率视角揭示了异常检测的核心假设：**正常数据聚集在高概率密度区域，异常数据落在低概率密度区域**。后续章节的各种方法本质上都是在用不同方式估计或近似这一概率密度。

{% hint style="caution" %}
⚠️ **异常 ≠ 离群值 ≠ 噪声**

虽然 "anomaly" 和 "outlier" 在文献中常被混用，但严格来说：**离群值**是统计概念（偏离中心趋势的极端值），**异常**是领域概念（有实际意义的罕见模式）。一个离群值可能只是测量噪声，而一个异常可能在数值上并不极端——例如，信用卡欺诈交易的金额可能完全正常，但交易时间和地点的组合构成异常。
{% endhint %}

### 7.1.1.2 三种异常类型

并非所有异常都是"数值极端"的孤立点。Chandola 等人（2009）将异常划分为三类，每类对应不同的检测策略。

{% block type="definition" title="定义 7.2　异常的三种类型" %}
根据异常的结构特征，可分为以下三类：

**（1）点异常（Point Anomaly）**：单个数据点相对于整体数据分布显著偏离。形式化地，对于数据点 $$\mathbf{x}_i$$，若其异常评分 $$s(\mathbf{x}_i)$$ 超过阈值：

$$
s(\mathbf{x}_i) > \tau \tag{7.2}
$$

则 $$\mathbf{x}_i$$ 为点异常。这是最基本、最常见的异常类型。

**（2）上下文异常（Contextual Anomaly）**：数据点在特定上下文中异常，但在其他上下文中可能正常。形式化地，数据点 $$\mathbf{x}_i$$ 包含**上下文属性** $$\mathbf{c}_i$$（如时间、位置）和**行为属性** $$\mathbf{b}_i$$（如温度、交易金额），若：

$$
p(\mathbf{b}_i \mid \mathbf{c}_i) < \tau \tag{7.3}
$$

则 $$\mathbf{x}_i$$ 在上下文 $$\mathbf{c}_i$$ 下为异常。例如，30°C 的气温在夏天正常，在冬天则为上下文异常。

**（3）集体异常（Collective Anomaly）**：一组数据点作为整体构成异常，但其中每个单独的数据点可能是正常的。形式化地，设子序列 $$\mathcal{S} = \{\mathbf{x}_{t_1}, \mathbf{x}_{t_2}, \ldots, \mathbf{x}_{t_k}\} \subset \mathcal{D}$$，若：

$$
p(\mathcal{S}) < \tau \quad \text{但} \quad \forall\, \mathbf{x}_i \in \mathcal{S}: p(\mathbf{x}_i) \geq \tau \tag{7.4}
$$

则 $$\mathcal{S}$$ 为集体异常。例如，心电图中单个心跳正常，但一段异常节律的组合可能指示心律失常。
{% endblock %}

三种异常类型的直观理解可参见图 7.1.1：点异常是"空间中的孤立点"，上下文异常是"在错误时间出现的正确值"，集体异常是"一群正常点的异常组合"。

{% hint style="think" %}
🤔 **思考：异常类型如何影响方法选择？**

对于点异常，基于距离（§7.3）或密度（§7.4）的方法通常有效。但上下文异常需要建模条件分布 $$p(\mathbf{b} \mid \mathbf{c})$$，这要求方法能区分上下文属性和行为属性——统计方法（§7.2）在此有天然优势。集体异常则需要捕捉序列或子图的**联合模式**，单点方法无能为力，通常需要时序或图结构的专用技术。选择检测方法前，首先判断你面对的是哪种异常类型。
{% endhint %}

## 7.1.2 异常检测的数学框架

### 7.1.2.1 评分函数范式

绝大多数异常检测方法可以统一为一个两步范式：首先对每个数据点计算**异常评分**（越高越异常），然后根据**阈值**将点划分为正常或异常。以下定义将此范式形式化。

{% block type="definition" title="定义 7.3　异常评分函数（Anomaly Scoring Function）" %}
**异常评分函数**是一个映射 $$s: \mathbb{R}^d \to \mathbb{R}$$，对数据集 $$\mathcal{D}$$ 中的每个点 $$\mathbf{x}_i$$ 赋予一个实数值 $$s(\mathbf{x}_i)$$，使得：

$$
s(\mathbf{x}_i) \propto -\log p(\mathbf{x}_i) \tag{7.5}
$$

即异常评分与概率密度负相关——密度越低的点，评分越高，越可能是异常。

不同的异常检测方法对应不同的评分函数：
- **统计方法**（§7.2）：$$s(\mathbf{x}) = |\mathbf{x} - \boldsymbol{\mu}|/\sigma$$（标准化偏差）或马氏距离；
- **距离方法**（§7.3）：$$s(\mathbf{x}) = d_k(\mathbf{x})$$（到第 $$k$$ 近邻的距离）；
- **密度方法**（§7.4）：$$s(\mathbf{x}) = \text{LOF}(\mathbf{x})$$（局部离群因子）；
- **隔离方法**：$$s(\mathbf{x}) = 2^{-E[h(\mathbf{x})]/c(n)}$$（隔离森林的归一化路径长度）。
{% endblock %}

评分函数的统一视角揭示了一个重要事实：各种异常检测方法的**本质差异在于如何估计概率密度**（或其代理量），而非在于最终的决策逻辑。有了评分函数，下一步是确定决策边界。

{% block type="definition" title="定义 7.4　阈值决策与检测规则" %}
给定异常评分函数 $$s$$ 和阈值 $$\tau \in \mathbb{R}$$，**异常检测规则**定义为：

$$
\hat{y}_i = \begin{cases}
1 \quad (\text{异常}), & \text{if } s(\mathbf{x}_i) > \tau \\
0 \quad (\text{正常}), & \text{if } s(\mathbf{x}_i) \leq \tau
\end{cases} \tag{7.6}
$$

阈值 $$\tau$$ 的选择直接控制**检出率**（recall）和**误报率**（false positive rate）之间的权衡：
- $$\tau$$ 过低：大量正常点被误判为异常（高误报率）；
- $$\tau$$ 过高：真正的异常被遗漏（低检出率）。

最优阈值 $$\tau^*$$ 取决于具体应用的损失结构。在 Neyman–Pearson 框架下：

$$
\tau^* = \arg\max_{\tau}\; \text{TPR}(\tau) \quad \text{s.t.} \quad \text{FPR}(\tau) \leq \alpha \tag{7.7}
$$

其中 $$\text{TPR}(\tau) = P(\hat{y}=1 \mid y=1)$$ 为真正例率，$$\text{FPR}(\tau) = P(\hat{y}=1 \mid y=0)$$ 为假正例率，$$\alpha$$ 为可接受的最大误报率。
{% endblock %}

评分-阈值框架的优势在于：它将**模型构建**（设计评分函数 $$s$$）与**决策优化**（选择阈值 $$\tau$$）解耦。这意味着同一个评分函数可以根据不同业务需求调整阈值——在医疗诊断中降低 $$\tau$$（宁可误报、不漏检），在垃圾邮件过滤中提高 $$\tau$$（避免误杀正常邮件）。

### 7.1.2.2 异常检测的三种问题设定

根据标签信息的可用程度，异常检测问题可分为三种设定，这对方法选择有决定性影响。

| 问题设定 | 标签情况 | 典型方法 | 适用场景 |
|---------|---------|---------|---------|
| **有监督**（Supervised） | 正常与异常标签均完整 | 分类器（SVM、随机森林） | 已知欺诈模式 |
| **半监督**（Semi-supervised） | 仅正常数据有标签 | 单类 SVM、自编码器 | 正常行为已知，异常未知 |
| **无监督**（Unsupervised） | 无任何标签 | LOF、iForest、统计检验 | 纯探索性分析 |

在实践中，**无监督设定最为常见**——因为异常本身就是罕见且多变的，收集足够多的异常标签既昂贵又不现实。本章主要讨论无监督方法（§7.2–7.5），§7.6 简要介绍基于分类的半监督/有监督方法。

## 7.1.3 评估指标与类不平衡挑战

### 7.1.3.1 混淆矩阵与基本指标

异常检测是一个典型的**类不平衡**问题：异常类（正类）的比例 $$\pi$$ 通常远小于 1%。这导致传统分类评估指标（如准确率）产生严重误导。

{% block type="definition" title="定义 7.5　异常检测的评估指标" %}
给定真实标签 $$y_i \in \{0, 1\}$$（0=正常，1=异常）和预测标签 $$\hat{y}_i \in \{0, 1\}$$，定义：

|  | 预测正常 ($$\hat{y}=0$$) | 预测异常 ($$\hat{y}=1$$) |
|--|----------------------|----------------------|
| **实际正常** ($$y=0$$) | TN（真负例） | FP（假正例） |
| **实际异常** ($$y=1$$) | FN（假负例） | TP（真正例） |

**准确率**（Accuracy）：

$$
\text{Acc} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}} \tag{7.8}
$$

**精确率**（Precision）和**召回率**（Recall）：

$$
\text{Prec} = \frac{\text{TP}}{\text{TP} + \text{FP}}, \quad \text{Rec} = \frac{\text{TP}}{\text{TP} + \text{FN}} \tag{7.9}
$$

**F1 分数**：精确率与召回率的调和均值：

$$
F_1 = \frac{2 \cdot \text{Prec} \cdot \text{Rec}}{\text{Prec} + \text{Rec}} = \frac{2\,\text{TP}}{2\,\text{TP} + \text{FP} + \text{FN}} \tag{7.10}
$$
{% endblock %}

这些指标中，准确率在异常检测中几乎毫无意义。以下定理严格说明了原因。

### 7.1.3.2 准确率失效定理

{% block type="theorem" title="定理 7.1　类不平衡下准确率的失效" %}
设异常比例为 $$\pi = n_1 / n$$（其中 $$n_1 = |\{i: y_i = 1\}|$$ 为异常点数），$$\pi \ll 1$$。一个**全部预测为正常**的"平凡分类器"（$$\hat{y}_i \equiv 0$$）的准确率为：

$$
\text{Acc}_\text{trivial} = 1 - \pi \tag{7.11}
$$

当 $$\pi = 0.01$$（1% 异常率）时，$$\text{Acc}_\text{trivial} = 99\%$$。因此，任何实际分类器只要准确率低于 $$99\%$$ 就"不如什么都不做"。这说明**准确率在高度不平衡的异常检测问题中是一个几乎无信息量的指标**。
{% endblock %}

{% hint style="proof" %}
**证明** 当 $$\hat{y}_i \equiv 0$$ 时，$$\text{TP} = 0$$，$$\text{FP} = 0$$，$$\text{FN} = n_1$$，$$\text{TN} = n_0 = n - n_1$$。代入准确率公式：

$$
\text{Acc}_\text{trivial} = \frac{0 + (n - n_1)}{n} = 1 - \frac{n_1}{n} = 1 - \pi
$$

当 $$\pi = 0.01$$ 时，$$\text{Acc}_\text{trivial} = 0.99$$。$$\blacksquare$$
{% endhint %}

定理 7.1 揭示了一个实践中极为重要的教训：**永远不要用准确率评估异常检测系统**。替代方案是使用关注少数类表现的指标——精确率、召回率、F1 分数，以及下面介绍的 ROC 和 PR 曲线。

{% hint style="caution" %}
⚠️ **异常检测中的"高准确率陷阱"**

初学者常犯的错误是报告异常检测器的准确率为 99%，并以此宣称模型"非常好"。但如果数据中只有 1% 的异常，99% 的准确率意味着模型可能**一个异常都没检测到**——它只是学会了"永远说正常"。在审查异常检测结果时，务必首先检查 Recall：如果 Recall 为 0，模型毫无价值。
{% endhint %}

### 7.1.3.3 ROC 曲线与 PR 曲线

评分函数 $$s$$ 不直接给出硬分类决策，而是产生一个**排序**（ranking）。ROC 和 PR 曲线提供了与阈值无关的评估视角。

{% block type="definition" title="定义 7.6　ROC 曲线与 AUC" %}
**ROC 曲线**（Receiver Operating Characteristic）以假正例率 FPR 为横轴、真正例率 TPR 为纵轴，绘制不同阈值 $$\tau$$ 下 $$(\text{FPR}(\tau), \text{TPR}(\tau))$$ 的轨迹：

$$
\text{TPR}(\tau) = \frac{\text{TP}(\tau)}{\text{TP}(\tau) + \text{FN}(\tau)}, \quad \text{FPR}(\tau) = \frac{\text{FP}(\tau)}{\text{FP}(\tau) + \text{TN}(\tau)} \tag{7.12}
$$

ROC 曲线下面积（AUC）衡量分类器在所有阈值下的综合表现：

$$
\text{AUC} = \int_0^1 \text{TPR}(\text{FPR})\, d\text{FPR} = P(s(\mathbf{x}^+) > s(\mathbf{x}^-)) \tag{7.13}
$$

其中 $$\mathbf{x}^+$$ 为随机异常样本，$$\mathbf{x}^-$$ 为随机正常样本。AUC 等于**随机抽取一个正样本和一个负样本，分类器正确排序的概率**。
{% endblock %}

AUC 的概率解读（式 7.13）是评估异常检测器排序质量的直觉基础：一个完美的检测器对所有异常点的评分都高于所有正常点，$$\text{AUC} = 1$$；随机猜测的 $$\text{AUC} = 0.5$$。然而，ROC 曲线在类不平衡场景下可能过于乐观，以下定理给出解释。

{% block type="theorem" title="定理 7.2　ROC–AUC 的基率不变性" %}
ROC 曲线对**类先验概率的变化具有不变性**：即若保持正负类各自内部的评分分布不变，仅改变正负类的比例 $$\pi$$，ROC 曲线的形状和 AUC 值不变。

形式化地，设正类评分分布为 $$f_1(s)$$，负类评分分布为 $$f_0(s)$$。则 $$\text{TPR}(\tau) = \int_\tau^\infty f_1(s)\,ds$$ 和 $$\text{FPR}(\tau) = \int_\tau^\infty f_0(s)\,ds$$ 均不依赖于 $$\pi$$。因此 AUC 也不依赖于 $$\pi$$。
{% endblock %}

{% hint style="proof" %}
**证明** 由定义，$$\text{TPR}(\tau) = P(s > \tau \mid y = 1) = \int_\tau^\infty f_1(s)\,ds$$，$$\text{FPR}(\tau) = P(s > \tau \mid y = 0) = \int_\tau^\infty f_0(s)\,ds$$。两者均为条件于类标签的概率，与先验 $$\pi = P(y=1)$$ 无关。因此以 $$(\text{FPR}, \text{TPR})$$ 参数化的 ROC 曲线以及其面积 AUC 均不随 $$\pi$$ 变化。$$\blacksquare$$
{% endhint %}

定理 7.2 看似是 ROC 的优点（不受类不平衡影响），但在异常检测中恰恰构成**缺陷**：当 $$\pi = 0.001$$ 时，即使 FPR 仅为 0.01，对应的假正例数也高达 $$0.01 \times 0.999n \approx 10n_1$$——误报数量是真实异常的 10 倍！ROC 曲线无法反映这种绝对数量的不对称性。因此，**PR 曲线**在异常检测中更具实际参考价值。

{% block type="definition" title="定义 7.7　PR 曲线与 AP" %}
**PR 曲线**（Precision-Recall Curve）以召回率 Recall 为横轴、精确率 Precision 为纵轴，绘制不同阈值下 $$(\text{Rec}(\tau), \text{Prec}(\tau))$$ 的轨迹。

**平均精确率**（Average Precision, AP）为 PR 曲线下面积：

$$
\text{AP} = \int_0^1 \text{Prec}(\text{Rec})\, d\text{Rec} \approx \sum_{k=1}^{K} (\text{Rec}_k - \text{Rec}_{k-1}) \cdot \text{Prec}_k \tag{7.14}
$$

与 ROC 不同，PR 曲线的基线（随机分类器）不是 0.5，而是 $$\pi$$。当 $$\pi = 0.01$$ 时，随机分类器的 AP 仅为 $$0.01$$——这使得 PR 曲线能更灵敏地区分不同检测器在少数类上的表现差异。
{% endblock %}

{% hint style="tip" %}
💡 **实践建议：何时用 ROC，何时用 PR？**

- 当正负类比例相对均衡（$$\pi > 10\%$$）时，ROC 和 PR 曲线均可使用。
- 当 $$\pi < 5\%$$（如欺诈检测、入侵检测），**优先使用 PR 曲线和 AP**——ROC-AUC 可能虚高，掩盖了大量误报问题。
- 在工业部署中，通常关注的是"在固定误报率下能检出多少异常"——这直接对应 ROC 曲线上的一个操作点（operating point）。
{% endhint %}

## 7.1.4 异常检测的核心挑战

### 7.1.4.1 标签稀缺与定义模糊

异常检测最根本的挑战是**缺乏明确的"真相"**。

在分类任务中，我们有明确的标签（猫/狗、垃圾邮件/正常邮件）。但在异常检测中，"什么是异常"往往是模糊的、领域相关的、随时间变化的。信用卡消费 5000 元，在普通用户看来可能是异常，但对高收入用户则完全正常——**异常的定义依赖于上下文**。

更棘手的是标签获取成本极高：标注一笔交易是否为欺诈通常需要人工调查员核实，每个案例可能耗费数小时。这使得有监督方法很难获得足够的训练数据。

### 7.1.4.2 类不平衡

定理 7.1 已经揭示了类不平衡对评估指标的影响。在算法层面，不平衡同样带来困难：

{% block type="example" title="例 7.1　类不平衡的数量级效应" %}
某银行每天处理 100 万笔信用卡交易，其中约 0.1%（1000 笔）为欺诈。即使一个检测器的假正例率仅为 0.5%，每天仍会产生：

$$
\text{FP} = 0.005 \times 999{,}000 = 4{,}995 \text{ 笔误报}
$$

远超真实欺诈数（1000 笔）。调查员需要审核约 6000 笔可疑交易（TP + FP），其中只有约 17% 是真正的欺诈——这意味着大量人力浪费在误报上。此时，精确率仅为：

$$
\text{Prec} = \frac{1000}{1000 + 4995} \approx 16.7\%
$$

要将精确率提升至 50% 以上，FPR 需降至 $$0.1\%$$ 以下——这对检测算法提出了极高要求。
{% endblock %}

### 7.1.4.3 高维失效

在高维空间中，异常检测面临**维度灾难**（§5.7 已讨论）。随着维度 $$d$$ 增加，数据点之间的距离趋于均匀，密度估计变得不可靠。具体表现为：

- **距离集中现象**：高维空间中最远点和最近点的距离之比趋于 1，基于距离的异常检测（§7.3）失去区分能力；
- **密度估计失败**：准确估计高维概率密度需要的样本量随维度指数增长，使 LOF（§7.4）等方法在高维下性能退化；
- **子空间异常**：异常可能仅在某些特征子集中显现，全空间分析会将异常信号淹没在无关维度的噪声中。

### 7.1.4.4 概念漂移

在实际系统中，"正常"的定义会随时间变化。电商平台在"双十一"期间的交易模式与平日截然不同；网络流量在工作日和周末有不同的基线。如果异常检测模型在历史数据上训练后不更新，它会在新的正常模式下大量误报——这就是**概念漂移**（concept drift）问题。

应对概念漂移的常见策略包括：（1）使用滑动窗口仅保留最近数据；（2）在线学习方法实时更新模型；（3）周期性重新训练。§7.7 的欺诈检测案例将展示概念漂移的具体应对方案。

## 7.1.5 异常检测方法分类

本章后续各节将详细介绍五大类异常检测方法。下表提供了各方法的全景对比，供读者在学习过程中对照查阅。

| 方法类别 | 代表算法 | 核心思想 | 优势 | 局限 | 章节 |
|---------|---------|---------|------|------|------|
| 统计方法 | Z-score, Grubbs, M-距离 | 假设数据服从已知分布，偏离分布的点为异常 | 理论严谨、有统计显著性检验 | 需要分布假设，高维困难 | §7.2 |
| 基于距离 | KNN, 全局距离 | 离最近邻远的点为异常 | 无分布假设，直觉清晰 | 受维度灾难影响，$$O(n^2)$$ 复杂度 | §7.3 |
| 基于密度 | LOF, LOCI | 局部密度显著低于邻居的点为异常 | 能检测局部异常，对不均匀分布鲁棒 | 参数敏感（$$k$$ 值），高维退化 | §7.4 |
| 基于聚类 | DBSCAN, K-means | 不属于任何簇或距簇心远的点为异常 | 同时完成聚类与异常检测 | 聚类假设不总是成立 | §7.5 |
| 基于分类 | One-Class SVM, iForest | 学习正常类边界或利用隔离原理 | 可处理高维数据，iForest 线性复杂度 | 需要调参，单类 SVM 对核函数敏感 | §7.6 |

各方法之间并非完全独立——例如 LOF 可视为"基于密度的距离方法"，隔离森林可视为"基于随机划分的聚类方法"。理解它们的**共同数学基础**（即异常评分函数，定义 7.3）有助于在实际问题中灵活选择和组合方法。

{% hint style="frontier" %}
🚀 **前沿视野：深度异常检测**

近年来，深度学习为异常检测带来了新范式。**深度自编码器**（Deep Autoencoder）通过学习数据的低维表示来检测重构误差高的点；**变分自编码器**（VAE）提供了概率性的异常评分；**对抗生成网络**（GAN）通过生成正常数据的分布来识别异常。2020 年以来，**自监督学习**方法（如对比学习）在无标签异常检测中展现出强大性能（Shenkar & Wolf, 2022）。Transformer 架构也被应用于时间序列异常检测（Tuli et al., 2022），在多变量时序数据上取得了超越传统方法的效果。这些方法将在第十章（深度学习）中进一步探讨。
{% endhint %}

## 案例 7.1：合成数据上的异常检测全流程

{% block type="case" title="案例 7.1　从评分到评估——异常检测的完整工作流" %}
**问题背景**：为了深入理解异常检测的评分-阈值-评估框架，我们在可控的合成数据上演示完整流程。数据包含一个二维正态分布的正常类和散布在外围的异常点，模拟信用卡交易中"正常消费模式"与"异常消费模式"的场景。

**分析目标**：(1) 计算基于距离和基于密度的异常评分；(2) 对比阈值选择对精确率和召回率的影响；(3) 绘制 ROC 和 PR 曲线并计算 AUC 和 AP。
{% endblock %}

```python
# ============================================================
# 代码 7.1：异常检测评分-阈值-评估完整流程
# 数据集：合成二维数据（正态分布 + 均匀分布异常）
# 依赖库：numpy, sklearn
# ============================================================
# --- 1. 导入与数据生成 ---
import numpy as np
from sklearn.metrics import (precision_recall_curve, roc_curve, auc,
                             f1_score, precision_score, recall_score)
from sklearn.neighbors import NearestNeighbors
np.random.seed(42)
n_normal, n_anomaly = 950, 50
X_normal = np.random.multivariate_normal([0, 0], [[1, 0.5], [0.5, 1]], n_normal)
X_anomaly = np.random.uniform(-6, 6, (n_anomaly, 2))
X = np.vstack([X_normal, X_anomaly])
y_true = np.array([0]*n_normal + [1]*n_anomaly)
# --- 2. 计算异常评分（KNN距离） ---
k = 10
nn = NearestNeighbors(n_neighbors=k)
nn.fit(X)
distances, _ = nn.kneighbors(X)
scores_knn = distances[:, -1]  # 第k近邻距离作为异常评分
# --- 3. 评估指标计算 ---
fpr, tpr, _ = roc_curve(y_true, scores_knn)
prec_arr, rec_arr, _ = precision_recall_curve(y_true, scores_knn)
roc_auc = auc(fpr, tpr)
pr_ap = auc(rec_arr, prec_arr)
print(f"ROC-AUC: {roc_auc:.4f}")
print(f"PR-AP:   {pr_ap:.4f}")
# --- 4. 阈值分析 ---
thresholds = np.linspace(scores_knn.min(), scores_knn.max(), 100)
best_f1, best_tau = 0, 0
for tau in thresholds:
    y_pred = (scores_knn > tau).astype(int)
    f1 = f1_score(y_true, y_pred, zero_division=0)
    if f1 > best_f1:
        best_f1 = f1
        best_tau = tau
y_pred_best = (scores_knn > best_tau).astype(int)
print(f"最优阈值: {best_tau:.3f}")
print(f"最优 F1:  {best_f1:.4f}")
print(f"Precision: {precision_score(y_true, y_pred_best):.4f}")
print(f"Recall:    {recall_score(y_true, y_pred_best):.4f}")
```

**结果分析**：

1. **KNN 距离评分**有效区分了正常与异常——正常点的第 $$k$$ 近邻距离集中在较低值，异常点的距离显著更高，这与定义 7.3 的"密度越低、评分越高"一致；
2. **ROC-AUC 高于 PR-AP**——在异常比例仅 5% 的设定下，ROC-AUC 通常高估了检测器的实际表现，印证了定理 7.2 关于 ROC 基率不变性的分析；
3. **最优阈值**通过 F1 最大化确定——在精确率和召回率之间取得了平衡，适合不确定具体业务损失结构时的默认选择。

{% hint style="ideology" %}
🌏 **课程思政：异常检测与隐私保护的张力**

异常检测技术在反欺诈、反洗钱中发挥了巨大价值，但同时也引发了深刻的伦理问题。银行使用异常检测监控每一笔交易，意味着所有用户的消费行为都在被持续分析——这是否构成对隐私的侵犯？当系统将某位客户标记为"可疑"时，可能导致账户冻结、信用受损，而如果这是一次误报（FP），客户将承受巨大的不便甚至经济损失。

中国《个人信息保护法》（2021）和欧盟 GDPR 均要求：（1）数据处理须有合法目的，（2）用户有权知道被自动化决策影响的理由。异常检测系统的设计者有责任确保模型的可解释性（为什么标记为异常？）、公平性（是否对特定群体有偏见？）和透明性（用户能否申诉？）。在定理 7.1 的启示下，我们还应关注：降低 FPR 不仅是技术追求，更是社会责任——每一个误报背后都可能是一个无辜用户的不公正待遇。
{% endhint %}

| 关键词 | 核心概念 |
|--------|---------|
| 异常（定义 7.1） | 概率密度低于阈值的数据点，$$p(\mathbf{x}_i) < \tau$$，由不同生成机制产生 |
| 三种类型（定义 7.2） | 点异常（孤立偏离）、上下文异常（条件偏离）、集体异常（群体偏离） |
| 评分函数（定义 7.3） | $$s: \mathbb{R}^d \to \mathbb{R}$$，统一各方法的核心抽象 |
| 阈值决策（定义 7.4） | $$\hat{y}_i = \mathbb{1}[s(\mathbf{x}_i) > \tau]$$，Neyman–Pearson 框架 |
| 准确率失效（定理 7.1） | 平凡分类器 $$\text{Acc} = 1 - \pi \approx 99\%$$，准确率无信息量 |
| ROC 基率不变（定理 7.2） | ROC/AUC 不依赖类先验 $$\pi$$，在极端不平衡下过于乐观 |
| PR 曲线（定义 7.7） | 基线为 $$\pi$$，对不平衡更敏感，是异常检测的首选评估工具 |
| 四大挑战 | 标签稀缺、类不平衡、高维失效、概念漂移 |

下一节（§7.2）将从最经典的统计学视角出发，介绍基于分布假设的异常检测方法——它们是整个异常检测理论大厦的基石，也是理解后续非参数方法的必要前置知识。

### B 级（数学推导）

**B1.** 对于混合分布 $$p(\mathbf{x}) = (1-\pi)\,p_0(\mathbf{x}) + \pi\,p_1(\mathbf{x})$$，推导后验概率 $$P(y=1 \mid \mathbf{x})$$。证明：当 $$\pi \to 0$$ 时，即使 $$p_1(\mathbf{x}) \gg p_0(\mathbf{x})$$，后验概率也可能很低——这从贝叶斯视角解释了异常检测的固有困难。

**D4.** 使用 `sklearn.datasets.make_blobs` 生成 3 个簇的二维数据，手动添加 5% 的均匀分布异常点。实现 3 种异常评分方法：(1) 马氏距离、(2) KNN 距离、(3) LOF（可使用 `sklearn.neighbors.LocalOutlierFactor`），绘制三者的异常评分热力图和 ROC 曲线对比。

### E 级（案例分析）

**E1.** 在 **Credit Card Fraud Detection** 数据集（Kaggle，包含约 28 万笔交易，其中 0.17% 为欺诈）上完成异常检测分析：(1) 数据探索——分析异常比例、特征分布；(2) 使用隔离森林（Isolation Forest）和 LOF 进行检测；(3) 绘制两种方法的 ROC 和 PR 曲线，计算 AUC 和 AP；(4) 分析最优阈值下的精确率和召回率，讨论在实际反欺诈场景中的部署建议。

**E2.** 在 **KDD Cup 1999** 网络入侵检测数据集（或其精简版 NSL-KDD）上：(1) 将多类攻击标签统一为"正常/异常"二分类；(2) 使用 One-Class SVM 在仅正常数据上训练半监督模型；(3) 对比有监督随机森林和半监督 One-Class SVM 的检测效果；(4) 分析哪些攻击类型最容易被检测，哪些最容易被遗漏，并解释原因。

### F 级（开放探讨）

**F1.** **方法论思辨**：异常检测的定义（定义 7.1）要求 $$p(\mathbf{x}_i) < \tau$$，但在无监督设定下我们既不知道 $$p$$ 也不知道 $$\tau$$。这是否意味着异常检测本质上是一个"不适定问题"（ill-posed problem）？讨论：(1) 在什么条件下异常检测可以给出可靠的结果？(2) 不同方法对"异常"的隐含假设是什么？(3) 如何验证异常检测结果的正确性——如果连标签都没有？

**F2.** **前沿探索**：阅读 Liu et al. (2012) 关于隔离森林的论文。隔离森林的核心思想是"异常点更容易被随机划分隔离"，这与定义 7.3 的密度视角有什么联系？它如何避免了显式的密度估计？讨论隔离森林在高维数据中为何比 LOF 更鲁棒。
