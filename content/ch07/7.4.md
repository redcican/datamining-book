# 7.4 基于密度的方法（LOF）

上一节的 KNN 异常检测使用"到第 $k$ 近邻的距离"作为异常评分。这一方法简洁有效，但存在一个根本性缺陷：**它是全局方法，对所有区域使用统一的距离标准**。考虑一个包含两个密度迥异的簇的数据集——密集簇 $C_1$（点间距 $\approx 0.5$）和稀疏簇 $C_2$（点间距 $\approx 3.0$）。此时，$C_2$ 中任何一个完全正常的点，其 KNN 距离都远大于 $C_1$ 中的异常点。全局阈值无法同时适应两个簇——调低阈值会漏检 $C_1$ 中的异常，调高阈值会误报 $C_2$ 中的正常点。

Breunig 等人（2000）提出的**局部离群因子**（Local Outlier Factor, LOF）优雅地解决了这一问题。LOF 的核心洞察是：**异常不应该用绝对距离衡量，而应该用相对于局部邻域密度的偏离程度衡量**。一个点的 LOF 值表示"它的局部密度相对于其邻居的局部密度有多低"——如果一个点的密度显著低于其邻居，那么无论这个邻域是密集还是稀疏，该点都是局部异常。

LOF 的提出标志着异常检测从"全局视角"到"局部视角"的范式转变，其影响深远——后续的许多方法（如 LoOP、LOCI、COF）都沿袭了这一局部化思想。LOF 至今仍是 scikit-learn 中最常用的无监督异常检测算法之一。

## 7.4.1 为什么需要局部密度

### 7.4.1.1 KNN 全局方法的困境

§7.3 中的 KNN 方法将异常评分定义为 $s(\mathbf{x}_i) = d_k(\mathbf{x}_i)$——到第 $k$ 近邻的距离。这一评分的隐含假设是：**所有区域的"正常距离"是相同的**。但现实数据往往具有**异质密度**（heterogeneous density）——不同区域的数据密度差异悬殊。

考虑图 7.4.1 的情境：密集簇 $C_1$ 中点间的典型距离为 $\delta_1$，稀疏簇 $C_2$ 中点间的典型距离为 $\delta_2 \gg \delta_1$。设异常点 $\mathbf{o}$ 位于 $C_1$ 附近，其 KNN 距离为 $d_k(\mathbf{o}) = 3\delta_1$。此时：

$$
d_k(\mathbf{o}) = 3\delta_1 < \delta_2 \leq d_k(\mathbf{x}_j), \quad \forall\, \mathbf{x}_j \in C_2 \tag{7.49}
$$

即异常点 $\mathbf{o}$ 的 KNN 距离**小于** $C_2$ 中所有正常点的 KNN 距离。任何全局阈值 $\tau$ 都面临两难：$\tau < \delta_2$ 会漏检 $\mathbf{o}$（假阴性），$\tau \geq \delta_2$ 会误报 $C_2$ 中所有点（假阳性）。

### 7.4.1.2 局部化的核心思想

解决异质密度问题的关键在于：**不用绝对距离，而用相对于邻域的距离来评分**。直觉上，一个点是否"异常"不取决于它的 KNN 距离有多大，而取决于它的 KNN 距离**相对于其邻居的 KNN 距离**有多大：

$$
\text{局部异常度} \approx \frac{\text{自身的"稀疏程度"}}{\text{邻居的"稀疏程度"}} \tag{7.50}
$$

如果这个比值 $\approx 1$，说明该点与邻居一样密集（或一样稀疏）——它是正常的。如果比值 $\gg 1$，说明该点比邻居稀疏得多——它是局部异常。这一思想对密度均匀和异质密度数据都适用。Breunig 等人（2000）将这一直觉严格形式化为 LOF 框架，下面逐步构建其数学定义。

## 7.4.2 LOF 的数学框架

LOF 的构建分为四个层次，每一层在前一层的基础上添加一个概念：$k$-距离 → 可达距离 → 局部可达密度 → 局部离群因子。

### 7.4.2.1 k-距离

{% block type="definition" title="定义 7.19　k-距离（k-distance）" %}
设数据集 $\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$，正整数 $k \leq n - 1$。数据点 $\mathbf{x}$ 的 **$k$-距离**定义为到第 $k$ 个最近邻的距离：

$$
k\text{-dist}(\mathbf{x}) = d_k(\mathbf{x}) = \text{dist}(\mathbf{x}, \mathbf{x}_{(k)}) \tag{7.51}
$$

其中 $\mathbf{x}_{(k)}$ 是 $\mathcal{D} \setminus \{\mathbf{x}\}$ 中距 $\mathbf{x}$ 第 $k$ 近的点。$k$-距离的 **$k$-邻域** $N_k(\mathbf{x})$ 包含所有距离不超过 $k\text{-dist}(\mathbf{x})$ 的点：

$$
N_k(\mathbf{x}) = \{\mathbf{y} \in \mathcal{D} \setminus \{\mathbf{x}\} : \text{dist}(\mathbf{x}, \mathbf{y}) \leq k\text{-dist}(\mathbf{x})\} \tag{7.52}
$$

注意 $|N_k(\mathbf{x})| \geq k$（当存在距离恰好等于 $k\text{-dist}(\mathbf{x})$ 的多个点时，$|N_k(\mathbf{x})| > k$）。
{% endblock %}

$k$-距离与 §7.3 中的定义 7.15 本质相同，但 LOF 文献中习惯使用 $k\text{-dist}$ 记号。需要注意 $k$-邻域 $N_k(\mathbf{x})$ 可能包含超过 $k$ 个点——这是距离相同（ties）时的处理方式，保证了算法的确定性。

### 7.4.2.2 可达距离

可达距离是 LOF 框架的第一个关键创新。它在真实距离的基础上引入了一个"下限截断"，起到**平滑**作用。

{% block type="definition" title="定义 7.20　可达距离（Reachability Distance）" %}
数据点 $\mathbf{x}$ 相对于 $\mathbf{y}$ 的 **$k$-可达距离**定义为：

$$
\text{reach\text{-}dist}_k(\mathbf{x}, \mathbf{y}) = \max\{k\text{-dist}(\mathbf{y}),\; \text{dist}(\mathbf{x}, \mathbf{y})\} \tag{7.53}
$$

即可达距离是真实距离 $\text{dist}(\mathbf{x}, \mathbf{y})$ 与参考点 $\mathbf{y}$ 的 $k$-距离中的较大者。
{% endblock %}

可达距离的 $\max$ 操作有深刻的含义。当 $\mathbf{x}$ 在 $\mathbf{y}$ 的 $k$-邻域内（即 $\text{dist}(\mathbf{x}, \mathbf{y}) \leq k\text{-dist}(\mathbf{y})$）时，可达距离被"截断"为 $k\text{-dist}(\mathbf{y})$——无论 $\mathbf{x}$ 与 $\mathbf{y}$ 有多近，可达距离都不会低于 $k\text{-dist}(\mathbf{y})$。这一截断的作用是**减少密集区域内点间距离的统计波动**，使后续的密度估计更加稳定。

{% hint style="caution" %}
⚠️ **可达距离不是对称的**

注意 $\text{reach\text{-}dist}_k(\mathbf{x}, \mathbf{y}) \neq \text{reach\text{-}dist}_k(\mathbf{y}, \mathbf{x})$，因为截断项取决于参考点的 $k$-距离。例如，若 $\mathbf{x}$ 在密集区域（$k\text{-dist}(\mathbf{x})$ 小）而 $\mathbf{y}$ 在稀疏区域（$k\text{-dist}(\mathbf{y})$ 大），则 $\text{reach\text{-}dist}_k(\mathbf{x}, \mathbf{y})$ 由 $k\text{-dist}(\mathbf{y})$ 主导，而 $\text{reach\text{-}dist}_k(\mathbf{y}, \mathbf{x})$ 可能由 $\text{dist}(\mathbf{x}, \mathbf{y})$ 主导。这一不对称性是 LOF 能区分不同密度区域的关键所在。
{% endhint %}

### 7.4.2.3 局部可达密度

有了可达距离，我们可以定义每个点的局部密度。

{% block type="definition" title="定义 7.21　局部可达密度（Local Reachability Density, LRD）" %}
数据点 $\mathbf{x}$ 的 **局部可达密度**定义为其到 $k$ 个近邻的平均可达距离的倒数：

$$
\text{lrd}_k(\mathbf{x}) = \left(\frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \text{reach\text{-}dist}_k(\mathbf{x}, \mathbf{y})\right)^{-1} \tag{7.54}
$$

直观地，$\text{lrd}_k(\mathbf{x})$ 估计了 $\mathbf{x}$ 附近的局部密度——平均可达距离越小（邻居越密集），LRD 越大。
{% endblock %}

LRD 的关键性质在于：对于密集区域中的点，平均可达距离小，LRD 大；对于稀疏区域中的点，平均可达距离大，LRD 小。但**LRD 本身不能直接用于异常检测**——因为稀疏区域的正常点天然有较低的 LRD，这与 KNN 距离面临同样的异质密度问题。LOF 的最后一步将 LRD 相对化。

### 7.4.2.4 局部离群因子

{% block type="definition" title="定义 7.22　局部离群因子（Local Outlier Factor, LOF）" %}
数据点 $\mathbf{x}$ 的**局部离群因子** $\text{LOF}_k(\mathbf{x})$ 定义为其 $k$-邻域内各点的 LRD 与自身 LRD 之比的平均值：

$$
\text{LOF}_k(\mathbf{x}) = \frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})} \tag{7.55}
$$

等价地：

$$
\text{LOF}_k(\mathbf{x}) = \frac{\overline{\text{lrd}}_k(N_k(\mathbf{x}))}{\text{lrd}_k(\mathbf{x})} \tag{7.56}
$$

其中 $\overline{\text{lrd}}_k(N_k(\mathbf{x})) = \frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \text{lrd}_k(\mathbf{y})$ 是邻域的平均 LRD。
{% endblock %}

LOF 的直觉非常优雅：它衡量的是"**邻居的密度与自身密度之比**"。

- **LOF $\approx 1$**：该点的密度与其邻居相当——无论是在密集区域还是稀疏区域，只要与邻居一致，就是正常的；
- **LOF $\gg 1$**：该点的密度显著低于邻居——它比周围的点"更孤立"，是局部异常；
- **LOF $< 1$**：该点比邻居更密集——这在理论上可能出现（如簇的中心点），但通常接近 1。

正是这一相对化操作使 LOF 天然适应异质密度：稀疏簇 $C_2$ 中的正常点虽然 LRD 低，但其邻居（同样在 $C_2$ 中）的 LRD 也低，所以比值 $\approx 1$。只有真正"比邻居更孤立"的点才会获得高 LOF 值。

{% hint style="think" %}
🤔 **思考：LOF 何时退化为 KNN？**

如果数据密度完全均匀（所有区域的局部密度相同），则所有点的 LRD 近似相等，LOF $\approx 1$ 对所有正常点成立，异常点的 LOF 仅取决于其 KNN 距离与邻居 KNN 距离的比值——此时 LOF 与 KNN 异常评分的排序大致相同。换言之，**LOF 是 KNN 方法在异质密度下的推广**，均匀密度是其特殊情形。这意味着当你不确定数据密度是否均匀时，使用 LOF 是更安全的选择。
{% endhint %}

### 7.4.2.5 LOF 的性质

以下两个定理刻画了 LOF 的数学性质。

{% block type="theorem" title="定理 7.9　LOF 的界" %}
设 $\mathbf{x}$ 的 $k$-邻域为 $N_k(\mathbf{x})$，定义：

$$
\text{direct\_min}(\mathbf{x}) = \min_{\mathbf{y} \in N_k(\mathbf{x})} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})}, \quad \text{direct\_max}(\mathbf{x}) = \max_{\mathbf{y} \in N_k(\mathbf{x})} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})} \tag{7.57}
$$

则 LOF 满足：

$$
\text{direct\_min}(\mathbf{x}) \leq \text{LOF}_k(\mathbf{x}) \leq \text{direct\_max}(\mathbf{x}) \tag{7.58}
$$

特别地，LOF $\geq 0$（因为 LRD $> 0$）。当 $\mathbf{x}$ 深埋于一个密度均匀的簇中时，$\text{lrd}_k(\mathbf{y}) \approx \text{lrd}_k(\mathbf{x})$ 对所有 $\mathbf{y} \in N_k(\mathbf{x})$，故 $\text{LOF}_k(\mathbf{x}) \approx 1$。
{% endblock %}

{% hint style="proof" %}
**证明**：

由 LOF 的定义（式 7.55），$\text{LOF}_k(\mathbf{x})$ 是 $|N_k(\mathbf{x})|$ 个比值 $\text{lrd}_k(\mathbf{y})/\text{lrd}_k(\mathbf{x})$ 的算术平均。由算术平均的性质，

$$
\min_{\mathbf{y}} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})} \leq \frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})} \leq \max_{\mathbf{y}} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})} \tag{7.59}
$$

对于密度均匀簇中的内部点，所有近邻的 LRD 近似相等：$\text{lrd}_k(\mathbf{y}) \approx \text{lrd}_k(\mathbf{x})$。因此，每个比值 $\approx 1$，平均值 $\text{LOF}_k(\mathbf{x}) \approx 1$。 $\blacksquare$
{% endhint %}

以下定理给出 LOF 更深刻的密度比解释。

{% block type="theorem" title="定理 7.10　LOF 的密度比解释" %}
设 $\rho(\mathbf{x})$ 为 $\mathbf{x}$ 处的真实局部密度。在密度变化平缓（Lipschitz 连续）的条件下，当 $n \to \infty$，$k/n \to 0$ 时，LOF 渐近等价于邻域平均密度与自身密度之比：

$$
\text{LOF}_k(\mathbf{x}) \xrightarrow{P} \frac{\bar{\rho}(N_k(\mathbf{x}))}{\rho(\mathbf{x})} \tag{7.60}
$$

其中 $\bar{\rho}(N_k(\mathbf{x})) = \frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \rho(\mathbf{y})$ 是邻域的平均真实密度。
{% endblock %}

{% hint style="proof" %}
**证明要点**：

当 $n$ 充分大时，LRD 是局部密度的一致估计量。记 $\text{lrd}_k(\mathbf{x}) \xrightarrow{P} c \cdot \rho(\mathbf{x})$，其中 $c$ 是与 $k$ 和距离度量相关的常数（对所有点相同）。将此代入 LOF 定义：

$$
\text{LOF}_k(\mathbf{x}) = \frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \frac{\text{lrd}_k(\mathbf{y})}{\text{lrd}_k(\mathbf{x})} \xrightarrow{P} \frac{1}{|N_k(\mathbf{x})|}\sum_{\mathbf{y} \in N_k(\mathbf{x})} \frac{c \cdot \rho(\mathbf{y})}{c \cdot \rho(\mathbf{x})} = \frac{\bar{\rho}(N_k(\mathbf{x}))}{\rho(\mathbf{x})} \tag{7.61}
$$

常数 $c$ 在分子分母中抵消，LOF 只依赖密度的**比值**，不依赖绝对密度。这正是 LOF 能适应异质密度的数学本质。 $\blacksquare$
{% endhint %}

定理 7.10 揭示了 LOF 的深层含义：**LOF 测量的是局部密度梯度**。如果一个点的密度低于其邻域平均密度（即处于密度的"下坡"方向），LOF $> 1$；如果密度与邻域一致，LOF $\approx 1$。异常点正是那些处于密度"悬崖"上的点——密度急剧下降。

### 7.4.2.6 计算示例

{% block type="example" title="例 7.4　LOF 的逐步计算" %}
考虑一维数据集 $\mathcal{D} = \{1, 2, 3, 10, 11, 12, 30\}$，取 $k = 2$，使用欧氏距离。

**第 1 步：计算 $k$-距离**

| 点 $x$ | 2-NN | $k\text{-dist}(x)$ |
|--------|------|-------------------|
| 1 | \{2, 3\} | 2 |
| 2 | \{1, 3\} | 1 |
| 3 | \{2, 1\} | 2 |
| 10 | \{11, 12\} | 2 |
| 11 | \{10, 12\} | 1 |
| 12 | \{11, 10\} | 2 |
| 30 | \{12, 11\} | 19 |

**第 2 步：计算可达距离**（以 $x = 30$ 为例）

$N_2(30) = \{12, 11\}$

- $\text{reach\text{-}dist}_2(30, 12) = \max\{k\text{-dist}(12),\, |30-12|\} = \max\{2, 18\} = 18$
- $\text{reach\text{-}dist}_2(30, 11) = \max\{k\text{-dist}(11),\, |30-11|\} = \max\{1, 19\} = 19$

**第 3 步：计算 LRD**

$\text{lrd}_2(30) = \left(\frac{18 + 19}{2}\right)^{-1} = \frac{2}{37} \approx 0.054$

同理计算邻居的 LRD：$\text{lrd}_2(12) = \left(\frac{\max\{1,1\} + \max\{2,2\}}{2}\right)^{-1} = (1.5)^{-1} \approx 0.667$

$\text{lrd}_2(11) = \left(\frac{\max\{2,1\} + \max\{2,1\}}{2}\right)^{-1} = (2)^{-1} = 0.5$

**第 4 步：计算 LOF**

$$
\text{LOF}_2(30) = \frac{1}{2}\left(\frac{\text{lrd}_2(12)}{\text{lrd}_2(30)} + \frac{\text{lrd}_2(11)}{\text{lrd}_2(30)}\right) = \frac{1}{2}\left(\frac{0.667}{0.054} + \frac{0.5}{0.054}\right) \approx \frac{1}{2}(12.35 + 9.26) \approx 10.8
$$

$\text{LOF}_2(30) \approx 10.8 \gg 1$，确认 $x = 30$ 是强异常。相比之下，簇内点（如 $x = 2$）的 LOF $\approx 1$。
{% endblock %}

## 7.4.3 算法描述与复杂度

### 7.4.3.1 LOF 算法

{% block type="algorithm" title="算法 7.3　LOF 异常检测" %}
**输入**：数据集 $\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$，近邻数 $k$，距离函数 $\text{dist}$

**输出**：每个点的 LOF 值 $\{\text{LOF}_k(\mathbf{x}_1), \ldots, \text{LOF}_k(\mathbf{x}_n)\}$

1. **KNN 查询**：对每个 $\mathbf{x}_i$，查询 $k$-邻域 $N_k(\mathbf{x}_i)$ 和 $k\text{-dist}(\mathbf{x}_i)$
2. **可达距离**：对每对 $(\mathbf{x}_i, \mathbf{y})$ 其中 $\mathbf{y} \in N_k(\mathbf{x}_i)$，计算 $\text{reach\text{-}dist}_k(\mathbf{x}_i, \mathbf{y}) = \max\{k\text{-dist}(\mathbf{y}), \text{dist}(\mathbf{x}_i, \mathbf{y})\}$
3. **局部可达密度**：$\text{lrd}_k(\mathbf{x}_i) = |N_k(\mathbf{x}_i)| \,\Big/\, \sum_{\mathbf{y} \in N_k(\mathbf{x}_i)} \text{reach\text{-}dist}_k(\mathbf{x}_i, \mathbf{y})$
4. **局部离群因子**：$\text{LOF}_k(\mathbf{x}_i) = \frac{1}{|N_k(\mathbf{x}_i)|}\sum_{\mathbf{y} \in N_k(\mathbf{x}_i)} \text{lrd}_k(\mathbf{y}) \,\Big/\, \text{lrd}_k(\mathbf{x}_i)$
5. **排序与检测**：按 LOF 值降序排列，LOF $\gg 1$ 的点为异常
{% endblock %}

### 7.4.3.2 复杂度分析

| 步骤 | 朴素实现 | 索引加速（KD-tree） |
|------|---------|-------------------|
| KNN 查询 | $O(n^2 d)$ | $O(n \cdot k \cdot d \cdot \log n)$ |
| 可达距离 | $O(nk)$ | $O(nk)$ |
| LRD 计算 | $O(nk)$ | $O(nk)$ |
| LOF 计算 | $O(nk)$ | $O(nk)$ |
| **总计** | $O(n^2 d)$ | $O(n \cdot k \cdot d \cdot \log n + nk)$ |

计算瓶颈在第一步的 KNN 查询——与 §7.3 的 KNN 异常检测相同。LOF 在 KNN 之上增加的额外开销仅为 $O(nk)$，可以忽略不计。因此，**LOF 的计算代价与 KNN 异常检测几乎相同，但检测效果在异质密度场景下显著更优**。

{% hint style="engineering" %}
🔧 **工程实践：大规模 LOF 的实现技巧**

在生产环境中部署 LOF 时，以下优化至关重要：(1) **增量更新**：当新数据到来时，不必重算所有点的 LOF——只需更新受影响的点（新点的 $k$-邻域内的点）的 LRD 和 LOF，时间复杂度从 $O(n)$ 降至 $O(k^2)$；(2) **近似计算**：使用 HNSW 或 LSH 进行近似 KNN 查询，牺牲少量精度换取 10–100 倍的加速；(3) **并行化**：步骤 2–4 对各点独立，天然适合 GPU 或多线程并行。scikit-learn 的 `LocalOutlierFactor` 已利用了 Ball-tree 索引，对于 $n < 10^5$ 的数据通常在数秒内完成。
{% endhint %}

## 7.4.4 参数选择与变体

### 7.4.4.1 参数 k 的影响

LOF 的唯一超参数是近邻数 $k$。与 KNN 方法类似（§7.3.3.3），$k$ 的选择需要在局部性和稳定性之间权衡：

- **$k$ 过小**（如 $k = 1, 2$）：LRD 估计基于极少的近邻，统计不稳定。个别噪声点可能导致 LOF 值剧烈波动；
- **$k$ 过大**（如 $k > n/4$）：邻域过大，局部密度变为全局密度，LOF 退化为全局方法，丧失局部自适应性；
- **合理范围**：Breunig 等人（2000）建议 $k \in [10, 50]$。实践中，$k = 20$ 是常用的默认值。

{% hint style="tip" %}
💡 **多 k 融合策略（mLOF）**

当无法确定最优 $k$ 时，可对多个 $k$ 值计算 LOF 并取最大值：

$$
\text{mLOF}(\mathbf{x}) = \max_{k \in \{k_1, k_2, \ldots, k_m\}} \text{LOF}_k(\mathbf{x}) \tag{7.62}
$$

例如取 $k \in \{10, 20, 30, 50\}$。由于 LOF 的归一化特性（正常点的 LOF $\approx 1$ 对所有 $k$ 成立），取最大值不会显著增加误报率，但能捕获在不同尺度上显现的异常。
{% endhint %}

### 7.4.4.2 LOF 的变体

LOF 的成功催生了一系列改进方法：

| 方法 | 改进点 | 输出 | 参考文献 |
|------|--------|------|---------|
| **LoOP** | 基于概率模型，输出在 $[0, 1]$ 内的异常概率 | $P(\text{outlier})$ | Kriegel et al. (2009) |
| **LOCI** | 自动确定参数，使用多粒度计数 | 异常标签 | Papadimitriou et al. (2003) |
| **COF** | 考虑链式距离，处理非球形簇 | COF 值 | Tang et al. (2002) |
| **INFLO** | 考虑反向 KNN，改善边界点处理 | INFLO 值 | Jin et al. (2006) |

其中 **LoOP**（Local Outlier Probabilities）尤为值得关注——它将 LOF 的无量纲评分转化为概率值，使得不同数据集上的结果可以直接比较。LoOP 假设每个点到其近邻的距离服从半正态分布，利用此分布计算异常概率 $P(\text{outlier} \mid \mathbf{x}) \in [0, 1]$。

{% hint style="frontier" %}
🚀 **前沿视野：从 LOF 到深度密度估计**

LOF 的核心思想——"局部密度异常"——在深度学习时代得到了新的演绎。**深度自编码器**（Autoencoder）将数据映射到低维潜在空间后计算重构误差，本质上是在学习后的流形上估计"局部正常性"。Zong 等人（2018）提出的 **DAGMM**（Deep Autoencoding Gaussian Mixture Model）将自编码器的潜在表示与 GMM 密度估计结合，可视为 LOF 思想在非线性空间中的推广。更近期的 **Flow-based** 方法（如 Normalizing Flows）直接学习精确的概率密度函数，避免了 LOF 的 KNN 近似——在图像和时序异常检测中展现出卓越性能。这些进展表明，LOF 的"局部密度比"思想具有超越具体算法的持久价值。
{% endhint %}

## 7.4.5 LOF 与 KNN 的系统对比

| 维度 | KNN 异常检测（§7.3） | LOF（本节） |
|------|---------------------|------------|
| **评分函数** | $s(\mathbf{x}) = d_k(\mathbf{x})$（绝对距离） | $s(\mathbf{x}) = \text{LOF}_k(\mathbf{x})$（相对密度比） |
| **异质密度** | ❌ 全局阈值无法适应 | ✅ 局部自适应 |
| **均匀密度** | ✅ 简单有效 | ✅ 退化为 KNN（等价） |
| **参数** | $k$（近邻数） | $k$（近邻数，相同） |
| **输出解释** | 距离值（量纲依赖） | 无量纲比值（LOF $\approx 1$ = 正常） |
| **计算代价** | $O(n^2d)$ / $O(nkd\log n)$ | 相同（额外 $O(nk)$ 可忽略） |
| **适用场景** | 密度均匀、低维 | 密度异质、多尺度 |

{% hint style="caution" %}
⚠️ **LOF 不是万能的**

LOF 虽然解决了异质密度问题，但仍有局限：(1) **高维退化**——与所有基于距离的方法一样，LOF 在高维下受距离集中现象影响（定理 7.8）；(2) **全局异常可能被忽略**——如果异常点恰好位于两个稀疏簇之间且距离与簇内距离相当，LOF 可能给出接近 1 的值；(3) **时间复杂度**——LOF 的 KNN 查询步骤仍为 $O(n^2d)$，对于百万级数据需要近似方法加速。对于高维或超大规模数据，考虑使用 §7.6 中的隔离森林。
{% endhint %}

## 案例 7.4：信用卡交易异常检测

{% block type="case" title="案例 7.4　多密度场景下的信用卡交易异常检测" %}
**问题背景**：某银行的信用卡交易数据包含不同消费群体：日常消费者（小额高频，交易密集）和商务客户（大额低频，交易稀疏）。欺诈交易隐藏在两个群体中，表现为偏离各自群体正常模式的异常。传统全局方法将所有商务客户的正常大额交易误判为异常。

**分析目标**：(1) 使用 LOF 在多密度数据上检测异常，展示其局部自适应优势；(2) 与 KNN 全局方法对比；(3) 分析参数 $k$ 对检测效果的影响。
{% endblock %}

```python
# ============================================================
# 代码 7.4：LOF 异常检测（从零实现 vs sklearn 对比）
# 数据集：合成信用卡交易数据（双密度群体 + 注入欺诈）
# 依赖库：numpy, scipy, sklearn
# ============================================================
# --- 1. 导入与数据生成 ---
import numpy as np
from scipy.spatial.distance import cdist
from sklearn.neighbors import LocalOutlierFactor, NearestNeighbors
from sklearn.metrics import roc_auc_score
np.random.seed(42)
# 日常消费者：密集簇（小额高频，200人）
n_daily = 200
X_daily = np.column_stack([
    np.random.normal(50, 8, n_daily),     # 金额（元）
    np.random.normal(25, 3, n_daily),     # 频次（次/月）
    np.random.normal(5, 1.5, n_daily)])   # 单笔时长（秒）
# 商务客户：稀疏簇（大额低频，60人）
n_biz = 60
X_biz = np.column_stack([
    np.random.normal(5000, 800, n_biz),   # 金额
    np.random.normal(3, 1.5, n_biz),      # 频次
    np.random.normal(120, 30, n_biz)])    # 单笔时长
# 欺诈交易：各群体中注入异常
n_fraud = 15
X_fraud_daily = np.column_stack([
    np.random.normal(200, 30, n_fraud//3*2),
    np.random.normal(40, 5, n_fraud//3*2),
    np.random.normal(1, 0.5, n_fraud//3*2)])  # 日常群体中的异常
X_fraud_biz = np.column_stack([
    np.random.normal(500, 100, n_fraud//3),
    np.random.normal(50, 10, n_fraud//3),
    np.random.normal(2, 1, n_fraud//3)])      # 商务群体中的异常
X = np.vstack([X_daily, X_biz, X_fraud_daily, X_fraud_biz])
y_true = np.array([0]*n_daily + [0]*n_biz + [1]*len(X_fraud_daily) + [1]*len(X_fraud_biz))
# 标准化
X_std = (X - X.mean(axis=0)) / X.std(axis=0)
# --- 2. 从零实现 LOF ---
def lof_from_scratch(X, k):
    """从零实现 LOF 算法"""
    n = len(X)
    dist_matrix = cdist(X, X, metric='euclidean')
    np.fill_diagonal(dist_matrix, np.inf)
    # 步骤 1: KNN 查询
    knn_indices = np.argsort(dist_matrix, axis=1)[:, :k]
    k_dists = np.array([dist_matrix[i, knn_indices[i, -1]] for i in range(n)])
    # 步骤 2: 可达距离
    reach_dists = np.zeros((n, k))
    for i in range(n):
        for j_idx, j in enumerate(knn_indices[i]):
            reach_dists[i, j_idx] = max(k_dists[j], dist_matrix[i, j])
    # 步骤 3: LRD
    lrd = 1.0 / reach_dists.mean(axis=1)
    # 步骤 4: LOF
    lof_scores = np.zeros(n)
    for i in range(n):
        neighbor_lrds = lrd[knn_indices[i]]
        lof_scores[i] = neighbor_lrds.mean() / lrd[i]
    return lof_scores
# --- 3. LOF 检测 ---
k = 20
lof_scores = lof_from_scratch(X_std, k)
auc_lof = roc_auc_score(y_true, lof_scores)
print(f"LOF (从零实现, k={k}): AUC={auc_lof:.4f}")
# --- 4. KNN 基线对比 ---
nn = NearestNeighbors(n_neighbors=k)
nn.fit(X_std)
dists, _ = nn.kneighbors(X_std)
knn_scores = dists[:, -1]
auc_knn = roc_auc_score(y_true, knn_scores)
print(f"KNN (k={k}):            AUC={auc_knn:.4f}")
# --- 5. sklearn 验证 ---
clf = LocalOutlierFactor(n_neighbors=k, contamination='auto')
clf.fit_predict(X_std)
lof_sk = -clf.negative_outlier_factor_  # sklearn 返回负值
auc_sk = roc_auc_score(y_true, lof_sk)
print(f"LOF (sklearn, k={k}):   AUC={auc_sk:.4f}")
# --- 6. 不同 k 值对比 ---
for k_val in [5, 10, 20, 50]:
    scores = lof_from_scratch(X_std, k_val)
    auc = roc_auc_score(y_true, scores)
    print(f"LOF k={k_val:2d}: AUC={auc:.4f}")
```

**结果分析**：

1. **LOF 显著优于 KNN**：在双密度数据上，LOF 的 AUC 通常高于 KNN——因为 KNN 将商务客户的正常大额交易误判为异常，而 LOF 通过局部密度归一化避免了这一问题；
2. **从零实现与 sklearn 一致**：两者的 AUC 几乎完全相同，验证了我们的实现正确性。sklearn 返回负值（`negative_outlier_factor_`），取反后即为标准 LOF 值；
3. **$k$ 值稳定性**：LOF 在 $k \in [10, 50]$ 范围内 AUC 变化很小，证实了 LOF 对参数选择的鲁棒性——这比 KNN 的 $k$ 敏感度更低，因为 LOF 的相对化操作消除了 $k$ 对绝对距离尺度的影响。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| $k$-距离（定义 7.19） | $k\text{-dist}(\mathbf{x})$：到第 $k$ 近邻的距离，LOF 的基础度量 |
| 可达距离（定义 7.20） | $\text{reach\text{-}dist}_k(\mathbf{x}, \mathbf{y}) = \max\{k\text{-dist}(\mathbf{y}), d(\mathbf{x},\mathbf{y})\}$，平滑截断 |
| 局部可达密度（定义 7.21） | $\text{lrd}_k(\mathbf{x})$：平均可达距离的倒数，估计局部密度 |
| 局部离群因子（定义 7.22） | $\text{LOF}_k(\mathbf{x}) = \overline{\text{lrd}}_k(N_k) / \text{lrd}_k(\mathbf{x})$，密度比 |
| LOF 的界（定理 7.9） | $\text{LOF} \approx 1$ 当密度均匀，$\text{LOF} \gg 1$ 当局部异常 |
| 密度比解释（定理 7.10） | LOF 渐近等价于邻域平均密度与自身密度之比 |
| 时间复杂度 | 与 KNN 相同（额外 $O(nk)$），瓶颈在 KNN 查询 |
| LOF 变体 | LoOP（概率化）、LOCI（自动参数）、COF（链式距离） |

LOF 通过**局部密度归一化**实现了对异质密度的自适应，是距离方法（§7.3）的自然升级。其核心贡献在于证明了"异常是相对的"——同一个 KNN 距离在密集区域可能意味着异常，在稀疏区域可能完全正常。LOF 将这一认识编码为精确的数学框架。

然而，无论 KNN 还是 LOF，都是纯粹从**数据几何**（距离/密度）出发的无监督方法。下一节（§7.5）将介绍基于聚类的异常检测方法——它们先将数据划分为若干组，再根据数据点与所属簇的关系判断异常。这种"先结构化，再检测"的策略为异常检测引入了更丰富的结构信息。

### E 级（案例分析）

**E1.** 在 **Credit Card Fraud Detection** 数据集（Kaggle，包含 284,807 笔交易，492 笔欺诈）上：(1) 选取 PCA 特征（V1–V28）和 Amount、Time；(2) 由于数据量大，随机采样 10,000 笔正常交易 + 全部欺诈交易；(3) 使用 LOF（$k = 20$）和 KNN 方法检测欺诈；(4) 对比两者的 PR-AP（注意极端不平衡）；(5) 分析 LOF 是否有效——如果效果不佳，推测原因。

**E2.** 在 **Satellite** 数据集（UCI / OpenML #40900）上使用 LOF 检测异常类别（将少数类视为异常）。分析：(1) 不同 $k$ 值对检测率的影响；(2) 与 §7.2 的马氏距离方法和 §7.3 的 KNN 方法对比；(3) 绘制三种方法的 ROC 曲线。

### F 级（开放探讨）

**F1.** **LOF 的局部性极限**：LOF 假设每个点的"正常"应参照其 $k$-邻域来判断。但如果异常点形成一个小型密集簇（如一群协同作案的欺诈者），LOF 可能给这些点较低的值（因为它们彼此是"邻居"，密度一致）。讨论：(1) 这种"群体掩蔽"现象的具体条件；(2) 有什么方法可以缓解？（提示：考虑使用不同尺度的 $k$ 值，或结合全局方法。）

**F2.** **从 LOF 到图神经网络异常检测**：LOF 的核心是"比较一个节点与其邻居的属性"——这与图神经网络（GNN）中的消息传递机制（message passing）有惊人的相似性。讨论：(1) 如何将 LOF 框架转化为图上的异常检测？(2) GNN-based 异常检测（如 GraphSAGE-AD, DOMINANT）与 LOF 的概念联系；(3) 图结构如何提供超越 KNN 图的更丰富的邻域信息？
