# 7.5 基于聚类的方法

前面两节分别介绍了基于距离（§7.3）和基于密度（§7.4）的异常检测方法。这些方法直接从数据点的几何关系出发，计算每个点的"孤立程度"或"局部密度偏离"。本节引入一种不同的策略：**先用聚类算法将数据划分为若干组，再根据数据点与所属簇的关系判断异常**。

这种"先结构化，再检测"的两阶段策略有独特的优势。聚类将无序的数据组织为有结构的簇——每个簇代表一种"正常模式"。在此基础上，异常被定义为不符合任何正常模式的数据点。这一框架与许多业务场景天然吻合：用户被分为不同的消费群体（簇），偏离所属群体行为模式的交易就是潜在的欺诈。

聚类方法在异常检测中的应用可追溯到 Smith 和 Jain（1988），他们首次系统地将聚类结果用于离群值判别。此后，He 等人（2003）提出了 CBLOF（Cluster-Based Local Outlier Factor），将聚类信息与局部密度估计结合。近年来，基于 DBSCAN 的噪声点检测因其自动识别异常的特性而广泛应用于空间数据分析和网络安全。

## 7.5.1 聚类与异常检测的关系

### 7.5.1.1 两阶段框架

{% block type="definition" title="定义 7.23　基于聚类的异常检测框架" %}
给定数据集 $\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\} \subset \mathbb{R}^d$，基于聚类的异常检测采用**两阶段框架**：

**阶段 1（结构化）**：使用聚类算法将 $\mathcal{D}$ 划分为 $K$ 个簇 $\{C_1, C_2, \ldots, C_K\}$（及可能的噪声集 $C_{\text{noise}}$），每个簇 $C_j$ 的代表参数为 $(\boldsymbol{\mu}_j, r_j, |C_j|)$（中心、半径、大小）。

**阶段 2（评分）**：基于聚类结构定义异常评分函数 $s: \mathbb{R}^d \to \mathbb{R}$，并根据阈值 $\tau$ 判断异常：

$$
s(\mathbf{x}_i) = f\left(\text{dist}(\mathbf{x}_i, \boldsymbol{\mu}_{c(i)}),\; |C_{c(i)}|,\; r_{c(i)}\right) \tag{7.63}
$$

其中 $c(i) = \arg\min_j \text{dist}(\mathbf{x}_i, \boldsymbol{\mu}_j)$ 是 $\mathbf{x}_i$ 所属的簇编号。
{% endblock %}

这一框架的灵活性在于：阶段 1 可以使用任何聚类算法（K-means、DBSCAN、GMM 等），阶段 2 的评分函数 $f$ 可以根据不同的异常模式设计。以下定义了三种基本的聚类异常模式。

### 7.5.1.2 三种聚类异常模式

{% block type="definition" title="定义 7.24　三种基于聚类的异常模式" %}
基于聚类结构，异常可分为以下三类：

**（1）不属于任何簇的点**：数据点 $\mathbf{x}_i$ 与所有簇中心的距离都超过阈值：

$$
\min_{j=1,\ldots,K} \text{dist}(\mathbf{x}_i, \boldsymbol{\mu}_j) > \tau_1 \tag{7.64}
$$

这类异常是真正的"离群点"，它们位于所有簇的边界之外。在 DBSCAN 中，这些点被自然标记为噪声。

**（2）属于小簇的点**：数据点 $\mathbf{x}_i$ 所属的簇 $C_{c(i)}$ 的大小极小：

$$
|C_{c(i)}| < \tau_2 \tag{7.65}
$$

小簇可能代表异常模式——少数几个点聚在一起形成的"微簇"往往比孤立离群点更值得关注，因为它们可能代表有组织的异常行为（如协同欺诈）。

**（3）簇边缘的点**：数据点 $\mathbf{x}_i$ 虽然属于某个簇，但位于该簇的边缘，远离簇中心：

$$
\text{dist}(\mathbf{x}_i, \boldsymbol{\mu}_{c(i)}) > \alpha \cdot r_{c(i)} \tag{7.66}
$$

其中 $r_{c(i)}$ 是簇 $C_{c(i)}$ 的半径（如平均距离或标准差），$\alpha > 1$ 是倍数阈值。
{% endblock %}

{% hint style="think" %}
🤔 **思考：聚类异常 vs 距离/密度异常**

三种聚类异常模式与前面章节的方法有什么关系？模式（1）对应 KNN 距离方法——远离所有簇心即 KNN 距离大。模式（3）类似 LOF——簇边缘点的密度低于簇中心。但模式（2）是聚类方法独有的——KNN 和 LOF 无法识别"一组聚在一起但整体异常"的点，因为这些点在局部密度上与正常点无异。这揭示了聚类方法的独特价值：它不仅检测**个体异常**，还能识别**群体异常**。
{% endhint %}

## 7.5.2 基于 K-means 的异常检测

### 7.5.2.1 距离评分

K-means 是最简单的聚类方法（§5.1），它将数据划分为 $K$ 个簇，每个簇由其质心 $\boldsymbol{\mu}_j$ 表示。基于 K-means 的异常评分直接使用到簇心的距离。

{% block type="definition" title="定义 7.25　K-means 异常评分" %}
设 K-means 将数据划分为 $K$ 个簇 $\{C_1, \ldots, C_K\}$，质心为 $\{\boldsymbol{\mu}_1, \ldots, \boldsymbol{\mu}_K\}$。数据点 $\mathbf{x}_i$ 被分配到最近的簇 $c(i) = \arg\min_j \|\mathbf{x}_i - \boldsymbol{\mu}_j\|$。定义两种异常评分：

**（1）绝对距离评分**：

$$
s_{\text{abs}}(\mathbf{x}_i) = \|\mathbf{x}_i - \boldsymbol{\mu}_{c(i)}\| \tag{7.67}
$$

**（2）相对距离评分**（归一化至簇的尺度）：

$$
s_{\text{rel}}(\mathbf{x}_i) = \frac{\|\mathbf{x}_i - \boldsymbol{\mu}_{c(i)}\|}{\bar{r}_{c(i)}} \tag{7.68}
$$

其中 $\bar{r}_j = \frac{1}{|C_j|}\sum_{\mathbf{x} \in C_j} \|\mathbf{x} - \boldsymbol{\mu}_j\|$ 是簇 $C_j$ 内点到质心的平均距离。
{% endblock %}

相对距离评分 $s_{\text{rel}}$ 对不同大小的簇进行了归一化——在大簇中距离 5 个单位可能是正常的，但在紧凑的小簇中可能是异常。这种归一化思想与 LOF 的局部密度归一化（§7.4）异曲同工。

以下定理揭示了 K-means 距离评分与统计方法的深层联系。

{% block type="theorem" title="定理 7.11　K-means 距离评分与马氏距离的联系" %}
设簇 $C_j$ 中的数据近似服从多元正态分布 $\mathcal{N}(\boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j)$。若各簇的协方差矩阵为球形（$\boldsymbol{\Sigma}_j = \sigma_j^2 \mathbf{I}_d$），则 K-means 的相对距离评分与马氏距离成正比：

$$
s_{\text{rel}}(\mathbf{x}_i) \propto D_M(\mathbf{x}_i; \boldsymbol{\mu}_{c(i)}, \boldsymbol{\Sigma}_{c(i)}) \tag{7.69}
$$

当协方差矩阵非球形时，K-means 距离评分低估了沿协方差短轴方向的偏离。
{% endblock %}

{% hint style="proof" %}
**证明**：

当 $\boldsymbol{\Sigma}_j = \sigma_j^2 \mathbf{I}_d$ 时，马氏距离为：

$$
D_M(\mathbf{x}_i; \boldsymbol{\mu}_j, \boldsymbol{\Sigma}_j) = \sqrt{(\mathbf{x}_i - \boldsymbol{\mu}_j)^\top (\sigma_j^2 \mathbf{I}_d)^{-1} (\mathbf{x}_i - \boldsymbol{\mu}_j)} = \frac{\|\mathbf{x}_i - \boldsymbol{\mu}_j\|}{\sigma_j} \tag{7.70}
$$

而簇内平均距离 $\bar{r}_j = E[\|\mathbf{x} - \boldsymbol{\mu}_j\|]$。对于 $d$ 维球形正态分布，$\|\mathbf{x} - \boldsymbol{\mu}_j\| / \sigma_j \sim \chi(d)$（卡方分布的平方根），故 $\bar{r}_j = \sigma_j \cdot E[\chi(d)] = \sigma_j \cdot \sqrt{2}\,\Gamma((d+1)/2)/\Gamma(d/2)$。因此：

$$
s_{\text{rel}}(\mathbf{x}_i) = \frac{\|\mathbf{x}_i - \boldsymbol{\mu}_j\|}{\bar{r}_j} = \frac{D_M(\mathbf{x}_i) \cdot \sigma_j}{\sigma_j \cdot E[\chi(d)]} = \frac{D_M(\mathbf{x}_i)}{E[\chi(d)]} \tag{7.71}
$$

即 $s_{\text{rel}} \propto D_M$，比例常数仅依赖维度 $d$。

当 $\boldsymbol{\Sigma}_j$ 非球形时，K-means 使用欧氏距离无法感知协方差结构——沿长轴方向的偏离被低估，沿短轴方向的偏离被高估。此时应使用 GMM 聚类（§5.5）并基于马氏距离评分。 $\blacksquare$
{% endhint %}

### 7.5.2.2 K 值的选择

K-means 异常检测中，$K$ 的选择对检测结果有重要影响：

- **$K$ 过小**：多个真实簇被合并为一个大簇，簇内方差增大，正常点也可能获得较大的距离评分（假阳性）；
- **$K$ 过大**：异常点可能自成一簇或被少数正常点"吸引"，导致异常点距离其（错误的）质心很近（假阴性）；
- **实用建议**：使用 §5.6 中的轮廓系数或肘部法则选择 $K$，或者尝试多个 $K$ 值并比较检测结果的稳定性。

{% hint style="caution" %}
⚠️ **K-means 异常检测的局限性**

K-means 假设簇是球形的——这意味着它只能检测沿径向方向的偏离。对于非凸簇（如半月形、环形数据），K-means 的质心可能落在数据稀疏区域，导致许多正常点到质心的距离很大。此外，K-means 对初始化敏感——不同的运行可能产生不同的聚类结果，从而影响异常评分的稳定性。使用 K-means++ 初始化和多次运行取最优可缓解这一问题。
{% endhint %}

## 7.5.3 基于 DBSCAN 的异常检测

### 7.5.3.1 噪声点即异常

DBSCAN（§5.2）是一种基于密度的聚类算法，它将数据划分为密集区域（簇）和稀疏区域（噪声）。与 K-means 不同，DBSCAN **自动标记不属于任何簇的点为噪声**——这些噪声点正是潜在的异常。

{% block type="definition" title="定义 7.26　DBSCAN 异常检测" %}
设 DBSCAN 以参数 $(\varepsilon, \textit{MinPts})$ 对数据集 $\mathcal{D}$ 进行聚类，产生簇 $\{C_1, \ldots, C_K\}$ 和噪声集 $C_{\text{noise}}$。基于 DBSCAN 的异常检测定义为：

$$
\hat{y}_i = \begin{cases}
1 \quad (\text{异常}), & \text{if } \mathbf{x}_i \in C_{\text{noise}} \\
0 \quad (\text{正常}), & \text{if } \mathbf{x}_i \in C_j \text{ for some } j
\end{cases} \tag{7.72}
$$

即**噪声点 = 异常点**。更精细地，可为噪声点定义异常程度评分：

$$
s_{\text{DBSCAN}}(\mathbf{x}_i) = \min_{j=1,\ldots,K} \text{dist}(\mathbf{x}_i, C_j) \tag{7.73}
$$

其中 $\text{dist}(\mathbf{x}_i, C_j) = \min_{\mathbf{x} \in C_j} \text{dist}(\mathbf{x}_i, \mathbf{x})$ 是 $\mathbf{x}_i$ 到最近簇的距离。
{% endblock %}

DBSCAN 异常检测的独特优势在于：

1. **无需指定 $K$**：簇数由数据密度自动确定；
2. **处理非凸形状**：DBSCAN 可以发现任意形状的簇，不受球形假设限制；
3. **自然的异常标记**：不需要额外的评分函数——噪声点即异常。

### 7.5.3.2 参数选择

DBSCAN 的两个参数 $(\varepsilon, \textit{MinPts})$ 直接影响异常检测结果：

- **$\varepsilon$ 过小**：大部分点被标记为噪声（过多假阳性），只有最密集的核心区域形成簇；
- **$\varepsilon$ 过大**：几乎所有点被归入簇中（过多假阴性），甚至异常点也被"吸收"；
- **$\textit{MinPts}$ 的选择**：通常取 $\textit{MinPts} = 2d$（$d$ 为维度），或 $\textit{MinPts} = \lceil\ln n\rceil$。

{% hint style="tip" %}
💡 **k-距离图选择 $\varepsilon$**

实用的参数选择方法是绘制 $k$-距离图（$k = \textit{MinPts}$）：对每个点计算到第 $k$ 近邻的距离，按升序排列并绘图。图中出现"拐点"（knee point）的位置对应的距离即为合适的 $\varepsilon$——拐点之前的点是簇内点（距离小），拐点之后是噪声/异常点（距离大）。
{% endhint %}

## 7.5.4 CBLOF：聚类与局部密度的融合

### 7.5.4.1 大簇与小簇

He 等人（2003）提出的 CBLOF（Cluster-Based Local Outlier Factor）将聚类信息与局部密度评估结合，对不同大小的簇采用不同的评分策略。

{% block type="definition" title="定义 7.27　CBLOF 异常评分（He et al., 2003）" %}
设聚类结果产生按大小排序的簇 $C_1, C_2, \ldots, C_K$，其中 $|C_1| \geq |C_2| \geq \cdots \geq |C_K|$。将簇分为**大簇集合** $\mathcal{L}$ 和**小簇集合** $\mathcal{S}$：

$$
\mathcal{L} = \{C_j : |C_j| \geq \beta \cdot n / K\}, \quad \mathcal{S} = \{C_j : |C_j| < \beta \cdot n / K\} \tag{7.74}
$$

其中 $\beta \in (0, 1)$ 为比例参数。CBLOF 评分定义为：

$$
\text{CBLOF}(\mathbf{x}_i) = \begin{cases}
|C_{c(i)}| \cdot \|\mathbf{x}_i - \boldsymbol{\mu}_{c(i)}\|, & \text{if } C_{c(i)} \in \mathcal{L} \\
|C_{c(i)}| \cdot \min_{C_j \in \mathcal{L}} \|\mathbf{x}_i - \boldsymbol{\mu}_j\|, & \text{if } C_{c(i)} \in \mathcal{S}
\end{cases} \tag{7.75}
$$
{% endblock %}

CBLOF 的巧妙之处在于它同时利用了**两种信息**：距离（到簇心有多远）和簇大小（所属簇有多小）。对于大簇中的点，评分正比于到簇心的距离乘以簇大小（大簇中远离中心的点获得高分）。对于小簇中的点，评分正比于到最近大簇的距离乘以小簇大小——小簇本身被视为潜在异常，其成员的"异常程度"取决于该小簇离正常簇有多远。

### 7.5.4.2 CBLOF 的统一视角

CBLOF 的设计思想可以从 §7.1 的评分函数框架（定义 7.3）理解：它将"正常"定义为"属于大簇的中心区域"，将"异常"定义为"远离大簇"（无论是大簇内的边缘点还是小簇中的点）。乘以簇大小的操作起到了**加权**作用——大簇中的边缘异常需要更大的距离才能获得高分（因为大簇本身更可信），而小簇的异常只需要较小的距离即可触发。

## 7.5.5 算法描述

{% block type="algorithm" title="算法 7.4　基于聚类的异常检测" %}
**输入**：数据集 $\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$，聚类方法 $\textit{method} \in \{\text{kmeans}, \text{dbscan}, \text{cblof}\}$，参数集 $\Theta$

**输出**：异常评分 $\{s_1, \ldots, s_n\}$ 及异常标签 $\{\hat{y}_1, \ldots, \hat{y}_n\}$

1. **聚类**：
   - 若 $\textit{method} = \text{kmeans}$：运行 K-means 得到 $K$ 个簇和质心
   - 若 $\textit{method} = \text{dbscan}$：运行 DBSCAN 得到簇和噪声集
   - 若 $\textit{method} = \text{cblof}$：运行任意聚类算法，按大小划分大簇/小簇
2. **异常评分**：
   - K-means：$s_i = \|\mathbf{x}_i - \boldsymbol{\mu}_{c(i)}\| / \bar{r}_{c(i)}$
   - DBSCAN：簇内点 $s_i = 0$；噪声点 $s_i = \min_j \text{dist}(\mathbf{x}_i, C_j)$
   - CBLOF：按定义 7.27 计算
3. **阈值与标签**：
   - 按评分降序排列，取 top-$\alpha$ 为异常；或使用自然断点（如 DBSCAN 的噪声标记）
{% endblock %}

### 复杂度分析

| 方法 | 聚类阶段 | 评分阶段 | 总计 |
|------|---------|---------|------|
| K-means | $O(nKd \cdot T)$ | $O(nKd)$ | $O(nKdT)$ |
| DBSCAN | $O(n^2 d)$ / $O(nd\log n)$ | $O(1)$（标签已有） | $O(n^2 d)$ |
| CBLOF | $O(nKd \cdot T)$ | $O(n|\mathcal{L}|d)$ | $O(nKdT)$ |

其中 $T$ 为 K-means 的迭代次数。聚类方法的计算代价通常低于 KNN/LOF 的 $O(n^2d)$——特别是 K-means 对于小 $K$ 值非常高效，适合大规模数据的快速异常检测。

## 7.5.6 与距离/密度方法的对比

{% block type="theorem" title="定理 7.12　聚类方法与 KNN 方法的关系" %}
在以下特殊条件下，K-means 异常评分与 KNN 异常评分渐近等价：

(1) 数据由 $K$ 个等大小、球形高斯簇组成，簇间距离远大于簇内方差；
(2) K-means 正确恢复了簇结构（即每个点被分配到正确的簇）。

此时，设 $\mathbf{x}_i \in C_j$，其 $k$ 近邻 $N_k(\mathbf{x}_i) \subset C_j$（近邻全在同簇内），则：

$$
d_k(\mathbf{x}_i) \approx g(k, |C_j|, \sigma_j) + \|\mathbf{x}_i - \boldsymbol{\mu}_j\| \cdot h(k, d) \tag{7.76}
$$

其中 $g, h$ 是与簇参数和维度相关的函数。KNN 距离随到簇心距离单调递增，故两种评分给出相同的异常排序。
{% endblock %}

{% hint style="proof" %}
**证明要点**：

在球形高斯簇中，距离簇心越远的点，其 $k$ 近邻也越远——因为远离中心的区域密度更低。形式化地，设 $\mathbf{x}_i$ 到簇心的距离为 $r_i = \|\mathbf{x}_i - \boldsymbol{\mu}_j\|$。在 $\mathbf{x}_i$ 附近，数据密度近似为 $\rho(\mathbf{x}_i) \propto \exp(-r_i^2/(2\sigma_j^2))$，$k$ 近邻距离 $d_k(\mathbf{x}_i) \propto \rho(\mathbf{x}_i)^{-1/d}$。因此 $d_k$ 是 $r_i$ 的单调递增函数，两种评分的排序一致。

然而，当簇非球形或近邻跨簇时，这一等价性不再成立——聚类方法使用全局簇结构信息，而 KNN 仅使用局部距离信息，两者可能给出不同的排序。 $\blacksquare$
{% endhint %}

以下表格总结了三类方法的系统对比：

| 维度 | KNN（§7.3） | LOF（§7.4） | 聚类方法（本节） |
|------|------------|------------|---------------|
| **信息利用** | 局部距离 | 局部密度比 | 全局簇结构 + 距离 |
| **异质密度** | ❌ | ✅ | 部分（相对评分） |
| **非凸数据** | ✅ | ✅ | K-means ❌, DBSCAN ✅ |
| **群体异常** | ❌ | ❌ | ✅（小簇检测） |
| **可解释性** | 中等 | 中等 | ✅（"属于哪个簇"） |
| **参数** | $k$ | $k$ | $K$; $\varepsilon, \textit{MinPts}$ |
| **计算代价** | $O(n^2d)$ | $O(n^2d)$ | $O(nKdT)$（更快） |
| **适用场景** | 均匀密度、低维 | 异质密度 | 有簇结构、需可解释 |

{% hint style="frontier" %}
🚀 **前沿视野：深度聚类与异常检测**

传统聚类方法在高维原始数据（如图像、文本）上表现不佳。近年来，**深度聚类**方法将表示学习与聚类目标联合优化，在学习到的低维特征空间中进行聚类。Xie 等人（2016）提出的 DEC（Deep Embedded Clustering）使用自编码器学习特征表示，然后在潜在空间中运行 K-means 式的聚类。将此思路应用于异常检测，可以先用深度聚类在潜在空间中建立簇结构，再使用本节的聚类异常评分——这种方法在图像异常检测（如工业缺陷检测）中取得了优异性能。Deep SVDD（Ruff et al., 2018）可以被视为 $K=1$ 的"深度单簇"异常检测方法，它学习将正常数据映射到一个紧凑的超球内，超球外的点即为异常。
{% endhint %}

{% hint style="ethics" %}
🌏 **课程思政：算法公平性与聚类偏见**

当聚类异常检测应用于社会场景（如信用评分、犯罪预测）时，需要警惕**算法偏见**。如果训练数据中某个少数群体（如少数民族、低收入人群）的行为模式与主流群体不同，聚类算法可能将该群体单独聚为一个"小簇"，进而被异常检测系统标记为"异常"。这种基于统计少数性（statistical minority）的"异常"判定可能导致系统性歧视。我国《个人信息保护法》（2021）第二十四条明确规定，利用个人信息进行自动化决策不得对个人在交易价格等方面实行不合理的差别待遇。数据科学家在部署聚类异常检测时，应当审计"小簇异常"标签是否与受保护特征（性别、民族、年龄）相关，并确保异常判定的公平性。
{% endhint %}

## 案例 7.5：客户行为异常分析

{% block type="case" title="案例 7.5　基于聚类的客户行为异常检测" %}
**问题背景**：某电商平台有数十万注册用户，用户行为特征包括浏览时长、下单频率、退货率、客单价等。运营团队希望识别行为异常的用户——可能是刷单用户（高频低价）、恶意退货者（高退货率）或被盗号用户（行为模式突变）。

**分析目标**：(1) 使用 K-means 聚类将用户分为不同行为群体；(2) 基于聚类距离评分识别各群体中的行为异常用户；(3) 使用 DBSCAN 验证并补充 K-means 的检测结果；(4) 与 KNN/LOF 方法对比检测效果。
{% endblock %}

```python
# ============================================================
# 代码 7.5：基于聚类的异常检测（K-means + DBSCAN + CBLOF）
# 数据集：合成电商用户行为数据（多群体 + 注入异常）
# 依赖库：numpy, sklearn
# ============================================================
# --- 1. 导入与数据生成 ---
import numpy as np
from sklearn.cluster import KMeans, DBSCAN
from sklearn.preprocessing import StandardScaler
from sklearn.neighbors import NearestNeighbors, LocalOutlierFactor
from sklearn.metrics import roc_auc_score
np.random.seed(42)
# 用户群体1：普通消费者（大量，密集）
n1 = 300
X1 = np.column_stack([
    np.random.normal(30, 5, n1),    # 浏览时长(分)
    np.random.normal(3, 1, n1),     # 下单频率(次/月)
    np.random.normal(0.05, 0.02, n1),  # 退货率
    np.random.normal(150, 30, n1)])  # 客单价(元)
# 用户群体2：高价值客户（中量，中等密度）
n2 = 100
X2 = np.column_stack([
    np.random.normal(60, 10, n2),
    np.random.normal(8, 2, n2),
    np.random.normal(0.03, 0.01, n2),
    np.random.normal(500, 80, n2)])
# 用户群体3：偶尔购买者（少量，稀疏）
n3 = 50
X3 = np.column_stack([
    np.random.normal(10, 8, n3),
    np.random.normal(0.5, 0.3, n3),
    np.random.normal(0.10, 0.05, n3),
    np.random.normal(80, 40, n3)])
# 异常用户：刷单（高频低价）+ 恶意退货（高退货率）
n_anom = 20
X_shua = np.column_stack([
    np.random.normal(5, 2, n_anom//2),
    np.random.normal(50, 10, n_anom//2),   # 异常高频
    np.random.normal(0.01, 0.005, n_anom//2),
    np.random.normal(10, 5, n_anom//2)])   # 异常低价
X_refund = np.column_stack([
    np.random.normal(20, 5, n_anom//2),
    np.random.normal(5, 2, n_anom//2),
    np.random.normal(0.70, 0.1, n_anom//2),  # 异常高退货率
    np.random.normal(200, 50, n_anom//2)])
X = np.vstack([X1, X2, X3, X_shua, X_refund])
y_true = np.array([0]*(n1+n2+n3) + [1]*n_anom)
scaler = StandardScaler()
X_std = scaler.fit_transform(X)
# --- 2. K-means 异常检测 ---
km = KMeans(n_clusters=3, random_state=42, n_init=10)
labels_km = km.fit_predict(X_std)
# 相对距离评分
dists_to_center = np.array([np.linalg.norm(X_std[i] - km.cluster_centers_[labels_km[i]])
                            for i in range(len(X_std))])
cluster_radii = np.array([dists_to_center[labels_km == j].mean() for j in range(3)])
scores_km = np.array([dists_to_center[i] / cluster_radii[labels_km[i]]
                       for i in range(len(X_std))])
auc_km = roc_auc_score(y_true, scores_km)
print(f"K-means 相对距离: AUC={auc_km:.4f}")
# --- 3. DBSCAN 异常检测 ---
db = DBSCAN(eps=1.5, min_samples=10)
labels_db = db.fit_predict(X_std)
# 噪声点标记为异常
scores_db = (labels_db == -1).astype(float)
n_noise = (labels_db == -1).sum()
n_clusters = len(set(labels_db)) - (1 if -1 in labels_db else 0)
print(f"DBSCAN: {n_clusters} 个簇, {n_noise} 个噪声点")
# 为了计算AUC，给非噪声点一个小的评分差异
nn_db = NearestNeighbors(n_neighbors=10)
nn_db.fit(X_std)
dists_db, _ = nn_db.kneighbors(X_std)
scores_db_cont = dists_db[:, -1] * (labels_db == -1).astype(float)
scores_db_cont[labels_db != -1] = 0
auc_db = roc_auc_score(y_true, dists_db[:, -1] * (1 + 9*(labels_db == -1)))
print(f"DBSCAN 评分:       AUC={auc_db:.4f}")
# --- 4. 对比：KNN 和 LOF ---
nn = NearestNeighbors(n_neighbors=20)
nn.fit(X_std)
dists_knn, _ = nn.kneighbors(X_std)
auc_knn = roc_auc_score(y_true, dists_knn[:, -1])
print(f"KNN (k=20):        AUC={auc_knn:.4f}")
clf = LocalOutlierFactor(n_neighbors=20, contamination='auto')
clf.fit_predict(X_std)
auc_lof = roc_auc_score(y_true, -clf.negative_outlier_factor_)
print(f"LOF (k=20):        AUC={auc_lof:.4f}")
# --- 5. 方法汇总 ---
print("\n方法对比汇总:")
print(f"  K-means:  AUC={auc_km:.4f}")
print(f"  DBSCAN:   AUC={auc_db:.4f}")
print(f"  KNN:      AUC={auc_knn:.4f}")
print(f"  LOF:      AUC={auc_lof:.4f}")
```

**结果分析**：

1. **K-means**：通过相对距离评分能有效检测大部分异常用户——刷单用户的高频低价特征使其远离所有正常簇的质心，退货异常者的高退货率同样偏离正常模式；
2. **DBSCAN**：自动将异常用户标记为噪声点，无需指定异常比例。其检测效果取决于 $\varepsilon$ 和 MinPts 的选择——参数不当时可能漏检或误报；
3. **与 KNN/LOF 对比**：在此数据上，各方法性能接近。聚类方法的独特优势在于**可解释性**——它能告诉业务方"这个异常用户偏离的是哪个用户群体"，而不仅仅给出一个评分数字。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 两阶段框架（定义 7.23） | 先聚类、再评分——将无序数据组织为有结构的簇，基于簇关系检测异常 |
| 三种异常模式（定义 7.24） | 不属于任何簇、小簇异常、簇边缘异常 |
| K-means 评分（定义 7.25） | $s_{\text{rel}} = \|\mathbf{x} - \boldsymbol{\mu}\| / \bar{r}$，归一化到簇尺度 |
| 球形等价性（定理 7.11） | 球形高斯簇下，K-means 距离评分 $\propto$ 马氏距离 |
| DBSCAN 检测（定义 7.26） | 噪声点 = 异常点，无需额外评分函数 |
| CBLOF（定义 7.27） | $\text{CBLOF} = |C| \cdot d$，同时考虑簇大小与距离 |
| 聚类-KNN 等价性（定理 7.12） | 球形簇下，K-means 距离与 KNN 距离排序一致 |
| 独特优势 | 可检测群体异常（小簇）；高可解释性；计算效率高 |

聚类方法将异常检测从"纯粹的几何分析"提升到了"基于数据结构的语义分析"——它不仅回答"这个点有多远/多稀疏"，还回答"这个点不像哪一类正常模式"。这种语义层面的异常检测在业务场景中尤为重要。

然而，无论是距离方法（§7.3）、密度方法（§7.4）还是聚类方法（本节），都是**无监督**的——它们不利用任何已知的异常标签。下一节（§7.6）将介绍基于分类的方法，它们将异常检测转化为一个（半）监督学习问题，利用已知的正常/异常标签构建判别模型。其中的**隔离森林**（Isolation Forest）和**One-Class SVM** 是两种最具代表性的方法。
