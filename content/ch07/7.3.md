# 7.3 基于距离的方法

上一节的统计方法依赖于"数据服从某一已知分布"的假设——正态分布下有 Z-score 和马氏距离，非参数情况下有 KDE。然而，在许多实际场景中，数据的分布形式完全未知，甚至可能呈现高度不规则的形状（多峰、非凸、流形结构）。此时，统计方法的分布假设成为根本性瓶颈。

**基于距离的方法**提供了一种更具鲁棒性的替代方案。其核心思想简洁而有力：**正常数据点彼此靠近，异常数据点远离大多数正常点**。这一思想不需要任何分布假设——只需要一个距离函数即可度量数据点之间的"远近"。Knorr 和 Ng（1998）首次将这一直觉形式化为 $$\text{DB}(p, D)$$-outlier 定义，标志着异常检测从统计假设检验范式向数据驱动范式的转变。此后，Ramaswamy 等人（2000）和 Angiulli 与 Pizzuti（2002）发展了基于 $$k$$ 近邻距离的异常评分方法，将距离异常检测推向实用。

本节从 Knorr-Ng 的经典定义出发，建立 KNN 异常检测的统一框架，分析距离度量的选择策略，讨论计算加速方法，并严格推导高维距离集中现象对距离方法的根本性挑战。

## 7.3.1 从分布假设到数据驱动

### 7.3.1.1 统计方法的局限性

回顾 §7.2 的方法框架：Z-score 和 Grubbs 检验假设正态分布，马氏距离假设多元正态分布，即使 KDE 也需要选择核函数和带宽。当数据呈现以下特征时，这些假设可能严重失效：

1. **多模态分布**：数据由多个簇组成，每个簇有自己的中心和形状。此时，单一正态模型的均值和协方差无法刻画数据的真实结构；
2. **非凸结构**：数据分布呈现弯曲的流形形态（如半月形、螺旋形），不符合任何简单的参数分布；
3. **异质密度**：不同区域的数据密度差异悬殊——稀疏区域的正常点可能被统计方法误判为异常。

基于距离的方法避开了这些问题。它们不试图估计全局分布 $$p(\mathbf{x})$$，而是直接利用数据点之间的**局部距离关系**来判断异常。其哲学基础是：

$$
\text{正常} \iff \text{有近邻} \quad\quad \text{异常} \iff \text{没有近邻}  \tag{7.31}
$$

这一思路是纯数据驱动的——只需要一个距离函数 $$d(\cdot, \cdot)$$ 和数据集本身，不需要任何参数化的分布假设。从 §7.1 的评分函数视角看，距离方法用**到近邻的距离**作为概率密度的代理量：距离大 $$\approx$$ 密度低 $$\approx$$ 可能异常。下面将这一直觉严格形式化。

## 7.3.2 基于距离的异常定义

### 7.3.2.1 DB(p, D)-outlier

Knorr 和 Ng（1998）给出了第一个不依赖分布假设的异常定义，它只使用两个直观参数：距离阈值 $$D$$ 和比例参数 $$p$$。

{% block type="definition" title="定义 7.14　DB(p, D)-outlier（Knorr & Ng, 1998）" %}
设数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\} \subset \mathbb{R}^d$$，距离函数 $$\text{dist}: \mathbb{R}^d \times \mathbb{R}^d \to \mathbb{R}_{\geq 0}$$。对于参数 $$p \in (0, 1)$$ 和 $$D > 0$$，数据点 $$\mathbf{x}_i$$ 是 $$\text{DB}(p, D)$$-outlier，当且仅当数据集中至少有比例 $$p$$ 的点与 $$\mathbf{x}_i$$ 的距离超过 $$D$$：

$$
\frac{|\{\mathbf{x}_j \in \mathcal{D} : \text{dist}(\mathbf{x}_i, \mathbf{x}_j) > D\}|}{n} \geq p \tag{7.32}
$$

等价地，$$\mathbf{x}_i$$ 的 $$D$$-邻域内点的数量不超过 $$n(1 - p)$$：

$$
|N_D(\mathbf{x}_i)| \leq n(1 - p), \quad N_D(\mathbf{x}_i) = \{\mathbf{x}_j \in \mathcal{D} : \text{dist}(\mathbf{x}_i, \mathbf{x}_j) \leq D\} \tag{7.33}
$$
{% endblock %}

定义 7.14 的直觉非常清晰：如果一个点的"邻居"太少（$$D$$-邻域内的点不超过 $$n(1-p)$$ 个），那么它就是异常。参数 $$p$$ 控制"太少"的阈值——例如，$$p = 0.98$$ 表示至少 98% 的点不在其邻域内，只有不超过 2% 的点是其邻居。

{% hint style="caution" %}
⚠️ **DB(p, D) 的参数选择困难**

DB 定义虽然优雅，但在实践中面临严重的参数选择问题：$$D$$ 的最优值高度依赖数据的尺度和密度分布，而 $$p$$ 的选择需要对异常比例的先验估计。更关键的是，**固定的 $$D$$ 无法适应异质密度**——在密集区域，$$D$$ 可能过大（漏检）；在稀疏区域，$$D$$ 可能过小（误报）。这一局限性催生了基于 $$k$$ 近邻距离的改进方法。
{% endhint %}

### 7.3.2.2 从固定阈值到自适应排序

DB 定义使用固定距离阈值 $$D$$ 做二值判断（是/否异常），无法生成**连续的异常评分**。Ramaswamy 等人（2000）提出了关键改进：用 $$k$$ 近邻距离作为连续评分函数，将异常检测从二值判断转化为排序问题。这一改进的优势在于：（1）不需要预设 $$D$$；（2）生成的评分可直接用于 ROC/PR 评估（§7.1）；（3）参数 $$k$$ 的含义比 $$D$$ 更直观且更稳定。

## 7.3.3 KNN 异常检测

### 7.3.3.1 k 近邻距离

{% block type="definition" title="定义 7.15　k 近邻距离（k-th Nearest Neighbor Distance）" %}
设数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$$，距离函数 $$\text{dist}(\cdot, \cdot)$$，正整数 $$k \leq n - 1$$。数据点 $$\mathbf{x}_i$$ 的 **$$k$$ 近邻集合** $$N_k(\mathbf{x}_i)$$ 是 $$\mathcal{D} \setminus \{\mathbf{x}_i\}$$ 中距离 $$\mathbf{x}_i$$ 最近的 $$k$$ 个点。

$$\mathbf{x}_i$$ 的 **$$k$$ 近邻距离**定义为到第 $$k$$ 个最近邻的距离：

$$
d_k(\mathbf{x}_i) = \max_{\mathbf{x}_j \in N_k(\mathbf{x}_i)} \text{dist}(\mathbf{x}_i, \mathbf{x}_j) \tag{7.34}
$$

即 $$d_k(\mathbf{x}_i)$$ 是以 $$\mathbf{x}_i$$ 为中心、恰好包含 $$k$$ 个点的最小球的半径。等价地，将 $$\mathcal{D} \setminus \{\mathbf{x}_i\}$$ 中的点按到 $$\mathbf{x}_i$$ 的距离升序排列为 $$\mathbf{x}_{(1)}, \mathbf{x}_{(2)}, \ldots, \mathbf{x}_{(n-1)}$$，则：

$$
d_k(\mathbf{x}_i) = \text{dist}(\mathbf{x}_i, \mathbf{x}_{(k)}) \tag{7.35}
$$
{% endblock %}

$$k$$ 近邻距离有一个直观的几何解释：想象在每个数据点 $$\mathbf{x}_i$$ 处放一个气球，不断膨胀直到恰好包含 $$k$$ 个其他点。正常点位于密集区域，气球很小就能包含 $$k$$ 个邻居；异常点远离密集区域，气球必须膨胀很大。因此，气球的半径 $$d_k(\mathbf{x}_i)$$ 自然地度量了 $$\mathbf{x}_i$$ 的"孤立程度"。

### 7.3.3.2 KNN 异常评分函数

基于 $$k$$ 近邻距离，文献中发展了多种异常评分函数，它们在对近邻信息的利用方式上有所不同。

{% block type="definition" title="定义 7.16　KNN 异常评分函数" %}
设 $$N_k(\mathbf{x}_i) = \{\mathbf{x}_{(1)}, \ldots, \mathbf{x}_{(k)}\}$$ 为 $$\mathbf{x}_i$$ 的 $$k$$ 近邻集合（按距离升序），定义三种异常评分函数：

**（1）最大距离评分**（Ramaswamy et al., 2000）——仅使用第 $$k$$ 个近邻的距离：

$$
s_{\max}(\mathbf{x}_i) = d_k(\mathbf{x}_i) = \text{dist}(\mathbf{x}_i, \mathbf{x}_{(k)}) \tag{7.36}
$$

**（2）平均距离评分**（Angiulli & Pizzuti, 2002）——使用所有 $$k$$ 个近邻距离的均值：

$$
s_{\text{avg}}(\mathbf{x}_i) = \frac{1}{k}\sum_{j=1}^{k} \text{dist}(\mathbf{x}_i, \mathbf{x}_{(j)}) \tag{7.37}
$$

**（3）加权距离评分**——赋予较远近邻更大权重（指数衰减）：

$$
s_w(\mathbf{x}_i) = \sum_{j=1}^{k} \frac{1}{j} \cdot \text{dist}(\mathbf{x}_i, \mathbf{x}_{(j)}) \tag{7.38}
$$

三种评分函数的异同：$$s_{\max}$$ 最简单但对 $$k$$ 值敏感；$$s_{\text{avg}}$$ 通过平均化减少了单个近邻的影响，更稳定；$$s_w$$ 兼顾近邻和远邻的信息，但引入了额外的权重设计。
{% endblock %}

以下定理揭示了 DB 异常定义与 KNN 评分之间的内在联系。

{% block type="theorem" title="定理 7.6　DB(p, D)-outlier 与 KNN 评分的等价性" %}
设 $$k = \lfloor n(1 - p) \rfloor$$，则 $$\mathbf{x}_i$$ 是 $$\text{DB}(p, D)$$-outlier 当且仅当其 $$k$$ 近邻距离超过 $$D$$：

$$
\mathbf{x}_i \text{ 是 DB}(p, D)\text{-outlier} \iff d_k(\mathbf{x}_i) > D \tag{7.39}
$$

即 DB 异常检测等价于对 KNN 距离施加阈值 $$D$$。
{% endblock %}

{% hint style="proof" %}
**证明**：

由定义 7.14，$$\mathbf{x}_i$$ 是 $$\text{DB}(p, D)$$-outlier 当且仅当 $$|N_D(\mathbf{x}_i)| \leq n(1 - p)$$。

设 $$k = \lfloor n(1 - p) \rfloor$$。将 $$\mathcal{D} \setminus \{\mathbf{x}_i\}$$ 中的点按距离升序排列。

$$(\Rightarrow)$$ 若 $$|N_D(\mathbf{x}_i)| \leq k$$，则距离不超过 $$D$$ 的点至多 $$k$$ 个。因此第 $$k$$ 个最近邻的距离 $$d_k(\mathbf{x}_i) > D$$（否则 $$D$$-邻域内至少有 $$k$$ 个点，矛盾）。

$$(\Leftarrow)$$ 若 $$d_k(\mathbf{x}_i) > D$$，则在距离不超过 $$D$$ 的范围内，点的数量不足 $$k$$ 个（因为第 $$k$$ 个最近邻已在 $$D$$ 之外）。故 $$|N_D(\mathbf{x}_i)| < k \leq n(1-p)$$，即 $$\mathbf{x}_i$$ 是 $$\text{DB}(p, D)$$-outlier。 $$\blacksquare$$
{% endhint %}

定理 7.6 的意义在于：DB 定义中令人困扰的参数 $$D$$ 变成了 KNN 评分上的阈值，而参数 $$p$$ 则自然转化为 $$k$$ 值的选择（$$k \approx n(1-p)$$）。在实践中，$$k$$ 比 $$(p, D)$$ 组合更容易调节——通常取 $$k \in [5, 50]$$ 即可获得良好效果。

### 7.3.3.3 参数 k 的选择

$$k$$ 值的选择对检测结果有显著影响：

- **$$k$$ 过小**（如 $$k = 1$$）：评分 $$d_1(\mathbf{x}_i)$$ 仅取决于最近的一个邻居，容易受到噪声干扰——一个随机靠近异常点的正常点可能使异常点的评分偏低（掩蔽效应的距离版本）；
- **$$k$$ 过大**（如 $$k > n/2$$）：评分趋向于到数据集中心的平均距离，丧失了局部性——无法区分不同密度区域中的异常。

{% hint style="tip" %}
💡 **k 值选择的实用准则**

1. **经验法则**：$$k = \lfloor\sqrt{n}\rfloor$$ 是常用的起始值，在样本量 $$n \in [100, 10000]$$ 的范围内通常表现良好；
2. **稳定性检验**：在 $$k \in [k_0/2,\, 2k_0]$$ 范围内变化 $$k$$，若异常排名前 $$m$$ 的点变化不大，则结果稳定；
3. **多 k 融合**：计算多个 $$k$$ 值下的评分，取平均或取最大值，可提高鲁棒性。
{% endhint %}

### 7.3.3.4 算法描述

{% block type="algorithm" title="算法 7.2　KNN 异常检测" %}
**输入**：数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$$，近邻数 $$k$$，异常比例 $$\alpha$$（或阈值 $$\tau$$），距离函数 $$\text{dist}$$，评分类型 $$\textit{type} \in \{\max, \text{avg}\}$$

**输出**：异常评分 $$\{s_1, \ldots, s_n\}$$ 及异常标签 $$\{\hat{y}_1, \ldots, \hat{y}_n\}$$

1. **构建索引**（可选）：对 $$\mathcal{D}$$ 构建 KD-tree 或 Ball-tree 空间索引
2. **KNN 查询**：对每个 $$\mathbf{x}_i \in \mathcal{D}$$，查询其 $$k$$ 个最近邻 $$N_k(\mathbf{x}_i)$$ 及对应距离
3. **异常评分**：对每个 $$\mathbf{x}_i$$，计算评分：
   - 若 $$\textit{type} = \max$$：$$s_i = d_k(\mathbf{x}_i)$$
   - 若 $$\textit{type} = \text{avg}$$：$$s_i = \frac{1}{k}\sum_{j=1}^{k} \text{dist}(\mathbf{x}_i, \mathbf{x}_{(j)})$$
4. **阈值确定**：
   - 若给定 $$\alpha$$：$$\tau = s_{(\lceil n(1-\alpha) \rceil)}$$（第 $$\lceil n(1-\alpha) \rceil$$ 大的评分）
   - 若给定 $$\tau$$：直接使用
5. **标签分配**：$$\hat{y}_i = \mathbb{1}[s_i > \tau]$$
{% endblock %}

算法 7.2 的时间复杂度取决于 KNN 查询的实现。朴素方法需要对每个点计算到所有其他点的距离，总复杂度为 $$O(n^2 d)$$。使用空间索引可以显著加速——但加速效果与数据维度密切相关，这将在 §7.3.5 中详细讨论。

{% block type="example" title="例 7.3　KNN 异常评分的计算" %}
考虑一维数据集 $$\mathcal{D} = \{1, 2, 3, 4, 5, 20\}$$，取 $$k = 2$$，使用欧氏距离。

对于每个点，找其 $$k = 2$$ 个最近邻并计算评分：

| 点 $$x_i$$ | 2 近邻 | $$d_1$$ | $$d_2$$ | $$s_{\max} = d_2$$ | $$s_{\text{avg}} = (d_1+d_2)/2$$ |
|----------|--------|-------|-------|------------------|-------------------------------|
| 1 | \{2, 3\} | 1 | 2 | 2 | 1.5 |
| 2 | \{1, 3\} | 1 | 1 | 1 | 1.0 |
| 3 | \{2, 4\} | 1 | 1 | 1 | 1.0 |
| 4 | \{3, 5\} | 1 | 1 | 1 | 1.0 |
| 5 | \{4, 3\} | 1 | 2 | 2 | 1.5 |
| 20 | \{5, 4\} | 15 | 16 | **16** | **15.5** |

点 $$x = 20$$ 的 KNN 评分远高于其他点——无论使用 $$s_{\max}$$ 还是 $$s_{\text{avg}}$$，它都被明确识别为异常。注意边界点（$$x=1, x=5$$）的评分略高于内部点，但远低于真正的异常。

## 7.3.4 距离度量的选择

### 7.3.4.1 Minkowski 距离族

KNN 异常检测的效果高度依赖距离函数的选择。最常用的距离函数族是 Minkowski 距离。

{% block type="definition" title="定义 7.17　Minkowski 距离族" %}
对于 $$\mathbf{x}, \mathbf{y} \in \mathbb{R}^d$$ 和参数 $$p \geq 1$$，**Minkowski 距离**定义为：

$$
d_p(\mathbf{x}, \mathbf{y}) = \left(\sum_{i=1}^{d} |x_i - y_i|^p\right)^{1/p} \tag{7.40}
$$

特殊情形：
- $$p = 1$$：**曼哈顿距离**（Manhattan / $$L_1$$），$$d_1(\mathbf{x}, \mathbf{y}) = \sum_{i=1}^{d} |x_i - y_i|$$
- $$p = 2$$：**欧氏距离**（Euclidean / $$L_2$$），$$d_2(\mathbf{x}, \mathbf{y}) = \sqrt{\sum_{i=1}^{d} (x_i - y_i)^2}$$
- $$p \to \infty$$：**切比雪夫距离**（Chebyshev / $$L_\infty$$），$$d_\infty(\mathbf{x}, \mathbf{y}) = \max_{1 \leq i \leq d} |x_i - y_i|$$

其等距线（即 $$\{\mathbf{y}: d_p(\mathbf{x}, \mathbf{y}) = r\}$$）的几何形状随 $$p$$ 变化：$$L_1$$ 为菱形，$$L_2$$ 为圆形，$$L_\infty$$ 为方形。
{% endblock %}

### 7.3.4.2 度量选择的实践指南

不同距离度量适用于不同的数据特征：

| 度量 | 等距线 | 特点 | 适用场景 |
|------|--------|------|---------|
| $$L_1$$（曼哈顿） | 菱形 | 对单个维度的极端值敏感 | 稀疏高维数据、特征独立性强 |
| $$L_2$$（欧氏） | 圆形 | 各方向等权，直觉自然 | 低中维连续数据、各维度可比 |
| $$L_\infty$$（切比雪夫） | 方形 | 仅由最大偏差决定 | 需检测任一维度极端偏离时 |

{% hint style="caution" %}
⚠️ **距离度量必须与数据尺度匹配**

若特征未经标准化，欧氏距离会被大尺度特征主导。例如，"年龄"（0–100）和"收入"（0–1,000,000）之间的欧氏距离几乎完全由收入决定。**在使用任何距离方法前，必须对特征进行标准化处理**（如 Z-score 标准化或 min-max 归一化），使每个维度对距离的贡献大致相等。
{% endhint %}

{% hint style="think" %}
🤔 **思考：距离度量与异常类型的匹配**

考虑一个二维数据集，正常数据沿 $$x_1 = x_2$$ 的对角线分布。异常点 A 位于 $$(5, 0)$$，异常点 B 位于 $$(3, 3.5)$$。在 $$L_2$$ 下，A 可能比 B 更显著（离对角线更远）。但在 $$L_\infty$$ 下呢？在 $$L_1$$ 下呢？读者可自行验证：度量的选择不仅影响异常的**评分**，还可能改变异常的**排序**。这意味着没有"最优"度量——选择应基于领域知识和异常类型。
{% endhint %}

## 7.3.5 计算加速

### 7.3.5.1 朴素方法的瓶颈

算法 7.2 的核心开销在于 KNN 查询。朴素实现需要对每个点 $$\mathbf{x}_i$$ 计算到所有其他 $$n - 1$$ 个点的距离，总时间复杂度为 $$O(n^2 d)$$。对于大规模数据（$$n > 10^5$$），这一复杂度难以承受。空间索引结构通过预处理组织数据，使 KNN 查询避免遍历所有点。

### 7.3.5.2 KD-tree

KD-tree（$$k$$-dimensional tree）是一种二叉空间划分树：每个内部节点沿一个坐标维度将空间一分为二，叶节点存储少量数据点。查询时，从根节点出发，利用剪枝策略跳过不可能包含最近邻的子树。

{% block type="theorem" title="定理 7.7　KD-tree 的查询复杂度" %}
对于 $$n$$ 个 $$d$$ 维数据点：
- **构建时间**：$$O(n d \log n)$$
- **单次 $$k$$-NN 查询**的期望时间复杂度：$$O(k \cdot d \cdot \log n)$$（当 $$d$$ 较小时）
- **总复杂度**（$$n$$ 次查询）：$$O(k \cdot n \cdot d \cdot \log n)$$

然而，KD-tree 的性能随维度 $$d$$ 迅速退化。当 $$d > 20$$ 时，剪枝效果急剧下降，查询时间退化为 $$O(n d)$$——与朴素方法无异。
{% endblock %}

{% hint style="proof" %}
**证明要点**：

KD-tree 的构建通过递归划分实现：每层选择一个维度和中位数作为切分点，将数据均分为两半。树的深度为 $$O(\log n)$$，每层需要 $$O(nd)$$ 的中位数选择和划分操作，故构建时间为 $$O(nd \log n)$$。

查询时，最坏情况下需要访问所有节点（$$O(n)$$），但在低维且数据分布均匀时，剪枝策略使平均访问节点数为 $$O(\log n)$$。设 $$B(d)$$ 为 $$d$$ 维空间中需要回溯检查的子树比例。Friedman 等人（1977）证明，当数据均匀分布时，$$B(d) \approx 2d/n^{1/d}$$。当 $$d$$ 较大时，$$n^{1/d} \to 1$$，故 $$B(d) \to 2d$$，此时几乎所有子树都需要检查。这解释了 KD-tree 在高维下退化的原因。 $$\blacksquare$$
{% endhint %}

### 7.3.5.3 Ball-tree

Ball-tree 使用超球体（而非超矩形）划分空间，在高维下通常比 KD-tree 更稳健。每个节点定义一个包含其所有子点的最小包围球，查询时利用三角不等式进行剪枝。Ball-tree 的构建时间为 $$O(nd\log n)$$，查询复杂度在高维下通常优于 KD-tree，但仍会随维度增长而退化。

### 7.3.5.4 近似方法

当精确 KNN 查询的成本过高时，可使用近似最近邻（Approximate Nearest Neighbor, ANN）方法：

| 方法 | 时间复杂度 | 精度保证 | 代表实现 |
|------|----------|---------|---------|
| 局部敏感哈希（LSH） | $$O(n^{1-1/c} \cdot d)$$ | $$(c, r)$$-近似 | `falconn` |
| 随机投影树 | $$O(d \log n)$$ 每次查询 | 概率性 | `annoy` |
| 图索引 (HNSW) | $$O(d \log n)$$ 每次查询 | 经验性高精度 | `hnswlib`, `faiss` |

对于异常检测的特定场景，近似 KNN 的精度损失通常可接受——因为我们关注的是异常**排名**而非精确距离值。下一节将讨论一个更根本的问题：在高维空间中，距离本身可能丧失区分能力。

## 7.3.6 高维距离集中现象

### 7.3.6.1 距离失效的根源

在 §5.7（高维聚类的挑战）中，我们已初步讨论了维度诅咒。对于基于距离的异常检测，高维空间带来一个尤为致命的问题：**所有点对之间的距离趋于相等**——最近邻和最远邻的距离几乎一样。这意味着"距离"丧失了区分正常与异常的能力。

{% block type="definition" title="定义 7.18　相对对比度（Relative Contrast）" %}
设 $$\mathcal{D} \subset \mathbb{R}^d$$ 为数据集，$$d_{\max}$$ 和 $$d_{\min}$$ 分别为最大和最小成对距离：

$$
d_{\max} = \max_{i \neq j} \text{dist}(\mathbf{x}_i, \mathbf{x}_j), \quad d_{\min} = \min_{i \neq j} \text{dist}(\mathbf{x}_i, \mathbf{x}_j) \tag{7.41}
$$

**相对对比度**定义为：

$$
\text{RC}(\mathcal{D}) = \frac{d_{\max} - d_{\min}}{d_{\min}} \tag{7.42}
$$

当 $$\text{RC} \to 0$$ 时，最近距离与最远距离趋于相等，距离丧失区分能力。
{% endblock %}

以下定理严格刻画了高维下距离集中的速度。

{% block type="theorem" title="定理 7.8　高维距离集中定理（Beyer et al., 1999）" %}
设 $$\mathbf{x}_1, \mathbf{x}_2, \ldots$$ 为独立同分布的随机向量序列，$$\mathbf{x}_i \in \mathbb{R}^d$$，各分量独立且具有有限的四阶矩。设 $$D_d = \|\mathbf{x}\|_2 = \sqrt{\sum_{i=1}^{d} x_i^2}$$ 为到原点的欧氏距离。若各分量 $$x_i$$ 满足 $$\operatorname{Var}(x_i^2) = \sigma^4 < \infty$$ 且 $$E[x_i^2] = \mu_2 > 0$$，则：

$$
\frac{\operatorname{Var}(D_d^2)}{(E[D_d^2])^2} = \frac{d \cdot \sigma^4}{(d \cdot \mu_2)^2} = \frac{\sigma^4}{d \cdot \mu_2^2} \xrightarrow{d \to \infty} 0 \tag{7.43}
$$

即 $$D_d^2$$ 的相对方差以 $$O(1/d)$$ 的速度趋于零。

由 Chebyshev 不等式，这意味着：

$$
\frac{D_d^2}{E[D_d^2]} \xrightarrow{P} 1 \quad (d \to \infty) \tag{7.44}
$$

所有点到原点的距离（因而所有成对距离）趋于相同，相对对比度 $$\text{RC} \xrightarrow{P} 0$$。
{% endblock %}

{% hint style="proof" %}
**证明**：

设 $$Y_i = x_i^2$$，则 $$D_d^2 = \sum_{i=1}^{d} Y_i$$。由于各 $$Y_i$$ 独立同分布，

$$
E[D_d^2] = d \cdot E[Y_i] = d \cdot \mu_2 \tag{7.45}
$$

$$
\operatorname{Var}(D_d^2) = d \cdot \operatorname{Var}(Y_i) = d \cdot \sigma^4 \tag{7.46}
$$

因此：

$$
\frac{\operatorname{Var}(D_d^2)}{(E[D_d^2])^2} = \frac{d \cdot \sigma^4}{d^2 \cdot \mu_2^2} = \frac{\sigma^4}{d \cdot \mu_2^2} \tag{7.47}
$$

当 $$d \to \infty$$ 时，上式 $$\to 0$$。由 Chebyshev 不等式：

$$
P\left(\left|\frac{D_d^2}{E[D_d^2]} - 1\right| > \epsilon\right) \leq \frac{\operatorname{Var}(D_d^2)}{\epsilon^2 (E[D_d^2])^2} = \frac{\sigma^4}{\epsilon^2 \cdot d \cdot \mu_2^2} \to 0 \tag{7.48}
$$

故 $$D_d^2 / E[D_d^2] \xrightarrow{P} 1$$。

对于两点间的距离 $$\|\mathbf{x}_i - \mathbf{x}_j\|^2$$，由于 $$\mathbf{x}_i - \mathbf{x}_j$$ 的各分量仍满足独立、有限四阶矩的条件，同样的论证成立。因此，所有成对距离的平方都集中在同一常数附近，$$\text{RC} \xrightarrow{P} 0$$。 $$\blacksquare$$
{% endhint %}

图 7.3.3 通过实验直观展示了距离集中现象：随着维度增加，成对距离的分布变得越来越集中，相对对比度趋于零。

### 7.3.6.2 应对策略

面对高维距离集中，实践中有以下应对策略：

1. **降维预处理**：先使用 PCA、t-SNE 或 UMAP 将数据降至 $$d' \leq 20$$ 维，再应用 KNN 异常检测。降维可能损失信息，但保留了距离的区分能力；
2. **子空间方法**：不在全空间中计算距离，而是在多个低维子空间中分别检测，然后融合结果（如 Feature Bagging，Lazarevic & Kumar, 2005）；
3. **切换到非距离方法**：当 $$d > 50$$ 时，考虑使用隔离森林（§7.6）或基于密度的方法（§7.4 的 LOF 在中等维度下仍然有效），它们对距离集中的敏感度较低。

高维下距离方法的局限性并不否定其价值——对于低中维数据（$$d \leq 20$$），KNN 异常检测仍然是最简单、最可靠的方法之一。关键是理解方法的适用边界，在合适的场景中选择合适的工具。

{% hint style="frontier" %}
🚀 **前沿视野：自适应距离与度量学习**

近年来，**度量学习**（Metric Learning）为距离方法注入了新的活力。其核心思想是：不使用固定的欧氏距离，而是从数据中**学习一个最优距离函数**。具体地，学习一个正定矩阵 $$\mathbf{M}$$，使得马氏距离 $$d_\mathbf{M}(\mathbf{x}, \mathbf{y}) = \sqrt{(\mathbf{x}-\mathbf{y})^\top \mathbf{M} (\mathbf{x}-\mathbf{y})}$$ 能更好地区分正常与异常。Pang 等人（2019）提出的 DevNet（Deep Anomaly Detection Network）和 Ruff 等人（2018）提出的 Deep SVDD 本质上都在学习一个非线性距离映射，将正常数据映射到紧凑区域，使异常数据在学习后的空间中"距离更远"。这些方法将距离异常检测从固定度量推向了可学习度量的新阶段。
{% endhint %}

## 案例 7.3：网络入侵检测

{% block type="case" title="案例 7.3　基于 KNN 距离的网络入侵检测" %}
**问题背景**：某企业网络每天产生大量连接记录，每条记录包含多个特征（持续时间、协议类型、传输字节数、错误率等）。正常连接占绝大多数，但少量恶意连接（端口扫描、DDoS 攻击等）隐藏其中。目标是使用 KNN 距离方法自动识别异常连接。

**分析目标**：(1) 构建 KNN 异常检测器，对比不同 $$k$$ 值和评分函数的效果；(2) 分析距离方法在多维网络数据上的表现；(3) 比较 KNN 方法与 §7.2 中统计方法的检测性能。
{% endblock %}

```python
# ============================================================
# 代码 7.3：KNN 异常检测（从零实现 vs sklearn 对比）
# 数据集：合成网络连接数据（多维正态 + 注入异常模式）
# 依赖库：numpy, scipy, sklearn
# ============================================================
# --- 1. 导入与数据生成 ---
import numpy as np
from scipy.spatial.distance import cdist
from sklearn.neighbors import NearestNeighbors
from sklearn.metrics import roc_auc_score
np.random.seed(42)
n_normal, n_anomaly, d = 800, 40, 6
features = ['持续时间', '字节数', '包数', '错误率', '服务数', '连接数']
# 正常连接：6维多元正态
mu_normal = np.array([100, 5000, 50, 0.01, 3, 10])
cov = np.eye(d) * np.array([20, 1000, 10, 0.005, 1, 3])**2
X_normal = np.random.multivariate_normal(mu_normal, cov, n_normal)
# 异常连接类型1：端口扫描（高连接数、低字节数、高服务数）
X_anom1 = np.random.multivariate_normal(mu_normal, cov, n_anomaly // 2)
X_anom1[:, 5] += 5 * np.sqrt(cov[5, 5])   # 连接数异常高
X_anom1[:, 1] -= 3 * np.sqrt(cov[1, 1])   # 字节数异常低
X_anom1[:, 4] += 4 * np.sqrt(cov[4, 4])   # 服务数异常高
# 异常连接类型2：DDoS（极高包数、短持续时间）
X_anom2 = np.random.multivariate_normal(mu_normal, cov, n_anomaly // 2)
X_anom2[:, 2] += 6 * np.sqrt(cov[2, 2])   # 包数异常高
X_anom2[:, 0] -= 3 * np.sqrt(cov[0, 0])   # 持续时间异常短
X = np.vstack([X_normal, X_anom1, X_anom2])
y_true = np.array([0]*n_normal + [1]*(n_anomaly))
# 标准化（距离方法的必要预处理）
X_std = (X - X.mean(axis=0)) / X.std(axis=0)
# --- 2. 从零实现 KNN 异常检测 ---
def knn_anomaly_score(X, k, score_type='max'):
    """计算每个点的 KNN 异常评分（从零实现）"""
    n = len(X)
    dist_matrix = cdist(X, X, metric='euclidean')
    np.fill_diagonal(dist_matrix, np.inf)  # 排除自身
    scores = np.zeros(n)
    for i in range(n):
        sorted_dists = np.sort(dist_matrix[i])[:k]
        if score_type == 'max':
            scores[i] = sorted_dists[-1]       # 第k近邻距离
        elif score_type == 'avg':
            scores[i] = sorted_dists.mean()    # 平均近邻距离
    return scores
# --- 3. 不同 k 值的对比 ---
for k in [5, 10, 20, 50]:
    scores = knn_anomaly_score(X_std, k, 'max')
    auc = roc_auc_score(y_true, scores)
    print(f"k={k:2d}, score=max:  AUC={auc:.4f}")
# --- 4. 不同评分函数的对比 ---
k_best = 10
for st in ['max', 'avg']:
    scores = knn_anomaly_score(X_std, k_best, st)
    auc = roc_auc_score(y_true, scores)
    print(f"k={k_best}, score={st:3s}: AUC={auc:.4f}")
# --- 5. sklearn 验证 ---
nn = NearestNeighbors(n_neighbors=k_best, metric='euclidean')
nn.fit(X_std)
dists, _ = nn.kneighbors(X_std)
scores_sk = dists[:, -1]  # 第 k 近邻距离
auc_sk = roc_auc_score(y_true, scores_sk)
print(f"sklearn 验证: AUC={auc_sk:.4f}")
# --- 6. 与马氏距离对比 ---
from scipy import stats
mu_hat = X_std.mean(axis=0)
S_inv = np.linalg.inv(np.cov(X_std.T))
D_M2 = np.array([((x - mu_hat) @ S_inv @ (x - mu_hat)) for x in X_std])
auc_maha = roc_auc_score(y_true, D_M2)
print(f"马氏距离对比: AUC={auc_maha:.4f}")
```

**结果分析**：

1. **k 值影响**：$$k$$ 在 $$[5, 50]$$ 范围内，AUC 变化通常不超过 0.02，说明 KNN 方法对 $$k$$ 的选择具有一定鲁棒性。过小的 $$k$$（如 $$k=1$$）可能受噪声干扰，过大的 $$k$$（如 $$k=100$$）丧失局部性；
2. **评分函数影响**：$$s_{\text{avg}}$$ 通常比 $$s_{\max}$$ 略优——平均化减少了单个近邻的随机波动；
3. **与马氏距离对比**：两种方法的 AUC 接近，但 KNN 方法**不需要正态性假设**。当数据分布不规则（如多簇结构）时，KNN 的优势更明显。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| DB(p, D)-outlier（定义 7.14） | $$D$$-邻域内点数 $$\leq n(1-p)$$，首个不依赖分布假设的异常定义 |
| $$k$$ 近邻距离（定义 7.15） | $$d_k(\mathbf{x}_i)$$：以 $$\mathbf{x}_i$$ 为中心恰好包含 $$k$$ 个点的最小球半径 |
| KNN 评分函数（定义 7.16） | 最大距离 $$s_{\max}$$、平均距离 $$s_{\text{avg}}$$、加权距离 $$s_w$$ |
| DB-KNN 等价性（定理 7.6） | DB(p,D)-outlier $$\iff$$ $$d_k > D$$，其中 $$k = \lfloor n(1-p) \rfloor$$ |
| Minkowski 距离（定义 7.17） | $$d_p(\mathbf{x}, \mathbf{y}) = (\sum|x_i-y_i|^p)^{1/p}$$，$$L_1$$/$$L_2$$/$$L_\infty$$ |
| KD-tree 复杂度（定理 7.7） | 查询 $$O(k \cdot d \cdot \log n)$$（低维），高维退化至 $$O(nd)$$ |
| 相对对比度（定义 7.18） | $$\text{RC} = (d_{\max} - d_{\min})/d_{\min}$$，度量距离的区分能力 |
| 距离集中（定理 7.8） | $$\text{Var}(D_d^2)/E[D_d^2]^2 = O(1/d)$$，高维下所有距离趋于相等 |

基于距离的方法以其**简单性和通用性**成为异常检测的基础工具——不需要分布假设，不需要训练过程，只需计算距离即可。其核心局限是全局性：对于异质密度的数据，一个远离稀疏簇的正常点可能比密集簇中的异常点有更大的 KNN 距离。下一节（§7.4）介绍的局部离群因子（LOF）正是针对这一局限性提出的——它不直接用距离评分，而是将距离**相对于局部邻域密度进行归一化**，从而实现对异质密度的自适应。

**B2.** 对于 $$n$$ 个独立同分布于 $$\mathcal{N}(0, \mathbf{I}_d)$$ 的点，证明成对欧氏距离的期望为 $$E[\|\mathbf{x}_i - \mathbf{x}_j\|^2] = 2d$$，方差为 $$\operatorname{Var}(\|\mathbf{x}_i - \mathbf{x}_j\|^2) = 8d$$。由此推导相对对比度的收敛速率。

**B3.** 证明 Minkowski 距离满足三角不等式（对任意 $$p \geq 1$$）。（提示：使用 Minkowski 不等式。）

**B4.** 设数据分布为均匀分布 $$\text{Uniform}([0,1]^d)$$，推导 $$k$$ 近邻距离 $$d_k(\mathbf{x})$$ 的期望值与 $$n, k, d$$ 的关系。（提示：利用 $$k$$ 阶统计量的分布。）

### C 级（数值计算）

**C1.** 对一维数据集 $$\{1, 3, 5, 7, 9, 50\}$$，手工计算每个点在 $$k = 2$$ 时的 $$s_{\max}$$ 和 $$s_{\text{avg}}$$ 评分。按评分从大到小排序，取 top-1 为异常。验证 $$x = 50$$ 被正确检测。

**C2.** 对二维数据集 $$\{(0,0), (1,0), (0,1), (1,1), (10, 10)\}$$，分别使用 $$L_1$$、$$L_2$$、$$L_\infty$$ 距离计算点 $$(10, 10)$$ 到其他所有点的距离。验证三种度量给出不同的距离值但相同的异常排名。

**C3.** 生成 $$n = 100$$ 个标准正态点在 $$d = 2, 10, 100, 1000$$ 维空间中，计算所有成对欧氏距离的均值和标准差。验证 $$\text{CV} = \text{std}/\text{mean}$$ 随 $$d$$ 递减。

### D 级（编程实践）

**D1.** 从零实现 KNN 异常检测器（不使用 sklearn 的 NearestNeighbors），支持 $$s_{\max}$$、$$s_{\text{avg}}$$ 两种评分函数和欧氏距离。在二维合成数据（一个主簇 + 5 个离群点）上测试，绘制散点图并用颜色标注异常评分。

**D2.** 实现基于 KD-tree 的加速版 KNN 异常检测器（可使用 `scipy.spatial.KDTree`）。在 $$n = 10000, d = 5$$ 的数据上对比朴素方法和 KD-tree 方法的运行时间。绘制运行时间随 $$n$$（从 1000 到 50000）变化的曲线。

**D3.** 比较 KNN 异常检测（$$k = 10, s_{\max}$$）、马氏距离（§7.2）和 Z-score 在以下三种数据上的 ROC-AUC：(1) 单簇正态 + 随机异常；(2) 双簇数据 + 异常在两簇之间；(3) 半月形数据 + 异常在凹陷处。绘制三种方法的 ROC 曲线并分析各方法的适用范围。

**D4.** 实现"多 $$k$$ 融合"策略：对 $$k \in \{5, 10, 20, 50\}$$ 分别计算 KNN 评分，然后通过（a）取最大值、（b）取均值、（c）排名融合三种方式合并评分。在合成数据上比较单一 $$k$$ 与融合策略的 AUC 稳定性。

### E 级（案例分析）

**E1.** 在 **KDD Cup 1999**（或 NSL-KDD）网络入侵检测数据集上：(1) 选取数值型特征（持续时间、字节数、错误率等）；(2) 标准化后使用 KNN 异常检测（$$k = 20$$）；(3) 计算 ROC-AUC 和 PR-AP；(4) 分析 KNN 方法能检测到哪些攻击类型（DoS、Probe、U2R、R2L），不能检测到哪些；(5) 尝试不同的距离度量（$$L_1, L_2, L_\infty$$），对比效果。

**E2.** 在 **Mammography**（乳腺 X 光检测）数据集上（OpenML #311），使用 KNN 异常检测识别恶性肿瘤样本。分析：(1) 不同 $$k$$ 值对检测率的影响；(2) 标准化方式（Z-score vs min-max）对结果的影响；(3) 与基于密度的 LOF 方法（§7.4）的对比。

### F 级（开放探讨）

**F1.** **距离方法的局部性扩展**：KNN 异常检测是全局方法——它对所有区域使用相同的阈值。讨论：(1) 在异质密度数据上，这会导致什么问题？(2) §7.4 的 LOF 如何解决这一问题？(3) 能否设计一种方法，在保留 KNN 简洁性的同时引入局部自适应性？（提示：考虑使用 KNN 距离的局部百分位排名。）

**F2.** **距离集中的反面**：定理 7.8 证明了高维下距离集中是不可避免的——但这是在所有特征都独立且同质的假设下。讨论：(1) 如果数据实际上位于一个低维流形上（如 1000 维空间中的 10 维流形），距离集中是否仍然成立？(2) 流形假设如何为高维距离方法提供"逃生通道"？(3) 自编码器的潜在空间是否可以被视为一种隐式的流形距离？
