# 附录 D 数据挖掘竞赛攻略

数据挖掘竞赛是将课堂知识转化为实战能力的最佳途径。本附录提供从入门到进阶的竞赛攻略。

## D.2 竞赛标准化工作流

### 阶段划分

| 阶段 | 时间分配 | 关键活动 | 交付物 | 对应本书章节 |
|------|---------|---------|--------|------------|
| **① 理解题目** | 5% | 阅读题目、理解评估指标、分析数据说明 | 题目理解文档 | §14.2 |
| **② 数据探索** | 15% | EDA、分布分析、缺失值/异常值检查 | EDA 报告 | §2 |
| **③ 基线构建** | 10% | 最简模型 + 首次提交 | 基线分数 | §14.4（MVM） |
| **④ 特征工程** | 35% | 特征构造、选择、变换 | 特征集合 | §2 |
| **⑤ 模型迭代** | 20% | 多模型尝试、超参数调优 | 模型候选集 | §3, §4 |
| **⑥ 融合提交** | 10% | 模型集成、后处理、最终提交 | 最终预测 | §4 |
| **⑦ 复盘总结** | 5% | 阅读获胜方案、总结经验 | 学习笔记 | §14.7 |

### 第一天行动清单

1. 下载全部数据文件，用 `df.info()` 和 `df.describe()` 快速了解数据结构
2. 检查训练集和测试集的**特征分布一致性**——差异大则说明可能存在时间偏移
3. 理解**评估指标**——AUC、RMSE、F1、还是自定义指标？指标决定优化方向
4. 理清数据的**主键和关系**——单表 vs 多表关联？时间序列 vs 独立样本？
5. 构建最简基线并提交一次——确认提交格式正确，建立排行榜参照锚点
6. 设置**本地验证方案**——分割方式应模拟线上评估逻辑

### EDA 检查清单

| 检查项 | 方法 | 注意事项 |
|--------|------|---------|
| 数据规模 | `df.shape` | 行数决定模型选择，列数决定特征策略 |
| 数据类型 | `df.dtypes` | 注意"伪数值"（如邮编 10001 是类别非数值） |
| 缺失值 | `df.isnull().sum()` | 缺失模式比缺失率更重要——是随机还是有规律？ |
| 目标分布 | `df["target"].value_counts()` | 不平衡比 > 10:1 需特殊处理 |
| 特征分布 | 直方图 + 箱线图 | 注意偏态、离群值、多峰分布 |
| 特征相关性 | 相关系数矩阵 | 高相关特征对（>0.95）可能需要去除 |
| 训练/测试一致性 | 特征分布对比 | 分布差异大的特征应谨慎使用 |
| 时间结构 | 按时间排序观察趋势 | 存在时间信号 → 必须用时间分割 |

{% hint style="warning" %}
⚠️ **本地验证 ≈ 线上得分**

如果本地验证分数和线上排行榜分数差距大，说明验证方案有问题（通常是数据泄漏或分割方式不当）。在修复验证方案之前，不要相信任何实验结论。
{% endhint %}

## D.3 特征工程速查

特征工程通常贡献竞赛最终成绩的 50–80%。以下是按数据类型分类的常用技巧：

### 数值特征

| 技巧 | 方法 | 适用场景 | 代码示例 |
|------|------|---------|---------|
| 基本变换 | `log1p`, `sqrt`, Box-Cox | 右偏分布 | `np.log1p(df["col"])` |
| 分箱离散化 | 等频/等宽分箱 | 非线性关系 | `pd.qcut(df["col"], q=10)` |
| 交互特征 | 乘法、除法、差值 | 交互效应 | `df["A"] * df["B"]` |
| 统计聚合 | groupby + mean/std/max | 一对多关系 | `df.groupby("id")["val"].transform("mean")` |
| 偏离度 | 当前值 - 均值 | 标识异常程度 | `df["col"] - df["col"].mean()` |
| 多项式 | 二次/三次项 | 线性模型增强 | `PolynomialFeatures(degree=2)` |
| 排名特征 | 值的排名/百分位 | 消除量纲影响 | `df["col"].rank(pct=True)` |

### 类别特征

| 技巧 | 方法 | 适用模型 | 注意事项 |
|------|------|---------|---------|
| 独热编码 | `pd.get_dummies()` | 线性模型 | 类别数 < 20 |
| 标签编码 | `LabelEncoder()` | 树模型 | 树模型能处理无序编码 |
| 目标编码 | 用目标均值替代类别 | 所有模型 | **必须在 CV 内部计算**，否则泄漏 |
| 频率编码 | 用出现频率替代 | 所有模型 | 快速、无泄漏风险 |
| 类别组合 | 拼接两个类别特征 | 树模型 | `df["A"].astype(str) + "_" + df["B"].astype(str)` |
| WoE 编码 | 基于 IV 值的编码 | 逻辑回归 | 常用于信用评分 |

### 时间特征

| 技巧 | 方法 | 适用场景 |
|------|------|---------|
| 时间分解 | 年、月、日、星期几、小时 | 周期性模式 |
| 滞后特征 | `shift(1)`, `shift(7)`, `shift(30)` | 时间序列（§8, §13.7） |
| 滚动统计 | `rolling(7).mean()`, `.std()` | 趋势和波动 |
| 时间差 | 距上次事件的秒数/天数 | 衰减效应 |
| 节假日标记 | 是否节假日、工作日/周末 | 行为模式变化 |
| 傅里叶特征 | `sin(2π·hour/24)`, `cos(2π·hour/24)` | 循环编码 |

### 文本特征

| 技巧 | 方法 | 适用场景 |
|------|------|---------|
| TF-IDF | `TfidfVectorizer()` | 传统文本分类（§9） |
| 文本统计 | 字数、字符数、句子数 | 简单但常有效 |
| N-gram | 二元组、三元组 | 捕获短语 |
| 词嵌入均值 | 对所有词向量取平均 | 短文本表示 |
| Transformer 嵌入 | BERT / Sentence-BERT 输出 | 高精度文本表示 |

## D.4 超参数调优

### LightGBM 调优路线图

LightGBM 是竞赛中使用率最高的模型。推荐的调优顺序：

| 步骤 | 参数 | 搜索范围 | 影响 |
|------|------|---------|------|
| 1 | `n_estimators` | 先设大值（如 5000）+ early_stopping | 最基本 |
| 2 | `learning_rate` | [0.01, 0.1] | 与 n_estimators 联动 |
| 3 | `max_depth` / `num_leaves` | max_depth: [3, 12]; num_leaves: [15, 255] | 模型复杂度 |
| 4 | `min_child_samples` | [5, 100] | 防过拟合 |
| 5 | `subsample` / `colsample_bytree` | [0.5, 1.0] | 随机性 |
| 6 | `reg_alpha` / `reg_lambda` | [0, 10] | 正则化 |

```python
import lightgbm as lgb

params = {
    "objective": "binary",
    "metric": "auc",
    "n_estimators": 5000,
    "learning_rate": 0.05,
    "num_leaves": 63,
    "max_depth": -1,
    "subsample": 0.8,
    "colsample_bytree": 0.8,
    "reg_alpha": 0.1,
    "reg_lambda": 1.0,
    "verbose": -1,
    "random_state": 42
}

model = lgb.LGBMClassifier(**params)
model.fit(X_train, y_train,
          eval_set=[(X_val, y_val)],
          callbacks=[lgb.early_stopping(100), lgb.log_evaluation(50)])
```

### Optuna 自动调优

```python
import optuna

def objective(trial):
    params = {
        "n_estimators": 5000,
        "learning_rate": trial.suggest_float("lr", 0.01, 0.1, log=True),
        "num_leaves": trial.suggest_int("num_leaves", 15, 255),
        "min_child_samples": trial.suggest_int("min_child", 5, 100),
        "subsample": trial.suggest_float("subsample", 0.5, 1.0),
        "colsample_bytree": trial.suggest_float("colsample", 0.5, 1.0),
    }
    model = lgb.LGBMClassifier(**params, random_state=42, verbose=-1)
```

## D.5 模型选择与融合策略

### 常用模型

| 模型 | 优势 | 劣势 | 竞赛使用频率 |
|------|------|------|------------|
| **LightGBM** | 速度快、内存低、类别特征原生支持 | 小数据易过拟合 | ★★★★★ |
| **XGBoost** | 稳定、社区成熟 | 比 LightGBM 慢 | ★★★★☆ |
| **CatBoost** | 类别特征优秀、调参少 | 训练较慢 | ★★★☆☆ |
| **Neural Network** | 非结构化数据（图像/文本） | 需 GPU、调参复杂 | ★★★☆☆ |
| **线性模型** | 快速、可解释 | 无法捕获非线性 | ★★☆☆☆ |

### 融合策略

| 方法 | 实现方式 | 提升幅度 | 复杂度 | 代码量 |
|------|---------|---------|--------|--------|
| **简单平均** | 多模型预测算术平均 | 小 | ★☆☆ | 1 行 |
| **加权平均** | 按 CV 表现分配权重 | 中 | ★☆☆ | 3 行 |
| **Rank 平均** | 对预测排名后平均 | 中 | ★★☆ | 5 行 |
| **Stacking** | 一层预测作为二层输入 | 中-大 | ★★★ | 30+ 行 |
| **Blending** | Holdout 上训练元模型 | 中 | ★★☆ | 20 行 |
| **异构融合** | 树模型 + NN 融合 | 最大 | ★★★ | 视情况 |

```python
# 加权融合示例
pred_lgb = lgb_model.predict_proba(X_test)[:, 1]
pred_xgb = xgb_model.predict_proba(X_test)[:, 1]
pred_cat = cat_model.predict_proba(X_test)[:, 1]
final_pred = 0.4 * pred_lgb + 0.35 * pred_xgb + 0.25 * pred_cat

# Rank 平均（消除模型间预测尺度差异）
from scipy.stats import rankdata
r_lgb = rankdata(pred_lgb) / len(pred_lgb)
r_xgb = rankdata(pred_xgb) / len(pred_xgb)
r_cat = rankdata(pred_cat) / len(pred_cat)
final_pred = (r_lgb + r_xgb + r_cat) / 3
```

{% hint style="tip" %}
💡 **多样性比准确性更重要**

融合的关键不是每个基模型都很强，而是基模型之间**犯不同的错误**。一个 AUC=0.88 的 LightGBM 和一个 AUC=0.85 的 NN 融合后的效果，通常优于两个 AUC=0.88 的 LightGBM 变种的融合。多样性的来源：不同算法类型、不同特征子集、不同随机种子、不同超参数。
{% endhint %}

## D.6 竞赛常见陷阱

| 陷阱 | 症状 | 对策 | 严重程度 |
|------|------|------|---------|
| **本地/线上不一致** | CV 很高但 LB 很低 | 检查验证方案是否模拟线上逻辑 | ★★★★★ |
| **过拟合排行榜** | Public LB 高但 Private LB 大幅下滑 | 信任 CV，不追逐 Public LB | ★★★★☆ |
| **目标编码泄漏** | 模型表现异常好 | 目标编码必须在 CV 每折内部计算 | ★★★★★ |
| **时序随机分割** | 用"未来"预测"过去" | 时间数据必须时间分割（§14.4） | ★★★★★ |
| **忽略评估指标** | 优化 AUC 但评估 F1 | 确保损失函数与评估指标对齐 | ★★★★☆ |
| **特征爆炸** | 太多无用特征 | 用 Permutation Importance 清理 | ★★★☆☆ |
| **忘记 seed** | 结果不可复现 | 全局设置随机种子 | ★★★☆☆ |
| **最后一天大改** | 新想法没充分验证就提交 | 最终提交选 CV 最稳定的版本 | ★★★★☆ |
| **忽略后处理** | 预测值超出合理范围 | clip 到 [0,1] 或根据业务规则修正 | ★★☆☆☆ |

## D.7 竞赛学习路径

### 推荐入门竞赛（Kaggle）

| 竞赛 | 任务类型 | 难度 | 学习重点 | 建议用时 |
|------|---------|------|---------|---------|
| Titanic | 二分类 | ★☆☆ | 流程熟悉、特征工程入门 | 1–2 天 |
| House Prices | 回归 | ★★☆ | 特征工程、正则化 | 2–3 天 |
| Digit Recognizer | 多分类 | ★★☆ | CNN 入门 | 2–3 天 |
| Store Sales | 时间序列 | ★★★ | 时间特征、滞后变量 | 3–5 天 |
| Tabular Playground | 多种 | ★★☆ | 月度练习，题目多样 | 2–4 天 |

### 进阶路线

1. **获得第一个铜牌**：挑选一个活跃竞赛，完整走完 D.2 工作流
2. **学习获胜方案**：每次竞赛结束后阅读金牌解决方案（Discussion 区），理解差距
3. **组队参赛**：找互补技能的队友（一人做特征、一人做模型、一人做融合）
4. **建立个人模板**：将重复使用的代码（EDA、CV、融合）模板化

### 从竞赛到工业的差距

| 维度 | 竞赛 | 工业项目 |
|------|------|---------|
| 数据 | 干净、固定、标签完整 | 脏、持续变化、标签稀缺 |
| 目标 | 单一指标最优化 | 多目标平衡（精度 + 延迟 + 成本 + 可解释性） |
| 模型 | 复杂融合可接受 | 需可部署、可维护、可解释 |
| 评估 | 排行榜即结果 | 线上 A/B 测试、长期监控（§14.5） |
| 团队 | 个人或小组 | 跨职能协作（§14.6） |
| 生命周期 | 提交即结束 | 持续维护和更新（§14.5） |
| 特征工程 | 越多越好 | 需考虑实时可计算性 |

竞赛是磨练技术能力的优秀训练场，但不要将竞赛思维直接带入工业项目。本书 §14 章的项目管理方法论正是弥合这一差距的桥梁。
