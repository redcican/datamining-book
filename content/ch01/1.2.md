# 1.2 数据挖掘的基本任务与应用领域

面对一个具体的业务问题，数据科学家首先要做的并非选择算法，而是**识别挖掘任务的类型**——这一判断决定了数据需求、模型结构与评估标准的全部走向。正如外科医生在手术前必须明确诊断，数据挖掘工程师在动用任何算法之前，必须厘清面前的问题究竟属于哪一类挖掘任务。1.1 节已经确立了数据挖掘的形式化定义；本节在此基础上，系统介绍数据挖掘的五类基本任务，建立"业务问题 → 挖掘任务"的映射框架，并以九大行业为坐标勾勒应用领域的全景图。

{% hint style="info" %}
📋 **本节学习目标**

完成本节学习后，读者应能够：

1. 以监督学习目标函数的语言，形式化描述分类任务与回归任务，并指出二者在输出空间 $\mathcal{Y}$ 上的本质差异；
2. 用簇内散度目标函数 $J(\mathcal{C})$ 定义聚类任务，并说明与有监督任务在"是否需要标签"上的根本区别；
3. 区分关联规则挖掘与分类任务的研究对象，复述支持度、置信度、提升度的含义；
4. 给定一个业务问题，运用"输出类型 × 标签可用性"决策矩阵，准确识别其对应的挖掘任务类型；
5. 从金融、医疗、制造等不少于五个领域，各列举至少一个典型的数据挖掘应用，并说明其对应的任务类型。
{% endhint %}

## 1.2.1 任务分类框架：描述性任务与预测性任务

数据挖掘的所有任务，均可沿**两个正交维度**加以分类。

第一个维度是**是否需要有标注的训练样本**。若每个训练样本 $(\mathbf{x}_i, y_i)$ 都附有人工标注的目标值 $y_i$，则称任务为**有监督**（supervised）；若训练集仅含特征矩阵 $\mathbf{X}$，无需任何标注，则称任务为**无监督**（unsupervised）。这一维度从数据采集的角度直接决定了项目成本——有监督任务往往需要耗费大量人工标注资源，而无监督任务则可在原始数据上直接进行。

第二个维度是**是否以预测单个样本的输出为目标**。若任务的目标是对新到来的样本 $\mathbf{x}$ 预测其对应的标量或离散类别，则称为**预测性任务**（predictive task）；若任务目标是揭示数据整体的结构或规律——如哪些样本天然聚集在一起，或哪些变量倾向于共同出现——则称为**描述性任务**（descriptive task）。

将上述两个维度交叉，数据挖掘的五类核心任务便浮现出来（如图 1.2.1 所示）：

- **预测性任务**（需要标签 $y$）
  - **分类**（Classification）：$y \in \{c_1, c_2, \ldots, c_K\}$，有限离散集合
  - **回归**（Regression）：$y \in \mathbb{R}$，连续值
- **描述性任务**（仅需特征矩阵 $\mathbf{X}$）
  - **聚类**（Clustering）：将样本划分为若干自然类别
  - **关联规则挖掘**（Association Rule Mining）：发现变量间的共现模式
  - **异常检测**（Anomaly Detection）：识别偏离正常分布的样本

{% hint style="caution" %}
⚠️ **任务类型的错误识别代价极高**

将一个分类问题误建模为回归问题（例如将"是否违约"编码为 0/1 后直接拟合线性回归），会导致模型在决策边界的概率估计严重失真。同样，将一个本质上无监督的探索性问题强行套用有监督框架，则要么因缺乏标签而无法训练，要么因标签质量低劣导致模型学习到错误规律。**任务识别是数据挖掘项目最重要的第一步。**
{% endhint %}

了解了整体框架，下面依次对五类任务给出严格的形式化定义与直觉解释，为后续各章的算法学习奠定共同的语言基础。

## 1.2.2 预测性任务：分类与回归

预测性任务的本质是**从已标注的历史数据中归纳出函数 $f$，并将其泛化应用于未见过的新样本**。用概率语言表述：目标是找到最小化期望预测误差的映射 $f: \mathbb{R}^d \to \mathcal{Y}$：

$$
f^* = \arg\min_f \; \mathbb{E}_{(\mathbf{x}, y) \sim P(\mathcal{X}, \mathcal{Y})} \bigl[\ell\bigl(f(\mathbf{x}),\, y\bigr)\bigr]
\tag{1.5}
$$

其中 $\ell(\cdot, \cdot)$ 为**损失函数**（loss function），$P(\mathcal{X}, \mathcal{Y})$ 为数据的联合分布。分类与回归的核心区别，正在于输出空间 $\mathcal{Y}$ 与损失函数 $\ell$ 的选取。

### 分类任务

分类是预测性任务中最基础的形式：当输出空间 $\mathcal{Y}$ 为有限个离散类别时，目标转化为寻找从特征空间到类别集合的最优判别映射。以下给出其严格的形式化定义。

{% block type="definition" label="1.3" title="分类任务（Classification）" %}
给定训练集 $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n$，其中 $\mathbf{x}_i \in \mathbb{R}^d$，$y_i \in \mathcal{Y} = \{c_1, c_2, \ldots, c_K\}$ 为有限离散标签集（$K$ 为类别数）。**分类任务**的目标是学习判别函数 $f: \mathbb{R}^d \to \mathcal{Y}$，使得在新样本上的**0-1 损失**期望值（即错误率）最小化：

$$
f^* = \arg\min_f \; \mathbb{E}\bigl[\mathbf{1}[f(\mathbf{x}) \neq y]\bigr]
$$

当 $K = 2$ 时称为**二分类**（binary classification），当 $K \geq 3$ 时称为**多分类**（multi-class classification）。
{% endblock %}

分类任务是数据挖掘中最广泛应用的任务类型。其核心挑战在于**泛化**（generalization）：函数 $f$ 对训练集的拟合精度高，并不保证对未见样本同样有效。如何在欠拟合（under-fitting）与过拟合（over-fitting）之间取得最优平衡，是第 3 章分类算法的核心主题。

{% block type="example" label="1.2" title="垃圾邮件过滤（二分类）" %}
**问题**：给定一封邮件，判断其是否为垃圾邮件。

**特征向量** $\mathbf{x} \in \mathbb{R}^d$：以 TF-IDF 权重编码的词袋向量，$d$ 为词汇表大小（通常 $10^4 \sim 10^5$）。

**标签**：$y \in \{0, 1\}$，$0$ 表示正常邮件，$1$ 表示垃圾邮件。

**目标**：学习函数 $f$，使得对新到达的邮件，预测准确率最大化。在此场景下，**误判垃圾邮件为正常**（漏报，False Negative）的代价通常低于**误判正常邮件为垃圾**（误报，False Positive）的代价——因为后者会导致用户错过重要信息。这一权衡体现了**任务定义必须嵌入业务成本结构**的原则。

**典型算法**：朴素贝叶斯（第 3 章）、支持向量机（第 3 章）、逻辑回归。
{% endblock %}

### 回归任务

当输出空间从有限离散集合扩展为连续实数轴时，预测性任务转变为回归。回归模型的目标是拟合一条能够准确描述特征与连续目标之间数量关系的函数，广泛应用于价格预测、趋势外推与风险量化等场景。

{% block type="definition" label="1.4" title="回归任务（Regression）" %}
给定训练集 $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n$，其中 $y_i \in \mathbb{R}$ 为**连续数值**目标（如房价、温度、销售量）。**回归任务**的目标是学习函数 $f: \mathbb{R}^d \to \mathbb{R}$，使得**均方误差**（Mean Squared Error, MSE）最小化：

$$
f^* = \arg\min_f \; \mathbb{E}\bigl[(f(\mathbf{x}) - y)^2\bigr]
\tag{1.6}
$$

最简单的线性回归假设 $f(\mathbf{x}) = \mathbf{w}^{\!\top}\mathbf{x} + b$，其中权重向量 $\mathbf{w} \in \mathbb{R}^d$ 与偏置 $b \in \mathbb{R}$ 通过最小化训练集上的 MSE 求解：

$$
(\hat{\mathbf{w}}, \hat{b}) = \arg\min_{\mathbf{w}, b} \frac{1}{n} \sum_{i=1}^n ({\mathbf{w}^{\!\top}\mathbf{x}_i + b - y_i})^2
\tag{1.7}
$$
{% endblock %}

{% block type="example" label="1.3" title="住宅价格预测（回归）" %}
**问题**：根据房屋特征（面积、卧室数、地理位置等），预测其市场成交价。

**特征向量** $\mathbf{x} = (x_1, x_2, \ldots, x_d)^{\!\top}$，包含数值型（面积 $x_1 = 120\,\text{m}^2$）与类别型特征（地区编码经 One-Hot 编码转换）。

**目标变量** $y \in \mathbb{R}$，如 $y = 350\,\text{万元}$。

**关键挑战**：（1）原始价格通常右偏分布，常取对数变换 $\tilde{y} = \ln(y)$ 以接近正态分布；（2）各特征量纲差异悬殊，须标准化（见第 2 章）；（3）位置类别特征的编码方式对模型影响显著。

**典型算法**：线性回归、决策树回归、梯度提升树（XGBoost）。
{% endblock %}

{% hint style="caution" %}
⚠️ **分类与回归的边界案例**

"预测评分"型问题（如 1–5 分的用户评分）介于分类与回归之间：若将 5 个等级视为有序离散类别，则为**有序分类**（ordinal classification）；若将评分视为连续值建模，则为回归。实践中常将其建模为回归任务，因为 MSE 损失能够编码"预测为 4 分时目标为 5 分"的误差小于"预测为 1 分时目标为 5 分"这一信息，而 0-1 损失则无法区分这一差异。
{% endhint %}

预测性任务依赖标注数据，而现实中标注往往稀缺且昂贵。描述性任务则无需标签，直接从数据结构中挖掘洞察——这是下一节的主题。

## 1.2.3 描述性任务：聚类、关联规则与异常检测

描述性任务的核心哲学是：**数据本身即包含结构，挖掘任务的目标是将这一结构显式化、可解释化**。没有预先定义的"正确答案"，评估标准因此更加多元。

### 聚类

描述性任务无需事先提供标注，直接从特征矩阵 $\mathbf{X}$ 的内在结构中发现规律。聚类是其中最基础的形式：它将样本集合自动划分为若干自然分组，使组内样本尽可能相似、组间样本尽可能相异，而无需事先告知算法"正确分组"应是什么。以下给出目标函数的形式化定义。

{% block type="definition" label="1.5" title="聚类任务（Clustering）" %}
给定无标注数据集 $\mathcal{D} = \{\mathbf{x}_i\}_{i=1}^n$，**聚类任务**的目标是将 $\mathcal{D}$ 划分为 $K$ 个不相交的非空子集（簇）$\mathcal{C} = \{C_1, C_2, \ldots, C_K\}$，满足：

$$
C_k \neq \emptyset, \quad \bigcup_{k=1}^K C_k = \mathcal{D}, \quad C_j \cap C_k = \emptyset \; (j \neq k)
$$

最广泛使用的**簇内离散度**（within-cluster scatter）目标函数为：

$$
J(\mathcal{C}) = \sum_{k=1}^{K} \sum_{\mathbf{x}_i \in C_k} \|\mathbf{x}_i - \boldsymbol{\mu}_k\|_2^2, \quad \boldsymbol{\mu}_k = \frac{1}{|C_k|}\sum_{\mathbf{x}_i \in C_k} \mathbf{x}_i
\tag{1.8}
$$

其中 $\boldsymbol{\mu}_k \in \mathbb{R}^d$ 为第 $k$ 个簇的**质心**（centroid）。最优划分 $\mathcal{C}^* = \arg\min_{\mathcal{C}} J(\mathcal{C})$。
{% endblock %}

目标函数（1.8）具有直觉上的自然性：最小化簇内方差等价于最大化同簇样本的"凝聚度"，同时（在固定总方差的前提下）隐式最大化了不同簇之间的"分离度"。这是第 4 章 $k$-means 算法的优化目标。

{% hint style="mathbg" %}
📐 **数学基础：聚类的 NP-困难性**

最小化目标函数（1.8）是一个 NP-困难（NP-hard）组合优化问题（Even et al., 1975）。当 $d \geq 2$ 时，穷举 $K^n$ 种可能划分在计算上不可行。$k$-means 算法通过交替更新质心与簇分配的**期望-最大化**（EM）启发式策略，在多项式时间内求解近似解，但不保证全局最优。这一计算复杂性是数据挖掘研究的持续动力之一。
{% endhint %}

{% block type="example" label="1.4" title="电商用户行为聚类" %}
**场景**：某电商平台拥有 800 万注册用户，希望在不借助任何外部标签的情况下，发现用户群体的自然分层，以支持精准营销策略制定。

**特征向量**：以 RFM 模型构建三维特征 $\mathbf{x} = (R, F, M)^{\!\top}$，其中 $R$（Recency）为最近一次购买距今天数，$F$（Frequency）为过去 90 天内购买频次，$M$（Monetary）为过去 90 天总消费金额（元）。

**聚类结果**（$K=4$）：
- $C_1$（高价值活跃用户）：$R < 7$，$F > 15$，$M > 2000$，占比 8%
- $C_2$（中价值潜力用户）：$R < 30$，$5 < F \leq 15$，$500 < M \leq 2000$，占比 27%
- $C_3$（低频但高单价）：$R < 60$，$F \leq 5$，$M > 1000$，占比 12%
- $C_4$（流失风险用户）：$R > 60$，$F \leq 3$，$M < 200$，占比 53%

**业务价值**：对 $C_4$ 发放召回优惠券，对 $C_1$ 推送高端会员权益，差异化运营成本仅为全量推送的 30%，而 GMV 提升效果相当。

注意：$K=4$ 的选择依赖业务经验与**轮廓系数**（silhouette coefficient）等内部指标的综合判断，详见第 4 章。
{% endblock %}

### 关联规则挖掘

关联规则挖掘已在 1.1.5 节的购物篮案例中初步引入。与聚类和分类的根本区别在于：**关联规则挖掘不以单个样本为预测或划分单位，而以变量间的共现关系为研究对象**——其输出是一组形如"购买 $X$ 则倾向于同时购买 $Y$"的蕴含规则，而非样本标签。以下给出其形式化定义。

{% block type="definition" label="1.6" title="关联规则挖掘（Association Rule Mining）" %}
设项目全集为 $\mathcal{I} = \{i_1, i_2, \ldots, i_m\}$，事务数据库 $\mathcal{T} = \{t_1, t_2, \ldots, t_N\}$，其中每条事务 $t_k \subseteq \mathcal{I}$。**关联规则**是形如 $X \Rightarrow Y$ 的蕴含关系（$X, Y \subseteq \mathcal{I}$，$X \cap Y = \emptyset$），其质量由三个度量刻画：

$$
\text{supp}(X \Rightarrow Y) = \frac{|\{t \in \mathcal{T} : X \cup Y \subseteq t\}|}{N}, \qquad
\text{conf}(X \Rightarrow Y) = \frac{\text{supp}(X \cup Y)}{\text{supp}(X)}, \qquad
\text{lift}(X \Rightarrow Y) = \frac{\text{conf}(X \Rightarrow Y)}{\text{supp}(Y)}
$$

其中**支持度**（$\text{supp}$）衡量 $X \cup Y$ 在事务库中出现的频率，**置信度**（$\text{conf}$）衡量"已知 $X$ 出现时 $Y$ 也出现"的条件概率，**提升度**（$\text{lift}$）衡量共现是否超越统计独立基线（$\text{lift} > 1$ 表示正相关，$= 1$ 表示独立，$< 1$ 表示负相关）。**关联规则挖掘**的目标是高效枚举 $\mathcal{T}$ 中所有满足最小支持度阈值 $\sigma_{\min}$ 与最小置信度阈值 $\rho_{\min}$ 的规则集合 $\mathcal{R} = \{X \Rightarrow Y : \text{supp} \geq \sigma_{\min},\, \text{conf} \geq \rho_{\min}\}$。其完整算法（Apriori、FP-Growth）将在第六章系统展开。
{% endblock %}

### 异常检测

在真实数据中，少量样本可能因欺诈行为、设备故障或人工录入错误而显著偏离总体分布。异常检测旨在无须依赖大量标注数据的前提下，通过构造评分函数量化每个样本与"正常模式"的偏离程度，进而将高评分样本判定为异常。

{% block type="definition" label="1.7" title="异常检测任务（Anomaly Detection）" %}
给定数据集 $\mathcal{D} = \{\mathbf{x}_i\}_{i=1}^n$，**异常检测**的目标是识别与"正常"数据生成分布显著偏离的样本子集 $\mathcal{A} \subset \mathcal{D}$。形式上，构造异常评分函数 $s: \mathbb{R}^d \to \mathbb{R}$（值越大表示越异常），将 $s(\mathbf{x}) > \theta$（$\theta$ 为决策阈值）的样本判定为异常：

$$
\hat{\mathcal{A}} = \{\mathbf{x}_i \in \mathcal{D} \mid s(\mathbf{x}_i) > \theta\}
\tag{1.9}
$$

根据标签可用性，异常检测可进一步分为：
- **无监督异常检测**：仅使用 $\mathcal{D}$，假设绝大多数样本为正常；
- **半监督异常检测**：仅有正常样本的标签，通过学习正常分布边界识别异常；
- **有监督异常检测**：同时拥有正常与异常样本的标签（此时退化为二分类，但通常面临严重类别不平衡）。
{% endblock %}

{% hint style="think" %}
🤔 **思考**：对于银行欺诈检测，正常交易与欺诈交易的比例约为 1000:1。若直接将其建模为有监督二分类问题，"全部预测为正常"的哑模型准确率已超过 99.9%，却毫无实用价值。这一现象揭示了什么？评估此类任务应使用什么指标替代准确率？（提示：Precision、Recall、F1、AUC-ROC——见第 3 章。）
{% endhint %}

{% hint style="frontier" %}
🚀 **前沿视野：自监督与基础模型重塑任务边界（2022–2025）**

传统任务分类体系在大型自监督预训练模型（Foundation Models）兴起后面临挑战。以 GPT-4 为代表的大语言模型在预训练阶段无需任务标签，通过语言建模目标在海量文本上学习通用表征；部署阶段仅需极少量提示（few-shot prompting）即可完成分类、回归、聚类摘要等多种任务。这模糊了"有监督/无监督"的边界。

在结构化数据领域，TabPFN（Hollmann et al., 2022）通过在合成数据集上元学习，实现了无需超参数调优的"即插即用"分类；UniTS（Liu et al., 2024）则探索了一个模型统一处理分类、回归、异常检测的多任务架构。

然而，这些模型在可解释性、数据安全合规与小样本精度上的局限，使得经典算法在高风险决策（医疗诊断、金融风控）领域仍占据核心地位。**掌握五类基本任务的形式化定义，是理解任何新模型"在做什么"的认知基础，而非可被跳过的历史遗产。**
{% endhint %}

图 1.2.3 以合成数据直观展示了上述五类任务的典型算法输出：SVM 的非线性决策边界（分类）、SVR 的曲线拟合（回归）、$k$-means 的簇划分（聚类）、支持度-置信度散点图（关联规则）以及孤立森林的异常评分等高面（异常检测）。读者可据此建立各任务"输入-输出"形态的视觉直觉，为后续章节的算法细节学习做好铺垫。

## 1.2.4 任务选择的决策框架

面对一个真实业务问题，如何系统性地确定挖掘任务类型？以下决策框架将"业务问题 → 数据特征 → 挖掘任务"的推理链路结构化：

**第一问：目标输出是什么形式？**

若目标是对**每个新样本**做出预测（预测收益率、预测是否违约、预测设备剩余寿命），进入预测性任务分支。若目标是**描述数据整体结构**（哪些客户天然相似、哪些商品倾向共购），进入描述性任务分支。

**第二问（预测性分支）：目标变量 $y$ 是连续的还是离散的？**

- $y$ 为有限离散类别 → **分类**（如二分类：欺诈/正常；多分类：情感/中立/负面）
- $y$ 为连续数值，或有序数值但连续近似合理 → **回归**（如销售额预测、温度预测）

**第三问（描述性分支）：研究对象是样本间相似度还是变量间共现关系？**

- 目标是将样本分组 → **聚类**（样本是操作单元）
- 目标是发现变量/项目的共现规律 → **关联规则挖掘**（项集是操作单元）
- 目标是识别"不寻常"的个体 → **异常检测**（通常与其他任务结合）

以上三问可组合为"任务识别决策矩阵"：

| 业务描述关键词 | 输出形式 | 需要标签 | 挖掘任务 |
|---|---|---|---|
| 预测、判断、分类、识别、诊断 | 离散类别 | 是 | 分类 |
| 预测、估计、预报、量化 | 连续数值 | 是 | 回归 |
| 分群、细分、画像、分层 | 样本分组 | 否 | 聚类 |
| 关联、推荐、组合、同购 | 规则集合 | 否 | 关联规则 |
| 检测、识别异常、监控、预警 | 异常评分 | 可选 | 异常检测 |

{% hint style="tip" %}
💡 **技巧：业务问题往往对应多个任务的组合**

一个实际数据产品通常涉及多个任务的流水线。以信用评分为例：首先用**聚类**将客户分为若干风险群体（无监督）；再在每个群体内用**分类**判断是否违约（有监督）；同时用**异常检测**实时标记欺诈行为（无监督/半监督）；最终用**回归**预测违约损失金额（有监督）。将问题拆解为子任务，分别选择最优算法，是工业级数据挖掘的常见范式。
{% endhint %}

明确了任务类型之后，下一步便是将任务放置于具体的行业场景——数据挖掘的真正价值，最终体现在对实际问题的解决上。

数据挖掘的应用边界几乎与人类组织活动的范围相重合。图 1.2.2 以九宫格形式呈现了九大典型行业及其代表性挖掘任务，以下逐一展开说明最核心的应用场景。

<figcaption><strong>图 1.2.2</strong>　数据挖掘典型应用领域全景（3×3 宫格）。每个格子标注所属领域名称、英文名，以及三类代表性挖掘任务及其对应的任务类型。九大领域覆盖金融、医疗、零售、制造、交通、教育、社交媒体、政务与科学研究。</figcaption>

**金融科技（FinTech）**　金融行业是数据挖掘最早大规模商业化的领域之一。**信用评分**（credit scoring）以借款人的历史还款行为、收入水平、负债比率等特征为输入，输出违约概率或信用等级，本质为二分类或有序分类任务。**欺诈检测**（fraud detection）面临极端类别不平衡（正常交易：欺诈交易 ≈ 10,000:1），通常建模为无监督或半监督异常检测，近实时处理每秒万级交易流。**量化交易**则是典型的时序回归问题——预测资产价格变动，但同时面临市场的非平稳性与过拟合风险。

**医疗健康（Healthcare）**　医疗数据挖掘的特殊性在于标注数据的获取门槛极高（需要临床专家标注）且错误决策代价极大（直接影响患者健康）。**疾病预测**（disease prediction）利用电子病历（EHR）、基因组数据预测疾病发生风险（分类）；**医学影像分析**（如 CT 图像中的肿瘤检测）以深度卷积网络为主，本质为有监督分类；**药物发现**（drug discovery）通过预测分子结构与靶点蛋白的结合强度，将早期筛选周期从数年压缩至数月（回归）。

**零售电商（E-Commerce）**　这是关联规则挖掘与推荐系统的核心应用场景。阿里巴巴、亚马逊的"你可能还喜欢"模块基于**协同过滤**（collaborative filtering），本质是在用户-商品交互矩阵上进行的矩阵补全（回归）或排序（分类）任务。**需求预测**（demand forecasting）预测各 SKU 在未来 7/14/30 天的销售量，直接影响采购计划与库存成本（时序回归，见第 7 章）。

**智能制造（Manufacturing）**　**预测性维护**（predictive maintenance）利用设备传感器数据（振动、温度、电流）预测设备何时可能发生故障，在停机前完成维修（分类或时序预测），将计划外停机损失降低 30%–50%。**质量控制**（quality inspection）通过图像识别自动检测产品缺陷，是制造业应用最广的分类任务。

{% hint style="engineering" %}
🔧 **工程实践：挖掘任务的工业部署挑战**

学术环境中，数据挖掘模型通常在静态数据集上训练和评估；工业部署则面临全然不同的挑战：

1. **数据漂移（Data Drift）**：生产环境中的数据分布随时间变化（如消费习惯受政策/季节影响），需要持续监控模型性能并触发再训练。
2. **延迟约束**：金融欺诈检测要求推理延迟 < 50ms，推荐系统要求 < 100ms，这制约了可部署模型的复杂度。
3. **反馈闭环**：模型的预测会影响未来的数据分布（如推荐系统影响用户点击行为），造成"幸存者偏差"——必须引入探索（exploration）机制打破正反馈循环。
4. **可解释性要求**：金融信贷场景的监管要求模型必须对拒贷决策提供可理解的理由（《欧盟 AI 法案》、《银行业消费者保护实施办法》），这使得黑盒模型在此类场景受到限制。

因此，工业界的任务选择不仅取决于预测精度，还取决于推理效率、可解释性与模型可维护性的综合权衡。
{% endhint %}

应用领域的多样性揭示了一个核心规律：**不同行业面临的挖掘任务类型相似，但数据特性、评估标准与部署约束各异**。掌握这些共性与差异，是数据科学家将领域知识与算法知识有机结合的前提。

## 1.2.6 案例：医疗数据挖掘——ICU 患者死亡风险预测

{% block type="case" label="1.2" title="ICU 患者院内死亡风险预测（MIMIC-III 数据集）" %}

**业务背景**

重症监护室（ICU）收治的患者面临极高的死亡风险，而医疗资源（床位、护理人力、呼吸机）极为有限。若能在患者入 ICU 24 小时内，利用早期生命体征与实验室检测数据，准确预测其院内死亡风险，则可：（1）提前为高风险患者配置重症护理资源；（2）对低风险患者早期制定转科计划，释放 ICU 床位；（3）辅助临床医生做出知情决策。

**数据描述**

MIMIC-III（Medical Information Mart for Intensive Care，Johnson et al., 2016）是由麻省理工学院与贝斯以色列医疗中心联合维护的去标识化 ICU 数据库，覆盖 2001–2012 年约 $n = 46{,}520$ 次 ICU 住院，$d \approx 48$ 个早期特征（前 24 小时内的生命体征均值、最大值、最小值，以及 17 项实验室检测值）。

| 特征类别 | 具体指标 | 类型 |
|---|---|---|
| **生命体征** | 心率、血压（收缩/舒张/平均）、体温、呼吸频率、血氧饱和度 | 数值型，连续 |
| **实验室检测** | 白细胞计数、肌酐、乳酸、血小板、胆红素等 | 数值型，含缺失 |
| **人口统计** | 年龄、性别 | 混合型 |
| **基础病史** | 主要诊断 ICD 编码 | 类别型（高维） |

**挖掘目标**

预测变量：$y \in \{0, 1\}$，$y=1$ 表示患者在本次住院期间死亡（院内死亡率约 11.5%）。

任务类型：**有监督二分类**——但由于正负样本比约为 1:8，须特别处理类别不平衡。

**评估指标选择**

在 ICU 死亡预测中，**漏报**（预测为低风险但实际死亡，False Negative）的代价远高于**误报**（预测为高风险但实际存活，False Positive）。因此，选用 **AUC-ROC** 与 **AUPRC**（精确率-召回率曲线下面积）作为主要指标，而非准确率。

**分析结果**

现有文献中，基线 SAPS-II 评分系统（传统临床评分）在此任务上 AUC-ROC ≈ 0.74；基于梯度提升树（XGBoost）的数据驱动模型可达 AUC-ROC ≈ 0.88；加入序列建模（LSTM 对时序生命体征建模）可进一步提升至 0.89–0.91。

**关键挑战**

（1）**缺失值比例高**：实验室检测缺失率在 30%–60% 之间，插补策略选择对结果影响显著（见第 2 章）。（2）**特征时序性**：24 小时内的生命体征并非静态，需要特征聚合或时序建模（见第 7 章）。（3）**部署伦理**：模型辅助决策而非替代医生，其输出须配备置信区间与可解释性说明，以满足临床审查要求。

**思考延伸**　本案例中，如果将死亡风险视为 0–100% 的连续概率值（回归建模），与直接二分类相比，各有什么优势和局限？（提示：标定（calibration）——预测的概率是否与实际频率相符——对临床决策支持至关重要。）
{% endblock %}

```python
print(f"  {'任务':<12} {'接口方法':<22} {'输出'}")
print(f"  {'-'*52}")
print(f"  {'分类':<12} {'fit() + predict()':<22} 离散类别标签")
print(f"  {'回归':<12} {'fit() + predict()':<22} 连续实数值")
print(f"  {'聚类':<12} {'fit_predict()':<22} 簇编号 0,1,...,K-1")
print(f"  {'关联规则':<12} {'fit() + rules':<22} (X→Y, supp, conf)")
print(f"  {'异常检测':<12} {'fit_predict()':<22} +1(正常)/-1(异常)")
```

{% hint style="mathbg" %}
📐 **数学基础：scikit-learn 的 Estimator API**

scikit-learn 的所有算法均遵循统一的估计器接口（Estimator API）：
- `fit(X, y=None)`：接受样本矩阵 $\mathbf{X} \in \mathbb{R}^{n \times d}$（和可选标签 $\mathbf{y}$），估计模型参数；
- `predict(X)`：对新样本矩阵 $\mathbf{X}' \in \mathbb{R}^{m \times d}$ 输出预测；
- `transform(X)`：用于预处理器（Scaler、PCA 等），将 $\mathbf{X}$ 映射到新空间。

`Pipeline([step1, step2, ...])` 将多个转换器与最终估计器串联，确保 `fit_transform` 只在训练集上计算，避免测试集泄露（data leakage，见第 2 章详述）。这是工业级代码的基本规范。
{% endhint %}

## 本节小结

本节系统建立了数据挖掘任务的分类体系，核心要点如下：

1. **两个维度**：以"是否需要标签"与"是否以单样本预测为目标"为轴，将挖掘任务分为预测性（分类、回归）与描述性（聚类、关联规则、异常检测）两大类。
2. **五类任务形式化**：分类最小化期望 0-1 损失（定义 1.3）；回归最小化 MSE（定义 1.4）；聚类最小化簇内离散度 $J(\mathcal{C})$（定义 1.5）；关联规则挖掘通过支持度与置信度阈值枚举共现规律集合（定义 1.6）；异常检测通过评分函数 $s(\mathbf{x})$ 与阈值 $\theta$ 识别偏离样本（定义 1.7）。
3. **任务识别框架**：以"输出类型 × 标签可用性"决策矩阵，将自然语言业务需求映射至挖掘任务类型，避免"任务错配"导致的项目失败。
4. **九大应用领域**：金融、医疗、零售、制造、交通、教育、社交媒体、政务、科研——各领域任务类型相似但数据特性与部署约束各异（如图 1.2.2）。
5. **工业部署差异**：学术基准与生产部署的核心差异在于数据漂移、延迟约束、反馈闭环与可解释性要求，这决定了算法选择不能仅以精度为唯一标准。

{% hint style="ideology" %}
🌏 **课程思政：算法公平性与数据挖掘的社会责任**

数据挖掘的五类任务无一例外地涉及对人的判断与决策。**分类算法**用于信用审批与医疗诊断；**回归模型**决定保险定价与薪资预测；**聚类**划定用户群体并据此差异化服务。当这些算法的训练数据本身含有历史偏见时，模型便可能自动化并放大歧视：

- Amazon 的人才简历筛选算法（2014–2018）因训练数据中男性工程师占比过高，学会了对含有"女子大学"字样的简历降权，被迫废弃；
- 美国 COMPAS 刑事风险评估系统被研究发现，对黑人被告的累犯风险存在系统性高估（ProPublica, 2016）；
- 中国部分平台的"大数据杀熟"现象，本质是以聚类结果为依据的价格歧视。

**技术无法独立解决公平性问题，但技术人员有责任将公平性纳入设计目标**。2021 年施行的《算法推荐管理规定》、《互联网信息服务算法推荐管理规定》（2022），要求算法提供者不得利用算法实施"不公平竞争"和"损害用户权益"的行为。数据挖掘工程师需要从数据采集阶段开始关注代表性偏差，在模型评估阶段引入分群公平指标（如人口均等、机会均等），并建立可追溯的模型决策记录。
{% endhint %}

### 关键术语表

| 中文术语 | 英文 | 简要定义 |
|---|---|---|
| 预测性任务 | Predictive Task | 以学习函数 $f: \mathbb{R}^d \to \mathcal{Y}$ 对新样本预测输出为目标的挖掘任务 |
| 描述性任务 | Descriptive Task | 以揭示数据整体结构（分组、关联、异常）为目标的挖掘任务 |
| 分类 | Classification | $y \in \{c_1,\ldots,c_K\}$，最小化期望 0-1 损失的有监督任务 |
| 回归 | Regression | $y \in \mathbb{R}$，最小化期望均方误差的有监督任务 |
| 聚类 | Clustering | 将无标注数据划分为若干自然分组，最小化簇内离散度 |
| 关联规则 | Association Rule | 形如 $X \Rightarrow Y$ 的共现规律，以支持度、置信度、提升度量化 |
| 异常检测 | Anomaly Detection | 通过评分函数 $s(\mathbf{x})$ 识别偏离正常分布的样本 |
| 轮廓系数 | Silhouette Score | 量化聚类质量的内部指标，取值 $[-1,1]$，越大越好 |
| 类别不平衡 | Class Imbalance | 分类任务中不同类别样本数量悬殊的现象（如欺诈检测） |
| 数据漂移 | Data Drift | 生产环境中输入数据分布随时间变化，导致模型性能下降的现象 |
| 损失函数 | Loss Function | 量化预测值与真实值差异的函数 $\ell(f(\mathbf{x}), y)$ |
