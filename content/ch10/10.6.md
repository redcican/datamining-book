# 10.6 链接预测方法

前几节讨论了图的特征提取（§10.2）、社区发现（§10.3）、节点分类（§10.4）和频繁子图挖掘（§10.5），这些方法分析的是图的**当前结构**。本节转向一个面向**未来**的问题：给定一个不完整的图，**预测哪些尚未观测到的边在未来最可能出现**——即**链接预测**（Link Prediction）。

链接预测在社交网络（"你可能认识的人"）、生物网络（预测蛋白质相互作用）、推荐系统（预测用户-商品关系）和知识图谱补全中具有重要应用。其核心思想简洁而深刻：**两个节点越相似，它们之间越可能存在连接**。

## 10.6.1 问题设定

{% block type="definition" title="定义 10.27　链接预测" %}
给定图 $$G = (V, E)$$ 在时刻 $$t$$ 的快照，**链接预测**的目标是对所有不存在边的节点对 $$(u, v) \notin E$$ 计算一个**评分函数**：

$$
s(u, v): V \times V \rightarrow \mathbb{R} \tag{10.57}
$$

使得评分越高的节点对在未来（时刻 $$t + \Delta t$$）越可能形成新的连接。

链接预测问题可以看作一个**排序问题**：将所有未连接的节点对按评分降序排列，排名靠前的节点对被预测为最可能出现的新边。
{% endblock %}

链接预测的核心挑战在于：未连接的节点对数量为 $$O(n^2)$$ 级别，远超已有边数 $$|E|$$，且绝大多数节点对永远不会形成边。这意味着链接预测面临严重的**类别不平衡**问题——正例（实际会出现的边）远少于负例。

## 10.6.2 基于局部相似度的方法

最直观的链接预测方法基于**局部拓扑相似度**——如果两个节点有许多公共邻居，它们很可能存在连接。

{% block type="definition" title="定义 10.28　局部相似度指标" %}
设 $$\Gamma(u) = \{v \mid (u, v) \in E\}$$ 为节点 $$u$$ 的邻居集合。常用的局部相似度指标包括：

**公共邻居**（Common Neighbors, CN）：
$$
s_{\text{CN}}(u, v) = |\Gamma(u) \cap \Gamma(v)| \tag{10.58}
$$

**Jaccard 系数**：
$$
s_{\text{Jaccard}}(u, v) = \frac{|\Gamma(u) \cap \Gamma(v)|}{|\Gamma(u) \cup \Gamma(v)|} \tag{10.59}
$$

**Adamic-Adar 指标**：
$$
s_{\text{AA}}(u, v) = \sum_{w \in \Gamma(u) \cap \Gamma(v)} \frac{1}{\ln d(w)} \tag{10.60}
$$

**优先连接**（Preferential Attachment, PA）：
$$
s_{\text{PA}}(u, v) = d(u) \cdot d(v) \tag{10.61}
$$

四种指标的直觉：CN 直接计数公共邻居；Jaccard 通过归一化消除度偏差；AA 给低度的公共邻居更高权重（"稀有邻居"更有价值）；PA 基于"富者更富"假设（高度节点更容易获得新边）。
{% endblock %}

{% hint style="info" %}
🤔 **思考：为什么 Adamic-Adar 指标有效？**

考虑两个用户 $$u$$ 和 $$v$$ 的共同好友 $$w$$：如果 $$w$$ 只有 3 个好友（低度），那么 $$u$$ 和 $$v$$ 通过 $$w$$ 相连的证据很强——$$w$$ 的少数好友之间关系密切。但如果 $$w$$ 有 1000 个好友（高度），则 $$u$$ 和 $$v$$ 仅仅因为都认识"社交名人" $$w$$ 而碰巧是公共邻居，证据较弱。Adamic-Adar 通过 $$1/\ln d(w)$$ 的加权反映了这种直觉。
{% endhint %}

## 10.6.3 基于全局路径的方法

局部相似度指标简洁高效，但存在一个根本性的局限：它们的"视野"仅限于节点的直接邻居（1-hop）。考虑两个处于不同社区的节点——例如社交网络中的两个人，他们没有共同好友（$$|\Gamma(u) \cap \Gamma(v)| = 0$$），但通过两三个中间人就能连接起来。对于这种情况，所有基于公共邻居的指标（CN、Jaccard、AA）都给出零分，完全无法捕获这种潜在的连接可能。然而，在真实网络中，跨社区的"桥接边"虽然稀少，却往往具有重要的社会或功能意义。

**基于路径**的方法通过考虑节点之间**所有长度的路径**来突破这一局限。其核心直觉是：即使两个节点没有直接的公共邻居，如果它们之间存在大量的短路径（2-hop、3-hop），那么它们之间形成边的可能性仍然很高。通过对不同长度的路径赋予递减的权重，这类方法在捕获全局拓扑信息的同时，仍然优先关注近距离的连接。

{% block type="definition" title="定义 10.29　全局路径指标" %}
**Katz 指标**：考虑两节点之间所有长度的路径，短路径权重高、长路径权重低：

$$
s_{\text{Katz}}(u, v) = \sum_{\ell=1}^{\infty} \beta^\ell \cdot |\text{paths}_{u,v}^{(\ell)}| \tag{10.62}
$$

其中 $$\beta \in (0, 1)$$ 为衰减因子，$$|\text{paths}_{u,v}^{(\ell)}|$$ 为 $$u$$ 到 $$v$$ 的长度为 $$\ell$$ 的路径数。

**随机游走重启**（Random Walk with Restart, RWR）：从节点 $$u$$ 出发进行随机游走，每步以概率 $$\alpha$$ 重启回到 $$u$$，$$s_{\text{RWR}}(u, v)$$ 为稳态时到达 $$v$$ 的概率：

$$
\mathbf{q}_u = (1 - \alpha) \mathbf{P}^T \mathbf{q}_u + \alpha \, \mathbf{e}_u \tag{10.63}
$$

其中 $$\mathbf{P} = \mathbf{D}^{-1}\mathbf{A}$$ 为转移矩阵（§10.4），$$\mathbf{e}_u$$ 为 $$u$$ 的单位向量。稳态分布 $$\mathbf{q}_u$$ 的第 $$v$$ 分量即为 $$s_{\text{RWR}}(u, v)$$。
{% endblock %}

Katz 指标的定义虽然直观——对所有长度的路径加权求和——但直接枚举路径的计算代价极高：长度为 $$\ell$$ 的路径数量随 $$\ell$$ 指数增长，逐一遍历在稍大的网络上就不可行。然而，图论中有一个经典而优雅的结论：**邻接矩阵 $$\mathbf{A}$$ 的 $$\ell$$ 次幂 $$(\mathbf{A}^\ell)_{uv}$$ 恰好等于 $$u$$ 到 $$v$$ 的长度为 $$\ell$$ 的路径数**。这个代数-组合对应关系是将 Katz 指标从"枚举路径"转化为"矩阵求逆"的关键桥梁——它把一个组合优化问题转化为线性代数运算，使得我们可以利用 Neumann 级数的收敛性得到闭式解。

{% block type="theorem" title="定理 10.6　Katz 指标的矩阵闭式解" %}
利用 $$|\text{paths}_{u,v}^{(\ell)}| = (\mathbf{A}^\ell)_{uv}$$（邻接矩阵的 $$\ell$$ 次幂给出长度为 $$\ell$$ 的路径数），Katz 指标的矩阵形式为：

$$
\mathbf{S}_{\text{Katz}} = \sum_{\ell=1}^{\infty} \beta^\ell \mathbf{A}^\ell = (\mathbf{I} - \beta \mathbf{A})^{-1} - \mathbf{I} \tag{10.64}
$$

当 $$\beta < 1/\rho(\mathbf{A})$$（$$\rho(\mathbf{A})$$ 为 $$\mathbf{A}$$ 的谱半径）时，Neumann 级数收敛，闭式解存在。

**证明**　$$\sum_{\ell=1}^{\infty} \beta^\ell \mathbf{A}^\ell = \sum_{\ell=0}^{\infty} (\beta \mathbf{A})^\ell - \mathbf{I}$$。当 $$\beta \rho(\mathbf{A}) < 1$$ 时，Neumann 级数 $$\sum_{\ell=0}^{\infty} (\beta \mathbf{A})^\ell = (\mathbf{I} - \beta \mathbf{A})^{-1}$$ 收敛。$$\blacksquare$$
{% endblock %}

{% hint style="warning" %}
⚠️ **易错点：Katz 指标的计算代价**

虽然 Katz 指标有优雅的矩阵闭式解（方程 10.64），但计算 $$(\mathbf{I} - \beta \mathbf{A})^{-1}$$ 需要 $$O(n^3)$$ 的矩阵求逆，对大规模网络不可行。实际中通常使用截断近似（只计算 $$\ell \leq L$$ 的路径）或迭代方法。对于百万节点级的网络，局部指标（CN、AA）因其 $$O(d^2)$$ 的计算复杂度而更实用。
{% endhint %}

## 10.6.4 方法对比

至此，我们介绍了两大类链接预测方法，它们体现了**局部性与全局性的根本权衡**——这一权衡贯穿整个图挖掘领域。

局部指标（CN、Jaccard、AA、PA）仅利用节点的直接邻域信息，计算代价与节点度的平方成正比（$$O(d^2)$$），在大规模稀疏网络上极其高效。然而，当两个节点处于不同的社区、没有公共邻居时，所有基于公共邻居的指标都给出零分——即使它们通过多条短路径相连、在全局拓扑上高度相关。这正是全局路径方法的用武之地：Katz 指标通过加权求和所有长度的路径来捕获间接连接，RWR 通过模拟随机游走来度量全局可达性，它们能发现局部指标遗漏的跨社区潜在连接。但代价是 $$O(n^3)$$ 或 $$O(n^2)$$ 的计算复杂度，限制了在大规模网络上的直接应用。

实际应用中的选择取决于网络规模和任务需求：对于社交网络的"好友推荐"（社区内预测为主），局部指标通常足够且实时性好；对于知识图谱补全（需要发现远距离的语义关联），全局方法或学习型方法（§10.7 GNN）更为适合。下表系统对比了各指标的关键特性。

| 指标 | 信息范围 | 时间复杂度 | 优势 | 局限 |
|------|---------|-----------|------|------|
| CN | 1-hop 邻居 | $$O(d^2)$$ | 简单高效 | 无公共邻居时为零 |
| Jaccard | 1-hop 邻居 | $$O(d^2)$$ | 度归一化 | 同上 |
| Adamic-Adar | 1-hop 邻居 | $$O(d^2)$$ | 区分邻居质量 | 同上 |
| PA | 0-hop（仅度） | $$O(1)$$ | 最快 | 忽略拓扑 |
| Katz | 全局路径 | $$O(n^3)$$ | 捕获全局结构 | 计算代价高 |
| RWR | 全局（随机游走） | $$O(n^2)$$ | 个性化评分 | 需要迭代收敛 |

## 10.6.5 链接预测评估

设计好了评分函数之后，一个同样重要的问题是：**如何公平地评估链接预测方法的性能？**这个问题比初看起来更具挑战性。

首先，链接预测面临极端的**类别不平衡**。一个有 $$n$$ 个节点的图最多有 $$n(n-1)/2$$ 个可能的边，而实际边数 $$|E|$$ 通常远小于这个上界——在稀疏的社交网络中，不存在边的节点对可能比已有边多 100 倍以上。如果一个模型简单地预测"所有节点对都不会形成边"，它的准确率（Accuracy）仍然可以超过 99%，但这样的模型毫无实用价值。因此，链接预测中**不能使用准确率**作为评估指标。

其次，评估需要模拟"已知过去、预测未来"的场景。标准做法是**隐藏一部分已有的边**（模拟"未观测到的边"），在剩余图上计算评分，然后检验被隐藏的边是否获得了比随机选取的不存在的边更高的评分。这本质上是一个**排序质量**的衡量——我们关心的不是分类阈值，而是正样本是否排在负样本前面。

**AUC**（Area Under the ROC Curve）恰好衡量了这种排序质量：它等于"随机选取一条真实的被隐藏边和一条不存在的边，前者的评分高于后者的概率"。AUC = 0.5 意味着模型的排序能力与随机猜测无异，AUC = 1.0 意味着所有正样本的评分都高于所有负样本。以下算法描述了完整的评估流程。

{% block type="algorithm" title="算法 10.7　链接预测评估" %}
**输入**：图 $$G = (V, E)$$，链接预测评分函数 $$s(\cdot, \cdot)$$，测试比例 $$r = 0.2$$

**输出**：AUC 评分

**步骤 1（数据划分）**：随机移除 $$r \cdot |E|$$ 条边作为测试集 $$E_{\text{test}}$$，剩余边作为训练集 $$E_{\text{train}}$$；从不存在的边中随机采样 $$|E_{\text{test}}|$$ 条作为负样本 $$E_{\text{neg}}$$

**步骤 2（评分计算）**：在训练图 $$G_{\text{train}} = (V, E_{\text{train}})$$ 上，对每个 $$(u, v) \in E_{\text{test}} \cup E_{\text{neg}}$$ 计算 $$s(u, v)$$

**步骤 3（AUC 计算）**：

$$
\text{AUC} = \frac{\sum_{(u,v) \in E_{\text{test}}} \sum_{(u',v') \in E_{\text{neg}}} \mathbb{1}[s(u,v) > s(u',v')]}{|E_{\text{test}}| \cdot |E_{\text{neg}}|} \tag{10.65}
$$

即随机选取一个正样本和一个负样本，正样本评分高于负样本的概率。AUC = 0.5 表示随机猜测，AUC = 1.0 表示完美预测。
{% endblock %}

## 10.6.6 案例分析

{% block type="case" title="案例 10.6　社交网络中的好友推荐" %}

**问题背景**：在社交网络中预测用户之间可能建立的新关系，是"你可能认识的人"推荐功能的核心技术。我们在 Zachary 空手道俱乐部网络上演示链接预测的完整流程。

**数据说明**：Karate Club 网络包含 34 个节点和 78 条边。随机移除 20% 的边（约 16 条）作为测试集，在剩余网络上进行链接预测。

**方法与步骤**：

```python
# --- 1. 加载数据与划分训练/测试集 ---
import networkx as nx
import numpy as np
np.random.seed(42)
G = nx.karate_club_graph()
edges = list(G.edges())
np.random.shuffle(edges)
n_test = int(0.2 * len(edges))
test_edges = edges[:n_test]
train_edges = edges[n_test:]
G_train = nx.Graph()
G_train.add_nodes_from(G.nodes())
G_train.add_edges_from(train_edges)
# --- 2. 生成负样本 ---
non_edges = list(nx.non_edges(G_train))
np.random.shuffle(non_edges)
neg_edges = non_edges[:n_test]
# --- 3. 计算局部相似度评分 ---
from sklearn.metrics import roc_auc_score
pairs = test_edges + neg_edges
y_true = [1]*len(test_edges) + [0]*len(neg_edges)
cn_scores = [len(list(nx.common_neighbors(G_train, u, v)))
             for u, v in pairs]
aa_scores = [sum(1/np.log(G_train.degree(w))
                 for w in nx.common_neighbors(G_train, u, v)
                 if G_train.degree(w) > 1)
             for u, v in pairs]
pa_scores = [G_train.degree(u) * G_train.degree(v)
             for u, v in pairs]
# --- 4. 计算 AUC 并输出 ---
for name, scores in [("CN", cn_scores), ("AA", aa_scores),
                     ("PA", pa_scores)]:
    auc_val = roc_auc_score(y_true, scores)
    print(f"{name}: AUC = {auc_val:.4f}")
```

**结果分析**：

1. **局部指标有效**：Adamic-Adar 在 Karate Club 上达到 AUC ≈ 0.85，仅利用 1-hop 邻居信息就能准确预测大部分缺失边——这得益于社交网络的强同质性（朋友的朋友也是朋友）。

2. **社区内预测更准**：预测正确率最高的边集中在社区内部——同一社区的节点共享更多公共邻居。跨社区的边（如节点 0 和节点 33 之间的"桥接边"）预测难度更大。

3. **度偏差效应**：优先连接（PA）倾向于预测高度节点之间的连接，在度分布不均匀的网络上有偏差。Jaccard 和 AA 通过归一化缓解了这一问题。

4. **实际应用**：链接预测的排序结果直接对应推荐列表——评分最高的 Top-K 节点对即为推荐候选。在工业级社交网络中，通常结合图嵌入（§10.7 GNN）和用户画像特征进一步提升预测精度。

{% endblock %}

## 习题

### A 级（概念理解）

**A1.** 解释链接预测与节点分类（§10.4）的区别。它们各自预测的对象是什么？

**A2.** 比较四种局部相似度指标的优缺点。在什么情况下优先连接（PA）会优于 Adamic-Adar？

**A3.** 为什么链接预测中通常使用 AUC 而不是准确率（Accuracy）作为评估指标？
