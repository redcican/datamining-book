# 10.5 图中频繁模式挖掘

前几节分别讨论了图的表示（§10.1）、节点与图的特征（§10.2–§10.4）、社区结构（§10.3）。这些方法关注的是**整体统计量**或**节点属性**。本节转向另一个核心问题：在一组图中发现**反复出现的子结构**——即**频繁子图挖掘**（Frequent Subgraph Mining）。

频繁子图挖掘是第二章频繁模式挖掘（§2.1–§2.6）在图数据上的自然推广。在事务数据中，频繁模式是反复出现的项集；在图数据中，频繁模式是反复出现的**子图**。这个推广带来了根本性的计算挑战：子图匹配（子图同构）是 NP 完全问题，搜索空间也从项的子集格扩展到了指数级的子图空间。

## 10.5.1 频繁子图挖掘的基本概念

### 10.5.1.1 子图与子图同构

{% block type="definition" title="定义 10.22　子图同构" %}
设 $$G = (V, E, \ell_V, \ell_E)$$ 和 $$g = (V', E', \ell_{V'}, \ell_{E'})$$ 为两个带标签的图（$$\ell_V, \ell_E$$ 分别为节点和边的标签函数）。称 $$g$$ 是 $$G$$ 的**子图同构像**（Subgraph Isomorphism），记为 $$g \subseteq_{\text{iso}} G$$，若存在单射 $$\varphi: V' \rightarrow V$$，满足：

1. **节点标签保持**：$$\forall v \in V', \ \ell_V(\varphi(v)) = \ell_{V'}(v)$$
2. **边结构保持**：$$\forall (u, v) \in E', \ (\varphi(u), \varphi(v)) \in E$$
3. **边标签保持**：$$\forall (u, v) \in E', \ \ell_E(\varphi(u), \varphi(v)) = \ell_{E'}(u, v)$$

子图同构判定是 **NP 完全**问题（Cook, 1971），这是频繁子图挖掘面临的基本计算障碍。
{% endblock %}

子图同构与第二章中项集的子集关系 $$I' \subseteq I$$ 类似，但复杂度截然不同：判断一个项集是否为另一个的子集只需 $$O(|I|)$$ 时间，而子图同构判定在最坏情况下需要指数时间。

### 10.5.1.2 支持度与频繁子图

{% block type="definition" title="定义 10.23　图支持度与频繁子图" %}
给定图数据库 $$\mathcal{D} = \{G_1, G_2, \ldots, G_N\}$$ 和子图模式 $$g$$，$$g$$ 在 $$\mathcal{D}$$ 中的**支持度**（Support）定义为包含 $$g$$ 的图的比例：

$$
\text{sup}(g, \mathcal{D}) = \frac{|\{G_i \in \mathcal{D} \mid g \subseteq_{\text{iso}} G_i\}|}{|\mathcal{D}|} \tag{10.48}
$$

给定最小支持度阈值 $$\sigma_{\min} \in (0, 1]$$，**频繁子图集合**为：

$$
\mathcal{F}(\mathcal{D}, \sigma_{\min}) = \{g \mid \text{sup}(g, \mathcal{D}) \geq \sigma_{\min}\} \tag{10.49}
$$

频繁子图挖掘的目标是高效地枚举 $$\mathcal{F}(\mathcal{D}, \sigma_{\min})$$ 中的所有子图。
{% endblock %}

{% hint style="warning" %}
⚠️ **易错点：基于图的支持度 vs 基于出现次数的支持度**

方程 (10.48) 使用的是"基于图"的支持度（Transaction-based Support）：一个模式在一个图中无论出现多少次，只计数一次。另一种定义是计算模式在所有图中出现的**总次数**，但这种定义不满足反单调性（一个模式的出现次数可能超过其子模式），因此在频繁子图挖掘中通常使用基于图的定义。
{% endhint %}

## 10.5.2 反单调性与搜索空间

### 10.5.2.1 反单调性

回顾 §2.2 中 Apriori 原理的核心——**反单调性**（Anti-monotonicity）：如果一个项集不是频繁的，那么它的所有超集也不是频繁的。这个性质在图数据上同样成立。

{% block type="theorem" title="定理 10.5　子图频率的反单调性" %}
设 $$g$$ 和 $$g'$$ 为两个图模式。若 $$g \subseteq_{\text{iso}} g'$$（$$g$$ 是 $$g'$$ 的子图），则：

$$
\text{sup}(g', \mathcal{D}) \leq \text{sup}(g, \mathcal{D}) \tag{10.50}
$$

即超图的支持度不超过其子图的支持度。

**证明**　若 $$g \subseteq_{\text{iso}} g'$$ 且 $$g' \subseteq_{\text{iso}} G_i$$（即 $$g'$$ 出现在 $$G_i$$ 中），由子图同构的传递性可得 $$g \subseteq_{\text{iso}} G_i$$。因此 $$\{G_i \mid g' \subseteq_{\text{iso}} G_i\} \subseteq \{G_i \mid g \subseteq_{\text{iso}} G_i\}$$，从而 $$\text{sup}(g') \leq \text{sup}(g)$$。$$\blacksquare$$
{% endblock %}

反单调性意味着可以采用与 Apriori 类似的**逐层搜索**策略：先找频繁的小子图，再在此基础上扩展为更大的子图。一旦某个子图不频繁，就无需再考虑包含它的更大子图。

### 10.5.2.2 搜索空间的挑战

尽管反单调性提供了剪枝能力，子图搜索空间仍远比项集空间复杂。对于 $$n$$ 个不同标签的节点：

$$
\text{可能的连通子图数量} = O\!\left(\sum_{k=1}^{n} \binom{n}{k} \cdot 2^{\binom{k}{2}}\right) \tag{10.51}
$$

这个数量随节点数**双指数增长**。实际算法必须在搜索策略和剪枝技术上做出精心设计。

频繁子图挖掘的两大主流策略是：

| 策略 | 代表算法 | 核心思想 | 类比 |
|------|---------|---------|------|
| **Apriori 策略** | AGM、FSG | 逐层生成候选，检验频率 | 类似 §2.2 Apriori |
| **模式增长策略** | gSpan、MoFa | 从小模式逐步扩展，无候选生成 | 类似 §2.4 FP-Growth |

## 10.5.3 基于 Apriori 的方法

基于 Apriori 的频繁子图挖掘方法遵循**候选生成-检验**（Candidate Generation and Testing）范式，与 §2.2 中的 Apriori 算法结构相同。

**基本流程**：
1. **初始化**：找出所有频繁的单节点和单边子图；
2. **候选生成**：将两个 $$k$$-边频繁子图合并，生成 $$(k+1)$$-边候选子图；
3. **频率检验**：对每个候选子图，通过子图同构检验计算其支持度；
4. **剪枝**：丢弃不频繁的候选，保留频繁子图；
5. **迭代**：重复步骤 2–4，直到无法生成新的频繁子图。

代表性算法包括：

- **AGM**（Apriori-based Graph Mining，Inokuchi et al., 2000）：使用邻接矩阵表示图，通过矩阵连接操作生成候选；
- **FSG**（Frequent Subgraph Discovery，Kuramochi & Karypis, 2001）：使用边列表表示，优化候选生成和同构检验。

{% hint style="info" %}
🤔 **思考：Apriori 策略的局限性**

Apriori 策略在频繁子图挖掘中面临两个严重瓶颈：(1) **候选爆炸**——两个 $$k$$-边子图可能合并出大量非同构的 $$(k+1)$$-边候选，数量远超项集的候选数；(2) **同构检验代价**——每次频率检验都需要执行子图同构测试（NP 完全），而不像项集支持度计算那样只需线性扫描。这两个问题促使研究者提出了**模式增长**策略。
{% endhint %}

## 10.5.4 gSpan 算法

gSpan（graph-based Substructure Pattern mining，Yan & Han, 2002）是最具影响力的频繁子图挖掘算法。它采用**模式增长**策略，通过定义图的**规范编码**来避免冗余搜索，无需显式生成候选。

### 10.5.4.1 DFS 编码

gSpan 的核心创新是将图表示为唯一的**DFS 编码**（DFS Code），从而将图的同构判定转化为编码序列的比较。

{% block type="definition" title="定义 10.24　DFS 编码" %}
对图 $$g$$ 执行深度优先搜索，得到 DFS 树 $$T$$。DFS 编码是按 DFS 发现顺序记录的**边序列** $$\text{code}(g, T) = (e_1, e_2, \ldots, e_m)$$，其中每条边 $$e_k$$ 表示为五元组：

$$
e_k = (i_k, j_k, \ell_{V}(v_{i_k}), \ell_{E}(v_{i_k}, v_{j_k}), \ell_{V}(v_{j_k})) \tag{10.52}
$$

其中 $$i_k, j_k$$ 为节点在 DFS 树中的发现序号（0-based），$$\ell_V, \ell_E$$ 分别为节点和边的标签。

边分为两类：**前向边**（Forward Edge，$$i_k < j_k$$，发现新节点）和**后向边**（Backward Edge，$$i_k > j_k$$，连接到已访问节点）。
{% endblock %}

一个图可能有多棵不同的 DFS 树，因此可能有多个 DFS 编码。gSpan 通过定义编码的**字典序**，选择**最小 DFS 编码**作为规范表示。

{% block type="definition" title="定义 10.25　最小 DFS 编码" %}
在所有 DFS 编码上定义字典序 $$\preceq$$：对两个 DFS 编码 $$\alpha = (e_1, \ldots, e_p)$$ 和 $$\beta = (e_1', \ldots, e_q')$$，$$\alpha \prec \beta$$ 当且仅当存在 $$t$$ 使得 $$e_i = e_i'$$（$$i < t$$）且 $$e_t \prec e_t'$$（按五元组的字典序比较）。

图 $$g$$ 的**最小 DFS 编码** $$\text{min-code}(g)$$ 是其所有 DFS 编码中字典序最小的：

$$
\text{min-code}(g) = \min_{\preceq} \{\text{code}(g, T) \mid T \text{ 是 } g \text{ 的 DFS 树}\} \tag{10.53}
$$

最小 DFS 编码具有**规范性**：两个图同构当且仅当它们的最小 DFS 编码相同。
{% endblock %}

最小 DFS 编码的规范性保证了 gSpan 不会重复搜索同构的子图——每个子图只会以其最小 DFS 编码的形式被发现一次。

### 10.5.4.2 右最扩展

gSpan 通过**右最扩展**（Rightmost Extension）策略逐步增长模式。设当前模式的 DFS 树的右最路径（Rightmost Path）为从根到最后发现节点的路径。扩展只在右最路径上的节点进行：

$$
\text{扩展规则：向右最路径上的节点添加一条新边（前向边或后向边）} \tag{10.54}
$$

右最扩展保证了每个子图恰好被枚举一次——它等价于在 DFS 编码的字典序空间中进行有序搜索。

### 10.5.4.3 算法流程

{% block type="algorithm" title="算法 10.6　gSpan 频繁子图挖掘" %}
**输入**：图数据库 $$\mathcal{D} = \{G_1, \ldots, G_N\}$$，最小支持度 $$\sigma_{\min}$$

**输出**：频繁子图集合 $$\mathcal{F}$$

**步骤 1（初始化）**：扫描 $$\mathcal{D}$$，找出所有频繁的单边子图 $$\mathcal{S}_1$$

**步骤 2（模式增长）**：对每个频繁单边 $$e \in \mathcal{S}_1$$：
$$\quad$$ 调用 $$\textsc{SubgraphMining}(\mathcal{D}, e, \sigma_{\min})$$

**子过程 $$\textsc{SubgraphMining}(\mathcal{D}, g, \sigma_{\min})$$**：
$$\quad$$ **if** $$\text{code}(g) \neq \text{min-code}(g)$$ **then return** $$\quad$$ $$\triangleright$$ 剪枝：非规范编码
$$\quad$$ 将 $$g$$ 加入 $$\mathcal{F}$$
$$\quad$$ **for each** 右最扩展 $$g' = g \oplus e'$$ **do**
$$\quad\quad$$ 计算 $$\text{sup}(g', \mathcal{D})$$
$$\quad\quad$$ **if** $$\text{sup}(g') \geq \sigma_{\min}$$ **then**
$$\quad\quad\quad$$ $$\textsc{SubgraphMining}(\mathcal{D}, g', \sigma_{\min})$$

**复杂度**：每个频繁子图被枚举恰好一次；主要开销在子图同构检验上。实际运行速度通常比 Apriori 方法快一个数量级。
{% endblock %}

## 10.5.5 闭频繁子图与最大频繁子图

当支持度阈值较低时，频繁子图的数量可能非常庞大。与 §2.3 中闭频繁项集的思想类似，可以通过**压缩表示**减少输出规模。

{% block type="definition" title="定义 10.26　闭频繁子图与最大频繁子图" %}

**闭频繁子图**（Closed Frequent Subgraph）：频繁子图 $$g$$ 是闭的，当且仅当不存在真超图 $$g'$$（$$g \subsetneq_{\text{iso}} g'$$）使得 $$\text{sup}(g') = \text{sup}(g)$$：

$$
\nexists \, g' \supsetneq_{\text{iso}} g: \ \text{sup}(g', \mathcal{D}) = \text{sup}(g, \mathcal{D}) \tag{10.55}
$$

**最大频繁子图**（Maximal Frequent Subgraph）：频繁子图 $$g$$ 是最大的，当且仅当不存在频繁的真超图：

$$
\nexists \, g' \supsetneq_{\text{iso}} g: \ \text{sup}(g', \mathcal{D}) \geq \sigma_{\min} \tag{10.56}
$$

三者关系：最大频繁子图 $$\subseteq$$ 闭频繁子图 $$\subseteq$$ 频繁子图。闭频繁子图保留了完整的支持度信息（无损压缩），最大频繁子图丢失了部分支持度信息（有损压缩）。
{% endblock %}

代表性算法包括 CloseGraph（闭频繁子图，Yan & Han, 2003）和 SPIN（最大频繁子图）。在实际应用中，闭频繁子图是最常用的压缩表示，因为它在大幅减少输出数量的同时不丢失任何信息。

## 10.5.6 方法对比

频繁子图挖掘算法的发展历程与频繁项集挖掘（§2.1–§2.4）高度平行：从最初的 Apriori 风格的逐层候选生成-检验方法（AGM、FSG），到后来的模式增长方法（gSpan），再到针对输出冗余的压缩表示方法（CloseGraph）。但图数据的特殊性——子图同构的 NP 完全性和搜索空间的双指数增长——使得图上的每一步改进都比项集上更为关键。gSpan 通过最小 DFS 编码将同构判定转化为编码比较，配合右最扩展实现无冗余搜索，成为该领域的事实标准。而 GASTON 则另辟蹊径，利用"路径→树→图"的结构层次逐步增加搜索空间的复杂度。下表比较了各主要算法的核心策略与适用场景。

| 方法 | 策略 | 规范形式 | 优势 | 局限 |
|------|------|---------|------|------|
| AGM | Apriori（逐层） | 邻接矩阵 | 实现简单 | 候选爆炸、同构检验慢 |
| FSG | Apriori（逐层） | 规范标签 | 优化剪枝 | 仍受候选数限制 |
| gSpan | 模式增长 | 最小 DFS 编码 | 无候选、高效 | 密集图上 DFS 编码检验代价高 |
| CloseGraph | 模式增长 | 最小 DFS 编码 | 输出压缩 | 额外的闭合检验开销 |
| GASTON | 混合 | 路径→树→图 | 利用结构层次 | 实现复杂 |

## 10.5.7 案例分析

{% block type="case" title="案例 10.5　合成分子图的频繁子结构发现" %}

**问题背景**：在化学信息学中，分子可以表示为图——原子是节点（标签为元素符号），化学键是边（标签为键类型）。发现频繁出现的子结构（如官能团）有助于理解化合物的共同特征，是药物发现和毒性预测的重要工具。我们用合成数据演示频繁子图发现的基本流程。

**数据说明**：生成 100 个合成分子图，每个图包含 8–15 个节点（元素：C, N, O, S），并在其中嵌入几种常见子结构（三角形、路径、星形），使这些子结构以不同频率出现。

**方法与步骤**：

```python
# --- 1. 生成合成分子图 ---
import networkx as nx
import numpy as np
np.random.seed(42)
elements = ['C', 'N', 'O', 'S']
graphs = []
for _ in range(100):
    n = np.random.randint(8, 16)
    G = nx.erdos_renyi_graph(n, 0.25)
    for node in G.nodes():
        G.nodes[node]['label'] = np.random.choice(elements, p=[0.5, 0.2, 0.2, 0.1])
    graphs.append(G)
# --- 2. 嵌入频繁子结构 ---
# 在 80% 的图中嵌入三角形 (C-C-C)
for G in graphs[:80]:
    nodes = list(G.nodes())[:3]
    for u, v in [(nodes[0], nodes[1]), (nodes[1], nodes[2]), (nodes[0], nodes[2])]:
        G.add_edge(u, v)
        G.nodes[u]['label'] = 'C'
# --- 3. 枚举子图并统计支持度 ---
# 对每个图检查是否包含特定模式
def has_triangle(G):
    """检查图是否包含三角形"""
    for node in G.nodes():
        neighbors = list(G.neighbors(node))
        for i in range(len(neighbors)):
            for j in range(i+1, len(neighbors)):
                if G.has_edge(neighbors[i], neighbors[j]):
                    return True
    return False
triangle_support = sum(has_triangle(G) for G in graphs) / len(graphs)
print(f"三角形支持度: {triangle_support:.2f}")
```

**结果分析**：

1. **子结构频率差异**：嵌入的三角形子结构在约 95% 的图中出现（高支持度），而四节点完全图等复杂模式的支持度随阈值降低急剧下降——体现了反单调性。

2. **支持度分布**：小模式（单边、路径）的支持度普遍较高，大模式的支持度迅速下降。这种**长尾分布**与项集频率分布（§2.1）具有相同的特征。

3. **闭子图压缩**：在最小支持度 30% 下，频繁子图有数百个，而闭频繁子图只有十几个——压缩比超过 90%。

4. **应用价值**：发现的高频子结构（如三角形、特定路径）对应化学中的**官能团**。它们可以作为分子分类的特征（§10.4），也可以用于新化合物的结构预测。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 子图同构（定义 10.22） | 保持节点/边标签的结构映射，NP 完全问题 |
| 频繁子图（定义 10.23） | 支持度 $$\geq \sigma_{\min}$$ 的子图模式 |
| 反单调性（定理 10.5） | 超图支持度 $$\leq$$ 子图支持度，允许剪枝搜索空间 |
| DFS 编码（定义 10.24–10.25） | 图的规范表示，将同构判定转化为编码比较 |
| gSpan（算法 10.6） | 模式增长 + 最小 DFS 编码 + 右最扩展，高效无候选 |
| 闭/最大频繁子图（定义 10.26） | 压缩频繁子图集的无损/有损表示 |

## 习题

### A 级（概念理解）

**A1.** 解释子图同构的定义。为什么子图同构判定是 NP 完全的，而子集判定只需线性时间？

**A2.** 频繁子图挖掘中的"基于图的支持度"和"基于出现次数的支持度"有何区别？为什么前者满足反单调性而后者不一定？

**A3.** 比较基于 Apriori 和基于模式增长的频繁子图挖掘策略。它们分别对应 §2.2 和 §2.4 中的哪些算法？

**A4.** 解释闭频繁子图和最大频繁子图的区别。哪个保留了更多信息？

### B 级（数学推导）

**B1.** 证明反单调性（定理 10.5）。进一步说明为什么基于出现次数的支持度不满足反单调性（构造反例）。

**B2.** 给定一个 4 节点环图（$$v_0 - v_1 - v_2 - v_3 - v_0$$，所有节点标签为 A，所有边标签为 1），写出其所有可能的 DFS 编码，并找出最小 DFS 编码。

**B3.** 设图数据库包含 $$N$$ 个图，每个图有 $$n$$ 个节点，图密度为 $$\rho$$。分析在这些图上进行一次子图同构检验的平均时间复杂度（与 $$n$$ 和 $$\rho$$ 的关系）。

### C 级（数值计算）

**C1.** 对以下 4 个图组成的数据库：$$G_1 = K_3$$（三角形），$$G_2 = P_4$$（4 节点路径），$$G_3 = C_4$$（4 节点环），$$G_4 = K_3$$ 加一条挂边。计算以下模式的支持度：(1) 单条边；(2) 路径 $$P_3$$；(3) 三角形 $$K_3$$。

**C2.** 对 C1 中的数据库，设 $$\sigma_{\min} = 0.5$$，列出所有频繁子图。哪些是闭频繁子图？哪些是最大频繁子图？

**C3.** 对三角形 $$K_3$$（节点标签全为 A，边标签全为 1），列出所有 DFS 树和对应的 DFS 编码，确定最小 DFS 编码。

### D 级（编程实践）

**D1.** 实现一个简单的频繁子图枚举器：给定一组 NetworkX 图，枚举所有频繁的 $$k$$-边连通子图（$$k \leq 3$$）。使用暴力子图同构检验（NetworkX 的 `is_isomorphic` 或 `GraphMatcher`）。

**D2.** 生成 200 个合成图（3 类结构：环形、树形、随机），在其中嵌入不同的子结构模式。统计各模式的支持度，绘制支持度分布直方图。

**D3.** 实现 gSpan 的简化版本（限制模式大小 $$\leq 4$$ 条边），在合成图数据库上测试。比较不同 $$\sigma_{\min}$$ 值下发现的频繁子图数量。

### E 级（案例分析）

**E1.** 在 NetworkX 内置的社交网络（如 `karate_club_graph`）上，枚举所有频繁三角形、四边形和五边形子图。分析哪些频繁子图与社区结构（§10.3）相关。

**E2.** 生成合成"分子图"数据集（原子标签 C/N/O/S，随机连边），嵌入特定的官能团模式。实现频繁子图发现，验证能否准确恢复嵌入的模式。

### F 级（开放探讨）

**F1.** **频繁子图挖掘的可扩展性**：当前的频繁子图挖掘算法在大规模图数据库（百万级图、千节点级图）上仍面临可扩展性挑战。讨论：(1) 近似算法（随机采样、概率计数）如何加速频繁子图挖掘？(2) 分布式框架（MapReduce、Spark）如何并行化 gSpan？(3) 深度学习方法是否可以替代传统的枚举方法？

**F2.** **从频繁子图到图神经网络**：频繁子图可以作为 GNN 的先验知识。讨论：(1) 如何将频繁子图用作 GNN 的特征（如将每个节点是否属于某频繁子图作为特征）？(2) WL 子树核（§10.2）与频繁子图挖掘有何联系？(3) GNN 的聚合机制是否隐式地发现了频繁子结构？
