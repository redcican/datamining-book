# 10.1 图数据表示与存储

第九章介绍了**文本**这种非结构化数据的挖掘方法。本章转向另一种广泛存在且具有丰富结构的数据类型：**图**（Graph）。

图是描述**实体之间关系**的通用数学工具——社交网络中的好友关系、引文网络中的引用关系、交通网络中的路线连接、蛋白质相互作用网络中的结合关系、知识图谱中的语义关联，这些现实世界的关系数据天然地以图的形式存在。与表格数据的独立同分布假设不同，图数据的核心特征是**节点之间的依赖关系**——一个节点的属性和行为受其邻居节点的强烈影响。

本节是图数据挖掘的起点——**如何用数学语言精确地表示图？** 正如文本挖掘需要先将文本转化为向量（§9.2），图挖掘的第一步是建立图的形式化表示和高效的存储结构。图的矩阵表示不仅是存储工具，更是后续谱分析（§10.2）、社区发现（§10.3）和图神经网络（§10.7）的数学基础。

## 10.1.1 图的基本概念

### 10.1.1.1 从关系数据到图

{% block type="definition" title="定义 10.1　图的基本术语" %}
**图**（Graph）是由顶点集合和边集合组成的数学结构，用于描述实体之间的关系：

**无向图**：$$G = (V, E)$$，其中 $$V = \{v_1, v_2, \ldots, v_n\}$$ 为**顶点集**（Vertex Set），$$E \subseteq \{(v_i, v_j) : v_i, v_j \in V, i \neq j\}$$ 为**边集**（Edge Set）。无向边 $$(v_i, v_j) = (v_j, v_i)$$。

**有向图**：$$G = (V, E)$$，其中边 $$(v_i, v_j)$$ 有方向，表示从 $$v_i$$ 指向 $$v_j$$ 的关系，且 $$(v_i, v_j) \neq (v_j, v_i)$$。

**加权图**：$$G = (V, E, w)$$，其中权函数 $$w: E \rightarrow \mathbb{R}^+$$ 为每条边赋予正实数权重：

$$
w_{ij} = w(v_i, v_j) \geq 0 \tag{10.1}
$$

$$|V| = n$$ 称为图的**阶**（Order），$$|E| = m$$ 称为图的**大小**（Size）。
{% endblock %}

图的分类取决于应用场景：社交网络中的好友关系是**无向**的（A 是 B 的朋友，则 B 也是 A 的朋友）；引文网络中的引用关系是**有向**的（论文 A 引用论文 B，但 B 未必引用 A）；交通网络中的路段是**加权**的（不同路段有不同的距离或行驶时间）。

### 10.1.1.2 节点度与度序列

{% block type="definition" title="定义 10.2　节点度与度序列" %}
在无向图 $$G = (V, E)$$ 中，顶点 $$v$$ 的**度**（Degree）是与其相连的边的数量：

$$
d(v) = |\{u \in V : (v, u) \in E\}| \tag{10.2}
$$

在有向图中，区分**入度**（In-degree）和**出度**（Out-degree）：

$$
d_{\text{in}}(v) = |\{u \in V : (u, v) \in E\}|, \quad d_{\text{out}}(v) = |\{u \in V : (v, u) \in E\}| \tag{10.3}
$$

将所有顶点的度按降序排列得到**度序列**（Degree Sequence）$$(d_1 \geq d_2 \geq \cdots \geq d_n)$$。图的**度分布**（Degree Distribution）$$P(k) = |\{v \in V : d(v) = k\}| / n$$ 描述了度值的概率分布。
{% endblock %}

图论中的一条基本性质——**握手定理**：无向图中所有节点的度之和等于边数的 2 倍：

$$
\sum_{v \in V} d(v) = 2m \tag{10.4}
$$

这是因为每条边为两个端点各贡献 1 度。这个简单的等式在验证图数据完整性和推导图的其他性质时非常有用。

## 10.1.2 图的矩阵表示

矩阵是图的代数表示，它将图的拓扑结构转化为线性代数对象，使得图论问题可以用矩阵运算来求解。

### 10.1.2.1 邻接矩阵

{% block type="definition" title="定义 10.3　邻接矩阵" %}
图 $$G = (V, E)$$ 的**邻接矩阵**（Adjacency Matrix）$$\mathbf{A} \in \mathbb{R}^{n \times n}$$ 定义为：

$$
A_{ij} = \begin{cases} w_{ij} & \text{if } (v_i, v_j) \in E \\ 0 & \text{otherwise} \end{cases} \tag{10.5}
$$

对于无权图，$$w_{ij} = 1$$。无向图的邻接矩阵是**对称的**：$$\mathbf{A} = \mathbf{A}^T$$。

邻接矩阵的幂 $$\mathbf{A}^k$$ 具有重要的组合意义：$$(\mathbf{A}^k)_{ij}$$ 表示从 $$v_i$$ 到 $$v_j$$ 的**长度为 $$k$$ 的路径数**。
{% endblock %}

邻接矩阵的幂次性质提供了强大的分析工具——例如，$$\text{tr}(\mathbf{A}^3) / 6$$ 计算了图中三角形的数量（每个三角形在迹的求和中被计数 6 次——3 个起点 × 2 个方向）。这种"将图问题转化为矩阵运算"的思路贯穿本章。

### 10.1.2.2 度矩阵与拉普拉斯矩阵

{% block type="definition" title="定义 10.4　度矩阵与拉普拉斯矩阵" %}
**度矩阵**（Degree Matrix）$$\mathbf{D} \in \mathbb{R}^{n \times n}$$ 是以节点度为对角元素的对角矩阵：

$$
D_{ii} = d(v_i) = \sum_{j=1}^{n} A_{ij}, \quad D_{ij} = 0 \text{ (}i \neq j\text{)} \tag{10.6}
$$

**拉普拉斯矩阵**（Laplacian Matrix）定义为度矩阵与邻接矩阵的差：

$$
\mathbf{L} = \mathbf{D} - \mathbf{A} \tag{10.7}
$$

其元素为：

$$
L_{ij} = \begin{cases} d(v_i) & \text{if } i = j \\ -1 & \text{if } (v_i, v_j) \in E \\ 0 & \text{otherwise} \end{cases} \tag{10.8}
$$

**归一化拉普拉斯矩阵**（Normalized Laplacian）定义为：

$$
\mathbf{L}_{\text{sym}} = \mathbf{D}^{-1/2} \mathbf{L} \mathbf{D}^{-1/2} = \mathbf{I} - \mathbf{D}^{-1/2} \mathbf{A} \mathbf{D}^{-1/2} \tag{10.9}
$$

其中 $$\mathbf{D}^{-1/2}$$ 为度矩阵的逆平方根。
{% endblock %}

拉普拉斯矩阵是图的谱分析的核心工具——它的特征值和特征向量编码了图的全局结构信息。

{% block type="theorem" title="定理 10.1　拉普拉斯矩阵的谱性质" %}
设 $$G = (V, E)$$ 为无向图，$$\mathbf{L} = \mathbf{D} - \mathbf{A}$$ 为其拉普拉斯矩阵，特征值为 $$0 = \lambda_1 \leq \lambda_2 \leq \cdots \leq \lambda_n$$，则：

1. **半正定性**：对任意向量 $$\mathbf{x} \in \mathbb{R}^n$$，$$\mathbf{x}^T \mathbf{L} \mathbf{x} = \sum_{(v_i, v_j) \in E} (x_i - x_j)^2 \geq 0$$；
2. **最小特征值**：$$\lambda_1 = 0$$，对应的特征向量为全 1 向量 $$\mathbf{1}$$；
3. **连通性**：零特征值的重数等于图的**连通分量数** $$c$$，即 $$\lambda_1 = \cdots = \lambda_c = 0 < \lambda_{c+1}$$；
4. **代数连通度**：第二小特征值 $$\lambda_2$$（Fiedler 值）度量了图的连通强度——$$\lambda_2$$ 越大，图越"紧密连通"。

**分析**　性质 1 的证明直接展开即可：$$\mathbf{x}^T \mathbf{L} \mathbf{x} = \mathbf{x}^T (\mathbf{D} - \mathbf{A}) \mathbf{x} = \sum_i d(v_i) x_i^2 - 2 \sum_{(v_i,v_j) \in E} x_i x_j = \sum_{(v_i,v_j) \in E} (x_i^2 + x_j^2 - 2x_i x_j) = \sum_{(v_i,v_j) \in E} (x_i - x_j)^2 \geq 0$$。这个二次型的物理意义是：如果将 $$x_i$$ 视为节点 $$v_i$$ 的"信号值"，则 $$\mathbf{x}^T \mathbf{L} \mathbf{x}$$ 衡量了信号在相邻节点间的**总变差**——变差越小，信号越"平滑"。性质 3 说明，连通图恰好有一个零特征值。这些性质是谱聚类（§10.3）和图卷积网络（§10.7）的数学基础。$$\blacksquare$$
{% endblock %}

### 10.1.2.3 关联矩阵

{% block type="definition" title="定义 10.5　关联矩阵" %}
无向图 $$G = (V, E)$$ 的**关联矩阵**（Incidence Matrix）$$\mathbf{B} \in \mathbb{R}^{n \times m}$$（$$n$$ 为顶点数，$$m$$ 为边数）定义为：

$$
B_{ik} = \begin{cases} 1 & \text{if vertex } v_i \text{ is an endpoint of edge } e_k \\ 0 & \text{otherwise} \end{cases} \tag{10.10}
$$

对于有向图，若为边 $$e_k = (v_i, v_j)$$ 分配方向，则 $$B_{ik} = +1$$（起点），$$B_{jk} = -1$$（终点）。

关联矩阵与拉普拉斯矩阵之间存在优美的关系：

$$
\mathbf{L} = \mathbf{B} \mathbf{B}^T \tag{10.11}
$$

这再次证明了拉普拉斯矩阵的半正定性（$$\mathbf{x}^T \mathbf{L} \mathbf{x} = \mathbf{x}^T \mathbf{B} \mathbf{B}^T \mathbf{x} = \|\mathbf{B}^T \mathbf{x}\|^2 \geq 0$$）。
{% endblock %}

{% hint style="info" %}
🤔 **思考：四种矩阵表示各自适用于什么场景？**

邻接矩阵 $$\mathbf{A}$$ 是最直观的表示，适合稠密图和需要快速查询边是否存在的场景。度矩阵 $$\mathbf{D}$$ 提供了节点级别的统计信息。拉普拉斯矩阵 $$\mathbf{L}$$ 是谱分析的核心——其特征值编码了图的全局结构，用于谱聚类（§10.3）和半监督学习。归一化拉普拉斯矩阵 $$\mathbf{L}_{\text{sym}}$$ 的特征值被约束在 $$[0, 2]$$ 范围内，使得不同大小的图具有可比较的谱特征，是图卷积网络（§10.7）的标准输入。
{% endhint %}

## 10.1.3 图的存储结构

### 10.1.3.1 邻接矩阵 vs 邻接表

图的矩阵表示虽然数学上优美，但在实际存储中需要考虑效率。邻接矩阵的存储空间为 $$O(n^2)$$，对于稀疏图（$$m \ll n^2$$）中大量的零元素是浪费的。

{% block type="definition" title="定义 10.6　图的存储结构" %}
图的两种基本存储结构：

**邻接矩阵存储**：使用 $$n \times n$$ 二维数组直接存储。
- 空间复杂度：$$O(n^2)$$
- 查询边 $$(v_i, v_j)$$ 是否存在：$$O(1)$$
- 遍历节点 $$v_i$$ 的所有邻居：$$O(n)$$

**邻接表存储**：每个节点维护一个链表（或动态数组），存储其所有邻居。
- 空间复杂度：$$O(n + m)$$
- 查询边 $$(v_i, v_j)$$ 是否存在：$$O(d(v_i))$$
- 遍历节点 $$v_i$$ 的所有邻居：$$O(d(v_i))$$

图的**密度**定义为实际边数与最大可能边数的比值：

$$
\rho = \frac{2m}{n(n-1)} \tag{10.12}
$$

当 $$\rho > 0.5$$ 时，邻接矩阵更优；当 $$\rho < 0.01$$ 时（如社交网络、引文网络），邻接表或 CSR（压缩稀疏行）格式显著更优。
{% endblock %}

### 10.1.3.2 BFS 遍历与连通性检测

图遍历是图分析的基本操作。广度优先搜索（BFS）从一个起始节点出发，逐层访问其邻居节点，可以用于检测图的连通性、计算最短路径等。

{% block type="algorithm" title="算法 10.1　广度优先搜索（BFS）" %}
**输入**：图 $$G = (V, E)$$（邻接表表示），起始节点 $$s$$

**输出**：从 $$s$$ 可达的所有节点及其距离 $$\text{dist}[v]$$

**步骤 1（初始化）**：$$\text{visited} \leftarrow \{s\}$$，$$\text{dist}[s] \leftarrow 0$$，队列 $$Q \leftarrow [s]$$

**步骤 2（逐层扩展）**：
$$\quad$$ **while** $$Q \neq \emptyset$$ **do**
$$\quad\quad$$ $$v \leftarrow Q.\text{dequeue}()$$
$$\quad\quad$$ **for each** $$u \in \text{Neighbors}(v)$$ **do**
$$\quad\quad\quad$$ **if** $$u \notin \text{visited}$$ **then**
$$\quad\quad\quad\quad$$ $$\text{visited}.\text{add}(u)$$
$$\quad\quad\quad\quad$$ $$\text{dist}[u] \leftarrow \text{dist}[v] + 1$$
$$\quad\quad\quad\quad$$ $$Q.\text{enqueue}(u)$$
$$\quad$$ **end while**

**步骤 3（返回）**：返回 $$\text{visited}$$, $$\text{dist}$$

**复杂度**：时间 $$O(n + m)$$，空间 $$O(n)$$。
{% endblock %}

BFS 的时间复杂度 $$O(n + m)$$ 与图的邻接表存储完美匹配——每个节点入队一次，每条边被检查两次（在无向图中）。如果使用邻接矩阵存储，遍历的时间复杂度会退化为 $$O(n^2)$$。

{% hint style="warning" %}
⚠️ **易错点：大规模图的存储选择**

真实世界的图通常非常稀疏——Facebook 社交网络有约 30 亿用户（节点），但平均每人只有约 300 个好友（平均度 $$\bar{d} \approx 300$$），密度 $$\rho \approx 10^{-7}$$。如果使用邻接矩阵存储，需要 $$30 \times 10^8 \times 30 \times 10^8 = 9 \times 10^{18}$$ 个元素——约 72 EB 的存储空间。而邻接表只需要 $$O(n \cdot \bar{d}) \approx 9 \times 10^{11}$$ 个元素——约 7.2 TB，可行得多。因此，对于大规模稀疏图，**永远不要使用邻接矩阵**。在 Python 中，scipy.sparse 的 CSR 格式是处理大规模稀疏图的标准选择。
{% endhint %}

## 10.1.4 图数据的统计特征

真实世界的图具有一系列独特的统计特征，这些特征区分了真实图与随机图（Erdős–Rényi 模型），也为图挖掘算法的设计提供了指导。

### 10.1.4.1 度分布与幂律

大多数真实网络的度分布不是均匀的或正态的，而是服从**幂律分布**（Power-law Distribution）：

$$
P(k) \propto k^{-\gamma}, \quad \gamma \in [2, 3] \tag{10.13}
$$

即少数节点拥有大量连接（**枢纽节点**，hub），大多数节点只有少量连接。这种"富者愈富"的现象在社交网络（少数意见领袖有数百万粉丝）、万维网（少数网站获得大量链接）、引文网络（少数论文被大量引用）中普遍存在，称为**无标度网络**（Scale-free Network，Barabási & Albert, 1999）。

### 10.1.4.2 聚集系数与小世界性质

{% block type="definition" title="定义 10.7　聚集系数" %}
节点 $$v$$ 的**局部聚集系数**（Local Clustering Coefficient）衡量了其邻居之间互相连接的程度：

$$
C(v) = \frac{2 |\{(u, w) \in E : u, w \in N(v)\}|}{d(v)(d(v) - 1)} \tag{10.14}
$$

其中 $$N(v)$$ 为 $$v$$ 的邻居集合。$$C(v) \in [0, 1]$$——当 $$C(v) = 1$$ 时，$$v$$ 的所有邻居互相认识；当 $$C(v) = 0$$ 时，$$v$$ 的邻居之间没有任何连接。

图的**全局聚集系数**为所有节点的平均值：

$$
\bar{C} = \frac{1}{n} \sum_{v \in V} C(v) \tag{10.15}
$$

图的**平均最短路径长度**定义为所有节点对之间最短路径的平均值：

$$
\bar{L} = \frac{1}{n(n-1)} \sum_{i \neq j} d(v_i, v_j) \tag{10.16}
$$

其中 $$d(v_i, v_j)$$ 为节点 $$v_i$$ 到 $$v_j$$ 的最短路径长度。
{% endblock %}

**小世界性质**（Small-world Property，Watts & Strogatz, 1998）是指真实网络同时具有：(1) **高聚集系数**（$$\bar{C}$$ 远大于等规模随机图的聚集系数 $$C_{\text{rand}} = \bar{d}/n$$）；(2) **短平均路径长度**（$$\bar{L}$$ 与等规模随机图的 $$L_{\text{rand}} = \ln n / \ln \bar{d}$$ 接近）。直觉上，"你的朋友也互相认识"（高聚集系数），但"六度分隔"就能连接任意两个人（短路径长度）。

{% hint style="tip" %}
🔧 **工程实践：NetworkX 图分析**

NetworkX 是 Python 中最流行的图分析库，提供了丰富的图构建、分析和可视化功能：

```python
import networkx as nx
# --- 1. 构建图 ---
G = nx.karate_club_graph()  # Zachary 空手道俱乐部
# --- 2. 基本统计 ---
print(f"节点数: {G.number_of_nodes()}")
print(f"边数: {G.number_of_edges()}")
print(f"平均度: {sum(dict(G.degree()).values()) / G.number_of_nodes():.2f}")
# --- 3. 度分布 ---
degrees = [d for _, d in G.degree()]
print(f"最大度: {max(degrees)}, 最小度: {min(degrees)}")
# --- 4. 聚集系数 ---
print(f"平均聚集系数: {nx.average_clustering(G):.4f}")
# --- 5. 平均最短路径长度 ---
print(f"平均路径长度: {nx.average_shortest_path_length(G):.4f}")
```

## 10.1.5 案例分析

{% block type="case" title="案例 10.1　社交网络图数据分析" %}

**问题背景**：Zachary 空手道俱乐部（Zachary's Karate Club）是图挖掘领域最经典的测试数据集——它记录了 1970 年代一个空手道俱乐部中 34 名成员之间的社交关系（78 条边）。俱乐部最终因内部矛盾分裂为两个子群，这一事实使其成为社区发现算法的标准基准。我们使用 NetworkX 构建并分析该图的基本统计特征。

**数据说明**：34 个节点（俱乐部成员），78 条无向边（成员之间的课外社交互动），无权重。

**方法与步骤**：

```python
# --- 1. 导入与构建 ---
import networkx as nx
import numpy as np
G = nx.karate_club_graph()
n, m = G.number_of_nodes(), G.number_of_edges()
# --- 2. 度统计 ---
degrees = dict(G.degree())
avg_deg = np.mean(list(degrees.values()))
print(f"节点数={n}, 边数={m}, 平均度={avg_deg:.2f}")
# --- 3. 矩阵表示 ---
A = nx.adjacency_matrix(G).toarray()  # 邻接矩阵
L = nx.laplacian_matrix(G).toarray()   # 拉普拉斯矩阵
print(f"三角形数: {np.trace(A @ A @ A) // 6}")
# --- 4. 图的统计特征 ---
clustering = nx.average_clustering(G)
avg_path = nx.average_shortest_path_length(G)
density = nx.density(G)
print(f"聚集系数={clustering:.4f}, 平均路径={avg_path:.4f}, 密度={density:.4f}")
# --- 5. 谱分析 ---
eigenvalues = np.linalg.eigvalsh(L)
print(f"最小特征值={eigenvalues[0]:.6f}")
print(f"Fiedler值={eigenvalues[1]:.6f}")
```

**结果分析**：

1. **基本统计**：34 个节点、78 条边、平均度 $$\bar{d} = 4.59$$、密度 $$\rho = 0.139$$。这是一个中等密度的小型社交网络。

2. **聚集系数**：$$\bar{C} = 0.5706$$——远高于等规模随机图的期望值 $$C_{\text{rand}} = \bar{d}/n = 0.135$$，表明"朋友的朋友也是朋友"的传递性很强。

3. **平均路径长度**：$$\bar{L} = 2.408$$——仅需约 2.4 步就能连接任意两个成员，验证了小世界性质。等规模随机图的期望路径长度 $$L_{\text{rand}} = \ln 34 / \ln 4.59 \approx 2.31$$，两者非常接近。

4. **谱分析**：拉普拉斯矩阵的最小特征值 $$\lambda_1 \approx 0$$（图连通），Fiedler 值 $$\lambda_2 \approx 0.4716$$，反映了中等程度的连通强度。Fiedler 值不大，暗示该图存在可以被"切割"的社区结构——这正是后来俱乐部分裂为两个子群的数学预兆。

5. **度分布**：节点 0（教练 Mr. Hi）和节点 33（管理员 Officer）的度最高（分别为 16 和 17），是社交网络中的枢纽节点。其余节点的度集中在 2–5 之间，呈现典型的右偏分布。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 图的基本术语（定义 10.1） | $$G = (V, E)$$，顶点集 $$V$$、边集 $$E$$，有向/无向/加权 |
| 节点度（定义 10.2） | $$d(v) = \|\{u : (v, u) \in E\}\|$$，握手定理 $$\sum d(v) = 2m$$ |
| 邻接矩阵（定义 10.3） | $$A_{ij} = 1$$ if $$(v_i, v_j) \in E$$，$$\mathbf{A}^k$$ 计数长度为 $$k$$ 的路径 |
| 拉普拉斯矩阵（定义 10.4） | $$\mathbf{L} = \mathbf{D} - \mathbf{A}$$，半正定，特征值编码图结构 |
| 关联矩阵（定义 10.5） | $$\mathbf{B} \in \mathbb{R}^{n \times m}$$，$$\mathbf{L} = \mathbf{B}\mathbf{B}^T$$ |
| 存储结构（定义 10.6） | 邻接矩阵 $$O(n^2)$$ vs 邻接表 $$O(n+m)$$，稀疏图选邻接表 |
| 聚集系数（定义 10.7） | 邻居互连程度 $$C(v) \in [0, 1]$$，真实图的 $$\bar{C}$$ 远高于随机图 |
| 拉普拉斯谱性质（定理 10.1） | 半正定、$$\lambda_1 = 0$$、零特征值数 = 连通分量数 |

### A 级（概念理解）

**A2.** 邻接矩阵的幂次 $$(\mathbf{A}^k)_{ij}$$ 表示从 $$v_i$$ 到 $$v_j$$ 的长度为 $$k$$ 的路径数。为什么 $$\text{tr}(\mathbf{A}^3) / 6$$ 计算的是三角形数量？"除以 6"的原因是什么？

**A3.** 比较邻接矩阵和邻接表在以下三个操作上的时间复杂度：(1) 判断两个节点之间是否有边；(2) 遍历一个节点的所有邻居；(3) 添加一条新边。在什么密度范围内应选择邻接矩阵？

**A4.** 解释拉普拉斯矩阵 $$\mathbf{L}$$ 的零特征值个数与图连通分量数的关系。如果一个图有 3 个连通分量，它的拉普拉斯矩阵有什么特征？

### B 级（数学推导）

**B1.** 证明握手定理：无向图中 $$\sum_{v \in V} d(v) = 2m$$。从邻接矩阵的角度给出另一种证明——利用 $$\sum_{i,j} A_{ij}$$ 的值。

**B2.** 证明定理 10.1 的性质 2：拉普拉斯矩阵 $$\mathbf{L}$$ 的最小特征值为 0，对应特征向量为 $$\mathbf{1}$$。提示：直接验证 $$\mathbf{L} \mathbf{1} = \mathbf{0}$$。

**B3.** 推导关联矩阵与拉普拉斯矩阵的关系 $$\mathbf{L} = \mathbf{B}\mathbf{B}^T$$。对一个 3 节点链图 $$v_1 - v_2 - v_3$$，显式写出 $$\mathbf{B}$$，验证 $$\mathbf{B}\mathbf{B}^T = \mathbf{L}$$。

**B4.** 归一化拉普拉斯矩阵 $$\mathbf{L}_{\text{sym}}$$ 的特征值在 $$[0, 2]$$ 范围内。证明 $$\lambda_{\max}(\mathbf{L}_{\text{sym}}) \leq 2$$，并给出等号成立的条件。

### C 级（数值计算）

**C1.** 对下图 $$G$$：$$V = \{A, B, C, D\}$$，$$E = \{(A,B), (A,C), (B,C), (C,D)\}$$。(1) 写出邻接矩阵 $$\mathbf{A}$$；(2) 写出度矩阵 $$\mathbf{D}$$；(3) 计算拉普拉斯矩阵 $$\mathbf{L} = \mathbf{D} - \mathbf{A}$$；(4) 计算 $$\mathbf{L}$$ 的特征值，验证最小特征值为 0。

**C2.** 对 C1 的图：(1) 计算每个节点的聚集系数 $$C(v)$$；(2) 计算全局聚集系数 $$\bar{C}$$；(3) 计算图密度 $$\rho$$；(4) 用 BFS 从节点 A 出发，计算到所有其他节点的最短距离。

**C3.** 对 C1 的图，计算邻接矩阵的 2 次幂 $$\mathbf{A}^2$$ 和 3 次幂 $$\mathbf{A}^3$$。验证 $$(\mathbf{A}^2)_{AC}$$ 等于 A 到 C 的长度为 2 的路径数，以及 $$\text{tr}(\mathbf{A}^3)/6$$ 等于图中三角形的数量。

### F 级（开放探讨）

**F1.** **异构图与多关系图**：本节讨论的是同构图（所有节点和边类型相同）。然而，真实场景中的图通常包含多种类型的节点和边——例如知识图谱中有"人""组织""地点"三种节点，以及"就职于""出生于""位于"等多种关系。讨论：(1) 如何扩展邻接矩阵来表示异构图？(2) 异构图的存储结构应如何设计？(3) 异构图对图挖掘算法提出了哪些新挑战？
