# 10.3 社区发现算法

上一节定义了节点中心性和节点相似度（§10.2），这些指标刻画了**单个节点**或**节点对**的性质。然而，真实网络中更普遍、更重要的结构是**社区**（Community）——一组内部连接紧密、外部连接稀疏的节点集合。

社区结构反映了网络的功能模块——社交网络中的兴趣圈子、引文网络中的研究领域、蛋白质相互作用网络中的功能模块、万维网中的主题网站群。发现这些隐藏的社区结构是理解复杂网络组织原理的关键，也是推荐系统（向同社区用户推荐内容）、网络治理（识别紧密群体）、生物信息学（发现蛋白质功能模块）等应用的基础。

本节介绍三种主流的社区发现方法：模块度优化、谱聚类和 Louvain 算法，并在经典的空手道俱乐部数据集上进行比较。

## 10.3.1 社区结构与模块度

### 10.3.1.1 什么是社区？

{% block type="definition" title="定义 10.14　社区结构" %}
给定图 $$G = (V, E)$$，**社区划分**（Community Partition）是将顶点集 $$V$$ 划分为 $$k$$ 个互不相交的子集 $$\mathcal{C} = \{C_1, C_2, \ldots, C_k\}$$，使得：

$$
\bigcup_{i=1}^{k} C_i = V, \quad C_i \cap C_j = \emptyset \ (\forall i \neq j) \tag{10.32}
$$

一个"好的"社区划分应满足：**社区内部**的边密度远高于**社区之间**的边密度。
{% endblock %}

直觉上，社区是"内紧外松"的节点集合。但如何将这个直觉量化为可优化的目标函数？Newman 和 Girvan 在 2004 年提出的**模块度**（Modularity）给出了优雅的答案。

### 10.3.1.2 模块度

{% block type="definition" title="定义 10.15　模块度（Modularity）" %}
给定图 $$G = (V, E)$$（$$m = |E|$$），社区划分 $$\mathcal{C}$$ 的**模块度**定义为：

$$
Q = \frac{1}{2m} \sum_{i,j} \left[ A_{ij} - \frac{k_i k_j}{2m} \right] \delta(c_i, c_j) \tag{10.33}
$$

其中：
- $$A_{ij}$$ 为邻接矩阵元素；
- $$k_i = d(v_i)$$ 为节点 $$v_i$$ 的度；
- $$c_i$$ 为节点 $$v_i$$ 所属的社区；
- $$\delta(c_i, c_j)$$ 为 Kronecker delta——当 $$c_i = c_j$$ 时为 1，否则为 0；
- $$k_i k_j / (2m)$$ 是在**度保持随机图**（Configuration Model）中节点 $$i$$ 和 $$j$$ 之间的期望边数。

模块度的取值范围为 $$Q \in [-1/2, 1)$$。$$Q > 0$$ 表示社区内的边密度超过随机期望——$$Q$$ 越大，社区结构越显著。实践中，$$Q > 0.3$$ 通常被认为存在显著的社区结构。
{% endblock %}

模块度的核心思想是**与零模型的偏差**：它衡量的是"实际边数"与"在保持度分布不变的随机图中期望的边数"之差。如果社区内的边数远超随机期望，说明社区结构是真实存在的，而非度分布的偶然产物。

写成矩阵形式，定义**模块度矩阵** $$\mathbf{B}$$：

$$
B_{ij} = A_{ij} - \frac{k_i k_j}{2m} \tag{10.34}
$$

则模块度可写为：

$$
Q = \frac{1}{2m} \text{tr}(\mathbf{S}^T \mathbf{B} \mathbf{S}) \tag{10.35}
$$

其中 $$\mathbf{S} \in \{0, 1\}^{n \times k}$$ 为社区指示矩阵（$$S_{ic} = 1$$ 当节点 $$i$$ 属于社区 $$c$$）。最大化模块度是 NP-hard 问题，需要近似算法。

{% hint style="warning" %}
⚠️ **易错点：模块度的分辨率极限**

模块度优化存在**分辨率极限**（Resolution Limit，Fortunato & Barthélemy, 2007）：当网络规模很大时，即使两个明显分离的小社区，模块度优化也可能将它们合并为一个大社区。直觉上，当子社区的内部边数小于 $$\sqrt{m}$$ 时，模块度无法将其分辨出来。因此，模块度适用于发现"中等粒度"的社区，对于极小或极大的社区可能失效。
{% endhint %}

## 10.3.2 谱聚类方法

### 10.3.2.1 图切割

{% block type="definition" title="定义 10.16　图切割" %}
给定图 $$G = (V, E)$$，将 $$V$$ 二分为 $$(S, \bar{S})$$ 的**切割代价**定义为两个子集之间的边数：

$$
\text{Cut}(S, \bar{S}) = \sum_{i \in S, j \in \bar{S}} A_{ij} \tag{10.36}
$$

最小化切割代价的朴素方法会倾向于将一个孤立节点切出去（代价为该节点的度）。为了获得**均衡的**划分，定义两种归一化切割：

**RatioCut**（按节点数均衡）：

$$
\text{RatioCut}(S, \bar{S}) = \frac{\text{Cut}(S, \bar{S})}{|S|} + \frac{\text{Cut}(S, \bar{S})}{|\bar{S}|} \tag{10.37}
$$

**NCut**（按度和均衡，Shi & Malik, 2000）：

$$
\text{NCut}(S, \bar{S}) = \frac{\text{Cut}(S, \bar{S})}{\text{vol}(S)} + \frac{\text{Cut}(S, \bar{S})}{\text{vol}(\bar{S})} \tag{10.38}
$$

其中 $$\text{vol}(S) = \sum_{i \in S} d(v_i)$$ 为子集 $$S$$ 的度和。
{% endblock %}

### 10.3.2.2 从离散优化到谱松弛

最小化 RatioCut 或 NCut 是 NP-hard 问题。谱聚类的核心洞察是：**将离散的指示向量松弛为连续向量**，问题就转化为拉普拉斯矩阵的特征值问题。

{% block type="theorem" title="定理 10.3　NCut 的谱松弛" %}
最小化二分 NCut 等价于如下受约束优化问题：

$$
\min_{\mathbf{f}} \frac{\mathbf{f}^T \mathbf{L} \mathbf{f}}{\mathbf{f}^T \mathbf{D} \mathbf{f}} \quad \text{s.t.} \quad \mathbf{D}\mathbf{f} \perp \mathbf{1} \tag{10.39}
$$

其中 $$\mathbf{L} = \mathbf{D} - \mathbf{A}$$ 为拉普拉斯矩阵，$$\mathbf{f}$$ 为松弛后的连续指示向量。

该问题的最优解是广义特征值问题 $$\mathbf{L}\mathbf{f} = \lambda \mathbf{D}\mathbf{f}$$ 的**第二小特征向量**（Fiedler 向量），等价于归一化拉普拉斯矩阵 $$\mathbf{L}_{\text{sym}} = \mathbf{D}^{-1/2}\mathbf{L}\mathbf{D}^{-1/2}$$ 的第二小特征向量。

**分析**　由 Rayleigh 商的性质，$$\mathbf{f}^T \mathbf{L} \mathbf{f} / (\mathbf{f}^T \mathbf{D} \mathbf{f})$$ 在正交于 $$\mathbf{D}^{1/2}\mathbf{1}$$（最小特征值 $$\lambda_1 = 0$$ 对应的方向）的子空间上取最小值时，$$\mathbf{f}$$ 为第二小特征值 $$\lambda_2$$ 对应的特征向量。$$\lambda_2$$（Fiedler 值）越小，图越"容易被切割"——这与 §10.1 定理 10.1 的代数连通度解释一致。$$\blacksquare$$
{% endblock %}

{% block type="algorithm" title="算法 10.3　谱聚类" %}
**输入**：图 $$G = (V, E)$$（或相似度矩阵），社区数 $$k$$

**输出**：社区划分 $$\mathcal{C} = \{C_1, \ldots, C_k\}$$

**步骤 1（构建拉普拉斯矩阵）**：计算 $$\mathbf{L}_{\text{sym}} = \mathbf{I} - \mathbf{D}^{-1/2}\mathbf{A}\mathbf{D}^{-1/2}$$

**步骤 2（特征分解）**：计算 $$\mathbf{L}_{\text{sym}}$$ 的前 $$k$$ 个最小特征值对应的特征向量 $$\mathbf{u}_1, \ldots, \mathbf{u}_k$$

**步骤 3（构建嵌入矩阵）**：令 $$\mathbf{U} = [\mathbf{u}_1, \ldots, \mathbf{u}_k] \in \mathbb{R}^{n \times k}$$，对每行归一化

**步骤 4（K-means 聚类）**：对 $$\mathbf{U}$$ 的 $$n$$ 行向量执行 K-means 聚类，得到 $$k$$ 个类别

**步骤 5（返回）**：节点 $$v_i$$ 属于其对应行向量的聚类结果

**复杂度**：特征分解 $$O(n^2 k)$$（稀疏矩阵可用 Lanczos 方法加速），K-means $$O(nkt)$$（$$t$$ 为迭代次数）。
{% endblock %}

## 10.3.3 Louvain 算法

谱聚类需要特征分解，对大规模网络（百万节点以上）计算开销较大。**Louvain 算法**（Blondel et al., 2008）通过贪心的模块度优化策略，在 $$O(n \log n)$$ 时间内高效地发现社区。

{% block type="definition" title="定义 10.17　Louvain 算法的两阶段策略" %}
Louvain 算法采用**多层级贪心聚合**策略，交替执行两个阶段：

**阶段一（模块度优化）**：对每个节点 $$v$$，尝试将其移入相邻社区——如果移动使模块度增加，则执行移动。当节点 $$v$$ 从社区 $$C_1$$ 移入社区 $$C_2$$ 时，模块度增量为：

$$
\Delta Q = \frac{1}{2m} \left[ \sum_{j \in C_2} A_{vj} - \frac{k_v \cdot \Sigma_{\text{tot}}(C_2)}{2m} \right] - \frac{1}{2m} \left[ \sum_{j \in C_1 \setminus \{v\}} A_{vj} - \frac{k_v \cdot (\Sigma_{\text{tot}}(C_1) - k_v)}{2m} \right] \tag{10.40}
$$

其中 $$\Sigma_{\text{tot}}(C)$$ 为社区 $$C$$ 中所有节点的度之和。重复扫描所有节点，直到没有移动能增加模块度。

**阶段二（图压缩）**：将阶段一得到的每个社区压缩为一个超节点，社区之间的边聚合为超边（权重为跨社区的总边数），社区内部的边成为超节点的自环。在压缩后的图上重复阶段一。

两个阶段交替执行，直到模块度不再增加。
{% endblock %}

{% block type="algorithm" title="算法 10.4　Louvain 社区发现" %}
**输入**：图 $$G = (V, E)$$

**输出**：社区划分 $$\mathcal{C}$$

**步骤 1（初始化）**：每个节点自成一个社区，$$c_i \leftarrow i$$

**步骤 2（阶段一——局部移动）**：
$$\quad$$ **repeat**
$$\quad\quad$$ $$\text{changed} \leftarrow \text{False}$$
$$\quad\quad$$ **for each** $$v \in V$$（随机顺序）**do**
$$\quad\quad\quad$$ 计算 $$v$$ 移入每个相邻社区的 $$\Delta Q$$（方程 10.40）
$$\quad\quad\quad$$ **if** $$\max \Delta Q > 0$$ **then**
$$\quad\quad\quad\quad$$ 将 $$v$$ 移入使 $$\Delta Q$$ 最大的社区
$$\quad\quad\quad\quad$$ $$\text{changed} \leftarrow \text{True}$$
$$\quad$$ **until** $$\neg \text{changed}$$

**步骤 3（阶段二——图压缩）**：
$$\quad$$ 将每个社区压缩为超节点，构建压缩图 $$G'$$
$$\quad$$ $$G \leftarrow G'$$

**步骤 4（终止判断）**：若模块度不再增加，返回 $$\mathcal{C}$$；否则转步骤 2

**复杂度**：实践中接近 $$O(n \log n)$$——每个节点只需检查其邻居社区，每轮扫描的计算量与边数成正比。
{% endblock %}

## 10.3.4 方法对比

谱聚类和 Louvain 分别代表了社区发现的两种范式：**自顶向下的谱分析**（通过全局特征分解找到最优切割）和**自底向上的贪心优化**（通过局部模块度增量逐步聚合）。除了这两种方法外，还有基于标签传播的方法（利用邻居投票快速扩散标签，速度最快但结果不稳定）和基于边介数的 Girvan-Newman 算法（逐步移除"桥梁边"来分离社区，理论清晰但速度极慢）。不同方法在计算效率、是否需要预设社区数、对分辨率极限的敏感性等方面各有优劣，实际选择时需要根据网络规模和应用需求权衡。

| 方法 | 核心思想 | 社区数 $$k$$ | 时间复杂度 | 优势 | 局限 |
|------|---------|-------------|-----------|------|------|
| 谱聚类 | 拉普拉斯特征向量 + K-means | 需指定 | $$O(n^2 k)$$ | 理论基础扎实 | 大规模图计算困难 |
| Louvain | 贪心模块度优化 | 自动确定 | $$O(n \log n)$$ | 快速、可扩展 | 分辨率极限 |
| Label Propagation | 邻居标签投票 | 自动确定 | $$O(m)$$ | 最快 | 结果不稳定 |
| Girvan-Newman | 移除高介数边 | 需指定 | $$O(n^2 m)$$ | 可解释性强 | 速度极慢 |

{% hint style="info" %}
🤔 **思考：社区发现 vs 聚类——有什么区别？**

社区发现与传统聚类（§5.1–§5.7）在形式上非常相似，但有本质区别：(1) 传统聚类基于**特征空间中的距离**（如欧氏距离），社区发现基于**拓扑连接**（如边的密度）；(2) 社区结构的评价标准是模块度和网络拓扑，而非轮廓系数等几何指标；(3) 社区发现需要处理**稀疏性**——大多数节点对之间没有边，这与稠密特征矩阵截然不同。谱聚类是两者的桥梁——它将图的拓扑结构通过拉普拉斯特征向量嵌入到欧氏空间，然后用 K-means 聚类。
{% endhint %}

## 10.3.5 案例分析

{% block type="case" title="案例 10.3　Zachary 空手道俱乐部社区发现" %}

**问题背景**：Zachary 空手道俱乐部（§10.1 案例 10.1）因内部矛盾分裂为两个子群。我们使用谱聚类和 Louvain 两种方法发现社区结构，并与真实的分裂结果对比。

**数据说明**：34 个节点（成员），78 条边（社交互动），真实标签为两个社区（Mr. Hi 派 vs Officer 派）。

**方法与步骤**：

```python
# --- 1. 加载数据 ---
import networkx as nx
import numpy as np
from sklearn.cluster import KMeans
G = nx.karate_club_graph()
true_labels = [0 if G.nodes[v]["club"] == "Mr. Hi" else 1 for v in G.nodes()]
# --- 2. 谱聚类 ---
L = nx.normalized_laplacian_matrix(G).toarray()
eigvals, eigvecs = np.linalg.eigh(L)
fiedler = eigvecs[:, 1]  # 第二小特征向量
spectral_labels = (fiedler > 0).astype(int)
# --- 3. Louvain ---
import community as community_louvain
partition = community_louvain.best_partition(G, random_state=42)
louvain_labels = [partition[v] for v in G.nodes()]
# --- 4. 评估 ---
from sklearn.metrics import normalized_mutual_info_score
nmi_spectral = normalized_mutual_info_score(true_labels, spectral_labels)
nmi_louvain = normalized_mutual_info_score(true_labels, louvain_labels)
Q_louvain = community_louvain.modularity(partition, G)
print(f"谱聚类 NMI={nmi_spectral:.4f}")
print(f"Louvain NMI={nmi_louvain:.4f}, Q={Q_louvain:.4f}")
```

**结果分析**：

1. **谱聚类**（$$k = 2$$）：利用 Fiedler 向量的正负将网络二分。NMI $$\approx 0.84$$——仅有 1–2 个边界节点被错误分类（如节点 2 和节点 8，它们与两个社区都有大量连接）。

2. **Louvain 算法**：自动发现 4 个社区，模块度 $$Q \approx 0.42$$。它将两个大社区各自进一步细分为两个子群，反映了社区内部更精细的结构。如果将 4 个社区合并为 2 个（按大类），NMI 可达 $$0.89$$。

3. **方法差异**：谱聚类需要预先指定 $$k = 2$$，结果与真实标签高度一致。Louvain 自动选择社区数，发现了更细粒度的结构——这不是"错误"，而是捕获了不同层级的社区。

4. **关键节点**：节点 0（Mr. Hi）和节点 33（Officer）分别是两个社区的核心，它们的 PageRank（§10.2）最高。节点 2（同时连接两个核心）是两种方法都容易误分的"边界节点"。

{% endblock %}

| 关键词 | 核心概念 |
|--------|---------|
| 社区结构（定义 10.14） | 内部紧密、外部稀疏的节点集合 |
| 模块度（定义 10.15） | $$Q = \frac{1}{2m}\sum [A_{ij} - k_i k_j / (2m)] \delta(c_i, c_j)$$，与零模型的偏差 |
| 图切割（定义 10.16） | RatioCut 和 NCut——均衡的最小切割 |
| 谱聚类（定理 10.3，算法 10.3） | NCut 松弛为拉普拉斯特征值问题，Fiedler 向量二分图 |
| Louvain（定义 10.17，算法 10.4） | 两阶段贪心——局部模块度优化 + 图压缩，$$O(n \log n)$$ |
