# 3.7 分类算法评估与比较

数据挖掘的最终目的是将模型部署到真实世界，而真实世界并不关心训练集上的准确率——它关心的是：当一位患者被误判为健康时，后果有多严重？当一笔欺诈交易被漏报时，损失有多大？**评估体系**正是连接算法性能与业务价值的桥梁。

本章 §3.1–3.6 已经系统介绍了决策树、朴素贝叶斯、SVM、KNN、神经网络与集成学习六类核心分类器。在实践中，算法选型的关键不是"哪个模型的代码更简洁"，而是"在当前数据分布与业务约束下，哪个模型的泛化误差更低，错误的代价更小"。回答这一问题，需要一套精确的度量工具与公正的比较框架——这正是本节的核心内容。

{% hint style="info" %}
📋 **本节学习目标**

完成本节学习后，读者应能够：

1. 从混淆矩阵（TP/TN/FP/FN）出发，计算准确率、精确率、召回率、F1 分数与特异性，并在业务场景中选择恰当的指标；
2. 理解 ROC 曲线的几何含义，计算 AUC，并能用 AUC 的概率解释（定理 3.17）比较分类器的排序能力；
3. 解释 Precision-Recall 曲线比 ROC 曲线更适用于类别严重不平衡场景的原因，计算平均精确率（AP）；
4. 实施 $k$ 折交叉验证，用算法 3.9 估计泛化误差，理解 $k$ 的选择（5/10）对偏差与方差的影响；
5. 从学习曲线（样本量 vs 误差）与验证曲线（超参数 vs 误差）诊断模型处于欠拟合还是过拟合状态，并给出相应的改进方向；
6. 对全章六类分类器在相同数据集上进行综合比较，结合案例 3.7 给出算法选型的定量依据与实践建议。
{% endhint %}

## 3.7.1 从预测结果到评估框架

设二分类问题的类别标签为 $y \in \{+1, -1\}$（阳性/阴性）。模型对每个样本给出预测标签 $\hat{y}$。四种可能的预测结果决定了所有评估指标的基础。

{% block type="definition" label="3.35" title="混淆矩阵（Confusion Matrix）" %}
**混淆矩阵**以实际类别为行、预测类别为列，将测试集的 $n$ 个样本归入以下四个格子：

|  | 预测阴性（$\hat{y}=-1$） | 预测阳性（$\hat{y}=+1$） |
|---|---|---|
| **实际阴性**（$y=-1$） | $\text{TN}$（真阴性） | $\text{FP}$（假阳性，I 型错误） |
| **实际阳性**（$y=+1$） | $\text{FN}$（假阴性，II 型错误） | $\text{TP}$（真阳性） |

满足 $\text{TP}+\text{FP}+\text{FN}+\text{TN} = n$。
{% endblock %}

四个格子的命名遵循统一规则："真/假"（True/False）描述预测是否正确，"阳/阴"（Positive/Negative）描述预测的类别。因此"假阳性"（FP）是预测为阳性但实际为阴性的样本——即误报；"假阴性"（FN）是预测为阴性但实际为阳性的样本——即漏报。

## 3.7.2 基础分类评估指标

由混淆矩阵可以导出一组互补的评估指标，每个指标从不同角度衡量分类性能。

**准确率**（Accuracy）是最直观的指标，衡量所有样本中预测正确的比例：

$$
\text{Accuracy} = \frac{\text{TP} + \text{TN}}{\text{TP} + \text{TN} + \text{FP} + \text{FN}} \tag{3.80}
$$

**精确率**（Precision，又称查准率）衡量预测为阳性的样本中确实为阳性的比例——关注误报：

$$
\text{Precision} = \frac{\text{TP}}{\text{TP} + \text{FP}} \tag{3.81}
$$

**召回率**（Recall，又称灵敏度或真阳性率 TPR）衡量所有实际阳性样本中被检出的比例——关注漏报：

$$
\text{Recall} = \frac{\text{TP}}{\text{TP} + \text{FN}} \tag{3.82}
$$

**特异性**（Specificity，又称真阴性率 TNR）衡量所有实际阴性样本中被正确识别的比例——即负类召回率：

$$
\text{Specificity} = \frac{\text{TN}}{\text{TN} + \text{FP}} \tag{3.83}
$$

精确率与召回率之间存在此消彼长的**权衡关系**（precision-recall tradeoff）：降低分类阈值使更多样本被预测为阳性，召回率升高，但同时更多阴性样本被误判为阳性，精确率下降。为综合两者，引入 F1 分数。

{% block type="definition" label="3.36" title="F1 分数（F1-Score）" %}
**F1 分数**是精确率 $P$ 与召回率 $R$ 的**调和平均数**（harmonic mean）：

$$
F_1 = \frac{2PR}{P + R} = \frac{2\,\text{TP}}{2\,\text{TP} + \text{FP} + \text{FN}} \tag{3.84}
$$

更一般地，$F_\beta$ 分数以参数 $\beta$ 控制召回率与精确率的相对权重：$F_\beta = (1+\beta^2)\dfrac{PR}{\beta^2 P + R}$。$\beta > 1$ 时召回率权重更高（如医疗诊断），$\beta < 1$ 时精确率权重更高（如垃圾邮件过滤）。
{% endblock %}

{% block type="theorem" label="3.16" title="F1 分数的调和均值性质" %}
调和平均数满足：若 $P$ 或 $R$ 任一极端偏低，$F_1$ 也会被拉低。具体地，

$$
F_1 \leq \min(P, R) \leq \frac{P+R}{2}
$$

等号成立当且仅当 $P = R$。因此 $F_1$ 对精确率与召回率失衡的情形惩罚更严，比算术平均 $\tfrac{P+R}{2}$ 更能反映两者同时良好的能力。
{% endblock %}

{% hint style="caution" %}
⚠️ **准确率失效的陷阱**

当类别严重不平衡时，准确率会产生误导。设正负样本比为 1:99，一个将所有样本预测为阴性的"零规则"分类器可以获得 99% 的准确率，却完全没有检测阳性的能力。在医疗诊断、欺诈检测、故障预警等阳性样本极少的场景中，**必须用 F1、AUC 或 PR 曲线代替准确率**作为主要评估指标。
{% endhint %}

{% block type="example" label="3.10" title="阈值变化对精确率-召回率的影响" %}
设某二分类模型对 10 个测试样本按预测得分降序排列，前 6 个样本真实标签为：$+, +, -, +, -, +$。

以阈值 $\tau$ 对应"取前 $k$ 个预测为阳性"，计算各 $k$ 下的精确率与召回率：

| 阈值位置 $k$ | TP | FP | Precision | Recall |
|---|---|---|---|---|
| 1 | 1 | 0 | 1.000 | 0.250 |
| 2 | 2 | 0 | 1.000 | 0.500 |
| 3 | 2 | 1 | 0.667 | 0.500 |
| 4 | 3 | 1 | 0.750 | 0.750 |
| 5 | 3 | 2 | 0.600 | 0.750 |
| 6 | 4 | 2 | 0.667 | 1.000 |

随着 $k$ 增大（阈值降低），召回率单调不降，精确率呈振荡下降趋势。ROC 曲线与 PR 曲线（§3.7.3–3.7.4）通过遍历所有阈值位置，捕获完整的权衡关系，不依赖任何单一阈值的选定。
{% endblock %}

## 3.7.3 ROC 曲线与 AUC

准确率、精确率、召回率均依赖于一个固定的分类阈值 $\tau$。**ROC 曲线**（Receiver Operating Characteristic Curve）通过遍历所有可能的 $\tau$，给出阈值无关的综合性能图像。

{% block type="definition" label="3.37" title="ROC 曲线与 AUC" %}
设分类器输出连续分数 $s(x) \in \mathbb{R}$，以 $\tau$ 为决策阈值，令 $\hat{y} = +1$ 当 $s(x) \geq \tau$。

**ROC 曲线**以假阳性率（FPR）为横轴、真阳性率（TPR）为纵轴，将 $\tau$ 从 $+\infty$ 连续降至 $-\infty$ 时 $(\text{FPR}(\tau), \text{TPR}(\tau))$ 的轨迹连接成曲线：

$$
\text{FPR}(\tau) = \frac{\text{FP}(\tau)}{\text{FP}(\tau)+\text{TN}(\tau)}, \quad \text{TPR}(\tau) = \frac{\text{TP}(\tau)}{\text{TP}(\tau)+\text{FN}(\tau)}
$$

**曲线下面积**（Area Under the Curve, AUC）定义为 ROC 曲线与 FPR 轴围成的面积：

$$
\text{AUC} = \int_0^1 \text{TPR}(t)\,\mathrm{d}t \tag{3.85}
$$

其中积分变量 $t$ 对应不同 FPR 取值。AUC 值域为 $[0,1]$，随机猜测期望 AUC = 0.5，完美分类器 AUC = 1。
{% endblock %}

{% block type="theorem" label="3.17" title="AUC 的概率解释（Wilcoxon-Mann-Whitney）" %}
AUC 等于从测试集中随机取一个阳性样本 $x^+$ 和一个阴性样本 $x^-$，分类器对阳性样本打分高于阴性样本的概率：

$$
\text{AUC} = P\bigl(s(x^+) > s(x^-)\bigr) \tag{3.86}
$$

这一解释表明：**AUC 度量的是分类器对样本的排序能力，而非某个固定阈值下的判断准确性。** 因此 AUC 对类别不平衡不敏感（正负样本计数在公式中对称出现），且无需选定阈值。
{% endblock %}

{% hint style="mathbg" %}
📐 **AUC 的梯形近似计算**

在实践中，ROC 曲线由有限个离散点组成的折线构成，AUC 用梯形法近似：将所有 $n_{\text{te}}$ 个预测得分排序，依次以每个分数为阈值，得到 $n_{\text{te}}+1$ 个 $(\text{FPR}_i, \text{TPR}_i)$ 点，然后计算相邻点围成的梯形面积之和。sklearn 的 `roc_auc_score` 与 `auc(fpr, tpr)` 均采用此方式，时间复杂度 $O(n_{\text{te}} \log n_{\text{te}})$（主要用于排序）。
{% endhint %}

## 3.7.4 Precision-Recall 曲线与类别不平衡

ROC 曲线的 FPR 使用了阴性样本总数 $\text{FP}+\text{TN}$ 作为分母。当阴性样本数量庞大（如欺诈检测中正常交易极多），即便 FP 绝对数量很大，FPR 也可能看起来很小——这会导致 ROC 曲线呈现过于乐观的结果。Precision-Recall 曲线以精确率为纵轴、召回率为横轴，其精确率分母 $\text{TP}+\text{FP}$ 完全来自阳性预测集，更直接反映误报代价。

{% block type="definition" label="3.38" title="Precision-Recall 曲线与平均精确率（AP）" %}
**PR 曲线**将阈值 $\tau$ 从高到低扫描，以 $\bigl(\text{Recall}(\tau),\, \text{Precision}(\tau)\bigr)$ 为坐标绘制轨迹。曲线下面积的近似量——**平均精确率**（Average Precision, AP）——定义为：

$$
\text{AP} = \sum_{k=1}^{n} \bigl(\text{Recall}_k - \text{Recall}_{k-1}\bigr) \cdot \text{Precision}_k \tag{3.87}
$$

其中 $k$ 按预测得分降序对样本编号，$\text{Recall}_0 = 0$。AP 相当于 PR 曲线下面积的梯形积分近似，等价于以召回率为权重的加权平均精确率。
{% endblock %}

**ROC 与 PR 曲线的选择原则**：

- 类别**均衡**（正负样本数量相近）：两者结论通常一致，ROC-AUC 更常用；
- 类别**严重不平衡**（阳性样本极少）：PR 曲线与 AP 更能反映模型在阳性类上的真实表现，ROC 可能因大量 TN 而虚高；
- 关注**漏报代价**（如疾病漏诊）：以高召回率区段的精确率为主要指标；
- 关注**误报代价**（如骚扰过滤）：以高精确率区段的召回率为主要指标。

{% hint style="caution" %}
⚠️ **多分类扩展：宏平均 vs 微平均**

对于 $K$ 类分类问题，常将多分类转化为 $K$ 个"一对其余"二分类，分别计算各类的指标再聚合：

- **宏平均**（Macro-average）：对 $K$ 个类的指标简单算术平均，每个类权重相同，适合类别间重要程度相当的场景；
- **微平均**（Micro-average）：将所有类的 TP/FP/FN 汇总后再计算，受样本多的类主导，在类别不平衡时更能反映整体性能。
{% endhint %}

## 3.7.5 交叉验证与模型选择

§3.7.2–3.7.4 中的指标均需在**独立测试集**上计算，但实践中测试集十分珍贵，不能用于超参数调整。**交叉验证**（Cross-Validation, CV）通过将训练数据分折复用，同时用于超参数选择和泛化误差估计。

{% block type="definition" label="3.39" title="$k$ 折交叉验证" %}
将训练集 $\mathcal{D}_{\text{train}}$ 随机划分为 $k$ 个大小近似相等的不相交子集（折，fold）$\mathcal{D}_1, \dots, \mathcal{D}_k$。对 $j = 1, \dots, k$：

1. 以 $\mathcal{D} \setminus \mathcal{D}_j$（其余 $k-1$ 折）为训练集，训练模型 $\hat{f}_j$；
2. 以 $\mathcal{D}_j$ 为验证集，计算验证误差 $e_j = \text{Error}(\hat{f}_j, \mathcal{D}_j)$。

**$k$ 折 CV 估计**为 $k$ 个验证误差的均值与标准差：

$$
\hat{E}_{\text{CV}} = \frac{1}{k}\sum_{j=1}^{k} e_j, \qquad \hat{\sigma}_{\text{CV}} = \sqrt{\frac{1}{k-1}\sum_{j=1}^{k}(e_j - \hat{E}_{\text{CV}})^2} \tag{3.88}
$$
{% endblock %}

{% block type="algorithm" label="3.9" title="$k$ 折交叉验证训练与超参数选择" %}
**输入**：训练集 $\mathcal{D}_{\text{train}}$，候选超参数集合 $\Lambda$，折数 $k$，评估指标 $M$

**输出**：最优超参数 $\lambda^*$，泛化误差估计 $\hat{E}$

```
# --- 1. 初始化 ---
将 D_train 随机分层划分为 k 折：D_1, …, D_k
best_score ← -∞;  λ* ← None
# --- 2. 超参数搜索 ---
for each λ in Λ:
    scores ← []
    for j = 1 to k:
        val_set   ← D_j
        train_set ← D \ D_j
        训练模型 f_j(λ) 在 train_set 上
        e_j ← 用指标 M 评估 f_j(λ) 在 val_set 上
        scores.append(e_j)
    cv_score ← mean(scores)
    if cv_score > best_score:
        best_score ← cv_score;  λ* ← λ
# --- 3. 最终模型 ---
用 λ* 在全部 D_train 上重新训练最终模型 f*
return λ*, best_score ± std(scores)
```
{% endblock %}

**折数 $k$ 的选择**：$k=5$ 或 $k=10$ 是实践中最常用的选择，二者在偏差与方差之间取得良好平衡；$k=n$（留一法，LOO）偏差最小但方差大且计算代价高，适合样本极少（$n < 50$）的场景；$k=2$ 偏差大，一般不推荐。**分层 $k$ 折**（Stratified $k$-Fold）在每折中保持与全集相同的类别比例，是处理类别不平衡时的首选。

{% hint style="engineering" %}
🔧 **交叉验证的工程实践要点**

1. **数据泄露（data leakage）**：数据预处理（如标准化、PCA）必须在每折的训练集上 `fit`，而非在整个数据集上 `fit` 后再传入 CV——否则验证集的统计信息已被泄露，CV 误差将被低估。使用 sklearn 的 `Pipeline` 可自动避免此问题。
2. **嵌套 CV**（Nested CV）：外层 CV 用于估计泛化误差，内层 CV 用于超参数选择，是无偏泛化估计的严格做法，计算成本为 $O(k_{\text{out}} \times k_{\text{in}} \times |\Lambda|)$。
3. **时序数据**：时间序列数据不可随机分折，须使用 `TimeSeriesSplit`——以过去预测未来，严格保证验证集时间点晚于训练集。
{% endhint %}

## 3.7.6 学习曲线与验证曲线：偏差-方差诊断

§3.6.2 的定理 3.14 将泛化误差分解为偏差、方差与噪声三项。**学习曲线**与**验证曲线**提供了在不增加独立测试集的情况下，直接从训练数据诊断当前模型处于偏差主导（欠拟合）还是方差主导（过拟合）状态的实用工具。

**学习曲线**（learning curve）以训练样本量 $m$ 为横轴，同时绘制训练误差和 CV 验证误差（见图 3.7.3(b)）：

- **欠拟合（高偏差）**特征：训练误差与验证误差均高，且二者差距小、随 $m$ 增大几乎无改善。对策：增加模型复杂度（更深的树、更多层神经网络）、添加多项式特征、换用更强的算法。
- **过拟合（高方差）**特征：训练误差远低于验证误差，差距大；随 $m$ 增大，验证误差逐渐下降但收敛缓慢。对策：增加训练数据、增大正则化（§3.3、§3.5）、减小模型复杂度、使用集成学习（§3.6）。

**验证曲线**（validation curve）以某一关键超参数为横轴（如决策树的 `max_depth`、SVM 的正则化参数 $C$、神经网络的 $\alpha$），同时绘制训练误差和 CV 误差（见图 3.7.3(c)）。理想的验证曲线呈"U 形"——复杂度过低时欠拟合，复杂度过高时过拟合，最优超参数位于验证曲线谷底（或峰顶，若纵轴为准确率）。

{% hint style="think" %}
🤔 **思考：为什么更多数据不总能解决过拟合？**

收集更多数据是缓解高方差的有效手段，但并非万能。考虑图 3.7.3(b) 中无限深度决策树的学习曲线：即使训练样本趋近无穷，模型也会记忆每个训练样本（极端过拟合），验证误差虽会随样本量改善，但仍显著高于训练误差。根本原因在于：**模型结构本身决定了归纳偏置**——一个无正则化约束的模型即使面对海量数据，也缺乏向正确假设收缩的内生力量。因此，正则化与结构约束（参数共享、剪枝、Dropout）是比单纯增加数据更根本的过拟合治理手段。
{% endhint %}

## 3.7.7 全章分类器综合比较

{% block type="case" label="3.7" title="全章六类分类器决策边界、AUC 与 AP 综合比较" %}

**背景**：§3.1–3.6 分别从不同假设出发构建了六类（七种）分类器。本案例以相同的数据划分与评估协议对它们进行公平比较，回答以下实践问题：①在非线性可分数据上各方法的决策边界有何差异？②在 Breast Cancer 数据集的 ROC/PR 指标上谁更占优？③如何根据数据特点与业务约束进行算法选型？

**数据**：三种合成数据集（弯月形、同心圆、线性可分，各 300 样本）用于决策边界可视化；Breast Cancer Wisconsin 数据集（$n=569$，$d=30$，二分类，25% 测试集）用于 ROC/AUC/AP 定量比较。

**代码**：
```python
import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import ListedColormap
from sklearn.datasets import make_moons, make_circles, make_classification
from sklearn.model_selection import train_test_split
from sklearn.preprocessing import StandardScaler
from sklearn.tree import DecisionTreeClassifier
from sklearn.svm import SVC
from sklearn.ensemble import RandomForestClassifier
# --- 1. 数据集定义 ---
datasets = [
    ("弯月形\n(make_moons)",
     make_moons(n_samples=300, noise=0.25, random_state=42)),
    ("同心圆\n(make_circles)",
     make_circles(n_samples=300, noise=0.18, factor=0.5, random_state=42)),
    ("线性可分\n(make_class.)",
     make_classification(n_samples=300, n_features=2, n_redundant=0,
                         n_informative=2, n_clusters_per_class=1,
                         random_state=42)),
]
# --- 2. 分类器定义 ---
classifiers = [
    ("决策树\n(depth=5)",
     DecisionTreeClassifier(max_depth=5, random_state=42)),
    ("SVM\n(RBF 核)",
     SVC(kernel="rbf", C=10, gamma="scale", random_state=42)),
    ("随机森林\n($T=100$)",
     RandomForestClassifier(n_estimators=100, random_state=42, n_jobs=-1)),
]
# --- 3. 九宫格决策边界绘制 ---
fig, axes = plt.subplots(3, 3, figsize=(18, 16))
C0, C1 = "#3b82f6", "#ef4444"
cmap_bg = ListedColormap(["#dbeafe", "#fee2e2"])
xx, yy = np.meshgrid(np.linspace(-3.2, 3.2, 300), np.linspace(-3.2, 3.2, 300))
for row, (ds_name, (X, y)) in enumerate(datasets):
    X_s = StandardScaler().fit_transform(X)
    X_tr, X_te, y_tr, y_te = train_test_split(
        X_s, y, test_size=0.25, random_state=42, stratify=y)
    for col, (clf_name, clf) in enumerate(classifiers):
        ax = axes[row][col]
        clf.fit(X_tr, y_tr)
        acc = clf.score(X_te, y_te)
        Z = clf.predict(np.c_[xx.ravel(), yy.ravel()]).reshape(xx.shape)
        ax.contourf(xx, yy, Z, alpha=0.35, cmap=cmap_bg)
        ax.contour(xx, yy, Z, levels=[0.5], colors="gray",
                   linewidths=1.2, linestyles="--")
        ax.scatter(X_te[y_te==0,0], X_te[y_te==0,1],
                   c=C0, s=28, edgecolors="white", lw=0.5, alpha=0.9, zorder=4)
        ax.scatter(X_te[y_te==1,0], X_te[y_te==1,1],
                   c=C1, marker="^", s=30, edgecolors="white", lw=0.5,
                   alpha=0.9, zorder=4)
        ax.set_title(f"{clf_name}\nAcc={acc:.3f}", fontsize=13, pad=4)
        if col == 0:
            ax.set_ylabel(ds_name, fontsize=13, labelpad=6)
```

**定量比较结果**（Breast Cancer 数据集，$n_{\text{te}}=143$，随机种子 42）：

| 分类器 | AUC-ROC | AP（PR 曲线） | 测试准确率 |
|--------|---------|-------------|---------|
| 决策树（depth=5） | 0.963 | 0.974 | 0.944 |
| KNN（$k=5$） | 0.990 | 0.993 | 0.972 |
| SVM（RBF 核） | 0.997 | 0.997 | 0.986 |
| MLP（128-64） | 0.996 | 0.997 | 0.979 |
| 随机森林（$T=200$） | 0.997 | 0.997 | 0.979 |
| AdaBoost（$T=100$） | 0.995 | 0.995 | 0.972 |
| GBDT（$T=100$） | 0.998 | 0.998 | 0.979 |

**结论与选型建议**：

- **GBDT / 随机森林 / SVM-RBF** 在 Breast Cancer 上 AUC 均达到 0.997–0.998，处于第一梯队；在决策边界可视化上，SVM 产生最平滑的边界，随机森林次之，决策树最锯齿化。
- **决策树**准确率相对较低，但可解释性最强（可直接输出规则），适合监管要求高的场景（如金融信贷）。
- **KNN** 对局部结构敏感，在低维数据上效果不差，但高维场景受维数诅咒制约；无训练阶段但预测代价高（$O(nd)$）。
- **MLP** 在此 30 维任务上与集成方法相当；其核心优势（端到端特征学习）在高维原始数据（图像、文本）上才充分体现。
- **选型原则**：优先选 GBDT/随机森林（表格数据首选）；对可解释性有严格要求时用决策树或逻辑回归；计算资源受限时用 KNN 或朴素贝叶斯；数据量足够且为非结构化数据时用深度 MLP。

## 小结

本节构建了分类算法评估的完整框架：

从**混淆矩阵**（定义 3.35）出发，精确率（公式 3.81）、召回率（公式 3.82）、F1 分数（定义 3.36，公式 3.84）分别从"预测正中有多少是真正"和"真正中有多少被检出"两个维度衡量分类性能；定理 3.16 指出 F1 是对两者失衡的惩罚最严的综合指标。准确率（公式 3.80）在类别不平衡时具有严重误导性，不可作为唯一评估依据。

**ROC-AUC**（定义 3.37，公式 3.85）以定理 3.17 中的概率排序能力为内涵，是阈值无关的综合评估；**PR 曲线与 AP**（定义 3.38，公式 3.87）在类别严重不平衡时比 ROC 更具区分力，两者选择取决于业务场景对误报与漏报代价的权衡。

**$k$ 折交叉验证**（定义 3.39，算法 3.9，公式 3.88）将数据的有限性转化为模型评估的可靠性，是实践中超参数选择与泛化误差估计的标准工具；工程上须特别注意避免数据泄露，在时序数据上采用时间感知分折。

**学习曲线与验证曲线**将偏差-方差分解（定理 3.14）可视化：前者以样本量为横轴诊断数据充足性，后者以超参数为横轴确定最优模型复杂度，二者合用可直接指导"应该收集更多数据还是调整正则化"的实践决策。

案例 3.7 的九宫格决策边界比较与 AUC/AP 排行榜表明：在 Breast Cancer 这类中等规模表格数据上，GBDT、随机森林与 SVM-RBF 处于第一梯队，MLP 紧随其后，单棵决策树以牺牲性能换取可解释性。实际选型应综合考虑性能上限、可解释性需求、计算资源与数据规模——没有任何算法在所有场景下都是最优的（"没有免费午餐"定理，Wolpert, 1996）。

**A1.** 一个医学诊断系统对 1000 个样本（100 名患者、900 名健康人）预测结果为：将所有样本预测为健康。计算该系统的准确率、精确率（患病类）、召回率（患病类）和 F1 分数，并说明为何准确率在此场景下是误导性的指标。

**A2.** 解释 ROC 曲线上以下三个点的含义：$(0, 0)$、$(0.5, 0.5)$ 和 $(0, 1)$。对应的分类器各是什么？若一个分类器的 ROC 曲线完全在对角线以下（AUC < 0.5），意味着什么？能否将其利用起来？

**A3.** 在 5 折 CV 中，每折验证集的大小约为全部训练集的 20%。与留一法（LOO-CV）相比，5 折 CV 的偏差和方差分别偏大还是偏小？解释原因并说明何时应优先使用 LOO-CV。

### B 级（公式推导）

**B1.** 证明 $F_1$ 分数可写为 $F_1 = \dfrac{2\,\text{TP}}{2\,\text{TP}+\text{FP}+\text{FN}}$，并从混淆矩阵的四个格子出发验证该公式等价于调和平均定义。

**B2.** 证明定理 3.17：AUC $= P(s(x^+) > s(x^-))$。（提示：将积分 $\int_0^1 \text{TPR}(t)\,\mathrm{d}t$ 改写为对所有阳性-阴性样本对的求和。）

**B3.** 设某分类器在 $k$ 折 CV 中得到验证分数 $e_1, \dots, e_k$。证明：若各折验证集大小相等，则 $\hat{E}_{\text{CV}} = \frac{1}{n}\sum_{i=1}^n \ell(y_i, \hat{f}_{-i}(x_i))$，其中 $\hat{f}_{-i}$ 是不包含样本 $i$ 所在折训练的模型——即 CV 估计等价于对每个样本使用留出该折的模型预测误差之均值。

### C 级（数值计算）

**C1.** 给定混淆矩阵（TP=80, FP=10, FN=20, TN=890），计算准确率、精确率、召回率、特异性和 F1 分数，并以业务语言（如"100 个真实欺诈中有多少被识别出"）解释每个指标。

**C2.** 对某分类器在 6 个测试样本上的预测得分（降序）为 0.9, 0.8, 0.7, 0.6, 0.5, 0.4，对应真实标签为 +, −, +, +, −, −。手动计算每个阈值下的 (FPR, TPR)，绘制 ROC 曲线草图，并用梯形法计算 AUC。

### D 级（编程实践）

### E 级（案例分析）

**E1.** 某医院引进 AI 辅助诊断系统（二分类：患病/健康），系统给出概率分数。从业务角度分析：（1）如何权衡假阳性（过度诊断）与假阴性（漏诊）的代价？（2）在罕见病筛查（患病率 0.1%）和常见病确诊（患病率 40%）两个场景下，应分别优先哪个指标（Precision、Recall、F1 或 AUC）？给出具体的阈值选择建议。

### F 级（开放探讨）

**F1.** "没有免费午餐"定理（Wolpert, 1996）指出：在所有可能的数据分布上对任意两个算法的性能取平均，二者表现等价。这是否意味着算法选择无关紧要？结合本章的实验结果，讨论该定理的实际含义与误解，并说明在真实世界数据分布的假设下，算法选型为何仍然有意义。
