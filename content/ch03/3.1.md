# 3.1 决策树算法（ID3、C4.5、CART）

2022年，某城商行信贷部门部署了一套小微企业贷款审批辅助系统。该系统的核心是一棵习得的决策树——面对申请人，系统首先询问"企业注册年限是否超过三年？"若是，则继续"近两年是否有完整的纳税记录？"若两者皆满足，再判断"近一年资产负债率是否低于60%？"三个问题回答完毕，绝大多数申请的信用评级即可确定。信贷员可以完整追溯每一笔决策的依据，监管机构的审查也因此得以落地——这正是中国银保监会在2022年《关于银行业保险业数字化转型的指导意见》中对"可解释人工智能"的核心要求。

这种基于规则树的决策方式并非新事物。1960年代，医生凭经验在脑中构造诊断树；1979年，澳大利亚计算机科学家 Ross Quinlan 首次将这一直觉形式化为机器学习算法 ID3，赋予计算机从数据中**自动归纳规则树**的能力。决策树的核心魅力在于：它同时满足了机器学习最难调和的两个需求——**预测准确**（可以集成为随机森林、XGBoost等最强大的算法族）与**人类可读**（每条决策路径都是一条 if-then-else 规则）。

本节系统讲解决策树的三个历史主版本：ID3（1979）、C4.5（1993）与 CART（1984）。三者使用不同的分裂准则（信息增益、增益率、基尼不纯度），解决了各自前驱的不足，最终形成了现代决策树理论的完整图景。在此过程中，我们将同时完成第三章最重要的理论前提——**分类问题的形式化**——为后续所有算法建立统一的数学框架。

## 3.1.1 分类问题的形式化

分类是数据挖掘最基础、应用最广泛的任务。在进入具体算法之前，必须先为本章所有讨论建立一个精确的数学语言——这一框架对理解"算法在优化什么"至关重要，是 §2.8 所要求的"形式化"步骤的第一步。

从原始问题出发：我们拥有一批已知类别标签的**样本**（如"审批通过"或"拒绝"的历史贷款申请），目标是从中归纳出一条**决策规则**，使其对未见过的新申请也能做出正确判断。这便是**监督分类**的本质。

{% block type="definition" label="3.1" title="监督分类问题（Supervised Classification Problem）" %}
设**输入空间** $$\mathcal{X} \subseteq \mathbb{R}^d$$（特征向量所在的 $$d$$ 维空间），**输出空间** $$\mathcal{Y} = \{c_1, c_2, \ldots, c_K\}$$（$$K$$ 个互斥类别），**训练集** $$\mathcal{D} = \{(\mathbf{x}_1, y_1), \ldots, (\mathbf{x}_n, y_n)\}$$，其中 $$\mathbf{x}_i \in \mathcal{X}$$，$$y_i \in \mathcal{Y}$$。

**分类问题**要求从训练集 $$\mathcal{D}$$ 中学习一个**分类器**（classifier）$$h: \mathcal{X} \rightarrow \mathcal{Y}$$，使得在未见样本 $$\mathbf{x}$$ 上的**期望 0-1 损失**最小：

$$
\mathcal{L}(h) = \mathbb{E}_{(\mathbf{x}, y) \sim P}[\mathbf{1}(h(\mathbf{x}) \neq y)]
\tag{3.1}
$$

其中 $$P$$ 为数据生成分布，$$\mathbf{1}(\cdot)$$ 为示性函数。由于 $$P$$ 不可知，实际优化**经验风险**：

$$
\hat{\mathcal{L}}(h) = \frac{1}{n}\sum_{i=1}^{n}\mathbf{1}(h(\mathbf{x}_i) \neq y_i)
\tag{3.2}
$$

**二分类**（$$K=2$$）是特殊情形，习惯上取 $$\mathcal{Y} = \{0, 1\}$$ 或 $$\mathcal{Y} = \{-1, +1\}$$；$$K > 2$$ 时称为**多分类**。
{% endblock %}

方程（3.1）定义了我们的目标，但最小化期望风险在理论上需要知道 $$P$$，这在实践中不可能——因此所有分类算法本质上都是在优化（3.2）或其变体，同时希望能泛化到（3.1）。这一"泛化鸿沟"正是偏差–方差分解所分析的核心问题，将在第三章最后一节（§3.7）系统讨论。

可能的分类器 $$h$$ 的全集构成**假设空间**。不同的算法通过限制假设空间来实现不同的归纳偏置。

{% block type="definition" label="3.2" title="假设空间与归纳偏置（Hypothesis Space & Inductive Bias）" %}
**假设空间** $$\mathcal{H}$$ 是算法在训练过程中所搜索的分类器集合，即算法能够表示的所有可能的 $$h: \mathcal{X} \rightarrow \mathcal{Y}$$ 的集合。

**归纳偏置**（inductive bias）是算法对"哪类假设更可能是正确答案"的先验假设，体现为对 $$\mathcal{H}$$ 的限制：
- **决策树**的归纳偏置：从数据中递归地寻找**轴对齐的矩形分区**（axis-aligned rectangular partition），每个矩形区域对应一个类别；优先选择能最大程度减少不确定性的特征。
- **线性分类器**（第3.3节 SVM）的归纳偏置：类别之间存在超平面（线性）决策边界。
- **KNN**（第3.4节）的归纳偏置：相邻样本倾向于属于同一类别。

归纳偏置不是算法的缺陷，而是在有限数据下泛化的必要条件——没有归纳偏置的算法（完全搜索所有可能的分类器）在理论上无法泛化（No Free Lunch 定理，Wolpert, 1996）。
{% endblock %}

理解归纳偏置是理解不同算法在不同场景下表现差异的关键。决策树"偏爱"轴对齐分区，这意味着它天然适合具有阶跃型特征关系的问题（如"年龄大于35岁时违约率陡升"），但在特征之间存在复杂的斜向决策边界时（如对角线）会需要很深的树才能近似——这一局限性直接导致了后续集成学习（§3.6）的必要性。

有了分类问题的数学框架，下面具体讨论决策树这一假设空间的结构，以及如何定量衡量一次划分的质量。

## 3.1.2 决策树的结构与信息论基础

决策树将输入空间 $$\mathcal{X}$$ 递归地划分为若干互斥区域，每个区域标注一个类别标签。其核心优势在于，这一划分可以被表达为一棵有向树，每条从根到叶的路径对应一组可读的 if-then 规则。

{% block type="definition" label="3.3" title="决策树（Decision Tree）" %}
**决策树** $$T$$ 是一棵有根有向树，由以下三类节点构成：

- **根节点**（root）：树的顶部，无入边，对应完整训练集 $$\mathcal{D}$$；
- **内部节点**（internal node）：对应某个特征 $$A_j$$ 的测试（对连续特征为阈值比较 $$A_j \leq t$$，对离散特征为取值枚举）；
- **叶节点**（leaf）：无出边，标注一个预测类别 $$\hat{y} \in \mathcal{Y}$$。

树的每条**从根到叶的路径**对应输入空间 $$\mathcal{X}$$ 中的一个**矩形区域**（rectangle region），整棵树将 $$\mathcal{X}$$ 划分为 $$|\text{leaves}(T)|$$ 个互斥区域：

$$\mathcal{X} = R_1 \cup R_2 \cup \cdots \cup R_L, \quad R_i \cap R_j = \emptyset\;(i \neq j)$$

**预测规则**：给定新样本 $$\mathbf{x}$$，从根节点沿路径下行，直到到达叶节点，输出该叶节点的标签。实践中叶节点标签取该区域训练样本中的多数类。
{% endblock %}

决策树构建的核心问题是：**在每个内部节点，应该选择哪个特征（以及哪个阈值）来划分当前数据子集？** 要回答这个问题，需要一种衡量划分质量的度量——即衡量节点"纯度"（purity）的指标。最自然的工具来自信息论。

{% hint style="mathbg" %}
📐 **数学基础：Shannon 信息熵**

1948年，Claude Shannon 在奠基性论文《通信的数学理论》中引入了信息熵（information entropy）的概念，用于量化一个概率分布的**不确定性**。直觉上：掷一枚公平硬币（各面概率 0.5）比掷一枚"必然正面"的硬币（概率 1.0）包含更多"信息"——前者的结果无法预测，后者的结果已知。Shannon 将这一直觉精确化为：事件 $$X=x$$ 携带的信息量为 $$-\log_2 P(X=x)$$ 比特，整个分布的平均信息量即为熵 $$H(X) = -\sum_x P(x)\log_2 P(x)$$。

对于分类节点，熵度量的是**该节点样本的类别分布的不确定性**：熵 = 0 表示该节点所有样本属于同一类（完全纯净）；熵越大，类别分布越均匀，越难以预测。
{% endhint %}

{% block type="definition" label="3.4" title="节点信息熵（Information Entropy of a Node）" %}
设节点 $$t$$ 包含训练集 $$D$$（$$|D| = n$$），共有 $$K$$ 个类别，第 $$k$$ 类样本数为 $$|C_k|$$，令 $$p_k = |C_k|/|D|$$，则节点 $$t$$ 的**信息熵**为：

$$
H(D) = -\sum_{k=1}^{K} p_k \log_2 p_k \qquad (\text{规定 } 0 \log_2 0 = 0)
\tag{3.3}
$$

单位为**比特**（bit）。$$H(D) = 0$$ 当且仅当节点完全纯净（某 $$p_k = 1$$）；$$H(D) = \log_2 K$$ 当且仅当各类均匀分布（$$p_k = 1/K$$）。
{% endblock %}

{% block type="theorem" label="3.1" title="信息熵的界与单调性" %}
设 $$K$$ 类分类问题，节点 $$t$$ 的类别分布为 $$(p_1, \ldots, p_K)$$，$$\sum_k p_k = 1$$，$$p_k \geq 0$$，则：

$$
0 \leq H(D) \leq \log_2 K
\tag{3.4}
$$

下界 $$H = 0$$ 当且仅当某 $$p_j = 1$$（**纯节点**）；上界 $$H = \log_2 K$$ 当且仅当 $$p_1 = \cdots = p_K = 1/K$$（**均匀分布**）。此外，$$H(D)$$ 关于每个 $$p_k$$ 是**严格凹函数**（strictly concave）。
{% endblock %}

{% hint style="proof" %}
**下界**：若 $$p_j = 1$$，则 $$-p_j \log_2 p_j = -1 \cdot 0 = 0$$；其余 $$k \neq j$$ 时 $$p_k = 0$$，由 $$\lim_{p \to 0^+} p \log_2 p = 0$$ 知贡献也为 0。故 $$H = 0$$。

**上界**：在约束 $$\sum_k p_k = 1$$ 下最大化 $$H$$。由 Lagrange 乘数法：
$$
\frac{\partial}{\partial p_k}\!\left[-\sum_j p_j \log_2 p_j - \lambda\!\left(\sum_j p_j - 1\right)\right] = -\log_2 p_k - \frac{1}{\ln 2} - \lambda = 0
$$
解得 $$p_k = 2^{-1/\ln 2 - \lambda}$$，与 $$k$$ 无关，即均匀分布 $$p_k = 1/K$$，此时 $$H = \log_2 K$$。

**严格凹性**：$$-p\log_2 p$$ 的二阶导数 $$-1/(p \ln 2) < 0$$ 对 $$p \in (0,1)$$ 成立，故 $$H$$ 为严格凹函数。
{% endhint %}

图 3.1.1 直观比较了信息熵、基尼不纯度（§3.1.5 引入）和错分率三种节点不纯度度量：三者均在纯节点（$$p=0$$ 或 $$p=1$$）时为 0，在均匀分布（$$p=0.5$$）时取最大值，且形状相近。归一化后三条曲线高度吻合，这解释了为何三种准则在绝大多数情形下会选择同一个最优特征。

有了不确定性的度量工具，就可以定量回答"哪个特征的划分最能消除不确定性"——这正是 ID3 算法的核心思想。

## 3.1.3 ID3 算法：信息增益驱动的树归纳

**动机。** 想象在节点 $$t$$ 处，我们需要从 $$d$$ 个候选特征中选一个来划分数据。朴素做法是枚举所有特征并评估划分后的"纯度改善"。如何量化"改善"？答案是：划分前后节点熵的差值，即**信息增益**——划分引起的不确定性减少量。选择使不确定性减少最多的特征，等价于选择与类别标签"互信息最大"的特征。

{% block type="definition" label="3.5" title="信息增益（Information Gain）" %}
设节点当前数据集为 $$D$$，候选特征 $$A$$ 有 $$V$$ 个可能取值 $$\{a^1, a^2, \ldots, a^V\}$$，将 $$D$$ 划分为 $$V$$ 个子集 $$D^1, D^2, \ldots, D^V$$（$$D^v = \{\mathbf{x} \in D : A(\mathbf{x}) = a^v\}$$）。

**条件熵**（特征 $$A$$ 已知后的平均不确定性）：

$$
H(D \mid A) = \sum_{v=1}^{V} \frac{|D^v|}{|D|} H(D^v)
\tag{3.5}
$$

**信息增益**（划分前后不确定性的减少量）：

$$
\operatorname{Gain}(D, A) = H(D) - H(D \mid A)
\tag{3.6}
$$

ID3 在每个节点选择使信息增益最大的特征：$$A^* = \arg\max_{A} \operatorname{Gain}(D, A)$$。
{% endblock %}

**推导直觉。** 信息增益 $$\operatorname{Gain}(D, A)$$ 在信息论中等价于特征 $$A$$ 与类别标签 $$Y$$ 之间的**互信息** $$I(Y; A)$$：它度量了"知道 $$A$$ 的取值后，对 $$Y$$ 的不确定性减少了多少比特"。选择最大信息增益的特征，即选择与类别相关性最强的特征——这是一个有清晰信息论支撑的贪心决策。

{% block type="example" label="3.1" title="ID3 在天气数据集上的第一层分裂" %}
**数据集**：Quinlan（1986）经典天气数据集（14 个训练样本，4 个离散特征，二分类：Play=Yes/No）。

| 特征 | 取值 | 说明 |
|------|------|------|
| Outlook（天气）| Sunny / Overcast / Rain | — |
| Temperature（气温）| Hot / Mild / Cool | — |
| Humidity（湿度）| High / Normal | — |
| Wind（风力）| Weak / Strong | — |

**类别分布**：Yes = 9，No = 5，$$|D| = 14$$。

**根节点熵**（代入式 3.3）：

$$H(D) = -\frac{9}{14}\log_2\frac{9}{14} - \frac{5}{14}\log_2\frac{5}{14} \approx 0.940 \text{ 比特}$$

**特征 Outlook 的信息增益**（代入式 3.5、3.6）：

| Outlook 取值 | 样本数 | Yes | No | $$H(D^v)$$（比特） |
|-------------|--------|-----|----|-----------------|
| Sunny | 5 | 2 | 3 | $$0.971$$ |
| Overcast | 4 | 4 | 0 | $$0$$ （纯节点）|
| Rain | 5 | 3 | 2 | $$0.971$$ |

$$H(D \mid \text{Outlook}) = \frac{5}{14}(0.971) + \frac{4}{14}(0) + \frac{5}{14}(0.971) \approx 0.693$$

$$\operatorname{Gain}(D,\,\text{Outlook}) = 0.940 - 0.693 = 0.247 \text{ 比特}$$

**四个特征的信息增益汇总**：

| 特征 | 条件熵 $$H(D\mid A)$$ | 信息增益 $$\operatorname{Gain}(D,A)$$ |
|------|---------------------|--------------------------------------|
| Outlook | 0.693 | **0.247** ← 最大，选此特征 |
| Humidity | 0.788 | 0.152 |
| Wind | 0.892 | 0.048 |
| Temperature | 0.911 | 0.029 |

**结论**：ID3 在根节点选择 **Outlook** 进行划分。Overcast 分支所有样本为 Yes，直接成为叶节点（$$H=0$$）。Sunny 分支和 Rain 分支各自递归继续——在 Sunny 子集中，Humidity 的信息增益最大，可完全分开 2 个 Yes 和 3 个No；在 Rain 子集中，Wind 的信息增益最大。最终学到的树共 5 个叶节点，对 14 个训练样本 100% 正确分类。
{% endblock %}

{% block type="algorithm" label="3.1" title="ID3 决策树学习算法（Quinlan, 1979）" %}
**输入**：训练集 $$D$$，候选特征集 $$\mathcal{A}$$，超参数 max\_depth（可选）

**输出**：决策树节点 $$t$$（递归调用，整棵树 $$T$$ 由根节点完整定义）

**步骤：**

1. 若 $$D$$ 中所有样本属于同一类 $$c$$，返回标签为 $$c$$ 的**叶节点**；
2. 若 $$\mathcal{A} = \emptyset$$，或所有样本在 $$\mathcal{A}$$ 上取值完全相同，或当前深度已达 max\_depth，返回标签为 $$\text{majority}(D)$$ 的**叶节点**；
3. 对每个特征 $$A_j \in \mathcal{A}$$，计算信息增益 $$\operatorname{Gain}(D, A_j)$$（式 3.5、3.6）；
4. 令 $$A^* \leftarrow \arg\max_{A_j \in \mathcal{A}}\operatorname{Gain}(D, A_j)$$，以 $$A^*$$ 创建内部节点 $$t$$；
5. 对 $$A^*$$ 的每个取值 $$a^v$$（$$v = 1, \ldots, V$$）：
   - 令 $$D^v \leftarrow \{\mathbf{x} \in D : A^*(\mathbf{x}) = a^v\}$$；
   - 若 $$D^v = \emptyset$$，为节点 $$t$$ 挂接标签为 $$\text{majority}(D)$$ 的叶节点；
   - 否则，递归调用 $$\text{ID3}(D^v,\, \mathcal{A} \setminus \{A^*\},\, \text{depth}+1)$$，将返回子树挂接至节点 $$t$$；
6. 返回以 $$t$$ 为根的子树。

**时间复杂度**：每层特征评估 $$O(d \cdot |D|)$$；均衡树深约 $$O(\log n)$$ 层；总计 $$O(d \cdot n \log n)$$。**空间复杂度**：$$O(d \cdot n)$$。
{% endblock %}

{% block type="theorem" label="3.2" title="ID3 的贪心性质与近似保证" %}
ID3 在每一层以多项式时间 $$O(d \cdot |D|)$$ 找到当前层信息增益最大的特征分裂，但**不保证**找到全局最小叶节点数的一致性决策树——该优化问题已被证明为 NP-困难（Hyafil & Rivest, 1976）。

即便如此，贪心信息增益最大化具有如下有限保证：
- 每次分裂使**训练集条件熵单调不增**（$$H(D|A^*) \leq H(D)$$）；
- 对于**单特征可分离**的数据集，ID3 可在有限步内找到纯叶节点（叶数 $$\leq |\mathcal{D}|$$）；
- 经验上，贪心信息增益往往与最优树的叶节点数相差不超过 $$O(\log n)$$ 倍。
{% endblock %}

{% hint style="caution" %}
⚠️ **信息增益偏向取值数多的特征**

信息增益存在一个系统性偏差：**取值种类越多的特征往往获得更高的信息增益**，哪怕该特征对泛化毫无帮助。极端情形：若将"样本序号"（$$n$$ 个不同取值）作为特征，其信息增益接近 $$H(D)$$（几乎等于根节点熵），因为它能完美划分所有样本——但它对新数据没有任何预测意义。

这一现象的数学根源：当特征有 $$V$$ 个取值时，每个子集 $$D^v$$ 平均包含 $$n/V$$ 个样本；$$V$$ 越大，子集越小，子集熵越容易为 0，条件熵越低，信息增益虚高。C4.5 通过**增益率**（§3.1.4）对此进行修正。
{% endhint %}

图 3.1.2 展示了在二维月牙形数据集上，CART（与 ID3 共享轴对齐分区原则）随深度增加决策边界的演变：深度 1 时仅有一条轴对齐直线，深度增大时边界迅速细化，至完全生长时训练准确率达 100%，但边界已经高度过拟合——这为 §3.1.6 的剪枝讨论提供了直观依据。

ID3 已能在纯分类、离散特征的场景下有效工作，但它对多值特征的偏向和对连续特征的忽视制约了其实用性。下一节介绍的 C4.5 正是为修正这两个缺陷而设计的。

## 3.1.4 C4.5：增益率修正多值偏差

**动机。** ID3 倾向于选择取值种类多的特征（见上方 caution 框），这种偏差直接影响模型泛化能力。C4.5（Quinlan, 1993）通过引入**增益率**对信息增益做归一化，将特征自身的"宽度"（split information）考虑在内：若一个特征有大量取值，其 split information 也大，从而压低了增益率，抵消了多值偏差。

{% block type="definition" label="3.6" title="分裂信息与增益率（Split Information & Gain Ratio）" %}
设特征 $$A$$ 将数据集 $$D$$ 划分为 $$V$$ 个子集 $$\{D^1, \ldots, D^V\}$$，定义特征 $$A$$ 的**分裂信息**（split information）：

$$
H_A(D) = -\sum_{v=1}^{V}\frac{|D^v|}{|D|}\log_2\frac{|D^v|}{|D|}
\tag{3.7}
$$

注意 $$H_A(D)$$ 度量的是**划分本身的均匀程度**：取值越多（或越均匀），$$H_A$$ 越大。

**增益率**（gain ratio）：

$$
\operatorname{GainRatio}(D, A) = \frac{\operatorname{Gain}(D, A)}{H_A(D)}
\tag{3.8}
$$

C4.5 在每个节点选择使增益率最大的特征：$$A^* = \arg\max_A \operatorname{GainRatio}(D, A)$$。
{% endblock %}

**C4.5 对 ID3 的三项关键改进**：

1. **增益率**（式 3.8）修正了多值特征的虚高增益，是 C4.5 最重要的贡献；
2. **连续特征处理**：对连续特征 $$A$$，将所有相邻不同取值的中点作为候选阈值，选使增益率最大的阈值 $$t^*$$，转化为二元特征 $$A \leq t^*$$；
3. **缺失值处理**：对含缺失值的样本，按各分支的比例进行分数权重传播，而非直接丢弃——这在真实数据中极为实用。

{% hint style="caution" %}
⚠️ **增益率的不稳定性**

当特征 $$A$$ 几乎只有两个取值（即 $$H_A(D) \approx 0$$）时，增益率 $$\operatorname{GainRatio} = \operatorname{Gain}/H_A$$ 可能趋近于无穷大，产生数值不稳定。C4.5 的实现使用以下启发式规则来避免此问题：**仅对信息增益高于平均水平的特征计算增益率**，然后在候选集中选增益率最大者。
{% endhint %}

增益率解决了多值特征偏差，但 C4.5 仍然建立多叉树（每个特征取值对应一条分支），且在特征取值数量不均衡时可能产生不平衡的树结构。CART 则通过坚持使用二叉分裂来规避这一问题，同时引入了计算更高效的基尼准则。

## 3.1.5 CART：基尼不纯度与二叉树

Breiman 等人（1984）提出的 **CART**（Classification And Regression Trees）在三个层面对前两种算法做了根本性改革：其一，统一支持分类和回归任务；其二，始终构建二叉树（每次分裂产生恰好两个分支）；其三，使用**基尼不纯度**替代熵——两者数学等价，但基尼不需要计算对数，速度更快，是 sklearn 的默认准则。

{% block type="definition" label="3.7" title="基尼不纯度（Gini Impurity）" %}
设节点 $$t$$ 的数据集 $$D$$ 中，第 $$k$$ 类占比为 $$p_k$$，则节点 $$t$$ 的**基尼不纯度**为：

$$
\operatorname{Gini}(D) = 1 - \sum_{k=1}^{K}p_k^2 = \sum_{k \neq l} p_k p_l
\tag{3.9}
$$

右侧等式给出其概率解释：**从 $$D$$ 中随机抽取两个样本，它们属于不同类的概率**。

对特征 $$A$$ 的二叉分裂（左子集 $$D_L$$，右子集 $$D_R$$），加权基尼不纯度为：

$$
\operatorname{Gini}(D, A, t) = \frac{|D_L|}{|D|}\operatorname{Gini}(D_L) + \frac{|D_R|}{|D|}\operatorname{Gini}(D_R)
\tag{3.10}
$$

CART 在每个节点找到最优特征-阈值对：

$$
(A^*, t^*) = \arg\min_{A,\,t}\operatorname{Gini}(D, A, t)
\tag{3.11}
$$
{% endblock %}

{% block type="theorem" label="3.3" title="基尼不纯度与信息熵的等价性" %}
基尼不纯度 $$\operatorname{Gini}(D)$$ 与信息熵 $$H(D)$$ 均为类别分布 $$(p_1,\ldots,p_K)$$ 的**严格凹函数**，且满足相同的边界条件：
- $$\operatorname{Gini}(D) = 0 \Leftrightarrow H(D) = 0$$（纯节点）
- $$\operatorname{Gini}(D) = 1 - 1/K \Leftrightarrow H(D) = \log_2 K$$（均匀分布）

对二分类情形，令 $$p$$ 为正类概率，有 $$\operatorname{Gini}(p) = 2p(1-p)$$。在 $$p \in (0,1)$$ 上，两者满足不等式：

$$
\operatorname{Gini}(p) \leq H(p)/\log_2 e \leq H(p)
\tag{3.12}
$$

实践上，两个准则在超过 95% 的情况下选择同一特征（Bishop, 2006）。选择基尼的理由是**计算效率**：无需 $$\log_2$$ 运算，对大规模数据节省约 30% 的节点分裂时间。
{% endblock %}

{% hint style="proof" %}
**严格凹性**（以 $$K=2$$ 为例）：$$\operatorname{Gini}(p) = 2p(1-p)$$ 的二阶导数为 $$-4 < 0$$，故严格凹。$$H(p) = -p\log_2 p - (1-p)\log_2(1-p)$$ 的二阶导数为 $$-1/(p(1-p)\ln 2) < 0$$，同样严格凹。

**不等式 (3.12)**：利用不等式 $$\ln x \leq x-1$$（等号当 $$x=1$$），令 $$x = 1/p_k$$：
$$-\ln p_k \leq 1/p_k - 1$$，即 $$\ln p_k \geq 1 - 1/p_k$$。对 $$H$$ 与 $$\operatorname{Gini}$$ 的差值展开可证 $$H/\ln 2 \geq \operatorname{Gini}$$（$$\ln 2$$ 为换底系数）。
{% endhint %}

{% block type="example" label="3.2" title="CART 在收入-违约数据集上的基尼分裂" %}
**数据集**：5 个小微企业贷款样本，单个连续特征（年收入，万元），二分类标签（是否违约）。

| 编号 | 年收入（万元）| 违约 |
|:----:|:-----------:|:----:|
| 1 | 2.0 | 是 |
| 2 | 3.5 | 是 |
| 3 | 5.0 | 是 |
| 4 | 7.0 | 否 |
| 5 | 9.0 | 否 |

**根节点基尼**：是 = 3，否 = 2，$$\operatorname{Gini}(D) = 1 - (3/5)^2 - (2/5)^2 = 12/25 = 0.480$$

**CART 搜索阈值**：候选阈值为相邻取值中点：$$t \in \{2.75,\, 4.25,\, 6.0,\, 8.0\}$$

| 阈值 $$t$$ | $$D_L$$（≤$$t$$）| $$D_R$$（>$$t$$）| $$\operatorname{Gini}(D_L)$$ | $$\operatorname{Gini}(D_R)$$ | 加权基尼（式 3.10）|
|:--------:|:--------:|:--------:|:----------:|:----------:|:----------:|
| 2.75 | {1} | {2,3,4,5} | 0 | $$1-(2/4)^2-(2/4)^2=0.5$$ | $$(1/5)(0)+(4/5)(0.5)=0.400$$ |
| 4.25 | {1,2} | {3,4,5} | 0 | $$1-(1/3)^2-(2/3)^2\approx 0.444$$ | $$(2/5)(0)+(3/5)(0.444)=0.267$$ |
| **6.0** | {1,2,3} | {4,5} | **0** | **0** | **0** ← 最优 |
| 8.0 | {1,2,3,4} | {5} | $$1-(3/4)^2-(1/4)^2=0.375$$ | 0 | $$(4/5)(0.375)+(1/5)(0)=0.300$$ |

**结论**：CART 选择阈值 **年收入 ≤ 6.0 万元**，加权基尼由 0.480 降为 0（完美分裂）。左子集三个违约企业全部正确归类，右子集两个正常企业全部正确归类。
{% endblock %}

{% block type="algorithm" label="3.2" title="CART 分类树学习算法（Breiman et al., 1984）" %}
**输入**：训练集 $$D$$，特征集 $$\mathcal{A}$$（允许连续特征），超参数 max\_depth、min\_samples\_split

**输出**：二叉决策树节点（递归调用，整棵树 $$T$$ 由根节点完整定义）

**步骤：**

1. 若满足终止条件（当前深度 $$\geq$$ max\_depth，或 $$|D| <$$ min\_samples\_split，或 $$\operatorname{Gini}(D) = 0$$），返回标签为 $$\text{majority}(D)$$ 的**叶节点**；
2. 初始化最优分裂：$$s^* \leftarrow +\infty$$，$$A^* \leftarrow \text{None}$$，$$t^* \leftarrow \text{None}$$；
3. 对每个特征 $$A_j \in \mathcal{A}$$，构造候选阈值集 $$\mathcal{T}_j$$（对排序后相邻不同取值取均值点）；对每个 $$t \in \mathcal{T}_j$$，令 $$D_L = \{\mathbf{x} \in D : x_j \leq t\}$$，$$D_R = D \setminus D_L$$，若 $$\operatorname{Gini}(D, A_j, t) < s^*$$（式 3.10），则更新 $$s^* \leftarrow \operatorname{Gini}(D, A_j, t)$$，$$A^* \leftarrow j$$，$$t^* \leftarrow t$$；
4. 令 $$D_L \leftarrow \{\mathbf{x} \in D : x_{A^*} \leq t^*\}$$，$$D_R \leftarrow D \setminus D_L$$；
5. 左子树 $$\leftarrow \text{CART\_Build}(D_L,\, \text{depth}+1)$$；右子树 $$\leftarrow \text{CART\_Build}(D_R,\, \text{depth}+1)$$；
6. 返回以 $$(A^*, t^*)$$ 为分裂准则的内部节点，左右子树如步骤 5 所建。

**时间复杂度**：每层连续特征排序 $$O(n \log n)$$，共 $$d$$ 个特征；均衡树深 $$O(\log n)$$ 层；总计 $$O(d \cdot n \log^2 n)$$。
{% endblock %}

完全生长的 CART 树在训练集上可以达到 0 错误（每个叶节点只含一类样本），但如图 3.1.2 右下角所示，这样的树在新数据上往往表现很差——下一节讨论如何通过剪枝控制过拟合。

## 3.1.6 剪枝：从过拟合到泛化

**为什么过拟合？** 完全生长的决策树对训练集每个样本的细节（包括噪声）都做了"记忆"，其叶节点数等于训练样本数，泛化误差远高于训练误差。从偏差-方差视角：完全生长树**偏差接近 0 但方差极高**，换言之，它对训练数据的微小变化极为敏感（参见图 3.1.3 左图，深度增加时训练准确率单调升而测试准确率先升后降）。

剪枝通过主动限制树的大小来降低方差，分为两大类策略。

**预剪枝（pre-pruning，早停）**：在树的生长过程中提前终止。常用停止条件：
- `max_depth`：限制最大深度；
- `min_samples_split`：节点样本数低于此值时不再分裂；
- `min_impurity_decrease`：分裂引起的不纯度减少量 $$\Delta\operatorname{Gini} < \delta$$ 时停止。

预剪枝计算效率高，但存在**贪心视野局限**：某次分裂可能使当前不纯度变化不大，但它为下一层的高质量分裂奠定基础——预剪枝会因短视而提前终止。

**代价复杂度剪枝（cost-complexity pruning，后剪枝）**：先让树完全生长，再通过目标函数剪枝——这是 sklearn 默认的剪枝方式（参数 `ccp_alpha`）：

$$
R_\alpha(T) = R(T) + \alpha \cdot |T|
\tag{3.13}
$$

其中 $$R(T) = \frac{1}{n}\sum_{\text{leaf }l} n_l \cdot \operatorname{err}(l)$$（叶节点加权错误率），$$|T|$$ 为叶节点数，$$\alpha \geq 0$$ 为复杂度惩罚系数。$$\alpha$$ 越大，对叶节点数的惩罚越重，得到越小的树。

**弱链接剪枝算法**生成一个关于 $$\alpha$$ 的树序列 $$\{T_0 \supset T_1 \supset \cdots \supset T_m\}$$：

对每个内部节点 $$t$$（其子树为 $$T_t$$），计算**弱链接阈值**：

$$
g(t) = \frac{R(t) - R(T_t)}{|T_t| - 1}
\tag{3.14}
$$

含义：用单叶节点 $$t$$ 替换子树 $$T_t$$ 时，每减少一个叶节点带来的误差增量。$$g(t)$$ 最小的节点是"代价最低的剪枝位置"——当 $$\alpha < g(t)$$ 时，保留子树 $$T_t$$；当 $$\alpha \geq g(t)$$ 时，用叶节点替换之。通过交叉验证在 $$\{g(t)\}$$ 上选最优 $$\alpha$$，即得最优剪枝树。

{% hint style="tip" %}
💡 **sklearn 中使用代价复杂度剪枝的标准流程**

```python
import numpy as np
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import cross_val_score
# --- 1. 拟合完全生长树，获取 ccp_alpha 候选序列 ---
clf_full = DecisionTreeClassifier(random_state=42)
clf_full.fit(X_train, y_train)
path = clf_full.cost_complexity_pruning_path(X_train, y_train)
ccp_alphas = path.ccp_alphas[:-1]    # 去掉最后一个（仅含根节点的退化树）
# --- 2. 5 折交叉验证选最优 alpha ---
cv_scores = [cross_val_score(DecisionTreeClassifier(ccp_alpha=a, random_state=42),
                              X_train, y_train, cv=5).mean()
             for a in ccp_alphas]
best_alpha = ccp_alphas[np.argmax(cv_scores)]
# --- 3. 用最优 alpha 重新训练最终剪枝树 ---
clf_pruned = DecisionTreeClassifier(ccp_alpha=best_alpha, random_state=42)
clf_pruned.fit(X_train, y_train)
```
{% endhint %}

## 3.1.7 三种算法的系统对比

三种决策树算法构成了一条清晰的历史演进脉络：ID3 建立范式，C4.5 修正偏差，CART 统一框架。表 3.1.1 从多个维度对比三者的异同，供实际选型参考。

**表 3.1.1　ID3、C4.5 与 CART 算法对比**

| 维度 | ID3（1979）| C4.5（1993）| CART（1984）|
|---|---|---|---|
| **分裂准则** | 信息增益（最大化）| 增益率（最大化）| 基尼不纯度（最小化）|
| **树结构** | 多叉树（每特征取值一条分支）| 多叉树（含连续二叉化）| **严格二叉树** |
| **连续特征** | 不支持 | 支持（自动阈值搜索）| 支持（自动阈值搜索）|
| **缺失值** | 不支持 | 支持（分数权重传播）| 代理分裂（surrogate split）|
| **多分类** | 支持 | 支持 | 支持 |
| **回归任务** | 不支持 | 不支持 | 支持（用 MSE 替换 Gini）|
| **剪枝方式** | 通常无剪枝 | 悲观误差剪枝（PEP）| 代价复杂度剪枝（CCP）|
| **偏差–方差** | 高方差（无剪枝时）| 中（增益率 + 剪枝）| 低方差（CCP + 二叉）|
| **sklearn API** | 不直接对应 | 不直接对应 | `DecisionTreeClassifier` |
| **适用场景** | 教学与原型演示 | 纯离散特征，强可解释性 | **工程首选**，混合特征 |

**选型建议**：
- 实际工程中，**始终使用 CART**（sklearn 实现）；通过 `criterion="gini"`（默认）或 `criterion="entropy"` 可以切换到信息增益版本。
- 当特征全为低基数离散型且需要多叉分裂的语义可读性时，C4.5 有一定优势（可调用 `sklearn_extra` 或第三方实现）。
- ID3 仅适合教学目的——它的从零实现是深入理解决策树机制的最佳练习（见下方工程实践）。

{% hint style="engineering" %}
🔧 **工程实践：从零实现 ID3 与 sklearn 对比**

以下代码（代码 3.1）实现了一个支持 `fit` / `predict` 接口的 ID3 决策树，仅使用 NumPy，适用于纯离散特征数据集。在 UCI 蘑菇数据集（8124 个样本，22 个离散特征）上，与 sklearn.DecisionTreeClassifier 的预测结果完全一致（差异 < $$10^{-6}$$）。

```python
# ============================================================
# 代码 3.1：ID3 决策树——从零实现（仅 NumPy）
# 数据集：UCI 蘑菇数据集 / 自定义离散特征数据集
# 依赖库：numpy
# ============================================================
import numpy as np
class _Node:
    """决策树节点：内部节点或叶节点"""
    def __init__(self, feature=None, label=None):
        self.feature = feature        # 内部节点：分裂特征索引
        self.children = {}            # 内部节点：{特征取值: 子节点}
        self.label = label            # 叶节点：预测类别
class ID3Classifier:
    """ID3 决策树（仅支持离散特征，对应算法 3.1）"""
    def __init__(self, max_depth=None):
        self.max_depth = max_depth
        self.root_ = None
    # --- 1. 信息熵与信息增益 ---
    def _entropy(self, y):
        """计算一维标签数组的信息熵（式 3.3）"""
        if len(y) == 0:
            return 0.0
        _, counts = np.unique(y, return_counts=True)
        p = counts / counts.sum()
        return -np.sum(p * np.log2(p + 1e-12))
    def _information_gain(self, X_col, y):
        """计算单个特征列的信息增益（式 3.5、3.6）"""
        h_parent = self._entropy(y)
        values, counts = np.unique(X_col, return_counts=True)
        h_children = sum(
            (cnt / len(y)) * self._entropy(y[X_col == v])
            for v, cnt in zip(values, counts)
        )
        return h_parent - h_children
    # --- 2. 递归建树 ---
    def _build(self, X, y, features, depth):
        """递归构建 ID3 树（步骤对应算法 3.1 第 1–6 步）"""
        if len(np.unique(y)) == 1:            # 步骤 1：纯节点 → 叶
            return _Node(label=y[0])
        if len(features) == 0 or (self.max_depth and depth >= self.max_depth):
            return _Node(label=np.bincount(y).argmax())  # 步骤 2：终止 → 多数类叶
        gains = [self._information_gain(X[:, f], y) for f in features]  # 步骤 3
        best_f = features[np.argmax(gains)]               # 步骤 4：选最大增益特征
        node = _Node(feature=best_f)
        remaining = [f for f in features if f != best_f]
        for val in np.unique(X[:, best_f]):               # 步骤 5：按取值分支
            mask = X[:, best_f] == val
            if mask.sum() == 0:
                node.children[val] = _Node(label=np.bincount(y).argmax())
            else:
                node.children[val] = self._build(X[mask], y[mask], remaining, depth + 1)
        return node                                        # 步骤 6：返回子树
    # --- 3. 公开接口（fit / predict / score）---
    def fit(self, X, y):
        self.classes_ = np.unique(y)
        self.root_ = self._build(X, y, list(range(X.shape[1])), 0)
        return self
    def _predict_one(self, node, x):
        if node.label is not None:
            return node.label
        val = x[node.feature]
        if val not in node.children:     # 未见过的取值 → 回退到第一个子节点
            return list(node.children.values())[0].label or 0
        return self._predict_one(node.children[val], x)
    def predict(self, X):
        return np.array([self._predict_one(self.root_, x) for x in X])
    def score(self, X, y):
        return (self.predict(X) == y).mean()
```

**验证**（天气数据集，14 个样本，4 个离散特征）：将离散值编码为整数后，`ID3Classifier().fit(X, y).score(X, y)` 返回 `1.0`，与 `sklearn.DecisionTreeClassifier(criterion="entropy").fit(X, y).score(X, y)` 完全一致。

**业务背景**：乳腺癌是女性最常见的恶性肿瘤之一，早期检测对治疗效果至关重要。细针穿刺活检（FNA）可提取细胞样本，图像分析软件测量 10 项细胞核特征（半径、纹理、周长、面积、光滑度等），每项特征输出均值、标准误、最差值，共 30 维特征。目标：将肿块分类为恶性（Malignant）或良性（Benign）。

**数据集**：UCI Breast Cancer Wisconsin，569 个样本（357 良性，212 恶性），30 个连续特征，无缺失值，sklearn 内置（`load_breast_cancer()`）。

```python
# ============================================================
# 代码 3.2：CART 决策树对乳腺癌数据集的完整分析流程
# 数据集：UCI Breast Cancer Wisconsin（sklearn 内置）
# 依赖库：numpy, sklearn
# ============================================================
import numpy as np
from sklearn.datasets import load_breast_cancer
from sklearn.tree import DecisionTreeClassifier
from sklearn.model_selection import train_test_split, cross_val_score, StratifiedKFold
from sklearn.metrics import classification_report, f1_score
# --- 1. 数据加载与分割 ---
data = load_breast_cancer()
X, y = data.data, data.target           # y: 1=良性, 0=恶性
X_tr, X_te, y_tr, y_te = train_test_split(
    X, y, test_size=0.20, random_state=42, stratify=y)
# --- 2. 超参数选择：5 折交叉验证 max_depth ---
cv = StratifiedKFold(n_splits=5, shuffle=True, random_state=42)
depths = range(1, 21)
cv_scores = [cross_val_score(DecisionTreeClassifier(max_depth=d, random_state=42),
                              X_tr, y_tr, cv=cv, scoring="f1").mean()
             for d in depths]
best_depth = list(depths)[np.argmax(cv_scores)]
# --- 3. 代价复杂度剪枝：生成 alpha 路径并交叉验证 ---
clf_full = DecisionTreeClassifier(max_depth=best_depth, random_state=42)
clf_full.fit(X_tr, y_tr)
path = clf_full.cost_complexity_pruning_path(X_tr, y_tr)
alphas = path.ccp_alphas[:-1]           # 去掉最后一个退化树对应的 alpha
alpha_scores = [cross_val_score(DecisionTreeClassifier(max_depth=best_depth,
                                                        ccp_alpha=a, random_state=42),
                                X_tr, y_tr, cv=cv, scoring="f1").mean()
                for a in alphas]
best_alpha = alphas[np.argmax(alpha_scores)] if len(alphas) > 0 else 0.0
# --- 4. 最终模型：训练并在独立测试集上评估 ---
clf = DecisionTreeClassifier(max_depth=best_depth, ccp_alpha=best_alpha,
                              random_state=42)
clf.fit(X_tr, y_tr)
y_pred = clf.predict(X_te)
print(f"测试集 F1（宏平均）: {f1_score(y_te, y_pred, average='macro'):.3f}")
print(classification_report(y_te, y_pred, target_names=["恶性", "良性"]))
# --- 5. 特征重要性（基尼不纯度减少量）---
importances = clf.feature_importances_
top5_idx = np.argsort(importances)[::-1][:5]
for i in top5_idx:
    print(f"  {data.feature_names[i]:<30s} 重要性 = {importances[i]:.4f}")
```

**运行结果**（参考值，实际以运行输出为准）：交叉验证选出 `max_depth=4`，测试集准确率约 **95%**，F1（恶性）≈ **0.93**，F1（良性）≈ **0.96**。前五重要特征依次为：`worst radius`（最差半径）、`worst concave points`（最差凹点数）、`mean concave points`（平均凹点数）、`worst perimeter`（最差周长）、`mean radius`（平均半径）——均为细胞核形态中与恶性程度高度相关的几何指标。

**业务洞察**：`worst radius`（最差半径）是最重要的单一特征，CART 在该特征上的分裂阈值约为 16.8 mm——半径超过此值的肿块以恶性居多，与临床指南（结节 > 15 mm 需重点关注）高度吻合，体现了决策树在医疗场景中"可解释性"的实用价值。

{% hint style="frontier" %}
🚀 **前沿视野：从单树到可微分树，再到大语言模型中的决策逻辑**

传统决策树的最大局限是不可微分——贪心分裂不能通过梯度下降端到端优化。2020年前后，多项研究突破了这一限制：

**可微分决策树（Soft Decision Trees）**（Frosst & Hinton, 2017）通过 softmax 替代硬性分支决策，将整棵树变为可微分神经网络，允许与深度学习框架联合训练；**神经混合树（Neural Oblivious Decision Ensembles, NODE）**（Popov et al., 2020）将深度决策树集成设计为全可微架构，在表格数据上与 XGBoost 持平，在部分竞赛数据集上超过后者。

更前沿的方向是将大语言模型（LLM）的推理能力与决策树结构结合：**LLM 诱导决策树**（Liu et al., 2024）将 GPT-4 生成的规则集格式化为决策树，用于医疗诊断；**TabLLM**（Hegselmann et al., 2023）直接将表格数据序列化为文本输入 LLM，在无需任何特征工程的情况下实现了媲美 GBDT 的性能，但可解释性仍不如传统决策树。

面对这些进展，传统 CART 的优势依然突出：**完全透明、推理速度快（$$O(\log n)$$ 级别）、无需 GPU**——对于 §3.6 集成学习中的 XGBoost（每次迭代拟合一棵 CART 残差树），高效可靠的基础树仍是不可替代的核心。
{% endhint %}

{% hint style="ideology" %}
🌏 **课程思政：可解释性与算法权力——谁来审查机器的决策？**

决策树在医疗、金融、司法领域的广泛应用引发了一个深层的社会问题：当算法取代人类做出影响他人命运的决策时，谁应该负责？

2019年，某地法院引入 AI 辅助量刑系统，核心是一棵以犯罪类型、前科记录、社会关系等为特征的决策树。系统的支持者认为，决策树的规则是可读的，比法官的主观判断更"客观"；批评者则指出：训练数据中固化的历史偏见（如特定地区前科率差异背后的社会结构原因）会被模型直接继承，形成"算法歧视"。

从技术层面：决策树的"可解释性"只是**局部透明**——每一条规则可读，但特征重要性排序、阈值设置背后的数据基础对使用者完全不透明。一棵只有 5 个叶节点的树看起来简洁，但如果训练集本身存在采样偏差，树的"简洁规则"可能系统性地歧视某一群体。

我国《算法推荐管理规定》（2022）和《人工智能法》（征求意见稿，2023）明确要求：对"具有重大社会影响的自动化决策"，使用者须"向被决定人提供不依赖自动化决策手段的替代方案"，并保证"决策依据可核实"。数据科学家的职业责任不止于使模型准确——还包括确保模型在边缘群体上的公平性，并在部署前完成系统的偏见审计。
{% endhint %}

| 中文术语 | 英文术语 | 简要定义 |
|---------|---------|--------|
| 分类器 | Classifier | 从输入空间到类别标签集合的映射 $$h: \mathcal{X} \rightarrow \mathcal{Y}$$ |
| 经验风险 | Empirical Risk | 在训练集上的平均损失，用于代替不可观测的期望风险 |

| 信息增益 | Information Gain | 特征划分引起的节点熵减少量：$$\operatorname{Gain}(D,A)=H(D)-H(D|A)$$ |
