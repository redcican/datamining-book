# 3.6 集成学习算法

1990年，Robert Schapire 在其博士学位论文中证明了一个令人惊讶的结论：若干个"弱学习器"（weak learner，错误率仅略低于随机猜测的分类器）经过恰当组合，可以得到任意强的"强学习器"（strong learner）——这一结论即**提升定理**（Boosting Theorem），奠定了集成学习理论的基础。四年后，Freund 与 Schapire（1995）将其工程化为 AdaBoost 算法，在一系列标准数据集上大幅超越了当时最优的单一分类器，并因此荣获 2003 年 Gödel 奖。

与此同时，Breiman（1996）从截然不同的角度出发，以 **Bootstrap 聚合**（Bagging）的形式证明：通过随机重采样生成多个训练集并并行训练多棵决策树，集成后的模型方差可以系统性地降至接近零。Breiman 随后将 Bagging 与特征随机化相结合，在 2001 年提出了**随机森林**（Random Forest）——这一算法至今仍是表格数据上最稳定、最广泛使用的分类方法之一。

集成学习的核心哲学是：**单个学习器的错误是随机的，而多个独立学习器的错误是不相关的；不相关的错误可以通过平均而消除，相关的偏差却不能**。这一哲学在数学上对应偏差-方差分解框架——本节的出发点。

## 3.6.1 集成的直觉：多数表决与投票误差

在集成学习的直觉理解层面，一个经典的统计学思想实验至关重要。设有 $T$ 个**相互独立**的二分类器，每个分类器的错误率均为 $\varepsilon < 0.5$。对某一样本，采用多数表决（Majority Voting），集成错误当且仅当超过半数的分类器出错。

{% block type="theorem" label="3.13" title="多数表决误差界（Condorcet 陪审团定理推广）" %}
设 $T$ 个独立的二分类器，每个分类器的错误率均为 $\varepsilon \in (0, 0.5)$。多数表决集成的错误率 $\varepsilon_{\text{ens}}$ 满足：

$$
\varepsilon_{\text{ens}} = \sum_{k=\lceil T/2 \rceil}^{T} \binom{T}{k} \varepsilon^k (1-\varepsilon)^{T-k}
$$

由 Hoeffding 不等式，当 $T \to \infty$ 时：

$$
\varepsilon_{\text{ens}} \leq e^{-2T(\frac{1}{2}-\varepsilon)^2} \to 0
\tag{3.68}
$$

即集成误差以指数速率趋近于零。
{% endblock %}

{% hint style="proof" %}
由二项分布的尾界，$\Pr[\text{超过 } T/2 \text{ 个分类器出错}] = \Pr[Z \geq T/2]$，其中 $Z \sim \text{Binomial}(T, \varepsilon)$，均值 $\mathbb{E}[Z] = T\varepsilon < T/2$。Hoeffding 不等式给出 $\Pr[Z \geq T/2] \leq \exp\!\left(-2T\left(\frac{1}{2}-\varepsilon\right)^2\right)$，即 (3.68)。∎
{% endhint %}

定理 3.13 揭示了集成的两个关键前提：**（1）每个基学习器的错误率须低于 0.5；（2）基学习器之间须尽可能独立（不相关）**。现实中两者均难以完全满足，但 Bagging 和 Boosting 各自从不同角度逼近这两个条件：Bagging 通过数据随机化降低相关性，Boosting 通过自适应权重确保每个弱分类器的准确率超过随机基线。图 3.6.1 对比了两种框架的结构差异。

{% hint style="ideology" %}
🌏 **集成思想与集体智慧**

集成学习在认识论层面呼应了一个古老的智慧：群体判断在特定条件下优于个体。经济学家 Francis Galton 在 1907 年记录的"牛的体重猜测"实验中，787 名参与者的猜测中位数为 1207 磅，与实际重量 1198 磅仅相差 0.8%——任何单个参与者的准确性均不及群体中位数。这与集成学习的核心机制完全一致：个体误差若相互独立，则通过聚合可以系统消除随机噪声，留下信号。

然而，群体智慧也存在失效边界——当个体之间高度相关（如所有基分类器均使用相同的特征子集），或所有个体系统性地偏向同一错误方向（高偏差），集成便无能为力。**认识集成的局限，与认识集成的威力，同等重要。**
{% endhint %}

## 3.6.2 偏差-方差分解：集成的理论基础

多数表决误差界（定理 3.13）在独立性假设下优美，但现实中基学习器无法完全独立。更一般的分析框架是**偏差-方差分解**（Bias-Variance Decomposition），它精确刻画了集成降低方差的机制，而不依赖独立性假设。

本节推导均方预测误差的完整分解。设真实目标为 $y = f(\mathbf{x}) + \varepsilon$，其中 $f$ 为真实函数，$\varepsilon \sim (0, \sigma^2)$ 为不可约噪声；$\hat{f}(\mathbf{x})$ 为从训练集 $\mathcal{D}$ 学习到的预测函数，其随机性来自训练集的抽取。

{% block type="theorem" label="3.14" title="偏差-方差-噪声分解" %}
均方预测误差（Mean Squared Prediction Error）对任意测试点 $\mathbf{x}$ 可分解为：

$$
\mathbb{E}_{\mathcal{D},\varepsilon}\!\left[\bigl(y - \hat{f}(\mathbf{x})\bigr)^2\right]
= \underbrace{\bigl(f(\mathbf{x}) - \mathbb{E}_\mathcal{D}[\hat{f}(\mathbf{x})]\bigr)^2}_{\text{Bias}^2(\hat{f})}
+ \underbrace{\mathbb{E}_\mathcal{D}\!\left[\bigl(\hat{f}(\mathbf{x}) - \mathbb{E}_\mathcal{D}[\hat{f}(\mathbf{x})]\bigr)^2\right]}_{\text{Var}(\hat{f})}
+ \underbrace{\sigma^2}_{\text{噪声}}
\tag{3.69}
$$

三项分别为**偏差平方**（模型系统性误差的平方）、**方差**（模型对训练集波动的敏感程度）与**不可约噪声**（数据本身的随机性）。
{% endblock %}

{% hint style="proof" %}
记 $\bar{f} = \mathbb{E}_\mathcal{D}[\hat{f}(\mathbf{x})]$，$f = f(\mathbf{x})$，则

$$
\mathbb{E}_{\mathcal{D},\varepsilon}\!\left[(y - \hat{f})^2\right]
= \mathbb{E}\!\left[(f + \varepsilon - \hat{f})^2\right]
= \mathbb{E}\!\left[(f - \hat{f})^2\right] + 2\mathbb{E}[(f-\hat{f})\varepsilon] + \mathbb{E}[\varepsilon^2]
$$

由于 $\varepsilon$ 与 $\hat{f}$（通过训练集）相互独立，交叉项 $\mathbb{E}[(f-\hat{f})\varepsilon]=0$。对第一项展开，加减 $\bar{f}$：

$$
\mathbb{E}\!\left[(f - \hat{f})^2\right]
= \mathbb{E}\!\left[(f - \bar{f} + \bar{f} - \hat{f})^2\right]
= (f-\bar{f})^2 + 2(f-\bar{f})\underbrace{\mathbb{E}[\bar{f}-\hat{f}]}_{=0} + \mathbb{E}\!\left[(\hat{f}-\bar{f})^2\right]
$$

合并即得 (3.69)。∎
{% endhint %}

定理 3.14 揭示了集成学习的两类策略的数学本质：

**Bagging 降低方差**。若 $T$ 个基学习器的预测 $\hat{f}_1, \ldots, \hat{f}_T$ 两两相关系数为 $\rho$，且每个学习器方差为 $\sigma_b^2$，则集成平均 $\bar{f}_T = \frac{1}{T}\sum_{t=1}^T \hat{f}_t$ 的方差为：

$$
\text{Var}(\bar{f}_T)
= \frac{\sigma_b^2}{T} + \frac{T-1}{T}\,\rho\,\sigma_b^2
= \rho\,\sigma_b^2 + \frac{1-\rho}{T}\,\sigma_b^2
\tag{3.70}
$$

当 $T \to \infty$ 时，方差的下界为 $\rho\,\sigma_b^2$——仅由基学习器之间的相关性决定。因此，**Bagging 的核心目标是在保持较低偏差的同时，最大化基学习器之间的独立性**（降低 $\rho$）。

**Boosting 降低偏差**。Boosting 策略不依赖多样性，而是通过顺序地专注于前一轮未能拟合的残差，逐步减小集成的系统性偏差，代价是可能增大方差（需正则化控制）。

公式 (3.70) 同时说明了：即使 $T$ 很大，若 $\rho \approx 1$（基学习器高度相关），集成增益也几近于零。这正是随机森林引入特征随机化的动机。图 3.6.2 以实验曲线直观展示了这一理论结果。

{% hint style="mathbg" %}
📐 **公式 (3.70) 的含义**

将 (3.70) 改写为 $\text{Var}(\bar{f}_T) = \rho\,\sigma_b^2 + (1-\rho)\,\sigma_b^2/T$，可以看出方差由两部分构成：

- **不可消除部分** $\rho\,\sigma_b^2$：基学习器共享信息导致的"公共误差"，无论 $T$ 多大都无法通过平均消除；
- **可消除部分** $(1-\rho)\,\sigma_b^2/T$：相互独立的"特有误差"，随 $T$ 增大以 $1/T$ 速率趋近于零。

这直接说明了为什么**决策树（高方差、低偏差）是 Bagging 的理想基学习器**：其高方差在集成后可被大幅压缩，而低偏差不会因聚合而损失。相比之下，线性模型（低方差、高偏差）对 Bagging 的受益有限。
{% endhint %}

## 3.6.3 Bagging 与随机森林

Bagging 的核心是 **Bootstrap 重采样**：从 $n$ 个样本的原始训练集 $\mathcal{D}$ 中有放回地随机抽取 $n$ 个样本，得到子集 $\mathcal{D}_t$；对每个 $\mathcal{D}_t$ 独立训练基学习器 $h_t$。有放回采样意味着平均每个样本被采到的概率为 $1-(1-1/n)^n \approx 1-e^{-1} \approx 0.632$，约 36.8% 的样本不会出现在 $\mathcal{D}_t$ 中，这批样本构成该轮的**袋外集**（Out-of-Bag, OOB）。

{% block type="definition" label="3.30" title="Bagging（Bootstrap Aggregating）" %}
给定训练集 $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n$ 与基学习器 $\mathcal{A}$，**Bagging** 执行如下步骤：

对 $t = 1, \ldots, T$：
1. 从 $\mathcal{D}$ 有放回地均匀抽样 $n$ 次，得 $\mathcal{D}_t$（Bootstrap 子集）；
2. 训练基学习器 $h_t = \mathcal{A}(\mathcal{D}_t)$；

集成预测（分类）：

$$
H(\mathbf{x}) = \arg\max_{c \in \mathcal{Y}} \sum_{t=1}^T \mathbf{1}[h_t(\mathbf{x}) = c]
\tag{3.71}
$$

**OOB 误差估计**：对每个样本 $(\mathbf{x}_i, y_i)$，以未使用 $\mathbf{x}_i$ 的基学习器集合 $\mathcal{T}_i = \{t : \mathbf{x}_i \notin \mathcal{D}_t\}$ 进行预测，得到 OOB 预测 $\hat{y}_i^{\text{OOB}} = \arg\max_c \sum_{t \in \mathcal{T}_i} \mathbf{1}[h_t(\mathbf{x}_i)=c]$，OOB 误差为 $\frac{1}{n}\sum_{i=1}^n \mathbf{1}[\hat{y}_i^{\text{OOB}} \neq y_i]$。
{% endblock %}

OOB 误差的统计价值在于：每次预测仅使用未见过该样本的基学习器，这模拟了留一交叉验证（LOOCV）的评估效果，且无需额外训练——是大规模场景下超参数调优的高效替代方案。

随机森林在 Bagging 的基础上引入第二层随机化：每次分裂节点时，不从全部 $d$ 个特征中选择最优分裂，而是仅在随机抽取的 $m_{\text{try}}$ 个特征中寻找最优分裂（$m_{\text{try}} \ll d$，默认 $m_{\text{try}} = \lfloor\sqrt{d}\rfloor$）。这一操作进一步降低了基学习器之间的相关系数 $\rho$（公式 3.70），以略微牺牲单棵树精度（因特征候选集缩小）换取更强的集成增益。

{% block type="definition" label="3.31" title="随机森林（Random Forest）" %}
**随机森林**是在 Bagging 框架下，以完全生长的决策树为基学习器，并在每个节点的分裂时引入**特征随机子采样**的集成方法（Breiman, 2001）：

- 每棵树 $h_t$ 从 Bootstrap 子集 $\mathcal{D}_t$ 生长，不剪枝（最大深度不限）；
- 每个内部节点的分裂特征候选集从 $d$ 维中随机抽取 $m_{\text{try}}$ 维，默认：
  - 分类：$m_{\text{try}} = \lfloor\sqrt{d}\rfloor$
  - 回归：$m_{\text{try}} = \lfloor d/3 \rfloor$

集成预测同公式 (3.71)。**特征重要性**（Feature Importance）可由每个特征在所有树的所有分裂节点上贡献的不纯度下降总量均值估计：

$$
\text{FI}(j) = \frac{1}{T}\sum_{t=1}^T \sum_{\text{节点 } v \in h_t,\; \text{分裂特征} = j} \Delta\text{Gini}(v)
\tag{3.72}
$$

其中 $\Delta\text{Gini}(v)$ 为节点 $v$ 处加权基尼不纯度的降低量。
{% endblock %}

{% block type="algorithm" label="3.7" title="随机森林训练与 OOB 评估" %}
**输入**：训练集 $\mathcal{D}=\{(\mathbf{x}_i,y_i)\}_{i=1}^n$，树数量 $T$，特征子集大小 $m_\text{try}$

**输出**：基学习器集合 $\{h_t\}_{t=1}^T$，OOB 误差估计，特征重要性向量

1. 初始化 OOB 投票矩阵 $V \in \mathbb{R}^{n \times K}$ 为全零（$K$ 为类别数）
2. **对** $t = 1, \ldots, T$：
   1. 有放回抽取 $n$ 个样本，得 Bootstrap 子集 $\mathcal{D}_t$，记录袋外集 $\text{OOB}_t = \mathcal{D} \setminus \mathcal{D}_t$
   2. 从 $\mathcal{D}_t$ 生长决策树 $h_t$：每个节点分裂时从 $d$ 维中随机抽 $m_{\text{try}}$ 维，选最优分裂，生长至纯节点（不剪枝）
   3. 对 $\text{OOB}_t$ 中每个 $(\mathbf{x}_i, y_i)$，将 $h_t(\mathbf{x}_i)$ 对应的列 $V[i, h_t(\mathbf{x}_i)]$ 加 1
   4. 更新特征重要性累加器（公式 3.72）
3. OOB 误差 $= \frac{1}{n}\sum_{i=1}^n \mathbf{1}[\arg\max_c V[i,c] \neq y_i]$
4. **返回** $\{h_t\}$，OOB 误差，特征重要性（归一化至和为 1）

**时间复杂度**：$O(T \cdot n \cdot d^{1/2} \cdot \log n)$（每棵树构建），可完全并行化。
{% endblock %}

{% block type="example" label="3.9" title="Bootstrap 子集的覆盖率与 OOB 期望比例" %}
对训练集大小 $n$，有放回抽取 $n$ 次，单个样本**未被**抽中的概率为 $(1-1/n)^n$，这是 OOB 样本的期望比例。

当 $n=100$：$(1-0.01)^{100} = 0.99^{100} \approx 0.366$

当 $n=1000$：$(1-0.001)^{1000} \approx e^{-1} \approx 0.368$

**结论**：无论 $n$ 多大，OOB 比例稳定收敛至 $e^{-1} \approx 36.8\%$。这意味着每棵树平均在约 $63.2\%$ 的样本上训练，另外约 $36.8\%$ 的样本作为天然的"验证集"——无需额外划分数据，OOB 估计的计算开销已被包含在随机森林的正常训练过程中。
{% endblock %}

{% hint style="caution" %}
⚠️ **随机森林的局限：外推与高维稀疏**

随机森林在以下场景中表现欠佳：

1. **外推能力缺失**：决策树（包括随机森林）的预测值由训练集中叶节点样本的标签均值决定，因此对训练数据范围之外的区域预测恒为某一常数值，无法外推——这在时序预测中尤为危险（见 §7 章）。
2. **高维极稀疏特征**：当 $d \gg n$ 且真正相关的特征仅有少数几个时，随机特征子采样可能导致每棵树的分裂普遍错过有效特征，集成质量大幅下降（此时 $m_{\text{try}}$ 应增大）。
3. **类别特征高基数**：若某特征取值超过 100 种，基于 Gini 的节点分裂会系统性地偏向高基数特征，引入偏差（Strobl et al., 2007）。

在这些场景中，梯度提升树（§3.6.5）通常是更稳健的选择。
{% endhint %}

## 3.6.4 提升法：AdaBoost 与指数损失最小化

AdaBoost（Adaptive Boosting）由 Freund 与 Schapire（1997）提出，是第一个具备严格理论保证的实用 Boosting 算法。其核心机制是：在每一轮训练中，**增大被前一轮基学习器误分类样本的权重**，使新的基学习器集中学习这些"难样本"，并以自适应系数 $\alpha_t$ 加权合并各轮的预测。

以下从损失函数的角度推导 AdaBoost 的权重更新规则，揭示其内在的数学结构。

{% block type="definition" label="3.32" title="AdaBoost 集成模型" %}
设训练集 $\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n$，$y_i \in \{-1, +1\}$，基学习器 $\mathcal{A}$。AdaBoost 的集成模型为：

$$
H(\mathbf{x}) = \text{sign}\!\left(F(\mathbf{x})\right), \quad F(\mathbf{x}) = \sum_{t=1}^T \alpha_t h_t(\mathbf{x})
\tag{3.73}
$$

其中 $h_t(\mathbf{x}) \in \{-1, +1\}$ 为第 $t$ 轮的弱分类器，$\alpha_t > 0$ 为其权重。训练目标是最小化**指数损失**（Exponential Loss）：

$$
\mathcal{L}_{\text{exp}}(F) = \frac{1}{n}\sum_{i=1}^n e^{-y_i F(\mathbf{x}_i)}
\tag{3.74}
$$
{% endblock %}

指数损失 $e^{-yF(\mathbf{x})}$ 对正确分类的样本（$yF > 0$）给予低惩罚，对错误分类的样本（$yF < 0$）给予指数级高惩罚——这正是"自适应增大难样本权重"的数学来源。

**权重更新的推导**。设第 $t$ 轮已有 $F_{t-1}(\mathbf{x}) = \sum_{s=1}^{t-1}\alpha_s h_s(\mathbf{x})$，需要确定新的 $(h_t, \alpha_t)$ 使 $\mathcal{L}_{\text{exp}}$ 最小。固定 $F_{t-1}$，令 $\tilde{\mathcal{L}} = \sum_{i=1}^n e^{-y_i F_{t-1}(\mathbf{x}_i)} \cdot e^{-y_i \alpha_t h_t(\mathbf{x}_i)}$。记 $w_i^{(t)} = e^{-y_i F_{t-1}(\mathbf{x}_i)}$（归一化后即为第 $t$ 轮的样本权重），则：

$$
\tilde{\mathcal{L}} = \sum_{i: h_t(\mathbf{x}_i) = y_i} w_i^{(t)} e^{-\alpha_t} + \sum_{i: h_t(\mathbf{x}_i) \neq y_i} w_i^{(t)} e^{+\alpha_t}
\tag{3.75}
$$

对 $h_t$ 最优化等价于最小化 $\sum_{i=1}^n w_i^{(t)} \mathbf{1}[h_t(\mathbf{x}_i)\neq y_i]$——即加权分类误差，这正是"以样本权重 $w_i^{(t)}$ 训练弱分类器"的依据。令加权误差率 $\varepsilon_t = \frac{\sum_i w_i^{(t)}\mathbf{1}[h_t\neq y_i]}{\sum_i w_i^{(t)}}$，对 $\alpha_t$ 求导并令其为零：

$$
\frac{\partial \tilde{\mathcal{L}}}{\partial \alpha_t} = -e^{-\alpha_t}(1-\varepsilon_t) + e^{\alpha_t}\varepsilon_t = 0
\implies
\alpha_t = \frac{1}{2}\ln\!\frac{1-\varepsilon_t}{\varepsilon_t}
\tag{3.76}
$$

权重更新：$w_i^{(t+1)} \propto w_i^{(t)} \cdot e^{-\alpha_t y_i h_t(\mathbf{x}_i)}$，即误分类样本权重乘以 $e^{\alpha_t} > 1$，正确分类样本权重乘以 $e^{-\alpha_t} < 1$。

{% block type="algorithm" label="3.8" title="AdaBoost（Freund & Schapire, 1997）" %}
**输入**：训练集 $\{(\mathbf{x}_i, y_i)\}_{i=1}^n$（$y_i \in \{-1,+1\}$），基学习器 $\mathcal{A}$，轮数 $T$

**输出**：集成分类器 $H(\mathbf{x}) = \text{sign}(F(\mathbf{x}))$

1. 初始化均匀权重 $w_i^{(1)} = 1/n$，$i = 1, \ldots, n$
2. **对** $t = 1, \ldots, T$：
   1. 用加权训练集 $\{(\mathbf{x}_i, y_i, w_i^{(t)})\}$ 训练弱分类器 $h_t = \mathcal{A}(\mathcal{D}, \mathbf{w}^{(t)})$
   2. 计算加权误差率 $\varepsilon_t = \frac{\sum_{i=1}^n w_i^{(t)}\mathbf{1}[h_t(\mathbf{x}_i)\neq y_i]}{\sum_{i=1}^n w_i^{(t)}}$
   3. **若** $\varepsilon_t \geq 0.5$，终止（弱学习器不满足条件）
   4. 计算分类器权重 $\alpha_t = \frac{1}{2}\ln\!\dfrac{1-\varepsilon_t}{\varepsilon_t}$（公式 3.76）
   5. 更新样本权重 $w_i^{(t+1)} = w_i^{(t)} \cdot e^{-\alpha_t y_i h_t(\mathbf{x}_i)}$，归一化至和为 1
3. **返回** $H(\mathbf{x}) = \text{sign}\!\left(\sum_{t=1}^T \alpha_t h_t(\mathbf{x})\right)$
{% endblock %}

AdaBoost 的训练误差具有严格的收敛保证（见定理 3.15）。

{% block type="theorem" label="3.15" title="AdaBoost 训练误差上界（Freund & Schapire, 1997）" %}
设每一轮弱分类器的加权误差率满足 $\varepsilon_t \leq \frac{1}{2} - \gamma$（$\gamma > 0$），则 AdaBoost 经 $T$ 轮后的训练误差满足：

$$
\frac{1}{n}\sum_{i=1}^n \mathbf{1}[H(\mathbf{x}_i) \neq y_i] \leq e^{-2\gamma^2 T}
\tag{3.77}
$$

即训练误差以指数速率趋近于零。
{% endblock %}

{% hint style="proof" %}
指数损失是 0-1 损失的上界：$\mathbf{1}[H(\mathbf{x}_i)\neq y_i] \leq e^{-y_i F(\mathbf{x}_i)}$（因为 $H$ 预测错误当且仅当 $y_i F \leq 0$，此时 $e^{-y_i F} \geq 1$）。故 $\frac{1}{n}\sum_i \mathbf{1}[\cdot] \leq \frac{1}{n}\sum_i e^{-y_i F(\mathbf{x}_i)}$。将总损失展开为 $T$ 轮乘积，利用每轮权重更新可以证明 $\frac{1}{n}\sum_i e^{-y_i F(\mathbf{x}_i)} = \prod_{t=1}^T 2\sqrt{\varepsilon_t(1-\varepsilon_t)} \leq e^{-2\gamma^2 T}$，其中最后一步利用了 $\sqrt{\varepsilon(1-\varepsilon)} \leq e^{-\gamma^2/2}$（当 $\varepsilon \leq 1/2 - \gamma$）。∎
{% endhint %}

{% hint style="caution" %}
⚠️ **AdaBoost 的实际局限**

定理 3.15 仅保证**训练误差**的收敛，对**泛化误差**（测试误差）不作直接保证。实践中，AdaBoost 对含噪标注数据极为敏感：当某些样本的标签错误时，这些样本在反复误分类后会获得极高权重，导致后续基学习器过度拟合噪声点——即 AdaBoost 在高噪声场景下容易过拟合。GBDT（§3.6.5）通过使用连续损失函数与收缩系数（Shrinkage）缓解了这一问题。
{% endhint %}

## 3.6.5 梯度提升树与 XGBoost

**梯度提升树**（Gradient Boosting Decision Tree, GBDT）由 Friedman（2001）提出，将 Boosting 从离散标签上的"权重调整"推广到任意可微损失函数上的**函数空间梯度下降**，是目前竞赛与工业界最广泛使用的集成方法之一。

核心思想：将集成模型 $F(\mathbf{x}) = \sum_{t=1}^T f_t(\mathbf{x})$（$f_t$ 为回归树）视为函数空间中的加法模型，每轮 $t$ 通过最速下降法，沿损失函数 $\mathcal{L}$ 对当前预测 $F_{t-1}(\mathbf{x})$ 的负梯度方向（即"伪残差"）拟合新的基学习器 $f_t$。

{% block type="definition" label="3.33" title="梯度提升（Gradient Boosting）" %}
设损失函数 $\mathcal{L}(y, F) = \sum_{i=1}^n \ell(y_i, F(\mathbf{x}_i))$，当前模型为 $F_{t-1}$。第 $t$ 轮的**负梯度残差**（伪残差，Pseudo-Residual）为：

$$
r_{ti} = -\left[\frac{\partial \ell(y_i, F(\mathbf{x}_i))}{\partial F(\mathbf{x}_i)}\right]_{F = F_{t-1}}, \quad i = 1, \ldots, n
\tag{3.78}
$$

用回归树 $f_t$ 拟合 $\{(\mathbf{x}_i, r_{ti})\}_{i=1}^n$，然后以**收缩系数**（Shrinkage / Learning Rate）$\eta \in (0,1]$ 更新模型：

$$
F_t(\mathbf{x}) = F_{t-1}(\mathbf{x}) + \eta \cdot f_t(\mathbf{x})
\tag{3.79}
$$

对均方损失 $\ell = \frac{1}{2}(y-F)^2$，负梯度恰好等于残差 $r_{ti} = y_i - F_{t-1}(\mathbf{x}_i)$；对对数损失，伪残差对应每个样本的"预测概率 – 真实标签"。
{% endblock %}

梯度提升框架的优雅之处在于其通用性：只需替换损失函数 $\ell$，便可将同一框架适配到分类（对数损失）、回归（MSE/MAE）、排序（LambdaRank 损失）等不同任务，无需改变算法主体。

**XGBoost 的工程优化**（Chen & Guestrin, 2016）在 GBDT 基础上引入三项关键改进：

1. **二阶 Taylor 展开**：不仅使用一阶梯度 $g_i = \partial\ell/\partial\hat{y}_i^{(t-1)}$，还使用二阶梯度 $h_i = \partial^2\ell/(\partial\hat{y}_i^{(t-1)})^2$，使叶节点权重具有解析最优解 $w_j^* = -\sum_{i\in j}g_i / (\sum_{i\in j}h_i + \lambda)$，避免对每个候选分裂重新拟合；

2. **节点分裂增益**（精确扫描 + 近似分位数）：分裂增益 $\text{Gain} = \frac{1}{2}\!\left[\frac{G_L^2}{H_L+\lambda} + \frac{G_R^2}{H_R+\lambda} - \frac{(G_L+G_R)^2}{H_L+H_R+\lambda}\right] - \gamma$，其中 $\gamma$ 为最小增益阈值（树的正则化），$\lambda$ 为叶节点 L2 正则系数；

3. **稀疏感知算法**：专为缺失值与稀疏特征设计的分裂路径策略，使 XGBoost 在包含大量缺失值的真实数据上无需显式插补。

{% hint style="engineering" %}
🔧 **XGBoost/LightGBM 的工程实践要点**

在工业界 Tabular 数据竞赛（Kaggle）与生产系统中，梯度提升树占据主导地位。以下是最重要的调参经验：

**核心超参数**：
- `n_estimators`（树数量）+ `learning_rate`（收缩系数）：两者负相关，建议固定 `learning_rate=0.05–0.1`，用早停（Early Stopping on validation set）确定最优 `n_estimators`；
- `max_depth`（GBDT 通常 3–6 层）+ `min_child_weight`/`min_samples_leaf`：控制树的复杂度，防止过拟合；
- `subsample`（列/行采样比例，类似 Bagging 随机性，范围 0.6–0.9）：引入随机性降低过拟合。

**LightGBM vs XGBoost**：LightGBM 的 Leaf-wise 生长策略（按最大增益叶节点生长，而非按层生长）在稀疏高维数据上收敛更快，但更容易过拟合，需配合 `num_leaves` 限制；XGBoost 的 Level-wise 策略更稳定，适合小数据集。

**实际经验**：在竞赛中，CatBoost（处理类别特征的专门实现）+ LightGBM + XGBoost 三模型 Stacking（§3.6.6）是成熟的高分方案。
{% endhint %}

## 3.6.6 Stacking：元学习框架

Bagging 与 Boosting 均基于**同质集成**（homogeneous ensemble）：所有基学习器属于同一类算法（均为决策树）。**Stacking**（Stacked Generalization，Wolpert, 1992）则是一种**异质集成**（heterogeneous ensemble）框架：组合来自不同算法族的基学习器，通过元学习器（Meta-Learner）在基学习器的预测上进行二次建模。

{% block type="definition" label="3.34" title="Stacking（堆叠泛化）" %}
**Stacking** 将集成分为两层：

**第一层（基学习器层）**：$M$ 个不同类型的基学习器 $\{h_1^{\text{base}}, \ldots, h_M^{\text{base}}\}$（如决策树、SVM、神经网络），每个基学习器的输出 $\hat{p}_m(\mathbf{x}) = h_m^{\text{base}}(\mathbf{x})$ 作为"元特征"（Meta-Feature）。

**第二层（元学习器层）**：一个元学习器 $h^{\text{meta}}$ 将 $M$ 个基学习器的预测向量 $(\hat{p}_1, \ldots, \hat{p}_M)^\top$ 作为输入，输出最终预测。

为防止数据泄露，元特征须以 $k$ 折交叉验证的**袋外预测**（Hold-Out Prediction）生成：对第 $k$ 折，用其余 $k-1$ 折训练的基学习器对第 $k$ 折的样本做预测，从而确保每个样本的元特征来自未见过该样本的模型。
{% endblock %}

{% hint style="caution" %}
⚠️ **Stacking 的数据泄露陷阱**

在 Stacking 中，若直接用全量训练集训练基学习器并生成元特征，则元学习器的训练数据（元特征）已包含了对测试集（元学习器本身的评估集）特征的隐式信息——这是一种**目标泄露**（Target Leakage），会导致对泛化性能的乐观估计。

**正确做法**：使用 $k$ 折 OOF（Out-of-Fold）预测生成元特征，或保留专门的元训练集（Holdout Meta-Train），确保元学习器在训练时从未见过任何样本的真实标签。Sklearn 的 `StackingClassifier` 已实现此保护。
{% endhint %}

{% hint style="think" %}
🤔 **思考**：Stacking 与 Bagging 的区别不仅在于基学习器的异质性，更在于信息流向。Bagging 中各基学习器之间无信息传递（并行独立），元特征只是等权多数表决；Stacking 的元学习器可以学习"何时相信哪个基学习器"——例如，SVM 可能在线性可分区域更可靠，而随机森林在高度非线性区域更准确。这种"可学习的异质权重"是 Stacking 通常优于简单投票的根本原因。请思考：什么场景下 Stacking 相对 Bagging 的增益最大？
{% endhint %}

## 3.6.7 案例：随机森林手写数字识别与全章算法对比

{% block type="case" label="3.6" title="随机森林特征重要性分析与全章六类算法综合对比" %}
**数据集**：`sklearn.datasets.load_digits`，$n=1797$，$d=64$（$8 \times 8$ 像素灰度值），$K=10$（数字 0–9）。按 75%/25% 分层划分，特征使用 `StandardScaler` 标准化。

**随机森林基准实验**：

```python
import numpy as np
from sklearn.datasets import load_digits
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.preprocessing import StandardScaler
from sklearn.tree import DecisionTreeClassifier
from sklearn.ensemble import (RandomForestClassifier, BaggingClassifier,
                               AdaBoostClassifier, GradientBoostingClassifier)
from sklearn.neighbors import KNeighborsClassifier
from sklearn.svm import SVC
from sklearn.neural_network import MLPClassifier
from sklearn.metrics import accuracy_score
# --- 1. 数据准备 ---
data = load_digits()
X, y = data.data, data.target
X_tr, X_te, y_tr, y_te = train_test_split(
    X, y, test_size=0.25, random_state=42, stratify=y)
scaler = StandardScaler()
X_tr_s = scaler.fit_transform(X_tr)
X_te_s  = scaler.transform(X_te)
# --- 2. 随机森林：OOB 准确率 + 特征重要性 ---
rf_best = RandomForestClassifier(
    n_estimators=200, oob_score=True, random_state=42, n_jobs=-1)
rf_best.fit(X_tr_s, y_tr)
print(f"RF OOB 准确率: {rf_best.oob_score_:.4f}")
print(f"RF 测试准确率: {accuracy_score(y_te, rf_best.predict(X_te_s)):.4f}")
importances = rf_best.feature_importances_.reshape(8, 8)  # 重塑为 8×8 热图
# --- 3. 全章算法 5 折 CV 准确率对比 ---
classifiers = {
    "决策树 (depth=None)": DecisionTreeClassifier(max_depth=None, random_state=42),
    "KNN (k=3)":           KNeighborsClassifier(n_neighbors=3, n_jobs=-1),
    "SVM (RBF)":           SVC(kernel="rbf", C=10, gamma="scale", random_state=42),
    "MLP (128-64)":        MLPClassifier(hidden_layer_sizes=(128, 64),
                                          activation="relu", solver="adam",
                                          max_iter=500, random_state=42),
    "随机森林 (T=200)":     RandomForestClassifier(n_estimators=200, random_state=42, n_jobs=-1),
    "AdaBoost (T=100)":    AdaBoostClassifier(n_estimators=100,
                                              algorithm="SAMME", random_state=42),
    "GBDT (T=100)":        GradientBoostingClassifier(n_estimators=100,
                                                       max_depth=3, learning_rate=0.1,
                                                       random_state=42),
}
results = {}
for name, clf in classifiers.items():
    cv_scores = cross_val_score(clf, X_tr_s, y_tr, cv=5,
                                scoring="accuracy", n_jobs=-1)
    results[name] = (cv_scores.mean(), cv_scores.std())
    print(f"{name:<25} CV 准确率: {cv_scores.mean():.4f} ± {cv_scores.std():.4f}")
```

| 算法 | 5 折 CV 准确率 | 测试准确率 |
|---|---|---|
| 决策树（无限深度） | 约 0.84 | 约 0.85 |
| KNN（$k=3$） | 约 0.97 | 约 0.97 |
| SVM（RBF 核） | **约 0.99** | **约 0.99** |
| MLP（128-64） | 约 0.98 | 约 0.98 |
| 随机森林（$T=200$） | 约 0.98 | 约 0.98 |
| AdaBoost（$T=100$） | 约 0.96 | 约 0.96 |
| GBDT（$T=100$） | 约 0.98 | 约 0.98 |

**关键观察**：

1. **随机森林的特征重要性**（图 3.6.4(a)，$8\times8$ 热图）呈现出符合视觉直觉的空间结构——中心区域像素的重要性显著高于边缘像素，这与笔画信息集中于字形中部的认知一致；

🚀 **前沿视野：梯度提升的现代演进（2022–2025）**

梯度提升树在结构化（表格）数据上的统治地位面临来自深度学习的挑战，但尚未被撼动：

- **TabNet**（Arik & Pfister, 2021）：基于注意力机制的表格数据神经网络，集成了特征选择与解释性，但在多数 Kaggle 竞赛中仍逊于 GBDT；
- **FT-Transformer / TabPFN**（Hollmann et al., 2022）：通过在大量合成数据集上元学习，实现了无需超参数调优的"即插即用"分类，在小数据集（$n < 1000$）上有时超越 XGBoost；
- **AutoGluon TabularPredictor**（Erickson et al., 2020）：自动化集成 XGBoost、LightGBM、CatBoost、神经网络，通过 Stacking 组合多层模型，在 70 个 OpenML 数据集的元评测中，整体排名高于任何单一算法；
- **GBDT + LLM**（2024–2025）：在结构化数据的特征工程阶段引入大语言模型（生成交叉特征描述、类别特征嵌入），再以 GBDT 建模，在部分金融风控场景中取得新的基准。

**结论**：对于表格数据，GBDT 仍是最可靠的基线；对于小数据集，TabPFN 值得尝试；对于追求最优性能的竞赛场景，多类 GBDT + 神经网络的 Stacking 是当前最优实践。掌握本节的理论基础，是理解所有这些演进方向的共同前提。

## 本节小结

本节从集成学习的两个核心理论框架出发，系统建立了 Bagging 与 Boosting 的数学基础：

1. **偏差-方差分解**（定理 3.14，公式 3.69–3.70）：均方误差 = Bias² + Variance + 噪声。集成 $T$ 个相关系数为 $\rho$ 的基学习器，方差下界为 $\rho\sigma_b^2$，因此降低基学习器间的相关性（随机化）比单纯增加数量更关键；

2. **Bagging 与随机森林**（定义 3.30–3.31，算法 3.7）：通过 Bootstrap 重采样并行训练多棵不剪枝决策树，以特征随机子采样（$m_\text{try}=\sqrt{d}$）进一步去相关；OOB 误差（公式 3.71）提供了无需额外数据划分的泛化估计；特征重要性（公式 3.72）给出可解释的特征贡献排名；

3. **AdaBoost**（定义 3.32，算法 3.8，定理 3.15）：从指数损失最小化推导出自适应样本权重更新（公式 3.75–3.76），集成误差以 $e^{-2\gamma^2 T}$ 速率指数下降（公式 3.77），但对噪声敏感；

4. **梯度提升树 / XGBoost**（定义 3.33）：将 Boosting 推广至任意可微损失函数的函数空间梯度下降（公式 3.78–3.79），XGBoost 以二阶 Taylor 展开、稀疏感知与列采样实现了工业级性能；

5. **Stacking**（定义 3.34）：通过 OOF 预测将异质基学习器的输出作为元特征，元学习器学习最优组合权重；数据泄露防护是正确实现的关键。

全章六类算法对比（图 3.6.4(c)）表明：在中等维度表格任务上，集成方法（随机森林、GBDT）与核方法（SVM）和深度模型（MLP）共同构成第一梯队，彼此差异通常在 1%–2% 以内；单棵决策树由于高方差，通常不宜直接部署。

下一节（§3.7）将建立统一的分类算法评估体系——混淆矩阵、ROC/AUC、精确率-召回率曲线与 $k$ 折交叉验证——为本章所有算法的横向比较提供严格的评估工具。

**A5.** 下列说法哪些正确？请逐条判断：（a）随机森林中 $m_{\text{try}}$ 越大，基学习器之间的相关性越低；（b）AdaBoost 的 $\alpha_t$ 当 $\varepsilon_t \to 0$ 时趋近于正无穷；（c）GBDT 对均方损失的伪残差就是普通残差 $y_i - F_{t-1}(\mathbf{x}_i)$；（d）Stacking 的元学习器必须使用逻辑回归；（e）随机森林无法进行回归任务。

### B 级（数学推导）

**B1.** 证明定理 3.13：$T$ 个独立分类器（每个错误率为 $\varepsilon < 0.5$）多数表决的集成误差满足公式 (3.68)。（提示：令 $Z \sim \text{Binomial}(T,\varepsilon)$，使用 Hoeffding 不等式 $\Pr[Z \geq \mu + t] \leq e^{-2t^2/T}$，令 $\mu = T\varepsilon$，$t = T/2 - T\varepsilon$。）

**B2.** 推导公式 (3.70)：$T$ 个等方差 $\sigma_b^2$、两两相关系数为 $\rho$ 的随机变量的平均方差。（提示：$\text{Var}(\sum_t X_t / T) = \frac{1}{T^2}\sum_{s,t}\text{Cov}(X_s,X_t)$，将对角项与非对角项分开计算。）

**B3.** 从定义 3.33 出发，对均方损失 $\ell(y,F) = (y-F)^2/2$ 推导伪残差 $r_{ti} = y_i - F_{t-1}(\mathbf{x}_i)$；对对数损失（逻辑回归）$\ell(y,F) = \log(1+e^{-yF})$（$y\in\{-1,+1\}$），推导对应的伪残差表达式，并说明其与 sigmoid 输出的关系。

**B4.** 证明 AdaBoost 的训练误差上界（定理 3.15 的核心步骤）：证明经过 $T$ 轮后，$\frac{1}{n}\sum_i e^{-y_i F(\mathbf{x}_i)} = \prod_{t=1}^T 2\sqrt{\varepsilon_t(1-\varepsilon_t)}$。（提示：利用每轮权重归一化的条件，递推展开。）
