# 13.1 零售领域购物篮分析

**购物篮分析**（Market Basket Analysis）是数据挖掘最经典的商业应用之一——通过分析顾客的交易记录，发现商品之间的关联关系（"买了 A 的顾客也倾向于买 B"），为**交叉销售**（Cross-Selling）、**捆绑促销**和**货架布局优化**提供数据驱动的决策依据。

§6 系统地介绍了关联规则挖掘的理论基础——Apriori 算法（§6.2）、FP-Growth 算法（§6.3）、兴趣度量（§6.4）。本节将这些方法应用于真实的零售交易数据，按照标准数据挖掘项目流程——**业务理解 → 数据准备 → 探索性分析 → 建模 → 可视化 → 业务解读**——展示从原始数据到可行动洞察的完整过程。

## 13.1.1 业务背景与数据集

### 问题定义

一家英国在线零售商记录了 2010–2011 年间约 54 万笔交易。公司希望回答以下业务问题：

1. **哪些商品经常被一起购买？** —— 用于在网站上设置"经常一起买"推荐；
2. **哪些关联关系具有强预测性？** —— 即买了 A 的顾客，有多大概率同时购买 B；
3. **如何利用这些信息优化促销策略？** —— 捆绑定价、交叉推荐。

这正是 §6.1 定义的关联规则挖掘问题：从事务数据库中发现满足最小支持度和最小置信度的关联规则 $$X \Rightarrow Y$$。

### 数据集

**UCI Online Retail** 数据集（Daqing Chen et al., 2012）包含以下字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| InvoiceNo | 发票编号（交易 ID） | 536365 |
| StockCode | 商品编码 | 85123A |
| Description | 商品描述 | WHITE HANGING HEART T-LIGHT HOLDER |
| Quantity | 购买数量 | 6 |
| InvoiceDate | 交易时间 | 2010-12-01 08:26:00 |
| UnitPrice | 单价（英镑） | 2.55 |
| CustomerID | 客户 ID | 17850 |
| Country | 国家 | United Kingdom |

原始数据共 541,909 行，每行对应一笔交易中的一件商品。需要将其转换为**事务格式**——每行代表一笔完整交易，列代表商品是否出现。

## 13.1.2 完整案例：从原始数据到商业洞察

{% block type="case" %}
**案例 13.1**（零售购物篮关联规则挖掘全流程）

**背景**：使用 UCI Online Retail 数据集，挖掘英国市场的商品关联规则，为交叉销售提供数据支撑。

**工具**：pandas（数据操作）、mlxtend（Apriori + 关联规则）、matplotlib（可视化）。

**第一步：数据加载与清洗。** 原始数据中包含退货记录（InvoiceNo 以"C"开头）、缺失的商品描述以及数量为负的异常记录，需要逐一过滤。此外，由于不同国家的消费习惯差异较大，我们聚焦于交易量最大的英国市场。

```python
import pandas as pd
import numpy as np
import matplotlib.pyplot as plt
from matplotlib.colors import Normalize, TwoSlopeNorm
from mlxtend.frequent_patterns import apriori, association_rules
from mlxtend.preprocessing import TransactionEncoder
plt.rcParams["font.sans-serif"] = ["SimHei", "DejaVu Sans"]
plt.rcParams["axes.unicode_minus"] = False
BLUE, RED, GREEN = "#2563eb", "#dc2626", "#16a34a"
ORANGE, PURPLE, GRAY = "#ea580c", "#9333ea", "#64748b"
PALETTE = [BLUE, RED, GREEN, ORANGE, PURPLE, "#0d9488", GRAY]
# --- 1. 数据加载 ---
url = ("https://archive.ics.uci.edu/ml/machine-learning-databases"
       "/00352/Online%20Retail.xlsx")
df = pd.read_excel(url)
print(f"原始数据: {df.shape[0]} 行, {df['InvoiceNo'].nunique()} 笔交易")
# --- 2. 数据清洗 ---
steps = [("原始数据", len(df))]
df = df.dropna(subset=["InvoiceNo", "Description"])
steps.append(("移除缺失值", len(df)))
df = df[~df["InvoiceNo"].astype(str).str.startswith("C")]
steps.append(("移除退货 (C-)", len(df)))
df = df[df["Quantity"] > 0]
steps.append(("移除异常数量", len(df)))
df = df[df["Country"] == "United Kingdom"]
steps.append(("筛选英国市场", len(df)))
print(f"清洗后: {df.shape[0]} 行, {df['InvoiceNo'].nunique()} 笔交易")
print(f"保留比例: {len(df)/steps[0][1]:.1%}")
# 典型结果:
# 原始数据: 541909 行, 25900 笔交易
# 清洗后: 485694 行, 18194 笔交易
# 保留比例: 89.6%
```

ax2.set_xlabel("出现次数")
ax2.set_title("(b) Top-15 高频商品（英国市场）", fontweight="bold")

**第三步：构建事务矩阵与 Apriori 频繁项集挖掘。** 将原始的"每行一件商品"格式按 InvoiceNo 分组为事务列表，再通过 One-Hot 编码转换为事务矩阵（§6.2）。随后使用 Apriori 算法挖掘频繁项集，设定最小支持度为 2%（即至少出现在 364 笔交易中）。最后从频繁项集生成关联规则，按提升度排序。

```python
# --- 4. 构建事务矩阵 ---
transactions = (df.groupby("InvoiceNo")["Description"]
                .apply(list).tolist())
print(f"事务数: {len(transactions)}, 平均商品数: "
      f"{np.mean([len(t) for t in transactions]):.1f}")
te = TransactionEncoder()
te_array = te.fit_transform(transactions)
basket = pd.DataFrame(te_array, columns=te.columns_)
print(f"事务矩阵: {basket.shape}")
# --- 5. Apriori 频繁项集挖掘 ---
freq_items = apriori(basket, min_support=0.02, use_colnames=True)
freq_items["length"] = freq_items["itemsets"].apply(len)
print(f"\n频繁项集数: {len(freq_items)}")
print(f"  1-项集: {(freq_items['length']==1).sum()}")
print(f"  2-项集: {(freq_items['length']==2).sum()}")
print(f"  3-项集: {(freq_items['length']==3).sum()}")
# --- 6. 关联规则生成与筛选 ---
rules = association_rules(freq_items, metric="lift",
                          min_threshold=1.5,
                          num_itemsets=len(freq_items))
rules = rules.sort_values("lift", ascending=False)
print(f"\n关联规则数 (lift≥1.5): {len(rules)}")
print(f"\nTop-10 规则 (按提升度):")
for _, r in rules.head(10).iterrows():
    ant = ", ".join(sorted(r["antecedents"]))
    con = ", ".join(sorted(r["consequents"]))
    print(f"  {ant:45s} → {con:45s}  "
          f"sup={r['support']:.3f} conf={r['confidence']:.2f} "
          f"lift={r['lift']:.2f}")
high_conf = rules[(rules["confidence"] >= 0.6) & (rules["lift"] >= 2.0)]
print(f"\n高置信度规则 (conf≥0.6, lift≥2.0): {len(high_conf)} 条")
for _, r in high_conf.head(5).iterrows():
    ant = ", ".join(sorted(r["antecedents"]))
    con = ", ".join(sorted(r["consequents"]))
    print(f"  {ant} → {con}")
    print(f"    置信度={r['confidence']:.2f}, 提升度={r['lift']:.2f}")
# 典型结果:
# 事务矩阵: (18194, 4058)
# 频繁项集数: 392 (296 个 1-项集, 92 个 2-项集, 4 个 3-项集)
# 关联规则数 (lift≥1.5): 208
# 最高提升度: WOODEN STAR → WOODEN HEART (lift=27.46)
# 高置信度规则 (conf≥0.6, lift≥2.0): 35 条
```

**第四步：关联规则可视化——支持度-置信度散点图。** 使用散点图展示所有规则在支持度-置信度空间中的分布，点的颜色和大小映射提升度。标注支持度和置信度阈值线，定位同时满足高支持、高置信、高提升的"黄金规则"（§6.4）。

```python
# --- 7. 支持度-置信度-提升度散点图 ---
fig, ax = plt.subplots(figsize=(12, 8))
sups = rules["support"].values
confs = rules["confidence"].values
lifts = rules["lift"].values
norm = Normalize(vmin=1.0, vmax=min(lifts.max(), 8.0))
sizes = np.clip(lifts * 30, 20, 300)
sc = ax.scatter(sups, confs, c=lifts, s=sizes, cmap="RdYlBu_r",
                norm=norm, alpha=0.7, edgecolors="white", linewidths=0.5)
cbar = plt.colorbar(sc, ax=ax, shrink=0.8)
cbar.set_label("提升度 (Lift)", fontsize=12)
ax.axvline(x=0.03, color=BLUE, linestyle="--", linewidth=1.5,
           alpha=0.6, label="min_support = 3%")
ax.axhline(y=0.6, color=GREEN, linestyle="--", linewidth=1.5,
           alpha=0.6, label="min_confidence = 60%")
golden = rules[(rules["support"] >= 0.03) &
               (rules["confidence"] >= 0.6) &
               (rules["lift"] >= 2.0)]
if len(golden) > 0:
    ax.fill_between([0.03, sups.max() * 1.05], 0.6, 1.05,
                    alpha=0.08, color=ORANGE,
                    label=f"黄金区域 ({len(golden)} 条规则)")
top5 = rules.nlargest(5, "lift")
for idx, (_, r) in enumerate(top5.iterrows()):
    ant = ", ".join(sorted(r["antecedents"]))[:25]
    con = ", ".join(sorted(r["consequents"]))[:25]
    ax.annotate(
        f"{ant}→\n{con}\nlift={r['lift']:.1f}",
        xy=(r["support"], r["confidence"]), fontsize=7.5,
        xytext=(10, 12 + (idx % 3) * 8),
        textcoords="offset points",
        arrowprops=dict(arrowstyle="->", color=RED, lw=0.8),
        bbox=dict(boxstyle="round,pad=0.3", facecolor="lightyellow",
                  edgecolor="gray", alpha=0.9))
ax.set_xlabel("支持度 (Support)")
ax.set_ylabel("置信度 (Confidence)")
ax.set_title("关联规则的支持度-置信度-提升度分布")
ax.legend(fontsize=10, loc="lower right")
ax.set_xlim(-0.005, sups.max() * 1.1)
ax.set_ylim(-0.02, 1.05)
plt.tight_layout()
plt.show()
```

**第五步：商品关联热力图。** 取出现频率最高的 12 种商品，计算两两之间的提升度并绘制热力图。颜色以 lift = 1（独立）为中心，蓝色表示负相关、红色表示正相关，可以直观定位强关联商品对。

```python
# --- 8. 高频商品提升度热力图 ---
top_n = 12
top_items = df["Description"].value_counts().head(top_n).index.tolist()
df_top = df[df["Description"].isin(top_items)]
invoice_items = (df_top.groupby("InvoiceNo")["Description"]
                 .apply(set).reset_index())
n_trans = df["InvoiceNo"].nunique()
item_support = {}
for item in top_items:
    count = invoice_items["Description"].apply(
        lambda s: item in s).sum()
    item_support[item] = count / n_trans
lift_matrix = pd.DataFrame(np.ones((top_n, top_n)),
                           index=top_items, columns=top_items)
for i, item_a in enumerate(top_items):
    for j, item_b in enumerate(top_items):
        if i == j:
            lift_matrix.iloc[i, j] = np.nan
            continue
        co_count = invoice_items["Description"].apply(
            lambda s: item_a in s and item_b in s).sum()
        co_sup = co_count / n_trans
        expected = item_support[item_a] * item_support[item_b]
        lift_matrix.iloc[i, j] = co_sup / expected if expected > 0 else 0
fig, ax = plt.subplots(figsize=(12, 10))
short_names = [d[:28] + "…" if len(d) > 28 else d for d in top_items]
vmin = max(0, np.nanmin(lift_matrix.values) - 0.1)
vmax = np.nanmax(lift_matrix.values)
norm = TwoSlopeNorm(vmin=vmin, vcenter=1.0, vmax=min(vmax, 6.0))
im = ax.imshow(lift_matrix.values, cmap="RdYlBu_r",
               norm=norm, aspect="auto")
cbar = plt.colorbar(im, ax=ax, shrink=0.8, pad=0.02)
cbar.set_label("提升度 (Lift)", fontsize=12)
for i in range(top_n):
    for j in range(top_n):
        val = lift_matrix.iloc[i, j]
        if np.isnan(val):
            continue
        color = "white" if val > 3.0 or val < 0.5 else "black"
        weight = "bold" if val > 2.0 else "normal"
        ax.text(j, i, f"{val:.1f}", ha="center", va="center",
                fontsize=8, color=color, fontweight=weight)
ax.set_xticks(range(top_n))
ax.set_yticks(range(top_n))
ax.set_xticklabels(short_names, rotation=45, ha="right", fontsize=9)
ax.set_yticklabels(short_names, fontsize=9)
ax.set_title("Top-12 高频商品关联提升度热力图", fontsize=14, pad=15)
ax.set_xticks(np.arange(-0.5, top_n, 1), minor=True)
ax.set_yticks(np.arange(-0.5, top_n, 1), minor=True)
ax.grid(which="minor", color="white", linewidth=1.5)
ax.grid(which="major", visible=False)
plt.tight_layout()
plt.show()
# 典型结果:
# 提升度范围: 0.90 – 6.52
# 最强关联: LUNCH BAG RED × LUNCH BAG BLACK SKULL (lift=6.52)
#           JUMBO BAG RED × JUMBO BAG PINK (lift=6.36)
```

## 13.1.3 业务解读与实践要点

### 从规则到行动

挖掘出关联规则后，数据挖掘团队需要与业务团队协作，将规则转化为可落地的商业行动。以下是一个典型的规则评估框架：

| 评估维度 | 问题 | 示例 |
|---------|------|------|
| **统计显著性** | 规则的支持度是否足够高？ | support > 2% 确保规则基于足够多的交易 |
| **预测强度** | 置信度和提升度是否令人信服？ | confidence > 50% 且 lift > 2 |
| **可行动性** | 规则是否可以转化为具体的业务动作？ | "星星装饰 → 心形装饰"可设置交叉推荐 |
| **非平凡性** | 规则是否揭示了非显而易见的关联？ | "牙刷 → 牙膏"是显而易见的，价值有限 |
| **盈利性** | 基于规则的行动是否能带来可衡量的收入增长？ | 交叉推荐的转化率提升 |

{% hint style="warning" %}
⚠️ **关联不等于因果**

购物篮分析发现的是商品之间的**共现关联**，而非因果关系。"买了 A 的顾客也买了 B"并不意味着"A 导致了 B 的购买"——两者可能都是由一个潜在因素驱动的（如季节性促销、节日礼品需求）。在制定业务策略时，应将关联规则作为**假设生成器**，通过 A/B 测试验证其商业效果。
{% endhint %}

### 大规模数据的工程考量

UCI Online Retail 数据集仅有约 54 万行，Apriori 可以在秒级完成。但在真实的零售场景中，数据规模可能达到数亿级——此时需要注意：

1. **FP-Growth 替代 Apriori**：当频繁项集数量庞大时，FP-Growth（§6.3）避免了候选项集生成，内存和时间效率都显著优于 Apriori；
2. **时间窗口**：分析通常限定在特定时间窗口（如近 3 个月），既减少数据量又确保规则的时效性；
3. **商品层次**：原始的 SKU 级别（如"红色 S 码 T 恤"）可能过于精细导致支持度过低。通过商品类目层次聚合（如"T 恤" → "上装" → "服装"），可以在不同粒度上发现规则（§6.5）；
4. **在线更新**：当新交易持续到来时，可以使用增量式频繁项集更新算法，避免每次全量重算。

### 支持度阈值的选择

本案例使用了 2% 的最小支持度。这一参数对结果影响极大——阈值降低会导致频繁项集数量指数增长。在 4,058 种商品的数据集中，2% 意味着至少 364 笔交易包含该项集，这是一个合理的平衡点。实际项目中建议从较高阈值（如 5%）开始，逐步降低并观察规则质量的变化，直到找到业务价值与计算成本的最佳平衡。
