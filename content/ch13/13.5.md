# 13.5 推荐系统实现

§6 介绍的**关联规则**能从购买记录中发现"啤酒 → 尿布"这类群体层面的购买规律，但它无法为每个用户生成个性化推荐——同一条规则对所有用户都一样。**推荐系统**的核心目标是为每个用户预测其对未消费内容的偏好，从而实现"千人千面"的个性化推荐。

推荐算法的核心思想是**协同过滤**（Collaborative Filtering）：如果用户 A 和用户 B 过去喜欢了很多相同的电影，那么 A 喜欢但 B 没看过的电影，很可能也是 B 会喜欢的。这一思想的数学实现依赖**用户-物品评分矩阵**——一个巨大而极度稀疏的矩阵，推荐问题本质上是矩阵补全问题。

## 13.5.1 业务背景与数据集

### 问题定义

推荐系统的核心任务是**评分预测**和**物品排序**：

1. **评分预测**：给定用户 $$u$$ 和物品 $$i$$，预测用户对该物品的评分 $$\hat{r}_{ui}$$；
2. **Top-K 推荐**：对用户 $$u$$，从所有未消费物品中选出预测评分最高的 K 个推荐。

推荐算法大致分为两类：

| 类别 | 方法 | 数据需求 | 优势 | 劣势 |
|------|------|---------|------|------|
| **基于内容** | 分析物品属性（类型、导演等） | 物品元数据 | 不依赖他人行为 | 推荐同质化 |
| **协同过滤** | 利用用户行为的集体智慧 | 用户-物品交互 | 能发现意外惊喜 | 冷启动问题 |

本节聚焦**协同过滤**方法，包括基于记忆（Memory-based）的 KNN 方法和基于模型（Model-based）的矩阵分解方法。

### 数据集

**MovieLens 100K**（Harper & Konstan, 2015）是推荐系统研究中最常用的基准数据集：

| 属性 | 值 |
|------|-----|
| 评分数 | 100,000 |
| 用户数 | 943 |
| 电影数 | 1,682 |
| 评分范围 | 1–5（整数） |
| 稀疏度 | 93.7%（每用户平均仅评价了 6.3% 的电影） |
| 平均评分 | 3.53 |
| 每用户评分数 | 20–737（平均 106） |
| 数据来源 | GroupLens（`files.grouplens.org`） |

{% hint style="tip" %}
💡 **稀疏性是核心挑战**

943 个用户 × 1,682 部电影 = 1,586,126 个可能的评分，但实际只有 100,000 个（6.3%）。推荐系统需要从这 6.3% 的已知评分中推断剩余 93.7% 的缺失值——这本质上是一个大规模的矩阵补全问题。
{% endhint %}

**案例 13.5**（基于协同过滤与矩阵分解的电影推荐系统）

```python
    url = "https://files.grouplens.org/datasets/movielens/ml-100k.zip"
```

**第四步：协同过滤与矩阵分解。** 实现四种推荐方法并评估预测精度。用户协同过滤和物品协同过滤均使用调整后的余弦相似度（Adjusted Cosine Similarity），NMF 使用 scikit-learn 实现，缺失值填充为用户均值。

```python
# --- 5. 协同过滤与矩阵分解 ---
# 训练/测试划分（80/20, 按用户分层）
rng = np.random.RandomState(42)
ratings_0 = ratings.copy()
ratings_0["user_id"] -= 1
ratings_0["item_id"] -= 1
train_idx, test_idx = [], []
for uid, group in ratings_0.groupby("user_id"):
    indices = group.index.tolist()
    rng.shuffle(indices)
    split = max(1, int(len(indices) * 0.2))
    test_idx.extend(indices[:split])
    train_idx.extend(indices[split:])
train_df = ratings_0.loc[train_idx].reset_index(drop=True)
test_df = ratings_0.loc[test_idx].reset_index(drop=True)
# 构建评分矩阵
R = np.zeros((n_users, n_items))
for _, row in train_df.iterrows():
    R[row["user_id"], row["item_id"]] = row["rating"]
user_means = np.array([R[u][R[u]>0].mean() if (R[u]>0).any()
                        else 3.0 for u in range(n_users)])
# 方法 1: User Mean Baseline
def predict_user_mean(test_df):
    return np.array([user_means[r["user_id"]]
                     for _, r in test_df.iterrows()])
# 方法 2: User-based CF (k=30)
R_cent = R.copy()
for u in range(n_users):
    m = R[u] > 0
    R_cent[u][m] -= user_means[u]
    R_cent[u][~m] = 0
user_sim = cosine_similarity(R_cent)
np.fill_diagonal(user_sim, 0)
def predict_user_cf(test_df, k=30):
    preds = []
    for _, row in test_df.iterrows():
        u, i = row["user_id"], row["item_id"]
        rated_users = np.where(R[:, i] > 0)[0]
        if len(rated_users) == 0:
            preds.append(user_means[u]); continue
        sims = user_sim[u, rated_users]
        top_k = np.argsort(np.abs(sims))[-k:]
        s = sims[top_k]
        denom = np.abs(s).sum()
        if denom == 0:
            preds.append(user_means[u])
        else:
            w = (s * (R[rated_users[top_k], i]
                      - user_means[rated_users[top_k]])).sum()
            preds.append(user_means[u] + w / denom)
    return np.clip(preds, 1, 5)
# 方法 3: Item-based CF (k=30, 调整余弦)
item_sim = cosine_similarity(R_cent.T)
np.fill_diagonal(item_sim, 0)
def predict_item_cf(test_df, k=30):
    preds = []
    for _, row in test_df.iterrows():
        u, i = row["user_id"], row["item_id"]
        rated_items = np.where(R[u] > 0)[0]
        if len(rated_items) == 0:
            preds.append(user_means[u]); continue
        sims = item_sim[i, rated_items]
        pos = sims > 0
        if pos.sum() == 0:
            preds.append(user_means[u]); continue
        pi, ps = rated_items[pos], sims[pos]
        if len(ps) > k:
            tk = np.argsort(ps)[-k:]
            pi, ps = pi[tk], ps[tk]
        preds.append((ps * R[u, pi]).sum() / ps.sum())
    return np.clip(preds, 1, 5)
# 方法 4: NMF (k=20)
R_filled = R.copy()
for u in range(n_users):
    R_filled[u][R[u]==0] = user_means[u]
nmf = NMF(n_components=20, init="nndsvda", random_state=42,
          max_iter=500)
W = nmf.fit_transform(R_filled)
H = nmf.components_
R_nmf = np.clip(W @ H, 1, 5)
def predict_nmf(test_df):
    return np.array([R_nmf[r["user_id"], r["item_id"]]
                     for _, r in test_df.iterrows()])
# --- 6. 模型评估 ---
actuals = test_df["rating"].values
methods = {
    "User Mean": predict_user_mean(test_df),
    "User-based CF": predict_user_cf(test_df),
    "Item-based CF": predict_item_cf(test_df),
    "NMF (k=20)": predict_nmf(test_df),
}
print(f"{'方法':<18s} {'RMSE':>8s} {'MAE':>8s}")
results = {}
for name, preds in methods.items():
    rmse = np.sqrt(np.mean((actuals - preds)**2))
    mae = np.mean(np.abs(actuals - preds))
    results[name] = {"RMSE": rmse, "MAE": mae}
    print(f"  {name:<16s} {rmse:>8.4f} {mae:>8.4f}")
# 绘图
fig, ax = plt.subplots(figsize=(12, 6))
names = list(results.keys())
x = np.arange(len(names))
width = 0.3
for i, metric in enumerate(["RMSE", "MAE"]):
    vals = [results[n][metric] for n in names]
    offset = (i - 0.5) * width
    bars = ax.bar(x + offset, vals, width, label=metric,
                  color=[BLUE, ORANGE][i], alpha=0.85,
                  edgecolor="white")
    for bar, val in zip(bars, vals):
        ax.text(bar.get_x() + bar.get_width()/2,
                bar.get_height() + 0.01,
                f"{val:.3f}", ha="center", fontsize=9)
ax.set_xticks(x)
ax.set_xticklabels(names, fontsize=11)
ax.set_ylabel("误差")
ax.set_title("四种推荐方法性能对比", fontweight="bold")
ax.legend(fontsize=11)
plt.tight_layout()
plt.show()
# 典型结果:
# User Mean:      RMSE=1.042, MAE=0.835
# User-based CF:  RMSE=0.930, MAE=0.727
# Item-based CF:  RMSE=0.927, MAE=0.725
# NMF (k=20):     RMSE=0.988, MAE=0.782
```

**第五步：NMF 隐因子调优与推荐生成。** 图 13.5.4 展示了隐因子维度 k 对 NMF 性能的影响，以及为特定用户生成的 Top-10 推荐列表。

```python
# --- 7. 隐因子维度调优 ---
ks = [5, 10, 15, 20, 30, 50, 75, 100]
rmse_list, mae_list = [], []
for k in ks:
    model = NMF(n_components=k, init="nndsvda",
                random_state=42, max_iter=500)
    W_k = model.fit_transform(R_filled)
    H_k = model.components_
    R_pred = np.clip(W_k @ H_k, 1, 5)
    preds = np.array([R_pred[r["user_id"], r["item_id"]]
                      for _, r in test_df.iterrows()])
    rmse_list.append(np.sqrt(np.mean((actuals - preds)**2)))
    mae_list.append(np.mean(np.abs(actuals - preds)))
    print(f"  k={k:>3d}  RMSE={rmse_list[-1]:.4f}  MAE={mae_list[-1]:.4f}")
best_k = ks[np.argmin(rmse_list)]
print(f"最佳 k = {best_k}")
# --- 8. 推荐示例 ---
# 使用最佳 k 生成推荐
model = NMF(n_components=best_k, init="nndsvda",
            random_state=42, max_iter=500)
W = model.fit_transform(R_filled)
R_pred = np.clip(W @ model.components_, 1, 5)
demo_user = 0  # 选取一个活跃用户
rated = set(train_df[train_df["user_id"]==demo_user]["item_id"])
scores = R_pred[demo_user].copy()
for i in rated:
    scores[i] = -1
top10 = np.argsort(scores)[-10:][::-1]
print(f"\n用户 {demo_user} Top-10 推荐:")
for rank, iid in enumerate(top10, 1):
    title = movies[movies["item_id"]==iid+1]["title"].values
    title = title[0] if len(title) else f"Movie {iid+1}"
    print(f"  {rank:>2d}. {title[:40]:<42s} 预测={scores[iid]:.2f}")
# 绘图
fig, (ax1, ax2) = plt.subplots(1, 2, figsize=(15, 6.5),
```

## 13.5.3 业务解读与实践要点

### 协同过滤的两种思路

案例 13.5 实现了两种协同过滤的"对偶"视角：

| 维度 | User-based CF | Item-based CF |
|------|--------------|---------------|
| 核心思想 | "人以群分"——找相似用户 | "物以类聚"——找相似物品 |
| 相似度矩阵 | $$943 \times 943$$（用户×用户） | $$1682 \times 1682$$（物品×物品） |
| 更新频率 | 新用户 → 重算整行 | 新物品 → 重算整列 |
| 适用场景 | 用户少、物品多 | 用户多、物品少（如电商） |

在 Amazon 等大型电商平台上，用户数远大于商品数，且物品之间的相似性比用户之间的更稳定——因此 **Item-based CF 是工业界的主流选择**。

### 冷启动问题

协同过滤的致命弱点是**冷启动**——新用户和新物品没有历史评分，相似度无法计算：

| 冷启动类型 | 问题 | 常见解决方案 |
|-----------|------|------------|
| **新用户** | 无历史评分，无法计算相似度 | 基于内容的推荐 + 热门物品补充 |
| **新物品** | 无人评价，不会被推荐 | 基于物品属性的内容匹配 |
| **系统冷启动** | 评分数据极少 | 人工策划 + 引导用户评分 |

{% hint style="warning" %}
⚠️ **信息茧房与推荐伦理**

协同过滤倾向于推荐与用户历史偏好相似的内容，可能导致**信息茧房**（Filter Bubble）——用户被困在自己的偏好回音室中，接触不到多样化内容。实际推荐系统需要在**相关性**和**多样性**之间取得平衡，例如在 Top-K 中混入一定比例的探索性推荐。
{% endhint %}

### 从 CF 到深度学习

案例中的 NMF 可以看作推荐系统深度化的起点：

| 阶段 | 方法 | 特点 |
|------|------|------|
| 经典 | KNN + CF | 简单直观，可解释性好 |
| 矩阵分解 | NMF / SVD | 学习隐因子，处理稀疏性 |
| 深度学习 | NCF / DeepFM | 隐因子 + 深层交互，容量更大 |
| 前沿 | Transformer | 序列推荐，捕捉时序偏好变化 |

## 本节小结

| 关键概念 | 说明 |
|---------|------|
| 用户-物品评分矩阵 | 推荐系统的核心数据结构，极度稀疏（本例 93.7%） |
| 协同过滤 | 利用群体行为推断个体偏好，分 User-based 和 Item-based |
| 调整余弦相似度 | 减去用户均值后计算余弦，消除评分标准差异 |
| 非负矩阵分解（NMF） | 将评分矩阵分解为用户因子和物品因子的乘积 |
| 冷启动问题 | 新用户/新物品无历史数据，是 CF 方法的根本限制 |

**D1.** 实现基于 SGD 的矩阵分解（Funk SVD）：直接在已知评分上优化 $$\min_{W,H} \sum_{(u,i) \in \Omega} (r_{ui} - \mathbf{w}_u \cdot \mathbf{h}_i)^2 + \lambda(\|\mathbf{w}_u\|^2 + \|\mathbf{h}_i\|^2)$$，对比其与 sklearn NMF 的性能差异。
