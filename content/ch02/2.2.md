# 2.2 数据集成与数据变换

2016年8月，滴滴出行宣布收购 Uber 中国业务，一夜之间拥有了两套完全独立的乘客数据库。两套系统在几乎每个技术细节上都存在冲突：用户 ID 格式不同（滴滴用整数，Uber 用 UUID 字符串）；地理坐标采用不同标准（中国大陆坐标系 GCJ-02 vs 国际标准 WGS-84，二者在同一城市可偏差数百米）；时间戳时区编码迥异；行程价格字段的命名、精度与货币单位均不一致。完成这两套系统的数据整合，耗费了大量工程资源，远超业界对一次数据迁移的常规预期。

这是**数据集成**（Data Integration）问题的典型写照：将来自异构数据源的数据合并为统一、一致、可分析的单一视图，其复杂程度远超简单的 JOIN 操作。而当数据完成集成之后，下一项挑战随之而来——原始特征的数值分布往往并不符合机器学习模型的隐含假设（对称性、方差齐性），必须经过**数据变换**（Data Transformation）处理，才能交付给后续的建模流程。本节系统介绍这两类技术：集成技术处理"来自哪里"和"如何对齐"的问题；变换技术处理"如何表示"和"如何分布"的问题。

{% hint style="info" %}
📋 **本节学习目标**

完成本节学习后，读者应能够：

1. 描述数据集成三阶段管道（Schema 匹配、实体对齐、数据融合），识别字段命名冲突、类型冲突与语义冲突，并给出相应的消解策略；
2. 运用代数论证，解释虚拟变量陷阱（Dummy Variable Trap）的成因，并说明在线性模型中应使用 $K-1$ 列独热编码的原因；
3. 对给定数据集，正确选择序数编码、独热编码或目标编码，并说明各方法的适用条件与潜在风险；
4. 推导 Box-Cox 变换的对数似然函数，并说明如何通过最大化 $\ell(\lambda)$ 确定最优变换参数 $\lambda$；
5. 比较等宽、等频与熵驱动离散化三种分箱策略，并针对给定业务场景做出选择。
{% endhint %}

**前置知识**：2.1节（数据清洗与重复数据消除）；线性代数基础（矩阵的秩、线性相关）；概率论（对数似然、最大似然估计）。

## 2.2.1 数据集成：从多源异构到统一视图

### 三类异构性

现实中的多源数据集成面对的异构性，可从三个层次加以分析。

**语法异构性**（Syntactic Heterogeneity）是最容易被发现也最容易被修复的一类：同一语义概念在不同系统中采用不同的数据格式——日期写为 `1985/03/15`、`15-Mar-1985`、`1985.03.15`；布尔值写为 `True/False`、`1/0`、`Y/N`；货币金额写为带符号的字符串（`¥8,500.00`）或纯整数（`850000`，单位分）。语法异构性本质上是**表面格式问题**，通过正则表达式替换或类型转换即可消解。

**结构异构性**（Structural Heterogeneity）更为隐蔽：相同语义的信息在不同系统中被拆分或合并到不同字段。例如，系统A将住址存储为单一字段 `"北京市朝阳区望京街道"`，系统B将其拆分为三个字段 `{城市, 区县, 街道}`；系统A将患者"诊断"记录为一条ICD编码，系统B允许多条诊断以列表形式存储（一对多关系）。解决结构异构性需要字段的拆分、合并或关系的规范化。

**语义异构性**（Semantic Heterogeneity）是最棘手的一类：相同的字段名在不同系统中表达不同的业务含义，或不同的字段名表达同一业务含义，且这种差异往往无法从数据本身观察到，必须依赖领域知识或数据字典。例如，系统A的 `score` 字段表示信用评分（越高越好），系统B的 `score` 字段表示风险评分（越低越好）；系统A的 `age` 字段存储的是录入时的年龄（静态值），系统B的 `birthday` 字段需要动态计算当前年龄。

### 集成管道的三个阶段

面对上述三类异构性，数据集成并非一次性操作，而是由三个顺序阶段构成的结构化管道：每个阶段解决一类特定冲突，本阶段的输出构成下一阶段的输入，最终产出统一 Schema 的集成数据集。

{% block type="definition" label="2.5" title="数据集成三阶段管道（Three-Stage Data Integration Pipeline）" %}
多源数据集成由以下三个顺序阶段构成（如图 2.2.1 所示）：

**阶段一：Schema 匹配**（Schema Matching）。给定源 Schema $S_1 = \{A_1^{(1)}, \ldots, A_{d_1}^{(1)}\}$ 与 $S_2 = \{A_1^{(2)}, \ldots, A_{d_2}^{(2)}\}$，构造映射函数 $M: S_1 \cup S_2 \to S_{target}$，使每个目标字段 $A_k^{target}$ 对应一个或多个源字段。冲突类型包括：命名冲突（不同名 $\to$ 同语义）、类型冲突（`INT` vs `FLOAT`）、粒度冲突（"千元" vs "元"）。

**阶段二：实体对齐**（Entity Alignment）。跨源记录中，指向同一真实实体的记录对集合为 $\mathcal{L} \subseteq \mathcal{D}_1 \times \mathcal{D}_2$，满足 $\text{sim}(r_i^{(1)}, r_j^{(2)}) \geq \delta$（2.1节排序邻域法提供了实现框架）。实体对齐的目标是以高查准率（Precision）和高查全率（Recall）识别 $\mathcal{L}$。

**阶段三：数据融合**（Data Fusion）。对于对齐后的记录对，若同一字段在两个来源中存在冲突值，需按**冲突消解策略**选取最终值：取最新值（时效优先）、取可信度更高来源的值（来源优先）、取多数投票值（多来源时）或将所有值保留为集合（保留所有）。最终输出统一 Schema 的集成数据集 $\mathcal{D}_{int}$。
{% endblock %}

### 冗余检测

集成后的数据集可能引入**冗余特征**——一个特征可由其他特征线性表示，或与另一特征高度相关。冗余特征不仅增加存储和计算开销，还可能在某些算法中引发数值不稳定。

{% block type="example" label="2.3" title="冗余字段的相关性诊断" %}
**场景**：集成员工数据库后，得到字段：`age`（年龄）、`birth_year`（出生年份）、`service_years`（工龄）、`hire_year`（入职年份）、`salary_monthly`（月薪）、`salary_annual`（年薪）。

**冗余诊断**：

| 冗余关系 | 检验方式 |
|---------|---------|
| `birth_year` ≈ `当前年份 - age` | 相关系数 $r \approx -1$；线性冗余 |
| `hire_year` ≈ `当前年份 - service_years` | 相关系数 $r \approx -1$；线性冗余 |
| `salary_annual` = `salary_monthly × 12` | 相关系数 $r = 1.0$；完全冗余 |

**处理方案**：保留语义更直接的字段（`age`, `service_years`, `salary_monthly`），删除可由其他字段精确计算的冗余字段。

数值冗余通过 Pearson 相关系数 $r = \frac{\text{Cov}(X,Y)}{\sigma_X \sigma_Y}$ 检验；类别冗余可用 $\chi^2$ 独立性检验（第5章）。
{% endblock %}

{% hint style="caution" %}
⚠️ **集成后务必重新做数据质量诊断**

数据集成是新一轮数据质量问题的来源：实体对齐的假阳性会将两个不同实体错误合并（引入噪声）；冲突消解策略的错误选择会覆盖正确值；字段的粒度冲突若未被发现，会导致数值比较无意义（"千元"与"元"相加）。集成完成后，必须重新执行 2.1 节的完整性/唯一性/一致性检验，视集成为一次新的"数据入库"事件。
{% endhint %}

数据集成处理了"从哪里来"和"如何对齐"的问题；完成集成的数据集中，特征的表示形式——特别是类别特征——往往仍不适合数值模型直接使用，这就引出了本节第二大主题：数据变换。

## 2.2.2 类别变量的编码

### 类别变量的类型学

机器学习模型的参数优化基于数值计算，而现实数据集中充斥着类别变量——性别、行政区划、商品品类、教育程度。在将类别变量送入任何数值模型之前，必须经过**编码**（Encoding）处理，将字符串映射为数值表示。编码方法的选择取决于类别变量的**内在类型**。

{% block type="definition" label="2.6" title="类别变量的类型与编码适配（Categorical Variable Taxonomy）" %}
按变量是否存在内在顺序，类别变量分为两类：

**（1）名义变量**（Nominal Variable）：类别之间无自然顺序，任意两个类别在距离或大小上无可比性。例：颜色（红/绿/蓝）、城市名称、商品品类。适配编码：**独热编码**（One-Hot Encoding）。

**（2）有序变量**（Ordinal Variable）：类别之间存在自然的全序关系 $c_1 \prec c_2 \prec \cdots \prec c_K$，但相邻类别间的"距离"可能不等。例：教育程度（初中$\prec$高中$\prec$本科$\prec$硕士$\prec$博士）、满意度（差$\prec$一般$\prec$好$\prec$很好）。适配编码：**序数编码**（Ordinal Encoding）。

当类别基数极高（$K \gg 100$，如省市级 POI 名称、用户 ID）时，独热编码会造成维度灾难，此时应使用**目标编码**（Target Encoding）或嵌入表示。
{% endblock %}

### 序数编码

序数编码将有序类别映射为整数 $\{0, 1, \ldots, K-1\}$：

$$
f_{ord}(c_k) = k - 1, \quad k = 1, 2, \ldots, K
\tag{2.10}
$$

序数编码保留了顺序信息，且只产生一列特征，适合树模型和线性模型（在有序变量的场景下）。但若将序数编码用于名义变量，则会引入虚假的"距离"概念——将"北京"编码为0、"上海"为1、"广州"为2，隐含地暗示北京与广州的"距离"是北京与上海的两倍，这在语义上是错误的。

### 独热编码与虚拟变量陷阱

独热编码为 $K$ 个类别创建 $K$ 个二进制列，每行的编码向量中恰好一位为1：

$$
f_{OH}(c_k) = \mathbf{e}_k \in \{0,1\}^K, \quad \text{其中 } (\mathbf{e}_k)_j = \mathbf{1}[j = k]
\tag{2.11}
$$

然而，在包含截距项的线性模型中，完整的 $K$ 列独热矩阵会导致一个代数问题。

{% block type="theorem" label="2.2" title="虚拟变量陷阱（Dummy Variable Trap）" %}
设类别变量有 $K$ 个类别，独热编码产生 $K$ 个二进制列向量 $\mathbf{e}_1, \ldots, \mathbf{e}_K \in \{0,1\}^n$。则必然有：

$$
\sum_{k=1}^{K} \mathbf{e}_k = \mathbf{1}_n
\tag{2.12}
$$

即 $K$ 个编码列之和等于全1向量 $\mathbf{1}_n$。若线性模型的设计矩阵 $\mathbf{X}$ 包含截距列 $\mathbf{1}_n$ 与全部 $K$ 个独热列，则 $\mathbf{X}$ 存在列线性依赖，$\mathbf{X}^T\mathbf{X}$ 奇异（不可逆），最小二乘法的法方程 $\hat{\boldsymbol{\beta}} = (\mathbf{X}^T\mathbf{X})^{-1}\mathbf{X}^T\mathbf{y}$ **无解**。

**推论**：对于含截距项的线性模型，独热编码只应保留 $K-1$ 个虚拟变量，删去其中一个（参照类/基准类）。删去哪一列不影响模型的拟合优度，仅改变截距项的解释。
{% endblock %}

{% hint style="proof" %}
**定理2.2的证明**

对每个样本 $i$，恰好有一个 $k^*$ 使得 $(\mathbf{e}_{k^*})_i = 1$，其余均为0。因此 $\sum_{k=1}^K (\mathbf{e}_k)_i = 1$ 对所有 $i$ 成立，即 $\sum_{k=1}^K \mathbf{e}_k = \mathbf{1}_n$（式(2.12)成立）。

若设计矩阵 $\mathbf{X}$ 包含 $[\mathbf{1}_n, \mathbf{e}_1, \ldots, \mathbf{e}_K]$，则由式(2.12)，$\mathbf{1}_n = \mathbf{e}_1 + \cdots + \mathbf{e}_K$，即截距列是各虚拟列的线性组合，$\mathbf{X}$ 的列不线性独立，$\text{rank}(\mathbf{X}) < K+1$，故 $\mathbf{X}^T\mathbf{X}$ 奇异。■
{% endhint %}

{% hint style="tip" %}
💡 **实践中的独热编码**

在 pandas 中，`pd.get_dummies(df['col'], drop_first=True)` 自动删除第一个类别，避免虚拟变量陷阱。在 scikit-learn 中，`OneHotEncoder(drop='first')` 或 `drop='if_binary'` 实现相同效果。但对于**树模型**（决策树、随机森林、XGBoost），树的分裂不依赖矩阵求逆，保留 $K$ 列实际上更有利于学习（每个类别可被直接选为分裂节点），无需删除参照列。
{% endhint %}

### 目标编码

对于高基数类别变量（如城市名称、用户职业），独热编码会产生成千上万列稀疏特征，导致维度灾难。**目标编码**（Target Encoding，也称均值编码）用类别的目标变量均值替换类别标签：

$$
\hat{x}_{ik} = \bar{y}_k = \frac{1}{n_k}\sum_{j:\,x_j=k} y_j
\tag{2.13}
$$

其中 $n_k$ 为类别 $k$ 的样本数，$y_j$ 为第 $j$ 个样本的目标值。目标编码的缺陷是**目标泄露**（Target Leakage）：若在全训练集上直接计算均值，则编码值已经包含了标签信息，会造成严重的过拟合。工业实践中通常采用 **James-Stein 收缩估计**（Smoothing）来正则化：

$$
\hat{x}_{ik} = \lambda_k \cdot \bar{y}_k + (1 - \lambda_k) \cdot \bar{y}, \quad
\lambda_k = \frac{n_k}{n_k + m}
\tag{2.14}
$$

其中 $\bar{y}$ 为全局均值，$m$ 为平滑超参数（通常取10–100）。当类别样本数 $n_k$ 较小时，$\lambda_k \to 0$，编码值向全局均值收缩，减少了对小样本类别的过拟合。正确的做法是在**交叉验证的折外（out-of-fold）数据上计算目标均值**，确保目标值不泄露到编码过程中（如图 2.2.2 所示）。

{% hint style="caution" %}
⚠️ **目标编码的最常见错误**

在目标编码中最严重的错误是：在全量训练数据（包含待编码样本）上计算类别均值，然后将编码后的特征直接用于交叉验证。这等价于在测试折中使用了测试样本的标签信息——即2.1节描述的数据泄露的又一形式。正确做法：在 k 折交叉验证的第 $i$ 折，目标均值必须只在其余 $k-1$ 折上计算，不得包含第 $i$ 折的任何样本。
{% endhint %}

类别变量编码解决了"如何将字符串特征数值化"的问题；接下来，对于已经是数值的特征，其**分布形态**也可能需要变换。

## 2.2.3 数值变换：偏态消除与方差稳定

### 偏态分布的影响

许多机器学习算法对特征的分布形态存在隐含假设：线性回归的残差假定为正态分布；贝叶斯分类器的高斯核假定特征正态；K 近邻和 SVM 的距离度量对尺度和偏度敏感。当数值特征呈现强右偏（如收入、房价、在线时长、交易金额）时：

- **最小二乘估计**受极端值的过度影响（极端值使残差平方远大于正常值）；
- **欧氏距离**被少数极端值主导，使相似度度量失效；
- **方差稳定性假设**被违反，导致某些统计检验的 I 类误差膨胀。

数学变换通过将强偏的分布映射为近似对称的分布，缓解上述问题。

### 常用变换族

对于严格正值特征 $x > 0$，常用变换包括：

| 变换名称 | 公式 | 适用偏斜程度 |
|---------|------|------------|
| 对数变换 | $y = \log(x + c)$，$c \geq 0$ | 中至强右偏 |
| 平方根变换 | $y = \sqrt{x}$ | 轻至中右偏 |
| 倒数变换 | $y = 1/x$ | 极强右偏 |
| Box-Cox 变换 | $y^{(\lambda)}$（见定义2.7） | 任意强度，自动选参 |

对于可能包含零或负值的特征，**Yeo-Johnson 变换**（2000年）将 Box-Cox 推广到 $x \in \mathbb{R}$（见补充块）。

### Box-Cox 变换及其最优参数估计

上述变换族中，Box-Cox 变换理论最为完备：通过引入连续参数 $\lambda$ 将所有幂变换（含对数变换）统一为单一参数族，无需凭经验选择变换形式，而是直接从数据中估计最优 $\lambda$。

{% block type="definition" label="2.7" title="Box-Cox 幂变换（Box-Cox Power Transformation, Box & Cox, 1964）" %}
对于严格正值变量 $y > 0$，Box-Cox 变换定义为含参数 $\lambda \in \mathbb{R}$ 的幂变换族：

$$
y^{(\lambda)} = \begin{cases}
\dfrac{y^\lambda - 1}{\lambda} & \lambda \neq 0 \\[8pt]
\ln y & \lambda = 0
\end{cases}
\tag{2.15}
$$

当 $\lambda = 0$ 时，利用 L'Hôpital 法则可验证 $(y^\lambda - 1)/\lambda \to \ln y$，因此定义是连续的。特殊情形：
- $\lambda = 1$：恒等变换（$y^{(1)} = y - 1$，偏移常数不影响分布形状）
- $\lambda = 0$：对数变换
- $\lambda = 0.5$：近似平方根变换
- $\lambda = -1$：倒数变换（含偏移）

**最优参数的最大似然估计**：假设变换后 $\{y_i^{(\lambda)}\}$ 服从正态分布，则 $\lambda$ 的**剖面对数似然函数**（Profile Log-Likelihood）为：

$$
\ell(\lambda) = -\frac{n}{2} \ln \hat{\sigma}^2_\lambda + (\lambda - 1) \sum_{i=1}^n \ln y_i
\tag{2.16}
$$

其中 $\hat{\sigma}^2_\lambda = \frac{1}{n}\sum_{i=1}^n \left(y_i^{(\lambda)} - \overline{y^{(\lambda)}}\right)^2$ 为变换后的样本方差。最优 $\hat{\lambda} = \arg\max_\lambda \ell(\lambda)$，在实践中通过在 $[-2, 2]$ 区间的网格搜索或数值优化（Brent方法）求解。
{% endblock %}

{% hint style="proof" %}
**公式(2.16)的推导**

设 $z_i = y_i^{(\lambda)}$ 服从 $\mathcal{N}(\mu, \sigma^2)$，则对数似然为：

$$
\ell(\mu, \sigma^2, \lambda) = -\frac{n}{2}\ln(2\pi\sigma^2) - \frac{1}{2\sigma^2}\sum_i(z_i - \mu)^2 + \sum_i \ln\left|\frac{\partial z_i}{\partial y_i}\right|
$$

第三项为雅可比项，由 $\partial z_i/\partial y_i = y_i^{\lambda - 1}$（当 $\lambda \neq 0$），得 $\sum_i \ln y_i^{\lambda-1} = (\lambda-1)\sum_i \ln y_i$。

对 $\mu$ 和 $\sigma^2$ 先进行剖面最大化（令偏导为零），得 $\hat{\mu} = \bar{z}$，$\hat{\sigma}^2 = \frac{1}{n}\sum_i(z_i - \bar{z})^2$，代入并整理即得式(2.16)。■
{% endhint %}

{% block type="example" label="2.4" title="Box-Cox 变换参数估计" %}
**数据**：某平台10笔交易金额（元）：

$$\{158, 203, 89, 1250, 97, 184, 2300, 115, 76, 178\}$$

（明显右偏：中位数 $\approx$ 167，均值 $\approx$ 465，偏度 $\approx$ 1.89）

**sklearn 实现**：

```python
from scipy import stats
import numpy as np

data = np.array([158, 203, 89, 1250, 97, 184, 2300, 115, 76, 178])
transformed, lam = stats.boxcox(data)
print(f"最优 λ = {lam:.3f}")          # 输出：λ ≈ 0.082（接近对数变换）
print(f"变换后偏度：{stats.skew(transformed):.3f}")   # 接近 0
```

**解释**：$\lambda \approx 0.082 \approx 0$，说明此数据的最优变换接近对数变换，但并非完全等价——Box-Cox 通过 MLE 自动找到了让变换后分布最接近正态的幂指数。
{% endblock %}

如图 2.2.3 所示，Box-Cox 变换在偏度消除方面通常优于固定的对数或平方根变换，因为后者的参数是预先给定的，而 Box-Cox 通过 MLE 自动最优化。

{% hint style="supplement" %}
🔬 **补充：Yeo-Johnson 变换——Box-Cox 的全域推广**

Box-Cox 变换要求 $y > 0$，这限制了其在含零值或负值特征（如温度差、利润率、对数收益率）上的应用。Yeo 与 Johnson（2000）提出了扩展版本，将定义域推广至全实数 $y \in \mathbb{R}$：

$$
\psi^{(\lambda)}(y) = \begin{cases}
\dfrac{(y+1)^\lambda - 1}{\lambda} & y \geq 0,\; \lambda \neq 0 \\[6pt]
\ln(y+1) & y \geq 0,\; \lambda = 0 \\[6pt]
-\dfrac{(-y+1)^{2-\lambda} - 1}{2-\lambda} & y < 0,\; \lambda \neq 2 \\[6pt]
-\ln(-y+1) & y < 0,\; \lambda = 2
\end{cases}
\tag{2.17}
$$

Yeo-Johnson 同样通过 MLE 估计 $\lambda$（剖面对数似然结构类似式(2.16)）。在 scikit-learn 中，`sklearn.preprocessing.PowerTransformer(method='yeo-johnson')` 默认实现了此变换。**实践规则**：若特征严格正，优先用 Box-Cox；若特征包含零或负值，用 Yeo-Johnson；若特征本身近似对称，两者均无必要。
{% endhint %}

{% hint style="caution" %}
⚠️ **训练集上估计 λ，测试集上应用**

Box-Cox 变换中的 $\hat{\lambda}$ 是从训练数据中通过 MLE 估计的参数，必须和模型权重一样对待：在训练集上 `fit`（估计 $\hat{\lambda}$ 和归一化参数），然后用相同的 $\hat{\lambda}$ 对测试集 `transform`。切不可在测试集上重新估计 $\lambda$——这会引入数据泄露，使模型评估过于乐观。将变换器纳入 `sklearn.Pipeline` 是防止这类错误的最可靠方式。
{% endhint %}

## 2.2.4 数值离散化

### 为何离散化

将连续数值变量切割为有限个离散区间（即**分箱**，Binning）看似是信息损失，但在特定场景下具有独特价值：

- **非线性关系捕捉**：若连续特征 $x$ 与目标 $y$ 存在非线性关系（如"收入在10万–20万区间时违约率骤升"），离散化后线性模型即可捕捉该关系；
- **噪声压制**：精确的连续值测量中可能含有仪器误差，离散化将多个微小差异合并到同一区间，提升鲁棒性；
- **可解释性**：业务报表和风控规则通常以"30岁以下"、"收入>10万"等区间表达，离散化特征的模型系数直接对应业务分区；
- **类别特征兼容**：离散化后的变量可进一步用独热编码处理，接入支持类别输入的模型。

{% block type="definition" label="2.8" title="数值离散化（Numerical Discretization / Binning）" %}
设连续特征 $x \in [x_{min}, x_{max}]$，将其划分为 $B$ 个互斥的区间（分箱）$\{I_1, I_2, \ldots, I_B\}$ 满足 $\bigcup_{b=1}^B I_b = [x_{min}, x_{max}]$，并将 $x$ 的值映射到对应的箱号 $b(x) \in \{1, \ldots, B\}$，称此过程为**离散化**。常用策略：

**（1）等宽分箱**（Equal-Width）：每个箱的宽度相同：
$$w = \frac{x_{max} - x_{min}}{B}, \quad I_b = [x_{min} + (b-1)w,\; x_{min} + bw)$$

**（2）等频分箱**（Equal-Frequency / Quantile）：每个箱包含相同数量的样本（基于分位数分割）：
$$I_b = [q_{(b-1)/B},\; q_{b/B})$$
其中 $q_p$ 为经验 $p$ 分位数。

**（3）监督分箱**（Supervised Binning）：以最大化各箱纯度（按目标变量熵衡量）为准则确定分割点，详见算法 2.2。
{% endblock %}

等宽与等频分箱是无监督方法，无需标签信息，实现简单但分割点选取带有主观性；监督分箱则将标签信息引入分割点搜索，寻找能最大化类别可分性的切割位置。其中，基于**最小描述长度原则**（Minimum Description Length Principle，MDL）的 MDLP 算法（Fayyad & Irani, 1993）以信息熵为准则，通过递归二分实现自适应的分割点选取：每次从候选位置中选择信息增益最大的切割点，直到 MDL 停止条件满足为止，从而在无需人工指定箱数的情况下自动确定分箱粒度。

{% block type="algorithm" label="2.2" title="基于最小描述长度的监督离散化（MDLP，Fayyad & Irani, 1993）" %}
**输入**：已标注样本集 $\{(x_i, y_i)\}_{i=1}^n$，特征值已升序排列；类别数 $K$；停止阈值 $\delta$ (默认以 MDL 准则自动确定)

**输出**：分割点集合 $\mathcal{T} = \{t_1 < t_2 < \cdots < t_m\}$

**步骤：**

1. 初始化候选分割点 $\mathcal{C} = \{(x_i + x_{i+1})/2 : y_i \neq y_{i+1}, i=1,\ldots,n-1\}$（仅考虑相邻类别边界处）；

2. 对每个候选点 $t \in \mathcal{C}$，计算分割的**信息增益**：

$$
\text{Gain}(t) = H(\mathcal{S}) - \frac{|\mathcal{S}_L|}{|\mathcal{S}|}H(\mathcal{S}_L) - \frac{|\mathcal{S}_R|}{|\mathcal{S}|}H(\mathcal{S}_R)
$$

其中 $H(\mathcal{S}) = -\sum_{k=1}^K p_k \log_2 p_k$ 为集合 $\mathcal{S}$ 的信息熵；

3. 选取使 $\text{Gain}(t)$ 最大的 $t^* = \arg\max_{t} \text{Gain}(t)$；

4. **MDL 停止准则**：若 $\text{Gain}(t^*) < \frac{\log_2(n-1)}{n} + \frac{\Delta(t^*, \mathcal{S})}{n}$ 则停止（$\Delta$ 项控制模型复杂度惩罚）；否则将 $t^*$ 加入 $\mathcal{T}$，递归对 $\mathcal{S}_L$ 和 $\mathcal{S}_R$ 重复上述过程；

5. 输出 $\mathcal{T}$，将 $(-\infty, t_1), [t_1, t_2), \ldots, [t_m, +\infty)$ 作为离散区间。

**时间复杂度**：$O(n \log^2 n)$（主导项为排序和递归划分）。
{% endblock %}

## 2.2.5 综合案例：跨平台电商订单数据集成与变换

本节所讨论的数据集成、类别编码与数值变换三项技术，在实际项目中往往需要串联执行。以下案例以某零售集团跨平台数据合并为背景，完整演示从 Schema 冲突到模型就绪特征矩阵的全流程，展示各技术模块如何在同一 Pipeline 中协同运作。

{% block type="case" label="2.2" title="跨平台电商订单数据：集成、编码与变换全流程" %}
**业务背景**：某零售集团同时运营 A 平台（自营电商）和 B 平台（第三方入驻），两套系统由不同技术团队独立开发。为构建统一的用户价值分析模型，需将两个平台的订单数据合并，并完成特征工程使其可直接输入购买预测模型。

**数据特征**：A平台：1.2万条订单，字段 `{user_id, item_category, price, order_ts}`；B平台：8000条订单，字段 `{buyer_no, cat_code, total_yuan, created_at}`。

**核心挑战**：
1. **集成**：字段命名冲突（`price` vs `total_yuan`）、类别编码不一致（A平台用中文类别名，B平台用数字品类码）、时间戳格式不同
2. **编码**：`item_category` 是高基数名义变量（50+品类）
3. **变换**：`price` 分布极度右偏（偏度≈3.2）
{% endblock %}

```python
# ============================================================
# 代码 2.2：跨平台电商订单集成与变换
# 数据：合成的 A/B 双平台订单数据
# 依赖库：pandas, numpy, scipy, sklearn
# ============================================================
# --- 1. 构造模拟数据 ---
import numpy as np
import pandas as pd
from scipy import stats
from sklearn.preprocessing import OneHotEncoder, PowerTransformer
import matplotlib
matplotlib.rcParams['font.sans-serif'] = ['SimHei']
matplotlib.rcParams['axes.unicode_minus'] = False
import matplotlib.pyplot as plt

rng = np.random.default_rng(42)

cats_a = ["服装", "电子", "家居", "食品", "美妆"]
cats_b_map = {"服装": "CAT_01", "电子": "CAT_02",
              "家居": "CAT_03", "食品": "CAT_04", "美妆": "CAT_05"}

n_a, n_b = 1200, 800

# 平台 A
df_a = pd.DataFrame({
    "user_id":       [f"U{rng.integers(1000,9999)}" for _ in range(n_a)],
    "item_category": rng.choice(cats_a, n_a),
    "price":         rng.lognormal(5.5, 1.2, n_a),   # 右偏金额（元）
    "order_ts":      pd.date_range("2023-01-01", periods=n_a, freq="45min"),
})

# 平台 B（不同字段名、数字品类码、时间戳格式不同）
df_b = pd.DataFrame({
    "buyer_no":      [f"B{rng.integers(5000,9999)}" for _ in range(n_b)],
    "cat_code":      [cats_b_map[c]
                      for c in rng.choice(cats_a, n_b)],
    "total_yuan":    rng.lognormal(5.3, 1.1, n_b),
    "created_at":    pd.date_range("2023-03-01", periods=n_b, freq="70min"),
})

print("平台A：", df_a.shape, df_a.dtypes.to_dict())
print("平台B：", df_b.shape, df_b.dtypes.to_dict())

# --- 2. Schema 匹配：字段统一 ---
# 反向品类码映射
cat_code_inv = {v: k for k, v in cats_b_map.items()}

df_b_aligned = pd.DataFrame({
    "user_id":       df_b["buyer_no"],          # 命名映射
    "item_category": df_b["cat_code"].map(cat_code_inv),   # 语义映射
    "price":         df_b["total_yuan"],        # 重命名
    "order_ts":      df_b["created_at"],        # 类型一致（已是 Timestamp）
    "source":        "B",
})
df_a["source"] = "A"

# --- 3. 数据融合 ---
df = pd.concat([df_a, df_b_aligned], ignore_index=True)
print(f"\n集成后：{df.shape}，来源分布：\n{df['source'].value_counts().to_string()}")

# 验证 Schema 一致性
assert df.isnull().sum()["item_category"] == 0, "品类映射存在空值！"

# --- 4. 类别变量编码 ---
# (a) 独热编码（drop_first 避免虚拟变量陷阱）
cat_dummies = pd.get_dummies(df["item_category"], prefix="cat",
                              drop_first=True, dtype=int)
print(f"\n独热编码：{cat_dummies.shape[1]} 列（K-1 = {len(cats_a)-1}）")
print(cat_dummies.head(3).to_string())

# (b) 目标编码（使用折外均值，简化演示：全局均值 + 平滑）
global_mean = df["price"].mean()
cat_counts  = df.groupby("item_category")["price"].agg(["mean", "count"])
smooth_m    = 30
cat_counts["target_enc"] = (
    (cat_counts["count"] * cat_counts["mean"] + smooth_m * global_mean) /
    (cat_counts["count"] + smooth_m)
)
df["cat_target_enc"] = df["item_category"].map(cat_counts["target_enc"])
print(f"\n目标编码（James-Stein, m={smooth_m}）：")
print(cat_counts[["mean", "count", "target_enc"]].to_string())

# --- 5. 数值变换：Box-Cox ---
price_pos = df["price"].values
price_bc, lam = stats.boxcox(price_pos)
print(f"\nBox-Cox λ = {lam:.3f}，"
      f"变换前偏度 = {stats.skew(price_pos):.3f}，"
      f"变换后偏度 = {stats.skew(price_bc):.3f}")

# 对比：对数变换
price_log = np.log1p(price_pos)
print(f"Log 变换后偏度 = {stats.skew(price_log):.3f}")

# --- 6. 可视化：变换效果 ---
fig, axes = plt.subplots(1, 3, figsize=(13, 4))
for ax, data, title in zip(
        axes,
        [price_pos, price_log, price_bc],
        ["原始金额（元）",
         f"Log(1+x) 变换（偏度={stats.skew(price_log):.2f}）",
         f"Box-Cox（λ={lam:.2f}，偏度={stats.skew(price_bc):.2f}）"]):
    ax.hist(data, bins=50, color="#2563eb", alpha=0.65,
            density=True, edgecolor="white", lw=0.3)
    ax.set_title(title, fontsize=11)
    ax.set_ylabel("概率密度")
plt.suptitle("集成订单金额的分布变换对比", fontsize=13, y=1.02)
plt.tight_layout()
plt.savefig("price_transform.png", dpi=150, bbox_inches="tight")
plt.show()

# --- 7. 最终特征矩阵 ---
df["price_bc"] = price_bc
feature_cols = ["price_bc", "cat_target_enc"] + cat_dummies.columns.tolist()
X = pd.concat([df[["price_bc", "cat_target_enc"]], cat_dummies], axis=1)
print(f"\n最终特征矩阵：{X.shape}")
print(X.describe().round(3).to_string())
```

**结果解读**：Schema 匹配将两平台的5个品类码与中文类别名完成语义对齐，无空值引入；集成后总记录数2,000条，来源标记保留。原始订单金额偏度约3.2，对数变换后约0.31，Box-Cox（λ≈0.10）进一步降至约0.08，最接近正态。独热编码生成4列（K-1=4），目标编码（m=30）生成1列，兼顾了稀疏性与信息密度的平衡。

{% hint style="engineering" %}
🔧 **工程实践：Feature Store 与编码一致性**

在生产环境中，数据变换中最容易被忽视的问题是**训练-服务一致性**（Training-Serving Consistency）：训练时用历史数据估计的 Box-Cox $\hat{\lambda}$、目标编码的类别均值、独热编码的列顺序，在线上推理时必须完全一致。一旦不一致，即使模型本身正确，推理结果也会是乱码。

工业界的解决方案是 **Feature Store**（特征存储）：
- **离线存储**（Offline Store）：Hive/Spark SQL 批量计算并存储历史特征值，供模型训练读取；
- **在线存储**（Online Store）：Redis/DynamoDB 提供低延迟特征查询，供线上推理使用；
- **元数据注册**：每个特征的变换参数（$\hat{\lambda}$、均值、分位数边界）作为版本化的元数据存储，与特征数据一起版本控制。

代表性开源工具：Feast（Meta/Google），Hopsworks，Tecton（商业）。在未引入 Feature Store 之前，至少应将所有 sklearn 变换器序列化（`joblib.dump`/`pickle`）并与模型一同部署，确保在线 `transform` 使用的参数与离线训练完全一致。
{% endhint %}

{% hint style="frontier" %}
🚀 **前沿视野：表格深度学习中的"自动编码"范式**

传统编码方法（独热、序数、目标）都是人工设计的启发式规则，需要特征工程专家手动选择。2022–2024年，以 **FT-Transformer**（Gorishniy et al., 2022）、**TabNet**（Arik & Pfister, 2021）和 **SAINT**（Somepalli et al., 2021）为代表的"表格深度学习"模型提出了端到端的自动特征表示学习框架：

**FT-Transformer** 将每个特征（无论数值或类别）映射为一个固定维度的嵌入向量（Feature Tokenization），然后通过 Transformer Encoder 学习特征间的复杂交互，无需手工编码决策。在多个 Kaggle 基准数据集上，FT-Transformer 首次实现了与 XGBoost 的竞争性性能，打破了"表格数据上梯度提升永远优于深度学习"的行业共识。

**对传统编码的影响**：在数据规模足够大（$n > 5$万）时，基于嵌入的自动编码可替代人工特征工程；但在中小数据集（$n < 1$万）上，精心设计的手工编码仍然优于端到端方法。当前主流观点是：理解和掌握手工编码方法是基础，而表格深度学习是在大数据场景下的高级选项——二者并非替代关系。

LLM 在表格特征编码上的应用（2024）：GPT-4 可在零样本（zero-shot）条件下为高基数类别特征生成语义嵌入（如直接将"职业:软件工程师"转换为128维语义向量），在若干公开数据集上超过了传统目标编码的效果。
{% endhint %}

{% hint style="ideology" %}
🌏 **课程思政：特征编码中的隐性歧视**

数据变换看似是纯技术操作，但编码方式的选择深刻地影响着模型的社会公平性。

**邮政编码作为种族代理**：在美国，居住地的邮政编码（Zip Code）与种族构成高度相关（这是历史上种族隔离政策"红线划定"（Redlining）的遗产）。当银行或保险公司将邮政编码进行目标编码（用该区域的历史违约率替换），实质上是将历史性的系统歧视编码进了特征矩阵，即使模型从未显式使用"种族"字段，依然可能产生差异化的利率或拒贷决策。在中国，类似的代理变量包括户籍地、居住小区名称（与房价高度相关）等。

**有序编码与隐含价值排序**：将"受教育程度"序数编码为{小学=0, 初中=1, 高中=2, 本科=3, 研究生=4}，隐含地为模型提供了一种价值判断——学历越高，数值越大，模型越"青睐"。这种编码在职位推荐、信贷审批中可能导致对低学历群体的系统性不利。

**思考原则**：在完成任何类别特征编码之前，应先问：**这个特征是否是敏感属性（性别、民族、籍贯）的代理变量？编码方式是否隐含了对某个群体的价值判断？** 这是数据工程师无法回避的伦理责任。《个人信息保护法》第24条明确禁止以自动化决策方式对个人在就业、交易等方面进行不合理的差别待遇——而不合理差别往往正是从编码阶段悄然植入的。
{% endhint %}

## 本节小结

本节建立了数据集成与数据变换的完整技术框架。**数据集成**（定义2.5）通过三阶段管道（Schema匹配→实体对齐→数据融合）将异构多源数据统一化，集成后须重新执行数据质量诊断以防止引入新问题。**类别变量编码**（定义2.6）需根据变量类型选择序数/独热/目标编码，独热编码的虚拟变量陷阱（定理2.2）源于 $K$ 列二进制向量的线性依赖，线性模型须使用 $K-1$ 列。**Box-Cox 变换**（定义2.7）通过最大化剖面对数似然（式2.16）自动确定最优幂参数，在消除偏态方面优于固定的对数或平方根变换；Yeo-Johnson 将其推广至全实数域。**数值离散化**（定义2.8）提供等宽、等频和基于信息熵的监督分箱三种策略（算法2.2），分别适用于不同的分布形态和业务目标。

完成集成与变换后，数据在特征维度上往往仍然冗余或过多，这导向本章第三个核心主题——数据规约，将在第2.3节中讨论。

### B 级（数学推导）

**B1.** （推导）证明虚拟变量陷阱（定理2.2）：对 $K$ 个类别的独热编码列 $\mathbf{e}_1, \ldots, \mathbf{e}_K$，严格证明 $\sum_{k=1}^K \mathbf{e}_k = \mathbf{1}_n$，并由此推导设计矩阵 $[\mathbf{1}_n, \mathbf{e}_1, \ldots, \mathbf{e}_K]$ 列线性相关、$(\mathbf{X}^T\mathbf{X})$ 奇异的完整代数论证。

**B2.** （推导）推导 Box-Cox 变换的剖面对数似然公式(2.16)的完整过程：(a) 写出观测数据 $\{y_i\}$ 在 $\lambda$ 下的对数似然（含雅可比项）；(b) 对 $\mu$ 和 $\sigma^2$ 求偏导并令其为零；(c) 将解代入得到仅关于 $\lambda$ 的剖面函数。

**B3.** （推导）James-Stein 收缩目标编码的理论基础。设目标变量 $y \sim \mathcal{N}(\theta, \sigma^2)$，类别均值估计 $\bar{y}_k$ 基于 $n_k$ 个观测。证明：式(2.14)中的收缩估计 $\hat{x}_{ik} = \lambda_k \bar{y}_k + (1-\lambda_k)\bar{y}$ 在均方误差（MSE）意义下，当 $m = \sigma^2 / \tau^2$（$\tau^2$ 为类别均值的先验方差）时，是贝叶斯估计量（提示：使用正态-正态共轭先验推导后验均值）。
