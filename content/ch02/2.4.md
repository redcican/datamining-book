# 2.4 特征选择与特征工程

2012 年，Kaggle 举办了一场广告点击率预测竞赛。冠军团队的核心贡献不是更复杂的模型，而是一个精妙的特征工程决策：将用户 ID 与广告主 ID 的**组合**作为输入特征，即所谓**交互特征**（interaction feature）。这一改动将模型 AUC 提升了 4 个百分点——相当于把模型结构从逻辑回归换成梯度提升树的提升幅度。Pedro Domingos（2012）在综述中明确指出：**"特征工程是机器学习中真正的艺术；数据与特征决定了模型的上限，算法只是逼近这个上限"**。

前三节处理了数据的质量问题（清洗）、来源问题（集成与变换）和维度问题（规约）。本节关注的是更主动的一步：**如何从原始属性中挖掘或构造出信息密度更高的特征**，使模型以更少的维度捕捉更多的目标信息。特征选择（Feature Selection）与特征工程（Feature Engineering）是这一过程的两翼：前者在已有特征中挑选最有价值的子集，后者通过变换和组合创造全新的派生特征。

6. 识别特征工程 Pipeline 中的数据泄露陷阱，确保所有变换仅在训练集上拟合。

## 2.4.1 特征的价值：从属性到信息

"特征"在机器学习语境中，是指经过设计或选择后输入模型的可计算量。原始数据集中的每一列都是候选特征，但并非每列都值得保留。一个特征的价值取决于三个维度：**与目标的相关性**（relevance）、**与其他特征的非冗余性**（non-redundancy）和**可计算性**（computability）。

{% hint style="think" %}
🤔 **思考**：若某特征与目标变量 $$y$$ 的 Pearson 相关系数 $$r = 0$$，是否说明该特征对预测 $$y$$ 毫无价值？请考虑 $$y = x^2$$（$$x \sim \mathcal{N}(0, 1)$$）的情形，计算 $$\mathrm{Corr}(x, y)$$ 并思考其含义。
{% endhint %}

{% block type="definition" label="2.13" title="互信息（Mutual Information）" %}
设随机变量 $$X$$ 与 $$Y$$ 的联合分布为 $$p(x, y)$$，边缘分布分别为 $$p(x)$$ 与 $$p(y)$$。$$X$$ 与 $$Y$$ 之间的**互信息**（Mutual Information）定义为：

$$
I(X; Y) = \sum_{x,y} p(x, y) \log \frac{p(x, y)}{p(x)\,p(y)}
$$

对于连续变量，以积分代替求和。等价表达：

$$
I(X; Y) = H(X) - H(X \mid Y) = H(Y) - H(Y \mid X)
\tag{2.25}
$$

其中 $$H(X) = -\sum_x p(x)\log p(x)$$ 为 $$X$$ 的香农熵，$$H(X|Y)$$ 为条件熵。

**核心性质**：$$I(X;Y) \geq 0$$，当且仅当 $$X$$ 与 $$Y$$ 统计独立时取零。互信息无需假设线性关系，能检测任意形式的统计依赖。
{% endblock %}

在特征选择中，互信息量化了特征 $$X_j$$ 对目标 $$Y$$ 的"信息贡献"：

$$
I(X_j; Y) = H(Y) - H(Y \mid X_j)
\tag{2.26}
$$

这正是**信息增益**（Information Gain）的形式化表达——知道特征 $$X_j$$ 之后，目标 $$Y$$ 的不确定性减少了多少。互信息与 Pearson 相关系数的关键差异在于：后者仅度量线性依赖，前者度量**一切形式**的统计依赖（见图 2.4.1(b)）。

特征选择方法按其与模型的关系，分为三大类：

{% block type="definition" label="2.14" title="过滤式特征选择（Filter Feature Selection）" %}
**过滤式方法**独立于下游模型，直接按特征与目标的统计相关性对特征评分，选取前 $$K$$ 个（或分数超过阈值 $$\theta$$ 的）特征。

常用评分准则（数值目标）：
- **Pearson 相关系数**：$$|r_{jY}| = |\mathrm{Corr}(X_j, Y)|$$，仅捕捉线性关系；
- **互信息**：$$I(X_j; Y)$$（式 2.26），捕捉任意统计依赖，连续变量用 $$k$$-NN 估计（Kraskov et al., 2004）；
- **F 统计量**：基于方差分析，检验特征与目标的线性关联显著性；
- **方差阈值**：过滤方差极低（几乎常数）的特征。

常用评分准则（类别目标）：卡方统计量：

$$
\chi^2 = \sum_{k=1}^{K}\sum_{l=1}^{L} \frac{(O_{kl} - E_{kl})^2}{E_{kl}}
\tag{2.27}
$$

其中 $$O_{kl}$$ 为观测频数，$$E_{kl}$$ 为期望频数，用于检验离散特征与类别目标的独立性。

**优点**：计算效率高（$$O(nd)$$），适合高维初筛；**缺点**：忽略特征间交互，无法识别联合相关（单独无信息、组合有信息的特征对）。
{% endblock %}

{% block type="example" label="2.8" title="手动计算互信息：离散变量示例" %}
**场景**：预测信用违约（$$Y$$：0=正常，1=违约），特征为收入等级（$$X$$：低/中/高）。

**联合频率表**（基于 1000 条记录）：

| $$X \backslash Y$$ | 正常（0） | 违约（1） | 合计 |
|:---:|:---:|:---:|:---:|
| 低收入 | 160 | 140 | 300 |
| 中收入 | 380 | 120 | 500 |
| 高收入 | 180 | 20 | 200 |
| **合计** | **720** | **280** | **1000** |

**步骤 1 — 边缘熵**：$$H(Y) = -0.72\log_2(0.72) - 0.28\log_2(0.28) \approx 0.863$$ 比特。

**步骤 2 — 条件熵**：
$$H(Y|X) = 0.30 \cdot H(Y|X=\text{低}) + 0.50 \cdot H(Y|X=\text{中}) + 0.20 \cdot H(Y|X=\text{高})$$

其中 $$H(Y|X=\text{低}) = -\tfrac{160}{300}\log_2\tfrac{160}{300} - \tfrac{140}{300}\log_2\tfrac{140}{300} \approx 0.997$$，类似计算中/高收入项约为 0.863 和 0.469：

$$H(Y|X) \approx 0.30 \times 0.997 + 0.50 \times 0.863 + 0.20 \times 0.469 \approx 0.826 \text{ 比特}$$

**步骤 3 — 互信息**：$$I(X; Y) = H(Y) - H(Y|X) \approx 0.863 - 0.826 = 0.037$$ 比特。

**结论**：收入等级提供约 0.037 比特信息，信息增益率约 4.3%。高收入群体违约率（10%）远低于低收入（47%），说明收入等级弱相关但有统计意义。
{% endblock %}

## 2.4.2 包裹式与嵌入式特征选择

过滤式方法忽略了特征与模型的交互——某些特征单独弱相关但组合强相关。**包裹式方法**和**嵌入式方法**通过引入模型反馈来弥补这一缺陷。

{% block type="definition" label="2.15" title="包裹式特征选择（Wrapper Feature Selection）" %}
**包裹式方法**将特征子集选择视为搜索问题：以下游模型的交叉验证性能作为评分函数，在特征子集空间中搜索最优子集。

**递归特征消除**（Recursive Feature Elimination, RFE）是最常用的包裹式策略：
1. 在全部 $$d$$ 个特征上训练模型，获得特征重要性评分 $$\{s_j\}$$；
2. 移除重要性最低的 $$\Delta$$ 个特征（$$\Delta \geq 1$$）；
3. 重复上述过程，直至保留特征数达到指定值 $$K$$；
4. 配合交叉验证（RFECV）自动确定最优 $$K$$。

**优点**：考虑特征间交互，性能通常优于过滤式；**缺点**：计算代价为 $$O(d/\Delta)$$ 次模型训练，当 $$d$$ 很大时开销巨大。
{% endblock %}

RFE 框架中，最优保留特征数 $$K$$ 本身是一个超参数——手动指定容易欠选或过选。标准做法是引入**交叉验证**：对每一个候选 $$K$$ 估计泛化性能，选取 CV 性能饱和时的最小 $$K$$，既保证模型能力，又最大程度地降低过拟合风险。算法 2.4 给出完整的 RFECV 流程，其 CV 性能曲线即为图 2.4.2(a) 所示。

{% block type="algorithm" label="2.4" title="带交叉验证的递归特征消除（RFECV）" %}
**输入**：数据 $$(\mathbf{X}, \mathbf{y})$$，估计器 $$f$$，折数 $$v$$，步长 $$\Delta$$

**输出**：最优特征子集 $$S^*$$，最优特征数 $$K^*$$

1. $$S \leftarrow \{1, \ldots, d\}$$（初始化特征集为全集）
2. **while** $$|S| > 1$$：
   1. 用 $$v$$ 折 CV 在特征子集 $$S$$ 上评估估计器，记录均值分数 $$\text{score}[|S|]$$
   2. 在全训练集上训练 $$f$$，获得各特征重要性 $$\{s_j\}_{j \in S}$$
   3. $$S \leftarrow S \setminus \arg\text{min}_{|\Delta|} \{s_j\}$$（移除最不重要的 $$\Delta$$ 个）
3. $$K^* \leftarrow \arg\max_k \text{score}[k]$$
4. **返回**：对应 $$K^*$$ 时保留的特征子集 $$S^*$$

**复杂度**：共执行 $$\lceil d/\Delta \rceil$$ 次模型训练（每次含 $$v$$ 折），总体为 $$O((d/\Delta) \cdot v \cdot T_f)$$，$$T_f$$ 为单次模型训练时间。
{% endblock %}

**嵌入式方法**将特征选择内建于模型训练过程，最典型的是 **Lasso 回归**（Least Absolute Shrinkage and Selection Operator）：

{% block type="definition" label="2.16" title="Lasso 嵌入式特征选择（Lasso Regression）" %}
Lasso 回归在最小二乘目标上附加 L1 正则化惩罚：

$$
\hat{\boldsymbol{\beta}} = \arg\min_{\boldsymbol{\beta}} \left\{ \frac{1}{n}\|\mathbf{y} - \mathbf{X}\boldsymbol{\beta}\|_2^2 + \alpha \|\boldsymbol{\beta}\|_1 \right\}
\tag{2.28}
$$

其中 $$\alpha \geq 0$$ 为正则化强度，$$\|\boldsymbol{\beta}\|_1 = \sum_j |\beta_j|$$。

**与 Ridge 的关键差异**：Ridge（L2 正则化）将系数均匀收缩但不归零；Lasso（L1 正则化）倾向于将部分系数**精确收缩至零**，从而自动实现特征选择。这一差异源于 L1 球的"角点"几何结构——最优解倾向于落在坐标轴上（某些 $$\beta_j = 0$$）。
{% endblock %}

Lasso 产生零系数的直觉来自几何图像：L1 约束域（$$\ell_1$$ 球）在坐标轴上有尖锐的顶点，而 L2 约束域（$$\ell_2$$ 球）是光滑的球面。损失函数的等高线与约束域的切点极大概率落在 $$\ell_1$$ 球的顶点（即某些 $$\beta_j = 0$$ 的坐标轴上），而不会落在 $$\ell_2$$ 球的光滑曲面上。定理 2.4 将这一几何直觉转化为严格的数量上界。

{% block type="theorem" label="2.4" title="Lasso 稀疏性定理" %}
设 $$\mathbf{X} \in \mathbb{R}^{n \times d}$$ 为设计矩阵，$$\mathrm{rank}(\mathbf{X}) = r$$。对于 Lasso 问题（式 2.28），存在阈值 $$\alpha^* > 0$$ 使得：当 $$\alpha \geq \alpha^*$$ 时，最优解 $$\hat{\boldsymbol{\beta}}$$ 满足 $$\|\hat{\boldsymbol{\beta}}\|_0 \leq r$$，即非零系数的数量不超过设计矩阵的秩。

**推论**：随着 $$\alpha$$ 从零增大，系数路径 $$\hat{\boldsymbol{\beta}}(\alpha)$$ 是分段线性的（LARS 算法，Efron et al., 2004）；各系数依次收缩至零，最后归零的特征具有最强的预测贡献（见图 2.4.2(b)）。

**证明思路**：KKT 最优条件要求次梯度包含零：$$\frac{2}{n}\mathbf{X}^{\top}(\mathbf{y} - \mathbf{X}\hat{\boldsymbol{\beta}}) = \alpha \mathbf{s}$$，其中 $$s_j = \mathrm{sign}(\hat{\beta}_j)$$（$$\hat{\beta}_j \neq 0$$ 时）或 $$s_j \in [-1, 1]$$（$$\hat{\beta}_j = 0$$ 时）。当 $$\alpha$$ 足够大时，所有次梯度约束可在零处满足。$$\square$$
{% endblock %}

{% hint style="think" %}
🤔 **思考**：图 2.4.2(b) 的 Lasso 路径中，随着 $$\alpha$$ 从左（小，欠正则化）到右（大，过正则化）增大，系数依次收缩至零。能否从路径图中直接读出"最重要的前 3 个特征"？这与 RFE 给出的排名一致吗？
{% endhint %}

{% block type="example" label="2.9" title="Lasso 路径：稀疏化过程的可视化解读" %}
**场景**：15 个特征的回归任务（n=300），其中 5 个为真实信号特征。

**图 2.4.2(b) 解读**：

- **最左侧**（$$\alpha \to 0$$）：所有系数取近似普通最小二乘（OLS）估计值，15 条路径均非零；
- **中段**（CV 最优 $$\alpha^*$$，红色虚线处）：约 10 条路径已归零，仅保留 5 个信号特征——与真实特征集高度吻合；
- **最右侧**（$$\alpha$$ 过大）：所有系数归零，模型退化为截距项。

**与 RFE 的比较**（图 2.4.2(c)）：三种方法（互信息过滤、RFE、Lasso）在本例中对 5 个信号特征的识别结果高度一致，说明在线性数据上不同方法趋于收敛；在非线性数据上，Lasso 和 F 统计量会漏掉非线性特征，而互信息和基于树的 RFE 则更稳健。
{% endblock %}

## 2.4.3 特征构造：从原始属性创造新特征

特征**选择**在已有特征中挑选子集；特征**构造**则主动创造新的派生特征，捕捉原始特征无法直接表达的信息结构。

### 多项式特征扩展

对于 $$p$$ 个原始特征 $$\mathbf{x} = (x_1, \ldots, x_p)^{\top}$$，$$d$$ 阶多项式特征映射为：

$$
\phi^{(d)}(\mathbf{x}) = \bigl\{x_1^{a_1} x_2^{a_2} \cdots x_p^{a_p} : a_i \geq 0,\ \textstyle\sum_i a_i \leq d\bigr\}
\tag{2.29}
$$

特征数从 $$p$$ 扩展到 $$\binom{p + d}{d}$$（组合增长）。例如 $$p=2, d=2$$：原始 2 维变为 6 维 $$\{1, x_1, x_2, x_1^2, x_1 x_2, x_2^2\}$$。

**适用场景**：线性模型需要拟合非线性关系时（如 $$y \approx a + bx + cx^2$$）；配合 Lasso 可自动选择有效的高阶项（见图 2.4.3(b)）。

**注意**：$$d$$ 阶多项式扩展后维度随 $$p$$ 呈组合增长，大规模数据需结合正则化（Lasso）或核方法（核 PCA）控制复杂度。

### 分箱离散化

将连续特征 $$X_j$$ 划分为 $$B$$ 个区间，以区间编号（或区间均值）替代原始数值，捕捉非单调的分段关系（见图 2.4.3(a)）。区间边界 $$\{t_b\}$$ 可等宽（uniform）、等频（quantile）或基于目标监督优化（见 2.2.4 节 MDLP 算法）。分箱离散化对异常值鲁棒，且允许线性模型隐式建模分段非线性关系。

### 交互特征

交互特征捕捉两个特征**联合作用**的效果，典型形式为乘积项 $$x_{ij} = x_i \cdot x_j$$。当目标与单个特征均无线性关联、但与两个特征的积有关时，交互项是不可或缺的（见图 2.4.3(c) 的 XOR 分类问题）。

在领域知识指导下构造的交互特征往往远比机械枚举所有乘积对更有效。例如：
- 电商场景：`用户历史点击率 × 商品品类匹配度`；
- 金融场景：`负债总额 / 月收入`（债务收入比）；
- 医疗场景：`用药剂量 × 肾功能指标`（剂量-代谢交互）。

{% hint style="caution" %}
⚠️ **特征构造的维度陷阱**

多项式扩展与交互项枚举会使特征维度爆炸式增长：2 阶多项式将 100 个特征扩展为 5151 个，3 阶则超过 17 万。在不加正则化的情况下，高维特征空间极易过拟合。**正确做法**：多项式扩展后搭配 Lasso（L1 正则化）进行稀疏选择；或预先通过领域知识缩小候选交互对的范围，避免机械枚举。
{% endhint %}

{% hint style="engineering" %}
🔧 **工程实践：sklearn Pipeline 中的特征工程**

**黄金法则：所有特征工程步骤必须纳入 Pipeline，且仅在训练集上 fit。**

```python
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, PolynomialFeatures
from sklearn.feature_selection import SelectKBest, mutual_info_regression, RFECV
from sklearn.linear_model import LassoCV, LinearRegression
from sklearn.model_selection import KFold
# ── 方案一：互信息粗筛 + Lasso 精选 ──────────────────────────────────────
pipe_filter = Pipeline([
    ("scaler",  StandardScaler()),
    ("mi_sel",  SelectKBest(mutual_info_regression, k=20)),
    ("lasso",   LassoCV(cv=5, max_iter=5000)),
])
pipe_filter.fit(X_train, y_train)
print(f"Lasso 选出非零特征数：{(pipe_filter['lasso'].coef_ != 0).sum()}")
# ── 方案二：多项式扩展 + Lasso ─────────────────────────────────────────────
pipe_poly = Pipeline([
    ("scaler", StandardScaler()),
    ("poly",   PolynomialFeatures(degree=2, include_bias=False)),
    ("lasso",  LassoCV(cv=5, max_iter=5000)),
])
pipe_poly.fit(X_train, y_train)
# ── 方案三：RFECV（包裹式）─────────────────────────────────────────────────
pipe_rfe = Pipeline([
    ("scaler", StandardScaler()),
    ("rfe",    RFECV(LinearRegression(), step=1,
                     cv=KFold(5, shuffle=True, random_state=42),
                     scoring="r2")),
])
pipe_rfe.fit(X_train, y_train)
print(f"RFE 最优特征数：{pipe_rfe['rfe'].n_features_}")
```

**常见数据泄露陷阱及规避**：

| 错误做法 | 泄露原因 | 正确做法 |
|---------|---------|---------|
| 在全集上 fit `SelectKBest` 再 split | 测试集信息影响特征选择 | 在 Pipeline 内，仅对训练折 fit |
| 全集计算目标编码均值 | 测试集目标泄露到训练特征 | 使用 `TargetEncoder(cv=5)` 交叉编码 |
| 全集 fit `StandardScaler` 再 fit Lasso | 测试集均值/方差影响归一化 | 禁止对 `X_test` 调用 `.fit_transform()` |

**特征重要性记录**：使用 MLflow 记录 `pipe['lasso'].coef_` 和 `pipe['rfe'].support_`，确保特征工程方案可复现。每次特征集变更视为模型版本迭代，应单独记录并对比性能。
{% endhint %}

{% hint style="supplement" %}
🔬 **补充：基于树模型的特征重要性**

除 Lasso 系数外，基于树的集成模型（随机森林、XGBoost）提供另一类嵌入式特征重要性评分。**基尼重要性**（Mean Decrease in Impurity, MDI）计算每个特征在所有树的所有分裂中平均减少的杂质量；**排列重要性**（Permutation Importance）通过随机打乱单个特征值后测量模型性能下降来评估重要性，更为可靠（Breiman, 2001；Strobl et al., 2008）。

```python
from sklearn.ensemble import RandomForestRegressor

# perm.importances_mean: 每个特征的排列重要性均值（越大越重要）
```

## 2.4.5 案例：泰坦尼克号生存预测的特征工程

{% block type="case" label="2.4" title="特征工程实践：泰坦尼克号生存预测" %}
**背景。** 泰坦尼克号数据集是特征工程的经典教学案例。原始特征包括：`Pclass`（船舱等级）、`Sex`（性别）、`Age`（年龄，含缺失值）、`SibSp`（兄弟姐妹/配偶数）、`Parch`（父母/子女数）、`Fare`（票价）、`Embarked`（登船港口）、`Cabin`（舱位号，大量缺失）、`Name`（姓名字符串）。

**目标**：设计特征工程方案，使逻辑回归模型的 10 折 CV 准确率达到或超过梯度提升树（原始特征）的基线。

```python
# ============================================================
# 代码 2.4：泰坦尼克号生存预测的特征工程
# 数据集：Titanic（Kaggle 竞赛 train.csv）
# 依赖库：pandas, numpy, scikit-learn
# ============================================================
# --- 1. 导入依赖 ---
import pandas as pd
import numpy as np
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score
# --- 2. 特征工程函数 ---
def engineer_features(df):
    df = df.copy()
    # (a) 家庭规模：组合 SibSp + Parch，独行旅客单独标记
    df["FamilySize"] = df["SibSp"] + df["Parch"] + 1
    df["IsAlone"]    = (df["FamilySize"] == 1).astype(int)
    # (b) 从 Name 提取称谓（头衔隐含社会阶层与性别角色）
    df["Title"] = df["Name"].str.extract(r",\s*([^\.]+)\.")
    rare = ["Lady", "Countess", "Capt", "Col", "Don",
            "Dr", "Major", "Rev", "Sir", "Jonkheer", "Dona"]
    df["Title"] = df["Title"].replace(rare, "Rare")
    df["Title"] = df["Title"].replace({"Mlle": "Miss", "Ms": "Miss",
                                        "Mme": "Mrs"})
    title_map = {"Mr": 1, "Miss": 2, "Mrs": 3, "Master": 4, "Rare": 5}
    df["Title"] = df["Title"].map(title_map).fillna(0)
    # (c) Fare 分箱（等频，4 组）——缓解票价右偏分布的影响
    df["FareBin"] = pd.qcut(df["Fare"].fillna(df["Fare"].median()),
                             q=4, labels=False)
    # (d) Age 分箱（等宽，5 组）+ 缺失值填充中位数
    df["Age"] = df["Age"].fillna(df["Age"].median())
    df["AgeBin"] = pd.cut(df["Age"], bins=[0, 16, 32, 48, 64, 100],
                          labels=False)
    # (e) 甲板字母（Cabin 首字母；缺失记为 'U'——Unknown）
    df["Deck"] = df["Cabin"].fillna("U").str[0]
    deck_map = {"A": 1, "B": 2, "C": 3, "D": 4, "E": 5,
                "F": 6, "G": 7, "T": 8, "U": 0}
    df["Deck"] = df["Deck"].map(deck_map).fillna(0)
    # (f) 交互特征：性别 × 舱等（捕捉"女性优先 + 头等舱"联合效应）
    df["Sex_num"]  = df["Sex"].map({"male": 0, "female": 1})
    df["SexClass"] = df["Sex_num"] * df["Pclass"]
    features = ["Pclass", "Sex_num", "AgeBin", "FareBin",
                "FamilySize", "IsAlone", "Title", "Deck", "SexClass"]
    return df[features]
# --- 3. 模型评估（10 折交叉验证）---
# 数据来源：Kaggle Titanic train.csv（kaggle competitions download -c titanic）
df = pd.read_csv("data/titanic_train.csv")
y  = df["Survived"]
X  = engineer_features(df)
pipe = Pipeline([
    ("scaler", StandardScaler()),
    ("clf",    LogisticRegression(max_iter=500, random_state=42)),
])
scores = cross_val_score(pipe, X, y, cv=10, scoring="accuracy")
print(f"工程特征 + 逻辑回归：{scores.mean():.3f} ± {scores.std():.3f}")
```

**关键工程特征解读：**

| 构造特征 | 工程逻辑 | 信息增益（近似） |
|---------|---------|:---:|
| `Title`（称谓） | 姓名头衔编码社会阶层（Master=男童，Rare=贵族） | 0.18 比特 |
| `SexClass`（性别 x 舱等） | 捕捉"女性优先"政策在不同舱等的差异效应 | 0.12 比特 |
| `FamilySize`（家庭规模） | 独行旅客 vs 家庭旅行者，逃生策略不同 | 0.07 比特 |
| `FareBin`（票价分箱） | 票价非线性分布，分箱比原始值更稳健 | 0.09 比特 |
| `Deck`（甲板层） | 舱位接近救生艇区域的程度（A/B 层距甲板最近） | 0.06 比特 |

**洞察**：`SexClass`（性别 × 舱等）这一交互特征，单独使用时信息增益仅约 0.05 比特，但与 `Sex_num` 和 `Pclass` 联合使用时，使模型捕捉到"一等舱女性几乎全部获救"而"三等舱男性几乎无人获救"的**差异化规律**——这正是交互特征的核心价值所在。
{% endblock %}

{% hint style="ideology" %}
🌏 **课程思政：特征工程与算法公平性**

特征工程的选择不是中立的技术决策，其背后隐含社会价值判断。以信用评分模型为例：若将"邮政编码"作为特征，模型可能学到种族或阶层的代理变量（proxy variable），在美国历史上的"红线划区"（redlining）政策中，邮政编码曾被用于系统性地拒绝少数族裔的贷款申请。即便工程师无意歧视，这一特征也会将历史偏见固化在模型中。

中国《人工智能伦理规范》（2021）明确提出 AI 系统应"消除偏见歧视"；欧盟 AI 法案（2024）要求高风险 AI 系统记录并评估敏感特征的使用情况。实践中，有以下几类处理方法：

1. **特征审计**：识别与受保护属性（种族、性别、年龄）高度相关的代理特征，评估其引入歧视的风险；
2. **因果推断**：区分特征与目标的因果关系和虚假相关，优先保留因果特征；
3. **公平感知特征选择**：在互信息或 F 统计量之外，加入公平性约束（如人口统计均等、机会均等）作为特征评分的附加条件。

特征工程的艺术，不仅在于提升模型精度，更在于确保被学习的规律是公平、可解释、对社会负责的规律。
{% endhint %}

{% hint style="frontier" %}
🚀 **前沿视野：自动特征工程与深度特征学习**

手工特征工程依赖领域专家知识，耗时费力。近年来，自动化方法正逐步重塑这一领域：

**AutoFE（Auto Feature Engineering）**：`featuretools`（Deep Feature Synthesis, DFS）通过自动遍历实体关系图，生成跨表的聚合特征（如"用户过去 7 天的平均购买金额"）。OpenFE（2023）进一步引入 LightGBM 快速评估特征重要性，在数百万候选特征中高效筛选。

**神经网络自动提取特征**：Transformer、ResNet 等深度模型在预训练阶段自动学习数据的层次化表示，本质上以端到端方式执行特征工程。BERT 的 [CLS] 向量、ImageNet 预训练 ResNet 的 FC 层激活，均是高度凝练的工程特征——只是从"手工设计"变成了"数据驱动习得"。

**因果特征选择**：传统相关性选择无法区分因果关系与伪相关。基于因果发现（PC 算法、LiNGAM）的特征选择方法试图仅保留目标变量的直接原因特征，在分布偏移环境下更具鲁棒性（Peters et al., 2014）。这一方向与工业界对模型可信性的需求高度契合，是近年 AutoML 研究的重要前沿。
{% endhint %}

## 本节小结

特征选择与特征工程是连接原始数据与高性能模型的关键桥梁。

| 方法 | 类别 | 核心机制 | 计算复杂度 | 适用场景 |
|------|------|---------|:---------:|---------|
| 方差阈值 | 过滤式 | $$\mathrm{Var}(X_j) \geq \theta$$ | $$O(nd)$$ | 快速剔除常数/低信息特征 |
| 互信息过滤 | 过滤式 | $$I(X_j; Y) \geq \theta$$ | $$O(nd\log n)$$ | 非线性特征与目标关联 |
| 卡方检验 | 过滤式 | $$\chi^2$$ 统计量 | $$O(nd)$$ | 离散特征与类别目标 |
| RFE / RFECV | 包裹式 | 迭代移除最弱特征 | $$O(d/\Delta \cdot v \cdot T_f)$$ | 中小维度、需最优子集 |
| Lasso | 嵌入式 | L1 正则化使系数稀疏 | $$O(ndK)$$ | 线性可加模型、高维稀疏 |
| 多项式扩展 | 特征构造 | $$\phi^{(d)}(\mathbf{x})$$，组合维度增长 | $$O(p^d / d!)$$ | 线性模型拟合非线性关系 |
| 分箱离散化 | 特征构造 | 连续值 → 区间编号 | $$O(nd)$$ | 非单调关系、异常值鲁棒性 |
| 交互特征 | 特征构造 | $$x_{ij} = x_i \cdot x_j$$ | $$O(p^2)$$ | 联合效应、领域驱动工程 |

**关键术语**

| 中文 | 英文 | 简要定义 |
|------|------|---------|
| 互信息 | Mutual Information | 两变量共享信息量，捕捉任意统计依赖 |
| 信息增益 | Information Gain | 条件熵减少量，$$H(Y)-H(Y|X)$$ |
| 递归特征消除 | Recursive Feature Elimination | 迭代移除最弱特征的包裹式方法 |
| Lasso 回归 | Lasso Regression | L1 正则化使系数稀疏，自动特征选择 |
| 正则化路径 | Regularization Path | 系数随正则化强度 $$\alpha$$ 变化的轨迹 |
| 交互特征 | Interaction Feature | 两个原始特征乘积，捕捉联合效应 |
| 数据泄露 | Data Leakage | 测试集信息污染训练特征的错误做法 |
