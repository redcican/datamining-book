# 2.6 缺失值处理策略

2004年，英国医学杂志（BMJ）的一项系统综述震动了临床试验界：对100项随机对照试验的审查发现，46%的试验存在结局数据缺失，而其中仅17%描述了任何处理缺失的方法——绝大多数研究者直接删除缺失记录，默认参与者退出（dropout）是随机事件。然而现实是，退出率最高的往往是病情最重、副作用最明显的患者：他们的退出本身就是疗效信息的一部分。这是经典的**非随机缺失（MNAR）**场景：删除这些记录不仅浪费数据，更系统性地高估了治疗效果。这一现象深刻揭示：缺失值的处理策略不是技术附录，而是决定结论是否可信的核心科学决策。

2.1节在概率框架（MCAR/MAR/MNAR）下识别并诊断了缺失机制；本节在此基础上，系统讨论与每种机制对应的处理策略——从直接删除、统计量单变量填补，到 KNN 与回归的多变量插值，再到能够正确传播不确定性的多重填补算法 MICE。每种方法都有其适用前提与偏差代价，理解这些代价是在真实工程场景中做出合理选择的前提。

{% hint style="info" %}

完成本节学习后，读者应能够：

1. 根据缺失率与缺失机制（MCAR/MAR/MNAR），在策略决策框架中选择合适的处理方法；
2. 证明在 MCAR 假设下均值填补对总体均值的无偏性，并说明其导致方差低估的原因；
3. 描述 KNN 填补与回归填补如何利用变量间的相关结构，改善 MAR 场景下的填补精度；

5. 在 sklearn Pipeline 中正确放置填补器，避免测试集信息泄露。

## 2.6.1 策略选择框架

缺失值处理没有万能的最优方案。合理选择依赖两个正交维度：**缺失率**（missing rate）决定了信息损失的严重程度，**缺失机制**（missing mechanism）决定了哪类方法在统计意义上是无偏的。

图 2.6.1 将这两个维度整合为一个决策流程。其核心逻辑如下：当缺失率超过 40% 时，填补的信息增益已十分有限，通常应优先考虑删除该特征或补充数据采集；在缺失率可接受的范围内，方法选择完全由缺失机制主导——MCAR 可用简单统计量填补，MAR 应采用利用变量间相关结构的多变量方法，MNAR 则需要借助领域知识和敏感性分析，任何基于观测数据的填补都无法消除其固有偏差。

## 2.6.2 删除法

当缺失率很低（< 5%）且机制为 MCAR 时，最简便可靠的方案是**删除含缺失值的记录**，直接在完整记录上建模。

{% block type="definition" label="2.21" title="完整案例分析（Complete Case Analysis, CCA）" %}
设数据矩阵 $$\mathbf{X} \in \mathbb{R}^{n \times d}$$ 中，令完整记录集合为：

$$
\mathcal{C} = \{i : R_{ij} = 1, \; \forall j = 1, \ldots, d\}
\tag{2.34}
$$

**完整案例分析**（又称 listwise deletion，逐行删除）仅在 $$\mathcal{C}$$ 上的子矩阵 $$\mathbf{X}_\mathcal{C}$$ 上进行后续分析。等价地，**成对删除**（pairwise deletion）在计算每对变量的协方差时分别使用各自的完整记录集合，可保留更多信息，但导致协方差矩阵非正定。
{% endblock %}

完整案例分析在理论上的有效性完全依赖于 MCAR 假设。

{% block type="theorem" label="2.6" title="MCAR 下完整案例分析的无偏性" %}
设特征 $$j$$ 的真实总体均值为 $$\mu_j$$，缺失机制为 MCAR，即 $$P(R_{ij} = 1 \,|\, \mathbf{X}) = P(R_{ij} = 1) = p$$（与数据值无关）。则完整案例样本均值：

$$
\hat{\mu}_j^{\mathrm{CCA}} = \frac{1}{|\mathcal{C}|} \sum_{i \in \mathcal{C}} x_{ij}
\tag{2.35}
$$

满足 $$\mathbb{E}[\hat{\mu}_j^{\mathrm{CCA}}] = \mu_j$$，即完整案例估计量无偏；但其方差为 $$\mathrm{Var}(\hat{\mu}_j^{\mathrm{CCA}}) = \sigma_j^2 / (np)$$，相比全样本估计量增大了 $$1/p$$ 倍（统计效能损失）。
{% endblock %}

{% hint style="proof" %}
**证明**

由 MCAR 条件，$$R_{ij}$$ 独立于 $$x_{ij}$$，故：

$$
\mathbb{E}[x_{ij} \,|\, R_{ij} = 1] = \mathbb{E}[x_{ij}] = \mu_j
$$

完整案例集合 $$\mathcal{C}$$ 中每条记录均为独立同分布的完整观测，因此：

$$
\mathbb{E}[\hat{\mu}_j^{\mathrm{CCA}}] = \mu_j \qquad \square
$$

方差部分：$$|\mathcal{C}|$$ 的期望值为 $$np$$，由大数定律 $$|\mathcal{C}| / n \xrightarrow{p} p$$，故 $$\mathrm{Var}(\hat{\mu}_j^{\mathrm{CCA}}) \approx \sigma_j^2 / (np)$$。
{% endhint %}

{% hint style="caution" %}
⚠️ **MCAR 之外：删除引入偏差**

若缺失机制为 MAR 或 MNAR，完整案例不再是总体的随机样本，$$\mathbb{E}[x_{ij} \,|\, R_{ij}=1] \neq \mu_j$$，任何基于 $$\mathcal{C}$$ 的估计量均存在系统性偏差。以血压数据为例（见图 2.1.2 中 MNAR 情形）：高血压患者拒绝测量，完整案例中血压值系统性偏低，删除后的均值估计严重低估真实均值。
{% endhint %}

## 2.6.3 统计量单变量填补

当缺失率在 5%–40% 之间、机制为 MCAR 时，更常见的做法是**保留所有记录，将缺失值替换为从观测数据计算的统计量**。这类方法统称为单变量填补，因为每列独立填补，不利用其他列的信息。

{% block type="definition" label="2.22" title="统计量单变量填补（Univariate Statistical Imputation）" %}
设特征 $$j$$ 的观测值集合为 $$\{x_{ij} : R_{ij} = 1\}$$，定义以下三种填补值：

**均值填补**（Mean Imputation）：

$$
\hat{x}_j^{\mathrm{mean}} = \bar{x}_j^{\mathrm{obs}} = \frac{\sum_{i: R_{ij}=1} x_{ij}}{n_j^{\mathrm{obs}}}
\tag{2.36}
$$

**中位数填补**（Median Imputation）：

$$
\hat{x}_j^{\mathrm{med}} = \mathrm{Median}\{x_{ij} : R_{ij}=1\}
\tag{2.37}
$$

**众数填补**（Mode Imputation）：$$\hat{x}_j^{\mathrm{mode}} = \arg\max_v \sum_{i: R_{ij}=1} \mathbf{1}[x_{ij}=v]$$，主要用于类别变量。

填补后的完整矩阵为：$$\tilde{x}_{ij} = x_{ij}$$ 若 $$R_{ij}=1$$，否则 $$\tilde{x}_{ij} = \hat{x}_j^{\star}$$（$$\star$$ 为 mean/med/mode 之一）。
{% endblock %}

### 均值填补的偏差分析

均值填补在 MCAR 下对总体均值是无偏的，但会低估方差，这是一个经常被忽视的缺陷。

{% block type="theorem" label="2.7" title="均值填补的无偏性与方差低估" %}
设特征 $$j$$ 的总体均值为 $$\mu_j$$、方差为 $$\sigma_j^2$$，缺失机制为 MCAR，观测率为 $$p = n_j^{\mathrm{obs}}/n$$。均值填补后完整数据集的样本均值 $$\bar{\tilde{x}}_j$$ 满足：

**（1）无偏性**：$$\mathbb{E}[\bar{\tilde{x}}_j] = \mu_j$$。

**（2）方差低估**：填补后样本方差 $$\hat{\sigma}_{\tilde{j}}^2$$ 的期望满足：

$$
\mathbb{E}[\hat{\sigma}_{\tilde{j}}^2] = p \cdot \sigma_j^2 < \sigma_j^2
\tag{2.38}
$$

即均值填补将方差压缩为真实方差的 $$p$$ 倍，导致填补后数据的变异性系统性低估。
{% endblock %}

{% hint style="proof" %}
**证明**

**无偏性**：填补后均值为 $$\bar{\tilde{x}}_j = \frac{1}{n}[\sum_{i:R_{ij}=1} x_{ij} + (n - n_j^{\mathrm{obs}}) \cdot \bar{x}_j^{\mathrm{obs}}]$$。化简得 $$\bar{\tilde{x}}_j = \bar{x}_j^{\mathrm{obs}}$$。在 MCAR 下，$$\mathbb{E}[\bar{x}_j^{\mathrm{obs}}] = \mu_j$$，故无偏。$$\square$$

**方差低估**：填补后方差由两部分组成：观测值对均值的偏差之和，加上填补值（均等于 $$\bar{x}_j^{\mathrm{obs}}$$）对均值的偏差（恒为0）之和。因此：

$$
\hat{\sigma}_{\tilde{j}}^2 = \frac{1}{n-1}\sum_{i=1}^n (\tilde{x}_{ij} - \bar{\tilde{x}}_j)^2 = \frac{1}{n-1}\sum_{i:R_{ij}=1} (x_{ij} - \bar{x}_j^{\mathrm{obs}})^2
$$

期望值约为 $$\frac{n_j^{\mathrm{obs}}}{n} \cdot \sigma_j^2 = p \cdot \sigma_j^2 < \sigma_j^2$$。$$\square$$
{% endhint %}

**MAR/MNAR 下的偏差**：若机制为 MAR，则 $$\mathbb{E}[x_j \,|\, R_{ij}=1] \neq \mu_j$$（缺失倾向于某类观测值），$$\bar{x}_j^{\mathrm{obs}}$$ 本身是有偏的，从而均值填补同时引入均值偏差和方差低估。以图 2.1.2 的 MAR 场景为例：年龄 < 35 的受访者更可能不填写血压值（此类人血压普遍较低），观测均值偏高，用其填补将系统性高估总体均值。

图 2.6.2 直观展示了均值填补（面板 b）对双峰分布的破坏：所有缺失值被替换为约 40（两峰的均值），在原本空旷的中间区域制造了一个人为的峰，完全歪曲了原始分布形状。KNN 与迭代填补（MICE）则基本保真了双峰结构。

## 2.6.4 多变量插值填补

单变量填补的根本局限在于**忽视了特征之间的相关结构**：若特征 $$j$$（血压）与特征 $$k$$（年龄）高度相关，且 $$k$$ 是完整观测的，那么已知 $$x_{ik}$$ 可以为 $$x_{ij}$$ 的缺失值提供远比全局均值更精确的预测。多变量插值填补方法正是利用这一相关结构，将填补问题转化为预测问题。

### 回归填补

最直接的多变量方法是以其余特征为预测变量，对缺失特征建立回归模型。

{% block type="definition" label="2.23" title="（随机）回归填补（Regression Imputation）" %}
对特征 $$j$$，以所有其余完整（或已填补）特征构成矩阵 $$\mathbf{X}_{-j}$$，在观测样本上拟合线性回归：

$$
\hat{\boldsymbol{\beta}}_j = \arg\min_{\boldsymbol{\beta}} \sum_{i: R_{ij}=1} (x_{ij} - \boldsymbol{\beta}^T \mathbf{x}_{i,-j})^2
\tag{2.39}
$$

对缺失样本 $$i$$（$$R_{ij}=0$$），预测填补值为 $$\hat{x}_{ij} = \hat{\boldsymbol{\beta}}_j^T \mathbf{x}_{i,-j}$$（确定性回归填补）。

**随机回归填补**（Stochastic Regression Imputation）在预测值基础上加入随机扰动：

$$
\tilde{x}_{ij} = \hat{\boldsymbol{\beta}}_j^T \mathbf{x}_{i,-j} + \varepsilon_i, \qquad \varepsilon_i \sim \mathcal{N}(0, \hat{\sigma}_j^2)
\tag{2.40}
$$

其中 $$\hat{\sigma}_j^2$$ 为回归残差方差。随机扰动项恢复了填补值的随机性，避免了确定性填补导致的相关性低估。
{% endblock %}

确定性回归填补虽消除了均值偏差，但仍存在方差低估：所有缺失值被预测到一条回归直线上，变异性小于真实数据。随机回归填补通过添加服从残差分布的噪声部分解决这一问题，但仍低估了回归参数本身的不确定性——这一问题需要多重填补（2.6.5节）才能彻底解决。

### KNN 填补

另一类利用相关结构的方法通过**局部相似性**代替全局回归，其核心思想是：缺失样本的特征值应与最相似的已观测样本接近。

{% block type="definition" label="2.24" title="KNN 填补（K-Nearest Neighbor Imputation）" %}
设样本 $$i$$ 在特征 $$j$$ 处缺失（$$R_{ij}=0$$），在完整特征的子空间（仅使用 $$R_{ij'}=1$$ 的维度）中，定义 $$i$$ 与其他样本的距离：

$$
d(i, k) = \left\|\mathbf{x}_{i, \mathcal{O}} - \mathbf{x}_{k, \mathcal{O}}\right\|_2, \qquad \mathcal{O} = \{j' : R_{ij'}=1\}
\tag{2.41}
$$

取 $$K$$ 个距离最小的已观测邻居 $$\mathcal{N}_K(i)$$（要求邻居在特征 $$j$$ 处有观测值），按距离反比加权填补：

$$
\tilde{x}_{ij} = \frac{\sum_{k \in \mathcal{N}_K(i)} w_k \cdot x_{kj}}{\sum_{k \in \mathcal{N}_K(i)} w_k}, \qquad w_k = \frac{1}{d(i,k) + \varepsilon}
\tag{2.42}
$$

其中 $$\varepsilon > 0$$ 为防止除零的小常数。等权版本（$$w_k = 1$$）取邻居均值。
{% endblock %}

{% block type="example" label="2.11" title="KNN 填补手动计算（K=3）" %}
**数据**（4 个样本，特征 $$x_1$$：年龄，$$x_2$$：收入，$$x_2$$ 在样本 3 处缺失）：

| 样本 | $$x_1$$（年龄） | $$x_2$$（收入） |
|------|--------------|--------------|
| 1    | 25           | 30,000       |
| 2    | 28           | 35,000       |
| 3    | 27           | **?**        |
| 4    | 50           | 80,000       |

**步骤**：在 $$x_1$$ 子空间中计算样本 3 与其他样本的距离：

- $$d(3,1) = |27-25| = 2$$
- $$d(3,2) = |27-28| = 1$$
- $$d(3,4) = |27-50| = 23$$

取 $$K=3$$：邻居为 $$\{1, 2, 4\}$$，距离反比权重：

$$w_1 = 1/2 = 0.50, \quad w_2 = 1/1 = 1.00, \quad w_4 = 1/23 \approx 0.043$$

归一化后填补值：

$$\tilde{x}_{3,2} = \frac{0.50 \times 30000 + 1.00 \times 35000 + 0.043 \times 80000}{0.50+1.00+0.043} \approx \frac{53440}{1.543} \approx 34634 \text{ 元}$$

直觉验证：样本 3 年龄 27 最接近样本 2（年龄 28），填补结果偏向 35,000，合理。
{% endblock %}

KNN 填补的主要优势是模型假设极弱（不需要线性关系），对非线性相关也有效。缺点是计算代价为 $$O(n^2 d)$$（需遍历所有样本对），超参数 $$K$$ 需通过交叉验证选择，且对高维数据中的**维数灾难**（见 2.3.1节）敏感。

## 2.6.5 多重填补：MICE

上述所有单次填补方法共享一个根本缺陷：**填补值被当作真实值处理**，下游分析（模型、置信区间）完全忽略了填补本身引入的不确定性。以回归分析为例：若将均值填补后的数据代入最小二乘回归，标准误将被低估，$$t$$ 统计量将虚高，I 类错误率将膨胀。

**多重填补**（Multiple Imputation, MI）的核心思想是：不给出单一的"最佳填补值"，而是生成 $$M$$ 套合理的完整数据集，分别分析，再按 Rubin 合并规则（Rubin's Rules）汇总结果，从而正确量化填补不确定性。

{% block type="algorithm" label="2.6" title="链式方程多重填补（MICE，van Buuren & Groothuis-Oudshoorn, 2011）" %}
**输入**：不完整数据矩阵 $$\mathbf{X} \in \mathbb{R}^{n \times d}$$；填补数量 $$M$$；迭代轮次 $$T$$

**输出**：$$M$$ 套完整数据集 $$\tilde{\mathbf{X}}^{(1)}, \ldots, \tilde{\mathbf{X}}^{(M)}$$

**对每次填补 $$m = 1, \ldots, M$$：**

```text
1. 初始化：对每列 j，将缺失值随机抽取该列观测值填充
2. 对每轮迭代 t = 1, …, T：
   对每列 j（按缺失量由少到多排序）：
     a. 令 X_{-j}  = 当前其余各列（已观测 + 上轮填补值）
     b. 在 X_{-j} 和 x_j 的观测部分拟合回归模型 f_j
     c. 从 f_j 的后验预测分布抽取参数（贝叶斯抽样）
     d. 以抽取的参数生成缺失位置的预测值并填入
3. 记录当前完整矩阵为 X̃^(m)
```

**合并阶段（Rubin's Rules）**：对任意目标统计量 $$\theta$$（如回归系数），设 $$\hat{\theta}^{(m)}$$、$$V^{(m)}$$ 分别为第 $$m$$ 套数据上的估计量和其方差：

$$
\bar{\theta} = \frac{1}{M}\sum_{m=1}^M \hat{\theta}^{(m)}, \quad
\bar{V} = \frac{1}{M}\sum_{m=1}^M V^{(m)}, \quad
B = \frac{1}{M-1}\sum_{m=1}^M (\hat{\theta}^{(m)} - \bar{\theta})^2
\tag{2.43}
$$

总方差（Total Variance）：

$$
T = \bar{V} + \left(1 + \frac{1}{M}\right) B
\tag{2.44}
$$

其中 $$\bar{V}$$ 为组内方差（sampling uncertainty），$$B$$ 为组间方差（imputation uncertainty），$$(1+1/M)B$$ 项的存在正是 MI 相对单一填补的核心优势——它显式量化了"因为数据缺失，我们对填补值本身也不确定"这一来源的变异。
{% endblock %}

{% block type="example" label="2.12" title="MICE 合并规则的数值示例" %}
**场景**：对某临床数据集进行 $$M=5$$ 次多重填补，估计某药物的平均疗效 $$\theta$$（血压降低量，单位 mmHg）。五套完整数据的估计结果如下：

| $$m$$ | $$\hat{\theta}^{(m)}$$ | $$V^{(m)}$$（方差） |
|-----|---------------------|------------------|
| 1   | 8.2                 | 1.44             |
| 2   | 9.1                 | 1.69             |
| 3   | 7.8                 | 1.21             |
| 4   | 8.6                 | 1.56             |
| 5   | 9.3                 | 1.96             |

**合并**：$$\bar{\theta} = (8.2+9.1+7.8+8.6+9.3)/5 = 8.60$$ mmHg

$$\bar{V} = (1.44+1.69+1.21+1.56+1.96)/5 = 1.572$$

$$B = [(8.2-8.6)^2+(9.1-8.6)^2+(7.8-8.6)^2+(8.6-8.6)^2+(9.3-8.6)^2] / 4 = 0.325$$

$$T = 1.572 + (1+1/5) \times 0.325 = 1.572 + 0.390 = 1.962$$

合并标准误 $$= \sqrt{1.962} = 1.401$$，对应 95% 置信区间：$$8.60 \pm 1.96 \times 1.401 = [5.85, 11.35]$$ mmHg。

若直接对单次填补数据计算（以 $$m=1$$ 为例）：点估计 8.2，SE $$= \sqrt{1.44} = 1.200$$，CI $$= [5.85, 10.55]$$——区间偏窄，因为忽略了组间方差 $$B$$ 所代表的填补不确定性。
{% endblock %}

实践建议：$$M$$ 通常取 5–20（缺失率越高需要越大的 $$M$$）；迭代轮次 $$T$$ 通常取 10–20（以收敛诊断图确认）；MICE 要求 MAR 假设成立，在 MNAR 下仍有偏差，但提供了比单一填补更诚实的不确定性量化。

## 2.6.6 工程实践：Pipeline 中的填补器

{% hint style="engineering" %}
🔧 **工程实践：填补器必须在 Pipeline 内，绝不在 Pipeline 外**

缺失值填补中最常见的数据泄露错误：**在划分训练/测试集之前**（或在 Pipeline 外）对全量数据拟合填补器，再分别变换训练集和测试集。这使测试集的缺失值被训练集的统计量（如均值、回归系数）"污染"，导致模型性能虚高。

```python
# ============================================================
# 代码 2.6：填补器与 Pipeline 的正确 vs 错误用法对比
# 数据集：合成数据（n=500，3 个特征，25% 随机缺失）
# 依赖库：numpy, scikit-learn
# ============================================================
# --- 1. 生成含缺失值的数据 ---
import numpy as np
from sklearn.datasets import make_regression
from sklearn.model_selection import cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.impute import SimpleImputer
from sklearn.linear_model import Ridge
rng = np.random.default_rng(0)
X, y = make_regression(n_samples=500, n_features=3, noise=10, random_state=0)
mask = rng.random(X.shape) < 0.25
X[mask] = np.nan
# --- 2. ❌ 错误做法：先在全量数据拟合填补器，再评估 ---
# imputer = SimpleImputer()
# X_filled = imputer.fit_transform(X)  # 泄露：测试集信息进入了填补统计量
# scores_wrong = cross_val_score(Ridge(), X_filled, y, cv=5)
# --- 3. ✅ 正确做法：填补器作为 Pipeline 的第一步 ---
pipe = Pipeline([
    ("imputer", SimpleImputer(strategy="mean")),
    ("model",   Ridge()),
])
scores = cross_val_score(pipe, X, y, cv=5, scoring="r2")
print(f"正确 Pipeline R²：{scores.mean():.3f} ± {scores.std():.3f}")
```

**原理**：`cross_val_score` 在每折内先在训练集上调用 `pipe.fit()`（触发 `imputer.fit_transform` 仅使用训练折数据），再对测试折调用 `pipe.transform()`（使用训练折拟合的均值填补测试折），完全隔离信息流。
{% endhint %}

## 2.6.7 综合案例：California Housing 缺失值填补效果量化

{% block type="case" label="2.6" title="California Housing：三种填补策略的 RMSE 对比" %}
**数据集**：加利福尼亚州房价数据（sklearn 内置，20,640 个样本，8 个数值特征）。

**挖掘目标**：人工注入 MAR 缺失后，比较均值填补、KNN 填补、迭代填补（MICE）三种策略对下游岭回归房价预测精度的影响，以 5 折交叉验证 RMSE 量化各策略的填补质量。

```python
# ============================================================
# 代码 2.6：California Housing 缺失值填补效果量化评估
# 数据集：California Housing（sklearn 内置）
# 依赖库：numpy, scikit-learn
# ============================================================
# --- 1. 加载数据与注入 MAR 缺失 ---
import numpy as np
from sklearn.datasets import fetch_california_housing
from sklearn.model_selection import cross_val_score
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler
from sklearn.impute import SimpleImputer, KNNImputer
from sklearn.experimental import enable_iterative_imputer  # noqa: F401（当前版本需要）
from sklearn.impute import IterativeImputer
from sklearn.linear_model import Ridge
X_raw, y = fetch_california_housing(return_X_y=True)
rng = np.random.default_rng(2024)
X = X_raw.copy().astype(float)
# MAR 设计：MedInc（特征0）缺失概率与 HouseAge（特征1）负相关
# 模拟"新房业主收入信息更完整"的现实场景
house_age = X[:, 1]
miss_prob = 0.05 + 0.35 * (house_age - house_age.min()) / (house_age.max() - house_age.min())
miss_mask = rng.random(len(X)) < miss_prob
X[miss_mask, 0] = np.nan
print(f"MedInc 缺失率：{miss_mask.mean():.1%}（设计为 MAR）")
# --- 2. 构建三种填补 Pipeline ---
strategies = {
    "均值填补（MCAR 基线）": Pipeline([
        ("imp", SimpleImputer(strategy="mean")),
        ("scl", StandardScaler()),
        ("reg", Ridge()),
    ]),
    "KNN 填补（k=5）": Pipeline([
        ("imp", KNNImputer(n_neighbors=5, weights="distance")),
        ("scl", StandardScaler()),
        ("reg", Ridge()),
    ]),
    "迭代填补（MICE，10次迭代）": Pipeline([
        ("imp", IterativeImputer(max_iter=10, random_state=42)),
        ("scl", StandardScaler()),
        ("reg", Ridge()),
    ]),
}
# --- 3. 5 折交叉验证 RMSE 对比 ---
for name, pipe in strategies.items():
    scores = cross_val_score(pipe, X, y, cv=5,
                             scoring="neg_root_mean_squared_error")
    rmse_mean = -scores.mean()
    rmse_std  = scores.std()
    print(f"{name}：RMSE = {rmse_mean:.4f} ± {rmse_std:.4f}")
```

**结果解读**：MAR 场景下，KNN 与迭代填补利用 HouseAge 与 MedInc 之间的相关性（年龄越大的房屋所在街区收入越低），为缺失的 MedInc 值提供比全局均值更精确的估计，从而在下游房价预测中获得更低的 RMSE。均值填补忽视了这一相关结构，将所有缺失值填充为同一常数，损失了预测信息。
{% endblock %}

## 本节小结

| 方法 | 适用机制 | 偏差（均值） | 偏差（方差） | 计算代价 | sklearn API |
|------|---------|------------|------------|---------|-------------|
| 完整案例分析（CCA） | MCAR，缺失率 < 5% | 无 | 无（但 $$n$$ 减少） | $$O(1)$$ | — |
| 均值/中位数填补 | MCAR | 无（均值无偏） | 低估（压缩为 $$p$$ 倍） | $$O(n)$$ | `SimpleImputer` |
| 众数填补 | MCAR，类别变量 | 无 | 低估 | $$O(n)$$ | `SimpleImputer` |
| KNN 填补 | MAR | 低（利用相关结构） | 有一定低估 | $$O(n^2 d)$$ | `KNNImputer` |
| 回归填补（确定性） | MAR | 低 | 低估 | $$O(nd^2)$$ | — |
| 随机回归填补 | MAR | 低 | 近似正确 | $$O(nd^2)$$ | — |
| MICE 多重填补 | MAR | 低 | 正确量化不确定性 | $$O(MTnd^2)$$ | `IterativeImputer` |
| 任何方法 | MNAR | **不可消除** | — | — | 需领域知识 |

核心结论：**缺失机制决定方法的上限，缺失率决定方法的可行性**。在 MCAR 下，均值填补对点估计无偏但低估方差，推断统计需用 MICE；在 MAR 下，多变量方法（KNN/回归/MICE）优于单变量方法；MNAR 是无解约束，任何算法只能缓解而无法消除偏差。
