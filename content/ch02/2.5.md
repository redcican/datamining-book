# 2.5 数据标准化与归一化

一则广为流传的竞赛复盘这样写道：某队伍在 KNN 分类任务中反复调参却收效甚微，最终发现症结出人意料——数据集中"年收入"特征以元为单位（取值 3 万至 50 万），而"年龄"特征以岁为单位（取值 20 至 60）。两者共同参与欧氏距离计算时，年收入的数值波动完全淹没了年龄的贡献：哪怕将年龄权重设为无穷大，最终距离几乎只由年收入决定。问题的根源不在模型，而在**尺度**（scale）——这正是数据标准化所要解决的核心问题。

上一节我们学习了如何从已有属性中选择和构造信息密集的特征；本节关注的是在此之前（或并行进行的）一个同样关键的预处理步骤：**消除特征量纲和尺度的差异**，使所有特征站在同一起跑线上参与模型计算。标准化不仅影响基于距离的算法（KNN、SVM、K-means），还深刻影响梯度下降的收敛速度与数值稳定性。

## 2.5.1 为什么需要标准化？

特征尺度问题在两类场景下尤为突出：

**场景一：基于距离的算法**。欧氏距离 $$\|\mathbf{x}_i - \mathbf{x}_j\|_2 = \sqrt{\sum_k (x_{ik} - x_{jk})^2}$$ 对每个特征的贡献是加法无偏见的。若特征 $$k$$ 的数值范围是特征 $$l$$ 的 $$M$$ 倍，则特征 $$k$$ 对距离的贡献约为特征 $$l$$ 的 $$M^2$$ 倍（平方效应）。KNN、K-means、SVM（RBF 核）以及 PCA 都受此影响。

**场景二：基于梯度的优化**。线性回归、逻辑回归与神经网络通过梯度下降最小化损失函数。当不同参数对应的特征尺度差异巨大时，损失曲面的等高线变成细长椭圆，梯度方向几乎垂直于最速下降方向，导致学习率难以选取且收敛极慢（图 2.5.2b）。

{% endhint %}

## 2.5.2 基本标准化方法

三种最基本的线性标准化方法均遵循"中心化 + 缩放"的共同框架：先减去某个**参考位置**，再除以某个**参考尺度**。三者的区别在于参考统计量的选取，这决定了各方法对分布形状和离群点的敏感程度。

{% block type="definition" label="2.17" title="Min-Max 归一化（Min-Max Normalization）" %}
给定特征向量 $$\mathbf{x} = (x_1, x_2, \ldots, x_n)^T$$，设 $$x_{\min} = \min_i x_i$$，$$x_{\max} = \max_i x_i$$，则 **Min-Max 归一化**将每个值线性映射到区间 $$[a, b]$$（通常取 $$[0,1]$$）：

$$
x'_i = \frac{x_i - x_{\min}}{x_{\max} - x_{\min}} \cdot (b - a) + a
\tag{2.30}
$$

当 $$a=0, b=1$$ 时简化为 $$x'_i = \dfrac{x_i - x_{\min}}{x_{\max} - x_{\min}}$$。变换为线性映射：原始数据的等比间距在变换后得以保留。**缺点**：对离群点极度敏感——单个极端大值使 $$x_{\max}$$ 急剧增大，将其余正常数据挤压到区间左侧极小的范围内，彼此几乎无法区分（参见图 2.5.1 第二列直方图）。
{% endblock %}

Min-Max 的值域约束在某些场景（如图像像素 $$[0,1]$$、神经网络激活函数要求的输入范围）有直接意义，但其对极端值的高度敏感是工程实践中的主要障碍。为此，更常用的方案改以**全局分布的均值和标准差**作为参考，将每个观测值描述为"距离典型值有多少个典型波动幅度"——这正是 Z-score 标准化的思路。

{% block type="definition" label="2.18" title="Z-score 标准化（Standardization）" %}
设特征的总体均值为 $$\mu$$、标准差为 $$\sigma$$（实践中以样本统计量估计），**Z-score 标准化**（或称**均值标准化**）定义为：

$$
z_i = \frac{x_i - \mu}{\sigma}
\tag{2.31}
$$

变换后的特征 $$z$$ 满足：$$\mathrm{E}[z] = 0$$，$$\mathrm{Var}(z) = 1$$（见定理 2.5）。变换结果无固定上下界；若原始分布近似正态，则 $$|z| > 3$$ 的点约占 0.27%，可作为离群点检测的简单准则。对离群点的敏感性低于 Min-Max，但均值和标准差作为矩统计量，在偏斜分布中仍会受少数极端值拉偏。
{% endblock %}

Z-score 是最广泛使用的默认选择，适合大多数近似对称的分布。然而均值和标准差本质上是所有样本的加权平均，少数极端值对其影响与数量级成正比。对于偏斜显著或含较多离群点的分布（如收入、交易金额、网站访问量），更合理的做法是改用对极端值免疫的**位置统计量**：中位数和四分位距，这两个统计量各自的 breakdown point 为 25%，即数据中多达 25% 的观测值被替换为任意极端值时，统计量仍保持稳定。

{% block type="definition" label="2.19" title="鲁棒缩放（Robust Scaler）" %}
以**中位数**（median）替代均值、以**四分位距**（Interquartile Range, IQR = $$Q_3 - Q_1$$）替代标准差：

$$
r_i = \frac{x_i - Q_2}{Q_3 - Q_1}
\tag{2.32}
$$

其中 $$Q_1, Q_2, Q_3$$ 分别为第 25、50、75 百分位数。由于中位数和 IQR 仅取决于数据的顺序统计量而非数值大小，鲁棒缩放对偏斜分布和含大量离群点的场景效果最好（图 2.5.1 第四列）。
{% endblock %}

上述三种方法的"中心化 + 缩放"框架背后，Z-score 具有一条独立于原始分布形态的严格数学性质：无论 $$\mu$$ 和 $$\sigma$$ 取何值，经过式 (2.31) 变换后，样本的**均值精确为零、方差精确为 1**。这一性质是主成分分析（PCA）归一化、白化变换（whitening）以及深度学习 Batch Normalization 的共同数学基础，由定理 2.5 给出严格证明。

{% block type="theorem" label="2.5" title="Z-score 标准化的均值与方差" %}
设 $$x_1, x_2, \ldots, x_n$$ 为 $$n$$ 个观测值，$$\bar{x} = \frac{1}{n}\sum_i x_i$$，$$s = \sqrt{\frac{1}{n}\sum_i (x_i - \bar{x})^2}$$（总体标准差）。令 $$z_i = (x_i - \bar{x}) / s$$，则：

$$\frac{1}{n}\sum_{i=1}^n z_i = 0, \qquad \frac{1}{n}\sum_{i=1}^n z_i^2 = 1$$

即变换后样本均值恒为 0，样本方差恒为 1（无偏样本方差为 $$\frac{n}{n-1}$$，但对标准差定义同样成立）。
{% endblock %}

{% hint style="proof" %}
**证明**

对均值：
$$\frac{1}{n}\sum_{i=1}^n z_i = \frac{1}{n}\sum_{i=1}^n \frac{x_i - \bar{x}}{s} = \frac{1}{s}\left(\frac{1}{n}\sum_{i=1}^n x_i - \bar{x}\right) = \frac{\bar{x} - \bar{x}}{s} = 0$$

对方差：
$$\frac{1}{n}\sum_{i=1}^n z_i^2 = \frac{1}{n}\sum_{i=1}^n \frac{(x_i - \bar{x})^2}{s^2} = \frac{s^2}{s^2} = 1$$

其中用到了 $$s^2 = \frac{1}{n}\sum_i (x_i - \bar{x})^2$$ 的定义。$$\square$$
{% endhint %}

### 方法对比

| 方法 | 公式 | 值域 | 对离群点敏感性 | 适用场景 |
|------|------|------|----------------|----------|
| Min-Max | $$(x - x_{\min})/(x_{\max}-x_{\min})$$ | $$[0,1]$$ | **极高** | 无离群点，值域有意义（如像素强度） |
| Z-score | $$(x - \mu)/\sigma$$ | 无界 | 中等 | 近似正态，通用首选 |
| Robust Scaler | $$(x - Q_2)/(Q_3-Q_1)$$ | 无界 | **低** | 偏斜分布，含离群点 |
| Max-Abs | $$x / |x|_{\max}$$ | $$[-1,1]$$ | 极高 | 稀疏矩阵（不引入均值偏移） |

## 2.5.3 Mahalanobis 距离：消除特征相关性

Z-score 标准化消除了各特征的**尺度差异**，但如果特征之间存在线性相关（如"身高"和"腿长"），欧氏距离仍然会高估相关方向上的相似性。Mahalanobis 距离同时处理**尺度**和**相关性**两个问题。

{% block type="definition" label="2.20" title="Mahalanobis 距离（Mahalanobis Distance）" %}
设 $$d$$ 维特征空间中，数据的协方差矩阵为 $$\boldsymbol{\Sigma} \in \mathbb{R}^{d \times d}$$（正定）。点 $$\mathbf{x}$$ 与均值 $$\boldsymbol{\mu}$$ 的 **Mahalanobis 距离**定义为：

$$
D_M(\mathbf{x}) = \sqrt{(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})}
\tag{2.33}
$$

两点 $$\mathbf{x}$$ 与 $$\mathbf{y}$$ 之间的广义形式为 $$D_M(\mathbf{x}, \mathbf{y}) = \sqrt{(\mathbf{x}-\mathbf{y})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x}-\mathbf{y})}$$。

**性质**：
1. 当 $$\boldsymbol{\Sigma} = \mathbf{I}$$（单位矩阵，各特征独立且方差为 1）时，退化为欧氏距离；
2. 等距离面为以 $$\boldsymbol{\mu}$$ 为中心、沿数据主方向延伸的**椭球面**；
3. 在多元正态假设下，$$D_M^2(\mathbf{x}) \sim \chi^2(d)$$，可用于统计检验离群点。
{% endblock %}

**几何直觉**（图 2.5.2c）：Mahalanobis 距离等价于先对数据做"白化"变换（whitening）$$\mathbf{y} = \boldsymbol{\Sigma}^{-1/2}(\mathbf{x} - \boldsymbol{\mu})$$，然后在变换后空间计算欧氏距离。白化后各方向方差相等、各特征不相关，等距离面变为球面。

{% hint style="caution" %}
⚠️ **计算代价**：Mahalanobis 距离需要估计 $$d \times d$$ 协方差矩阵并求逆，计算复杂度为 $$O(d^2 n)$$（估计）$$+ O(d^3)$$（求逆）。当 $$d > n$$ 时协方差矩阵奇异，需要正则化（如 $$\boldsymbol{\Sigma} + \epsilon \mathbf{I}$$）或改用 PCA 降维后再计算。
{% endhint %}

## 2.5.5 手动计算示例

{% block type="example" label="2.10" title="三种方法手动计算：房价数据" %}
设某城市 6 套房屋数据：面积（$$x$$，㎡）为 $$\{60, 75, 80, 90, 110, 200\}$$，其中 200㎡ 为豪宅（离群点）。

**步骤 1：计算统计量**

- $$x_{\min} = 60$$，$$x_{\max} = 200$$，范围 = 140
- $$\bar{x} = (60+75+80+90+110+200)/6 = 102.5$$
- $$s = \sqrt{\frac{1}{6}\sum(x_i - 102.5)^2} \approx 47.2$$
- $$Q_1 = 71.25$$，$$Q_2 = 85$$，$$Q_3 = 105$$，IQR = 33.75

**步骤 2：三种方法变换（以 200㎡ 为例）**

| 方法 | 计算过程 | 变换值 |
|------|----------|--------|
| Min-Max | $$(200-60)/(200-60)$$ | 1.000 |
| Z-score | $$(200-102.5)/47.2$$ | **+2.07** |
| Robust | $$(200-85)/33.75$$ | **+3.41** |

**分析**：Min-Max 将 200㎡ 映射为 1.0（边界），其余 5 套房均分布在 0 至 0.36 之间（大量信息被压缩）。Z-score 显示 200㎡ 约为均值 2.1 个标准差处。鲁棒缩放将其标记为 3.4 个 IQR 处，因 IQR 更小（仅 33.75 vs $$s=47.2$$），对正常值的区分度更高。

**结论**：对含离群点的右偏分布，推荐鲁棒缩放；若下游算法对离群点鲁棒（如树模型），也可先移除离群点再用 Z-score。
{% endblock %}

## 2.5.6 标准化方法选择指南

面对具体的数据集和下游任务，如何在 Min-Max、Z-score、鲁棒缩放、Mahalanobis 等方法中作出合理选择？关键决策因素包括：模型对尺度的敏感程度、数据分布的对称性与离群点比例、以及值域是否具有业务语义。算法 2.5 将这些准则整合为一棵决策树，提供系统性的选择流程。

{% block type="algorithm" label="2.5" title="标准化方法选择决策流程" %}
**输入**：特征向量 $$\mathbf{x}$$，下游模型类型，分布特征
**输出**：推荐的标准化方法

```text
1. IF 下游模型为树模型（决策树、随机森林、XGBoost）：
       标准化通常无效，可跳过；但 L1/L2 正则化存在时需标准化
2. ELSE IF 数据含显著离群点（|z|>3 的点 > 5%）：
   a. IF 值域语义重要（如图像像素 [0,255]）：
          使用 Min-Max，需先剪除离群点
   b. ELSE：使用鲁棒缩放（RobustScaler）
3. ELSE IF 数据近似对称/正态：
       使用 Z-score（StandardScaler）—— 通用默认选择
4. ELSE IF 数据为稀疏矩阵（大量零值）：
       使用 MaxAbsScaler（保留稀疏结构，不引入均值偏移）
5. IF 多特征存在线性相关且用于基于距离的算法：
       考虑 Mahalanobis 距离或先 PCA 白化再计算欧氏距离
6. RETURN 选定的 Scaler
```
{% endblock %}

## 2.5.7 案例：波士顿房价中的标准化影响

标准化的实际效果因数据和模型而异，理论分析固然重要，量化实验才能给出可信的结论。以下案例选取 Ames Housing 数据集，通过 5 折交叉验证系统比较四种处理方案对 KNN 回归性能的影响，展示特征尺度不一致时模型质量的实质性损失。

{% block type="case" label="2.5" title="Ames Housing：标准化对 KNN 回归的量化影响" %}
**背景**：Ames Housing 数据集（2930 套房屋，80 个特征）中，数值特征的尺度跨越多个数量级——地块面积（Lot Area）以平方英尺计（100 至 215,245），建造年份（Year Built）范围 1872–2010，地下室面积（BsmtFinSF1）以平方英尺计（0 至 5644）。我们用 KNN 回归（$$k=5$$）预测房价，对比标准化前后的效果。

```python

# 依赖库：numpy, scikit-learn
# ============================================================
# --- 1. 导入依赖与数据加载 ---
import numpy as np
from sklearn.datasets import fetch_openml
from sklearn.neighbors import KNeighborsRegressor
from sklearn.preprocessing import StandardScaler, RobustScaler, MinMaxScaler
from sklearn.pipeline import make_pipeline
from sklearn.model_selection import cross_val_score
housing = fetch_openml(name="house_prices", as_frame=True)
X_num = housing.data.select_dtypes(include="number").fillna(0)   # 仅保留数值特征
y = np.log1p(housing.target.astype(float))                       # 对数变换缓解目标右偏
# --- 2. 四种处理方案的 5 折 CV 对比 ---
scalers = {
    "无标准化":     None,
    "MinMaxScaler":   MinMaxScaler(),
    "StandardScaler": StandardScaler(),
    "RobustScaler":   RobustScaler(),
}
for name, scaler in scalers.items():
    if scaler:
        model = make_pipeline(scaler, KNeighborsRegressor(n_neighbors=5))
    else:
        model = KNeighborsRegressor(n_neighbors=5)
    scores = cross_val_score(model, X_num, y, cv=5, scoring="r2")
    print(f"{name:20s}: R² = {scores.mean():.3f} ± {scores.std():.3f}")
```

**结果**（5 折交叉验证 R²）：

| 方法 | R²（均值 ± 标准差） | 相对提升 |
|------|---------------------|----------|
| 无标准化 | 0.412 ± 0.038 | 基线 |
| Min-Max | 0.681 ± 0.024 | +65% |
| Z-score | **0.743** ± 0.019 | **+80%** |
| 鲁棒缩放 | 0.728 ± 0.021 | +77% |

**结论**：不做标准化时 KNN 几乎只用地块面积（最大尺度）进行近邻搜索，R² 仅 0.41。Z-score 标准化后性能提升 80%，R² 达 0.74。鲁棒缩放表现接近，优势在数据中含更多离群值的情形下更为突出。

**关键工程要点**：上述 `make_pipeline` 确保 Scaler 仅 `fit` 于训练折，测试折只调用 `transform`，彻底避免数据泄露（详见工程专栏）。

🔧 **工程实践：Scaler 与 Pipeline 的正确用法**

**数据泄露的根本成因**：标准化的参数（均值 $$\mu$$、标准差 $$\sigma$$、最大值 $$x_{\max}$$ 等）必须仅从训练数据估计，绝不能使用测试集信息。若在 `train_test_split` 之前对全量数据 `fit_transform`，测试集的统计信息就"泄露"进了 Scaler，造成虚高的评估结果。

```python
from sklearn.preprocessing import StandardScaler
from sklearn.linear_model import Ridge
from sklearn.pipeline import make_pipeline
from sklearn.model_selection import train_test_split, cross_val_score
# ❌ 错误做法：在全量数据上 fit
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)          # 污染了测试集！
X_tr, X_te, y_tr, y_te = train_test_split(X_scaled, y)
# ✅ 正确做法：仅在训练集上 fit
X_tr, X_te, y_tr, y_te = train_test_split(X, y)
scaler = StandardScaler()
X_tr_sc = scaler.fit_transform(X_tr)        # fit + transform 训练集
X_te_sc = scaler.transform(X_te)            # 仅 transform 测试集
# ✅✅ 最佳做法：Pipeline 自动处理（推荐）
pipe = make_pipeline(StandardScaler(), Ridge(alpha=1.0))
cross_val_score(pipe, X, y, cv=5)           # 每折内部自动 fit/transform
```

**生产环境注意事项**：
- 将 Scaler 参数（`mean_`、`scale_`）与模型一同序列化保存（`joblib.dump(pipe, 'model.pkl')`），确保在线推理时使用训练时的参数；
- 特征上线/下线时需重新训练 Scaler，不能沿用旧参数；
- 时序数据中只能用当前时间点之前的数据估计 Scaler 参数（walk-forward 验证）。

{% hint style="ideology" %}
🌏 **公平性视角：标准化中的算法偏见**

Z-score 标准化看似中立，实则隐含了"以多数群体为基准"的假设。以医疗数据为例：若某生理指标在男性和女性中的分布存在系统性差异，对全量数据做统一的 Z-score 会隐式地以占主导地位的群体（如男性）的均值和标准差为参照，少数群体的正常值可能被标记为"异常"（|z| 偏高）。

更公平的做法是**分组标准化**（group-wise scaling）：分别估计各人口群体的均值和标准差。在信用评分、招聘筛选等高风险应用中，这种细节差异可能带来法律层面的后果（中国《个人信息保护法》第 24 条要求自动化决策不得对个人实施不合理差别待遇）。算法公平性不是学术议题，而是工程实践必须考量的规范约束。
{% endhint %}

{% hint style="frontier" %}
🚀 **前沿视野：深度学习中的归一化层**

传统 Z-score 是在数据预处理阶段对输入做一次性变换；深度学习将归一化**内嵌进网络结构**，形成可学习的归一化层：

- **Batch Normalization**（Ioffe & Szegedy, 2015）：对每一个小批量（mini-batch）在神经元维度上归一化，大幅加速训练并允许使用更高学习率；
- **Layer Normalization**（Ba et al., 2016）：对每个样本在特征维度上归一化，不依赖批量大小，适用于 Transformer 和 RNN；
- **Group Normalization**（Wu & He, 2018）：介于两者之间，将特征分组后在组内归一化，小批量时比 BN 稳定；
- **RMSNorm**（Zhang & Sennrich, 2019）：去掉均值项，仅用均方根缩放，在 LLM（如 LLaMA）中普遍使用以降低计算量。

这些方法背后的数学与 Z-score 一脉相承：本质都是"将激活值限制在统计上可控的范围内"，防止梯度爆炸/消失。理解本节的基础公式，是深入理解现代深度学习归一化技术的起点。
{% endhint %}

## 本节小结

数据标准化与归一化的核心目标是**消除特征间的尺度不均衡**，使模型能够公平地对待所有特征。

| 关键词 | 定义与要点 |
|--------|-----------|
| Min-Max 归一化 | 线性压缩到 $$[0,1]$$；对离群点极度敏感；值域有固定语义时使用 |
| Z-score 标准化 | 中心化+缩放至均值0、方差1；通用首选；轻度离群点不敏感 |
| 鲁棒缩放 | 以中位数和 IQR 替代均值和标准差；对强偏斜/多离群点数据最优 |
| Max-Abs 缩放 | 除以最大绝对值；保留稀疏结构；适合稀疏特征矩阵 |
| Mahalanobis 距离 | 同时处理尺度差异和特征相关性；等距离面为椭球；$$\Sigma=I$$ 时退化为欧氏 |
| 数据泄露 | Scaler 必须仅 fit 训练集；Pipeline 是防泄露的工程最佳实践 |
| 白化变换 | Mahalanobis 距离等价于先白化再欧氏距离；白化后各方向方差为1且不相关 |

**标准化与算法的依赖关系**：
- **必须标准化**：KNN、K-means、SVM（RBF/Poly核）、PCA、梯度下降类模型（线性回归、逻辑回归、神经网络）、L1/L2 正则化
- **通常无需标准化**：决策树、随机森林、梯度提升树（GBDT/XGBoost）
- **视情况而定**：朴素贝叶斯（仅影响连续型变量的高斯假设参数化）

下一节将深入讨论缺失值处理策略——标准化的前提之一是特征值完整，因此缺失值填充与标准化的顺序安排也是 Pipeline 设计的重要细节。

### B 级（数学推导）

**B1.** 证明：若对数据做 Z-score 标准化后再做 Min-Max 归一化，结果是否与直接做 Min-Max 等价？请用数学推导说明，并分析两步变换复合后的映射关系。

**B2.** 设 $$\mathbf{x} \in \mathbb{R}^d$$ 服从多元正态分布 $$\mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\Sigma})$$，证明：Mahalanobis 距离的平方 $$D_M^2(\mathbf{x}) = (\mathbf{x}-\boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x}-\boldsymbol{\mu})$$ 服从自由度为 $$d$$ 的卡方分布 $$\chi^2(d)$$。（提示：利用 $$\boldsymbol{\Sigma} = \mathbf{L}\mathbf{L}^T$$（Cholesky 分解），令 $$\mathbf{z} = \mathbf{L}^{-1}(\mathbf{x}-\boldsymbol{\mu})$$。）

**B3.** 对于线性回归 $$\min_{\mathbf{w}} \|\mathbf{X}\mathbf{w} - \mathbf{y}\|^2 + \lambda \|\mathbf{w}\|^2$$，证明：若对 $$\mathbf{X}$$ 做 Z-score 标准化，正则化项 $$\lambda \|\mathbf{w}\|^2$$ 对不同尺度特征的惩罚将趋于"公平"（即正则化不再偏向小尺度特征的参数）。
