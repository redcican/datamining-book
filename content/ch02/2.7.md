# 2.7 异常值检测与处理

2022年，某城市商业银行的信用卡反欺诈系统面临一次棘手的误判：凌晨 3 时，一张长期在上海活跃的卡在巴黎刷卡消费 3,200 欧元。传统 Z-score 规则未将其标记为异常——该持卡人是商务出行者，历史消费记录中有多笔万元级别的大额支出，3,200 欧元并未突破 ±3σ 阈值。但安全分析师注意到：6 小时前该卡在上海有一笔消费记录（物理上不可能飞越），且前 24 小时内存在 5 笔逐渐递增的"探针性"小额消费——单独来看每笔都不异常，合在一起却构成典型的信用卡盗刷预热模式。这笔交易是**上下文异常**（凌晨 + 地理跳变）与**集体异常**（多笔探针构成异常模式）的叠加，而非任何单一维度上的极端值。

这一案例揭示了异常值检测的核心困难：**异常不总是"数值极大"，更多时候是"在特定上下文和局部邻域关系中偏离预期"**。2.1 节已就单变量情形介绍了 Z-score 与 IQR 两种基础检测方法（见图 2.1.3）；本节在此基础上，系统建立异常值的概念体系，推导 Grubbs 检验与 Mahalanobis 多元检验的统计理论，并深入讲解能处理多密度、非高斯分布场景的两类现代方法——**局部离群因子（LOF）**与**孤立森林（Isolation Forest）**——最终给出检测后的处理策略与工程实践指南。

4. 阐述孤立森林如何将异常检测转化为路径长度问题，说明其对高维数据和大规模数据集的优势；

## 2.7.1 异常值的概念体系与分类

2.1 节给出了统计异常值的基础定义（见定义 2.3）：取值远离正常范围的数据点。然而这一定义暗含了"全局正态分布"的假设，对于多密度、多尺度的真实数据并不充分。Hawkins（1980）给出了更宽泛的描述："异常值是一个与其他观测值偏离甚远、令人怀疑其由不同机制产生的观测。"在此基础上，现代文献（Chandola et al., 2009）将异常值细分为四种类型。

{% block type="definition" label="2.25" title="异常值的四类型分类（Taxonomy of Outliers）" %}
给定数据集 $\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$，依据异常的参照系不同，将异常值分为以下四类：

**（1）全局异常**（Global Outlier）：相对于**整体数据集**的分布而言，取值极端的单个数据点。形式上，若 $\mathbf{x}_i$ 落在全局分布的低概率尾部区域，即 $p(\mathbf{x}_i) < \delta$（$\delta$ 为阈值），则称 $\mathbf{x}_i$ 为全局异常。

**（2）局部异常**（Local Outlier）：相对于其**局部邻域**而言密度显著偏低的数据点。即使该点在全局分布中并不极端，若其局部密度远低于邻域中其他点的密度，则为局部异常。

**（3）上下文异常**（Contextual Outlier）：在特定**上下文**（时间、空间、条件）下偏离预期，而在其他上下文中属于正常范围的数据点。数据点 $\mathbf{x}_i$ 的异常性依赖于上下文属性（如月份、位置）与行为属性（如气温、消费额）的联合分布。

**（4）集体异常**（Collective Outlier）：**单独每个点不构成异常**，但一组点的联合模式相对于整体数据集呈现异常的集合 $\mathcal{A} \subset \mathcal{D}$。
{% endblock %}

图 2.7.1 从几何角度直观展示了这四种类型的区别：全局异常（面板 a）远离整体分布中心；局部异常（面板 b）夹在两个密度不同的簇之间，在全局上并不极端但在局部邻域中密度极低；上下文异常（面板 c）是时间序列中冬季出现的高温点，数值本身在夏季是正常的；集体异常（面板 d）是一组排列成不寻常线性模式的点，任意单点都不异常。

**方法与类型的对应关系**：全局异常可由 Z-score、IQR、Grubbs 检验或 Mahalanobis 距离检测；局部异常需要 LOF 或 DBSCAN 等基于密度的方法；上下文异常需在模型中显式引入上下文特征（如月份、地理位置）后再应用检测方法；集体异常则依赖序列分析或子群发现技术。本节重点讲解前两类的完整方法论。

理解了异常的分类，下一步是选择与数据分布假设相匹配的检测工具——当数据满足正态性假设时，经典统计检验提供了严格的假设检验框架；当分布未知或多峰时，需要无参数的密度或孤立方法。

## 2.7.2 统计检验方法

### Z-score 与 IQR：回顾与局限

Z-score 方法（三 σ 准则）和 IQR 方法已在 2.1 节（见定理 2.1 和图 2.1.3）中详细推导。这里简要回顾其局限：Z-score 假设数据服从正态分布，在强偏态或双峰分布下会产生大量漏报（正态尾部轻，真实异常被掩盖）；IQR 方法更鲁棒但同样是单变量方法，无法捕捉多特征联合异常。

### Grubbs 检验

对于小样本（$n < 30$）的单变量正态数据，Grubbs（1950）提出了一种更严格的假设检验框架，其零假设是"数据集中**不存在**异常值"。

{% block type="definition" label="2.26" title="Grubbs 检验（Grubbs' Test for Outliers）" %}
设 $\{x_1, \ldots, x_n\}$ 为正态总体 $N(\mu, \sigma^2)$ 的随机样本，样本均值为 $\bar{x}$，样本标准差为 $s$。**Grubbs 检验统计量**定义为：

$$
G = \frac{\max_{i=1}^{n} |x_i - \bar{x}|}{s}
\tag{2.45}
$$

即最极端点偏离均值的标准化距离。在显著性水平 $\alpha$ 下，拒绝"无异常值"的零假设（即判定存在异常点）的临界值为：

$$
G_{\mathrm{crit}}(n, \alpha) = \frac{n-1}{\sqrt{n}} \sqrt{\frac{t^2_{\alpha/(2n),\, n-2}}{n - 2 + t^2_{\alpha/(2n),\, n-2}}}
$$

其中 $t_{\alpha/(2n),\, n-2}$ 为自由度 $n-2$ 的 $t$ 分布在显著性水平 $\alpha/(2n)$（双侧 Bonferroni 校正）下的临界值。若 $G > G_{\mathrm{crit}}$，则 $\arg\max_i |x_i - \bar{x}|$ 被判定为异常值。
{% endblock %}

{% hint style="caution" %}
⚠️ **Grubbs 检验的适用条件**

Grubbs 检验仅适用于：（1）数据近似正态分布（可先用 Shapiro-Wilk 检验验证）；（2）样本量 $n \geq 7$；（3）每次只检测并移除**一个**最极端点，之后须重新计算 $\bar{x}$ 和 $s$ 再迭代检验（Grubbs 序贯过程）。若直接对大量离群点一次性判定，会因均值和方差受离群点扭曲而导致**掩蔽效应**（masking effect）。
{% endhint %}

### Mahalanobis 距离与多变量异常检测

单变量方法（Z-score、Grubbs）无法处理多个特征联合分布下的异常。2.5 节（见定义 2.20）已定义 Mahalanobis 距离为：$D_M(\mathbf{x}) = \sqrt{(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})}$。其用于异常检测的理论依据是以下定理，这也完成了 2.1 节（2.1.4 节补充块）与 2.5 节（习题 B2）中的证明承诺。

{% block type="theorem" label="2.8" title="Mahalanobis 距离的卡方分布（Chi-squared Distribution of Mahalanobis Distance）" %}
设 $\mathbf{x} \in \mathbb{R}^d$ 服从多元正态分布 $\mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\Sigma})$，其中 $\boldsymbol{\Sigma}$ 正定。则 Mahalanobis 距离的平方：

$$
D_M^2(\mathbf{x}) = (\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu}) \sim \chi^2(d)
\tag{2.46}
$$

即服从自由度为 $d$（特征维度数）的 $\chi^2$ 分布。因此，在显著性水平 $\alpha$ 下，若 $D_M^2(\mathbf{x}) > \chi^2_{1-\alpha,\, d}$（$\chi^2$ 分布的 $1-\alpha$ 分位数），则判定 $\mathbf{x}$ 为多变量异常值。
{% endblock %}

{% hint style="proof" %}
**证明**

由 $\boldsymbol{\Sigma}$ 正定，可进行 Cholesky 分解 $\boldsymbol{\Sigma} = \mathbf{L}\mathbf{L}^T$，其中 $\mathbf{L}$ 为下三角矩阵。令 $\mathbf{z} = \mathbf{L}^{-1}(\mathbf{x} - \boldsymbol{\mu})$，则：

$$
\mathbf{z} \sim \mathcal{N}(\mathbf{0},\, \mathbf{L}^{-1}\boldsymbol{\Sigma}(\mathbf{L}^{-1})^T) = \mathcal{N}(\mathbf{0},\, \mathbf{L}^{-1}\mathbf{L}\mathbf{L}^T(\mathbf{L}^{-1})^T) = \mathcal{N}(\mathbf{0},\, \mathbf{I})
$$

因此 $z_1, \ldots, z_d$ 相互独立的标准正态随机变量。注意到：

$$
D_M^2(\mathbf{x}) = (\mathbf{x}-\boldsymbol{\mu})^T(\mathbf{L}\mathbf{L}^T)^{-1}(\mathbf{x}-\boldsymbol{\mu}) = (\mathbf{L}^{-1}(\mathbf{x}-\boldsymbol{\mu}))^T (\mathbf{L}^{-1}(\mathbf{x}-\boldsymbol{\mu})) = \mathbf{z}^T\mathbf{z} = \sum_{k=1}^d z_k^2
$$

由 $\chi^2$ 分布的定义，$d$ 个独立标准正态变量的平方和服从 $\chi^2(d)$，故 $D_M^2(\mathbf{x}) \sim \chi^2(d)$。$\square$
{% endhint %}

{% block type="example" label="2.13" title="Mahalanobis 距离检测二维异常：尺度差异下的直觉" %}
**场景**：某传感器采集二维数据 $\mathbf{x} = (x_1, x_2)$，总体分布为 $\mathcal{N}(\boldsymbol{0}, \boldsymbol{\Sigma})$，其中 $\boldsymbol{\Sigma} = \mathrm{diag}(9, 1)$（$x_1$ 标准差为 3，$x_2$ 标准差为 1），$\boldsymbol{\mu} = \mathbf{0}$。设两个测试点 $A = (3, 0)$ 和 $B = (0, 3)$：

| 方法 | $A = (3, 0)$ | $B = (0, 3)$ | 判定 |
|------|-------------|-------------|------|
| 欧氏距离 $\|\mathbf{x} - \boldsymbol{\mu}\|$ | $\sqrt{9} = 3$ | $\sqrt{9} = 3$ | 两者**相同** |
| $D_M^2(\mathbf{x}) = x_1^2/9 + x_2^2/1$ | $9/9 + 0 = 1$ | $0 + 9/1 = 9$ | **显著不同** |

**检验**：$d=2$ 时，$\chi^2(2)$ 在显著性水平 $\alpha=0.025$ 下的临界值为 $\chi^2_{0.975,2} = 7.38$。

- $D_M^2(A) = 1 < 7.38$：**不能**判定为异常（$A$ 在 $x_1$ 方向仅 1 个标准差）。
- $D_M^2(B) = 9 > 7.38$：**判定为异常**（$B$ 在 $x_2$ 方向达 3 个标准差）。

**直觉**：欧氏距离将两点视为等距，但 Mahalanobis 距离正确地区分了"$x_1$ 偏离 1σ"与"$x_2$ 偏离 3σ"的本质差别——后者相对于本特征的正常范围而言远更异常。
{% endblock %}

{% hint style="tip" %}
💡 **实践中的协方差估计**

在样本量有限时，协方差矩阵 $\boldsymbol{\Sigma}$ 须从数据中估计，容易受到已存在的异常值污染（"鸡生蛋、蛋生鸡"问题）。推荐使用**最小协方差行列式估计量（MCD，Rousseeuw, 1984）**：在 $h = \lfloor (n+d+1)/2 \rfloor$ 个样本的所有子集中选择使协方差行列式最小的子集（等价于最密集的核心数据集），在此子集上估计 $\boldsymbol{\mu}$ 和 $\boldsymbol{\Sigma}$，再用全样本计算 $D_M^2$。sklearn 中对应 `EllipticEnvelope(support_fraction=0.75)`。
{% endhint %}

## 2.7.3 局部离群因子（LOF）

基于统计分布的方法（Z-score、Mahalanobis）隐含了全局密度均匀性假设：整个数据集来自同一分布，异常是其"尾部"。然而图 2.7.1(b) 所示的局部异常——夹在两个密度悬殊的簇之间的点——既不在全局分布的尾部，也不在任何单一特征的极端位置，却在**局部邻域**中显著偏离。Breunig 等人（2000）提出的**局部离群因子（LOF）**正是专门为捕捉这类局部密度偏差设计的。

{% hint style="mathbg" %}
📐 **数学基础：局部密度与可达距离**

LOF 的核心操作是"**用可达距离代替实际距离**"来平滑化局部密度的估计，防止邻居之间距离极小（近邻极近）时密度被夸大。设 $k$-距离是到第 $k$ 近邻的距离，则**可达距离**用 $k$-距离截断极近的邻居距离，避免密度计算退化。
{% endhint %}

{% block type="definition" label="2.27" title="LOF 算法形式化：k-距离、可达距离、局部可达密度与 LOF 分数" %}
对数据集 $\mathcal{D}$ 中的点 $p$，给定参数 $k$（邻居数），依次定义：

**（1）$k$-距离**（$k$-distance）：$p$ 到其第 $k$ 近邻的距离，记为 $k\text{-dist}(p)$。

$$
k\text{-dist}(p) = \min_{d'} \left\{d' : \left|\{q \in \mathcal{D} \setminus \{p\} : d(p,q) \leq d'\}\right| \geq k\right\}
\tag{2.47}
$$

**$k$-邻域**（$k$-neighborhood）：$N_k(p) = \{q \in \mathcal{D} \setminus \{p\} : d(p,q) \leq k\text{-dist}(p)\}$（距离不超过 $k$-距离的所有点，可能多于 $k$ 个，存在距离相等的并列情形）。

**（2）可达距离**（reachability distance）：点 $p$ 相对于点 $o$ 的 $k$-可达距离为：

$$
\mathrm{reach\text{-}dist}_k(p, o) = \max\bigl(k\text{-dist}(o),\; d(p, o)\bigr)
\tag{2.48}
$$

即：若 $p$ 与 $o$ 极近（$d(p,o) < k\text{-dist}(o)$），则可达距离被 $o$ 的 $k$-距离截断，防止密度估计虚高。

**（3）局部可达密度**（local reachability density, lrd）：

$$
\mathrm{lrd}_k(p) = \frac{|N_k(p)|}{\displaystyle\sum_{o \in N_k(p)} \mathrm{reach\text{-}dist}_k(p, o)}
\tag{2.49}
$$

即 $p$ 与其邻域中各点的平均可达距离之倒数——距离越小，密度越高。

**（4）LOF 分数**（local outlier factor）：

$$
\mathrm{LOF}_k(p) = \frac{1}{|N_k(p)|} \sum_{o \in N_k(p)} \frac{\mathrm{lrd}_k(o)}{\mathrm{lrd}_k(p)}
\tag{2.50}
$$

即 $p$ 的邻居的平均局部密度与 $p$ 自身局部密度之比。$\mathrm{LOF}_k(p) \approx 1$ 表示 $p$ 与邻居密度相当（正常点）；$\mathrm{LOF}_k(p) \gg 1$ 表示 $p$ 的密度远低于邻居（局部异常）。
{% endblock %}

以下定理说明了 LOF 在密度均匀区域的理论性质，为 LOF=1 作为"正常"判据提供了理论依据。

{% block type="theorem" label="2.9" title="均匀密度区域内 LOF≈1 的性质" %}
若数据点 $p$ 处于**局部密度均匀**的区域（即 $p$ 的所有 $k$-邻居 $o$ 满足 $d(p,o) \geq k\text{-dist}(o)$），则：

$$
\mathrm{LOF}_k(p) = 1
$$

更一般地，若 $p$ 及其所有邻居均处于密度均匀区域，则 $1 - \varepsilon \leq \mathrm{LOF}_k(p) \leq 1 + \varepsilon$，其中 $\varepsilon$ 随密度变化率减小而趋于 0。
{% endblock %}

{% hint style="proof" %}
**证明**

在密度均匀区域，对任意 $o \in N_k(p)$，有 $d(p,o) \geq k\text{-dist}(o)$（邻域足够大，截断不生效），故：

$$
\mathrm{reach\text{-}dist}_k(p,o) = \max(k\text{-dist}(o),\, d(p,o)) = d(p,o)
$$

从而 $\mathrm{lrd}_k(p) = |N_k(p)| / \sum_{o \in N_k(p)} d(p,o)$。类似地，对每个邻居 $o'$，由密度均匀性有 $\mathrm{lrd}_k(o') \approx \mathrm{lrd}_k(p)$（相同大小的邻域，相似的平均距离）。代入 LOF 定义：

$$
\mathrm{LOF}_k(p) = \frac{1}{|N_k(p)|}\sum_{o \in N_k(p)} \frac{\mathrm{lrd}_k(o)}{\mathrm{lrd}_k(p)} \approx \frac{1}{|N_k(p)|} \sum_{o \in N_k(p)} 1 = 1 \qquad \square
$$
{% endhint %}

{% block type="example" label="2.14" title="LOF 手动计算（k=2，四点一维数据集）" %}
**数据**（一维，四个点）：$p_1=0,\; p_2=1,\; p_3=2,\; p_4=10$。参数 $k=2$。

**步骤一：计算各点 $k$-距离与 $k$-邻域（$k=2$）**

| 点 | 到其余点的距离 | $k\text{-dist}$ | $N_2$ |
|----|--------------|-----------------|-------|
| $p_1$ | $d(p_1,p_2)=1,\;d(p_1,p_3)=2,\;d(p_1,p_4)=10$ | $2$ | $\{p_2,p_3\}$ |
| $p_2$ | $d(p_2,p_1)=1,\;d(p_2,p_3)=1,\;d(p_2,p_4)=9$  | $1$ | $\{p_1,p_3\}$ |
| $p_3$ | $d(p_3,p_1)=2,\;d(p_3,p_2)=1,\;d(p_3,p_4)=8$  | $2$ | $\{p_1,p_2\}$ |
| $p_4$ | $d(p_4,p_1)=10,\;d(p_4,p_2)=9,\;d(p_4,p_3)=8$ | $9$ | $\{p_2,p_3\}$ |

**步骤二：计算可达距离**（$\mathrm{reach\text{-}dist}_2(p,o) = \max(k\text{-dist}(o), d(p,o))$）

- $p_1$：$\mathrm{rd}(p_1,p_2) = \max(1,1) = 1$，$\mathrm{rd}(p_1,p_3) = \max(2,2) = 2$
- $p_2$：$\mathrm{rd}(p_2,p_1) = \max(2,1) = 2$，$\mathrm{rd}(p_2,p_3) = \max(2,1) = 2$
- $p_3$：$\mathrm{rd}(p_3,p_1) = \max(2,2) = 2$，$\mathrm{rd}(p_3,p_2) = \max(1,1) = 1$
- $p_4$：$\mathrm{rd}(p_4,p_2) = \max(1,9) = 9$，$\mathrm{rd}(p_4,p_3) = \max(2,8) = 8$

**步骤三：局部可达密度**（$\mathrm{lrd}_2(p) = |N_2|/\sum \mathrm{rd}$）

$$
\mathrm{lrd}(p_1) = \frac{2}{1+2} = \tfrac{2}{3}, \quad \mathrm{lrd}(p_2) = \frac{2}{2+2} = \tfrac{1}{2}, \quad \mathrm{lrd}(p_3) = \frac{2}{2+1} = \tfrac{2}{3}, \quad \mathrm{lrd}(p_4) = \frac{2}{9+8} = \tfrac{2}{17}
$$

**步骤四：LOF 分数**（$\mathrm{LOF}_2(p) = \frac{1}{|N_2|}\sum_{o \in N_2} \frac{\mathrm{lrd}(o)}{\mathrm{lrd}(p)}$）

$$
\mathrm{LOF}(p_1) = \tfrac{1}{2}\left(\frac{1/2}{2/3} + \frac{2/3}{2/3}\right) = \tfrac{1}{2}\left(\tfrac{3}{4}+1\right) = \tfrac{7}{8} \approx 0.875
$$

$$
\mathrm{LOF}(p_2) = \tfrac{1}{2}\left(\frac{2/3}{1/2} + \frac{2/3}{1/2}\right) = \tfrac{1}{2}\cdot\tfrac{8}{3} = \tfrac{4}{3} \approx 1.333
$$

$$
\mathrm{LOF}(p_3) = \tfrac{1}{2}\left(\frac{2/3}{2/3} + \frac{1/2}{2/3}\right) = \tfrac{7}{8} \approx 0.875
$$

$$
\mathrm{LOF}(p_4) = \tfrac{1}{2}\left(\frac{1/2}{2/17} + \frac{2/3}{2/17}\right) = \tfrac{1}{2} \cdot \frac{17}{2}\left(\frac{1}{2}+\frac{2}{3}\right) = \tfrac{17}{4}\cdot\frac{7}{6} \approx \mathbf{4.96}
$$

**结论**：密集簇内点 $p_1, p_3$ 的 LOF≈0.875（近似为 1），$p_2$ 的 LOF≈1.33（轻度边界效应），而孤立点 $p_4$ 的 LOF≈4.96 远大于 1，被正确检测为异常。
{% endblock %}

**LOF 的参数选择**：$k$ 的典型取值为 10–50，过小会使 LOF 对噪声敏感，过大则会将异常点混入邻域而降低检测能力。实践中常对多个 $k$ 值取 LOF 的最大值（$\mathrm{LOF}^{\max}_k$）以提高稳定性。**计算复杂度**为 $O(n^2 d)$（暴力邻域查询），大规模数据可用 kd-Tree 加速至 $O(n \log n \cdot d)$。

## 2.7.4 孤立森林

LOF 在中等规模数据上表现优异，但其 $O(n^2)$ 复杂度在百万级数据集上难以承受。Liu 等人（2008）提出的**孤立森林（Isolation Forest）**从完全不同的角度定义异常：**异常点稀少且与其他点距离远，因此用随机超平面递归分割时，将其孤立所需的切割次数远少于正常点**。这一思想将异常检测转化为随机树的路径长度问题，计算复杂度仅为 $O(n)$（树的期望深度为 $O(\log n)$），尤其适合高维大规模场景。

{% block type="definition" label="2.28" title="孤立树、路径长度与孤立森林异常分数" %}
**孤立树**（Isolation Tree, iTree）：对数据集 $\mathcal{D}$ 的子样本 $\psi \leq n$ 个点，递归地随机构建二叉树：
- 随机选取一个特征 $q$；随机选取该特征值区间 $[x_{\min}^q, x_{\max}^q]$ 内的切割点 $p$；
- 左子树包含 $x_q \leq p$ 的所有点，右子树包含 $x_q > p$ 的所有点；
- 递归直至每个叶节点只含一个点或达到最大深度。

**路径长度**（Path Length）：点 $\mathbf{x}$ 在一棵 iTree 中被孤立所经历的边数，记为 $h(\mathbf{x})$，即从根节点到包含 $\mathbf{x}$ 的叶节点的深度。

**期望路径长度归一化**：对包含 $n$ 个样本的 BST，搜索一个节点的平均路径长度期望为：

$$
c(n) = 2H(n-1) - \frac{2(n-1)}{n}, \qquad H(i) = \ln(i) + 0.5772\ldots \text{（欧拉常数）}
\tag{2.51}
$$

**孤立森林异常分数**：基于 $t$ 棵 iTree 的路径长度均值 $\mathbb{E}[h(\mathbf{x})]$，定义异常分数为：

$$
s(\mathbf{x}, \psi) = 2^{-\dfrac{\mathbb{E}[h(\mathbf{x})]}{c(\psi)}}
\tag{2.52}
$$

分数解读：$s \to 1$ 表示 $\mathbf{x}$ 路径极短（易被孤立，**强异常**）；$s \to 0.5$ 表示路径接近平均长度（**正常**）；$s \to 0$ 表示路径极长（深度嵌入密集区域）。
{% endblock %}

{% block type="algorithm" label="2.7" title="孤立森林（Isolation Forest，Liu et al., 2008）" %}
**输入**：数据集 $\mathcal{D}$，树的数量 $t$，子样本大小 $\psi$

**输出**：每个点的异常分数 $s(\mathbf{x}, \psi)$

```text
训练阶段：
  对每棵树 i = 1, …, t：
    1. 从 D 中随机无放回抽取 ψ 个点，构成子集 D_i
    2. 对 D_i 递归构建孤立树：
       a. 若 |当前节点| ≤ 1 或深度达到 ⌈log₂ψ⌉，返回叶节点
       b. 随机选取特征 q ∈ {1, …, d}
       c. 随机选取切割点 p ∈ [x_min^q, x_max^q]
       d. 左子树 ← 满足 x_q ≤ p 的子集；右子树 ← 满足 x_q > p 的子集
       e. 递归对左、右子树执行步骤 a-d

预测阶段：
  对待评估点 x：
    1. 在每棵 iTree 中沿路径遍历，记录路径长度 h_i(x)
    2. 计算平均路径长度 E[h(x)] = (1/t) Σ h_i(x)
    3. 计算异常分数 s(x, ψ) = 2^{-E[h(x)] / c(ψ)}

判定：s > 阈值（通常 0.5 + contamination / 2）则标记为异常
```

**推荐参数**：$t = 100$ 棵树（增大收益递减），$\psi = 256$（Liu 等人证明 $\psi = 256$ 时路径长度已充分收敛）。
{% endblock %}

图 2.7.3 直观展示了孤立森林的工作原理：面板 (a) 中注入的异常点（紫色边框）在得分热图中颜色最深（得分最高），而密集正常点区域颜色较浅；面板 (b) 的路径长度分布图清晰显示了正常点（蓝色峰）与异常点（红色竖线）的分离——异常点得分集中在高端，与正常点分布几乎不重叠。

{% hint style="supplement" %}
🔬 **补充：孤立森林 vs LOF 的适用场景对比**

| 维度 | LOF | 孤立森林 |
|------|-----|---------|
| 时间复杂度 | $O(n^2 d)$（暴力）/ $O(n\log n \cdot d)$（kd-Tree） | $O(nt)$（$t$ 棵树，每棵 $O(n)$） |
| 空间复杂度 | $O(n^2)$（需存所有距离） | $O(t\psi)$ |
| 适用分布 | 任意（无假设）| 任意（无假设）|
| 高维数据 | 受维数灾难影响（距离集中） | 随机切割对高维鲁棒 |
| 需预知异常比例 | 否（设阈值即可） | 推荐设 `contamination`；也可用 $s > 0.5$ |
| 局部异常 | **极优**（基于局部密度比） | 较弱（随机切割不考虑邻域结构） |
| 全局异常 | 有效 | **极优**（稀疏极端点被快速孤立） |
| 大规模数据（$n > 10^6$） | 不推荐 | **推荐**（次线性复杂度）|

**实践准则**：数据量 $< 10^4$、怀疑存在局部异常 → 优先 LOF；数据量 $\geq 10^5$、异常稀疏且全局分散 → 优先孤立森林；两者结合（取分数均值）可提升鲁棒性。
{% endhint %}

## 2.7.5 异常值处理策略

检测出异常值后，处理策略的选择取决于**异常来源**（数据录入错误 vs. 真实极端值）和**下游模型对异常的敏感度**（线性回归高度敏感，决策树/随机森林相对鲁棒）。

**策略一：删除法**。当异常点被明确判定为**数据录入错误**（如身高 `-1.70` 米、年龄 `200` 岁）且数量极少（< 1%）时，直接删除是最简洁的处理方式。删除前须记录删除数量与理由，并确认删除后样本分布不发生系统性偏移（否则等同于引入 MAR 缺失）。

**策略二：Winsorization（截尾/缩尾法）**。将超出分位数阈值的异常值替换为阈值本身，而不删除记录：

$$
\tilde{x}_i = \max\!\bigl(\min(x_i,\, q_{1-\alpha}),\; q_{\alpha}\bigr)
\tag{2.53}
$$

其中 $q_\alpha, q_{1-\alpha}$ 为样本 $\alpha$ 和 $1-\alpha$ 分位数（常取 $\alpha = 0.01$ 或 $0.05$）。Winsorization 保留了样本量，减弱了极端值的杠杆效应，同时避免了"真实极端值被删除"的信息损失，是金融与医学数据中的常用策略。

**策略三：对数/Box-Cox 变换**。对强右偏数据（如收入、房价、交易金额）进行对数变换 $y = \log(1 + x)$ 可将长尾压缩，使得极端值相对不那么"极端"，而不删除任何记录。Box-Cox 变换（见 2.2 节）提供了更一般的幂变换选择。此策略适用于**真实极端值**（如富豪收入）而非录入错误的情形。

**策略四：标记保留法**。将检测出的异常点标记为二值特征 $r_i = \mathbf{1}[\text{点 } i \text{ 为异常}]$，保留原始值，让模型同时学习数值本身与其异常状态：

$$
\mathbf{x}_i^{\mathrm{new}} = (x_{i1}, \ldots, x_{id}, r_i)
$$

此策略适用于**异常本身具有业务意义**的场景（如信用卡欺诈检测：异常状态就是标签的强预测因子），既保留了信息，又将异常检测的结论显式传递给下游模型。

{% hint style="caution" %}
⚠️ **不要在不理解原因的情况下直接删除异常值**

盲目删除异常值是数据分析中的常见错误。真实数据中的"异常"可能是：（1）最有价值的少数群体（高净值客户、罕见疾病患者）；（2）模型应该学习区分的关键样本（欺诈交易）；（3）系统性问题的线索（传感器漂移、流程缺陷）。规范流程是：**先检测、再分类（错误 vs. 真实极端）、再决定处理方式**，而非直接删除。
{% endhint %}

## 2.7.6 工程实践：多方法对比与 Pipeline 集成

{% hint style="engineering" %}
🔧 **工程实践：三种方法的统一对比框架与 Pipeline 集成**

实际项目中，不同异常检测方法对同一数据集的标记结果往往有分歧。推荐的工程实践是同时运行多种方法，对标记结果取**集成投票**（≥2 种方法均标记才判为异常），并将异常分数作为特征输入下游模型（而非直接删除）。

```python
# ============================================================
# 代码 2.7：三种异常检测方法对比（Z-score / LOF / 孤立森林）
# 数据集：合成双密度簇数据（含全局异常 + 局部异常）
# 依赖库：numpy, scikit-learn
# ============================================================
# --- 1. 构建含不同类型异常的数据集 ---
import numpy as np
from sklearn.neighbors import LocalOutlierFactor
from sklearn.ensemble import IsolationForest
from sklearn.preprocessing import StandardScaler
rng = np.random.default_rng(2024)
n_norm = 300
# 正常点：两个密度不同的簇
Xd = rng.normal([0, 0], 0.6, (200, 2))
Xs = rng.normal([6, 0], 1.5, (100, 2))
# 全局异常：远离所有簇
X_global = np.array([[-5.0, 4.0], [5.0, -5.0]])
# 局部异常：夹在两簇之间的稀疏区域
X_local = np.array([[2.5, 0.5], [3.2, -0.3]])
X_all = np.vstack([Xd, Xs, X_global, X_local])
true_label = np.array([0]*n_norm + [1]*2 + [1]*2)  # 0=正常, 1=异常
# --- 2. 方法一：Z-score（多变量版本：对所有特征取最大 Z 分数）---
scaler = StandardScaler()
X_std = scaler.fit_transform(X_all)
z_max = np.abs(X_std).max(axis=1)
z_pred = (z_max > 3.0).astype(int)
# --- 3. 方法二：LOF ---
lof = LocalOutlierFactor(n_neighbors=20, contamination=0.05)
lof_pred = (lof.fit_predict(X_all) == -1).astype(int)
lof_scores = -lof.negative_outlier_factor_
# --- 4. 方法三：孤立森林 ---
iso = IsolationForest(n_estimators=200, contamination=0.05, random_state=2024)
iso_pred = (iso.fit_predict(X_all) == -1).astype(int)
iso_scores = -iso.score_samples(X_all)
# --- 5. 集成投票：≥2 种方法判定才视为异常 ---
vote_score = z_pred + lof_pred + iso_pred
ensemble_pred = (vote_score >= 2).astype(int)
# --- 6. 评估（Precision / Recall 更合适于不平衡检测场景）---
from sklearn.metrics import precision_score, recall_score, f1_score
for name, pred in [("Z-score",    z_pred),
                   ("LOF",        lof_pred),
                   ("孤立森林",    iso_pred),
                   ("集成投票",    ensemble_pred)]:
    p = precision_score(true_label, pred, zero_division=0)
    r = recall_score(true_label, pred, zero_division=0)
    f = f1_score(true_label, pred, zero_division=0)
    print(f"{name:8s}  Precision={p:.2f}  Recall={r:.2f}  F1={f:.2f}")
```

**Pipeline 集成注意事项**：若将异常分数作为特征传入下游分类器，须在 `Pipeline` 内部完成异常检测（避免测试集信息泄露）。对于 LOF，sklearn 提供 `LocalOutlierFactor(novelty=True)` 支持 `fit`/`predict` 分离，可正确放入 Pipeline；孤立森林本身支持 `fit`/`predict` 分离。
{% endhint %}

## 2.7.7 综合案例：KDD Cup 1999 网络入侵检测

{% block type="case" label="2.7" title="KDD Cup 1999：基于孤立森林与 LOF 的网络入侵异常检测" %}
**数据集**：KDD Cup 1999 网络入侵数据集（sklearn 内置，`fetch_kddcup99`），采用其中 HTTP 流量子集（`subset='http'`）。每条记录为一次 TCP 连接，特征包含连接时长、传输字节数、标志位等；标签为 `'normal.'`（正常连接）或各类攻击类型（`neptune.`、`smurf.` 等）。异常检测任务：不使用标签，仅凭特征检测入侵流量。

**挖掘目标**：以无监督方式检测网络异常连接，比较孤立森林与 LOF 两种方法在精度与召回率上的差异，并分析各自误判案例的类型特征。

```python
# ============================================================
# 代码 2.7：KDD Cup 1999 网络入侵无监督异常检测
# 数据集：sklearn.datasets.fetch_kddcup99（HTTP 子集）
# 依赖库：numpy, scikit-learn
# ============================================================
# --- 1. 加载数据（HTTP 子集：约 5.8 万条，41 个数值特征）---
import numpy as np
from sklearn.datasets import fetch_kddcup99
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.ensemble import IsolationForest
from sklearn.neighbors import LocalOutlierFactor
from sklearn.metrics import classification_report
data = fetch_kddcup99(subset="http", percent10=True, random_state=42)
X_raw = data.data.astype(float)    # 数值特征矩阵（删除类别列后）
y_raw = data.target                # 标签（bytes, normal. 等）
# --- 2. 构建二值标签：'normal.' = 0，其余均为异常 = 1 ---
y_bin = (y_raw != b"normal.").astype(int)
print(f"样本数：{len(y_bin)}；异常比例：{y_bin.mean():.3%}")
# --- 3. 特征预处理：标准化 + PCA（高维稳定性）---
scaler = StandardScaler()
X_sc = scaler.fit_transform(X_raw)
pca = PCA(n_components=0.95, random_state=42)   # 保留 95% 方差
X_pca = pca.fit_transform(X_sc)
print(f"PCA 保留维度：{X_pca.shape[1]}")
# --- 4. 孤立森林（适合大规模，全局异常）---
contamination = y_bin.mean()
iso = IsolationForest(n_estimators=200, contamination=contamination,
                      random_state=42)
iso_pred = (iso.fit_predict(X_pca) == -1).astype(int)
# --- 5. LOF（局部异常，受规模限制取子样本）---
sample_idx = np.random.default_rng(42).choice(len(X_pca),
                                               size=min(10000, len(X_pca)),
                                               replace=False)
X_sub = X_pca[sample_idx]
y_sub = y_bin[sample_idx]
lof = LocalOutlierFactor(n_neighbors=20, contamination=contamination,
                         novelty=False)
lof_sub = (lof.fit_predict(X_sub) == -1).astype(int)
# --- 6. 评估对比 ---
print("\n── 孤立森林（全量数据）──")
print(classification_report(y_bin, iso_pred,
                             target_names=["正常", "异常"], digits=3))
print("── LOF（1 万样本子集）──")
print(classification_report(y_sub, lof_sub,
                             target_names=["正常", "异常"], digits=3))
```

**结果解读**：HTTP 子集中的网络攻击（如 neptune DoS 攻击）通常表现为异常高的字节数或连接数，属于全局异常，孤立森林对此类型检测率更高（Recall 通常 > 85%）。而某些"低慢型"攻击（慢速扫描）制造的异常连接与正常流量差异细微，属于局部异常，LOF 凭借对局部密度的敏感性具有优势。两种方法各有侧重，工程实践中通常取两者分数的加权组合以覆盖更多攻击类型。

**局限性**：KDD Cup 1999 数据集存在严重的样本重复问题（约 78% 的记录是重复的）；真实场景中的网络安全异常检测还需要处理时序上下文、协议语义等更复杂的特征工程。
{% endblock %}

{% hint style="frontier" %}
🚀 **前沿视野：深度学习驱动的异常检测**

传统方法（LOF、孤立森林）在非结构化数据（图像、时序、图结构）上面临特征工程瓶颈。2020 年以来，深度异常检测方法迅速成熟：

**自编码器（Autoencoder）重建误差**：训练 AE 在正常数据上最小化重建误差 $\|\mathbf{x} - \hat{\mathbf{x}}\|^2$；推断时，异常点因偏离训练分布而重建误差显著更高，可直接作为异常分数。变分自编码器（VAE）将重建概率 $p(\mathbf{x})$ 而非 MSE 作为分数，提供了更稳健的统计基础。

**Transformer 序列异常**：用 Transformer 在时序数据上建模正常连续模式，以预测误差检测时序异常点；DeepAnt（Munir et al., 2019）和 Anomaly Transformer（Xu et al., 2022）是代表性工作，后者引入"关联差异"（Association Discrepancy）机制，对时间序列异常点实现了细粒度的端到端检测。

**图神经网络异常检测**：DOMINANT（Ding et al., 2019）将 GAE 应用于图结构数据，以属性重建误差与结构重建误差的线性组合作为节点异常分数；在社交网络水军检测、金融交易图谱欺诈识别中表现优异。

**基础模型赋能**：2024 年出现的 AnomalyGPT 等工作将大语言模型的预训练知识迁移至工业图像异常检测，无需大量领域标注数据，零样本/少样本检测精度接近全监督水平。

理解本节的统计基础（局部密度、路径长度、分布假设）是评估和改进上述深度方法的必要前提：深度模型在特征表示层创新，但其异常判据的统计逻辑与 LOF、孤立森林一脉相承。
{% endhint %}

{% hint style="ideology" %}
🌏 **课程思政·技术向善：异常检测与数字化治理**

异常检测技术不仅是数据挖掘的核心工具，也是现代数字化治理的重要基础设施。

**金税三期/四期系统**：我国税务总局的金税系统综合运用统计异常检测与机器学习模型，从海量增值税发票数据中识别虚开发票、偷漏税等违法行为。2022 年，金税四期上线后，利用图神经网络对关联企业交易网络进行分析，识别"环开"发票等集体异常模式，有效维护了税收公平。这是本节"集体异常"概念在现实治理中的直接应用。

**食品安全监管**：市场监管总局通过对食品抽检数据库应用异常检测算法，对同一批次、同一产地的样品检测结果进行上下文对比，快速识别批次性污染事件（集体异常），实现从"抽检发现"到"溯源预警"的转变。

然而，技术的正当应用须以**法治为边界**。异常检测系统可能产生误报，误将正常个体标记为异常，引发不当处罚或歧视。我国《个人信息保护法》（2021）第 24 条明确规定：使用个人信息进行自动化决策时，须保证决策透明度和结果公平，并提供申诉途径。数据科学家有责任：（1）公开模型的误报率；（2）为被标记的个体提供人工复核机制；（3）定期审计模型是否对特定群体产生系统性偏差。技术能力越强，审慎使用的义务越重——这是数字时代数据科学职业伦理的核心要义。

## 本节小结

| 方法 | 适用异常类型 | 分布假设 | 时间复杂度 | sklearn API |
|------|------------|---------|-----------|-------------|
| Z-score（3σ准则） | 全局，单变量 | 正态 | $O(n)$ | `StandardScaler` |
| IQR | 全局，单变量，鲁棒 | 无 | $O(n\log n)$ | `IQRRange` 手动 |
| Grubbs 检验 | 全局，小样本，单变量 | 正态（严格） | $O(n)$ | — |
| Mahalanobis 距离 | 全局，多变量 | 多元正态 | $O(nd^2 + d^3)$ | `EllipticEnvelope` |
| LOF | **局部**，中等规模 | 无 | $O(n^2 d)$ | `LocalOutlierFactor` |
| 孤立森林 | 全局，大规模，高维 | 无 | $O(nt)$ | `IsolationForest` |

**核心结论**：选择方法须先判断异常类型与数据规模。分布已知（正态）→ Mahalanobis；分布未知、中等规模、局部异常 → LOF；大规模或高维、全局异常 → 孤立森林；两者集成投票可提升鲁棒性。处理策略上，**先查原因、后选方法**：录入错误删除，真实极端值 Winsorize 或变换，异常本身具有业务意义时保留并标记。

本节完成了数据预处理第二章的最后一个核心主题。至此，从数据质量诊断（2.1节）、集成与变换（2.2节）、规约（2.3节）、特征工程（2.4节）、标准化（2.5节）、缺失值处理（2.6节）到异常值检测（2.7节），构成了一套完整的数据预处理方法论。第三章将在这一"干净数据"的基础上，正式进入分类算法的核心内容。

## 关键术语

| 中文术语 | 英文术语 | 简要定义 |
|----------|----------|----------|
| 全局异常 | Global Outlier | 相对于整体分布取值极端的单个点 |
| 局部异常 | Local Outlier | 相对于局部邻域密度偏低的点（全局不一定极端） |
| 上下文异常 | Contextual Outlier | 在特定时间/空间上下文下偏离预期的点 |
| 集体异常 | Collective Outlier | 单独不异常但整体模式异常的一组点 |
| Grubbs 检验 | Grubbs' Test | 正态分布下检验单个最极端点的假设检验 |
| Mahalanobis 距离 | Mahalanobis Distance | 考虑特征相关性的多元统计距离（见定义 2.20） |
| LOF | Local Outlier Factor | 基于局部密度比的异常分数，内部点 LOF≈1 |
| 可达距离 | Reachability Distance | LOF 中用 k-距离截断的平滑化距离 |
| 孤立森林 | Isolation Forest | 基于随机分割路径长度的异常检测集成方法 |
| Winsorization | Winsorization | 将超出分位数阈值的值截断至阈值（缩尾法） |
| 掩蔽效应 | Masking Effect | 多个异常点相互"保护"导致单一检验失效的现象 |
