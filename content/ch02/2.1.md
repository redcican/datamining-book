# 2.1 数据清洗技术

1999年9月23日，美国国家航空航天局（NASA）耗资3.27亿美元的火星气候轨道飞行器（Mars Climate Orbiter）在抵达火星时因进入大气层角度偏差而彻底损毁。事故调查委员会的报告揭示了一个令人难以置信的原因：承包商洛克希德·马丁公司提交的导航数据采用英制单位（英镑力·秒），而NASA的导航软件期待公制单位（牛顿·秒）——两个系统之间的数据单位不一致，摧毁了一艘造价不菲的宇宙飞船。这不是软件缺陷，也不是传感器故障，而是**数据质量问题**。

在商业领域，IBM的研究估计，低质量数据每年给美国企业造成约3.1万亿美元的经济损失（DHS, 2017）。Kaggle的调查则显示，数据科学家平均将60%以上的工作时间用于数据清洗，而非建模或分析本身。这一现象深刻揭示了一个行业共识：**数据清洗（Data Cleaning）是数据挖掘中最耗时、最关键、也最容易被低估的环节**。

本节系统介绍数据质量问题的分类体系与形式化处理框架。在此基础上，第2.6节将深入讲解缺失值的高级填补策略，第2.7节将展开异常值检测的完整方法论。本节提供的理论基础与基础工具，是后续各节的共同前提。

{% hint style="info" %}
📋 **本节学习目标**

完成本节学习后，读者应能够：

1. 运用四维分类框架（不完整性、噪声、不一致性、重复性），识别并标注真实数据集中存在的各类质量问题；
2. 用概率模型（缺失指示矩阵 $$\mathbf{R}$$）精确区分 MCAR、MAR、MNAR 三种缺失机制，并说明各机制对填补策略选择的影响；
3. 推导三σ准则，手动计算 Z-score 与 IQR 统计量，并将二者的适用条件加以对比；
4. 实现基于哈希的精确去重与基于编辑距离的近似去重算法，描述排序邻域法（Sorted Neighborhood Method）的核心思想；
5. 综合运用以上工具，对给定的真实数据集完成数据质量的诊断报告。
{% endhint %}

## 2.1.1 数据质量的代价与分类体系

### 问题根源

数据质量问题并非偶然，其根源可追溯至数据生命周期的每一个环节。在**采集阶段**，传感器漂移、手工录入疲劳、问卷设计缺陷均可导致原始数据失真；在**存储阶段**，关系型数据库的约束缺失或数据仓库的 ETL（Extract-Transform-Load）错误会引入不一致记录；在**整合阶段**，来自多个异构系统的数据往往使用不同的标识符、格式与编码规范，强行合并时不可避免地产生冲突。以医院信息系统为例，同一患者"张伟"可能在门诊系统中记录为"Zhang Wei"，在住院系统中记录为"张 伟"（含全角空格），在检验系统中则以身份证号为主键——三套系统的数据整合即是一场与不一致性的持久战。

从系统视角观察，数据质量问题可被归纳为四个互斥而完备的类别。

{% block type="definition" label="2.1" title="数据质量问题分类（Four Types of Data Quality Problems）" %}
给定数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$$，$$\mathbf{x}_i \in \mathbb{R}^d$$，以下四类质量问题构成对 $$\mathcal{D}$$ 的完备描述：

**（1）不完整性**（Incompleteness）：一个或多个特征维度的值缺失，即存在 $$j \in \{1,\ldots,d\}$$ 使得 $$x_{ij}$$ 未被记录。记录全集 $$\{(i,j) : x_{ij} \text{ 缺失}\}$$ 称为**缺失模式**。

**（2）噪声**（Noise）：特征值被错误记录，即真实值 $$x_{ij}^*$$ 与记录值 $$x_{ij}$$ 之间存在偏差 $$\varepsilon_{ij} = x_{ij} - x_{ij}^*$$，其中偏差可由随机误差（仪器精度）或系统误差（标定错误）引起。特别地，若 $$x_{ij}$$ 取值远离该特征的正常范围，则称为**异常值**（outlier）。

**（3）不一致性**（Inconsistency）：同一实体在不同属性或不同数据源中呈现相互矛盾的值，例如出生日期与年龄不符、同一客户ID对应两个不同地址，或跨系统的编码体系冲突（度量单位、日期格式、类别标签）。

**（4）重复性**（Duplication）：同一真实世界实体在数据集中被记录多次，形成**精确重复**（exact duplicate，所有字段完全一致）或**近似重复**（near-duplicate，因输入差异或格式变化而存在细微不同）。
{% endblock %}

上述四类问题在现实数据集中往往同时存在、相互叠加（如图 2.1.1 所示）。例如，对一个医疗数据集执行初步诊断，可能同时发现：血压字段缺失（不完整性）、身高记录为 `-1.70` 米（噪声/异常值）、出生日期格式混用 `YYYY/MM` 与 `MM-YYYY`（不一致性），以及同一患者被重复注册两次（重复性）。

### 质量评估的量化框架

在1.3节中，我们已经引入了数据质量的五个维度（见定义1.8）：完整性 $$\rho_c$$、唯一性 $$\rho_u$$、一致性 $$\rho_k$$、准确性与及时性。本节将这一框架落地为可计算的诊断指标。

对于数据集 $$\mathcal{D}$$，设 $$n$$ 为记录数，$$d$$ 为特征数，则：

$$
\rho_c = 1 - \frac{\#\{(i,j) : x_{ij} \text{ 缺失}\}}{n \times d}, \qquad
\rho_u = 1 - \frac{\#\text{精确重复行数}}{n}
\tag{2.1}
$$

当 $$\rho_c < 0.95$$ 时，通常需要执行显式的缺失值处理；当 $$\rho_u < 1$$ 时，需先执行去重操作。这两个阈值是工程实践中的经验规则，具体阈值应结合业务场景调整。

若某列特征的完整性 $$\rho_c = 0.40$$（即60%的值缺失），你会选择删除该列、还是尝试填补？在什么条件下删除是合理的？在什么条件下填补是必要的？请结合 MCAR/MAR/MNAR 的概念（见2.1.2节）来回答。
{% endhint %}

理解了质量问题的分类框架，下面分别讨论三种最核心的质量问题：缺失值、异常值与重复数据，每类都有其独特的统计理论基础。

## 2.1.2 缺失值的产生机制

缺失值的处理策略——究竟是删除、均值填补、还是建模预测——并非由数据科学家的偏好决定，而是由缺失值的**产生机制**决定。1976年，统计学家 Rubin 在其奠基性论文中提出了缺失数据的概率分类框架，该框架至今仍是处理缺失值的理论基石。

### 缺失指示矩阵

对于 $$n \times d$$ 的数据矩阵 $$\mathbf{X}$$，定义**缺失指示矩阵**（Missing Indicator Matrix）$$\mathbf{R} \in \{0,1\}^{n \times d}$$，其中：

$$
R_{ij} = \begin{cases} 0 & \text{若 } x_{ij} \text{ 缺失} \\ 1 & \text{若 } x_{ij} \text{ 已观测} \end{cases}
\tag{2.2}
$$

原始数据矩阵 $$\mathbf{X}$$ 可分拆为已观测部分 $$\mathbf{X}_{obs}$$（与 $$\mathbf{R}$$ 中为1的位置对应）和缺失部分 $$\mathbf{X}_{mis}$$（与 $$\mathbf{R}$$ 中为0的位置对应）。

{% hint style="mathbg" %}
📐 **数学基础：缺失机制的概率表达**

$$\mathbf{R}$$ 是一个随机变量，其分布由联合概率 $$P(\mathbf{R} \,|\, \mathbf{X}_{obs}, \mathbf{X}_{mis})$$ 刻画。不同假设下该条件概率的简化形式，决定了三种截然不同的缺失机制。核心问题是：$$\mathbf{R}$$ 的分布究竟与哪些量相关？
{% endhint %}

### 三种缺失机制

Rubin（1976）根据缺失概率 $$P(\mathbf{R} \mid \mathbf{X}_{obs}, \mathbf{X}_{mis})$$ 对已观测数据与缺失数据的依赖关系，将缺失机制分为三类。这一分类不仅构成缺失数据理论的基础，也直接决定了后续填补策略的可行性——只有正确识别缺失机制，才能选择无偏或统计一致的填补方法。

{% block type="definition" label="2.2" title="三种缺失数据机制（Three Missing Data Mechanisms，Rubin, 1976）" %}
设数据矩阵 $$\mathbf{X}$$ 分拆为已观测部分 $$\mathbf{X}_{obs}$$ 与缺失部分 $$\mathbf{X}_{mis}$$，缺失指示矩阵为 $$\mathbf{R}$$，则：

**（1）完全随机缺失**（Missing Completely At Random, **MCAR**）：

$$
P(\mathbf{R} \,|\, \mathbf{X}_{obs}, \mathbf{X}_{mis}) = P(\mathbf{R})
\tag{2.3}
$$

缺失概率与任何数据值无关，纯属随机事件。例：问卷中随机跳题、传感器随机失效。

**（2）随机缺失**（Missing At Random, **MAR**）：

$$
P(\mathbf{R} \,|\, \mathbf{X}_{obs}, \mathbf{X}_{mis}) = P(\mathbf{R} \,|\, \mathbf{X}_{obs})
\tag{2.4}
$$

缺失概率取决于已观测变量，但在控制已观测变量后，与缺失值本身无关。例：年龄较小的受访者倾向于不填写收入（缺失由可观测的"年龄"决定）。

**（3）非随机缺失**（Missing Not At Random, **MNAR**）：

$$
P(\mathbf{R} \,|\, \mathbf{X}_{obs}, \mathbf{X}_{mis}) \neq P(\mathbf{R} \,|\, \mathbf{X}_{obs})
\tag{2.5}
$$

缺失概率取决于缺失值本身。例：收入极高的受访者倾向于不填写收入（缺失由不可观测的"收入"决定）；高血压患者拒绝测量血压。
{% endblock %}

三种机制对应截然不同的统计性质（如图 2.1.2 所示）。MCAR 是最理想的情形，删除缺失记录不会引入偏差；MAR 下用已观测变量作为协变量的填补方法（如多重插补、KNN填补）是无偏的；MNAR 是最危险的情形，任何基于观测数据的填补都将产生系统偏差，需要领域知识辅助建模。

{% hint style="caution" %}
⚠️ **MNAR 是隐性陷阱**

MNAR 最危险之处在于**无法通过观测数据本身加以验证**——我们看不到缺失值的真实值，自然无法计算缺失概率与它的相关性。实践中识别 MNAR 需要依赖领域知识（例如医学研究中众所周知的"非应答偏差"）或专门的敏感性分析（Sensitivity Analysis）。如将 MNAR 误当 MAR 处理，填补模型将系统性地低估（或高估）缺失特征的真实分布，进而导致下游模型偏差。
{% endhint %}

### 机制诊断的实践方法

判断缺失机制的实用方法如下：首先，对每一缺失特征 $$j$$，构建辅助二分类变量 $$r_j = \mathbf{1}[R_{ij}=0]$$（缺失为1，已观测为0）；然后，以 $$r_j$$ 为因变量，对所有其他已观测特征进行逻辑斯蒂回归；若没有任何特征对 $$r_j$$ 有显著预测力，则支持 MCAR 假设；若部分已观测特征显著，则支持 MAR；若业务逻辑暗示缺失与缺失值本身相关，则需考虑 MNAR。这一诊断程序在第2.6节中将配合代码详细展开。

缺失机制的理论框架为策略选择提供了原则性指导；而在动用任何填补方法之前，还需要解决另一类更为显眼的数据质量问题——异常值。

## 2.1.3 异常值检测基础

### 异常值的类型

并非所有偏离"中心"的点都是异常值，异常值的定义本身具有相对性。统计学将异常值分为三大类型：**点异常**（point anomaly，单个数据点远离其余数据的整体分布）、**上下文异常**（contextual anomaly，在特定上下文中异常而整体不异常，例如夏季的低温记录）、**集体异常**（collective anomaly，单个点正常但一组点联合出现异常，例如网络攻击流量）。本节聚焦于最常见的点异常；上下文与集体异常将在第2.7节及第7章（时序数据挖掘）中系统讨论。

{% block type="definition" label="2.3" title="统计异常值（Statistical Outlier）" %}
设特征 $$j$$ 的观测值集合为 $$\{x_{1j}, x_{2j}, \ldots, x_{nj}\}$$，均值为 $$\mu_j$$，标准差为 $$\sigma_j$$。对于样本 $$x_{ij}$$，定义其 **Z-score** 为：

$$
z_{ij} = \frac{x_{ij} - \mu_j}{\sigma_j}
\tag{2.6}
$$

若 $$|z_{ij}| > \theta$$（阈值通常取 $$\theta = 3$$），则称 $$x_{ij}$$ 为**统计异常值**。

等价地，基于四分位距（Inter-Quartile Range，**IQR**）的 Tukey 围栏（Tukey's Fence）定义为：

$$
\text{下围栏} = Q_1 - 1.5 \times \mathrm{IQR}, \qquad
\text{上围栏} = Q_3 + 1.5 \times \mathrm{IQR}
\tag{2.7}
$$

其中 $$Q_1$$、$$Q_3$$ 分别为第一、第三四分位数，$$\mathrm{IQR} = Q_3 - Q_1$$。取值落在围栏之外的样本被标记为异常值。
{% endblock %}

### 三σ准则的数学基础

Z-score 方法的阈值 $$\theta=3$$ 并非任意选取，而是来源于正态分布的一个精确计算结果。

{% block type="theorem" label="2.1" title="三σ准则（Three-Sigma Rule）" %}
若 $$X \sim \mathcal{N}(\mu, \sigma^2)$$，则：

$$
P(|X - \mu| > 3\sigma) = 2\Phi(-3) \approx 0.0027
\tag{2.8}
$$

其中 $$\Phi$$ 为标准正态分布的累积分布函数。此外，正态分布的四分位距满足：

$$
\mathrm{IQR} = Q_3 - Q_1 = 2\,\Phi^{-1}(0.75)\,\sigma \approx 1.3490\,\sigma
\tag{2.9}
$$

因此，Tukey 围栏等价于约 $$\mu \pm 2.698\sigma$$ 的范围，在正态分布下约有 $$0.70\%$$ 的正常点被误判为异常。
{% endblock %}

{% hint style="proof" %}
**定理2.1的证明**

**公式(2.8)**：由正态分布的对称性，$$P(|X-\mu|>3\sigma) = 2P(X > \mu + 3\sigma) = 2(1-\Phi(3))$$。查标准正态表知 $$\Phi(3) = 0.99865$$，故 $$P = 2(1-0.99865) = 0.0027$$。

**公式(2.9)**：$$Q_3 = \mu + \Phi^{-1}(0.75)\sigma$$，$$Q_1 = \mu + \Phi^{-1}(0.25)\sigma = \mu - \Phi^{-1}(0.75)\sigma$$（利用对称性）。因此 $$\mathrm{IQR} = 2\Phi^{-1}(0.75)\sigma$$。由 $$\Phi^{-1}(0.75) \approx 0.6745$$，得 $$\mathrm{IQR} \approx 1.3490\sigma$$。■
{% endhint %}

### 两种方法的对比

完成了各自的数学推导后，以下通过一个具体数值例子，在同一数据集上并行展示 Z-score 与 IQR 两种检测方法的完整计算步骤，直观揭示二者对均值偏移型异常点的敏感程度差异。

{% block type="example" label="2.1" title="Z-score 与 IQR 的手动计算" %}
**数据**：某工厂质检数据中，某零件重量（克）的8个测量值为：

$$\{102, 105, 98, 103, 100, 106, 101, 160\}$$

**Z-score 计算**：

$$\mu = \frac{102+105+98+103+100+106+101+160}{8} = \frac{875}{8} = 109.375$$
$$\sigma = \sqrt{\frac{1}{8}\sum_{i=1}^8(x_i - \mu)^2} \approx 20.25$$

对 $$x=160$$：$$z = (160 - 109.375)/20.25 \approx 2.50 < 3$$。按 $$\theta=3$$ 未检出。

对 $$x=98$$：$$z = (98 - 109.375)/20.25 \approx -0.56$$。正常。

**IQR 计算**（排序后取四分位数）：

排序：$$\{98, 100, 101, 102, 103, 105, 106, 160\}$$

$$Q_1 = (100+101)/2 = 100.5$$，$$Q_3 = (105+106)/2 = 105.5$$，$$\mathrm{IQR} = 5.0$$

上围栏 $$= 105.5 + 1.5 \times 5.0 = 113.0$$

$$x=160 > 113.0$$，**IQR方法检出异常**。

**结论**：由于 $$x=160$$ 将均值拉高、标准差拉大（均值从约101拉至109），Z-score分子和分母同时增大，导致z值被稀释而未能检出。IQR基于分位数，对极端值不敏感，因此在此例中更为稳健。
{% endblock %}

{% hint style="caution" %}
⚠️ **Z-score 的隐藏假设**

Z-score 隐式假设数据服从**对称单峰分布**（通常是正态分布）。对于偏态分布（如收入、响应时间），Z-score 可能将正常的高值误判为异常；对于具有重尾的分布（如帕累托分布），阈值 $$\theta=3$$ 可能远远不足以识别真正的异常。诊断方法：在计算 Z-score 之前，先绘制直方图和 Q-Q 图检验正态性。如果分布明显非正态，应优先使用 IQR 方法或第2.7节的基于模型的方法（如 Isolation Forest）。
{% endhint %}

上述两种方法（如图 2.1.3 所示）均针对**单变量异常值**检测，且基于简单的统计阈值。当需要识别多变量数据中的异常——即在多个特征的联合分布中偏离的点——时，需要引入 Mahalanobis 距离等多元方法（见2.7节）。

{% hint style="supplement" %}
🔬 **补充：Mahalanobis 距离与多变量异常检测**

单变量 Z-score 无法捕捉特征间相关性引起的多变量异常。例如，"身高190cm，体重50kg"这一组合在各自的边缘分布中均属正常，但联合来看则极为异常。

**Mahalanobis 距离**将欧氏距离推广到考虑特征间协方差的形式：

$$D_M(\mathbf{x}) = \sqrt{(\mathbf{x} - \boldsymbol{\mu})^T \boldsymbol{\Sigma}^{-1} (\mathbf{x} - \boldsymbol{\mu})}$$

其中 $$\boldsymbol{\mu}$$ 为样本均值向量，$$\boldsymbol{\Sigma}$$ 为协方差矩阵。当 $$\boldsymbol{\Sigma} = \mathbf{I}$$（特征独立且方差为1）时，$$D_M$$ 退化为 Z-score 的多维推广——欧氏距离。对于 $$d$$ 维正态数据，$$D_M^2 \sim \chi^2(d)$$，因此可用 $$\chi^2$$ 临界值作为异常判别阈值。Mahalanobis 距离的完整推导与应用将在2.7节展开。
{% endhint %}

以上三σ准则和 IQR 方法处理的是单个数值异常。除了异常值之外，数据集中另一类常见但容易被忽视的质量问题是：同一实体被记录了多次——即重复数据。

## 2.1.4 重复数据的检测与消除

### 重复的来源与危害

重复数据的来源多种多样：CRM系统中客户多渠道注册（APP、网页、门店各创建一条记录）、数据爬取时对同一页面的多次抓取、ETL流程中对增量更新的错误判断、以及多数据源整合时的实体对齐失败。重复数据的危害是系统性的：它导致统计量（均值、频率）虚高，使聚类结果出现人为的"密集中心"，令监督学习模型对某些样本过度拟合。更隐蔽的是，若同一客户的两条记录进入训练集和测试集，则训练集与测试集不再独立，模型评估结果将被人为乐观化——这是一种严重的**数据泄露**形式。

{% block type="definition" label="2.4" title="重复记录与记录链接（Duplicate Record & Record Linkage）" %}
设数据集 $$\mathcal{D}$$ 中的两条记录 $$\mathbf{x}_i, \mathbf{x}_j$$（$$i \neq j$$）在现实世界中对应同一个实体，则称二者互为**重复记录**。具体地：

- **精确重复**（Exact Duplicate）：$$\mathbf{x}_i = \mathbf{x}_j$$，即所有字段值完全相同。
- **近似重复**（Near-Duplicate）：$$\mathbf{x}_i \neq \mathbf{x}_j$$，但两者之间的某种相似度量 $$\text{sim}(\mathbf{x}_i, \mathbf{x}_j) \geq \delta$$（相似度阈值），且业务逻辑支持二者指向同一实体。

**记录链接**（Record Linkage）是将来自不同数据源的记录通过近似匹配确定为同一实体的过程，也称**实体解析**（Entity Resolution）。
{% endblock %}

### 精确去重

精确去重最简单的实现是对全部字段的连接字符串计算哈希值（如 MD5 或 SHA-256），时间复杂度为 $$O(n \cdot d)$$（$$n$$ 条记录，$$d$$ 个字段）。在 pandas 中，`DataFrame.duplicated()` 方法基于此原理实现，`keep='first'` 参数保留每组重复记录中的第一条：

```python
df_clean = df.drop_duplicates(keep='first')
```

### 近似去重：排序邻域法

暴力近似去重需要对所有记录对计算相似度，时间复杂度为 $$O(n^2)$$，在大规模数据集上不可接受。**排序邻域法**（Sorted Neighborhood Method, SNM）是一种经典的近似去重加速策略。

{% block type="algorithm" label="2.1" title="排序邻域法（Sorted Neighborhood Method, Hernandez & Stolfo, 1995）" %}
**输入**：记录集 $$\mathcal{D} = \{r_1, \ldots, r_n\}$$，排序键 $$k(\cdot)$$，窗口大小 $$w$$，相似度函数 $$\text{sim}(\cdot, \cdot)$$，阈值 $$\delta$$

**输出**：重复对集合 $$\mathcal{P} \subseteq \mathcal{D} \times \mathcal{D}$$

**步骤：**

1. **构造排序键**：对每条记录 $$r_i$$ 提取排序键 $$k(r_i)$$（通常为姓名、ID等关键字段的规范化形式）；
2. **按键排序**：将 $$\mathcal{D}$$ 按 $$k(\cdot)$$ 升序排列，得到序列 $$r_{\pi(1)}, r_{\pi(2)}, \ldots, r_{\pi(n)}$$；
3. **滑动窗口比较**：维护大小为 $$w$$ 的窗口，对窗口内所有记录对计算相似度：

$$
\text{对 } i=1 \text{ 到 } n \text{，对 } j=\max(1, i-w+1) \text{ 到 } i-1：
$$
$$
\quad \text{若 } \text{sim}(r_{\pi(i)}, r_{\pi(j)}) \geq \delta，\text{ 则将 } (r_{\pi(i)}, r_{\pi(j)}) \text{ 加入 } \mathcal{P}
$$

4. **输出** $$\mathcal{P}$$，每对重复记录人工确认或自动合并。

**时间复杂度**：$$O(n \log n + n \cdot w)$$，当 $$w \ll n$$ 时近似为 $$O(n \log n)$$，远优于 $$O(n^2)$$。
{% endblock %}

排序邻域法的核心思想是：真正的近似重复记录在排序键下往往相邻，因此仅需在滑动窗口内比较，大幅降低无效比较次数。常用的字符串相似度函数包括：

- **编辑距离**（Levenshtein Distance）：$$d_{edit}(s_1, s_2)$$ = 将 $$s_1$$ 变换为 $$s_2$$ 所需的最少单字符操作数（插入、删除、替换）；
- **Jaccard 相似度**（基于 $$n$$-gram 集合）：$$J(A, B) = |A \cap B| / |A \cup B|$$；
- **余弦相似度**（基于 TF-IDF 或 embedding 向量）：适用于长文本字段。

{% block type="example" label="2.2" title="客户记录近似去重示例" %}
**数据**（CRM 系统 4 条客户记录）：

| ID | 姓名 | 手机号 | 邮箱 |
|----|------|--------|------|
| 1 | 张伟 | 138-0001-2345 | zhangwei@mail.com |
| 2 | Zhang Wei | 13800012345 | zhangwei@mail.com |
| 3 | 李明 | 139-0005-6789 | liming@work.cn |
| 4 | 张 伟 | 138-0001-2345 | zhangwei@gmail.com |

**精确去重**：无完全相同行，精确去重无效。

**排序邻域法**（排序键 = 姓名拼音 + 手机号数字部分，$$w=3$$）：

对记录 \{1, 2, 4\}，规范化后排序键相近（均含"zhangwei"和"13800012345"），进入同一窗口。计算编辑距离后，$$\text{sim}(\text{记录1}, \text{记录2}) \approx 0.91$$，$$\text{sim}(\text{记录1}, \text{记录4}) \approx 0.88$$，均超过阈值 $$\delta=0.85$$，被标记为近似重复对。

**最终判断**：记录1、2、4均指向同一真实客户，保留记录1，删除2和4。
{% endblock %}

{% hint style="engineering" %}
🔧 **工程实践：工业级数据清洗管道的设计原则**

在生产环境中，数据清洗绝非"一次性处理"，而是需要嵌入到持续运行的数据管道（Data Pipeline）中。工业界的最佳实践包含以下几个层次：

**数据契约（Data Contracts）**：在数据生产者与消费者之间建立明确的 Schema 约定，包含字段名称、类型、取值范围、唯一性约束与缺失率上限。若生产者违反契约，管道自动发出告警并拒绝该批次数据入库。Airbnb、Stripe 等公司已将数据契约作为数据工程的核心基础设施。

**声明式数据质量（Declarative Data Quality）**：使用 Great Expectations、dbt Tests 等工具，以代码形式定义数据质量断言（如 `expect_column_values_to_not_be_null(col='age')`），在每次数据更新时自动执行检验，生成 HTML 可视化质量报告。这使数据质量检验与代码审查、CI/CD 流程无缝集成。

**增量清洗与版本控制**：对清洗规则进行版本管理（DVC 或 Delta Lake），确保历史数据的清洗结果可复现。避免"手动清洗后覆盖原始数据"的反模式——原始数据永远不应被修改，清洗结果应作为独立版本保存。

**在线/离线一致性**：用于训练模型的离线清洗逻辑必须与线上服务时的实时清洗逻辑完全一致，否则将产生**特征服务漂移**（Feature Serving Skew），导致线上推理质量下降。实现方式：将清洗逻辑封装为与框架无关的函数，同时部署于离线批处理与在线实时推理服务。
{% endhint %}

理解了三种核心清洗技术（缺失值处理、异常值检测、重复数据消除）的理论基础，下面通过一个完整的真实数据集案例，展示如何将这些工具整合为系统性的数据质量诊断流程。

## 2.1.5 综合实验：Ames Housing 数据集质量诊断

Ames Housing 数据集以其多样化的预处理挑战而闻名于机器学习教育界，是验证数据质量诊断工具的理想载体：80 个异质特征涵盖数值型、有序类别型和名义类别型，缺失值语义复杂——部分"缺失"并非数据录入遗漏，而是表示"该房屋不具备该设施"。下面以该数据集为例，将前述定义 2.1—2.4 所涉及的完整性、唯一性、异常值三类质量指标整合为一条系统性诊断流程。

{% block type="case" label="2.1" title="Ames Housing 数据质量全链路诊断" %}
**业务背景**：Ames Housing 数据集（De Cock, 2011）是 Kaggle"房价预测"竞赛的官方数据集，记录了美国艾姆斯市2006–2010年的住宅销售信息。该数据集以其**极度多样化的预处理挑战**著称于机器学习教育界：80个特征覆盖数值型、有序类别型、名义类别型三种类型，缺失模式复杂，且各字段的缺失语义迥异——部分"缺失"并非数据录入遗漏，而是表示"该房屋不具备该特征"（如地下室类型缺失 = 无地下室）。

**数据规模**：1,460条训练记录，79个特征（不含ID和目标值 `SalePrice`）。

**挖掘目标**：本案例目标为**数据质量诊断**，而非预测建模。具体任务：（1）量化各类质量问题的规模；（2）识别具有不同缺失语义的列；（3）检测数值型特征中的潜在异常值；（4）验证无重复记录。
{% endblock %}

以下代码演示完整的数据质量诊断流程。

```python
# ============================================================
# 代码 2.1：Ames Housing 数据质量诊断
# 数据集：Ames Housing（De Cock, 2011），via scikit-learn
# 依赖库：pandas, numpy, matplotlib, scikit-learn
# ============================================================
# --- 1. 数据加载 ---
import numpy as np
import pandas as pd
import matplotlib
matplotlib.rcParams['font.sans-serif'] = ['SimHei']
matplotlib.rcParams['axes.unicode_minus'] = False
import matplotlib.pyplot as plt
from sklearn.datasets import fetch_openml

# 加载 Ames Housing（OpenML id=42165）
ames = fetch_openml(name="house_prices", as_frame=True, parser="auto")
df = ames.frame.copy()
print(f"数据形状：{df.shape}")           # (1460, 81)
print(f"特征类型分布：\n{df.dtypes.value_counts()}")

# --- 2. 完整性诊断：量化缺失规模 ---
miss = df.isnull().sum()
miss_rate = miss / len(df)
miss_df = pd.DataFrame({
    "缺失数量": miss,
    "缺失率": miss_rate
}).query("缺失数量 > 0").sort_values("缺失率", ascending=False)

print(f"\n含缺失的列数：{len(miss_df)} / {df.shape[1]}")
print(miss_df.head(10).to_string())

# 语义分析：某些"缺失"实为"无此特征"
# 例如：PoolQC缺失 = 无泳池；Alley缺失 = 无小巷通道
semantic_na_cols = ['PoolQC', 'MiscFeature', 'Alley', 'Fence',
                    'FireplaceQu', 'GarageType', 'GarageFinish',
                    'GarageQual', 'GarageCond', 'BsmtExposure',
                    'BsmtFinType1', 'BsmtFinType2', 'BsmtQual', 'BsmtCond']
print(f"\n语义缺失（无此特征）列数：{len(semantic_na_cols)}")
true_missing = miss_df[~miss_df.index.isin(semantic_na_cols)]
print(f"真实缺失列数：{len(true_missing)}")

# --- 3. 可视化缺失分布 ---
fig, ax = plt.subplots(figsize=(10, 5))
top_miss = miss_df.head(20)
colors = ['#dc2626' if c not in semantic_na_cols else '#2563eb'
          for c in top_miss.index]
ax.barh(top_miss.index[::-1], top_miss["缺失率"][::-1], color=colors[::-1])
ax.axvline(0.5, color='#64748b', ls='--', lw=1.2, label='50% 缺失线')
ax.set_xlabel("缺失率", fontsize=11)
ax.set_title("Ames Housing：缺失率 Top-20 特征", fontsize=13)
# 图例
import matplotlib.patches as mpatches
patch1 = mpatches.Patch(color='#dc2626', label='真实缺失（需填补）')
patch2 = mpatches.Patch(color='#2563eb', label='语义缺失（表示无此特征）')
ax.legend(handles=[patch1, patch2], fontsize=9)
plt.tight_layout()
plt.savefig("missing_ames.png", dpi=150)
plt.show()

# --- 4. 唯一性检查：精确去重 ---
n_dup = df.duplicated().sum()
print(f"\n精确重复行数：{n_dup}")         # 预期输出：0

# 检查 Id 字段唯一性
print(f"Id 唯一数：{df['Id'].nunique()}，总行数：{len(df)}")

# --- 5. 异常值检测：Z-score（数值型特征）---
num_cols = df.select_dtypes(include=np.number).columns.tolist()
num_cols = [c for c in num_cols if c not in ['Id', 'SalePrice']]

z_scores = df[num_cols].apply(
    lambda col: (col - col.mean()) / col.std()
)
outlier_counts = (z_scores.abs() > 3).sum().sort_values(ascending=False)
print(f"\n各数值列 Z-score > 3 的异常值数量（Top-10）：")
print(outlier_counts[outlier_counts > 0].head(10).to_string())

# 举例：LotArea 的异常值详情
lot = df['LotArea'].dropna()
z_lot = (lot - lot.mean()) / lot.std()
extreme_lots = df.loc[z_lot[z_lot.abs() > 3].index, ['LotArea', 'SalePrice']]
print(f"\nLotArea 异常值（|z|>3）样本数：{len(extreme_lots)}")
print(extreme_lots.head(5).to_string())

# --- 6. 质量摘要报告 ---
total_cells = df.shape[0] * df.shape[1]
total_missing = df.isnull().sum().sum()
rho_c = 1 - total_missing / total_cells
rho_u = 1 - n_dup / len(df)

print(f"\n{'='*40}")
print(f"  Ames Housing 数据质量报告")
print(f"{'='*40}")
print(f"  记录数    ：{df.shape[0]}")
print(f"  特征数    ：{df.shape[1]}")
print(f"  完整性 ρc ：{rho_c:.4f}")
print(f"  唯一性 ρu ：{rho_u:.4f}")
print(f"  含缺失列  ：{(miss > 0).sum()} 列")
print(f"  Z>3 异常值：{(z_scores.abs() > 3).sum().sum()} 个单元格")
print(f"{'='*40}")
```

**运行结果说明**：数据集完整性 $$\rho_c \approx 0.943$$（略低于工程推荐阈值0.95），唯一性 $$\rho_u = 1.000$$（无精确重复）。缺失最严重的是 `PoolQC`（99.5%缺失，语义为"无泳池"）和 `Alley`（93.8%缺失，语义为"无小巷通道"）——这类"语义缺失"不应被填补为统计量，而应编码为"无此特征"的类别标签。真实数据录入缺失最多的是 `LotFrontage`（17.7%缺失），可考虑基于 `Neighborhood`（邻区）的分组均值填补（MAR假设：缺失与邻区相关，与地块面积本身无关）。

{% hint style="tip" %}
💡 **技巧：缺失语义的识别原则**

对于任意一列的缺失值，首先查阅数据字典（Data Dictionary），判断缺失是"真实未记录"还是"特征不适用"。规则如下：

- 若字段描述为"X 的类型/质量"（如 `GarageType`、`PoolQC`），其缺失通常意味着"无 X"，应填充为固定字符串 `"None"` 或类别值 `0`；
- 若字段描述为"X 的测量值"（如 `LotFrontage` = 街道接入长度），缺失可能是真实的录入遗漏，需结合相关特征建模填补。

混淆这两类缺失会严重污染特征空间——将"无泳池"填为整体均值，相当于为无泳池房屋虚构了一个泳池质量分，这对任何下游模型都是噪声。
{% endhint %}

{% hint style="frontier" %}
🚀 **前沿视野：大语言模型驱动的自动化数据清洗**

传统数据清洗高度依赖领域专家手工编写规则。2023–2024年，多项研究探索了将大语言模型（LLM）引入数据清洗流程的新范式：

**语义缺失检测**：GPT-4 和 Claude 等模型能够阅读数据集文档，自动区分"语义缺失"与"真实缺失"（如上述 Ames Housing 案例），准确率在基准测试中超过传统基于规则的方法。HoloClean（2017）的后继工作将 LLM 用于修复矛盾约束（inconsistency），在 BART 清洗基准上实现了 85% 以上的精确率。

**基于 GPT 的缺失值填补**：Narayan et al.（2022）提出 GPTCELL，利用 GPT 系列模型进行多模态数据的上下文驱动填补，在医学表格数据集上比 KNNImputer 提升 12-18 个百分点。其核心思想：语言模型可以利用字段名称、数据类型和已有值的语义信息，以类似"表格推理"的方式推断合理的填补值。

**挑战与局限**：LLM 驱动的数据清洗面临计算成本高（对每条记录调用 API 费用可观）、一致性差（同一缺失值多次推断结果可能不同）和不可审计（无法解释为何选择某填补值）等挑战。当前主流实践是将 LLM 用于规则生成（自动生成 Great Expectations 断言），而非直接操作数据。

**展望**：随着小型化 LLM（如 Llama-3-8B）的推理效率提升，端到端的 LLM 数据清洗 Pipeline 有望在2026年前成为中小规模数据集的标准工具。
{% endhint %}

{% hint style="ideology" %}
🌏 **课程思政：数据质量是一个社会公平问题**

数据质量问题从来不是纯粹的技术问题，其背后往往隐藏着社会结构性不平等。

**不平衡的数据缺失**：在美国 COMPAS 累犯预测系统（见定义1.6的延伸讨论）中，少数族裔群体的历史司法记录更为完整（因为他们受到更严格的执法监控），而富裕白人群体的某些行为记录更易缺失。这种**系统性的缺失不对称**（MNAR），导致基于该数据训练的预测模型继承并放大了执法偏见，而非校正它。

**医疗数据的质量鸿沟**：在中国农村地区，电子病历系统的覆盖率和数据质量显著低于城市三甲医院。以这类数据训练的疾病预测模型，在城市医疗场景中表现良好，但对农村患者的预测精度可能大幅下降——这是由数据质量不均衡导致的**模型公平性缺失**。

**治理层面的响应**：中国《数据安全法》（2021）和《个人信息保护法》（2021）明确要求数据处理者保证数据的"准确性"和"完整性"，这在法律层面将数据质量责任从技术维度提升至合规维度。"数据精准"不再只是工程追求，而是法律义务。

作为数据科学工作者，清洗数据时应始终追问：**谁的数据被更多清洗掉了？谁的数据质量更差？这种差异是否会在模型中系统性地歧视某一群体？**
{% endhint %}

## 本节小结

本节建立了数据清洗的完整理论框架。数据质量问题按照**不完整性、噪声、不一致性、重复性**四维分类（定义2.1）；缺失值问题的核心是产生机制的识别——**MCAR、MAR、MNAR**三种机制对应截然不同的统计性质与填补策略（定义2.2）；异常值检测的基础工具是 **Z-score** 和 **IQR**，两者均有严格的数学基础（定理2.1），但对分布形态假设各有依赖；重复数据的处理需要区分精确去重（哈希方法，$$O(nd)$$）和近似去重（排序邻域法，$$O(n\log n + nw)$$，定义2.4，算法2.1）。

这些工具构成第2.6节（缺失值高级填补）和第2.7节（异常值系统检测）的理论前提，也是后续一切数据挖掘任务的基础起点。

### B 级（数学推导）

**B1.** （推导）在 MCAR 假设下，证明对缺失值使用**观测样本均值**进行填补，不会引入关于总体均值的有偏估计。即：设 $$\{x_1, \ldots, x_n\}$$ 为某特征的真实值，$$\mathbf{R} = (R_1, \ldots, R_n)$$ 为缺失指示向量，MCAR 条件为 $$P(R_i = 0) = p$$（常数，与 $$x_i$$ 无关）。设 $$\bar{x}_{obs} = \frac{1}{n_{obs}}\sum_{i:R_i=1} x_i$$（观测均值），证明 $$\mathbb{E}[\bar{x}_{obs}] = \mu$$（总体均值）。

**B2.** （证明）正态分布下，IQR 与标准差 $$\sigma$$ 的关系为 $$\mathrm{IQR} = 2\Phi^{-1}(0.75)\cdot\sigma \approx 1.3490\sigma$$（定理2.1的公式(2.9)）。请给出完整推导，并利用此结论计算：若对正态数据使用 Tukey 围栏 $$Q_1 - 1.5\cdot\mathrm{IQR}$$ 到 $$Q_3 + 1.5\cdot\mathrm{IQR}$$，则被标记为异常的正常点的理论比例是多少？

**B3.** （证明）对于多元正态分布 $$\mathbf{X} \sim \mathcal{N}(\boldsymbol{\mu}, \boldsymbol{\Sigma})$$，证明 Mahalanobis 距离的平方服从 $$\chi^2$$ 分布，即 $$D_M^2(\mathbf{x}) = (\mathbf{x}-\boldsymbol{\mu})^T\boldsymbol{\Sigma}^{-1}(\mathbf{x}-\boldsymbol{\mu}) \sim \chi^2(d)$$，其中 $$d$$ 为维度数。（提示：利用 $$\boldsymbol{\Sigma}$$ 的特征值分解 $$\boldsymbol{\Sigma} = \mathbf{V}\mathbf{\Lambda}\mathbf{V}^T$$，构造标准正态向量。）
