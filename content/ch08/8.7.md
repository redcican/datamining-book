# 8.7 时间序列异常检测

上一节（§8.6）介绍了时间序列预测模型——利用历史模式预测未来值。本节将预测能力应用于一个关键任务：**异常检测**——识别时间序列中偏离正常模式的数据点或片段。

时间序列异常检测与第七章（§7.1–§7.7）介绍的一般异常检测有本质区别：**时间上下文**。在表格数据中，一个数据点是否异常只取决于它与其他样本的关系。但在时间序列中，同一个值在不同时间可以是正常或异常的——例如，服务器 CPU 使用率 95% 在白天高峰时段是正常的，但在凌晨 3 点则高度可疑。异常的判断必须考虑时间上下文。

本节介绍四种互补的检测方法。**统计控制图**适合稳态过程的在线监控；**基于预测的方法**利用预测模型的残差检测异常；**Matrix Profile Discord** 以无监督方式发现最独特的子序列；**自编码器方法**通过重构误差捕获复杂的非线性异常模式。

## 8.7.1 时间序列异常的分类

### 8.7.1.1 三种异常类型

时间序列中的异常比表格数据中的异常更加多样。根据异常的时间范围和上下文依赖性，可以分为三种类型。

{% block type="definition" title="定义 8.37　点异常（Point Anomaly）" %}
时间序列 $$\{x_1, \ldots, x_T\}$$ 中的**点异常**是单个时间步 $$t^*$$ 处的观测值 $$x_{t^*}$$ 显著偏离其预期值：

$$
|x_{t^*} - \mathbb{E}[x_{t^*} \mid x_{t^*-1}, x_{t^*-2}, \ldots]| > \tau \tag{8.106}
$$

其中 $$\tau$$ 为检测阈值。点异常是最简单的异常类型——单个值"太大"或"太小"。典型场景：传感器故障导致的尖峰值。
{% endblock %}

{% block type="definition" title="定义 8.38　上下文异常（Contextual Anomaly）" %}
观测值 $$x_{t^*}$$ 本身在全局范围内不算异常，但在其**时间上下文**中是异常的：

$$
x_{t^*} \in [\mu - 3\sigma, \mu + 3\sigma] \quad \text{但} \quad x_{t^*} \notin [\mu_{\text{ctx}} - 3\sigma_{\text{ctx}}, \mu_{\text{ctx}} + 3\sigma_{\text{ctx}}] \tag{8.107}
$$

其中 $$\mu, \sigma$$ 为全局统计量，$$\mu_{\text{ctx}}, \sigma_{\text{ctx}}$$ 为局部上下文的统计量。典型场景：夏季用电高峰期间突然出现冬季水平的低负荷。
{% endblock %}

{% block type="definition" title="定义 8.39　集体异常（Collective Anomaly）" %}
一组**连续**的观测值 $$\{x_{t^*}, x_{t^*+1}, \ldots, x_{t^*+L-1}\}$$ 构成的子序列整体上是异常的，但其中每个单独的点可能都不是点异常：

$$
d(T_{t^*,L}, \text{正常模式}) > \tau_{\text{coll}} \tag{8.108}
$$

其中 $$T_{t^*,L}$$ 为从 $$t^*$$ 开始、长度为 $$L$$ 的子序列。集体异常关注的是**形状和模式的异常**，而非单个值的大小。典型场景：心电图中一段异常的心律波形。
{% endblock %}

三种异常类型需要不同的检测方法：点异常可以用阈值法检测；上下文异常需要建立时间上下文模型；集体异常需要子序列级别的比较（如 Matrix Profile Discord）。

## 8.7.2 统计控制图方法

### 8.7.2.1 Shewhart 控制图

统计过程控制（SPC）是工业质量管理中的经典方法。**Shewhart 控制图**基于 3-sigma 规则：当过程处于统计控制状态时，观测值落在 $$\mu \pm 3\sigma$$ 之外的概率仅为 0.27%。

对于平稳时间序列，Shewhart 控制图的上下控制限为：

$$
\text{UCL} = \bar{x} + 3s, \quad \text{LCL} = \bar{x} - 3s \tag{8.109}
$$

其中 $$\bar{x}$$ 和 $$s$$ 由训练期（正常运行期）的数据估计。超出控制限的点被标记为异常。

Shewhart 控制图对**突发性大偏移**非常敏感（因为单个点就能超出 3-sigma），但对**缓慢漂移**反应迟钝——过程均值缓慢偏移 1-sigma 时，单个点很少超出控制限，但整体质量已经恶化。

### 8.7.2.2 CUSUM 控制图

{% block type="definition" title="定义 8.40　CUSUM 统计量" %}
**累积和**（CUSUM, Cumulative Sum）统计量通过累积残差来检测均值的持续偏移。定义上侧和下侧 CUSUM：

$$
C_t^+ = \max(0,\; C_{t-1}^+ + (x_t - \mu_0) - k) \tag{8.110}
$$

$$
C_t^- = \max(0,\; C_{t-1}^- - (x_t - \mu_0) - k) \tag{8.111}
$$

其中 $$\mu_0$$ 为过程的目标均值，$$k > 0$$ 为**松弛参数**（Slack Value，通常设为 $$\delta\sigma/2$$，$$\delta$$ 为期望检测的偏移量）。初始值 $$C_0^+ = C_0^- = 0$$。

当 $$C_t^+ > h$$ 或 $$C_t^- > h$$ 时发出告警，其中 $$h$$ 为**决策区间**（Decision Interval）。
{% endblock %}

CUSUM 的核心思想是"累积证据"：如果过程均值向上偏移，每个残差 $$(x_t - \mu_0)$$ 倾向于正值，$$C_t^+$$ 持续增长，最终突破阈值。松弛参数 $$k$$ 的作用是过滤正常随机波动——只有当偏移超过 $$k$$ 时才开始累积。

### 8.7.2.3 EWMA 控制图

{% block type="definition" title="定义 8.41　EWMA 统计量" %}
**指数加权移动平均**（EWMA）统计量对近期观测赋予更高权重：

$$
Z_t = \lambda x_t + (1 - \lambda) Z_{t-1}, \quad Z_0 = \mu_0 \tag{8.112}
$$

其中 $$\lambda \in (0, 1]$$ 为平滑参数。EWMA 控制限为：

$$
\text{UCL}_t = \mu_0 + L \cdot \sigma \sqrt{\frac{\lambda}{2 - \lambda}\left[1 - (1-\lambda)^{2t}\right]} \tag{8.113}
$$

其中 $$L$$ 为控制限宽度参数（通常 $$L \approx 3$$）。$$\lambda$$ 越小，EWMA 对历史数据的记忆越长，对小偏移的检测越敏感。
{% endblock %}

{% block type="theorem" title="定理 8.13　平均运行长度（ARL）" %}
控制图的性能通过**平均运行长度**（Average Run Length, ARL）衡量：

$$
\text{ARL}_0 = \frac{1}{\alpha} \tag{8.114}
$$

其中 $$\alpha$$ 为单次检验的误报率。对于 Shewhart 控制图（3-sigma），$$\alpha = 0.0027$$，因此 $$\text{ARL}_0 \approx 370$$——平均每 370 个时间步产生一次误报。

当过程均值偏移 $$\delta\sigma$$ 时，检测到偏移所需的平均时间步数为 $$\text{ARL}_1(\delta)$$。好的控制图应满足：

$$
\text{ARL}_0 \text{ 大（少误报）}, \quad \text{ARL}_1(\delta) \text{ 小（快检测）} \tag{8.115}
$$

CUSUM 和 EWMA 在检测小偏移（$$\delta < 2$$）时的 $$\text{ARL}_1$$ 显著小于 Shewhart 控制图，因为它们通过累积信息放大了微小偏移的信号。$$\blacksquare$$
{% endblock %}

## 8.7.3 基于预测的异常检测

### 8.7.3.1 残差异常评分

统计控制图假设过程是平稳的——均值和方差不随时间变化。但许多真实时间序列具有趋势、季节性和复杂的非线性模式。此时，更有效的方法是：先用预测模型捕获正常模式，然后将**预测残差**作为异常信号。

{% block type="definition" title="定义 8.42　基于预测的异常评分" %}
给定时间序列 $$\{x_1, \ldots, x_T\}$$ 和预测模型 $$f$$，时刻 $$t$$ 的**异常评分**定义为标准化预测残差：

$$
a_t = \frac{|x_t - \hat{x}_t|}{\hat{\sigma}_t} \tag{8.116}
$$

其中 $$\hat{x}_t = f(x_{t-1}, x_{t-2}, \ldots)$$ 为模型的单步预测值，$$\hat{\sigma}_t$$ 为预测误差的局部标准差（可通过滑动窗口估计）。当 $$a_t > \tau$$ 时，时刻 $$t$$ 被标记为异常。

自适应阈值可通过预测误差的分布动态调整：

$$
\tau_t = Q_{1-\alpha}(\{a_{t-W}, \ldots, a_{t-1}\}) \tag{8.117}
$$

其中 $$Q_{1-\alpha}$$ 为最近 $$W$$ 个评分值的 $$(1-\alpha)$$ 分位数。
{% endblock %}

基于预测的方法有一个重要优势：**任何预测模型都可以使用**。简单场景用 ARIMA，复杂场景用 LSTM 或 Transformer——异常检测的质量直接取决于预测模型对"正常模式"的拟合能力。

{% block type="algorithm" title="算法 8.7　滑动窗口异常检测" %}
**输入**：时间序列 $$\{x_1, \ldots, x_T\}$$，预测模型 $$f$$，窗口大小 $$W$$，显著性水平 $$\alpha$$

**输出**：异常时刻集合 $$\mathcal{A}$$

**步骤 1（初始化）**：
$$\quad$$ $$\mathcal{A} \leftarrow \emptyset$$
$$\quad$$ 用前 $$W$$ 个时间步初始化预测模型

**步骤 2（逐步检测）**：
$$\quad$$ **for** $$t = W + 1$$ **to** $$T$$ **do**
$$\quad\quad$$ $$\hat{x}_t \leftarrow f(x_{t-1}, \ldots, x_{t-W})$$
$$\quad\quad$$ $$e_t \leftarrow x_t - \hat{x}_t$$
$$\quad\quad$$ $$\hat{\sigma}_t \leftarrow \text{std}(e_{t-W}, \ldots, e_{t-1})$$
$$\quad\quad$$ $$a_t \leftarrow |e_t| / \hat{\sigma}_t$$
$$\quad\quad$$ $$\tau_t \leftarrow Q_{1-\alpha}(\{a_{t-W}, \ldots, a_{t-1}\})$$
$$\quad\quad$$ **if** $$a_t > \tau_t$$ **then** $$\mathcal{A} \leftarrow \mathcal{A} \cup \{t\}$$
$$\quad$$ **end for**

**步骤 3（返回）**：返回 $$\mathcal{A}$$
{% endblock %}

{% hint style="warning" %}
⚠️ **易错点：训练数据中的异常污染**

基于预测的方法假设训练数据是"正常的"——模型学习的是正常模式。如果训练数据中包含未标记的异常，模型会将异常模式也当作正常来学习，导致真正的异常被漏检。解决方案：(1) 先用简单方法（如 3-sigma）粗筛训练数据中的明显异常；(2) 使用鲁棒估计（如中位数替代均值）；(3) 迭代训练——第一轮检测异常，移除后重新训练。
{% endhint %}

## 8.7.4 基于表示学习的异常检测

### 8.7.4.1 自编码器重构误差

当时间序列的正常模式过于复杂（多变量、非线性交互），传统预测模型难以充分建模时，**自编码器**（Autoencoder）提供了一种"学习正常模式的压缩表示"的方法。

{% block type="definition" title="定义 8.43　基于重构的异常评分" %}
自编码器将输入窗口 $$\mathbf{x}_{t-W:t} = (x_{t-W}, \ldots, x_t)$$ 编码为低维表示 $$\mathbf{z} = g_{\text{enc}}(\mathbf{x}_{t-W:t})$$，再解码重构 $$\hat{\mathbf{x}}_{t-W:t} = g_{\text{dec}}(\mathbf{z})$$。**重构误差**作为异常评分：

$$
a_t = \|\mathbf{x}_{t-W:t} - \hat{\mathbf{x}}_{t-W:t}\|^2 \tag{8.118}
$$

训练阶段仅使用正常数据，使自编码器学习正常模式的低维流形。当输入包含异常模式时，自编码器无法准确重构（因为训练时没见过），重构误差增大。
{% endblock %}

自编码器方法的优势在于：(1) 无需手工设计特征或预测模型；(2) 自然支持多变量时间序列——编码器可以学习变量之间的正常交互模式；(3) 对集体异常敏感——整个窗口的重构质量反映了模式级别的异常。

### 8.7.4.2 变分自编码器与概率异常检测

标准自编码器给出的是确定性的重构误差。**变分自编码器**（VAE）进一步建模潜在空间的概率分布，给出异常的概率估计：

$$
a_t = -\log p(\mathbf{x}_{t-W:t} \mid \mathbf{z}) + D_{\text{KL}}(q(\mathbf{z} \mid \mathbf{x}) \| p(\mathbf{z})) \tag{8.119}
$$

其中第一项衡量重构质量，第二项衡量潜在表示偏离先验分布的程度。当输入是异常的，两项都倾向于增大。

## 8.7.5 方法对比与选择

### 8.7.5.1 四种方法的比较

| 方法 | 异常类型 | 是否需要标签 | 适用场景 | 复杂度 |
|------|---------|------------|---------|--------|
| Shewhart 控制图 | 点异常 | 否（需正常基线） | 平稳过程的大偏移 | $$O(1)$$ 每步 |
| CUSUM/EWMA | 点异常（小偏移） | 否（需正常基线） | 均值漂移检测 | $$O(1)$$ 每步 |
| 预测残差 | 点异常 + 上下文 | 否 | 有模式的序列 | 取决于预测模型 |
| Matrix Profile Discord | 集体异常 | 否 | 子序列级异常 | $$O(n^2)$$ |
| 自编码器 | 所有类型 | 否（需正常训练数据） | 复杂多变量序列 | $$O(W \cdot d)$$ 每步 |

### 8.7.5.2 选择指南

方法选择取决于三个关键因素：

1. **异常类型**：点异常 → 控制图或预测残差；上下文异常 → 预测残差；集体异常 → Matrix Profile 或自编码器。
2. **序列特征**：平稳序列 → 控制图最简单有效；有趋势/季节性 → 预测残差；复杂非线性 → 自编码器。
3. **实时性要求**：在线检测 → 控制图或预测残差（$$O(1)$$ 每步）；离线分析 → Matrix Profile。

{% hint style="info" %}
🤔 **思考：异常检测中的"正常"定义**

所有无监督异常检测方法都隐含一个假设：训练数据代表了"正常"。但什么是"正常"？在服务器监控中，正常可能包括日常的负载波动和定期的维护窗口。如果模型在节假日训练，会把低负载当作正常，在工作日就会大量误报。因此，**正常基线的选择**是异常检测系统成功的关键——它需要覆盖所有正常的运行模式（工作日/周末/节假日、高峰/低谷等）。
{% endhint %}

## 8.7.6 案例分析

{% block type="case" title="案例 8.7　服务器性能指标异常检测" %}

**问题背景**：云计算平台需要实时监控数千台服务器的性能指标（CPU 使用率、内存占用、网络流量等），自动识别异常事件——如内存泄漏（缓慢上升的集体异常）、突发流量（点异常）、服务降级（上下文异常）。我们使用合成的 CPU 使用率数据演示多方法融合的异常检测系统。

**数据说明**：模拟 7 天的分钟级 CPU 使用率数据（10,080 个时间步），包含日周期模式和三种注入异常。

**方法与步骤**：

```python
# --- 1. 生成合成 CPU 数据 ---
import numpy as np
np.random.seed(42)
T = 10080  # 7 天 × 24 × 60
t = np.arange(T)
# --- 2. 正常模式：日周期 + 随机波动 ---
daily = 30 * np.sin(2 * np.pi * t / 1440 - np.pi / 2) + 50
noise = np.random.normal(0, 3, T)
cpu = daily + noise
cpu = np.clip(cpu, 0, 100)
# --- 3. 注入三种异常 ---
# (a) 点异常：3 个突发尖峰
spike_locs = [2000, 5500, 8000]
for loc in spike_locs:
    cpu[loc] = 98
# (b) 上下文异常：凌晨异常高负载
ctx_start = 3 * 1440 + 180  # 第4天凌晨3点
cpu[ctx_start:ctx_start + 60] += 25
# (c) 集体异常：缓慢内存泄漏模式
leak_start = 5 * 1440
leak_len = 360  # 6 小时
cpu[leak_start:leak_start + leak_len] += np.linspace(0, 20, leak_len)
# --- 4. 方法 1：Shewhart 控制图 ---
mu_train = cpu[:1440].mean()
sigma_train = cpu[:1440].std()
ucl = mu_train + 3 * sigma_train
shewhart_anomalies = np.where(cpu > ucl)[0]
# --- 5. 方法 2：预测残差 ---
# 简化：使用滑动窗口均值预测
W = 60
predictions = np.convolve(cpu, np.ones(W)/W, mode='same')
residuals = np.abs(cpu - predictions)
residual_std = np.std(residuals[:1440])
pred_anomalies = np.where(residuals > 3 * residual_std)[0]
# --- 6. 方法 3：CUSUM ---
k = 0.5 * sigma_train
h = 4 * sigma_train
C_plus = np.zeros(T)
cusum_anomalies = []
for i in range(1, T):
    C_plus[i] = max(0, C_plus[i-1] + (cpu[i] - mu_train) - k)
    if C_plus[i] > h:
        cusum_anomalies.append(i)
        C_plus[i] = 0  # 重置
# --- 7. 评估 ---
print(f"Shewhart: {len(shewhart_anomalies)} alerts")
print(f"Prediction: {len(pred_anomalies)} alerts")
print(f"CUSUM: {len(cusum_anomalies)} alerts")
```

**结果分析**：

1. **Shewhart 控制图**：成功检测了 3 个尖峰异常（点异常），但对上下文异常和集体异常完全漏检——因为凌晨异常高负载和缓慢泄漏的瞬时值没有超出全局 3-sigma 限。

2. **预测残差方法**：检测到了尖峰异常和上下文异常（凌晨的实际值与周期预测值偏差大），但对缓慢泄漏的敏感度不足——因为泄漏速率低，单步残差不显著。

3. **CUSUM**：成功检测到缓慢泄漏——累积偏移逐步增大，最终触发告警。也检测到了尖峰异常（立即触发）。但对上下文异常的敏感度取决于 $$\mu_0$$ 是否考虑了日周期。

4. **方法融合**：三种方法的检测能力互补。实践中的最佳策略是**多方法融合**——将多个检测器的输出聚合（如取并集或加权投票），以覆盖不同类型的异常。

{% hint style="tip" %}
🔧 **工程实践：异常检测系统的部署**

生产级异常检测系统需要考虑：(1) **告警疲劳**——误报太多导致运维人员忽略告警，通常通过调高阈值或要求连续 $$k$$ 次超阈值才告警来缓解；(2) **概念漂移**——正常模式随时间变化（如业务增长导致基线上升），检测器需要定期重新训练或使用在线学习；(3) **多指标关联**——CPU、内存、网络流量之间存在正常的相关性，单指标异常可能是多指标联合异常的表征。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 点异常（定义 8.37） | 单个时间步的极端值 |
| 上下文异常（定义 8.38） | 值在时间上下文中异常，但全局不极端 |
| 集体异常（定义 8.39） | 连续子序列的整体模式异常 |
| CUSUM（定义 8.40） | 累积残差检测小偏移，松弛参数过滤噪声 |
| EWMA（定义 8.41） | 指数加权平滑统计量，对近期偏移敏感 |
| ARL（定理 8.13） | 平均运行长度，衡量误报率与检测速度的权衡 |
| 预测残差评分（定义 8.42） | 标准化残差 + 自适应阈值 |
| 重构误差（定义 8.43） | 自编码器重构正常模式，异常导致误差增大 |
| 滑动窗口检测（算法 8.7） | 在线逐步检测，$$O(1)$$ 每步 |

**E2.** 在工业设备振动数据上：(1) 将振动信号的 RMS 值作为监控指标；(2) 使用 CUSUM 和 Matrix Profile Discord 分别检测设备退化和突发故障；(3) 讨论两种方法在预测性维护中的互补作用。

### F 级（开放探讨）
