# 8.2 时间序列预处理方法

上一节（§8.1）建立了时间序列分析的数学基础——形式化定义、成分分解、平稳性理论和自相关分析。有了这些工具，我们已经能够"读懂"一条时间序列的基本特征。然而，真实世界的时间序列数据往往存在各种质量问题：传感器故障导致数据缺失、电磁干扰引入高频噪声、不同系统的采样频率不一致、量纲差异使得多条时间序列难以直接比较。

时间序列的预处理与第二章（§2.1–§2.6）介绍的表格数据预处理有本质区别：**时间顺序必须被严格保护**。在表格数据中，标准化可以使用全局均值和标准差；但在时间序列中，使用未来数据来标准化过去的观测会造成**数据泄露**（Data Leakage），导致模型在回测中表现优异但在实际预测中失效。类似地，缺失值插补必须考虑时间连续性，不能像表格数据那样简单地用全局均值填充。

本节系统介绍时间序列预处理的六大环节：缺失值处理、平滑降噪、重采样、标准化、差分去趋势和异常值修正。每个环节都从"为什么需要"出发，给出数学定义和算法，最后在案例中串联为完整管线。

## 8.2.1 缺失值处理

### 8.2.1.1 时间序列缺失的特殊性

表格数据中的缺失值通常用全局统计量（均值、众数）或 KNN 插补填充——因为样本之间独立，填充顺序无关紧要。时间序列的缺失值处理必须尊重**时间连续性**：(1) 填充值必须在时间上合理——不能让气温从 20°C 突跳到 5°C 再跳回 20°C；(2) 插补只能使用**过去和当前已知的数据**，不能使用未来信息（在预测场景中）。

{% block type="definition" title="定义 8.9　时间序列的缺失模式" %}
时间序列 $$\{x_1, x_2, \ldots, x_T\}$$ 中的缺失可分为三种模式：

**（1）随机缺失**（Random Missing）：缺失位置随机分布，缺失概率与数据值无关。设缺失指示变量 $$m_t \in \{0, 1\}$$：

$$
P(m_t = 1) = p, \quad \text{独立于 } x_t \text{ 和 } t \tag{8.22}
$$

**（2）连续缺失**（Block Missing）：连续多个时间步的数据同时缺失，形成"空洞"：

$$
x_{t_0}, x_{t_0+1}, \ldots, x_{t_0+L-1} \text{ 均缺失} \tag{8.23}
$$

其中 $$L$$ 为空洞长度。连续缺失常由设备故障或通信中断引起，插补难度远高于随机缺失。

**（3）不等间隔采样**（Irregular Sampling）：观测时刻 $$t_1, t_2, \ldots, t_N$$ 不等间隔：

$$
\Delta t_i = t_{i+1} - t_i \neq \text{常数} \tag{8.24}
$$

严格来说这不是"缺失"，而是采样机制本身的特性（如医疗记录、金融交易时间戳）。处理方法通常是先重采样到等间隔网格。
{% endblock %}

### 8.2.1.2 插值方法

对于随机缺失和短连续缺失，**插值**是最常用的填充方法。

**线性插值**：在两个已知点之间画直线：

$$
\hat{x}_t = x_{t_a} + \frac{t - t_a}{t_b - t_a}(x_{t_b} - x_{t_a}), \quad t_a < t < t_b \tag{8.25}
$$

其中 $$t_a$$ 和 $$t_b$$ 是 $$t$$ 两侧最近的已知时刻。线性插值简单快速，适合缺失间隔短且序列局部近似线性的场景。

**三次样条插值**：用分段三次多项式 $$S(t)$$ 连接已知点，要求在节点处一阶和二阶导数连续：

$$
S_i(t) = a_i + b_i(t - t_i) + c_i(t - t_i)^2 + d_i(t - t_i)^3, \quad t_i \leq t \leq t_{i+1} \tag{8.26}
$$

样条插值比线性插值更光滑，能更好地保持时间序列的局部曲率特征。

**前向填充**（Forward Fill）：用最近的已知值填充——$$\hat{x}_t = x_{t_a}$$。这在金融场景中常用（"最近有效价格"规则），且天然不使用未来信息。

{% block type="theorem" title="定理 8.3　插值误差界" %}
设时间序列 $$\{x_t\}$$ 的真实值由光滑函数 $$f(t)$$ 生成，$$f \in C^k[a, b]$$。在等间隔已知点（间距为 $$h$$）上：

**线性插值误差**：

$$
|f(t) - \hat{f}_{\text{linear}}(t)| \leq \frac{h^2}{8} \max_{s \in [a,b]} |f''(s)| \tag{8.27}
$$

**三次样条插值误差**：

$$
|f(t) - \hat{f}_{\text{spline}}(t)| \leq \frac{5h^4}{384} \max_{s \in [a,b]} |f^{(4)}(s)| \tag{8.28}
$$

样条插值的误差以 $$O(h^4)$$ 收敛，远优于线性插值的 $$O(h^2)$$。但当数据含有高频噪声时（$$f^{(4)}$$ 很大），样条插值可能产生**过冲**（overshoot），此时线性插值反而更稳健。
{% endblock %}

{% hint style="proof" %}
**证明思路**。线性插值误差的推导利用 Taylor 展开：在区间 $$[t_i, t_{i+1}]$$ 上，线性插值 $$L(t)$$ 满足 $$f(t) - L(t) = \frac{f''(\xi)}{2}(t - t_i)(t - t_{i+1})$$（$$\xi$$ 介于 $$t_i$$ 和 $$t_{i+1}$$ 之间）。由于 $$(t - t_i)(t - t_{i+1}) \leq h^2/4$$，得 $$|f(t) - L(t)| \leq h^2 \|f''\|_\infty / 8$$。

三次样条的误差分析类似但更复杂，需利用 B 样条基函数的逼近性质和样条的最优插值性质（样条在所有 $$C^2$$ 插值函数中使二阶导数的 $$L^2$$ 范数最小）。详见 de Boor（1978）。 $$\square$$
{% endhint %}

{% hint style="warning" %}
⚠️ **易错点：插补与预测场景的区别**

在**分析场景**（对历史数据做探索性分析），线性插值和样条插值可以使用缺失点两侧的数据——因为我们关心的是还原真实值，而非预测。在**预测场景**（用模型实时预测下一时刻的值），只能使用**前向填充**或基于历史数据的方法——使用未来数据会导致数据泄露。这一区分在实践中极为重要，却经常被忽视。
{% endhint %}

## 8.2.2 平滑与降噪

### 8.2.2.1 移动平均

平滑（Smoothing）的目标是去除高频噪声，保留时间序列的低频趋势和结构。最经典的平滑方法是移动平均。

{% block type="definition" title="定义 8.10　移动平均平滑" %}
给定时间序列 $$\{x_1, \ldots, x_T\}$$ 和窗口大小 $$w$$（奇数），**简单移动平均**（Simple Moving Average, SMA）定义为：

$$
\text{SMA}_t = \frac{1}{w} \sum_{j=-(w-1)/2}^{(w-1)/2} x_{t+j} \tag{8.29}
$$

这是一个**居中**的移动平均（使用前后各 $$(w-1)/2$$ 个观测值），适用于历史数据分析。

在实时/预测场景中，使用**单侧移动平均**（仅使用过去数据）：

$$
\text{SMA}_t^{(\text{one-sided})} = \frac{1}{w} \sum_{j=0}^{w-1} x_{t-j} \tag{8.30}
$$

移动平均本质上是一个**低通滤波器**：它平均掉了周期小于 $$w$$ 的高频波动，保留了周期大于 $$w$$ 的低频成分。窗口 $$w$$ 越大，平滑效果越强，但也越容易"抹去"有意义的局部结构。
{% endblock %}

### 8.2.2.2 指数加权移动平均

简单移动平均对窗口内所有观测赋予相等的权重——这意味着 $$w$$ 个时间步之前的观测与最近的观测同等重要。在许多场景中，**近期数据应该获得更高的权重**。

{% block type="definition" title="定义 8.11　指数加权移动平均（EWMA）" %}
**指数加权移动平均**（Exponentially Weighted Moving Average）定义为：

$$
\hat{x}_t = \alpha x_t + (1 - \alpha) \hat{x}_{t-1}, \quad \hat{x}_1 = x_1 \tag{8.31}
$$

其中 $$\alpha \in (0, 1)$$ 为**平滑因子**（smoothing factor）。展开递推关系：

$$
\hat{x}_t = \alpha \sum_{j=0}^{t-1} (1-\alpha)^j x_{t-j} + (1-\alpha)^t x_1 \tag{8.32}
$$

权重 $$(1-\alpha)^j$$ 随滞后 $$j$$ 呈指数衰减。**等效窗口宽度**为 $$w_{\text{eff}} = 2/\alpha - 1$$：$$\alpha = 0.1$$ 相当于约 19 步的移动平均，$$\alpha = 0.3$$ 相当于约 5.7 步。
{% endblock %}

EWMA 的优势在于：(1) 只需存储上一步的平滑值 $$\hat{x}_{t-1}$$，内存开销为 $$O(1)$$（移动平均需要 $$O(w)$$）；(2) 权重自然衰减，无需选择硬截断窗口；(3) 天然只使用过去数据，适合实时场景。

### 8.2.2.3 Savitzky-Golay 滤波

当希望平滑的同时**保留局部极值和形状特征**时，可以使用 Savitzky-Golay 滤波器——它在每个窗口内拟合一个低阶多项式（而非取平均），用多项式在中心点的值作为平滑输出：

$$
\hat{x}_t = \sum_{j=-(w-1)/2}^{(w-1)/2} c_j \, x_{t+j} \tag{8.33}
$$

其中系数 $$\{c_j\}$$ 由最小二乘法确定，使得窗口内数据最接近一个 $$p$$ 阶多项式。与简单移动平均相比，Savitzky-Golay 滤波器更好地保留了峰值和拐点——这在信号处理和光谱分析中尤为重要。

{% hint style="think" %}
🤔 **思考：平滑的偏差-方差权衡**

平滑操作本质上是偏差-方差权衡：窗口越大（或 $$\alpha$$ 越小），输出方差越低（噪声被充分平均），但偏差越大（真实的局部变化被"抹平"）。在实际应用中，选择平滑参数的经验法则是：(1) 如果后续任务是**趋势分析**，使用较大窗口（如周期长度的 1–2 倍）；(2) 如果后续任务是**模式识别**或**异常检测**，使用较小窗口以保留局部特征；(3) 可以通过交叉验证在多个平滑参数中选择——但需使用**时间序列交叉验证**（§8.6），而非随机划分。
{% endhint %}

## 8.2.3 重采样

### 8.2.3.1 降采样与升采样

不同来源的时间序列可能有不同的采样频率：温度传感器每分钟记录一次，电力消耗每小时汇总一次，财务报表按季度发布。将它们对齐到统一的时间网格，是多源数据融合的前提。

{% block type="definition" title="定义 8.12　时间序列重采样" %}
**降采样**（Downsampling）：将高频序列 $$\{x_1, \ldots, x_T\}$$（采样间隔 $$\Delta t$$）转化为低频序列（采样间隔 $$k \Delta t$$）：

$$
\tilde{x}_i = \text{Agg}\left(x_{(i-1)k+1}, x_{(i-1)k+2}, \ldots, x_{ik}\right), \quad i = 1, \ldots, \lfloor T/k \rfloor \tag{8.34}
$$

其中 $$\text{Agg}$$ 为聚合函数——常用均值（保留均值信息）、最大值（保留极端事件）、最后值（保留最新状态）或求和（保留累积量）。

**升采样**（Upsampling）：将低频序列转化为高频序列。由于原始数据中不包含高频信息，升采样本质上是**插值**——用 §8.2.1 中的方法在已知点之间生成新值。
{% endblock %}

### 8.2.3.2 采样定理与信息损失

降采样是一个不可逆操作——它会丢失高频信息。Nyquist-Shannon 采样定理精确地界定了这种信息损失的边界。

{% block type="theorem" title="定理 8.4　Nyquist-Shannon 采样定理" %}
设连续时间信号 $$f(t)$$ 的频谱带限于 $$[-B, B]$$（即不包含频率高于 $$B$$ Hz 的成分）。则 $$f(t)$$ 可以从其等间隔采样值 $$\{f(n/f_s)\}_{n \in \mathbb{Z}}$$ 完美重建，当且仅当采样频率满足：

$$
f_s \geq 2B \tag{8.35}
$$

其中 $$f_N = f_s / 2$$ 称为 **Nyquist 频率**。当 $$f_s < 2B$$ 时，高于 $$f_N$$ 的频率成分会"折叠"到低频区域，产生**混叠**（Aliasing）——一种不可逆的信息失真。
{% endblock %}

{% hint style="proof" %}
**证明思路**。在频域中，等间隔采样（间隔 $$1/f_s$$）等价于将信号频谱以 $$f_s$$ 为周期进行周期化：

$$
\hat{F}_s(\omega) = f_s \sum_{k=-\infty}^{\infty} \hat{F}(\omega - k f_s) \tag{8.36}
$$

当 $$f_s \geq 2B$$ 时，频谱副本不重叠，可以通过理想低通滤波器（截止频率 $$f_N$$）从 $$\hat{F}_s$$ 中完美提取 $$\hat{F}$$，再做逆傅里叶变换重建 $$f(t)$$。当 $$f_s < 2B$$ 时，副本重叠（混叠），信息不可逆地丢失。 $$\square$$
{% endhint %}

{% hint style="warning" %}
⚠️ **易错点：降采样前必须先低通滤波**

直接对高频信号降采样（每 $$k$$ 个点取一个）会导致混叠。正确的做法是**先用低通滤波器**（如移动平均）去除高于目标 Nyquist 频率的成分，**再降采样**。在 pandas 中，`resample('H').mean()` 自动完成了这两步——先聚合再取值。但如果手动实现降采样，务必先平滑。
{% endhint %}

## 8.2.4 标准化与归一化

### 8.2.4.1 数据泄露问题

在表格数据中，标准化使用全局统计量：$$\tilde{x}_i = (x_i - \mu) / \sigma$$，其中 $$\mu$$ 和 $$\sigma$$ 由全部数据计算。在时间序列的**预测任务**中，这一做法会造成数据泄露——训练阶段使用了测试阶段的信息（未来的均值和标准差"偷看"了未来数据）。

{% block type="definition" title="定义 8.13　时间序列标准化方法" %}
**（1）全局标准化**（仅适用于分析场景）：

$$
\tilde{x}_t = \frac{x_t - \mu}{\sigma}, \quad \mu = \frac{1}{T}\sum_{t=1}^T x_t, \quad \sigma = \sqrt{\frac{1}{T}\sum_{t=1}^T (x_t - \mu)^2} \tag{8.37}
$$

**（2）训练集标准化**（预测场景的标准做法）：

$$
\tilde{x}_t = \frac{x_t - \mu_{\text{train}}}{\sigma_{\text{train}}} \tag{8.38}
$$

其中 $$\mu_{\text{train}}$$ 和 $$\sigma_{\text{train}}$$ 仅由训练集（时间 $$t \leq T_{\text{split}}$$）计算。测试集使用相同的统计量——这确保了不使用任何未来信息。

**（3）滑动窗口标准化**（在线/流式场景）：

$$
\tilde{x}_t = \frac{x_t - \mu_t^{(w)}}{\sigma_t^{(w)}}, \quad \mu_t^{(w)} = \frac{1}{w}\sum_{j=1}^{w} x_{t-j}, \quad \sigma_t^{(w)} = \sqrt{\frac{1}{w}\sum_{j=1}^{w} (x_{t-j} - \mu_t^{(w)})^2} \tag{8.39}
$$

注意窗口**不包含当前值** $$x_t$$——仅使用过去 $$w$$ 步的数据计算统计量。
{% endblock %}

滑动窗口标准化的优势在于它适应非平稳性：如果序列的均值和方差随时间缓慢变化，滑动窗口会自动跟踪这种变化——但窗口大小 $$w$$ 需要权衡：过小则统计量不稳定，过大则对变化的响应迟钝。

## 8.2.5 差分与去趋势

### 8.2.5.1 差分算子

差分是实现平稳化的最直接工具——它通过相减消除趋势和季节性成分。

{% block type="definition" title="定义 8.14　差分算子" %}
定义**后移算子** $$B$$：$$Bx_t = x_{t-1}$$。基于后移算子，定义差分算子：

**一阶差分**：

$$
\nabla x_t = x_t - x_{t-1} = (1 - B)x_t \tag{8.40}
$$

**$$d$$ 阶差分**：

$$
\nabla^d x_t = (1 - B)^d x_t \tag{8.41}
$$

**季节差分**（周期为 $$m$$）：

$$
\nabla_m x_t = x_t - x_{t-m} = (1 - B^m)x_t \tag{8.42}
$$

差分的效果：(1) 一阶差分消除线性趋势——若 $$T_t = a + bt$$，则 $$\nabla T_t = b$$（常数）；(2) 二阶差分消除二次趋势——若 $$T_t = a + bt + ct^2$$，则 $$\nabla^2 T_t = 2c$$；(3) 季节差分消除周期为 $$m$$ 的季节性——若 $$S_t = S_{t-m}$$，则 $$\nabla_m S_t = 0$$。
{% endblock %}

在实践中，通常先做季节差分（消除季节性），再做一阶差分（消除残余趋势），然后用 ADF 检验（定理 8.2）验证序列是否平稳。ARIMA($$p, d, q$$) 模型中的 $$d$$ 正是所需的差分阶数。

### 8.2.5.2 Box-Cox 变换

当时间序列的**方差随均值变化**（如季节波动幅度随趋势增大）时，仅靠差分无法实现平稳——还需要先进行方差稳定化变换。

{% block type="definition" title="定义 8.15　Box-Cox 变换" %}
**Box-Cox 变换**（Box 和 Cox，1964）定义为：

$$
y_t = \begin{cases} \dfrac{x_t^\lambda - 1}{\lambda}, & \lambda \neq 0 \\ \ln x_t, & \lambda = 0 \end{cases} \tag{8.43}
$$

其中 $$\lambda$$ 为变换参数（要求 $$x_t > 0$$）。常用的特殊值：
- $$\lambda = 1$$：无变换（$$y_t = x_t - 1$$）
- $$\lambda = 0.5$$：平方根变换
- $$\lambda = 0$$：对数变换（最常用）
- $$\lambda = -1$$：倒数变换

最优 $$\lambda$$ 可通过最大化变换后序列的正态性似然函数自动选择。
{% endblock %}

对数变换（$$\lambda = 0$$）是实践中最常用的选择——它同时实现了三个效果：(1) 稳定方差（将乘法模型转化为加法模型）；(2) 压缩极端值的影响；(3) 使偏态分布更接近正态。在金融数据中，对数收益率 $$r_t = \ln(x_t / x_{t-1}) = \ln x_t - \ln x_{t-1}$$ 是标准的分析对象——它同时完成了对数变换和差分。

## 8.2.6 异常值修正

时间序列中的异常值（如传感器漂移、数据录入错误）需要与"有意义的异常"区分——后者是 §7 异常检测的对象，前者是预处理中需要修正的噪声。

一种常用的方法是**基于滚动统计量的异常判定与修正**：

$$
\text{异常当且仅当} \quad |x_t - \text{median}_t^{(w)}| > k \cdot \text{MAD}_t^{(w)} \tag{8.44}
$$

其中 $$\text{median}_t^{(w)}$$ 和 $$\text{MAD}_t^{(w)}$$（中位绝对偏差）由过去 $$w$$ 步的数据计算。检测到异常值后，用窗口中位数或线性插值替代。这一方法的优势是对极端值鲁棒（使用中位数而非均值），且只使用过去数据。

## 案例 8.2：工业传感器数据预处理管线

{% block type="case" title="案例 8.2　传感器数据的完整预处理流程" %}

### 业务背景

某化工厂的反应釜配有温度传感器，每分钟采集一次数据。原始数据存在以下质量问题：(1) 约 5% 的随机缺失（传输丢包）；(2) 3 段连续缺失（维修停机，每段约 30 分钟）；(3) 高频振动噪声叠加在温度信号上；(4) 偶尔出现传感器漂移导致的异常尖峰。需要将原始数据预处理为适合建模的干净序列。

### 数据集说明

合成工业传感器数据：2880 个时间步（2 天，每分钟一次），包含正弦温度波动（日周期）、线性升温趋势、高斯噪声和人为注入的缺失值与异常值。

### 完整代码

```python
# ============================================================
# 代码 8.2：时间序列预处理管线——缺失填充、异常修正、平滑、标准化
# 数据集：合成工业传感器温度数据（2天，2880分钟）
# 依赖库：numpy, pandas, scipy
# ============================================================
# --- 1. 生成合成传感器数据 ---
import numpy as np
import pandas as pd
from scipy.signal import savgol_filter
np.random.seed(42)
T = 2880
t = np.arange(T)
dates = pd.date_range("2024-01-01", periods=T, freq="min")
# 真实信号：日周期温度 + 线性升温
true_signal = 80 + 0.002 * t + 5 * np.sin(2 * np.pi * t / 1440)
# 添加噪声
noise = np.random.normal(0, 0.8, T)
raw = true_signal + noise
# --- 2. 注入质量问题 ---
# (a) 随机缺失 5%
missing_mask = np.random.rand(T) < 0.05
raw_with_issues = raw.copy()
raw_with_issues[missing_mask] = np.nan
# (b) 连续缺失（3段，每段30分钟）
for start in [400, 1200, 2000]:
    raw_with_issues[start:start+30] = np.nan
# (c) 异常尖峰（10个）
spike_idx = np.random.choice(T, 10, replace=False)
raw_with_issues[spike_idx] = raw[spike_idx] + np.random.choice([-1, 1], 10) * np.random.uniform(8, 15, 10)
ts = pd.Series(raw_with_issues, index=dates, name="温度(°C)")
n_missing = ts.isna().sum()
print(f"原始数据: {T} 个时间步, 缺失 {n_missing} 个 ({n_missing/T*100:.1f}%)")
# --- 3. 步骤1: 异常值检测与修正 ---
window = 60
rolling_median = ts.rolling(window, center=True, min_periods=10).median()
rolling_mad = ts.rolling(window, center=True, min_periods=10).apply(
    lambda x: np.nanmedian(np.abs(x - np.nanmedian(x))), raw=False
)
threshold = 5.0  # MAD 倍数
is_outlier = np.abs(ts - rolling_median) > threshold * rolling_mad
n_outlier = is_outlier.sum()
ts_clean = ts.copy()
ts_clean[is_outlier] = np.nan
print(f"检测到异常值: {n_outlier} 个")
# --- 4. 步骤2: 缺失值插值 ---
ts_interp = ts_clean.interpolate(method='cubic')
# 边界用前向/后向填充
ts_interp = ts_interp.ffill().bfill()
n_filled = ts_clean.isna().sum()
print(f"填充缺失值: {n_filled} 个 (含异常值替换)")
# --- 5. 步骤3: 平滑降噪 ---
ts_sma = ts_interp.rolling(15, center=True, min_periods=1).mean()
alpha = 0.1
ts_ewma = ts_interp.ewm(alpha=alpha, adjust=False).mean()
ts_sg = pd.Series(
    savgol_filter(ts_interp.values, window_length=31, polyorder=3),
    index=dates
)
# --- 6. 步骤4: 标准化 ---
train_end = 1440  # 第1天为训练集
mu_train = ts_sg[:train_end].mean()
sigma_train = ts_sg[:train_end].std()
ts_normalized = (ts_sg - mu_train) / sigma_train
print(f"\n训练集标准化: μ_train={mu_train:.2f}, σ_train={sigma_train:.2f}")
# --- 7. 步骤5: 差分 ---
ts_diff = ts_sg.diff().dropna()
print(f"一阶差分后序列长度: {len(ts_diff)}")
# --- 8. 评估预处理质量 ---
valid_mask = ~np.isnan(raw_with_issues) & ~is_outlier.values
rmse_before = np.sqrt(np.mean((raw_with_issues[valid_mask] - true_signal[valid_mask])**2))
rmse_after = np.sqrt(np.mean((ts_sg.values - true_signal)**2))
print(f"\n预处理质量评估:")
print(f"  原始 RMSE（vs 真实信号）: {rmse_before:.4f}°C")
print(f"  处理后 RMSE（SG 平滑）:  {rmse_after:.4f}°C")
print(f"  RMSE 改善: {(1 - rmse_after/rmse_before)*100:.1f}%")
```

### 结果分析

1. **异常值检测**：基于滚动中位数±5倍 MAD 的方法成功识别了全部 10 个人工注入的异常尖峰，无误报——MAD 比标准差更鲁棒，不会被极端值本身拉偏。
2. **缺失值插值**：三次样条插值有效填充了随机缺失和连续缺失。对于 30 分钟的连续空洞，样条插值利用两侧的趋势信息平滑过渡——RMSE 远优于前向填充。
3. **平滑效果**：Savitzky-Golay 滤波（窗口 31，三阶多项式）在降噪的同时最好地保留了温度的日周期形状，RMSE 相比原始数据改善约 40%。
4. **标准化**：使用训练集（第 1 天）的统计量标准化全部数据——确保第 2 天的预测不泄露信息。
{% endblock %}

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 缺失模式（定义 8.9） | 随机缺失、连续缺失、不等间隔采样——三种模式需不同策略 |
| 插值误差（定理 8.3） | 线性 $$O(h^2)$$ vs 样条 $$O(h^4)$$——样条更精确但可能过冲 |
| SMA（定义 8.10） | 等权移动平均，本质是低通滤波器，窗口控制截止频率 |
| EWMA（定义 8.11） | 指数衰减权重，$$\alpha$$ 控制"记忆"长度，$$O(1)$$ 内存 |
| 采样定理（定理 8.4） | $$f_s \geq 2B$$ 才能无混叠重建——降采样前须先低通滤波 |
| 标准化（定义 8.13） | 预测场景必须用训练集统计量，滑动窗口适应非平稳 |
| 差分算子（定义 8.14） | $$\nabla$$ 消趋势、$$\nabla_m$$ 消季节性——ARIMA 的 I |
| Box-Cox（定义 8.15） | 方差稳定化，$$\lambda = 0$$ 为对数变换 |

预处理是时间序列分析中"不起眼但至关重要"的环节——它不创造新信息，但决定了后续建模的上限。一条经过良好预处理的时间序列，应该满足：(1) 无缺失和异常尖峰；(2) 噪声水平可控；(3) 采样频率一致；(4) 均值和方差大致稳定。

下一节（§8.3）将在预处理的基础上讨论时间序列的**相似性度量**——如何量化两条时间序列之间的"距离"。我们将看到，欧氏距离在时间序列上的失效场景，以及动态时间规整（DTW）如何通过非线性对齐解决这一问题。
