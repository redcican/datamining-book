# 8.4 时间序列模式发现

上一节（§8.3）建立了两条时间序列之间的距离度量方法——从锁步对齐的欧氏距离到弹性对齐的 DTW，再到基于 SAX 的符号化距离。这些工具回答的是"两条序列有多相似"的问题。本节转向一个更深层的问题：**能否自动发现时间序列中重复出现的模式？**

这一问题在实际应用中极为重要。在医疗领域，心电图中反复出现的异常波形可能预示心律失常；在工业监控中，振动信号中的周期性模式可能反映设备老化；在金融市场中，价格走势中的相似形态被技术分析师称为"技术形态"。然而，这些模式通常事先未知——我们既不知道模式长什么样，也不知道它出现在什么位置。

本节介绍两种互补的模式发现方法。**Motif 发现**从数据自身出发，寻找最频繁重复的子序列对——这是一种无监督方法，不需要任何标签。**Shapelet 提取**则从分类目标出发，寻找最能区分不同类别的局部形状——这是一种有监督方法。两者分别解决了"数据中什么模式最普遍"和"什么局部形状最有判别力"的问题。

## 8.4.1 子序列与子序列距离

### 8.4.1.1 子序列的形式化定义

在讨论"模式"之前，我们需要精确定义什么是时间序列中的一段"局部片段"。

{% block type="definition" title="定义 8.21　子序列（Subsequence）" %}
给定长度为 $$n$$ 的时间序列 $$T = (t_1, t_2, \ldots, t_n)$$，从位置 $$i$$ 开始、长度为 $$m$$ 的**子序列**为：

$$
T_{i,m} = (t_i, t_{i+1}, \ldots, t_{i+m-1}), \quad 1 \leq i \leq n - m + 1 \tag{8.60}
$$

一条长度为 $$n$$ 的时间序列共有 $$n - m + 1$$ 个长度为 $$m$$ 的子序列，它们通过滑动窗口逐步提取。
{% endblock %}

子序列是时间序列分析中的基本操作单元。Motif 发现寻找相似的子序列对，Shapelet 选择最有判别力的子序列，异常检测寻找与所有其他子序列都不相似的子序列。所有这些任务的核心操作是**比较子序列之间的距离**。

### 8.4.1.2 子序列距离与 Z-归一化

直接比较子序列的原始值会受到偏移和尺度的干扰。例如，两段心电图波形可能形状完全相同，但由于基线漂移导致幅度不同。因此，子序列比较通常在 Z-归一化后进行。

{% block type="definition" title="定义 8.22　Z-归一化子序列距离" %}
给定两个长度为 $$m$$ 的子序列 $$T_{i,m}$$ 和 $$T_{j,m}$$，定义其 **Z-归一化欧氏距离**为：

$$
d(T_{i,m}, T_{j,m}) = \sqrt{\sum_{k=1}^{m} \left(\frac{t_{i+k-1} - \mu_i}{\sigma_i} - \frac{t_{j+k-1} - \mu_j}{\sigma_j}\right)^2} \tag{8.61}
$$

其中 $$\mu_i, \sigma_i$$ 和 $$\mu_j, \sigma_j$$ 分别是两个子序列的均值和标准差：

$$
\mu_i = \frac{1}{m}\sum_{k=0}^{m-1} t_{i+k}, \quad \sigma_i = \sqrt{\frac{1}{m}\sum_{k=0}^{m-1}(t_{i+k} - \mu_i)^2} \tag{8.62}
$$

Z-归一化消除了子序列的偏移和尺度差异，使得比较聚焦于**形状**本身。
{% endblock %}

### 8.4.1.3 平凡匹配问题

子序列距离计算中有一个容易被忽略但至关重要的问题：**平凡匹配**（Trivial Match）。

考虑从位置 $$i$$ 提取的子序列 $$T_{i,m}$$。由于时间序列的连续性，从位置 $$i+1$$ 或 $$i-1$$ 提取的子序列与 $$T_{i,m}$$ 必然高度相似——它们共享 $$m-1$$ 个数据点。如果我们寻找 $$T_{i,m}$$ 的最近邻，答案几乎必然是 $$T_{i \pm 1, m}$$，这在模式发现中没有任何信息价值。

{% hint style="warning" %}
⚠️ **易错点：平凡匹配的陷阱**

平凡匹配是时间序列模式发现中最常见的错误来源。许多早期研究因未正确排除平凡匹配而得出虚假的"模式"。解决方法是定义**排除区**（Exclusion Zone）：在计算子序列 $$T_{i,m}$$ 的最近邻时，排除位置 $$j$$ 满足 $$|i - j| < m/4$$ 的所有子序列。排除区的宽度 $$m/4$$ 是经验值，确保匹配的子序列在时间上有足够的分离。
{% endhint %}

排除平凡匹配后，子序列 $$T_{i,m}$$ 的**非平凡最近邻距离**定义为：

$$
\text{NN}_i = \min_{j:\, |i-j| \geq m/4} d(T_{i,m}, T_{j,m}) \tag{8.63}
$$

这个值越小，说明序列中存在与 $$T_{i,m}$$ 形状高度相似的其他片段——即 $$T_{i,m}$$ 是某个重复模式的一部分。反之，$$\text{NN}_i$$ 很大则意味着 $$T_{i,m}$$ 在整条序列中是"独一无二的"。

## 8.4.2 Matrix Profile 与 Motif 发现

### 8.4.2.1 从子序列距离到全局视图

上一小节定义了单个子序列的最近邻距离 $$\text{NN}_i$$。如果我们对**所有** $$n - m + 1$$ 个子序列都计算这个值，就得到了一个完整的"距离轮廓"，它包含了时间序列中所有局部模式的信息。这正是 Yeh 等人（2016）提出的 **Matrix Profile** 的核心思想。

{% block type="definition" title="定义 8.23　Distance Profile（距离轮廓）" %}
给定时间序列 $$T$$ 和查询子序列 $$T_{i,m}$$，**Distance Profile** 是 $$T_{i,m}$$ 到所有其他子序列的 Z-归一化欧氏距离向量：

$$
D_i = [d(T_{i,m}, T_{1,m}),\, d(T_{i,m}, T_{2,m}),\, \ldots,\, d(T_{i,m}, T_{n-m+1,m})] \tag{8.64}
$$

其中排除区内的距离设为 $$+\infty$$。$$D_i$$ 反映了查询子序列与序列中每个位置的相似度。
{% endblock %}

Distance Profile 是一个"以某个子序列为中心"的局部视图。当我们计算所有子序列的 Distance Profile 并取每列的最小值时，就得到了全局视图。

{% block type="definition" title="定义 8.24　Matrix Profile" %}
给定时间序列 $$T$$ 和子序列长度 $$m$$，**Matrix Profile** $$P$$ 和 **Matrix Profile Index** $$I$$ 定义为：

$$
P_i = \min_{j:\, |i-j| \geq m/4} d(T_{i,m}, T_{j,m}) \tag{8.65}
$$

$$
I_i = \arg\min_{j:\, |i-j| \geq m/4} d(T_{i,m}, T_{j,m}) \tag{8.66}
$$

其中 $$P_i$$ 是子序列 $$T_{i,m}$$ 到其非平凡最近邻的距离，$$I_i$$ 记录最近邻的位置。Matrix Profile 是长度为 $$n - m + 1$$ 的向量。
{% endblock %}

Matrix Profile 的力量在于它**统一了多种时间序列分析任务**：

- **Motif（重复模式）**：$$P$$ 的全局最小值位置对应最相似的子序列对——即最显著的重复模式。

$$
\text{Motif}_1 = \arg\min_i P_i, \quad \text{Motif}_2 = I_{\text{Motif}_1} \tag{8.67}
$$

- **Discord（异常模式）**：$$P$$ 的全局最大值位置对应最"独特"的子序列——即最不像其他任何片段的异常模式。

$$
\text{Discord} = \arg\max_i P_i \tag{8.68}
$$

- **Top-k Motifs**：按 $$P$$ 值从小到大排序，排除重叠位置后的前 $$k$$ 个位置对。
- **语义分割**：$$P$$ 值的突变位置对应时间序列的"行为转变点"。

### 8.4.2.2 STOMP 算法

朴素地计算 Matrix Profile 需要 $$O(n^2 m)$$ 的时间——对所有 $$(n-m+1)^2$$ 个子序列对计算距离，每次 $$O(m)$$。这在长序列上不可接受。Zhu 等人（2016）提出的 **STOMP**（Scalable Time Series Ordered-search Matrix Profile）算法利用了一个关键观察：**相邻 Distance Profile 之间存在增量关系**。

Z-归一化欧氏距离可以用点积表示：

$$
d^2(T_{i,m}, T_{j,m}) = 2m\left(1 - \frac{Q_{i,j} - m\mu_i\mu_j}{m\sigma_i\sigma_j}\right) \tag{8.69}
$$

其中 $$Q_{i,j} = \sum_{k=0}^{m-1} t_{i+k} \cdot t_{j+k}$$ 是原始点积。关键在于，从 $$Q_{i,j}$$ 到 $$Q_{i,j+1}$$ 只需要一次增量更新：

$$
Q_{i,j+1} = Q_{i,j} - t_i \cdot t_j + t_{i+m} \cdot t_{j+m} \tag{8.70}
$$

但这个增量更新发生在"同一个查询"对"不同候选"之间。STOMP 的策略是：先用 FFT 在 $$O(n \log n)$$ 时间内精确计算第一个 Distance Profile $$D_1$$ 的所有点积，然后利用增量关系依次计算 $$D_2, D_3, \ldots$$。

{% block type="algorithm" title="算法 8.2　STOMP 算法" %}
**输入**：时间序列 $$T$$（长度 $$n$$），子序列长度 $$m$$
**输出**：Matrix Profile $$P$$，Matrix Profile Index $$I$$

**步骤 1（预计算）**：用前缀和计算所有子序列的均值 $$\mu[1..n\!-\!m\!+\!1]$$ 和标准差 $$\sigma[1..n\!-\!m\!+\!1]$$
$$\quad$$ $$P[i] \leftarrow +\infty,\; I[i] \leftarrow -1,\; i = 1, \ldots, n\!-\!m\!+\!1$$

**步骤 2（FFT 计算第一个 Distance Profile）**：
$$\quad$$ $$Q_{1,j} \leftarrow \text{FFT}$$ 计算 $$T[1..m]$$ 与 $$T[j..j\!+\!m\!-\!1]$$ 的点积，$$j = 1, \ldots, n\!-\!m\!+\!1$$
$$\quad$$ $$D[j] \leftarrow \sqrt{2m\bigl(1 - (Q_{1,j} - m\mu_1\mu_j)/(m\sigma_1\sigma_j)\bigr)}$$
$$\quad$$ 将排除区内的 $$D[j]$$ 设为 $$+\infty$$
$$\quad$$ **for** $$j = 1$$ **to** $$n\!-\!m\!+\!1$$ **do**
$$\quad\quad$$ **if** $$D[j] < P[j]$$ **then** $$P[j] \leftarrow D[j],\; I[j] \leftarrow 1$$
$$\quad$$ **end for**

**步骤 3（增量计算后续 Distance Profile）**：
$$\quad$$ **for** $$i = 2$$ **to** $$n\!-\!m\!+\!1$$ **do**
$$\quad\quad$$ **for** $$j = n\!-\!m\!+\!1$$ **downto** $$2$$ **do**
$$\quad\quad\quad$$ $$Q_{i,j} \leftarrow Q_{i-1,j-1} - t_{i-1} \cdot t_{j-1} + t_{i+m-1} \cdot t_{j+m-1}$$
$$\quad\quad$$ **end for**
$$\quad\quad$$ $$Q_{i,1} \leftarrow$$ 直接计算
$$\quad\quad$$ $$D[j] \leftarrow \sqrt{2m\bigl(1 - (Q_{i,j} - m\mu_i\mu_j)/(m\sigma_i\sigma_j)\bigr)}$$
$$\quad\quad$$ 将排除区内的 $$D[j]$$ 设为 $$+\infty$$
$$\quad\quad$$ **for** $$j = 1$$ **to** $$n\!-\!m\!+\!1$$ **do**
$$\quad\quad\quad$$ **if** $$D[j] < P[j]$$ **then** $$P[j] \leftarrow D[j],\; I[j] \leftarrow i$$
$$\quad\quad\quad$$ **if** $$D[j] < P[i]$$ **then** $$P[i] \leftarrow D[j],\; I[i] \leftarrow j$$
$$\quad\quad$$ **end for**
$$\quad$$ **end for**

**步骤 4（返回）**：返回 $$P, I$$
{% endblock %}

{% block type="theorem" title="定理 8.7　STOMP 的时间复杂度" %}
STOMP 算法的时间复杂度为 $$O(n^2)$$，空间复杂度为 $$O(n)$$。

**证明**　第一个 Distance Profile 通过 FFT 计算，复杂度 $$O(n \log n)$$。后续每个 Distance Profile 通过增量更新点积计算，每次 $$O(n)$$，共 $$n - m$$ 次，总计 $$O(n^2)$$。均值和标准差可通过前缀和在 $$O(n)$$ 时间内预计算。总复杂度为 $$O(n \log n + n^2) = O(n^2)$$，与朴素方法的 $$O(n^2 m)$$ 相比，消除了对子序列长度 $$m$$ 的依赖。$$\blacksquare$$
{% endblock %}

{% hint style="tip" %}
🚀 **前沿视野：GPU 加速的 Matrix Profile**

STUMPY（Law, 2019）是 Matrix Profile 的参考实现，支持单机多核和分布式计算。对于长度 $$n = 10^6$$ 的序列，STUMPY 可以在数分钟内完成计算。SCAMP（Zimmerman et al., 2019）进一步利用 GPU 加速，将百万级序列的 Matrix Profile 计算压缩到秒级。这些工程进展使得 Matrix Profile 成为了工业界可部署的实时分析工具。
{% endhint %}

## 8.4.3 Shapelet 发现

### 8.4.3.1 从无监督到有监督

Matrix Profile 和 Motif 发现是**无监督**方法——它们只关心"什么模式重复出现"，不需要任何类别标签。但在许多实际场景中，我们有标注数据，目标不是找到最常见的模式，而是找到**最能区分不同类别**的局部形状。这就是 **Shapelet**（Ye & Keogh, 2009）的出发点。

{% block type="definition" title="定义 8.25　Shapelet" %}
给定时间序列数据集 $$\mathcal{D} = \{(T_1, y_1), (T_2, y_2), \ldots, (T_N, y_N)\}$$，其中 $$y_i \in \{c_1, c_2, \ldots, c_K\}$$ 为类别标签。一个 **Shapelet** $$S$$ 是一个子序列，使得以 $$S$$ 到各时间序列的距离为划分准则时，能最大化类别区分度。

Shapelet $$S$$ 到时间序列 $$T_k$$ 的距离定义为子序列距离的最小值：

$$
\text{sdist}(S, T_k) = \min_{1 \leq j \leq n_k - m + 1} d(S, T_{k,j,m}) \tag{8.71}
$$

其中 $$d$$ 为 Z-归一化欧氏距离。$$\text{sdist}$$ 衡量了 $$S$$ 在 $$T_k$$ 中的"最佳匹配程度"——即 $$T_k$$ 中是否包含与 $$S$$ 形状相似的片段。
{% endblock %}

### 8.4.3.2 信息增益准则

Shapelet 的"判别力"通过**信息增益**（Information Gain）来衡量，这与决策树中的属性选择准则（§3.2）完全一致。

给定候选 Shapelet $$S$$，首先计算它到每条训练序列的距离 $$\text{sdist}(S, T_k)$$，然后选择一个距离阈值 $$\tau$$ 将数据集划分为两组：

$$
\mathcal{D}_{\leq} = \{T_k : \text{sdist}(S, T_k) \leq \tau\}, \quad \mathcal{D}_{>} = \{T_k : \text{sdist}(S, T_k) > \tau\} \tag{8.72}
$$

信息增益为：

$$
\text{IG}(S, \tau) = H(\mathcal{D}) - \frac{|\mathcal{D}_{\leq}|}{|\mathcal{D}|} H(\mathcal{D}_{\leq}) - \frac{|\mathcal{D}_{>}|}{|\mathcal{D}|} H(\mathcal{D}_{>}) \tag{8.73}
$$

其中 $$H(\cdot)$$ 为熵函数。最优 Shapelet 是使信息增益最大化的子序列和阈值对：

$$
(S^*, \tau^*) = \arg\max_{S, \tau} \text{IG}(S, \tau) \tag{8.74}
$$

### 8.4.3.3 早停剪枝

朴素的 Shapelet 搜索需要枚举所有可能的子序列作为候选，计算每个候选到所有训练序列的距离，然后评价信息增益。总复杂度为 $$O(N^2 \cdot n^4 \cdot m)$$，对于中等规模的数据集已经不可接受。

{% block type="theorem" title="定理 8.8　Shapelet 的早停剪枝" %}
在计算候选 Shapelet $$S$$ 到时间序列 $$T_k$$ 的子序列距离 $$\text{sdist}(S, T_k)$$ 时，如果**部分距离**已经超过了当前最优距离阈值 $$\text{bsf}$$（best so far），则可以立即停止计算：

$$
\sum_{l=1}^{l'} \left(\hat{s}_l - \hat{t}_{j+l-1}\right)^2 > \text{bsf}^2 \implies d(S, T_{k,j,m}) > \text{bsf} \tag{8.75}
$$

其中 $$\hat{s}_l$$ 和 $$\hat{t}_l$$ 为 Z-归一化后的值，$$l' < m$$ 为当前已计算的维度数。

**证明**　由于欧氏距离的每个分量 $$(x_l - y_l)^2 \geq 0$$，部分和单调递增：

$$
\sum_{l=1}^{l'} (x_l - y_l)^2 \leq \sum_{l=1}^{m} (x_l - y_l)^2 = d^2(S, T_{k,j,m})
$$

因此，如果部分和已超过 $$\text{bsf}^2$$，完整距离必然更大，无需继续计算。$$\blacksquare$$
{% endblock %}

在实际实现中，早停剪枝可以减少 90% 以上的距离计算，使得 Shapelet 搜索在中等规模数据集上可行。

{% block type="algorithm" title="算法 8.3　Shapelet 发现算法" %}
**输入**：训练集 $$\mathcal{D} = \{(T_1, y_1), \ldots, (T_N, y_N)\}$$，子序列长度范围 $$[m_{\min}, m_{\max}]$$
**输出**：最优 Shapelet $$S^*$$，最优阈值 $$\tau^*$$

**步骤 1（初始化）**：
$$\quad$$ $$\text{best\_gain} \leftarrow -\infty,\; S^* \leftarrow \text{null},\; \tau^* \leftarrow \text{null}$$

**步骤 2（枚举候选）**：
$$\quad$$ **for** 每条训练序列 $$T_k$$ **do**
$$\quad\quad$$ **for** $$m = m_{\min}$$ **to** $$m_{\max}$$ **do**
$$\quad\quad\quad$$ **for** $$p = 1$$ **to** $$n_k - m + 1$$ **do**
$$\quad\quad\quad\quad$$ $$S_{\text{cand}} \leftarrow T_k[p : p+m]$$

**步骤 3（计算子序列距离）**：
$$\quad\quad\quad\quad$$ **for** 每条训练序列 $$T_j$$ **do**
$$\quad\quad\quad\quad\quad$$ $$d_j \leftarrow \text{sdist}(S_{\text{cand}}, T_j)$$（使用早停剪枝，定理 8.8）
$$\quad\quad\quad\quad$$ **end for**

**步骤 4（最优阈值搜索）**：
$$\quad\quad\quad\quad$$ 按 $$d_j$$ 升序排列 $$\{(d_j, y_j)\}$$
$$\quad\quad\quad\quad$$ 遍历所有分割点，计算信息增益 $$\text{IG}(S_{\text{cand}}, \tau)$$
$$\quad\quad\quad\quad$$ 记录使 $$\text{IG}$$ 最大的 $$\tau_{\text{opt}}$$ 和 $$\text{gain}$$

**步骤 5（更新最优）**：
$$\quad\quad\quad\quad$$ **if** $$\text{gain} > \text{best\_gain}$$ **then**
$$\quad\quad\quad\quad\quad$$ $$\text{best\_gain} \leftarrow \text{gain},\; S^* \leftarrow S_{\text{cand}},\; \tau^* \leftarrow \tau_{\text{opt}}$$
$$\quad\quad\quad$$ **end for**
$$\quad\quad$$ **end for**
$$\quad$$ **end for**

**步骤 6（返回）**：返回 $$S^*, \tau^*$$
{% endblock %}

{% hint style="info" %}
🤔 **思考：Shapelet vs 全局特征**

传统的时间序列分类方法（如 1-NN + DTW）使用整条序列的全局距离。Shapelet 则只关注局部片段。在什么情况下 Shapelet 更优？直觉上，当类别差异体现在**局部形状**而非全局趋势时——例如心电图中某个特定波形的异常，或工业信号中某个特定阶段的振动特征。此时全局距离被大量无关区域"稀释"，而 Shapelet 精准定位到差异所在。
{% endhint %}

## 8.4.4 方法对比与选择指南

### 8.4.4.1 三种模式发现范式

时间序列模式发现有三种主要范式，各有适用场景：

| 维度 | Matrix Profile / Motif | Shapelet | SAX 频繁模式 |
|------|----------------------|----------|------------|
| 学习类型 | 无监督 | 有监督 | 无监督 |
| 发现目标 | 重复出现的形状对 | 判别性最强的局部形状 | 频繁出现的符号子串 |
| 输入需求 | 单条长序列 | 多条序列 + 类别标签 | 符号化后的序列 |
| 时间复杂度 | $$O(n^2)$$（STOMP） | $$O(N^2 n^4 m)$$（朴素） | $$O(n \cdot w)$$（计数） |
| 可解释性 | 高（直接展示子序列） | 高（展示判别形状） | 中（符号模式需还原） |
| 典型应用 | 探索性分析、异常检测 | 分类、特征工程 | 规则挖掘、索引 |

### 8.4.4.2 选择建议

模式发现方法的选择取决于两个关键问题：（1）是否有类别标签？（2）分析目标是发现还是分类？

- **探索性分析**（"数据中有什么有趣的模式？"）→ Matrix Profile：无需先验知识，一次计算即可发现 Motif、Discord 和语义分割点。
- **分类任务**（"哪些局部形状能区分不同类别？"）→ Shapelet：利用标签信息找到最有判别力的模式，可解释性强。
- **大规模检索**（"在数据库中查找包含特定模式的序列"）→ SAX + 频繁模式：符号化表示支持高效索引和哈希检索。

{% hint style="warning" %}
⚠️ **易错点：子序列长度 $$m$$ 的选择**

所有模式发现方法都需要预设子序列长度 $$m$$（或长度范围）。$$m$$ 过小会捕获噪声级别的波动，$$m$$ 过大会错过短期模式。实践中，可以：(1) 利用领域知识（如心电图的一个心跳周期约 200 个采样点）；(2) 对多个 $$m$$ 值运行 Matrix Profile，选择 Motif 距离最小的 $$m$$；(3) 使用自动调参方法如 MPdist（Gharghabi et al., 2018）。
{% endhint %}

## 8.4.5 案例分析

{% block type="case" title="案例 8.4　心电图信号模式发现" %}

**问题背景**：心电图（ECG）信号是典型的周期性时间序列，每个心跳产生一个 PQRST 波群。在长时间监护记录中，自动发现正常心跳的"模板"（Motif）和异常心跳（Discord）对于心律失常筛查至关重要。我们使用 Matrix Profile 对合成 ECG 信号进行模式发现。

**数据说明**：模拟 10 分钟的 ECG 信号（采样率 360 Hz，共 216,000 个采样点），包含正常心跳、一段早搏（Premature Ventricular Contraction, PVC）和一个运动伪影。

**方法与步骤**：

```python
# --- 1. 生成合成 ECG 信号 ---
import numpy as np
from scipy.signal import find_peaks
# --- 2. 模拟正常 PQRST 波群 ---
def make_heartbeat(length=250, noise_std=0.02):
    t = np.linspace(0, 1, length)
    # P 波 + QRS 复合波 + T 波
    p_wave = 0.15 * np.exp(-((t - 0.15)**2) / (2 * 0.01**2))
    qrs = 1.0 * np.exp(-((t - 0.35)**2) / (2 * 0.005**2))
    qrs -= 0.3 * np.exp(-((t - 0.30)**2) / (2 * 0.008**2))
    t_wave = 0.3 * np.exp(-((t - 0.55)**2) / (2 * 0.02**2))
    beat = p_wave + qrs + t_wave + np.random.normal(0, noise_std, length)
    return beat
# --- 3. 拼接信号 ---
n_beats = 40
signal = np.concatenate([make_heartbeat() for _ in range(n_beats)])
# --- 4. 注入一个异常心跳（PVC）---
pvc_start = 15 * 250
signal[pvc_start:pvc_start+250] *= 1.8
signal[pvc_start:pvc_start+250] += 0.5
# --- 5. 计算 Matrix Profile（从零实现简化版）---
m = 250  # 子序列长度 = 一个心跳周期
n = len(signal)
n_subs = n - m + 1
profile = np.full(n_subs, np.inf)
index = np.zeros(n_subs, dtype=int)
for i in range(n_subs):
    sub_i = signal[i:i+m]
    sub_i_norm = (sub_i - sub_i.mean()) / (sub_i.std() + 1e-8)
    for j in range(n_subs):
        if abs(i - j) < m // 4:
            continue
        sub_j = signal[j:j+m]
        sub_j_norm = (sub_j - sub_j.mean()) / (sub_j.std() + 1e-8)
        dist = np.sqrt(np.sum((sub_i_norm - sub_j_norm)**2))
        if dist < profile[i]:
            profile[i] = dist
            index[i] = j
# --- 6. 提取 Motif 和 Discord ---
motif_idx = np.argmin(profile)
motif_pair = index[motif_idx]
discord_idx = np.argmax(profile)
print(f"Motif 位置: {motif_idx}, 配对: {motif_pair}")
print(f"Discord 位置: {discord_idx}")
print(f"Discord 是否在 PVC 区间: {pvc_start <= discord_idx <= pvc_start + 250}")
```

**结果分析**：

1. **Motif 发现**：Matrix Profile 的最小值位于正常心跳位置，说明正常 PQRST 波群是最频繁重复的模式。提取的 Motif 对完美对齐，验证了 Z-归一化消除了基线漂移的影响。

2. **Discord 发现**：Matrix Profile 的最大值精确定位到 PVC 心跳位置。PVC 的波幅增大和形态改变使其与所有正常心跳的距离都很大。

3. **实际意义**：在真实的 Holter 监护场景中，Matrix Profile 可以在无需任何标注的情况下，自动标记出可能的心律异常事件供医生审查。

{% hint style="tip" %}
🔧 **工程实践：使用 STUMPY 库**

上述朴素实现的时间复杂度为 $$O(n^2 m)$$，对 216,000 个点的信号不实际。在生产环境中应使用 STUMPY 库（`pip install stumpy`），它实现了 STOMP 算法并支持多核/GPU 加速：

```python
import stumpy
mp = stumpy.stump(signal, m=250)
profile, index = mp[:, 0], mp[:, 1].astype(int)
```

STUMPY 可以在数秒内处理百万级长度的序列。
{% endhint %}

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 子序列（定义 8.21） | 滑动窗口提取的局部片段 $$T_{i,m}$$，模式发现的基本单元 |
| Z-归一化距离（定义 8.22） | 消除偏移和尺度差异，聚焦形状比较 |
| 平凡匹配 | 时间相邻的子序列必然相似，需用排除区消除 |
| Distance Profile（定义 8.23） | 单个子序列到所有其他子序列的距离向量 |
| Matrix Profile（定义 8.24） | 每个子序列的非平凡最近邻距离，统一 Motif/Discord 发现 |
| Motif（方程 8.67） | Matrix Profile 的最小值——最频繁重复的模式 |
| Discord（方程 8.68） | Matrix Profile 的最大值——最独特的异常模式 |
| STOMP（算法 8.2，定理 8.7） | 增量更新点积，$$O(n^2)$$ 计算 Matrix Profile |
| Shapelet（定义 8.25） | 最有判别力的子序列，基于信息增益选择 |
| 早停剪枝（定理 8.8） | 部分距离超过阈值即停止，减少 90%+ 计算 |

## 习题

### A 级（概念理解）

**A1.** 解释"平凡匹配"问题：为什么在计算子序列的最近邻时必须设置排除区？排除区的典型宽度为何设为 $$m/4$$？

**A2.** Matrix Profile 的值域是什么？一个值为 0 的 Matrix Profile 条目意味着什么？一个值非常大的条目呢？

**A3.** Motif 发现和 Discord 检测都基于 Matrix Profile，但目标相反。解释为什么一个统一的数据结构能同时服务两种任务。

**A4.** 比较 Shapelet 和全局 DTW 分类的区别：在什么类型的时间序列数据上，Shapelet 分类可能优于 1-NN + DTW？

### B 级（数学推导）

**B1.** 证明 Z-归一化欧氏距离可以用点积表示（方程 8.69）。提示：展开 $$d^2 = \sum (\hat{x}_i - \hat{y}_i)^2$$，其中 $$\hat{x}_i = (x_i - \mu_x)/\sigma_x$$。

**B2.** 证明 STOMP 的增量更新公式（方程 8.70）。从 $$Q_{i,j}$$ 和 $$Q_{i,j+1}$$ 的定义出发，逐项比较。

**B3.** 信息增益准则（方程 8.73）为什么适合评价 Shapelet 的判别力？如果用 Gini 不纯度替代熵，结果会有什么变化？

### C 级（数值计算）

**C1.** 给定时间序列 $$T = (1, 3, 2, 4, 1, 3, 2, 5, 1, 3)$$，子序列长度 $$m = 3$$。(1) 列出所有 8 个子序列；(2) 计算 $$T_{1,3}$$ 到所有其他子序列的 Z-归一化欧氏距离（排除区宽度 1）；(3) 写出 $$T_{1,3}$$ 的 Distance Profile。

**C2.** 对 C1 中的时间序列，计算完整的 Matrix Profile。找出 Motif 位置和 Discord 位置。

**C3.** 给定两类时间序列：类 A = {(1,2,3,2,1), (1,2,4,2,1)}，类 B = {(1,1,1,2,3), (1,1,1,3,3)}。考虑候选 Shapelet $$S = (2, 3, 2)$$，计算 $$\text{sdist}(S, T_k)$$ 对所有 4 条序列的值，并计算最优阈值下的信息增益。

### D 级（编程实践）

**D1.** 从零实现 Matrix Profile 的朴素算法（$$O(n^2 m)$$），在合成数据上验证：生成一条包含 3 个重复模式和 1 个异常片段的时间序列，验证 Motif 和 Discord 是否被正确识别。

**D2.** 使用 STUMPY 库计算长时间序列（$$n \geq 10000$$）的 Matrix Profile，并与朴素实现的结果比较。记录两种方法的运行时间差异。

**D3.** 实现 Shapelet 发现算法（含早停剪枝），在 UCR 时间序列数据集的一个二分类子集上测试。比较 Shapelet 分类器与 1-NN + 欧氏距离的准确率。

### E 级（案例分析）

**E1.** 在真实 ECG 数据集（如 MIT-BIH Arrhythmia Database 的一个通道）上：(1) 用 Matrix Profile 找到 top-5 Motif 和 top-3 Discord；(2) 对照医学标注，分析 Discord 是否对应已知的心律异常事件；(3) 讨论子序列长度 $$m$$ 对发现结果的影响。

**E2.** 在工业振动信号数据上：(1) 用 Matrix Profile 分析设备正常运行和故障前兆的差异；(2) 用 Shapelet 提取故障判别特征；(3) 比较 Matrix Profile Discord 检测和 Shapelet 分类两种故障检测策略。

### F 级（开放探讨）

**F1.** **多维 Matrix Profile**：标准 Matrix Profile 处理一维时间序列。当输入是多维时间序列（如三轴加速度计）时，如何扩展 Matrix Profile 的定义？讨论 mSTOMP（Yeh et al., 2017）的核心思想：是在所有维度上同时匹配，还是在维度子集上匹配？后者能带来什么优势？

**F2.** **Shapelet 的可解释性与因果性**：Shapelet 提供了直观的可解释性——"因为信号在某个时段呈现这种形状，所以分类为异常"。但相关性不等于因果性。讨论：(1) Shapelet 的判别力是否意味着该局部形状是类别差异的"原因"？(2) 如何设计实验来验证 Shapelet 的因果意义？(3) 在医疗诊断等高风险场景中，如何向领域专家解释 Shapelet 的发现？
