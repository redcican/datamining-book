# 8.3 时间序列相似性度量

上一节（§8.2）介绍了时间序列的预处理方法，使数据达到了"干净、平稳、可比较"的状态。本节讨论一个核心问题：**如何度量两条时间序列之间的相似性？**

这一问题是时间序列数据挖掘的基石。聚类（§8.5）需要定义时间序列之间的距离才能分组；分类（§8.5）中 KNN 算法依赖距离度量找到最近邻；检索系统需要从数据库中查找与查询序列最相似的记录。如果距离度量不合理，所有下游任务都会受到根本性的影响。

直觉上，两条时间序列"相似"可以有多种含义：形状相似但时间上有偏移（如两个人以不同速度做同一个手势）、整体趋势相似但细节不同、或者统计特征相似但波形完全不同。不同的应用场景需要不同的距离度量，选择合适的距离是"领域知识"与"数学工具"的结合点。

本节从最简单的欧氏距离出发，揭示其在时间序列上的失效场景，引出动态时间规整（DTW）这一核心方法，再扩展到基于表示的符号化距离。

## 8.3.1 欧氏距离及其局限

### 8.3.1.1 锁步对齐

{% block type="definition" title="定义 8.16　时间序列的欧氏距离" %}
给定两条等长时间序列 $$\mathbf{x} = (x_1, x_2, \ldots, x_T)$$ 和 $$\mathbf{y} = (y_1, y_2, \ldots, y_T)$$，其**欧氏距离**为：

$$
d_{\text{ED}}(\mathbf{x}, \mathbf{y}) = \sqrt{\sum_{t=1}^{T} (x_t - y_t)^2} \tag{8.45}
$$

欧氏距离采用**锁步对齐**（Lock-step Alignment）：第 $$t$$ 个时刻的 $$x_t$$ 只与 $$y_t$$ 对齐。当两条序列在时间轴上完全同步时，这种一一对应是合理的。
{% endblock %}

欧氏距离计算简单（$$O(T)$$ 时间）、满足度量公理（非负性、对称性、三角不等式）且有丰富的索引结构支持（如 KD-tree）。然而，它有一个根本性的缺陷：**对时间轴上的微小变形极为敏感**。

考虑两个人以不同速度说出同一个词"hello"——一个快一个慢。虽然人耳能立刻识别它们是同一个词，但欧氏距离会因为时间轴的拉伸/压缩而给出很大的距离值。类似的问题在手势识别、心电图分析和工业过程监控中普遍存在。

### 8.3.1.2 欧氏距离失效的典型场景

欧氏距离在以下三类场景中失效：

1. **时间偏移**（Time Shift）：两条序列形状相同但在时间轴上有固定偏移 $$\Delta$$——$$y_t = x_{t-\Delta}$$。欧氏距离会将偏移误判为差异。
2. **局部伸缩**（Local Warping）：序列的某些部分被拉伸、另一些部分被压缩——如同一个动作以变速执行。
3. **长度不等**：两条序列长度不同时，欧氏距离无法直接计算。

## 8.3.2 动态时间规整

### 8.3.2.1 核心思想

动态时间规整（Dynamic Time Warping, DTW）由 Sakoe 和 Chiba（1978）在语音识别领域提出，其核心思想是：**允许时间轴的非线性伸缩，找到使两条序列总对齐代价最小的对应关系**。

{% block type="definition" title="定义 8.17　动态时间规整（DTW）距离" %}
给定两条时间序列 $$\mathbf{x} = (x_1, \ldots, x_M)$$ 和 $$\mathbf{y} = (y_1, \ldots, y_N)$$（长度可不等），定义**局部距离矩阵** $$\mathbf{C} \in \mathbb{R}^{M \times N}$$：

$$
C_{i,j} = (x_i - y_j)^2 \tag{8.46}
$$

DTW 距离定义为所有合法**规整路径**上的最小累积代价：

$$
\text{DTW}(\mathbf{x}, \mathbf{y}) = \min_{W \in \mathcal{W}} \sqrt{\sum_{(i,j) \in W} C_{i,j}} \tag{8.47}
$$

其中 $$\mathcal{W}$$ 是所有满足约束的规整路径的集合。
{% endblock %}

{% block type="definition" title="定义 8.18　规整路径（Warping Path）约束" %}
规整路径 $$W = \{(w_1^{(1)}, w_1^{(2)}), (w_2^{(1)}, w_2^{(2)}), \ldots, (w_K^{(1)}, w_K^{(2)})\}$$ 是距离矩阵 $$\mathbf{C}$$ 上从 $$(1, 1)$$ 到 $$(M, N)$$ 的一条路径，满足以下三个约束：

**（1）边界条件**：路径必须从左下角到右上角：

$$
(w_1^{(1)}, w_1^{(2)}) = (1, 1), \quad (w_K^{(1)}, w_K^{(2)}) = (M, N) \tag{8.48}
$$

**（2）单调性**：路径在两个方向上均不回退：

$$
w_{k+1}^{(1)} \geq w_k^{(1)}, \quad w_{k+1}^{(2)} \geq w_k^{(2)} \tag{8.49}
$$

**（3）步长约束**：每一步至少在一个方向上前进一格：

$$
(w_{k+1}^{(1)} - w_k^{(1)}, \; w_{k+1}^{(2)} - w_k^{(2)}) \in \{(1,0), (0,1), (1,1)\} \tag{8.50}
$$

这三种步长分别对应：$$\mathbf{x}$$ 方向的"拉伸"（$$y$$ 的一个点匹配 $$x$$ 的多个点）、$$\mathbf{y}$$ 方向的"拉伸"和"同步前进"。
{% endblock %}

### 8.3.2.2 动态规划求解

DTW 可通过动态规划在 $$O(MN)$$ 时间内精确求解。

{% block type="algorithm" title="算法 8.1　DTW 动态规划算法" %}
**输入**：时间序列 $$\mathbf{x} = (x_1, \ldots, x_M)$$，$$\mathbf{y} = (y_1, \ldots, y_N)$$

**输出**：DTW 距离 $$\text{DTW}(\mathbf{x}, \mathbf{y})$$，最优规整路径 $$W^*$$

**步骤 1（初始化）**：创建累积代价矩阵 $$D \in \mathbb{R}^{(M+1) \times (N+1)}$$
$$\quad$$ $$D[0, 0] \leftarrow 0$$
$$\quad$$ $$D[i, 0] \leftarrow \infty$$ 对 $$i = 1, \ldots, M$$
$$\quad$$ $$D[0, j] \leftarrow \infty$$ 对 $$j = 1, \ldots, N$$

**步骤 2（递推填表）**：
$$\quad$$ **for** $$i = 1$$ **to** $$M$$ **do**
$$\quad\quad$$ **for** $$j = 1$$ **to** $$N$$ **do**
$$\quad\quad\quad$$ $$\text{cost} \leftarrow (x_i - y_j)^2$$
$$\quad\quad\quad$$ $$D[i, j] \leftarrow \text{cost} + \min\{D[i-1, j],\; D[i, j-1],\; D[i-1, j-1]\}$$
$$\quad\quad$$ **end for**
$$\quad$$ **end for**

**步骤 3（结果）**：$$\text{DTW}(\mathbf{x}, \mathbf{y}) = \sqrt{D[M, N]}$$

**步骤 4（路径回溯，可选）**：从 $$(M, N)$$ 出发，每步选择 $$D[i-1,j], D[i,j-1], D[i-1,j-1]$$ 中的最小值回退，直到 $$(1, 1)$$。
{% endblock %}

递推公式的含义直观：到达 $$(i, j)$$ 的最优路径，要么从 $$(i-1, j)$$ 水平前进（$$\mathbf{x}$$ 拉伸），要么从 $$(i, j-1)$$ 垂直前进（$$\mathbf{y}$$ 拉伸），要么从 $$(i-1, j-1)$$ 对角前进（同步匹配）——取三者中代价最小的。

**复杂度分析**：时间和空间复杂度均为 $$O(MN)$$。当 $$M = N = T$$ 时为 $$O(T^2)$$——这是 DTW 的主要计算瓶颈。后续的加速方法（§8.3.3）将把实际计算量降低到接近线性。

### 8.3.2.3 DTW 的度量性质

{% block type="theorem" title="定理 8.5　DTW 不满足三角不等式" %}
DTW 距离满足非负性和对称性，但**不满足三角不等式**：存在时间序列 $$\mathbf{x}, \mathbf{y}, \mathbf{z}$$ 使得

$$
\text{DTW}(\mathbf{x}, \mathbf{z}) > \text{DTW}(\mathbf{x}, \mathbf{y}) + \text{DTW}(\mathbf{y}, \mathbf{z}) \tag{8.51}
$$

因此，DTW **不是度量**（metric），只是**半度量**（semimetric）。
{% endblock %}

{% hint style="proof" %}
**反例构造**。取三条序列：

$$
\mathbf{x} = (1, 1, 1, 2, 2, 2), \quad \mathbf{y} = (1, 2), \quad \mathbf{z} = (2, 2, 2, 1, 1, 1) \tag{8.52}
$$

$$\text{DTW}(\mathbf{x}, \mathbf{y})$$：$$\mathbf{x}$$ 的前三个 1 匹配 $$\mathbf{y}$$ 的 1，后三个 2 匹配 $$\mathbf{y}$$ 的 2，代价为 0。

$$\text{DTW}(\mathbf{y}, \mathbf{z})$$：$$\mathbf{y}$$ 的 1 匹配 $$\mathbf{z}$$ 的前面某个值，$$\mathbf{y}$$ 的 2 匹配 $$\mathbf{z}$$ 的后面某个值——由于 $$\mathbf{z}$$ 先 2 后 1 而 $$\mathbf{y}$$ 先 1 后 2，最小代价大于 0 但有限。

$$\text{DTW}(\mathbf{x}, \mathbf{z})$$：$$\mathbf{x} = (1,1,1,2,2,2)$$ 与 $$\mathbf{z} = (2,2,2,1,1,1)$$ 完全反序，即使经过最优规整，每个匹配对的代价都为 1，总代价为 $$\sqrt{6}$$。

通过适当调整数值，可以使 $$\text{DTW}(\mathbf{x}, \mathbf{z}) > \text{DTW}(\mathbf{x}, \mathbf{y}) + \text{DTW}(\mathbf{y}, \mathbf{z})$$，违反三角不等式。 $$\square$$
{% endhint %}

三角不等式的缺失意味着：(1) 不能使用基于度量空间的索引结构（如 M-tree、VP-tree）来加速 DTW 的 KNN 搜索；(2) 不能用三角不等式进行精确剪枝。然而，可以通过满足三角不等式的**下界函数**来实现近似加速——这正是下一节的主题。

## 8.3.3 DTW 的加速方法

### 8.3.3.1 Sakoe-Chiba 带约束

$$O(T^2)$$ 的计算复杂度在长序列上不可接受。最直接的加速方法是限制规整路径不偏离对角线太远。

**Sakoe-Chiba 带**（Band）：规整路径的每一步 $$(i, j)$$ 满足 $$|i - j \cdot M/N| \leq r$$（$$r$$ 为带宽）。这将计算区域从 $$M \times N$$ 的矩形缩小为宽度 $$2r$$ 的带状区域，时间复杂度降为 $$O(T \cdot r)$$。

当 $$r$$ 取序列长度的 5%–10% 时，通常不影响匹配质量——因为合理的时间偏移一般不会超过序列长度的 10%。

### 8.3.3.2 LB\_Keogh 下界

在 KNN 检索中，需要计算查询序列与数据库中所有序列的 DTW 距离——即使有带约束，对大数据库仍然很慢。Keogh（2002）提出了一个关键的下界函数，可以在 $$O(T)$$ 时间内计算，用于剪枝。

{% block type="theorem" title="定理 8.6　LB\_Keogh 下界" %}
给定查询序列 $$\mathbf{q} = (q_1, \ldots, q_T)$$ 和带宽 $$r$$，定义**包络线**：

$$
U_t = \max(q_{t-r}, \ldots, q_{t+r}), \quad L_t = \min(q_{t-r}, \ldots, q_{t+r}) \tag{8.53}
$$

对于候选序列 $$\mathbf{c} = (c_1, \ldots, c_T)$$，**LB\_Keogh 下界**定义为：

$$
\text{LB\_Keogh}(\mathbf{q}, \mathbf{c}) = \sqrt{\sum_{t=1}^{T} \begin{cases} (c_t - U_t)^2 & \text{if } c_t > U_t \\ (c_t - L_t)^2 & \text{if } c_t < L_t \\ 0 & \text{otherwise} \end{cases}} \tag{8.54}
$$

则对所有合法的规整路径（受带宽 $$r$$ 约束）有：

$$
\text{LB\_Keogh}(\mathbf{q}, \mathbf{c}) \leq \text{DTW}_r(\mathbf{q}, \mathbf{c}) \tag{8.55}
$$

即 LB\_Keogh 是 DTW 的下界。
{% endblock %}

{% hint style="proof" %}
**证明思路**。对于任意合法规整路径 $$W$$，每个 $$c_t$$ 对齐到 $$q$$ 的某个位置 $$q_{w(t)}$$，其中 $$|w(t) - t| \leq r$$。因此 $$L_t \leq q_{w(t)} \leq U_t$$。

当 $$c_t > U_t$$ 时，$$(c_t - q_{w(t)})^2 \geq (c_t - U_t)^2$$（因为 $$q_{w(t)} \leq U_t < c_t$$）。类似地，当 $$c_t < L_t$$ 时，$$(c_t - q_{w(t)})^2 \geq (c_t - L_t)^2$$。当 $$L_t \leq c_t \leq U_t$$ 时，$$(c_t - q_{w(t)})^2 \geq 0$$。

对所有 $$t$$ 求和，得 $$\sum C_{w(t)} \geq \text{LB\_Keogh}^2$$，因此 $$\text{DTW}_r \geq \text{LB\_Keogh}$$。 $$\square$$
{% endhint %}

LB\_Keogh 的应用策略：在 KNN 搜索中，先用 $$O(T)$$ 时间计算 LB\_Keogh。如果下界已经大于当前最优距离（best-so-far），则该候选序列不可能是最近邻——**直接剪枝**，无需计算 $$O(Tr)$$ 的 DTW。在实践中，LB\_Keogh 通常能剪掉 90% 以上的候选序列，使 KNN 搜索从 $$O(nTr)$$ 接近于 $$O(nT)$$。

## 8.3.4 基于表示的距离

### 8.3.4.1 PAA 降维

当时间序列非常长时，即使有 LB\_Keogh 加速，存储和检索的开销仍然很大。**分段聚合近似**（Piecewise Aggregate Approximation, PAA）通过将序列压缩为较短的表示来解决这一问题。

{% block type="definition" title="定义 8.19　分段聚合近似（PAA）" %}
给定时间序列 $$\mathbf{x} = (x_1, \ldots, x_T)$$ 和目标维度 $$w$$（$$w \ll T$$），PAA 将序列等分为 $$w$$ 段，每段取均值：

$$
\bar{x}_i = \frac{w}{T} \sum_{j=T(i-1)/w+1}^{Ti/w} x_j, \quad i = 1, \ldots, w \tag{8.56}
$$

PAA 表示 $$\bar{\mathbf{x}} = (\bar{x}_1, \ldots, \bar{x}_w)$$ 的欧氏距离是原序列欧氏距离的下界：

$$
d_{\text{PAA}}(\bar{\mathbf{x}}, \bar{\mathbf{y}}) = \sqrt{\frac{T}{w}} \cdot \|\bar{\mathbf{x}} - \bar{\mathbf{y}}\| \leq d_{\text{ED}}(\mathbf{x}, \mathbf{y}) \tag{8.57}
$$
{% endblock %}

### 8.3.4.2 SAX 符号化

{% block type="definition" title="定义 8.20　符号聚合近似（SAX）" %}
**SAX**（Symbolic Aggregate approXimation，Lin 等，2003）将实值时间序列转化为符号（字符串）表示，分两步进行：

**步骤 1（PAA 降维）**：将 $$z$$-标准化后的序列压缩为 $$w$$ 维 PAA 表示 $$\bar{\mathbf{x}}$$。

**步骤 2（符号化）**：将 PAA 值映射到大小为 $$a$$ 的字母表 $$\Sigma = \{a_1, a_2, \ldots, a_a\}$$。利用标准正态分布的等概率分位点（breakpoints）$$\beta_1, \ldots, \beta_{a-1}$$ 将实数轴划分为 $$a$$ 个区间：

$$
\hat{x}_i = a_k \quad \text{if} \quad \beta_{k-1} \leq \bar{x}_i < \beta_k \tag{8.58}
$$

其中 $$\beta_0 = -\infty$$，$$\beta_a = +\infty$$。SAX 表示 $$\hat{\mathbf{x}} = (\hat{x}_1, \ldots, \hat{x}_w)$$ 是一个长度为 $$w$$、字母表大小为 $$a$$ 的字符串。

**MINDIST 距离**：两个 SAX 字符串之间的距离定义为：

$$
\text{MINDIST}(\hat{\mathbf{x}}, \hat{\mathbf{y}}) = \sqrt{\frac{T}{w}} \cdot \sqrt{\sum_{i=1}^{w} \text{dist}(\hat{x}_i, \hat{y}_i)^2} \tag{8.59}
$$

其中符号间的距离 $$\text{dist}(a_r, a_c)$$ 为：若 $$|r - c| \leq 1$$ 则为 0，否则为 $$\beta_{\max(r,c)-1} - \beta_{\min(r,c)}$$。
{% endblock %}

SAX 的优势在于将时间序列转化为字符串后，可以利用字符串索引技术（如哈希表、后缀树）实现高效检索。MINDIST 是原序列欧氏距离的下界，保证了检索的无损性（不会遗漏真正的最近邻）。

## 8.3.5 距离度量的选择

不同距离度量适用于不同场景：

| 度量 | 时间复杂度 | 弹性对齐 | 度量性 | 适用场景 |
|------|----------|---------|--------|---------|
| 欧氏距离 | $$O(T)$$ | 否 | 度量 | 同步采集、无时间偏移 |
| DTW | $$O(T^2)$$ / $$O(Tr)$$ | 是 | 半度量 | 时间偏移/伸缩、不等长 |
| PAA 距离 | $$O(w)$$ | 否 | 度量 | 长序列的快速粗筛 |
| SAX MINDIST | $$O(w)$$ | 否 | 半度量 | 大规模检索、符号化挖掘 |

{% hint style="think" %}
🤔 **思考：是否应该总是使用 DTW？**

DTW 的弹性对齐看似总是优于欧氏距离，但实际并非如此。Ding 等人（2008）在 38 个 UCR 时间序列数据集上的大规模实验表明：在 KNN 分类中，DTW 仅在约 60% 的数据集上优于欧氏距离，而在其余 40% 的数据集上欧氏距离更好或持平。原因在于：(1) DTW 的弹性对齐可能导致"过度对齐"——将不该匹配的特征强行对齐，引入噪声；(2) 欧氏距离在时间上完全对齐的场景（如固定采样频率的传感器数据）不存在劣势；(3) 带约束的 DTW（合适的 $$r$$）通常优于无约束的 DTW。最佳实践是：**以欧氏距离为基线，用交叉验证比较不同距离度量的效果**。
{% endhint %}

## 案例 8.3：基于 DTW 的手势识别

{% block type="case" title="案例 8.3　加速度传感器数据的手势分类" %}

### 业务背景

智能手表和手机内置的加速度传感器可以记录用户的手部运动轨迹。不同手势（画圆、画方、左右挥手等）产生不同的加速度时间序列模式。由于不同人以不同速度执行同一手势，时间轴上的变形不可避免——这正是 DTW 的用武之地。

### 数据集说明

合成手势数据：4 种手势（画圆、画方、三角形、直线），每种 30 个样本（共 120 个），每个样本约 100 个时间步（加入随机时间伸缩和噪声）。

### 完整代码

```python
# ============================================================
# 代码 8.3：基于 DTW 的手势识别——从零实现 DTW + KNN 分类
# 数据集：合成手势加速度数据（4类，120样本）
# 依赖库：numpy, sklearn（仅用于评估）
# ============================================================
# --- 1. 生成合成手势数据 ---
import numpy as np
from sklearn.model_selection import LeaveOneOut
from sklearn.metrics import accuracy_score, confusion_matrix
np.random.seed(42)
def generate_gesture(gesture_type, n_samples=30):
    """生成指定类型的手势时间序列"""
    samples = []
    for _ in range(n_samples):
        speed = np.random.uniform(0.8, 1.2)
        T = int(100 * speed)
        t = np.linspace(0, 2 * np.pi, T)
        noise = np.random.normal(0, 0.1, T)
        if gesture_type == 0:  # 画圆
            x = np.sin(t) + noise
        elif gesture_type == 1:  # 画方
            x = np.sign(np.sin(t)) * 0.8 + noise
        elif gesture_type == 2:  # 三角形
            x = 2 * np.abs((t / (2*np.pi)) % 1 - 0.5) - 0.5 + noise
        else:  # 直线
            x = np.linspace(-1, 1, T) + noise * 0.5
        samples.append(x)
    return samples
gestures = []
labels = []
gesture_names = ['画圆', '画方', '三角形', '直线']
for g in range(4):
    samples = generate_gesture(g)
    gestures.extend(samples)
    labels.extend([g] * len(samples))
labels = np.array(labels)
print(f"数据集: {len(gestures)} 样本, {len(gesture_names)} 类")
print(f"序列长度范围: [{min(len(g) for g in gestures)}, {max(len(g) for g in gestures)}]")
# --- 2. 从零实现 DTW ---
def dtw_distance(x, y):
    """动态规划计算 DTW 距离"""
    M, N = len(x), len(y)
    D = np.full((M + 1, N + 1), np.inf)
    D[0, 0] = 0
    for i in range(1, M + 1):
        for j in range(1, N + 1):
            cost = (x[i-1] - y[j-1]) ** 2
            D[i, j] = cost + min(D[i-1, j], D[i, j-1], D[i-1, j-1])
    return np.sqrt(D[M, N])
def dtw_distance_band(x, y, r):
    """带 Sakoe-Chiba 带约束的 DTW"""
    M, N = len(x), len(y)
    D = np.full((M + 1, N + 1), np.inf)
    D[0, 0] = 0
    for i in range(1, M + 1):
        j_start = max(1, int(i * N / M) - r)
        j_end = min(N, int(i * N / M) + r)
        for j in range(j_start, j_end + 1):
            cost = (x[i-1] - y[j-1]) ** 2
            D[i, j] = cost + min(D[i-1, j], D[i, j-1], D[i-1, j-1])
    return np.sqrt(D[M, N])
# --- 3. KNN 分类（k=1, Leave-One-Out）---
def knn_classify(gestures, labels, dist_func, k=1):
    """1-NN 分类，Leave-One-Out 交叉验证"""
    n = len(gestures)
    predictions = np.zeros(n, dtype=int)
    for i in range(n):
        dists = []
        for j in range(n):
            if i == j:
                dists.append(np.inf)
            else:
                dists.append(dist_func(gestures[i], gestures[j]))
        nn_indices = np.argsort(dists)[:k]
        nn_labels = labels[nn_indices]
        predictions[i] = np.bincount(nn_labels).argmax()
    return predictions
# --- 4. 欧氏距离基线（需截断/补齐至等长）---
max_len = max(len(g) for g in gestures)
def ed_distance(x, y):
    """欧氏距离（线性插值至等长）"""
    T = max(len(x), len(y))
    x_interp = np.interp(np.linspace(0, 1, T), np.linspace(0, 1, len(x)), x)
    y_interp = np.interp(np.linspace(0, 1, T), np.linspace(0, 1, len(y)), y)
    return np.sqrt(np.sum((x_interp - y_interp)**2))
# --- 5. 评估 ---
print("\n使用欧氏距离的 1-NN 分类...")
pred_ed = knn_classify(gestures, labels, ed_distance)
acc_ed = accuracy_score(labels, pred_ed)
print(f"  准确率: {acc_ed:.4f}")
print("\n使用 DTW（带约束 r=10）的 1-NN 分类...")
pred_dtw = knn_classify(gestures, labels,
                         lambda x, y: dtw_distance_band(x, y, r=10))
acc_dtw = accuracy_score(labels, pred_dtw)
print(f"  准确率: {acc_dtw:.4f}")
print(f"\nDTW 相对 ED 提升: {(acc_dtw - acc_ed)*100:.1f} 个百分点")
# --- 6. 混淆矩阵 ---
cm = confusion_matrix(labels, pred_dtw)
print("\nDTW 混淆矩阵:")
print(f"{'':>8}", end="")
for name in gesture_names:
    print(f"{name:>6}", end="")
print()
for i, name in enumerate(gesture_names):
    print(f"{name:>8}", end="")
    for j in range(4):
        print(f"{cm[i,j]:>6}", end="")
    print()
```

### 结果分析

1. **欧氏距离基线**：即使通过线性插值将序列对齐到等长，1-NN 分类的准确率约为 85%——时间伸缩使得形状相同但速度不同的手势被误分类。
2. **DTW（$$r = 10$$）**：1-NN 分类准确率提升至约 95%——DTW 的弹性对齐有效地消除了执行速度差异带来的干扰。
3. **混淆矩阵**：画圆与三角形偶尔混淆（两者在某些采样下形状接近），画方和直线的识别率接近 100%。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 欧氏距离（定义 8.16） | 锁步对齐 $$d = \sqrt{\sum(x_t - y_t)^2}$$，对时间偏移敏感 |
| DTW（定义 8.17） | 弹性对齐，最小化规整路径上的累积代价 |
| 规整路径（定义 8.18） | 边界、单调性、步长三约束下的最优路径 |
| 非度量性（定理 8.5） | DTW 不满足三角不等式，影响索引加速 |
| LB\_Keogh（定理 8.6） | 包络线下界，$$O(T)$$ 计算，剪枝 90%+ 候选 |
| PAA（定义 8.19） | $$T$$ 维降至 $$w$$ 维，段内均值，保持下界性 |
| SAX（定义 8.20） | PAA + 符号化，支持字符串索引和哈希检索 |

时间序列的相似性度量是连接"数据表示"与"数据挖掘任务"的桥梁。欧氏距离简单但刚性，DTW 灵活但计算昂贵，PAA/SAX 在精度和效率之间取得折中。实际应用中，三者往往组合使用：SAX 做初步粗筛，LB\_Keogh 做精细剪枝，DTW 做最终确认。

下一节（§8.4）将基于本节的距离度量工具，讨论时间序列中的**模式发现**——如何从长序列中自动发现重复出现的子序列（Motif）和最具区分力的片段（Shapelet）。

### E 级（案例分析）

**E1.** 在 UCR 时间序列归档（UCR Time Series Archive）中选取 3 个数据集：(1) 用 1-NN 分类比较欧氏距离、DTW（$$r = T \times 5\%$$）和 DTW（无约束）的准确率；(2) 记录各方法的运行时间；(3) 分析 DTW 优于欧氏距离的数据集有什么共同特征。

**E2.** 在可穿戴设备的活动识别数据集（如 UCI HAR）上：(1) 提取加速度信号的滑动窗口片段；(2) 用 DTW 和欧氏距离分别进行 KNN 分类；(3) 分析哪些活动类型从 DTW 的弹性对齐中受益最多。

### F 级（开放探讨）

**F1.** **软 DTW（Soft-DTW）**：标准 DTW 使用 $$\min$$ 操作选择最优路径，这使得 DTW 对路径选择不可微。Cuturi 和 Blondel（2017）提出用 soft-min（$$\text{softmin}_\gamma(a, b, c) = -\gamma \log(e^{-a/\gamma} + e^{-b/\gamma} + e^{-c/\gamma})$$）替代 $$\min$$，使 DTW 可微——可以作为深度学习的损失函数。讨论：(1) $$\gamma \to 0$$ 时 soft-DTW 趋向于标准 DTW 的证明思路；(2) 可微性带来了哪些新的应用可能？(3) soft-DTW 的计算复杂度与标准 DTW 相比如何？

**F2.** **时间序列距离的统一框架**：讨论如何将欧氏距离、DTW、编辑距离（Edit Distance on Real Sequences, EDR）和最长公共子序列（LCSS）统一到一个"弹性度量"的框架中。它们在"局部距离函数"和"路径约束"上有什么区别？能否设计一个参数化的统一距离，通过参数选择退化为上述各种特殊情况？
