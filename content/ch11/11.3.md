# 11.3 循环神经网络与序列数据挖掘

§11.1 和 §11.2 分别介绍了全连接网络和卷积神经网络——前者处理固定维度的向量输入，后者利用局部连接处理二维网格结构的图像数据。然而，现实世界中大量重要的数据是以**序列**形式存在的：股票价格随时间波动，自然语言由词语顺序排列，DNA 由碱基按序串联，传感器持续产生时间戳信号。这些序列数据有两个关键特点：(1) **长度可变**——一句话可以是 5 个词也可以是 50 个词；(2) **时序依赖**——当前的状态取决于过去的历史，而且这种依赖可能跨越很长的距离（"那只在第一段出现的**猫**……后来**它**跳上了桌子"）。

全连接网络和 CNN 都无法自然地处理这类数据。MLP 要求固定维度的输入，无法直接接受长度可变的序列；CNN 虽然可以通过一维卷积处理序列，但其感受野受限于卷积核大小和网络深度，难以捕获长距离依赖。**循环神经网络**（Recurrent Neural Network, RNN）通过在隐藏层引入**自循环连接**来解决这一问题——网络在每个时间步接收当前输入和上一步的隐藏状态，将历史信息压缩为一个固定维度的状态向量逐步传递。这种"记忆"机制使 RNN 能够建模任意长度的序列依赖关系。

本节将从序列数据的特点出发，介绍基本 RNN 的结构与训练方法，分析其梯度消失/爆炸问题，然后引入 LSTM 和 GRU 两种门控机制的数学原理，最后通过时间序列预测案例将理论付诸实践。

## 11.3.1 序列数据与循环结构

在正式定义 RNN 之前，我们先理解序列数据的独特之处以及为何需要一种新的网络架构。

**序列数据的普遍性**。在数据挖掘的实际应用中，序列数据无处不在：金融领域的股票价格时间序列、自然语言处理中的文本序列、生物信息学中的 DNA/蛋白质序列、物联网中的传感器信号流、用户行为分析中的点击流序列等。这些数据的共同特征是：样本之间存在**时间或顺序上的依赖关系**——当前的观测值不仅取决于当前的输入特征，还取决于过去的历史状态。

**前馈网络的局限**。如果我们用一个固定窗口的 MLP 或 CNN 来处理序列，必须事先决定"回看多少步历史"，这带来两个问题：(1) 窗口太小会遗漏关键的长距离依赖；(2) 窗口太大会导致参数量爆炸（MLP）或需要极深的网络（CNN）。更根本地，前馈网络没有"记忆"——它独立地处理每个输入样本，无法在时间步之间传递信息。

**循环结构的核心思想**。RNN 的解决方案极为优雅：在隐藏层中引入一条从自身到自身的**循环连接**，使得网络在每个时间步 $$t$$ 不仅接收当前输入 $$\mathbf{x}_t$$，还接收上一时间步的隐藏状态 $$\mathbf{h}_{t-1}$$——后者是对所有历史信息 $$\mathbf{x}_1, \ldots, \mathbf{x}_{t-1}$$ 的压缩表示。这种机制类似于人类阅读文章的过程：我们不会在每个词处从头重新阅读整篇文章，而是维护一个不断更新的"理解状态"，将其与当前读到的词结合来推断含义。至关重要的是，RNN 的参数在所有时间步之间**共享**——这与 CNN 在空间维度上共享卷积核的思想一脉相承，只是共享的维度从空间转移到了时间。

## 11.3.2 基本循环神经网络

### RNN 的前向传播

理解了循环结构的直觉后，我们来形式化定义基本 RNN 的数学模型。

{% block type="definition" %}
**定义 11.14**（循环神经网络）　设输入序列 $$\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_T$$，其中 $$\mathbf{x}_t \in \mathbb{R}^d$$。循环神经网络在每个时间步 $$t$$ 维护一个隐藏状态 $$\mathbf{h}_t \in \mathbb{R}^h$$，通过以下递推关系更新：

$$
\mathbf{h}_t = \tanh\!\big(W_{hh}\, \mathbf{h}_{t-1} + W_{xh}\, \mathbf{x}_t + \mathbf{b}_h\big) \tag{11.44}
$$

$$
\mathbf{y}_t = W_{hy}\, \mathbf{h}_t + \mathbf{b}_y \tag{11.45}
$$

其中 $$W_{xh} \in \mathbb{R}^{h \times d}$$ 为输入-隐藏权重，$$W_{hh} \in \mathbb{R}^{h \times h}$$ 为隐藏-隐藏权重，$$W_{hy} \in \mathbb{R}^{o \times h}$$ 为隐藏-输出权重，$$\mathbf{b}_h, \mathbf{b}_y$$ 为偏置。初始隐藏状态 $$\mathbf{h}_0$$ 通常初始化为零向量。
{% endblock %}

式 (11.44) 是 RNN 的核心——它揭示了 RNN 与前馈网络的本质区别。在 MLP 中，每层的输出仅取决于该层的输入；而在 RNN 中，隐藏状态 $$\mathbf{h}_t$$ 同时取决于当前输入 $$\mathbf{x}_t$$ **和**上一步的隐藏状态 $$\mathbf{h}_{t-1}$$。这种递推关系使得 $$\mathbf{h}_t$$ 隐式地包含了从 $$\mathbf{x}_1$$ 到 $$\mathbf{x}_t$$ 的全部历史信息——尽管这些信息被压缩到了一个固定维度的向量中。

要注意的是，$$W_{xh}$$、$$W_{hh}$$、$$W_{hy}$$ 这三个权重矩阵在**所有时间步共享**。这意味着 RNN 的参数量与序列长度无关——无论序列有 10 步还是 10,000 步，网络的参数总量恒为 $$h \times d + h \times h + o \times h + h + o$$。这正是循环结构的参数效率优势：通过时间维度上的权值共享，RNN 可以处理任意长度的序列。

为了理解 RNN 的计算过程，一个有效的方法是将循环结构在时间轴上**展开**（Unrolling）：将 $$T$$ 个时间步的递推展开为 $$T$$ 个相同结构的网络层，每层接收对应时间步的输入 $$\mathbf{x}_t$$ 和前一层的隐藏状态 $$\mathbf{h}_{t-1}$$，如图 11.3.1 所示。展开后的 RNN 形式上等价于一个 $$T$$ 层的深层前馈网络——但所有层共享同一组参数。这一视角不仅帮助我们直观理解信息的流动，也是推导训练算法的基础。

### 时间反向传播

展开后的 RNN 在结构上等价于一个 $$T$$ 层的深层网络，因此可以直接应用反向传播算法来计算梯度——这就是**时间反向传播**（Backpropagation Through Time, BPTT）算法。

{% block type="definition" %}
**定义 11.15**（时间反向传播）　设总损失为各时间步损失之和 $$\mathcal{L} = \sum_{t=1}^{T} \mathcal{L}_t$$。BPTT 算法沿时间轴从后向前传播误差，损失对共享权重 $$W_{hh}$$ 的梯度为：

$$
\frac{\partial \mathcal{L}}{\partial W_{hh}} = \sum_{t=1}^{T} \sum_{k=1}^{t} \frac{\partial \mathcal{L}_t}{\partial \mathbf{h}_t} \left(\prod_{j=k+1}^{t} \frac{\partial \mathbf{h}_j}{\partial \mathbf{h}_{j-1}}\right) \frac{\partial \mathbf{h}_k}{\partial W_{hh}} \tag{11.46}
$$

其中，相邻隐藏状态之间的 Jacobian 矩阵为：

$$
\frac{\partial \mathbf{h}_t}{\partial \mathbf{h}_{t-1}} = \operatorname{diag}\!\big(1 - \mathbf{h}_t^2\big) \cdot W_{hh} \tag{11.47}
$$

（利用了 $$\tanh'(z) = 1 - \tanh^2(z)$$ 的性质。）
{% endblock %}

式 (11.46) 的关键在于连乘项 $$\prod_{j=k+1}^{t} \frac{\partial \mathbf{h}_j}{\partial \mathbf{h}_{j-1}}$$——损失从时间步 $$t$$ 传播到时间步 $$k$$ 的梯度需要经过 $$t - k$$ 个 Jacobian 矩阵的连续相乘。这种连乘结构与 §11.1.5 中深层前馈网络的梯度消失问题如出一辙，但在 RNN 中由于序列可能很长（$$T$$ 达数百甚至数千步），问题更加严重。

### 梯度消失与梯度爆炸

{% block type="theorem" %}
**定理 11.5**（RNN 的梯度消失与爆炸）　对于基本 RNN（式 11.44），从时间步 $$t$$ 回传到时间步 $$k$$ 的梯度范数满足：

$$
\left\|\prod_{j=k+1}^{t} \frac{\partial \mathbf{h}_j}{\partial \mathbf{h}_{j-1}}\right\| \le \left(\|W_{hh}\| \cdot \gamma\right)^{t-k} \tag{11.48}
$$

其中 $$\gamma = \max_j \|{\operatorname{diag}(1 - \mathbf{h}_j^2)}\| \le 1$$（因为 $$\tanh$$ 的导数值域为 $$(0, 1]$$）。

- 当 $$\|W_{hh}\| \cdot \gamma < 1$$ 时，梯度以 $$(\|W_{hh}\| \cdot \gamma)^{t-k}$$ 的速率**指数衰减**——这就是**梯度消失**，导致网络无法学习长距离依赖关系。
- 当 $$\|W_{hh}\|$$ 的最大奇异值 $$> 1/\gamma$$ 时，梯度可能**指数增长**——这就是**梯度爆炸**，导致参数更新过大，训练不稳定。

**证明思路**：由式 (11.47)，每步 Jacobian 的范数 $$\|\frac{\partial \mathbf{h}_j}{\partial \mathbf{h}_{j-1}}\| \le \|\operatorname{diag}(1 - \mathbf{h}_j^2)\| \cdot \|W_{hh}\| \le \gamma \cdot \|W_{hh}\|$$。$$t - k$$ 步连乘的范数由子乘法性（submultiplicativity）得到上界 $$(\gamma \|W_{hh}\|)^{t-k}$$。当该值小于 1 时指数衰减，大于 1 时指数增长。$$\blacksquare$$
{% endblock %}

定理 11.5 揭示了基本 RNN 的根本缺陷：**短期记忆**。梯度消失意味着损失对远处时间步参数的梯度趋近于零——网络实际上"忘记"了久远的输入。实验表明，基本 RNN 通常只能有效地建模约 10–20 步以内的依赖关系。对于梯度爆炸，一个常用的工程手段是**梯度裁剪**（Gradient Clipping）：当梯度范数超过阈值 $$\theta$$ 时，将其缩放回 $$\theta$$：$$\mathbf{g} \leftarrow \frac{\theta}{\|\mathbf{g}\|} \mathbf{g}$$。但梯度消失问题无法通过裁剪解决——需要从网络结构上进行根本改进，这正是 LSTM 的动机。

{% hint style="warning" %}
⚠️ **梯度消失 ≠ 信息消失**。隐藏状态 $$\mathbf{h}_t$$ 中确实包含了历史信息（式 11.44 的递推保证了这一点），但梯度消失使得网络**无法通过学习来利用**这些久远的信息——优化器"看不到"远处的误差信号，因此无法调整参数去捕获长距离依赖。问题出在训练过程，而非网络的表达能力。
{% endhint %}

## 11.3.3 长短期记忆网络

1997 年，Hochreiter 和 Schmidhuber 提出了**长短期记忆网络**（Long Short-Term Memory, LSTM），通过精心设计的**门控机制**从根本上解决了梯度消失问题。LSTM 的核心创新是引入了一个独立的**细胞状态**（Cell State）$$\mathbf{c}_t$$，它像一条"信息高速公路"贯穿整个时间轴——信息可以沿着这条高速公路几乎无损地传递，而三个可学习的"门"控制着信息的写入、保留和读取。

### LSTM 细胞结构

{% block type="definition" %}
**定义 11.16**（长短期记忆网络）　LSTM 在每个时间步 $$t$$ 维护隐藏状态 $$\mathbf{h}_t$$ 和细胞状态 $$\mathbf{c}_t$$，通过以下四组方程更新：

**遗忘门**（Forget Gate）——决定从细胞状态中**丢弃**哪些旧信息：

$$
\mathbf{f}_t = \sigma\!\big(W_f [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_f\big) \tag{11.49}
$$

**输入门**（Input Gate）——决定将哪些**新信息**写入细胞状态：

$$
\mathbf{i}_t = \sigma\!\big(W_i [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_i\big) \tag{11.50}
$$

$$
\tilde{\mathbf{c}}_t = \tanh\!\big(W_c [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_c\big) \tag{11.51}
$$

**细胞状态更新**——旧信息的选择性遗忘 + 新信息的选择性写入：

$$
\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t \tag{11.52}
$$

**输出门**（Output Gate）——决定从细胞状态中**读取**哪些信息作为当前隐藏状态：

$$
\mathbf{o}_t = \sigma\!\big(W_o [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_o\big) \tag{11.53}
$$

$$
\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t) \tag{11.54}
$$

其中 $$[\mathbf{h}_{t-1}, \mathbf{x}_t]$$ 表示向量拼接，$$\sigma$$ 为 Sigmoid 函数（输出值域 $$(0, 1)$$），$$\odot$$ 为逐元素乘法。
{% endblock %}

理解 LSTM 的关键在于理解**每个门的角色**及其协同工作方式：

- **遗忘门** $$\mathbf{f}_t$$（式 11.49）：输出值接近 0 表示"忘记"对应维度的旧信息，接近 1 表示"完整保留"。例如，在语言模型中，当遇到新的句子主语时，遗忘门可以清除旧主语的信息。

- **输入门** $$\mathbf{i}_t$$（式 11.50）和**候选值** $$\tilde{\mathbf{c}}_t$$（式 11.51）：$$\tilde{\mathbf{c}}_t$$ 是候选新信息（类似于基本 RNN 的隐藏状态更新），$$\mathbf{i}_t$$ 控制哪些候选信息被实际写入。两者的分离使网络可以独立决定"生成什么新信息"和"写入多少新信息"。

- **细胞状态更新**（式 11.52）：这是 LSTM 的**核心方程**。与基本 RNN 对隐藏状态做**乘法变换**（$$\mathbf{h}_t = \tanh(W_{hh} \mathbf{h}_{t-1} + \ldots)$$）不同，LSTM 对细胞状态做**加法更新**——$$\mathbf{c}_t$$ 是 $$\mathbf{c}_{t-1}$$ 的线性组合加上新信息。这种加法结构确保了梯度可以沿细胞状态几乎无损地回传：$$\frac{\partial \mathbf{c}_t}{\partial \mathbf{c}_{t-1}} = \operatorname{diag}(\mathbf{f}_t)$$，当遗忘门接近 1 时，梯度接近恒等映射，从根本上缓解了梯度消失问题。

- **输出门** $$\mathbf{o}_t$$（式 11.53）：控制细胞状态中的哪些信息被暴露给当前的隐藏状态输出。细胞状态是长期记忆的载体，而隐藏状态是短期工作记忆——输出门将两者解耦。

  <figcaption>图 11.3.2　LSTM 细胞结构详解。细胞状态 $$\mathbf{c}_{t-1} \to \mathbf{c}_t$$ 像一条"信息高速公路"贯穿上方，遗忘门（红色）控制旧信息的丢弃，输入门（绿色）控制新信息的写入，输出门（蓝色）控制信息的读取。⊗ 表示逐元素乘法，⊕ 表示逐元素加法，σ 和 tanh 表示激活函数。</figcaption>

{% hint style="tip" %}
💡 **LSTM 的直觉类比**：可以把 LSTM 想象成一个带有选择性记忆的笔记本。**细胞状态**是笔记本的内容；**遗忘门**是橡皮擦，决定擦掉哪些旧笔记；**输入门**是钢笔，决定写入哪些新笔记；**输出门**是一个滤镜，决定在当前任务中参考哪些笔记内容。关键在于：笔记本的内容可以长时间保持不变（遗忘门 ≈ 1，输入门 ≈ 0），这就是"长期记忆"；而"短期记忆"（隐藏状态 $$\mathbf{h}_t$$）则根据当前任务需要从笔记本中选择性地读取。
{% endhint %}

## 11.3.4 门控循环单元

LSTM 的四组方程涉及三个独立的门和一个细胞状态，参数量较大。2014 年，Cho 等人提出了**门控循环单元**（Gated Recurrent Unit, GRU），将 LSTM 的三个门简化为两个，并取消了独立的细胞状态——在保留门控核心思想的同时显著减少了参数量。

{% block type="definition" %}
**定义 11.17**（门控循环单元）　GRU 在每个时间步 $$t$$ 仅维护隐藏状态 $$\mathbf{h}_t$$，通过以下方程更新：

**更新门**（Update Gate）——兼具 LSTM 遗忘门和输入门的功能：

$$
\mathbf{z}_t = \sigma\!\big(W_z [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_z\big) \tag{11.55}
$$

**重置门**（Reset Gate）——控制历史信息对候选隐藏状态的影响：

$$
\mathbf{r}_t = \sigma\!\big(W_r [\mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_r\big) \tag{11.56}
$$

**候选隐藏状态**——在重置门控制下融合历史信息与当前输入：

$$
\tilde{\mathbf{h}}_t = \tanh\!\big(W_h [\mathbf{r}_t \odot \mathbf{h}_{t-1}, \mathbf{x}_t] + \mathbf{b}_h\big) \tag{11.57}
$$

**隐藏状态更新**——在旧状态与候选状态之间做凸组合：

$$
\mathbf{h}_t = (1 - \mathbf{z}_t) \odot \mathbf{h}_{t-1} + \mathbf{z}_t \odot \tilde{\mathbf{h}}_t \tag{11.58}
$$
{% endblock %}

GRU 的设计有两个巧妙之处。第一，**更新门** $$\mathbf{z}_t$$ 同时承担了 LSTM 中遗忘门和输入门的角色：当 $$\mathbf{z}_t \approx 0$$ 时，$$\mathbf{h}_t \approx \mathbf{h}_{t-1}$$（保留旧信息，类似遗忘门 = 1）；当 $$\mathbf{z}_t \approx 1$$ 时，$$\mathbf{h}_t \approx \tilde{\mathbf{h}}_t$$（接受新信息，类似输入门 = 1）。由于保留和更新的权重被约束为互补（$$(1-\mathbf{z}_t)$$ 和 $$\mathbf{z}_t$$），GRU 用一个门实现了两个功能。第二，**重置门** $$\mathbf{r}_t$$ 在计算候选状态时控制"回看多少历史"：当 $$\mathbf{r}_t \approx 0$$ 时，$$\tilde{\mathbf{h}}_t$$ 仅取决于当前输入 $$\mathbf{x}_t$$，相当于重置历史记忆；当 $$\mathbf{r}_t \approx 1$$ 时，历史信息完整参与计算。

**LSTM vs GRU**。下表从多个维度比较两种门控 RNN：

| 特性 | LSTM | GRU |
|------|------|-----|
| 门的数量 | 3（遗忘、输入、输出） | 2（更新、重置） |
| 状态变量 | $$\mathbf{h}_t$$ + $$\mathbf{c}_t$$ | 仅 $$\mathbf{h}_t$$ |
| 参数量（隐藏维度 $$h$$，输入维度 $$d$$） | $$4h(h+d+1)$$ | $$3h(h+d+1)$$ |
| 长距离依赖 | 优秀（独立细胞状态） | 良好 |
| 训练速度 | 较慢 | 较快（参数少 25%） |
| 适用场景 | 序列很长、依赖复杂 | 数据量较小、序列较短 |

{% hint style="tip" %}
💡 **如何选择 LSTM 还是 GRU**？经验法则：(1) 当序列很长且长期依赖是核心挑战时，优先选择 LSTM——其独立的细胞状态提供了更强的长期记忆能力；(2) 当计算资源有限或数据集较小时，GRU 是更务实的选择——参数更少意味着更不容易过拟合且训练更快；(3) 在多数实际任务中，两者的性能差异并不显著，建议都尝试后选择在验证集上更优的那个。
{% endhint %}

## 11.3.5 序列建模的扩展

基本的 RNN/LSTM/GRU 只沿一个方向（从左到右）处理序列，且只有一层隐藏层。在许多实际应用中，我们可以通过两种方式增强模型能力：双向处理和深层堆叠。

### 双向循环网络

在某些任务中，预测当前时间步的输出不仅需要过去的信息，还需要**未来的信息**。例如在命名实体识别中，判断"苹果"是水果还是公司，可能需要看它后面的词语（"苹果很甜" vs "苹果发布了新产品"）。**双向 RNN**（Bidirectional RNN）通过两个方向相反的 RNN 同时处理序列来解决这一问题。

{% block type="definition" %}
**定义 11.18**（双向循环网络）　双向 RNN 由一个**前向 RNN**（从 $$t=1$$ 到 $$t=T$$）和一个**后向 RNN**（从 $$t=T$$ 到 $$t=1$$）组成。两个方向的隐藏状态拼接后作为每个时间步的最终表示：

前向隐藏状态：$$\overrightarrow{\mathbf{h}}_t = \text{RNN}_{\text{fwd}}(\mathbf{x}_t, \overrightarrow{\mathbf{h}}_{t-1})$$

后向隐藏状态：$$\overleftarrow{\mathbf{h}}_t = \text{RNN}_{\text{bwd}}(\mathbf{x}_t, \overleftarrow{\mathbf{h}}_{t+1})$$

最终隐藏状态：$$\mathbf{h}_t = [\overrightarrow{\mathbf{h}}_t;\, \overleftarrow{\mathbf{h}}_t] \in \mathbb{R}^{2h}$$

其中 $$[\cdot;\, \cdot]$$ 表示向量拼接。前向和后向 RNN 各自拥有独立的参数。
{% endblock %}

双向 RNN 使每个时间步的表示同时包含了来自过去和未来的上下文信息，在自然语言处理（NER、机器翻译的编码器）和语音识别等需要全局上下文的任务中广泛使用。需要注意的是，双向 RNN 要求在推理时已知完整序列——因此不适用于实时预测或在线生成等只能获取历史信息的场景。

### 深层循环网络

另一种增强方向是在垂直维度上堆叠多层 RNN，形成**深层 RNN**（Deep RNN 或 Stacked RNN）。第一层 RNN 将原始输入序列 $$\mathbf{x}_t$$ 映射为第一层隐藏状态序列 $$\mathbf{h}_t^{(1)}$$，第二层 RNN 将 $$\mathbf{h}_t^{(1)}$$ 作为输入产生 $$\mathbf{h}_t^{(2)}$$，以此类推。这种堆叠使模型能够学习更高层次的时序抽象——低层捕获局部时序模式，高层捕获全局语义模式。实践中，2–3 层的 LSTM/GRU 堆叠是常见配置；过多层数需要配合残差连接等技术（类似于 §11.2.5 的 ResNet 思想）来保证训练稳定性。

## 11.3.6 方法对比

从基本 RNN 到 LSTM、GRU，再到双向和深层扩展，循环网络的设计始终围绕一个核心挑战：如何在保留长期记忆的同时高效地训练。下表总结了各类循环网络的核心特征：

| 架构 | 核心机制 | 参数量级 | 长期依赖 | 适用场景 |
|------|---------|---------|---------|---------|
| 基本 RNN | 隐藏状态递推 | $$h(h+d)$$ | 差（~10 步） | 短序列、简单模式 |
| LSTM | 细胞状态 + 三门控 | $$4h(h+d)$$ | 优秀 | 长序列、复杂依赖 |
| GRU | 两门控、无细胞状态 | $$3h(h+d)$$ | 良好 | 中等序列、资源受限 |
| 双向 LSTM | 前向 + 后向 | $$2 \times 4h(h+d)$$ | 优秀 | 需全局上下文 |
| 深层 LSTM | 多层堆叠 | $$L \times 4h(h+d)$$ | 优秀 | 复杂层次特征 |

从表中可以看出：(1) **门控机制是解决长期依赖的关键**——LSTM 的长期记忆能力远超基本 RNN，代价是参数量增加约 4 倍；(2) **GRU 以 75% 的参数量达到了接近 LSTM 的性能**，在参数效率上更具优势；(3) **双向和深层扩展可以进一步提升性能**，但需要根据具体任务权衡计算成本。值得注意的是，近年来基于注意力机制的 Transformer 架构在许多序列任务上已超越 RNN，但 RNN 在实时流式处理、资源受限场景和序列长度极端不一致的任务中仍有其独特优势。

## 案例 11.3：正弦波时间序列预测

理论分析完毕后，让我们通过一个时间序列预测的实践案例来验证 LSTM 捕获序列模式的能力。正弦波预测虽然简单，但它清晰地展示了 RNN 从历史观测中学习周期性模式并外推预测的核心能力——这一能力是所有更复杂的时间序列预测任务（金融、气象、工业传感器）的基础。

{% block type="case" %}
**案例 11.3**（正弦波时间序列预测）

**任务**：给定一段正弦信号 $$y(t) = \sin(2\pi t / 50)$$ 的历史观测值，使用 LSTM 预测未来的信号值。输入为过去 $$L = 30$$ 个时间步的序列，输出为下一个时间步的预测值。

**模型架构**：单层 LSTM（隐藏维度 64）+ 全连接输出层：

$$
\text{Input}(30 \times 1) \to \text{LSTM}(h{=}64) \to \text{FC}(1)
$$

取 LSTM 最后一个时间步的隐藏状态 $$\mathbf{h}_T$$ 送入全连接层产生标量预测值。
{% endblock %}

{% block type="algorithm" %}
**算法 11.3**（LSTM 时间序列预测训练流程）

---

**输入**：时间序列 $$\{y_1, y_2, \ldots, y_N\}$$，回看窗口长度 $$L$$，学习率 $$\eta$$，轮数 $$T$$

**输出**：训练好的 LSTM 参数 $$\boldsymbol{\theta}^*$$

1. 构建滑动窗口样本：$$\mathbf{x}_i = (y_i, \ldots, y_{i+L-1})$$，$$\hat{y}_i = y_{i+L}$$
2. 初始化 LSTM 参数 $$\boldsymbol{\theta}$$
3. **for** $$t = 1, 2, \ldots, T$$ **do**
4. 　　**for** 每个小批量 $$\mathcal{B}$$ **do**
5. 　　　　初始化 LSTM 状态 $$(\mathbf{h}_0, \mathbf{c}_0) = (\mathbf{0}, \mathbf{0})$$
6. 　　　　逐时间步前向传播（式 11.49–11.54）
7. 　　　　取最后时间步隐藏状态 $$\mathbf{h}_L$$ → 全连接层 → 预测 $$\hat{y}$$
8. 　　　　计算 MSE 损失：$$\mathcal{L} = \frac{1}{B}\sum_{i \in \mathcal{B}} (y_i - \hat{y}_i)^2$$
9. 　　　　BPTT 反向传播：计算 $$\nabla_{\boldsymbol{\theta}} \mathcal{L}$$
10. 　　　　参数更新：$$\boldsymbol{\theta} \leftarrow \text{Adam}(\boldsymbol{\theta},\, \nabla_{\boldsymbol{\theta}} \mathcal{L},\, \eta)$$
11. 　　**end for**
12. **end for**
13. **return** $$\boldsymbol{\theta}^* = \boldsymbol{\theta}$$

---
{% endblock %}

**PyTorch 实现**：

```python
import numpy as np
import torch
import torch.nn as nn
from torch.utils.data import DataLoader, TensorDataset

# --- 1. 生成正弦波数据 ---
t = np.linspace(0, 8 * np.pi, 500)
y = np.sin(t)
seq_len = 30

# --- 2. 构建滑动窗口样本 ---
X, Y = [], []
for i in range(len(y) - seq_len):
    X.append(y[i:i+seq_len])
    Y.append(y[i+seq_len])
X = torch.FloatTensor(np.array(X)).unsqueeze(-1)  # (N, 30, 1)
Y = torch.FloatTensor(np.array(Y)).unsqueeze(-1)  # (N, 1)
train_size = int(len(X) * 0.8)
train_loader = DataLoader(
    TensorDataset(X[:train_size], Y[:train_size]),
    batch_size=32, shuffle=True)

# --- 3. 定义 LSTM 模型 ---
class LSTMPredictor(nn.Module):
    def __init__(self, input_dim=1, hidden_dim=64):
        super().__init__()
        self.lstm = nn.LSTM(input_dim, hidden_dim, batch_first=True)
        self.fc = nn.Linear(hidden_dim, 1)

    def forward(self, x):
        out, (h_n, c_n) = self.lstm(x)
        return self.fc(h_n.squeeze(0))

model = LSTMPredictor()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.MSELoss()

# --- 4. 训练循环 ---
for epoch in range(100):
    model.train()
    for xb, yb in train_loader:
        pred = model(xb)
        loss = criterion(pred, yb)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

# --- 5. 测试评估 ---
model.eval()
with torch.no_grad():
    pred_all = model(X[train_size:]).numpy().flatten()
    true_all = Y[train_size:].numpy().flatten()
mse = np.mean((pred_all - true_all) ** 2)
print(f"测试 MSE: {mse:.6f}")
# 典型结果: MSE ≈ 0.0001 (100 epoch)
```

**结果分析**：LSTM 在仅 100 轮训练后即可将测试 MSE 降至 $$10^{-4}$$ 量级，预测值与真实正弦波几乎完全重合（见图 11.3.3b）。这个结果说明了两个要点：(1) LSTM 能够从历史观测中自动学习正弦信号的**周期性模式**，而无需任何关于信号频率或相位的先验知识；(2) LSTM 的门控机制有效地在 30 个时间步的回看窗口中保留了关键的相位信息——如果换用基本 RNN，由于梯度消失导致长距离依赖学习困难，预测精度会显著下降。在实际的数据挖掘应用中，同样的 LSTM 架构可以推广到金融时序预测（股价、汇率）、设备故障预测（传感器读数异常检测）和用户行为序列建模等更复杂的任务。

## 本节小结

本节系统介绍了循环神经网络家族的核心成员及其数学原理。从序列数据的特点出发，我们理解了 RNN 循环结构的设计动机和参数共享机制。通过 BPTT 的梯度分析，我们揭示了基本 RNN 的致命缺陷——梯度消失导致的短期记忆局限。LSTM 通过引入细胞状态和三个可学习的门，从根本上解决了长期依赖问题；GRU 则以更简洁的双门结构达到了接近的效果。双向和深层扩展进一步增强了模型的表达能力。以下表格汇总了本节的核心概念：

| 关键词 | 含义 |
|--------|------|
| 循环结构 | 隐藏层的自循环连接，使网络在时间步之间传递状态信息 |
| BPTT | 将 RNN 在时间轴上展开后应用反向传播，梯度沿时间逆向流动 |
| 梯度消失 | 梯度经过长距离传播后指数衰减，导致网络无法学习长期依赖 |
| LSTM | 通过细胞状态和三门控机制实现长期记忆，解决梯度消失问题 |
| GRU | 简化版门控 RNN，用两个门达到接近 LSTM 的效果 |
| 双向 RNN | 同时从正向和反向处理序列，每个位置获得完整上下文信息 |
