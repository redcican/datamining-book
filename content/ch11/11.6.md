# 11.6 深度强化学习基础

§11.1–§11.5 介绍的深度学习方法——无论是分类、序列建模还是数据生成——都遵循同一个范式：从**静态数据集**中学习。监督学习需要标注好的输入-输出对，无监督学习需要未标注的数据样本，生成模型需要大量训练样本来拟合数据分布。在这些范式中，学习者是一个被动的观察者——它接收数据、提取模式，但从不主动"行动"，也不关心自己的行动会如何改变未来观测到的数据。

然而，数据挖掘中有一类重要问题本质上是**序列决策**问题：推荐系统需要根据用户的实时反馈动态调整推荐策略，以最大化长期用户满意度而非单次点击率；自动化特征工程需要在庞大的特征变换空间中序贯选择最优的变换组合；网络入侵检测需要在信息不完全的情况下持续做出响应决策。这些问题的共同特征是：(1) 决策是序贯的——当前决策影响未来可用的选择；(2) 反馈是延迟的——一个决策的好坏可能要在很多步之后才能评判；(3) 环境是动态的——学习者的行动会改变环境状态。

**强化学习**（Reinforcement Learning, RL）正是为这类问题设计的学习框架。它的核心思想简洁而强大：一个**智能体**（Agent）通过与**环境**（Environment）的反复交互来学习最优行为策略——每一步观察环境状态、采取行动、获得奖励反馈，目标是最大化长期累积奖励。当我们用深度神经网络来表示强化学习中的策略或值函数时，就得到了**深度强化学习**（Deep RL）——它结合了深度学习强大的函数逼近能力和强化学习的序贯决策框架，在围棋（AlphaGo）、机器人控制和自动驾驶等领域取得了里程碑式的突破。

本节将从强化学习的基本交互框架出发，建立马尔可夫决策过程的数学基础，推导 Bellman 方程和 Q-learning 算法，然后介绍深度 Q 网络（DQN）及其两大核心创新——经验回放和目标网络，最后通过 CartPole 平衡控制案例将理论付诸实践。

## 11.6.1 从监督学习到强化学习

在 §11.1 中，我们建立了深度学习的基本范式：给定训练集 $$\{(\mathbf{x}_i, y_i)\}$$，通过最小化损失函数学习输入到输出的映射。这种范式假设数据是**独立同分布**（i.i.d.）的——每个样本的出现不受模型预测结果的影响。然而，在序列决策场景中，这个假设被彻底打破了：智能体在时刻 $$t$$ 的行动会改变时刻 $$t+1$$ 的环境状态，从而影响后续观测到的"数据"。智能体既是学习者也是数据的"产生者"——这种**交互式学习**本质上不同于被动的数据驱动学习。

强化学习的交互循环如下：在每个时间步 $$t$$，智能体观察当前**状态** $$s_t$$，根据其**策略** $$\pi$$ 选择一个**动作** $$a_t$$，环境响应产生**奖励** $$r_t$$ 并转移到新状态 $$s_{t+1}$$。智能体的目标不是最大化即时奖励，而是最大化从当前时刻到未来所有时刻的**累积奖励**。这种"延迟满足"的思想是强化学习区别于贪心策略的关键——就像下棋时为了最终获胜而牺牲一个棋子，或推荐系统为了长期留存而暂时推荐用户可能不点击的新品类。

{% hint style="warning" %}
⚠️ **监督学习 vs 强化学习：核心区别**

| 维度 | 监督学习 | 强化学习 |
|------|---------|---------|
| 数据 | 静态数据集，i.i.d. | 交互产生，序贯相关 |
| 反馈 | 即时标签（每个输入都有正确答案） | 延迟奖励（好坏可能若干步后才知道） |
| 目标 | 最小化当前预测误差 | 最大化长期累积奖励 |
| 探索 | 不需要（数据已给定） | 必须平衡探索与利用 |
| 因果性 | 预测不影响数据 | 行动影响后续状态和数据 |

强化学习中**探索-利用困境**（Exploration-Exploitation Dilemma）是一个基本挑战：智能体应该利用已知的好策略获取即时回报（利用），还是尝试未知的行动以发现可能更好的策略（探索）？
{% endhint %}

## 11.6.2 马尔可夫决策过程

### MDP 形式化定义

强化学习问题的标准数学框架是**马尔可夫决策过程**（Markov Decision Process, MDP）。MDP 将智能体-环境交互形式化为一个状态转移系统，其核心假设是**马尔可夫性**：未来的状态只取决于当前状态和动作，与历史无关（$$P(s_{t+1} | s_t, a_t, s_{t-1}, a_{t-1}, \ldots) = P(s_{t+1} | s_t, a_t)$$）。这一"无记忆"假设看似严格，但通过适当设计状态表示（如将历史信息编码入状态），大多数实际问题都可以近似满足。

{% block type="definition" %}
**定义 11.28**（马尔可夫决策过程）　马尔可夫决策过程由五元组 $$(\mathcal{S}, \mathcal{A}, P, R, \gamma)$$ 定义：

- **状态空间** $$\mathcal{S}$$：所有可能环境状态的集合；
- **动作空间** $$\mathcal{A}$$：智能体在每个状态可选的动作集合；
- **转移概率** $$P: \mathcal{S} \times \mathcal{A} \times \mathcal{S} \to [0, 1]$$：

$$
P(s' | s, a) = \Pr(s_{t+1} = s' \mid s_t = s, a_t = a) \tag{11.81}
$$

- **奖励函数** $$R: \mathcal{S} \times \mathcal{A} \to \mathbb{R}$$：执行动作后获得的即时奖励；
- **折扣因子** $$\gamma \in [0, 1)$$：衡量未来奖励的重要性。
{% endblock %}

折扣因子 $$\gamma$$ 的作用值得特别说明。智能体在一个**回合**（episode）中的**累积折扣回报**为：

$$
G_t = r_t + \gamma \, r_{t+1} + \gamma^2 r_{t+2} + \cdots = \sum_{k=0}^{\infty} \gamma^k r_{t+k} \tag{11.82}
$$

$$\gamma$$ 接近 0 时，智能体"目光短浅"，几乎只关心即时奖励；$$\gamma$$ 接近 1 时，智能体"深谋远虑"，赋予远期奖励与近期奖励几乎相同的权重。在实践中，$$\gamma$$ 通常取 0.99 或 0.995——足够重视长期回报，同时保证 $$G_t$$ 的有限性（几何级数收敛条件）。

### 策略与值函数

有了 MDP 的框架，我们需要定义智能体的行为准则和评估标准。策略描述了智能体"如何行动"，值函数则评估一个状态或状态-动作对"有多好"。

{% block type="definition" %}
**定义 11.29**（策略与值函数）

- **策略** $$\pi: \mathcal{S} \to \mathcal{P}(\mathcal{A})$$ 将每个状态映射到动作的概率分布。确定性策略直接给出 $$a = \pi(s)$$，随机策略给出 $$\pi(a | s) = \Pr(a_t = a \mid s_t = s)$$。

- **状态值函数** $$V^\pi(s)$$ 评估在状态 $$s$$ 遵循策略 $$\pi$$ 的期望累积回报：

$$
V^\pi(s) = \mathbb{E}_\pi\!\left[\sum_{k=0}^{\infty} \gamma^k r_{t+k} \,\Big|\, s_t = s\right] \tag{11.83}
$$

- **动作值函数** $$Q^\pi(s, a)$$ 评估在状态 $$s$$ 执行动作 $$a$$ 后遵循策略 $$\pi$$ 的期望累积回报：

$$
Q^\pi(s, a) = \mathbb{E}_\pi\!\left[\sum_{k=0}^{\infty} \gamma^k r_{t+k} \,\Big|\, s_t = s, a_t = a\right] \tag{11.84}
$$
{% endblock %}

值函数的核心在于其**递推结构**——当前时刻的值可以分解为即时奖励加上后续状态值的折扣。这种递推关系由 **Bellman 方程**刻画，它是强化学习中几乎所有算法的理论基石：

$$
V^\pi(s) = \sum_{a} \pi(a|s) \left[R(s, a) + \gamma \sum_{s'} P(s'|s,a) \, V^\pi(s')\right] \tag{11.85}
$$

$$
Q^\pi(s, a) = R(s, a) + \gamma \sum_{s'} P(s'|s,a) \sum_{a'} \pi(a'|s') \, Q^\pi(s', a') \tag{11.86}
$$

式 (11.85) 和 (11.86) 分别是 $$V^\pi$$ 和 $$Q^\pi$$ 的 **Bellman 期望方程**。直观地，一个状态的"价值"等于"现在能拿到的奖励"加上"未来能拿到的折扣价值"——这就像评估一笔投资：当前收益加上未来现金流的折现值。

### 最优策略

强化学习的终极目标是找到**最优策略** $$\pi^*$$——在每个状态都做出最优决策的策略。最优值函数满足 **Bellman 最优方程**：

$$
Q^*(s, a) = R(s, a) + \gamma \sum_{s'} P(s'|s,a) \max_{a'} Q^*(s', a') \tag{11.87}
$$

注意与式 (11.86) 的区别：期望方程中的 $$\sum_{a'} \pi(a'|s')Q$$ 被替换为 $$\max_{a'}Q$$——最优策略在每个状态都贪心地选择使 $$Q$$ 值最大的动作。

{% block type="theorem" %}
**定理 11.9**（Bellman 最优方程的不动点定理）　定义 Bellman 最优算子 $$\mathcal{T}^*$$：

$$
(\mathcal{T}^* Q)(s, a) = R(s, a) + \gamma \sum_{s'} P(s'|s,a) \max_{a'} Q(s', a')
$$

则 $$\mathcal{T}^*$$ 是 $$\gamma$$-压缩映射：对任意 $$Q_1, Q_2$$，

$$
\|\mathcal{T}^* Q_1 - \mathcal{T}^* Q_2\|_\infty \le \gamma \|Q_1 - Q_2\|_\infty
$$

由 Banach 不动点定理，$$\mathcal{T}^*$$ 存在唯一不动点 $$Q^*$$，且从任意初始 $$Q_0$$ 出发的迭代 $$Q_{n+1} = \mathcal{T}^* Q_n$$ 收敛到 $$Q^*$$。

**证明**：对任意 $$(s, a)$$，$$|(\mathcal{T}^*Q_1 - \mathcal{T}^*Q_2)(s,a)| = \gamma \big|\sum_{s'} P(s'|s,a)[\max_{a'}Q_1(s',a') - \max_{a'}Q_2(s',a')]\big|$$。由 $$|\max_a f(a) - \max_a g(a)| \le \max_a |f(a) - g(a)|$$，得 $$\le \gamma \sum_{s'} P(s'|s,a) \|Q_1 - Q_2\|_\infty = \gamma \|Q_1 - Q_2\|_\infty$$。因 $$\gamma < 1$$，由 Banach 定理结论成立。$$\blacksquare$$
{% endblock %}

定理 11.9 保证了值迭代算法的收敛性——无论初始估计多么不准确，只要反复应用 Bellman 最优算子，$$Q$$ 值就会收敛到唯一的最优解。然而，这个收敛保证建立在"能遍历所有状态-动作对"的假设上。当状态空间巨大（如围棋的 $$10^{170}$$ 种状态）或连续时，精确的表格式值迭代变得不可行——这正是深度强化学习要解决的问题。

## 11.6.3 深度 Q 网络

### 从 Q-learning 到函数逼近

经典的 **Q-learning** 算法（Watkins, 1989）是解 Bellman 最优方程的无模型方法：智能体不需要知道环境的转移概率 $$P$$，而是通过与环境交互来增量式地更新 $$Q$$ 值表。

{% block type="definition" %}
**定义 11.30**（Q-learning 更新规则）　在观察到转移 $$(s_t, a_t, r_t, s_{t+1})$$ 后，Q-learning 按以下规则更新 $$Q$$ 值：

$$
Q(s_t, a_t) \leftarrow Q(s_t, a_t) + \alpha \big[r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t)\big] \tag{11.88}
$$

其中 $$\alpha \in (0, 1)$$ 为学习率，方括号内的项称为**时序差分误差**（TD error）$$\delta_t = r_t + \gamma \max_{a'} Q(s_{t+1}, a') - Q(s_t, a_t)$$。
{% endblock %}

Q-learning 的优美之处在于它是**离策略**（off-policy）的——智能体可以使用任意的探索策略（如 $$\epsilon$$-贪心）收集数据，但学习到的 $$Q$$ 值始终收敛到最优策略的 $$Q^*$$。然而，经典 Q-learning 将 $$Q$$ 值存储在表格中（状态 $$\times$$ 动作），这在状态空间巨大或连续的场景中完全不可行。例如，Atari 游戏的每一帧是 $$210 \times 160 \times 3$$ 的像素图像，可能的状态数约为 $$256^{210 \times 160 \times 3}$$——根本无法用表格枚举。

2015 年，Mnih 等人在 *Nature* 上发表了 **DQN**（Deep Q-Network）的里程碑论文，用深度卷积神经网络 $$Q(s, a; \theta)$$ 替代 $$Q$$ 值表来逼近 $$Q^*$$。DQN 在 49 款 Atari 游戏中达到了人类水平的表现，标志着深度强化学习的诞生。

{% block type="definition" %}
**定义 11.31**（深度 Q 网络, DQN）　DQN 使用深度神经网络 $$Q(s, a; \theta)$$ 参数化动作值函数，通过最小化以下损失来训练：

$$
\mathcal{L}(\theta) = \mathbb{E}_{(s,a,r,s') \sim \mathcal{D}}\!\left[\big(r + \gamma \max_{a'} Q(s', a'; \theta^{-}) - Q(s, a; \theta)\big)^2\right] \tag{11.89}
$$

其中 $$\mathcal{D}$$ 为**经验回放缓冲区**存储的历史转移样本，$$\theta^{-}$$ 为**目标网络**的参数（定期从 $$\theta$$ 复制或软更新）。
{% endblock %}

式 (11.89) 本质上是将 Q-learning 的表格更新（式 11.88）转化为函数逼近的回归问题——目标值 $$y = r + \gamma \max_{a'} Q(s', a'; \theta^{-})$$ 充当"伪标签"，损失函数衡量当前 $$Q$$ 预测与目标值之间的均方误差。但这里有两个关键的技术挑战需要解决。

### 经验回放

第一个挑战是**数据相关性**。智能体与环境的交互产生的是一系列高度相关的状态转移 $$(s_t, a_t, r_t, s_{t+1})$$——连续的帧之间几乎相同，连续的状态-动作对在时间上强相关。如果直接用最新的交互数据做梯度更新（在线学习），网络会不断遗忘旧的经验，训练极不稳定。

{% block type="definition" %}
**定义 11.32**（经验回放与目标网络）

- **经验回放**（Experience Replay）：将智能体的交互转移 $$(s_t, a_t, r_t, s_{t+1})$$ 存入固定容量的缓冲区 $$\mathcal{D}$$（先进先出），训练时从 $$\mathcal{D}$$ 中**随机采样**小批量数据。随机采样打破了时间相关性，使得训练样本近似满足 i.i.d. 假设。

- **目标网络**（Target Network）：使用一个参数更新较慢的网络 $$Q(s, a; \theta^{-})$$ 计算目标值。$$\theta^{-}$$ 的更新方式为软更新（Polyak 平均）：

$$
\theta^{-} \leftarrow \tau \theta + (1 - \tau) \theta^{-}, \quad \tau \ll 1 \tag{11.90}
$$

或每隔 $$C$$ 步硬更新 $$\theta^{-} \leftarrow \theta$$。目标网络通过减缓目标值的变化来稳定训练。
{% endblock %}

经验回放与目标网络是 DQN 成功的两大支柱。没有经验回放，连续帧的强相关性会导致网络在局部经验中过拟合然后灾难性遗忘；没有目标网络，目标值 $$y = r + \gamma \max Q(s'; \theta)$$ 随 $$\theta$$ 的每次更新而剧烈变化，造成训练目标不断"移动"，优化过程在正反馈循环中发散。两者结合后，DQN 的训练过程近似于监督学习——从固定分布的缓冲区采样，对缓慢变化的目标做回归——这正是深度学习所擅长的。

{% hint style="tip" %}
💡 **经验回放的多重收益**

经验回放不仅解决了数据相关性问题，还带来了额外的好处：

1. **数据效率**：每个交互经验可以被多次采样学习，而非用过即弃；
2. **避免灾难性遗忘**：旧的经验持续参与训练，防止网络遗忘已学到的知识；
3. **稳定梯度**：随机采样的小批量数据覆盖不同状态和时间段，梯度方向更稳定。

后续工作（如优先经验回放）进一步改进了采样策略——让 TD 误差更大的经验被更频繁地回放，加速了学习过程。
{% endhint %}

## 11.6.4 DQN 改进与变体

### Double DQN

原始 DQN 存在一个系统性的问题——**过高估计**（Overestimation）：$$\max_{a'} Q(s', a'; \theta^{-})$$ 操作在选择最大 $$Q$$ 值的同时也放大了估计噪声，导致 $$Q$$ 值被系统性地高估。这种偏差会随着训练传播和累积，导致次优策略。

van Hasselt 等人（2016）提出了 **Double DQN**，通过**解耦**动作选择和值评估来消除过高估计：用在线网络 $$\theta$$ 选择最优动作，但用目标网络 $$\theta^{-}$$ 评估该动作的值：

$$
y^{\text{DDQN}} = r + \gamma \, Q\!\left(s', \arg\max_{a'} Q(s', a'; \theta);\, \theta^{-}\right) \tag{11.91}
$$

与原始 DQN 的目标 $$y = r + \gamma \max_{a'} Q(s', a'; \theta^{-})$$ 相比，Double DQN 的改动极其简单——仅一行代码的变化——但效果显著：由于 $$\theta$$ 和 $$\theta^{-}$$ 是不同时刻的参数，一个网络的估计偏差不太可能与另一个网络的偏差完全对齐，从而抵消了系统性过高估计。

### Dueling DQN

另一个重要改进是 **Dueling DQN**（Wang 等, 2016），它从**网络架构**层面改进了值函数的学习效率。核心观察是：在许多状态下，不同动作的价值差异很小——真正重要的是"处于这个状态本身好不好"（状态值），而非"在这个状态做哪个动作好"（动作优势）。Dueling 架构将 $$Q$$ 函数分解为两部分：

$$
Q(s, a; \theta) = V(s; \theta_V) + A(s, a; \theta_A) - \frac{1}{|\mathcal{A}|}\sum_{a'} A(s, a'; \theta_A) \tag{11.92}
$$

其中 $$V(s)$$ 为**状态值流**（Value stream），$$A(s, a)$$ 为**优势流**（Advantage stream），减去优势均值保证了可辨识性（$$V$$ 和 $$A$$ 的分解是唯一的）。在网络结构上，特征提取层共享，之后分叉为两个独立的全连接流，最终通过式 (11.92) 合并输出 $$Q$$ 值。

这种分解的好处在于：对于那些"做什么动作都差不多"的状态，网络可以高效地学习状态值 $$V$$，而不必为每个动作分别估计 $$Q$$——特别是在动作空间较大时，Dueling 架构的学习效率显著优于标准 DQN。

{% hint style="info" %}
📋 **DQN 家族的演进脉络**

DQN 的核心算法框架（经验回放 + 目标网络 + 函数逼近）自 2015 年确立后，后续改进主要集中在三个方向：

1. **估计精度**：Double DQN（消除过高估计）；
2. **网络架构**：Dueling DQN（V+A 分解，高效学习）；
3. **采样策略**：优先经验回放（按 TD 误差加权采样，加速学习）。

Rainbow（Hessel 等, 2018）将这些改进统一整合，在 Atari 基准上取得了大幅超越单一改进的性能，验证了各技术的互补性。
{% endhint %}

## 11.6.5 方法对比

回顾本节介绍的强化学习方法家族，下表总结了各核心方法的特点与适用场景：

| 方法 | 值函数表示 | 核心机制 | 过高估计 | 学习效率 | 典型应用 |
|------|---------|---------|---------|---------|---------|
| Q-learning | 表格 | TD 更新 | 有 | 低（需遍历） | 小规模离散问题 |
| DQN | 神经网络 | 经验回放 + 目标网络 | 有 | 中等 | Atari 游戏、离散控制 |
| Double DQN | 神经网络 | 解耦选择与评估 | 显著减少 | 中等 | 需精确估计的任务 |
| Dueling DQN | 神经网络（V+A 分解） | 架构改进 | 有 | 高 | 大动作空间 |
| Rainbow | 神经网络 | 多技术整合 | 最少 | 最高 | 通用 Atari 基准 |

从表中可以观察到：DQN 家族的演进遵循"保持核心框架不变、逐步改进各个环节"的渐进式创新路径。每一个改进都针对原始 DQN 的一个具体缺陷（过高估计、采样效率、架构表达力），且相互正交——这使得它们可以自由组合以获得叠加收益。

{% hint style="warning" %}
⚠️ **DQN 的局限性**

DQN 及其变体仅适用于**离散动作空间**——网络输出每个离散动作的 $$Q$$ 值，通过 $$\arg\max$$ 选择最优动作。对于**连续动作空间**（如机器人关节角度、推荐系统的出价金额），需要使用基于策略梯度的方法（如 DDPG、PPO、SAC），这些方法超出了本节"基础"的范围。
{% endhint %}

## 案例 11.6：CartPole 平衡控制

强化学习在数据挖掘中的应用包括推荐策略优化、自动化超参数搜索和动态资源分配等。这些实际应用的规模较大、环境复杂，不适合作为教学入门案例。本案例使用经典的 **CartPole**（倒立摆）环境来展示 DQN 的完整实现流程——尽管问题简单，但它完整地体现了深度强化学习的所有核心环节：状态观测、动作选择、经验收集、网络训练和策略改进。

{% block type="case" %}
**案例 11.6**（CartPole 平衡控制）

**任务**：在 CartPole-v1 环境中，通过左右推动小车来保持杆子直立，坚持 500 步即为成功。

**环境**：状态 $$s \in \mathbb{R}^4$$（小车位置、速度、杆角度、角速度），动作 $$a \in \{0, 1\}$$（向左/右推），每步存活奖励 +1。

**方法**：DQN with 经验回放（容量 10000）、目标网络（每 10 回合同步）、$$\epsilon$$-贪心探索（$$\epsilon$$ 从 1.0 衰减至 0.01）。
{% endblock %}

{% block type="algorithm" %}
**算法 11.6**（DQN 训练算法）

**输入**：环境 $$\mathcal{E}$$，在线网络 $$Q_\theta$$，目标网络 $$Q_{\theta^-}$$，缓冲区容量 $$N$$，批量大小 $$B$$，同步间隔 $$C$$
**输出**：训练好的策略 $$\pi(s) = \arg\max_a Q(s, a; \theta)$$

1. 初始化回放缓冲区 $$\mathcal{D} \leftarrow \emptyset$$，令 $$\theta^- \leftarrow \theta$$
2. **for** episode $$= 1, 2, \ldots$$ **do**
3. 　　获取初始状态 $$s_0 \leftarrow \mathcal{E}.\text{reset}()$$
4. 　　**for** $$t = 0, 1, \ldots$$ **do**
5. 　　　　以概率 $$\epsilon$$ 随机选择 $$a_t$$，否则 $$a_t \leftarrow \arg\max_a Q(s_t, a; \theta)$$
6. 　　　　执行 $$a_t$$，观察 $$r_t, s_{t+1}, \text{done}$$
7. 　　　　存入缓冲区 $$\mathcal{D} \leftarrow \mathcal{D} \cup \{(s_t, a_t, r_t, s_{t+1}, \text{done})\}$$
8. 　　　　**if** $$|\mathcal{D}| \ge B$$ **then**
9. 　　　　　　从 $$\mathcal{D}$$ 随机采样小批量 $$\{(s_j, a_j, r_j, s'_j, d_j)\}_{j=1}^B$$
10. 　　　　　计算目标 $$y_j = r_j + \gamma (1 - d_j) \max_{a'} Q(s'_j, a'; \theta^-)$$
11. 　　　　　更新 $$\theta \leftarrow \theta - \alpha \nabla_\theta \frac{1}{B}\sum_j (y_j - Q(s_j, a_j; \theta))^2$$
12. 　　　　**end if**
13. 　　　　**if** done **then break**
14. 　　**end for**
15. 　　每 $$C$$ 个 episode 同步 $$\theta^- \leftarrow \theta$$
16. 　　衰减 $$\epsilon$$
17. **end for**
{% endblock %}

```python
import torch
import torch.nn as nn
import numpy as np
from collections import deque
import random

# --- 1. Q 网络定义 ---
class QNet(nn.Module):
    def __init__(self, state_dim, action_dim):
        super().__init__()
        self.fc = nn.Sequential(
            nn.Linear(state_dim, 128), nn.ReLU(),
            nn.Linear(128, 128), nn.ReLU(),
            nn.Linear(128, action_dim)
        )
    def forward(self, x):
        return self.fc(x)

# --- 2. 经验回放缓冲区 ---
replay_buf = deque(maxlen=10000)

def sample_batch(buf, batch_size):
    batch = random.sample(buf, batch_size)
    s, a, r, s2, d = zip(*batch)
    return (torch.FloatTensor(np.array(s)),
            torch.LongTensor(a),
            torch.FloatTensor(r),
            torch.FloatTensor(np.array(s2)),
            torch.FloatTensor(d))

# --- 3. 初始化网络与优化器 ---
q_net = QNet(4, 2)          # CartPole: 4维状态, 2个动作
target_net = QNet(4, 2)
target_net.load_state_dict(q_net.state_dict())
optimizer = torch.optim.Adam(q_net.parameters(), lr=1e-3)
gamma, eps, eps_min, eps_decay = 0.99, 1.0, 0.01, 0.995
batch_size = 64

# --- 4. 训练循环 ---
import gymnasium as gym
env = gym.make("CartPole-v1")

for episode in range(300):
    state, _ = env.reset()
    total_reward = 0
    while True:
        # epsilon-贪心探索
        if random.random() < eps:
            action = env.action_space.sample()
        else:
            with torch.no_grad():
                action = q_net(torch.FloatTensor(state)).argmax().item()
        next_state, reward, done, trunc, _ = env.step(action)
        replay_buf.append((state, action, reward, next_state,
                           float(done or trunc)))
        state = next_state
        total_reward += reward
        # 经验回放训练
        if len(replay_buf) >= batch_size:
            s, a, r, s2, d = sample_batch(replay_buf, batch_size)
            q_val = q_net(s).gather(1, a.unsqueeze(1)).squeeze()
            with torch.no_grad():
                target = r + gamma * (1 - d) * target_net(s2).max(1)[0]
            loss = nn.MSELoss()(q_val, target)
            optimizer.zero_grad(); loss.backward(); optimizer.step()
        if done or trunc:
            break
    # 同步目标网络 + 衰减 epsilon
    if episode % 10 == 0:
        target_net.load_state_dict(q_net.state_dict())
    eps = max(eps_min, eps * eps_decay)
    if episode % 20 == 0:
        print(f"Episode {episode}: reward={total_reward:.0f}, eps={eps:.3f}")
# 典型结果: 约 100 episode 后稳定达到 500 步满分
```

**结果分析**：DQN 在 CartPole 环境上展现了典型的强化学习学习曲线。训练初期（前 30 回合），$$\epsilon$$ 较高，智能体以近乎随机的策略探索环境，平均每回合存活约 20–30 步。随着经验积累和 $$\epsilon$$ 衰减，智能体逐渐学会在关键时刻做出正确的推动方向选择。到约 100 回合时，智能体稳定达到 500 步满分（CartPole-v1 的上限）。值得注意的是，经验回放缓冲区在初期的快速填充阶段对训练效率至关重要——如果缓冲区太小或不使用回放，训练可能永远无法稳定收敛。图 11.6.3 展示了训练过程中的奖励曲线和各 DQN 变体的对比。

## 本节小结

本节从监督学习与强化学习的本质区别出发，建立了马尔可夫决策过程的数学框架，推导了值函数和 Bellman 方程，然后介绍了深度 Q 网络及其两大核心创新——经验回放和目标网络。DQN 的成功证明了深度学习的函数逼近能力与强化学习的序贯决策框架可以有效结合，为后续更复杂的深度强化学习方法（策略梯度、Actor-Critic）奠定了基础。以下表格汇总了本节的核心概念：

| 关键词 | 含义 |
|--------|------|
| 马尔可夫决策过程 | 由状态、动作、转移、奖励和折扣组成的序贯决策数学框架 |
| 状态/动作值函数 | 评估状态或状态-动作对在给定策略下的期望累积回报 |
| Bellman 方程 | 值函数的递推关系，是几乎所有 RL 算法的理论基石 |
| Q-learning | 无模型的离策略表格值迭代方法 |
| DQN | 用深度网络逼近 Q 函数，结合经验回放和目标网络 |
| 经验回放 | 存储并随机采样历史经验，打破时间相关性 |
| 目标网络 | 参数延迟更新的辅助网络，稳定训练目标 |
