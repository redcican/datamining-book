# 11.5 生成对抗网络

§11.4 介绍的变分自编码器（VAE）通过概率化的潜在空间实现了数据生成的能力——从标准正态先验中采样潜在变量，解码器即可生成新的数据样本。然而，VAE 生成的图像往往带有明显的"模糊感"：人脸的轮廓不够锐利，文字的边缘不够清晰。这是因为 VAE 的重建损失（像素级 MSE）本质上在优化"所有可能输出的平均"——当多个合理的输出都存在时，MSE 倾向于生成它们的均值，而均值恰好是一张模糊的图像。

能否不依赖像素级的重建比较，而让模型直接学习生成"以假乱真"的逼真样本？2014 年，Ian Goodfellow 等人提出了**生成对抗网络**（Generative Adversarial Network, GAN），用一个革命性的思路回答了这个问题：让两个神经网络**互相对抗**——一个**生成器**（Generator）努力制造逼真的"假样本"，一个**判别器**（Discriminator）努力区分真实样本和假样本。两者在对抗博弈中共同进化：生成器越来越擅长欺骗判别器，判别器越来越善于识别赝品，直到生成器产生的样本与真实数据无法区分——这时我们就获得了一个强大的数据生成模型。

这种对抗思想可以用一个生活中的类比来理解：生成器就像一个不断提高技艺的**造假画家**，判别器则是一位越来越敏锐的**鉴定专家**。画家从随机涂鸦开始，根据鉴定专家的反馈不断改进伪画质量；而鉴定专家则不断研究真画和伪画的细微差异。当画家的伪画达到连最好的鉴定专家也无法分辨的水平时，我们就说画家"学会了"真画的分布。

本节将从 GAN 的极小极大博弈框架出发，推导最优判别器和全局最优性的理论保证，分析模式崩溃等训练挑战，然后依次介绍 DCGAN、WGAN 和条件 GAN 三种核心变体，最后通过 MNIST 数字生成案例展示 GAN 在数据挖掘中的实际应用。

## 11.5.1 从重建到对抗

在 §11.4 中，我们看到 VAE 通过最大化证据下界（ELBO）来训练生成模型，其重建项 $$\|\mathbf{x} - \hat{\mathbf{x}}\|^2$$ 提供了一个明确的优化信号。这种逐像素的比较方式简单直观，但存在根本性的局限：它独立地衡量每个像素的差异，却忽略了像素之间的空间关系和整体语义。例如，一张向右平移一个像素的完美人脸照片在 MSE 意义下可能有很大的误差，但在视觉上几乎完美；而一张像素值恰好接近平均值的模糊图像在 MSE 意义下可能误差很小，但视觉质量极差。

GAN 采用了一种截然不同的策略：**不直接比较像素，而是训练一个神经网络来判断生成样本是否"像"真实数据**。这个判别网络可以学习到任意复杂的统计特征——纹理细节、结构一致性、风格连贯性——从而提供远比像素级 MSE 丰富的监督信号。正是这种"以网络评价网络"的对抗机制，使 GAN 能够生成视觉质量远超 VAE 的锐利、逼真的样本，也使其在数据挖掘中的数据增强、异常样本生成、缺失数据填补等场景中表现出独特的优势。

## 11.5.2 对抗训练框架

### 极小极大博弈

GAN 的核心是一个**二人零和博弈**：生成器 $$G$$ 和判别器 $$D$$ 各自优化一个相互对立的目标。生成器从随机噪声 $$\mathbf{z}$$ 出发生成假样本，目标是让判别器犯错；判别器接收真实样本和假样本，目标是正确区分两者。两者在博弈中迭代改进，直至达到纳什均衡。

{% block type="definition" %}
**定义 11.23**（生成对抗网络）　生成对抗网络由**生成器** $$G: \mathbb{R}^{n_z} \to \mathbb{R}^d$$ 和**判别器** $$D: \mathbb{R}^d \to [0, 1]$$ 组成，通过以下**极小极大博弈**联合训练：

$$
\min_G \max_D \; V(D, G) = \mathbb{E}_{\mathbf{x} \sim p_{\text{data}}}[\log D(\mathbf{x})] + \mathbb{E}_{\mathbf{z} \sim p_z}[\log(1 - D(G(\mathbf{z})))] \tag{11.70}
$$

其中 $$p_{\text{data}}$$ 为真实数据分布，$$p_z$$ 为先验噪声分布（通常取 $$\mathcal{N}(\mathbf{0}, I)$$ 或 $$\text{Uniform}[-1,1]^{n_z}$$），$$D(\mathbf{x})$$ 表示判别器认为 $$\mathbf{x}$$ 是真实样本的概率。
{% endblock %}

式 (11.70) 的极小极大目标直观地表达了对抗机制。从**判别器的角度**（$$\max_D$$）：第一项 $$\mathbb{E}[\log D(\mathbf{x})]$$ 希望对真实样本输出高概率（$$D(\mathbf{x}) \to 1$$），第二项 $$\mathbb{E}[\log(1 - D(G(\mathbf{z})))]$$ 希望对生成样本输出低概率（$$D(G(\mathbf{z})) \to 0$$）——判别器在做标准的二分类。从**生成器的角度**（$$\min_G$$）：只有第二项与 $$G$$ 有关，生成器希望最小化 $$\log(1 - D(G(\mathbf{z})))$$，即让 $$D(G(\mathbf{z})) \to 1$$——让判别器误认为假样本是真的。

### 最优判别器

对于固定的生成器 $$G$$，判别器的最优策略可以解析求解。这个结果不仅指导了理论分析，也建立了 GAN 目标函数与信息论中经典散度度量之间的深刻联系。

{% block type="theorem" %}
**定理 11.7**（最优判别器）　对于任意固定的生成器 $$G$$，最大化 $$V(D, G)$$ 的最优判别器为：

$$
D^*(\mathbf{x}) = \frac{p_{\text{data}}(\mathbf{x})}{p_{\text{data}}(\mathbf{x}) + p_g(\mathbf{x})} \tag{11.71}
$$

其中 $$p_g$$ 为生成器隐含定义的数据分布（即 $$\mathbf{x} = G(\mathbf{z})$$ 的分布）。

**证明**：对于任意 $$\mathbf{x}$$，$$V(D, G)$$ 关于 $$D$$ 的被积函数为 $$f(d) = p_{\text{data}}(\mathbf{x}) \log d + p_g(\mathbf{x}) \log(1 - d)$$。令 $$f'(d) = \frac{p_{\text{data}}(\mathbf{x})}{d} - \frac{p_g(\mathbf{x})}{1 - d} = 0$$，解得 $$d^* = \frac{p_{\text{data}}(\mathbf{x})}{p_{\text{data}}(\mathbf{x}) + p_g(\mathbf{x})}$$。由 $$f''(d^*) < 0$$ 知此为最大值点。$$\blacksquare$$
{% endblock %}

定理 11.7 揭示了一个优美的直觉：最优判别器的输出正是**贝叶斯后验概率**——在真假样本等先验的假设下，$$D^*(\mathbf{x})$$ 恰好等于"数据来自真实分布"的后验概率。当 $$p_g = p_{\text{data}}$$ 时，$$D^*(\mathbf{x}) = \frac{1}{2}$$，判别器完全无法区分真假——这正是 GAN 训练的理想终态。

### 全局最优性

将最优判别器 $$D^*$$ 代回博弈目标函数，我们可以证明 GAN 在全局最优时恰好等价于最小化两个分布之间的 **Jensen-Shannon 散度**（JSD）：

{% block type="theorem" %}
**定理 11.8**（GAN 全局最优性）　当判别器取最优 $$D^*$$ 时，生成器的目标函数变为：

$$
V(G, D^*) = 2 \, D_{\text{JS}}(p_{\text{data}} \| p_g) - 2\log 2 \tag{11.72}
$$

其中 Jensen-Shannon 散度定义为：

$$
D_{\text{JS}}(p \| q) = \frac{1}{2} D_{\text{KL}}\!\left(p \,\Big\|\, \frac{p+q}{2}\right) + \frac{1}{2} D_{\text{KL}}\!\left(q \,\Big\|\, \frac{p+q}{2}\right) \tag{11.73}
$$

由于 $$D_{\text{JS}} \ge 0$$，等号当且仅当 $$p_g = p_{\text{data}}$$ 时成立，全局最小值为 $$V^* = -2\log 2$$。

**证明思路**：将 $$D^*$$ 代入 $$V(D^*, G)$$，得 $$\int p_{\text{data}} \log \frac{p_{\text{data}}}{p_{\text{data}}+p_g} + p_g \log \frac{p_g}{p_{\text{data}}+p_g} \, d\mathbf{x}$$。加减 $$2\log 2$$，利用 KL 散度定义改写为式 (11.72)。$$\blacksquare$$
{% endblock %}

定理 11.8 的意义深远。它说明 GAN 的训练过程本质上是在最小化真实数据分布与生成分布之间的 JSD——一种对称、有界的分布距离度量。与 VAE 最小化 KL 散度 $$D_{\text{KL}}(q \| p)$$（前向 KL，倾向于"覆盖"所有模式但产生模糊）不同，JSD 对模式遗漏和模式扩展给予对称的惩罚，这在理论上有利于生成**锐利且多样**的样本。然而，正如我们将在 §11.5.3 中看到的，理论上的优美并不意味着训练上的便利——JSD 在两个分布支撑集不重叠时会饱和（常数 $$\log 2$$），导致梯度消失，这是 GAN 训练不稳定的根源之一。

## 11.5.3 训练算法与稳定性

### 交替优化算法

理论分析表明 GAN 存在全局最优解（$$p_g = p_{\text{data}}$$），但如何在实践中逼近这个解？Goodfellow 等人提出了**交替优化**策略：每一步先更新判别器若干次（使其逼近当前生成器下的最优 $$D^*$$），再更新生成器一次（在近似最优判别器的指导下改进生成质量）。

{% block type="algorithm" %}
**算法 11.5**（GAN 交替训练算法）

**输入**：训练数据 $$\{x_1, \ldots, x_n\}$$，噪声维度 $$n_z$$，判别器更新步数 $$n_d$$，学习率 $$\alpha$$
**输出**：训练好的生成器 $$G$$

1. 初始化生成器参数 $$\theta_G$$ 和判别器参数 $$\theta_D$$
2. **for** epoch $$= 1, 2, \ldots$$ **do**
3. 　　**for** $$k = 1, \ldots, n_d$$ **do**　　$$\triangleright$$ 更新判别器
4. 　　　　采样小批量真实数据 $$\{\mathbf{x}^{(i)}\}_{i=1}^m \sim p_{\text{data}}$$
5. 　　　　采样噪声 $$\{\mathbf{z}^{(i)}\}_{i=1}^m \sim p_z$$
6. 　　　　$$\theta_D \leftarrow \theta_D + \alpha \cdot \nabla_{\theta_D} \frac{1}{m}\sum_{i=1}^m \big[\log D(\mathbf{x}^{(i)}) + \log(1 - D(G(\mathbf{z}^{(i)})))\big]$$
7. 　　**end for**
8. 　　采样噪声 $$\{\mathbf{z}^{(i)}\}_{i=1}^m \sim p_z$$　　$$\triangleright$$ 更新生成器
9. 　　$$\theta_G \leftarrow \theta_G - \alpha \cdot \nabla_{\theta_G} \frac{1}{m}\sum_{i=1}^m \log(1 - D(G(\mathbf{z}^{(i)})))$$
10. **end for**
{% endblock %}

算法 11.5 中有一个关键的实践问题：在训练初期，生成器产生的样本质量极差，判别器可以轻松地将其与真实数据区分开。此时 $$D(G(\mathbf{z})) \approx 0$$，使得 $$\log(1 - D(G(\mathbf{z}))) \approx 0$$——生成器几乎得不到梯度信号，学习陷入停滞。Goodfellow 提出了一个实用的替代方案——**非饱和生成器损失**：

$$
\mathcal{L}_G^{\text{NS}} = -\mathbb{E}_{\mathbf{z} \sim p_z}[\log D(G(\mathbf{z}))] \tag{11.74}
$$

与原始的 $$\min_G \mathbb{E}[\log(1 - D(G(\mathbf{z})))]$$ 相比，式 (11.74) 在 $$D(G(\mathbf{z})) \approx 0$$ 时提供了更大的梯度——从"最小化被判为假的概率"转变为"最大化被判为真的概率"。虽然两者在全局最优时等价，但在训练初期的梯度行为截然不同，非饱和版本已成为实践中的标准选择。

### 模式崩溃

尽管 GAN 的理论保证令人信服，但实际训练中最常遇到的问题是**模式崩溃**（Mode Collapse）：生成器"偷懒"，只学会生成少数几种样本来反复欺骗判别器，而忽略了真实数据分布中的大量多样模式。

{% block type="definition" %}
**定义 11.24**（模式崩溃）　设真实数据分布 $$p_{\text{data}}$$ 包含 $$K$$ 个模式（如 10 类手写数字），若生成器的输出分布 $$p_g$$ 仅覆盖其中 $$k \ll K$$ 个模式，即存在 $$K - k$$ 个模式 $$\mathcal{M}_j$$ 满足 $$p_g(\mathbf{x}) \approx 0, \forall \mathbf{x} \in \mathcal{M}_j$$，则称生成器发生了模式崩溃。极端情况下 $$k = 1$$，生成器始终输出同一张（或几乎相同的）图像。
{% endblock %}

模式崩溃的根源在于极小极大博弈的动态特性。在每一步更新中，生成器只需要找到当前判别器的"弱点"——即判别器最难区分的生成模式——然后集中力量生成这种模式。判别器随后学会识别这种模式，生成器就切换到另一种模式，形成"猫鼠游戏"式的震荡，而非真正地覆盖所有模式。

{% hint style="tip" %}
💡 **GAN 训练的实用技巧**

在实践中，以下技巧有助于稳定 GAN 训练并缓解模式崩溃：

1. **标签平滑**：将真实样本的标签从 1.0 降低到 0.9，避免判别器过度自信；
2. **特征匹配**：让生成器匹配判别器中间层特征的统计量，而非直接最大化判别器输出；
3. **谱归一化**：对判别器权重矩阵进行谱归一化，控制 Lipschitz 常数；
4. **平衡训练**：保持判别器和生成器的能力大致匹配，避免一方过强；
5. **小批量多样性**：在判别器中加入小批量统计特征，鼓励生成多样的样本。

这些技巧并非银弹，不同任务和数据集往往需要不同的组合。§11.5.5 中介绍的 WGAN 从理论层面提供了更根本的解决方案。
{% endhint %}

## 11.5.4 深度卷积 GAN

### 从全连接到卷积架构

原始 GAN 论文使用全连接网络作为生成器和判别器，但在图像生成任务上效果有限——全连接层无法有效利用图像的空间结构。2015 年，Radford 等人提出了 **DCGAN**（Deep Convolutional GAN），将 CNN 的归纳偏置引入 GAN 框架，大幅提升了图像生成质量，并建立了一系列至今仍被广泛遵循的架构设计准则。

{% block type="definition" %}
**定义 11.25**（深度卷积生成对抗网络, DCGAN）　DCGAN 用**转置卷积**（Transposed Convolution）构建生成器，用**步幅卷积**（Strided Convolution）构建判别器，遵循以下架构准则：

1. 用步幅卷积（判别器）和转置卷积（生成器）替代所有池化层；
2. 在生成器和判别器中均使用**批归一化**（Batch Normalization），但生成器的输出层和判别器的输入层除外；
3. 去除所有全连接隐藏层，仅在判别器最终输出和生成器初始投影中使用；
4. 生成器使用 ReLU 激活（输出层使用 Tanh），判别器使用 LeakyReLU 激活。

生成器通过转置卷积逐步上采样：

$$
\mathbf{z} \in \mathbb{R}^{n_z} \xrightarrow{\text{reshape}} 4{\times}4{\times}512 \xrightarrow{\text{ConvT}} 8{\times}8{\times}256 \xrightarrow{\text{ConvT}} 16{\times}16{\times}128 \xrightarrow{\text{ConvT}} 32{\times}32{\times}C \tag{11.75}
$$

其中每一步 ConvT 包含转置卷积 → 批归一化 → ReLU（最后一步为 Tanh）。
{% endblock %}

DCGAN 的设计理念在于充分利用 CNN 的**局部感受野和平移等变性**：生成器通过转置卷积从粗糙的全局结构（$$4 \times 4$$）逐步细化到高分辨率的局部纹理（$$32 \times 32$$），每一层只需关注局部区域的细节；判别器则通过步幅卷积从局部纹理逐步提取到全局结构，判断整体的真实性。批归一化在每一层稳定了特征分布，使深层网络的训练成为可能。这套架构准则极其有效——DCGAN 成为了后续几乎所有图像 GAN 变体的基础架构。

## 11.5.5 Wasserstein GAN

### 分布距离的重新审视

§11.5.2 的理论分析表明，原始 GAN 本质上在最小化 JSD。然而 JSD 有一个严重的缺陷：当两个分布的支撑集（support）不重叠时，JSD 退化为常数 $$\log 2$$，梯度完全消失。在高维空间中（如图像的像素空间），真实数据分布和生成分布通常位于低维流形上——两个低维流形在高维空间中几乎必然不相交，导致 JSD 几乎处处饱和。这从根本上解释了为什么原始 GAN 的训练如此不稳定：判别器太容易完美分类，生成器就失去了改进方向。

2017 年，Arjovsky 等人提出了一种更适合衡量分布距离的度量——**Wasserstein 距离**（又称推土距离, Earth Mover's Distance），它即使在两个分布不重叠时也能提供有意义的梯度信号。

{% block type="definition" %}
**定义 11.26**（Wasserstein 距离与 WGAN）　两个分布 $$p$$ 和 $$q$$ 之间的 **Wasserstein-1 距离**为：

$$
W_1(p, q) = \inf_{\gamma \in \Pi(p, q)} \mathbb{E}_{(\mathbf{x}, \mathbf{y}) \sim \gamma}\big[\|\mathbf{x} - \mathbf{y}\|\big] \tag{11.76}
$$

其中 $$\Pi(p, q)$$ 为所有边际分布为 $$p$$ 和 $$q$$ 的联合分布（传输计划）的集合。直观地，$$W_1$$ 衡量将分布 $$p$$ "搬运"为 $$q$$ 所需的最小总工作量。

由 Kantorovich-Rubinstein 对偶定理，$$W_1$$ 可改写为：

$$
W_1(p, q) = \sup_{\|f\|_L \le 1} \left[\mathbb{E}_{\mathbf{x} \sim p}[f(\mathbf{x})] - \mathbb{E}_{\mathbf{x} \sim q}[f(\mathbf{x})]\right] \tag{11.77}
$$

其中上确界取遍所有 1-Lipschitz 函数 $$f$$。WGAN 将判别器替换为**评论器**（Critic）$$f_w$$（不再输出概率，而是一个实数分数），通过最大化式 (11.77) 估计 $$W_1$$ 距离。
{% endblock %}

Wasserstein 距离相较于 JSD 有一个关键优势：它对分布的"几何结构"敏感。即使两个分布完全不重叠，$$W_1$$ 仍然能衡量它们之间的实际距离——就像两座相隔一米和相隔一公里的山丘，虽然都不重叠，但推土距离能区分这两种情况，而 JSD 对两者给出相同的常数值。这意味着生成器在任何训练阶段都能获得有意义的梯度方向，从而稳定训练过程。

### 梯度惩罚

式 (11.77) 中的 1-Lipschitz 约束是 WGAN 的关键。原始 WGAN 通过简单地裁剪评论器的权重（$$w \leftarrow \text{clip}(w, -c, c)$$）来近似满足 Lipschitz 约束，但这种粗暴的方式导致评论器的容量受限且训练不稳定。Gulrajani 等人在 2017 年提出了更优雅的**梯度惩罚**（Gradient Penalty, GP）方法：

$$
\mathcal{L}_{\text{GP}} = \lambda \, \mathbb{E}_{\hat{\mathbf{x}} \sim p_{\hat{x}}}\!\left[\big(\|\nabla_{\hat{\mathbf{x}}} f_w(\hat{\mathbf{x}})\|_2 - 1\big)^2\right] \tag{11.78}
$$

其中 $$\hat{\mathbf{x}} = \epsilon \mathbf{x} + (1 - \epsilon) G(\mathbf{z})$$（$$\epsilon \sim \text{Uniform}[0,1]$$）为真实样本与生成样本之间的随机插值点，$$\lambda$$ 为惩罚系数（通常取 10）。梯度惩罚直接在训练过程中软约束评论器的梯度范数接近 1，确保 Lipschitz 条件被近似满足，同时不限制网络的表达能力。

{% hint style="info" %}
📋 **WGAN 的核心改进总结**

与原始 GAN 相比，WGAN 有三个关键变化：

1. **损失函数**：从 JSD（通过二分类交叉熵）改为 Wasserstein 距离估计；
2. **判别器 → 评论器**：输出从概率值 $$[0,1]$$ 变为无约束的实数分数（去掉 Sigmoid）；
3. **Lipschitz 约束**：通过梯度惩罚（WGAN-GP）确保评论器满足 1-Lipschitz 条件。

这些改进使得：(a) 训练损失与生成质量强相关，可用于监控训练进度；(b) 模式崩溃显著减少；(c) 超参数选择更加鲁棒。
{% endhint %}

## 11.5.6 条件生成对抗网络

### 从无条件到条件生成

原始 GAN 和 DCGAN 都是**无条件生成**模型——给定噪声 $$\mathbf{z}$$，生成器随机生成一个样本，我们无法控制生成结果的具体属性。在数据挖掘的许多应用中，我们需要**可控的生成**：生成特定类别的样本（如"生成数字 7"）、按条件生成数据增强样本、或在给定上下文下补全缺失数据。**条件 GAN**（Conditional GAN, cGAN）通过将**条件信息**引入生成器和判别器来实现这一目标。

{% block type="definition" %}
**定义 11.27**（条件生成对抗网络）　条件 GAN 在生成器和判别器中同时引入条件变量 $$\mathbf{y}$$（如类别标签、属性向量或其他模态的数据），极小极大目标变为：

$$
\min_G \max_D \; V(D, G) = \mathbb{E}_{\mathbf{x} \sim p_{\text{data}}}[\log D(\mathbf{x} | \mathbf{y})] + \mathbb{E}_{\mathbf{z} \sim p_z}[\log(1 - D(G(\mathbf{z} | \mathbf{y}) | \mathbf{y}))] \tag{11.79}
$$

在实现上，条件变量 $$\mathbf{y}$$ 通常通过**拼接**或**嵌入**的方式融入网络：在生成器中与噪声 $$\mathbf{z}$$ 拼接作为输入，在判别器中与数据 $$\mathbf{x}$$ 拼接后一起判别。
{% endblock %}

条件 GAN 的应用极其广泛。在数据挖掘中，cGAN 可以针对少数类生成合成样本来缓解**类别不平衡**问题（如在欺诈检测中增加欺诈样本）；在计算机视觉中，Pix2Pix（Isola 等, 2017）使用 cGAN 实现图像到图像的翻译（如语义分割图 → 真实照片）；在医学影像中，cGAN 可以在给定诊断标签的条件下生成对应的病理图像，用于训练数据增强。条件变量 $$\mathbf{y}$$ 的形式也不限于类别标签——它可以是文本描述（文本到图像生成）、另一幅图像（风格迁移）、甚至是时间序列的历史数据（条件时间序列生成）。

## 11.5.7 方法对比

回顾本节介绍的 GAN 家族及其与 VAE 的关系，下表总结了各核心方法的特点：

| 方法 | 损失/目标 | 生成质量 | 训练稳定性 | 多样性 | 条件生成 | 典型应用 |
|------|---------|---------|----------|-------|---------|---------|
| 原始 GAN | JSD | 中等 | 差 | 易模式崩溃 | 否 | 理论基础 |
| DCGAN | JSD + CNN | 较好 | 一般 | 一般 | 否 | 图像生成 |
| WGAN-GP | $$W_1$$ + GP | 好 | 好 | 好 | 否 | 稳定训练 |
| cGAN | JSD + 条件 | 好 | 一般 | 好 | 是 | 可控生成 |
| VAE (§11.4) | ELBO | 模糊 | 好 | 好 | 可扩展 | 表示学习 |

从表中可以观察到两个趋势：(1) **生成质量与训练稳定性的权衡**——原始 GAN 的理论框架优美但训练困难，WGAN 通过改进距离度量同时改善了两者；(2) **GAN 与 VAE 的互补性**——GAN 擅长生成锐利、逼真的样本但训练不稳定且缺乏有意义的潜在空间，VAE 训练稳定且潜在空间结构化但生成较模糊。近年来的研究趋势是将两者结合（如 VAE-GAN），利用 VAE 的编码器提供结构化潜在空间，同时用 GAN 的判别器替代像素级重建损失以提升生成锐度。

## 案例 11.5：基于 DCGAN 的 MNIST 数字生成

在数据挖掘中，生成模型不仅可以创造逼真的合成数据用于**数据增强**，还可以揭示数据分布的内在结构。本案例使用 DCGAN 在 MNIST 手写数字数据集上训练生成模型，展示如何从随机噪声生成逼真的数字图像，并通过 FID 指标定量评估生成质量。

{% block type="case" %}
**案例 11.5**（MNIST 数字生成）

**任务**：训练 DCGAN 生成 $$28 \times 28$$ 的手写数字图像。

**数据**：MNIST 训练集 60,000 张灰度图，归一化至 $$[-1, 1]$$。

**评估指标**：Fréchet Inception Distance（FID）衡量生成图像与真实图像的统计距离：

$$
\text{FID} = \|\boldsymbol{\mu}_r - \boldsymbol{\mu}_g\|^2 + \text{Tr}\!\left(\Sigma_r + \Sigma_g - 2(\Sigma_r \Sigma_g)^{1/2}\right) \tag{11.80}
$$

其中 $$(\boldsymbol{\mu}_r, \Sigma_r)$$ 和 $$(\boldsymbol{\mu}_g, \Sigma_g)$$ 分别为真实图像和生成图像在 Inception 网络特征空间中的均值和协方差。FID 越小，生成质量越高。
{% endblock %}

**实现要点**：生成器使用转置卷积逐步上采样（$$100 \to 256 \times 7 \times 7 \to 128 \times 14 \times 14 \to 1 \times 28 \times 28$$），判别器使用步幅卷积逐步下采样（$$1 \times 28 \times 28 \to 64 \times 14 \times 14 \to 128 \times 7 \times 7 \to 1$$），使用 Adam 优化器（$$\alpha = 0.0002, \beta_1 = 0.5$$），按算法 11.5 的交替策略训练，并采用非饱和损失（式 11.74）与标签平滑。

```python
import torch
import torch.nn as nn
from torch.utils.data import DataLoader
from torchvision import datasets, transforms

# --- 1. 数据准备 ---
transform = transforms.Compose([
    transforms.ToTensor(),
    transforms.Normalize([0.5], [0.5])  # 归一化至 [-1, 1]
])
train_data = datasets.MNIST("data", train=True, download=True,
                            transform=transform)
loader = DataLoader(train_data, batch_size=128, shuffle=True)
nz = 100  # 噪声维度

# --- 2. 生成器 (转置卷积上采样) ---
G = nn.Sequential(
    nn.ConvTranspose2d(nz, 256, 7, 1, 0, bias=False),
    nn.BatchNorm2d(256), nn.ReLU(True),
    nn.ConvTranspose2d(256, 128, 4, 2, 1, bias=False),
    nn.BatchNorm2d(128), nn.ReLU(True),
    nn.ConvTranspose2d(128, 1, 4, 2, 1, bias=False),
    nn.Tanh()
)

# --- 3. 判别器 (步幅卷积下采样) ---
D = nn.Sequential(
    nn.Conv2d(1, 64, 4, 2, 1, bias=False),
    nn.LeakyReLU(0.2, inplace=True),
    nn.Conv2d(64, 128, 4, 2, 1, bias=False),
    nn.BatchNorm2d(128), nn.LeakyReLU(0.2, inplace=True),
    nn.Flatten(),
    nn.Linear(128 * 7 * 7, 1), nn.Sigmoid()
)

# --- 4. 训练循环 ---
criterion = nn.BCELoss()
opt_D = torch.optim.Adam(D.parameters(), lr=0.0002, betas=(0.5, 0.999))
opt_G = torch.optim.Adam(G.parameters(), lr=0.0002, betas=(0.5, 0.999))

for epoch in range(20):
    for real, _ in loader:
        bs = real.size(0)
        # 标签平滑: 真实标签 0.9, 假标签 0.0
        real_label = torch.full((bs, 1), 0.9)
        fake_label = torch.zeros(bs, 1)

        # 更新判别器
        z = torch.randn(bs, nz, 1, 1)
        fake = G(z).detach()
        loss_D = criterion(D(real), real_label) + \
                 criterion(D(fake), fake_label)
        opt_D.zero_grad(); loss_D.backward(); opt_D.step()

        # 更新生成器 (非饱和损失)
        z = torch.randn(bs, nz, 1, 1)
        fake = G(z)
        loss_G = criterion(D(fake), torch.ones(bs, 1))
        opt_G.zero_grad(); loss_G.backward(); opt_G.step()

    print(f"Epoch {epoch+1}: D_loss={loss_D.item():.4f}, "
          f"G_loss={loss_G.item():.4f}")
# 典型结果: 20 epoch 后可生成清晰可辨的手写数字
```

**结果分析**：经过 20 个 epoch 的对抗训练，DCGAN 的生成器能够从 100 维随机噪声生成清晰可辨的手写数字图像。训练初期，生成的图像呈噪声状态，判别器轻松区分真假（$$D$$ loss 接近 0）。随着训练推进，生成图像逐渐出现数字的笔画结构，判别器的优势减小。到训练后期，两者达到近似均衡——生成图像在视觉上与真实 MNIST 样本难以区分，判别器的输出接近 0.5。典型的 FID 分数约为 10–20（越低越好），远优于基于像素级重建的 VAE 模型。图 11.5.3 展示了训练过程中的生成样本和损失曲线。

## 本节小结

本节系统介绍了生成对抗网络的对抗训练框架及其核心变体。从极小极大博弈目标出发，我们推导了最优判别器定理和全局最优性定理，建立了 GAN 与 Jensen-Shannon 散度的理论联系。针对原始 GAN 的训练不稳定性和模式崩溃问题，DCGAN 通过卷积架构提升了图像生成质量，WGAN 通过 Wasserstein 距离从根本上改善了训练稳定性，条件 GAN 则赋予了生成过程可控性。以下表格汇总了本节的核心概念：

| 关键词 | 含义 |
|--------|------|
| 生成器 / 判别器 | GAN 的两个对抗网络：生成器制造假样本，判别器区分真假 |
| 极小极大博弈 | GAN 的优化目标——生成器最小化、判别器最大化同一目标函数 |
| Jensen-Shannon 散度 | GAN 在最优判别器下等价最小化的分布距离 |
| 模式崩溃 | 生成器只覆盖数据分布的部分模式 |
| Wasserstein 距离 | WGAN 使用的更平滑的分布距离，即使分布不重叠也提供梯度 |
| 条件 GAN | 引入条件变量实现可控生成 |
