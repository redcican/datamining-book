# 11.4 自编码器与表示学习

前三节介绍的 MLP、CNN 和 RNN 都是**有监督学习**模型——它们需要标注数据 $$(\mathbf{x}, y)$$ 来训练。然而，在数据挖掘的实际场景中，大量数据是**没有标签**的：社交媒体上每天产生的数十亿张图片、电商平台的海量用户行为日志、工业物联网中持续涌入的传感器信号——为这些数据人工标注既昂贵又耗时。能否让模型在**没有标签**的情况下自动发现数据中的内在结构和有效表示？

**自编码器**（Autoencoder, AE）正是为回答这一问题而设计的无监督学习框架。其核心思想极为优雅：让网络学习将输入"压缩"为一个低维的**潜在表示**（Latent Representation），然后从这个压缩表示中"重建"出原始输入。如果网络能够在信息瓶颈的约束下成功重建输入，那么潜在表示必然捕获了数据中最本质的特征——噪声和冗余信息被迫丢弃，只有最重要的结构被保留。这种"通过重建来学习表示"的思想，使自编码器成为数据挖掘中降维、特征提取、异常检测和数据生成等任务的核心工具。

本节将从基本自编码器的数学定义出发，分析其与经典 PCA 的深刻联系，介绍稀疏和去噪两种正则化变体，然后重点推导**变分自编码器**（VAE）的概率框架和 ELBO 目标函数，最后通过 MNIST 异常检测案例展示自编码器在数据挖掘中的实际应用。

## 11.4.1 从降维到表示学习

在数据挖掘的经典方法中，**主成分分析**（PCA）是最常用的降维技术之一——它通过寻找数据方差最大的正交方向，将高维数据线性投影到低维子空间。PCA 简单高效，但有一个根本局限：它只能捕获数据中的**线性结构**。对于图像这样高度非线性的数据（例如，一只猫在不同光照、角度和姿态下的图像），数据的内在流形是一个弯曲的非线性子空间，线性投影无法有效表示。

自编码器可以看作 PCA 的**非线性推广**——用神经网络代替线性投影，使降维过程可以捕获复杂的非线性关系。更重要的是，自编码器学到的潜在表示不仅可以用于降维，还可以作为下游监督任务（如分类、聚类）的**特征输入**——这就是**表示学习**（Representation Learning）的核心思想：学习数据的中间表示，使后续任务更加容易。

## 11.4.2 基本自编码器

### 编码-解码框架

自编码器由两个互为"镜像"的子网络组成：**编码器**将输入压缩为低维潜在表示，**解码器**从潜在表示重建输入。中间的低维潜在层形成一个**信息瓶颈**（Information Bottleneck），迫使网络学习数据中最本质的特征。

{% block type="definition" %}
**定义 11.19**（自编码器）　自编码器由编码器 $$\phi$$ 和解码器 $$\psi$$ 组成：

**编码器**将输入映射到潜在空间：

$$
\mathbf{z} = f_\phi(\mathbf{x}) = \sigma\!\big(W_e \mathbf{x} + \mathbf{b}_e\big), \quad \mathbf{z} \in \mathbb{R}^k \tag{11.59}
$$

**解码器**从潜在表示重建输入：

$$
\hat{\mathbf{x}} = g_\psi(\mathbf{z}) = \sigma\!\big(W_d \mathbf{z} + \mathbf{b}_d\big), \quad \hat{\mathbf{x}} \in \mathbb{R}^d \tag{11.60}
$$

**训练目标**是最小化重建误差：

$$
\mathcal{L}_{\text{AE}} = \frac{1}{n} \sum_{i=1}^{n} \|\mathbf{x}_i - \hat{\mathbf{x}}_i\|^2 = \frac{1}{n} \sum_{i=1}^{n} \|\mathbf{x}_i - g_\psi(f_\phi(\mathbf{x}_i))\|^2 \tag{11.61}
$$

其中 $$k < d$$ 为潜在维度（瓶颈层宽度），$$\sigma$$ 为激活函数。当 $$k < d$$ 时称为**欠完备自编码器**（Undercomplete Autoencoder），瓶颈强制网络学习数据的压缩表示。
{% endblock %}

式 (11.61) 的训练目标直观而有力：网络必须在 $$k$$ 维的"瓶颈"中保留足够的信息，使得解码器能够重建原始的 $$d$$ 维输入。由于 $$k \ll d$$，大量信息必须被丢弃——网络被迫学习只保留**对重建最重要的特征**，而舍弃噪声和冗余。这个过程与 PCA 的"保留最大方差方向"在精神上是一致的，但自编码器通过非线性激活函数可以捕获远比线性投影丰富的结构。

### 线性自编码器与 PCA 的等价性

自编码器与 PCA 的联系不仅是直觉上的类比，而是可以严格证明的数学等价关系。当自编码器退化为线性模型时，它恰好学到了 PCA 的解：

{% block type="theorem" %}
**定理 11.6**（线性自编码器与 PCA 的等价性）　设编码器和解码器均为线性映射（无激活函数）：$$\mathbf{z} = W_e \mathbf{x}$$，$$\hat{\mathbf{x}} = W_d \mathbf{z}$$，且潜在维度为 $$k$$。最小化重建误差 $$\mathcal{L} = \frac{1}{n}\sum_i \|\mathbf{x}_i - W_d W_e \mathbf{x}_i\|^2$$ 的最优解满足：$$W_d W_e$$ 的列空间等于数据协方差矩阵 $$\Sigma = \frac{1}{n}X^\top X$$ 的前 $$k$$ 个特征向量张成的子空间——即 PCA 的主子空间。

**证明思路**：令 $$P = W_d W_e$$，最小化 $$\|X - XP^\top\|_F^2$$ 等价于寻找最优的秩-$$k$$ 投影矩阵。由 Eckart-Young 定理，最优解为 $$P = U_k U_k^\top$$，其中 $$U_k$$ 为 $$\Sigma$$ 的前 $$k$$ 个特征向量。因此线性自编码器的最优解与 PCA 完全一致。$$\blacksquare$$
{% endblock %}

定理 11.6 建立了自编码器与经典降维方法之间的桥梁：PCA 是线性自编码器的特例，而具有非线性激活函数的自编码器可以学习**非线性流形**上的降维——它能捕获数据中 PCA 无法发现的弯曲结构。这也是自编码器在图像、文本等复杂数据上表现优于 PCA 的理论基础。

## 11.4.3 正则化自编码器

基本的欠完备自编码器通过限制瓶颈维度 $$k < d$$ 来防止网络学到恒等映射。然而，如果编码器和解码器的容量足够大（例如很深的网络），即使 $$k < d$$，网络也可能只是学到了某种高效的"记忆"方式而非有意义的表示。**正则化自编码器**通过在损失函数中添加额外的约束，强制潜在表示具有特定的理想性质——即使在 $$k \ge d$$（过完备）的情况下也能学到有用的特征。

### 稀疏自编码器

{% block type="definition" %}
**定义 11.20**（稀疏自编码器）　稀疏自编码器在重建误差之上添加**稀疏性约束**，强制潜在表示中大部分神经元处于非激活状态。设第 $$j$$ 个隐藏神经元在所有训练样本上的平均激活值为 $$\hat{\rho}_j = \frac{1}{n}\sum_{i=1}^{n} z_j^{(i)}$$，目标稀疏度为 $$\rho$$（通常取 0.05），稀疏自编码器的损失函数为：

$$
\mathcal{L}_{\text{sparse}} = \mathcal{L}_{\text{AE}} + \beta \sum_{j=1}^{k} \text{KL}(\rho \| \hat{\rho}_j) = \mathcal{L}_{\text{AE}} + \beta \sum_{j=1}^{k} \left[\rho \log \frac{\rho}{\hat{\rho}_j} + (1-\rho) \log \frac{1-\rho}{1-\hat{\rho}_j}\right] \tag{11.62}
$$

其中 $$\beta$$ 为稀疏惩罚强度，$$\text{KL}(\rho \| \hat{\rho}_j)$$ 为 Bernoulli 分布之间的 KL 散度。
{% endblock %}

稀疏性约束的直觉来自神经科学：大脑皮层的神经元在任一时刻只有很少一部分处于活跃状态（约 1%–5%），这种稀疏激活模式被认为有助于学习更具区分性的特征。在数据挖掘中，稀疏表示使得每个样本只由少数几个"活跃特征"描述，天然适合作为聚类或分类的输入——类似于将一幅画用少数几种颜色来描述，每种颜色都代表一个显著的视觉模式。

### 去噪自编码器

另一种有效的正则化策略是**去噪自编码器**（Denoising Autoencoder, DAE），它从一个不同的角度出发：不是约束潜在表示的性质，而是约束训练过程的输入。

{% block type="definition" %}
**定义 11.21**（去噪自编码器）　去噪自编码器将输入 $$\mathbf{x}$$ 随机**腐蚀**为 $$\tilde{\mathbf{x}}$$（如随机置零、添加高斯噪声），然后要求网络从腐蚀后的输入重建**原始**的干净输入：

$$
\mathcal{L}_{\text{DAE}} = \frac{1}{n} \sum_{i=1}^{n} \|\mathbf{x}_i - g_\psi(f_\phi(\tilde{\mathbf{x}}_i))\|^2, \quad \tilde{\mathbf{x}}_i \sim q(\tilde{\mathbf{x}} | \mathbf{x}_i) \tag{11.63}
$$

其中 $$q(\tilde{\mathbf{x}} | \mathbf{x})$$ 为腐蚀分布，常见选择包括：(1) **掩码噪声**：以概率 $$p$$ 将每个维度置零；(2) **加性高斯噪声**：$$\tilde{\mathbf{x}} = \mathbf{x} + \boldsymbol{\epsilon}$$，$$\boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, \sigma^2 I)$$。
{% endblock %}

去噪自编码器的训练迫使网络学会从局部的、带噪声的观测中"推断"出完整的、干净的数据——这要求编码器不仅记住输入的表面形式，更要理解数据的内在结构和统计规律。从流形学习的角度看，去噪自编码器在学习将被噪声推离数据流形的点"拉回"到流形上——编码器学到的梯度方向近似指向数据流形的法线方向。这种几何直觉也解释了为什么去噪自编码器的潜在表示通常比基本自编码器更加鲁棒和语义化。

### 收缩自编码器

**收缩自编码器**（Contractive Autoencoder, CAE）通过惩罚编码器 Jacobian 矩阵的 Frobenius 范数来实现正则化：

$$
\mathcal{L}_{\text{CAE}} = \mathcal{L}_{\text{AE}} + \lambda \left\|\frac{\partial f_\phi(\mathbf{x})}{\partial \mathbf{x}}\right\|_F^2 \tag{11.64}
$$

Jacobian 惩罚使编码器对输入的微小扰动不敏感——即学到的表示在局部方向上"收缩"，只对数据流形上的切线方向敏感。这与去噪自编码器的效果类似，但从微分几何的角度提供了更精确的正则化。

## 11.4.4 变分自编码器

上述自编码器（基本、稀疏、去噪）都是**确定性**模型：给定输入 $$\mathbf{x}$$，编码器输出一个固定的潜在向量 $$\mathbf{z}$$。这种确定性映射在表示学习方面很有效，但有一个重要缺陷：它无法**生成**新的数据样本——我们不知道潜在空间中哪些区域对应有意义的数据，随机采样一个 $$\mathbf{z}$$ 后解码出的结果往往毫无意义。

**变分自编码器**（Variational Autoencoder, VAE）是 Kingma 和 Welling 于 2014 年提出的概率生成模型，它将自编码器框架与**贝叶斯推断**相结合：编码器不再输出一个固定的点，而是输出一个**概率分布**的参数——潜在表示成为一个随机变量，从而赋予潜在空间以明确的概率结构，使数据生成成为可能。

### 概率生成框架

VAE 假设数据 $$\mathbf{x}$$ 由一个不可观测的潜在变量 $$\mathbf{z}$$ 生成，生成过程为：

$$
\mathbf{z} \sim p(\mathbf{z}) = \mathcal{N}(\mathbf{0}, I), \quad \mathbf{x} \sim p_\theta(\mathbf{x} | \mathbf{z}) \tag{11.65}
$$

其中 $$p(\mathbf{z})$$ 为标准正态先验，$$p_\theta(\mathbf{x} | \mathbf{z})$$ 为由解码器网络参数化的似然函数。我们的目标是最大化数据的边际似然 $$p_\theta(\mathbf{x}) = \int p_\theta(\mathbf{x} | \mathbf{z}) p(\mathbf{z}) \, d\mathbf{z}$$，但这个积分在高维空间中是不可计算的（intractable）。

### ELBO 推导

VAE 的核心思想是引入一个**推断网络**（编码器）$$q_\phi(\mathbf{z} | \mathbf{x})$$ 来近似不可计算的真实后验 $$p_\theta(\mathbf{z} | \mathbf{x})$$，然后推导出一个可计算的**下界**作为优化目标。

{% block type="definition" %}
**定义 11.22**（变分自编码器）　变分自编码器最大化数据对数似然的**证据下界**（Evidence Lower BOund, ELBO）：

$$
\log p_\theta(\mathbf{x}) \ge \underbrace{\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}\big[\log p_\theta(\mathbf{x} | \mathbf{z})\big]}_{\text{重建项}} - \underbrace{D_{\text{KL}}\big(q_\phi(\mathbf{z} | \mathbf{x}) \| p(\mathbf{z})\big)}_{\text{正则化项}} = \text{ELBO}(\phi, \theta; \mathbf{x}) \tag{11.66}
$$

编码器输出潜在分布的参数：$$q_\phi(\mathbf{z} | \mathbf{x}) = \mathcal{N}(\boldsymbol{\mu}_\phi(\mathbf{x}),\, \operatorname{diag}(\boldsymbol{\sigma}^2_\phi(\mathbf{x})))$$。当先验为标准正态时，KL 散度有解析解：

$$
D_{\text{KL}}\big(q_\phi(\mathbf{z} | \mathbf{x}) \| \mathcal{N}(\mathbf{0}, I)\big) = -\frac{1}{2} \sum_{j=1}^{k} \big(1 + \log \sigma_j^2 - \mu_j^2 - \sigma_j^2\big) \tag{11.67}
$$
{% endblock %}

式 (11.66) 的两项有着清晰的直觉解释。**重建项** $$\mathbb{E}[\log p_\theta(\mathbf{x} | \mathbf{z})]$$ 衡量解码器从潜在表示重建原始数据的能力——这与基本自编码器的重建误差类似。**正则化项** $$D_{\text{KL}}(q_\phi \| p)$$ 约束后验分布接近标准正态先验——这使得潜在空间具有良好的结构：连续、平滑、且支持从先验分布直接采样生成新数据。两项之间存在张力：重建项鼓励编码器输出信息量大的、精确的潜在表示，正则化项则鼓励表示尽可能接近标准正态——VAE 在两者之间取得平衡。

### 重参数化技巧

ELBO 中的期望 $$\mathbb{E}_{q_\phi(\mathbf{z}|\mathbf{x})}[\cdot]$$ 需要从 $$q_\phi$$ 中采样，但采样操作是不可微的——无法通过反向传播计算梯度。Kingma 和 Welling 提出了一个精巧的**重参数化技巧**（Reparameterization Trick）来解决这一问题：

$$
\mathbf{z} = \boldsymbol{\mu}_\phi(\mathbf{x}) + \boldsymbol{\sigma}_\phi(\mathbf{x}) \odot \boldsymbol{\epsilon}, \quad \boldsymbol{\epsilon} \sim \mathcal{N}(\mathbf{0}, I) \tag{11.68}
$$

通过将随机性从网络内部"外推"到一个独立于参数的噪声变量 $$\boldsymbol{\epsilon}$$，采样操作变成了一个关于 $$\boldsymbol{\mu}$$ 和 $$\boldsymbol{\sigma}$$ 的**可微变换**——梯度可以正常地从 $$\mathbf{z}$$ 回传到编码器参数 $$\phi$$。

综合以上各项，VAE 的最终训练损失为：

$$
\mathcal{L}_{\text{VAE}} = -\text{ELBO} = \underbrace{\|\mathbf{x} - \hat{\mathbf{x}}\|^2}_{\text{重建损失}} + \underbrace{\frac{1}{2} \sum_{j=1}^{k} \big(\mu_j^2 + \sigma_j^2 - \log \sigma_j^2 - 1\big)}_{\text{KL 正则化}} \tag{11.69}
$$

{% hint style="tip" %}
💡 **VAE 与基本 AE 的关键区别**：基本自编码器的潜在空间是**离散的点**——每个输入映射到一个固定位置，点与点之间的空间没有定义。VAE 的潜在空间是**连续的概率分布**——每个输入映射到一个"云团"（由 $$\boldsymbol{\mu}$$ 和 $$\boldsymbol{\sigma}$$ 定义的高斯分布），且 KL 正则化确保这些云团与标准正态分布重叠。这使得潜在空间的任何位置都有明确的概率含义，从先验分布采样 $$\mathbf{z} \sim \mathcal{N}(\mathbf{0}, I)$$ 后解码即可生成有意义的新数据。
{% endhint %}

## 11.4.5 方法对比

回顾本节介绍的各类自编码器，它们在正则化策略、潜在空间性质和适用场景上各有特点。下表总结了各变体的核心差异：

| 变体 | 正则化方式 | 潜在空间性质 | 可否生成 | 典型应用 |
|------|----------|-----------|---------|---------|
| 基本 AE | 瓶颈维度 $$k < d$$ | 确定性、不连续 | 否 | 降维、特征提取 |
| 稀疏 AE | KL 稀疏惩罚 | 稀疏激活 | 否 | 特征选择、聚类 |
| 去噪 AE | 输入腐蚀 | 鲁棒表示 | 否 | 去噪、流形学习 |
| 收缩 AE | Jacobian 惩罚 | 局部不变 | 否 | 流形学习 |
| VAE | KL 散度正则化 | 连续、概率化 | 是 | 生成、异常检测 |

从表中可以看出两个重要趋势：(1) **正则化的核心目标**从限制容量（基本 AE）逐步演化为约束表示的理想性质（稀疏性、鲁棒性、连续性）；(2) **VAE 是唯一具备生成能力的变体**——其概率化的潜在空间使得"从表示到数据"的方向成为可能，这是确定性自编码器无法实现的。在数据挖掘实践中，特征提取任务优先考虑去噪或稀疏自编码器，异常检测任务优先考虑 VAE（利用重建误差或潜在空间概率），而数据增强和生成任务则必须使用 VAE 或其他生成模型（如 GAN，§11.5）。

## 案例 11.4：基于自编码器的 MNIST 异常检测

在数据挖掘中，**异常检测**（Anomaly Detection）是一项核心任务——识别偏离正常模式的稀有样本，如信用卡欺诈、设备故障或网络入侵。自编码器提供了一种直观的异常检测思路：用正常数据训练自编码器，使其学会重建正常模式；推理时，正常样本的重建误差较小，而异常样本（模型从未见过的模式）的重建误差显著偏大——以重建误差为异常分数，即可区分正常与异常。

{% block type="case" %}
**案例 11.4**（MNIST 异常检测）

**任务**：将 MNIST 中的数字"0"视为正常类别，其余数字（1–9）视为异常。仅用数字"0"的样本训练自编码器，在测试集上通过重建误差区分正常与异常样本。

**模型架构**：全连接自编码器，编码器 784→256→64→16，解码器 16→64→256→784（对称结构），隐藏层使用 ReLU 激活，输出层使用 Sigmoid（像素值归一化到 $$[0,1]$$）。
{% endblock %}

{% block type="algorithm" %}
**算法 11.4**（自编码器异常检测流程）

---

**输入**：正常训练集 $$\{\mathbf{x}_i\}_{i=1}^N$$，测试集 $$\{\mathbf{x}_j\}$$（含正常和异常），阈值百分位数 $$\alpha$$

**输出**：异常标签 $$\{a_j\}$$

1. 初始化自编码器参数 $$\boldsymbol{\theta}$$
2. **for** $$t = 1, 2, \ldots, T$$ **do**
3. 　　**for** 每个小批量 $$\mathcal{B}$$ **do**
4. 　　　　前向传播：$$\hat{\mathbf{x}} = g_\psi(f_\phi(\mathbf{x}))$$
5. 　　　　计算 MSE 重建损失：$$\mathcal{L} = \frac{1}{B}\sum_{i \in \mathcal{B}} \|\mathbf{x}_i - \hat{\mathbf{x}}_i\|^2$$
6. 　　　　反向传播并更新参数
7. 　　**end for**
8. **end for**
9. 对测试集计算重建误差：$$e_j = \|\mathbf{x}_j - g_\psi(f_\phi(\mathbf{x}_j))\|^2$$
10. 设定阈值 $$\tau$$ 为正常训练集重建误差的第 $$\alpha$$ 百分位数
11. **return** $$a_j = \mathbb{1}[e_j > \tau]$$（重建误差超过阈值则判定为异常）

---
{% endblock %}

**PyTorch 实现**：

```python
import torch
import torch.nn as nn
import torchvision
import torchvision.transforms as T
from torch.utils.data import DataLoader, Subset
import numpy as np

# --- 1. 准备数据：仅用数字 0 训练 ---
transform = T.Compose([T.ToTensor(), T.Lambda(lambda x: x.view(-1))])
full_train = torchvision.datasets.MNIST(
    root="./data", train=True, download=True, transform=transform)
full_test = torchvision.datasets.MNIST(
    root="./data", train=False, download=True, transform=transform)
normal_idx = [i for i, (_, y) in enumerate(full_train) if y == 0]
train_loader = DataLoader(Subset(full_train, normal_idx),
                          batch_size=128, shuffle=True)

# --- 2. 定义自编码器 ---
class Autoencoder(nn.Module):
    def __init__(self):
        super().__init__()
        self.encoder = nn.Sequential(
            nn.Linear(784, 256), nn.ReLU(),
            nn.Linear(256, 64), nn.ReLU(),
            nn.Linear(64, 16),
        )
        self.decoder = nn.Sequential(
            nn.Linear(16, 64), nn.ReLU(),
            nn.Linear(64, 256), nn.ReLU(),
            nn.Linear(256, 784), nn.Sigmoid(),
        )

    def forward(self, x):
        return self.decoder(self.encoder(x))

model = Autoencoder()
optimizer = torch.optim.Adam(model.parameters(), lr=1e-3)
criterion = nn.MSELoss()

# --- 3. 训练 ---
for epoch in range(30):
    model.train()
    for (x,), in zip(train_loader):
        recon = model(x)
        loss = criterion(recon, x)
        optimizer.zero_grad()
        loss.backward()
        optimizer.step()

# --- 4. 异常检测评估 ---
model.eval()
errors, labels = [], []
with torch.no_grad():
    for x, y in DataLoader(full_test, batch_size=256):
        x_flat = x.view(x.size(0), -1)
        recon = model(x_flat)
        err = ((x_flat - recon) ** 2).mean(dim=1)
        errors.extend(err.numpy())
        labels.extend((y != 0).numpy())  # 非 0 为异常
errors = np.array(errors)
labels = np.array(labels)

# --- 5. 计算 AUC ---
from sklearn.metrics import roc_auc_score
auc = roc_auc_score(labels, errors)
print(f"异常检测 AUC: {auc:.4f}")
# 典型结果: AUC ≈ 0.97
```

**结果分析**：仅用数字"0"训练的自编码器在异常检测任务上通常可达到 AUC ≈ 0.97 的优异性能。这个结果展示了自编码器异常检测的核心原理：模型在训练时只见过数字"0"的笔画模式，因此能够以很低的误差重建任何"0"的变体；但当输入是数字"5"或"8"时，这些笔画模式与"0"截然不同，解码器无法从 16 维潜在表示中重建出它们，产生显著的重建误差——重建误差自然成为了区分正常与异常的判据。图 11.4.3 直观展示了这一过程。

## 本节小结

本节系统介绍了自编码器家族及其在无监督表示学习中的核心地位。从编码-解码框架出发，我们理解了信息瓶颈如何迫使网络学习数据的本质特征，并通过定理 11.6 建立了自编码器与经典 PCA 的数学联系。稀疏、去噪和收缩三种正则化变体从不同角度增强了表示的质量。变分自编码器则通过引入概率框架，赋予潜在空间以连续、可采样的结构，实现了从"表示学习"到"数据生成"的质变。以下表格汇总了本节的核心概念：

| 关键词 | 含义 |
|--------|------|
| 自编码器 | 通过编码-解码的重建任务学习数据的压缩表示 |
| 信息瓶颈 | 低维潜在层迫使网络只保留最重要的信息 |
| 稀疏正则化 | 约束潜在表示中大部分神经元不激活，学习更具区分性的特征 |
| 去噪自编码器 | 从损坏的输入中重建原始数据，学习鲁棒的流形表示 |
| VAE | 将潜在空间概率化，支持数据生成和有意义的插值 |
| ELBO | VAE 的优化目标——重建项与 KL 正则项的权衡 |
