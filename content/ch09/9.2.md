# 9.2 文本表示模型

上一节（§9.1）完成了文本预处理的全部步骤——分词、规范化、停用词过滤和词干提取/词形还原，将原始文本转化为标准化的词项序列。然而，机器学习算法需要的是**数值向量**，而非词项序列。本节的核心问题是：**如何将一组词项映射为数学空间中的向量，同时尽可能保留文本的语义信息？**

这个问题的答案在过去三十年中经历了深刻的演变。从最简单的**词袋模型**（统计每个词出现了几次）到 **TF-IDF**（赋予"有区分力的词"更高权重），再到 **Word2Vec**（将每个词映射为稠密的语义向量），直至 **BERT**（同一个词在不同上下文中有不同的向量）。每一代方法都在解决前一代的核心局限——从"有没有这个词"到"这个词多重要"到"这个词是什么意思"到"这个词在这句话里是什么意思"。

## 9.2.1 词袋模型

### 9.2.1.1 文档-词项矩阵

最直观的文本向量化方式是：**忽略词序，只统计每个词出现了几次**。

{% block type="definition" title="定义 9.7　词袋模型（Bag of Words, BoW）" %}
给定词汇表 $$V = \{w_1, w_2, \ldots, w_{|V|}\}$$，文档 $$d$$ 的**词袋表示**为一个 $$|V|$$ 维向量：

$$
\mathbf{x}_d = \bigl(\text{tf}(w_1, d),\, \text{tf}(w_2, d),\, \ldots,\, \text{tf}(w_{|V|}, d)\bigr)^\top \tag{9.11}
$$

其中 $$\text{tf}(w_j, d)$$ 是词项 $$w_j$$ 在文档 $$d$$ 中的出现次数（§9.1 方程 9.1）。

对于 $$N$$ 个文档的语料库 $$\mathcal{D}$$，所有文档的词袋向量构成**文档-词项矩阵** $$\mathbf{X} \in \mathbb{R}^{N \times |V|}$$，其中 $$X_{ij} = \text{tf}(w_j, d_i)$$。
{% endblock %}

词袋模型的名字源于一个形象的比喻：将文档中的所有词倒入一个袋子里，只关心每个词有几个，不关心它们的原始顺序。例如，"猫追狗"和"狗追猫"在词袋模型中具有**完全相同的表示**——这显然丢失了重要的语义信息。

词袋模型有两个突出的问题：

- **高维稀疏性**：典型的英文语料库词汇表大小 $$|V| \sim 10^4$$–$$10^5$$，但每个文档通常只包含几十到几百个不重复词。文档-词项矩阵的非零率通常低于 1%——这是一个极度稀疏的矩阵。
- **等权假设**：所有词被同等对待。但"的""是""在"等高频功能词在几乎所有文档中都出现，对区分文档内容毫无帮助；而"机器学习""神经网络"等专业术语虽然低频，却是文档主题的关键标志。

第一个问题可以通过降维方法（如 LSA，§9.4）缓解。第二个问题正是 TF-IDF 要解决的。

## 9.2.2 TF-IDF 加权

### 9.2.2.1 逆文档频率

TF-IDF 的核心思想是：**一个词的重要性不仅取决于它在当前文档中出现得多频繁（TF），还取决于它在整个语料库中有多"稀有"（IDF）**。

{% block type="definition" title="定义 9.8　TF-IDF 权重" %}
给定词项 $$w$$、文档 $$d$$ 和语料库 $$\mathcal{D}$$（共 $$N$$ 个文档），定义：

**逆文档频率**（Inverse Document Frequency, IDF）：

$$
\text{idf}(w) = \log \frac{N}{\text{df}(w)} \tag{9.12}
$$

其中 $$\text{df}(w) = |\{d \in \mathcal{D} : w \in d\}|$$ 是包含词项 $$w$$ 的文档数量。

**TF-IDF 权重**：

$$
\text{tf-idf}(w, d) = \text{tf}(w, d) \times \text{idf}(w) \tag{9.13}
$$

文档 $$d$$ 的 TF-IDF 向量为 $$\mathbf{x}_d^{\text{tfidf}} = \bigl(\text{tf-idf}(w_1, d), \ldots, \text{tf-idf}(w_{|V|}, d)\bigr)^\top$$。
{% endblock %}

IDF 的直觉是：如果一个词在所有文档中都出现（$$\text{df}(w) = N$$），则 $$\text{idf}(w) = \log 1 = 0$$，该词的权重为零——它对区分文档没有贡献。反之，如果一个词只在极少数文档中出现，$$\text{idf}(w)$$ 很大——它是这些文档的强特征。

### 9.2.2.2 文档相似度

有了文档向量之后，最常用的相似度度量是余弦相似度：

$$
\text{sim}_{\cos}(\mathbf{x}_a, \mathbf{x}_b) = \frac{\mathbf{x}_a \cdot \mathbf{x}_b}{\|\mathbf{x}_a\| \cdot \|\mathbf{x}_b\|} = \frac{\sum_{j=1}^{|V|} x_{aj} x_{bj}}{\sqrt{\sum_{j} x_{aj}^2} \cdot \sqrt{\sum_{j} x_{bj}^2}} \tag{9.14}
$$

余弦相似度衡量两个向量方向的一致性，取值范围为 $$[0, 1]$$（TF-IDF 向量的所有分量非负）。$$\text{sim}_{\cos} = 1$$ 表示两个文档的词汇分布完全相同（可能长度不同），$$\text{sim}_{\cos} = 0$$ 表示两个文档没有任何共同词汇。

### 9.2.2.3 TF-IDF 的信息论解释

TF-IDF 的有效性不是偶然的——IDF 有深刻的信息论基础。

{% block type="theorem" title="定理 9.2　IDF 与自信息量的关系" %}
设词项 $$w$$ 在语料库中的文档频率为 $$\text{df}(w)$$，则随机选择一个文档包含 $$w$$ 的概率为 $$P(w) = \text{df}(w) / N$$。词项 $$w$$ 的**自信息量**（Self-Information）为：

$$
I(w) = -\log P(w) = \log \frac{N}{\text{df}(w)} = \text{idf}(w) \tag{9.15}
$$

因此，IDF 恰好等于词项的自信息量——**一个词越稀有，它携带的信息量越大**。

**证明**　由信息论中自信息量的定义 $$I(w) = -\log P(w)$$，将 $$P(w) = \text{df}(w) / N$$ 代入即得 $$I(w) = \log(N / \text{df}(w)) = \text{idf}(w)$$。TF-IDF 权重 $$\text{tf}(w,d) \times I(w)$$ 的含义是：**词项在文档中的出现次数乘以每次出现携带的信息量**，即该词项在该文档中贡献的总信息量。$$\blacksquare$$
{% endblock %}

这一结论解释了为什么 TF-IDF 在信息检索中如此成功——它本质上是在度量每个词项对文档内容的**信息贡献**。

{% hint style="warning" %}
⚠️ **易错点：TF-IDF 的变体**

TF-IDF 有多种变体，在不同的工具库和论文中可能使用不同的定义：(1) TF 的对数化：$$\text{tf}' = 1 + \log(\text{tf})$$，避免长文档中的词频主导；(2) IDF 的平滑：$$\text{idf}' = \log\bigl((N + 1) / (\text{df}(w) + 1)\bigr) + 1$$，避免除零和对罕见词的过度放大；(3) L2 归一化：对 TF-IDF 向量做 L2 归一化，使余弦相似度计算简化为内积。scikit-learn 的 `TfidfVectorizer` 默认使用平滑 IDF 和 L2 归一化。使用时需确认具体变体，否则不同工具的结果可能不一致。
{% endhint %}

## 9.2.3 分布式词表示

### 9.2.3.1 从稀疏到稠密

词袋模型和 TF-IDF 将每个词表示为一个 **one-hot 向量**——在 $$|V|$$ 维空间中，只有一个维度为 1，其余全为 0。这种表示有一个根本缺陷：**任意两个不同词的 one-hot 向量都是正交的**，无论它们的语义多么接近。例如，"猫"和"狗"的余弦相似度为 0，与"猫"和"经济学"完全相同——这显然不合理。

解决这一问题的关键洞察来自语言学家 J. R. Firth 在 1957 年的名言：

> *"You shall know a word by the company it keeps."*
> （一个词的含义由它的上下文决定。）

这就是**分布式假说**（Distributional Hypothesis）：如果两个词经常出现在相似的上下文中，它们的语义就相似。例如，"猫"和"狗"经常与"喂养""宠物""可爱"共现，因此它们的语义应该接近。

{% block type="definition" title="定义 9.9　词嵌入（Word Embedding）" %}
**词嵌入**是将词汇表中的每个词项 $$w \in V$$ 映射到一个低维稠密向量空间 $$\mathbb{R}^d$$（通常 $$d = 100$$–$$300$$）的映射函数：

$$
\phi: V \to \mathbb{R}^d, \quad w \mapsto \mathbf{v}_w \tag{9.16}
$$

好的词嵌入应满足：语义相似的词在向量空间中距离近，即 $$\text{sim}(w_1, w_2)$$ 高 $$\Leftrightarrow$$ $$\text{sim}_{\cos}(\mathbf{v}_{w_1}, \mathbf{v}_{w_2})$$ 高。
{% endblock %}

与 one-hot 向量（$$|V|$$ 维、稀疏）相比，词嵌入（$$d$$ 维、稠密）的优势是：(1) **维度低**——从 $$10^4$$–$$10^5$$ 降到 $$10^2$$–$$10^3$$；(2) **语义可比**——相似的词有相似的向量；(3) **泛化能力强**——即使训练时未见过"猫吃鱼"，如果见过"狗吃肉"，且"猫""狗"的向量接近、"鱼""肉"的向量接近，模型就能泛化。

### 9.2.3.2 Word2Vec: Skip-gram 模型

Word2Vec（Mikolov et al., 2013）是第一个大规模成功的词嵌入方法，它通过一个极其简洁的任务来学习词向量：**用中心词预测其上下文词**。

{% block type="definition" title="定义 9.10　Skip-gram 模型" %}
给定语料库中的词序列 $$w_1, w_2, \ldots, w_T$$，定义窗口大小 $$c$$，**Skip-gram** 模型的目标是最大化所有（中心词，上下文词）对的联合概率：

$$
\mathcal{L} = \sum_{t=1}^{T} \sum_{\substack{-c \leq j \leq c \\ j \neq 0}} \log P(w_{t+j} \mid w_t) \tag{9.17}
$$

其中条件概率由两组向量参数化——**中心词向量** $$\mathbf{v}_w$$ 和**上下文向量** $$\mathbf{u}_w$$：

$$
P(w_o \mid w_c) = \frac{\exp(\mathbf{u}_{w_o}^\top \mathbf{v}_{w_c})}{\sum_{w \in V} \exp(\mathbf{u}_w^\top \mathbf{v}_{w_c})} \tag{9.18}
$$
{% endblock %}

方程 (9.18) 的分母是一个对整个词汇表的求和（Softmax 归一化），计算代价为 $$O(|V|)$$——当词汇表有数十万个词时，这是不可接受的。Mikolov et al. (2013) 提出了**负采样**（Negative Sampling）来近似这个 Softmax。

$$
\log P(w_o \mid w_c) \approx \log \sigma(\mathbf{u}_{w_o}^\top \mathbf{v}_{w_c}) + \sum_{k=1}^{K} \mathbb{E}_{w_k \sim P_n(w)} \bigl[\log \sigma(-\mathbf{u}_{w_k}^\top \mathbf{v}_{w_c})\bigr] \tag{9.19}
$$

其中 $$\sigma(x) = 1/(1+e^{-x})$$ 是 Sigmoid 函数，$$K$$ 是负样本数（通常 $$K = 5$$–$$20$$），$$P_n(w) \propto \text{count}(w)^{3/4}$$ 是噪声分布。这将每次梯度更新的复杂度从 $$O(|V|)$$ 降到 $$O(K)$$。

{% block type="algorithm" title="算法 9.2　Skip-gram with Negative Sampling (SGNS)" %}
**输入**：语料库词序列 $$w_1, \ldots, w_T$$，嵌入维度 $$d$$，窗口大小 $$c$$，负采样数 $$K$$，学习率 $$\eta$$

**输出**：词向量矩阵 $$\mathbf{V} \in \mathbb{R}^{|V| \times d}$$

**步骤 1（初始化）**：随机初始化 $$\mathbf{v}_w, \mathbf{u}_w \sim \mathcal{U}(-0.5/d,\, 0.5/d)$$，$$\forall w \in V$$

**步骤 2（遍历语料）**：
$$\quad$$ **for** $$t = 1$$ **to** $$T$$ **do**
$$\quad\quad$$ **for** $$j \in [-c, c] \setminus \{0\}$$ **do**
$$\quad\quad\quad$$ 正样本：$$(w_t, w_{t+j})$$
$$\quad\quad\quad$$ 采样 $$K$$ 个负样本 $$w_1^-, \ldots, w_K^- \sim P_n(w)$$
$$\quad\quad\quad$$ 计算梯度：$$\nabla = \sigma(\mathbf{u}_{w_{t+j}}^\top \mathbf{v}_{w_t}) - 1$$
$$\quad\quad\quad$$ 更新正样本：$$\mathbf{u}_{w_{t+j}} \leftarrow \mathbf{u}_{w_{t+j}} - \eta \nabla \mathbf{v}_{w_t}$$
$$\quad\quad\quad$$ **for** $$k = 1$$ **to** $$K$$ **do**
$$\quad\quad\quad\quad$$ $$\nabla_k = \sigma(\mathbf{u}_{w_k^-}^\top \mathbf{v}_{w_t})$$
$$\quad\quad\quad\quad$$ $$\mathbf{u}_{w_k^-} \leftarrow \mathbf{u}_{w_k^-} - \eta \nabla_k \mathbf{v}_{w_t}$$
$$\quad\quad\quad$$ **end for**
$$\quad\quad\quad$$ 更新中心词：$$\mathbf{v}_{w_t} \leftarrow \mathbf{v}_{w_t} - \eta \sum \text{(对应梯度)}$$
$$\quad\quad$$ **end for**
$$\quad$$ **end for**

**步骤 3（返回）**：返回 $$\mathbf{V} = [\mathbf{v}_{w_1}, \ldots, \mathbf{v}_{w_{|V|}}]^\top$$
{% endblock %}

训练完成后，每个词获得一个 $$d$$ 维向量。这些向量具有惊人的**代数性质**——词向量之间的线性关系捕获了语义关系：

$$
\mathbf{v}_{\text{king}} - \mathbf{v}_{\text{man}} + \mathbf{v}_{\text{woman}} \approx \mathbf{v}_{\text{queen}} \tag{9.20}
$$

这意味着"king 与 man 的关系"和"queen 与 woman 的关系"在向量空间中被编码为**相同的方向**。类似的类比关系在"国家-首都""动词时态""形容词比较级"等语义维度上都成立。

### 9.2.3.3 Word2Vec 与矩阵分解的等价性

Word2Vec 看起来是一个基于神经网络的方法，但 Levy & Goldberg (2014) 揭示了它与经典矩阵分解之间的深层联系。

{% block type="theorem" title="定理 9.3　Skip-gram 与 PMI 矩阵分解的等价性" %}
设词对 $$(w, c)$$ 的**逐点互信息**（Pointwise Mutual Information, PMI）定义为：

$$
\text{PMI}(w, c) = \log \frac{P(w, c)}{P(w) P(c)} \tag{9.21}
$$

当 Skip-gram with Negative Sampling（SGNS）训练收敛时，学到的词向量满足：

$$
\mathbf{v}_w^\top \mathbf{u}_c = \text{PMI}(w, c) - \log K \tag{9.22}
$$

其中 $$K$$ 是负采样数。即 **SGNS 隐式地分解了经过偏移的 PMI 矩阵**。

**证明概要**　SGNS 的目标函数对每个词对 $$(w, c)$$ 可以写为：

$$J_{wc} = n_{wc} \log \sigma(\mathbf{v}_w^\top \mathbf{u}_c) + K \cdot n_w \cdot P_n(c) \cdot \log \sigma(-\mathbf{v}_w^\top \mathbf{u}_c)$$

对 $$\mathbf{v}_w^\top \mathbf{u}_c$$ 求导并令其为零，解得 $$\mathbf{v}_w^\top \mathbf{u}_c = \log \frac{n_{wc} \cdot |\mathcal{C}|}{n_w \cdot n_c} - \log K = \text{PMI}(w, c) - \log K$$。$$\blacksquare$$
{% endblock %}

这一结论具有深远意义：Word2Vec 并非"黑箱"——它本质上在做一件统计学家早就熟悉的事情：**估计词对的共现统计量并进行低秩近似**。PMI 度量的是两个词共现的频率是否超过了独立假设下的期望——PMI 高意味着两个词有显著的语义关联。

{% hint style="tip" %}
🚀 **前沿视野：从 Word2Vec 到 GloVe**

Word2Vec 是"预测式"模型（从局部上下文预测词），而 GloVe（Pennington et al., 2014）是"计数式"模型（直接对全局共现矩阵做加权分解）。GloVe 的目标函数为 $$\sum_{i,j} f(X_{ij})(\mathbf{v}_i^\top \mathbf{v}_j + b_i + b_j - \log X_{ij})^2$$，其中 $$X_{ij}$$ 是共现次数。定理 9.3 表明这两种看似不同的方法在数学上是等价的——都在分解 PMI 矩阵。实践中，Word2Vec 和 GloVe 在大多数任务上表现接近。
{% endhint %}

## 9.2.4 从词向量到文档向量

### 9.2.4.1 聚合策略

词嵌入给出的是每个**词**的向量，但许多任务（如文档分类、文档检索）需要整个**文档**的向量。最简单的方法是对文档中所有词的向量取平均。

{% block type="definition" title="定义 9.11　文档嵌入的聚合方法" %}
给定文档 $$d = (w_1, w_2, \ldots, w_L)$$ 及其词向量 $$\mathbf{v}_{w_1}, \ldots, \mathbf{v}_{w_L}$$，定义：

**平均词向量**：

$$
\mathbf{d}_{\text{avg}} = \frac{1}{L} \sum_{i=1}^{L} \mathbf{v}_{w_i} \tag{9.23}
$$

**TF-IDF 加权平均**：

$$
\mathbf{d}_{\text{tfidf}} = \frac{\sum_{i=1}^{L} \text{tf-idf}(w_i, d) \cdot \mathbf{v}_{w_i}}{\sum_{i=1}^{L} \text{tf-idf}(w_i, d)} \tag{9.24}
$$

TF-IDF 加权平均让重要的词（高 TF-IDF 权重）对文档向量贡献更大。
{% endblock %}

平均词向量简单且效果出奇地好——在许多文本分类任务上，它的表现接近更复杂的文档表示方法。TF-IDF 加权平均通常进一步提升效果，因为它降低了功能词的影响。

更复杂的文档嵌入方法包括 **Doc2Vec**（Le & Mikolov, 2014），它在 Skip-gram 的基础上增加一个文档向量参与预测上下文词。但在实践中，TF-IDF 加权平均的性价比通常最高。

## 9.2.5 上下文化词表示

### 9.2.5.1 静态表示的局限

Word2Vec 和 GloVe 为每个词分配**一个固定向量**，无论它出现在什么语境中。但自然语言中**一词多义**极其普遍：

- "苹果"在"苹果很甜"中指水果，在"苹果发布新手机"中指公司。
- "bank" 在 "river bank" 中指河岸，在 "bank account" 中指银行。

静态词向量只能给"苹果"一个向量——这个向量是所有含义的"平均"，无法精确表达任何一个含义。

{% block type="definition" title="定义 9.12　上下文化词表示（Contextualized Word Representation）" %}
**上下文化词表示**为每个词项 $$w_t$$ 在句子 $$S = (w_1, \ldots, w_n)$$ 中生成一个**依赖完整上下文的向量**：

$$
\mathbf{h}_t = f_\theta(w_t, w_1, \ldots, w_n) \tag{9.25}
$$

其中 $$f_\theta$$ 是参数化的编码器（如 Transformer）。**同一个词在不同句子中获得不同的向量**——"苹果"在"苹果很甜"中的向量与在"苹果发布新手机"中的向量不同。
{% endblock %}

上下文化表示的代表方法是 **BERT**（Devlin et al., 2019），它基于 Transformer 架构，通过**遮掩语言模型**（Masked Language Model）在大规模文本上预训练。BERT 的自注意力机制允许每个词"看到"句子中所有其他词，从而根据上下文动态调整自己的表示。

### 9.2.5.2 表示方法的演进总结

| 方法 | 维度 | 词序 | 语义 | 多义词 | 典型维度 |
|------|------|------|------|--------|---------|
| BoW/TF-IDF | $$\|V\|$$ | 忽略 | 弱 | 不区分 | $$10^4$$–$$10^5$$ |
| Word2Vec/GloVe | $$d$$ | 忽略 | 强 | 不区分 | 100–300 |
| ELMo | $$d$$ | 局部 | 强 | 区分 | 256–1024 |
| BERT/GPT | $$d$$ | 全局 | 极强 | 区分 | 768–1024 |

{% hint style="info" %}
🤔 **思考：如何选择文本表示方法？**

在资源充足的场景下，BERT 类预训练模型几乎在所有 NLP 任务上都优于传统方法。但在以下场景中，传统方法仍然是更好的选择：(1) **大规模信息检索**——TF-IDF + 倒排索引可以在毫秒级检索数百万文档，BERT 的推理速度无法满足这一要求；(2) **可解释性需求**——TF-IDF 的每个维度对应一个可解释的词，BERT 的 768 维向量难以解释；(3) **小数据场景**——当标注数据很少（<100 条）时，高维 BERT 表示可能过拟合，TF-IDF + 简单分类器更稳健；(4) **嵌入式设备**——Word2Vec 模型约 1.5 GB，BERT 模型约 400 MB–1.3 GB，在资源受限设备上 TF-IDF 是唯一可行的选择。
{% endhint %}

## 9.2.6 案例分析

{% block type="case" title="案例 9.2　新闻文本表示与相似度计算" %}

**问题背景**：给定一组新闻标题，使用不同的文本表示方法计算文档间的相似度矩阵，比较 BoW、TF-IDF 和平均词向量三种方法的效果差异。

**数据说明**：模拟 8 条新闻标题，覆盖体育和科技两个类别，部分标题在语义上相关但措辞不同。

**方法与步骤**：

```python
# --- 1. 导入工具 ---
import numpy as np
from collections import Counter
# --- 2. 模拟数据 ---
docs = [
    "football match championship goal victory",       # 体育
    "soccer game tournament score winning",            # 体育（同义词）
    "basketball player season team league",            # 体育
    "artificial intelligence machine learning model",  # 科技
    "deep learning neural network training",           # 科技（相关）
    "AI algorithm data prediction accuracy",           # 科技
    "football player transferred to new team",         # 体育
    "machine learning model prediction results",       # 科技
]
# --- 3. 构建词袋模型 ---
vocab = sorted(set(w for d in docs for w in d.split()))
word2idx = {w: i for i, w in enumerate(vocab)}
V = len(vocab)
bow_matrix = np.zeros((len(docs), V))
for i, d in enumerate(docs):
    for w in d.split():
        bow_matrix[i, word2idx[w]] += 1
# --- 4. 计算 TF-IDF ---
tf = bow_matrix.copy()
df = np.sum(bow_matrix > 0, axis=0)
idf = np.log(len(docs) / (df + 1e-8))
tfidf_matrix = tf * idf
# --- 5. 模拟词向量（随机 + 语义偏置）---
np.random.seed(42)
dim = 50
word_vecs = {}
sport_words = {"football", "soccer", "match", "game", "championship",
               "tournament", "goal", "score", "victory", "winning",
               "basketball", "player", "season", "team", "league",
               "transferred", "new"}
tech_words = {"artificial", "intelligence", "machine", "learning",
              "model", "deep", "neural", "network", "training",
              "AI", "algorithm", "data", "prediction", "accuracy",
              "results"}
for w in vocab:
    base = np.random.randn(dim) * 0.1
    if w in sport_words:
        base[:25] += np.random.randn(25) * 0.3 + 0.5
    elif w in tech_words:
        base[25:] += np.random.randn(25) * 0.3 + 0.5
    word_vecs[w] = base
# --- 6. 文档向量 = 平均词向量 ---
embed_matrix = np.zeros((len(docs), dim))
for i, d in enumerate(docs):
    words = d.split()
    embed_matrix[i] = np.mean([word_vecs[w] for w in words], axis=0)
# --- 7. 计算余弦相似度矩阵 ---
def cosine_sim_matrix(X):
    norms = np.linalg.norm(X, axis=1, keepdims=True) + 1e-8
    X_norm = X / norms
    return X_norm @ X_norm.T
sim_bow = cosine_sim_matrix(bow_matrix)
sim_tfidf = cosine_sim_matrix(tfidf_matrix)
sim_embed = cosine_sim_matrix(embed_matrix)
```

**结果分析**：

1. **BoW 相似度**：BoW 只能识别**字面重叠**——"football match championship goal victory"与"soccer game tournament score winning"的相似度为 0（无共享词），尽管两条标题语义高度相关。

2. **TF-IDF 相似度**：TF-IDF 本质上仍基于精确词匹配，但通过降低高频词权重、提升低频词权重，使得共享专业术语的文档相似度更高。然而，它同样无法捕获同义词关系。

3. **词向量平均相似度**：平均词向量可以捕获**语义相似性**——"football"和"soccer"在向量空间中接近，因此即使两条标题没有共享词，它们的文档向量仍然接近。同类新闻（体育/科技）内部的相似度显著高于跨类相似度。

4. **结论**：BoW 和 TF-IDF 适合**精确匹配**的场景（如关键词检索），词向量适合**语义匹配**的场景（如语义搜索、推荐）。实际系统常将两者结合——先用 TF-IDF 粗筛，再用词向量精排。

{% endblock %}

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 词袋模型（定义 9.7） | 文档表示为词频向量 $$\mathbf{x}_d \in \mathbb{R}^{\|V\|}$$，忽略词序 |
| TF-IDF（定义 9.8） | 词频 × 逆文档频率，IDF 等于自信息量（定理 9.2） |
| 余弦相似度 | 方程 (9.14)，度量两个文档向量方向的一致性 |
| 词嵌入（定义 9.9） | 稠密低维向量 $$\mathbf{v}_w \in \mathbb{R}^d$$，语义相似的词距离近 |
| Skip-gram（定义 9.10） | 用中心词预测上下文词，负采样降低计算复杂度（算法 9.2） |
| SGNS ≈ PMI 分解（定理 9.3） | Word2Vec 隐式分解偏移 PMI 矩阵 |
| 文档嵌入（定义 9.11） | 平均词向量或 TF-IDF 加权平均 |
| 上下文化表示（定义 9.12） | 同一词在不同语境中有不同向量（BERT） |
