# 9.1 文本预处理技术

第八章介绍了时间序列这种**有序数值**数据的挖掘方法。本章转向另一种无处不在但性质截然不同的数据类型：**文本**。

文本是人类知识传播和信息交流的主要载体——电子邮件、社交媒体帖子、新闻报道、学术论文、产品评论、法律文书，这些非结构化文本数据占据了企业数据存储的 80% 以上。然而，机器学习算法处理的是**数值向量**，无法直接理解自然语言。文本挖掘的核心挑战是：**如何将自然语言转化为数学对象，同时尽可能保留其语义信息？**

本节是这一转化过程的第一步——**预处理**。正如表格数据需要缺失值处理和标准化（§2.1–§2.6），时间序列需要平滑和差分（§8.2），文本数据也需要一系列清洗和规范化操作，才能成为下游分析的输入。预处理的质量直接决定了后续文本表示（§9.2）、分类（§9.3）、主题建模（§9.4）等任务的效果。

## 9.1.1 文本数据的形式化

### 9.1.1.1 从自然语言到数学对象

{% block type="definition" title="定义 9.1　文本数据的基本术语" %}
文本挖掘中的基本术语定义如下：

**语料库**（Corpus）：文本文档的集合 $$\mathcal{D} = \{d_1, d_2, \ldots, d_N\}$$，其中 $$N$$ 为文档数量。

**文档**（Document）：一段连续的文本，可以是一篇文章、一条推文或一个段落。

**词汇表**（Vocabulary）：语料库中所有不重复词项的集合 $$V = \{w_1, w_2, \ldots, w_{|V|}\}$$，其中 $$|V|$$ 为词汇表大小。

**词项**（Term）：词汇表中的一个元素——通常是一个词（word），也可以是 n-gram 或子词（subword）。

**词频**（Term Frequency）：词项 $$w$$ 在文档 $$d$$ 中出现的次数：

$$
\text{tf}(w, d) = \text{count}(w, d) \tag{9.1}
$$
{% endblock %}

文本与表格数据的根本区别在于：表格数据的特征是**预定义的**（如"年龄""收入"），而文本的"特征"是**从数据中涌现的**——词汇表 $$V$$ 由语料库本身决定，不同语料库产生不同的词汇表。这带来了两个挑战：(1) 词汇表可能非常大（数万到数百万个词）；(2) 词汇表是**稀疏的**——绝大多数文档只包含词汇表的极小子集。

### 9.1.1.2 文本预处理管线

{% block type="definition" title="定义 9.2　文本预处理管线" %}
文本预处理是将原始文本 $$d_{\text{raw}}$$ 转换为标准化的词项序列 $$d_{\text{proc}}$$ 的过程。标准的预处理管线包含以下步骤：

$$
d_{\text{raw}} \xrightarrow{\text{分词}} \mathbf{t} \xrightarrow{\text{规范化}} \mathbf{t}' \xrightarrow{\text{过滤}} \mathbf{t}'' \xrightarrow{\text{词干/还原}} d_{\text{proc}} \tag{9.2}
$$

各步骤的功能为：

1. **分词**（Tokenization）：将连续文本切分为词项序列 $$\mathbf{t} = (t_1, t_2, \ldots, t_L)$$
2. **规范化**（Normalization）：大小写统一、Unicode 规范化、标点和数字处理
3. **过滤**（Filtering）：去除停用词和低频/高频词
4. **词干提取/词形还原**（Stemming/Lemmatization）：将词项归约到基本形式
{% endblock %}

每个步骤都在**缩小词汇空间**——分词确定词汇的粒度，规范化合并同一词的不同书写形式，过滤去除无信息的词，词干提取合并同一词根的变体。这一系列操作的整体效果是**降低文本表示的维度**，同时尽可能保留语义信息。

## 9.1.2 分词

### 9.1.2.1 英文分词

英文（及大多数欧洲语言）的分词相对简单——词与词之间有**空格**作为天然分隔符。最基本的英文分词器只需按空格和标点拆分：

$$
\text{Tokenize}(\text{``Data mining is fun!''}) = [\text{``Data''}, \text{``mining''}, \text{``is''}, \text{``fun''}] \tag{9.3}
$$

然而，即使在英文中，分词也有细微的挑战：

- **缩写词**：``don't`` 应拆分为 ``do`` + ``n't`` 还是保持整体？
- **连字符词**：``state-of-the-art`` 是一个词还是四个词？
- **数字和单位**：``3.14``、``$100``、``10:30am`` 的边界在哪里？
- **网络文本**：URL、邮箱、表情符号（``:-)``, ``❤️``）需要特殊处理。

现代英文分词器（如 spaCy、NLTK 的 `word_tokenize`）使用**规则 + 异常词典**的组合策略来处理这些边界情况。

### 9.1.2.2 中文分词

中文分词面临更大的挑战：中文**没有空格分隔词语**，一串连续的汉字可能有多种切分方式。

{% block type="definition" title="定义 9.3　中文分词问题" %}
给定汉字序列 $$C = c_1 c_2 \cdots c_n$$（其中 $$c_i$$ 为单个汉字），**中文分词**是寻找最优的词序列 $$W^* = w_1 w_2 \cdots w_m$$（$$m \leq n$$），使得：

$$
W^* = \arg\max_{W} P(W \mid C) \tag{9.4}
$$

其中 $$P(W \mid C)$$ 为给定汉字串的条件下，词序列 $$W$$ 的概率。
{% endblock %}

例如，"研究生命的起源"可以切分为"研究/生命/的/起源"（研究生命的起源）或"研究生/命/的/起源"（研究生的命运），两种切分的语义完全不同。这种**歧义**是中文分词的核心难题。

**方法一：最大匹配法**

最大匹配（Maximum Matching）是最简单的基于词典的分词方法，分为正向和逆向两种。

{% block type="algorithm" title="算法 9.1　正向最大匹配分词" %}
**输入**：汉字序列 $$C = c_1 c_2 \cdots c_n$$，词典 $$\mathcal{L}$$，最大词长 $$M$$

**输出**：分词结果 $$W = [w_1, w_2, \ldots, w_m]$$

**步骤 1（初始化）**：$$W \leftarrow []$$，$$i \leftarrow 1$$

**步骤 2（贪心匹配）**：
$$\quad$$ **while** $$i \leq n$$ **do**
$$\quad\quad$$ **for** $$l = M$$ **downto** $$1$$ **do**
$$\quad\quad\quad$$ $$s \leftarrow c_i c_{i+1} \cdots c_{\min(i+l-1, n)}$$
$$\quad\quad\quad$$ **if** $$s \in \mathcal{L}$$ **then**
$$\quad\quad\quad\quad$$ $$W.\text{append}(s)$$
$$\quad\quad\quad\quad$$ $$i \leftarrow i + l$$
$$\quad\quad\quad\quad$$ **break**
$$\quad\quad$$ **end for**
$$\quad\quad$$ **if** 未匹配 **then** $$W.\text{append}(c_i)$$，$$i \leftarrow i + 1$$
$$\quad$$ **end while**

**步骤 3（返回）**：返回 $$W$$
{% endblock %}

最大匹配法简单高效（$$O(nM)$$），但无法解决歧义——它总是贪心地选择最长匹配，可能导致错误切分。

**方法二：统计分词**

统计分词将方程 (9.4) 展开为：

$$
W^* = \arg\max_{W} P(W) = \arg\max_{W} \prod_{i=1}^{m} P(w_i \mid w_{i-1}, \ldots, w_{i-k+1}) \tag{9.5}
$$

在二元语言模型（Bigram）下简化为：

$$
W^* = \arg\max_{W} \prod_{i=1}^{m} P(w_i \mid w_{i-1}) \tag{9.6}
$$

其中转移概率 $$P(w_i \mid w_{i-1})$$ 从大规模标注语料中估计。最优分词路径可以通过**维特比算法**（动态规划）在 $$O(nK)$$ 时间内求解（$$K$$ 为每个位置的候选词数）。

## 9.1.3 文本规范化

### 9.1.3.1 大小写规范化与标点处理

分词之后，需要对词项进行规范化处理，以减少词汇表的冗余。

- **大小写统一**：将所有词项转为小写。``The``、``the``、``THE`` 统一为 ``the``。但需注意专有名词（如 ``Apple`` 公司 vs ``apple`` 水果）可能因此丢失区分信息。
- **标点和特殊字符移除**：去除标点符号、HTML 标签、特殊字符等。
- **数字处理**：将所有数字替换为统一的占位符（如 ``<NUM>``），或直接移除。
- **Unicode 规范化**：统一全角/半角字符、Unicode 组合字符等。

### 9.1.3.2 停用词过滤

{% block type="definition" title="定义 9.4　停用词" %}
**停用词**（Stop Words）是在文档中高频出现但携带极少语义信息的词项。英文停用词包括冠词（a, an, the）、介词（in, on, at）、连词（and, or, but）等；中文停用词包括"的""了""在""是""和"等。

设停用词集合为 $$S$$，过滤操作定义为：

$$
\mathbf{t}'' = \{t \in \mathbf{t}' : t \notin S\} \tag{9.7}
$$

停用词列表通常包含 100–500 个词项。过滤后，词汇表大小和文档长度均显著缩小。
{% endblock %}

{% hint style="warning" %}
⚠️ **易错点：停用词过滤的副作用**

停用词过滤并非总是有益的：(1) 在**短文本**（如推文、搜索查询）中，停用词可能携带重要的语法信息——"to be or not to be" 去掉停用词后完全丧失了语义；(2) 在**情感分析**中，否定词（"not""no""don't"）是关键的情感翻转词，不应被过滤；(3) 在**精确检索**中，"the who"（乐队名）会因过滤而丢失。现代深度学习方法（如 BERT）通常**不使用停用词过滤**，而是让模型自己学习忽略无关词。
{% endhint %}

## 9.1.4 词干提取与词形还原

### 9.1.4.1 词干提取

{% block type="definition" title="定义 9.5　词干提取（Stemming）" %}
**词干提取**是将词项截断到其**词干**（Stem）的过程——通过规则去除后缀，使得同一词根的不同变体映射到同一形式：

$$
\text{stem}(\text{``running''}) = \text{stem}(\text{``runs''}) = \text{stem}(\text{``ran''}) = \text{``run''} \tag{9.8}
$$

最经典的词干提取算法是 **Porter Stemmer**（Porter, 1980），它通过约 60 条后缀删除规则（如 ``-ing`` → 去除，``-tion`` → ``-te``）逐步截断词项。
{% endblock %}

Porter Stemmer 的优势是简单快速（$$O(L)$$，$$L$$ 为词长），但存在两类错误：

- **过度提取**（Over-stemming）：不同含义的词被错误合并。如 ``university`` 和 ``universe`` 都被提取为 ``univers``。
- **提取不足**（Under-stemming）：同一含义的词未被合并。如 ``alumnus`` 和 ``alumni`` 具有不同后缀模式，Porter Stemmer 无法统一。

### 9.1.4.2 词形还原

{% block type="definition" title="定义 9.6　词形还原（Lemmatization）" %}
**词形还原**是将词项还原为其**词元**（Lemma）——词典中的标准形式：

$$
\text{lemma}(\text{``better''}) = \text{``good''}, \quad \text{lemma}(\text{``went''}) = \text{``go''} \tag{9.9}
$$

与词干提取不同，词形还原需要考虑**词性**（Part-of-Speech, POS）和**形态学规则**。例如，``saw`` 作为名词的词元是 ``saw``（锯子），作为动词的词元是 ``see``（看见）。
{% endblock %}

词形还原的输出是合法的词典词汇（而词干提取的输出可能不是真实的词，如 ``univers``），但代价是需要词典和词性标注器的支持，速度较慢。

| 维度 | 词干提取 | 词形还原 |
|------|---------|---------|
| 原理 | 规则截断后缀 | 词典查找 + 形态学 |
| 输出 | 可能不是真词 | 总是合法词汇 |
| 速度 | 快（规则匹配） | 较慢（需词典/POS） |
| 精度 | 低（会过度/不足提取） | 高（语言学正确） |
| 适用场景 | 信息检索（召回优先） | 文本分析（精度优先） |

## 9.1.5 预处理对词汇空间的影响

### 9.1.5.1 降维效果

{% block type="theorem" title="定理 9.1　预处理的词汇压缩比" %}
设原始语料库的词汇表大小为 $$|V_0|$$，经过完整预处理管线后词汇表大小为 $$|V_f|$$，则**压缩比**满足：

$$
\rho = \frac{|V_0| - |V_f|}{|V_0|} \tag{9.10}
$$

在典型的英文语料库上：
- 大小写规范化：压缩 $$5\%$$–$$15\%$$
- 停用词过滤：压缩 $$0.1\%$$–$$0.5\%$$（词汇表影响小，但文档长度显著缩短）
- 词干提取：压缩 $$20\%$$–$$40\%$$
- 综合效果：$$\rho \approx 30\%$$–$$50\%$$

**分析**　压缩比的大小取决于语料库的规模和多样性。大型语料库中，同一词根的变体更多（如科学论文中的专业术语），词干提取的压缩效果更显著。但过度压缩会导致语义损失——极端情况下，将所有词都映射到同一个词干，$$\rho = 100\%$$ 但信息全部丢失。预处理的目标是在降维和信息保留之间找到**最优折中**。$$\blacksquare$$
{% endblock %}

### 9.1.5.2 预处理对下游任务的影响

预处理步骤的选择不是"越多越好"，而应根据下游任务调整：

| 下游任务 | 推荐预处理 | 不推荐 |
|---------|----------|--------|
| 信息检索 | 词干提取、停用词去除 | — |
| 情感分析 | 保留否定词、保留标点 | 过度停用词过滤 |
| 命名实体识别 | 保留大小写 | 小写化 |
| 主题模型 | 停用词去除、低频词过滤 | — |
| BERT 微调 | 仅基本清洗 | 词干提取、停用词过滤 |

{% hint style="info" %}
🤔 **思考：预处理正在消失吗？**

早期的文本挖掘（2010 年之前）严重依赖精细的预处理——每个步骤都是手工设计的特征工程。现代预训练语言模型（如 BERT、GPT）使用**子词分词**（BPE、WordPiece），不需要词干提取或停用词过滤——模型从原始文本自动学习哪些信息重要、哪些可以忽略。预处理的角色正从"手工特征工程"转变为"基本数据清洗"（去除 HTML 标签、修复编码错误等）。然而，理解经典预处理方法仍然重要：(1) 在资源受限场景下（如嵌入式设备），轻量级的 TF-IDF + 预处理仍然是最实用的方案；(2) 理解预处理的逻辑有助于理解现代模型为什么有效。
{% endhint %}

## 9.1.6 案例分析

{% block type="case" title="案例 9.1　中文新闻文本预处理" %}

**问题背景**：中文新闻分类是 NLP 的经典任务。在将新闻文本输入分类器之前，需要进行完整的预处理——包括中文分词、停用词过滤、特殊字符清洗和词频统计。我们使用合成的中文新闻标题数据演示预处理管线。

**数据说明**：模拟 5 个类别（体育、科技、财经、娱乐、教育）的新闻标题，每类 20 条，共 100 条。

**方法与步骤**：

```python
# --- 1. 导入工具 ---
import re
import jieba
from collections import Counter
# --- 2. 模拟新闻标题 ---
titles = [
    "国足世预赛客场0比1负于日本队",
    "华为发布新一代麒麟芯片",
    "央行宣布下调存款准备金率",
    "周杰伦新专辑全球销量破百万",
    "教育部发布高考改革方案",
    # ... 更多标题
]
# --- 3. 中文停用词表 ---
stopwords = set("的了在是和有与到对等也被这那个一不".split())
# --- 4. 预处理函数 ---
def preprocess_chinese(text, stopwords):
    # 步骤 1：去除标点和特殊字符
    text = re.sub(r'[^\u4e00-\u9fa5a-zA-Z0-9]', '', text)
    # 步骤 2：分词
    words = jieba.lcut(text)
    # 步骤 3：去停用词 + 过滤单字词
    words = [w for w in words if w not in stopwords and len(w) > 1]
    return words
# --- 5. 批量处理 ---
all_words = []
for title in titles:
    words = preprocess_chinese(title, stopwords)
    all_words.extend(words)
# --- 6. 词频统计 ---
word_freq = Counter(all_words)
print("Top-10 高频词:", word_freq.most_common(10))
print(f"原始词汇表: {len(set(all_words))} 个不重复词")
```

**结果分析**：

1. **分词效果**：jieba 在新闻标题上的分词准确率约 95%。主要错误出现在专有名词（如人名、机构名）和新词（如"麒麟芯片"可能被拆分为"麒麟/芯片"）。通过添加用户词典可以解决大部分问题。

2. **词汇压缩**：原始标题的平均字符数约 15 个。经预处理后，每条标题的平均词数降至 5–7 个，停用词（如"的""在""是"）被有效去除。

3. **词频分布**：词频分布符合 **Zipf 定律**——少数高频词（如"发布""中国""改革"）出现频次远高于大量低频词。Top-10 高频词覆盖了约 15% 的总词频。

4. **类别区分**：不同类别的特征词高度不同——体育类包含"比赛""冠军""进球"，科技类包含"发布""芯片""技术"。这种词汇差异是文本分类的基础。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 语料库与词汇表（定义 9.1） | 文档集合 $$\mathcal{D}$$，词项集合 $$V$$，词频 $$\text{tf}(w, d)$$ |
| 预处理管线（定义 9.2） | 分词 → 规范化 → 过滤 → 词干/还原，逐步缩小词汇空间 |
| 中文分词（定义 9.3） | 最大匹配（贪心）或统计分词（Bigram + Viterbi） |
| 停用词（定义 9.4） | 高频低信息词，过滤可缩短文档但可能丢失语法信息 |
| 词干提取（定义 9.5） | 规则截断后缀，快速但可能不精确 |
| 词形还原（定义 9.6） | 词典查找标准形式，精确但较慢 |
| 压缩比（定理 9.1） | 预处理使词汇表缩小 30%–50%，降维与信息保留的权衡 |

**B2.** 证明词频分布的 Zipf 定律：设词频排名为 $$r$$，词频为 $$f(r)$$，则 $$f(r) \propto 1/r^\alpha$$（$$\alpha \approx 1$$）。从经验分布出发，推导 Zipf 定律对词汇表大小的约束。

**B3.** Porter Stemmer 的规则"如果词长 > 某阈值，则去除后缀 -ing"可以用什么条件来形式化？讨论阈值选择对过度提取率的影响。

**D3.** 构建一个完整的中文预处理管线（分词 → 去停用词 → 去标点 → 去低频词），在至少 500 条文本上运行。比较预处理前后的词汇表大小和文档平均长度。

### E 级（案例分析）

**E1.** 在真实的中文新闻数据集（如搜狗新闻语料）上：(1) 对 5 个类别的新闻进行预处理；(2) 统计各类别的 Top-20 特征词，分析类别区分度；(3) 比较"有预处理"和"无预处理"两种方案下 TF-IDF 向量的维度和稀疏度差异。

**E2.** 在英文产品评论数据集（如 Amazon Reviews）上：(1) 比较使用和不使用停用词过滤对情感分类准确率的影响；(2) 比较词干提取和词形还原对分类效果的差异；(3) 分析在哪些评论上预处理导致了分类错误。

### F 级（开放探讨）

**F1.** **子词分词的崛起**：BPE（Byte Pair Encoding）和 WordPiece 等子词分词方法已成为现代 NLP 的标准。它们将词拆分为更小的单元（如 ``playing`` → ``play`` + ``##ing``），从而用有限的子词词汇表覆盖任意词汇。讨论：(1) 子词分词如何解决传统词干提取和 OOV（Out-of-Vocabulary）问题？(2) 子词粒度的选择对模型性能的影响；(3) 子词分词是否使得传统预处理完全过时？

**F2.** **多语言分词的统一**：不同语言的分词规则差异巨大——英文有空格、中文无空格、日文混合假名和汉字、阿拉伯文从右到左。讨论如何设计一个**语言无关的分词器**——SentencePiece（Kudo & Richardson, 2018）的设计思路是什么？它如何在不依赖语言特定规则的情况下处理所有语言？
