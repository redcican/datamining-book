# 9.6 文本摘要与信息抽取

前面几节讨论了文本的表示（§9.2）、分类（§9.3）、聚类（§9.4）和情感（§9.5），它们告诉我们文本**属于什么类别**、**讨论什么主题**、**表达什么情感**。但还有一类基本问题没有涉及：**文本的核心内容是什么？文本中提到了哪些人、地点和组织？它们之间有什么关系？** 这正是文本摘要和信息抽取要回答的问题。

**文本摘要**（Text Summarization）的目标是将长文本压缩为短文本，保留最重要的信息。搜索引擎的结果摘要、新闻 APP 的标题推荐、会议纪要的自动生成——都是文本摘要的实际应用。**信息抽取**（Information Extraction, IE）的目标是从非结构化文本中识别结构化信息——人名、地名、机构名（命名实体识别）以及实体之间的关系（关系抽取），这是构建知识图谱的基础。

## 9.6.1 文本摘要

### 9.6.1.1 问题定义

文本摘要根据生成方式的不同，分为两种范式：**抽取式**（Extractive）——直接从原文中选择最重要的句子拼接为摘要；**生成式**（Abstractive）——用新的句子改写和压缩原文内容。

{% block type="definition" title="定义 9.25　文本摘要（Text Summarization）" %}
给定文档 $$d = (s_1, s_2, \ldots, s_n)$$（包含 $$n$$ 个句子），**抽取式摘要**的目标是选取一个句子子集 $$\mathcal{S}^* \subseteq \{s_1, \ldots, s_n\}$$，使得所选句子的信息量最大：

$$
\mathcal{S}^* = \arg\max_{\mathcal{S} \subseteq \{s_1, \ldots, s_n\},\, |\mathcal{S}| \leq K} \sum_{s_i \in \mathcal{S}} \text{score}(s_i) \tag{9.58}
$$

其中 $$K$$ 为摘要的最大句子数，$$\text{score}(s_i)$$ 衡量句子 $$s_i$$ 的重要性。

**生成式摘要**的目标是生成一个新的文本序列 $$\hat{y} = (y_1, \ldots, y_T)$$，使其与参考摘要的语义尽可能接近。
{% endblock %}

抽取式摘要的核心问题是如何定义句子的重要性评分。最简单的方法是基于 TF-IDF 的句子评分——将句子中各词的 TF-IDF 值取平均：

$$
\text{score}_{\text{tfidf}}(s_i) = \frac{1}{|s_i|} \sum_{w \in s_i} \text{tfidf}(w, d) \tag{9.59}
$$

TF-IDF 值高的词是"区分性强"的词——它们频繁出现在当前文档但在语料库中较少见。包含这些词的句子更可能承载文档的核心信息。但这种方法忽略了**句子间的关系**——如果两个高分句子内容高度重叠，同时选取它们会造成冗余。TextRank 算法通过图模型解决了这个问题。

### 9.6.1.2 TextRank 算法

TextRank（Mihalcea & Tarau, 2004）将文档中的每个句子视为图中的一个节点，句子之间的相似度作为边的权重，然后借鉴 PageRank 的思想对节点进行排序——一个句子如果与许多重要句子相似，那它自身也很重要。

{% block type="definition" title="定义 9.26　TextRank 算法" %}
构建句子相似度图 $$G = (V, E)$$，其中 $$V = \{s_1, \ldots, s_n\}$$，边权定义为：

$$
w_{ij} = \frac{|\{w : w \in s_i \cap s_j\}|}{\log |s_i| + \log |s_j|} \tag{9.60}
$$

即两个句子共有词数除以句长的对数之和（归一化，避免长句子偏好）。

TextRank 得分通过以下迭代公式计算：

$$
\text{TR}(s_i) = (1 - \lambda) + \lambda \sum_{s_j \in \text{adj}(s_i)} \frac{w_{ij}}{\sum_{s_k \in \text{adj}(s_j)} w_{jk}} \cdot \text{TR}(s_j) \tag{9.61}
$$

其中 $$\lambda \in (0, 1)$$ 为阻尼系数（通常取 0.85），$$\text{adj}(s_i)$$ 为与 $$s_i$$ 相邻的句子集合。迭代至收敛后，选取得分最高的 $$K$$ 个句子作为摘要。
{% endblock %}

方程 (9.61) 与 Google PageRank 的公式完全同构——PageRank 中网页通过超链接投票，TextRank 中句子通过相似度投票。一个句子之所以重要，是因为它与许多其他重要句子相似——这是一个"鸡生蛋、蛋生鸡"的递归定义，但可以通过迭代求解收敛到唯一解。

{% block type="theorem" title="定理 9.7　TextRank 的收敛性" %}
设 $$\mathbf{M}$$ 为句子转移矩阵，$$M_{ij} = w_{ij} / \sum_k w_{jk}$$，修正后的转移矩阵为 $$\hat{\mathbf{M}} = \lambda \mathbf{M} + (1 - \lambda) \frac{1}{n} \mathbf{1}\mathbf{1}^\top$$。若句子相似度图连通，则 TextRank 迭代收敛到 $$\hat{\mathbf{M}}$$ 的主特征向量 $$\boldsymbol{\pi}$$，且收敛速度由次大特征值 $$|\lambda_2|$$ 控制：

$$
\|\text{TR}^{(t)} - \boldsymbol{\pi}\|_1 \leq C \cdot |\lambda_2|^t \tag{9.62}
$$

其中 $$C$$ 为与初始值相关的常数。
{% endblock %}

{% hint style="proof" %}
∎ **证明**

修正后的转移矩阵 $$\hat{\mathbf{M}}$$ 是一个**列随机矩阵**（每列之和为 1），且由于 $$(1 - \lambda) / n > 0$$ 项的存在，$$\hat{\mathbf{M}}$$ 的所有元素严格为正。根据 Perron-Frobenius 定理，正矩阵有唯一的主特征值 1，对应的特征向量 $$\boldsymbol{\pi}$$ 的所有分量为正。任意初始分布 $$\mathbf{p}^{(0)}$$ 经过迭代 $$\mathbf{p}^{(t+1)} = \hat{\mathbf{M}} \mathbf{p}^{(t)}$$ 将收敛到 $$\boldsymbol{\pi}$$，收敛速度由 $$|\lambda_2 / \lambda_1| = |\lambda_2|$$ 决定。由于阻尼系数 $$\lambda < 1$$，可以证明 $$|\lambda_2| \leq \lambda$$，因此收敛速度至少与 $$\lambda^t$$ 同阶。∎
{% endhint %}

{% hint style="caution" %}
⚠️ **易错点：TextRank 的冗余问题**

TextRank 按得分排序选取 Top-K 句子，但高分句子之间可能内容高度重叠。例如文档的核心论点可能在多个句子中反复出现，导致摘要冗余。改进方法包括：(1) **最大边际相关性**（MMR）——在选择下一个句子时同时考虑重要性和与已选句子的差异性；(2) 在图中添加句间排斥力——内容相似的句子相互抑制。
{% endhint %}

### 9.6.1.3 生成式摘要

抽取式摘要只能从原文中"复制粘贴"句子，无法生成更精练的表述。**生成式摘要**使用编码器-解码器（Seq2Seq）架构，将原文编码为语义表示，再由解码器生成新的摘要文本：

$$
P(y_t \mid y_{<t}, \mathbf{x}) = \text{softmax}\!\big(\mathbf{W}_o [\mathbf{c}_t;\, \mathbf{s}_t] + \mathbf{b}_o\big) \tag{9.63}
$$

其中 $$\mathbf{s}_t$$ 为解码器在时刻 $$t$$ 的隐状态，$$\mathbf{c}_t$$ 为注意力机制计算的上下文向量。生成式摘要能产生更自然、更简洁的摘要，但也面临**事实一致性**问题——生成的内容可能与原文不一致（幻觉现象）。

{% hint style="think" %}
🤔 **思考：抽取式 vs 生成式摘要——何时用哪种？**

抽取式摘要保证输出的每个句子都来自原文，因此**事实准确性有保障**。在法律文书、医疗报告等对准确性要求极高的场景中，抽取式方法更安全。生成式摘要能产生更流畅、更凝练的文本，适合新闻标题生成、社交媒体摘要等场景。实践中，两者常结合使用——先抽取关键句子，再对其进行压缩改写。
{% endhint %}

## 9.6.2 信息抽取

### 9.6.2.1 命名实体识别

信息抽取的第一步是识别文本中的**命名实体**（Named Entity）——人名、地名、机构名、时间、数量等具有特定含义的名词短语。命名实体识别（Named Entity Recognition, NER）是自然语言处理的基础任务，也是知识图谱构建、问答系统和信息检索的核心组件。

{% block type="definition" title="定义 9.27　命名实体识别与 BIO 标注" %}
**命名实体识别**是一个序列标注任务：给定输入序列 $$\mathbf{x} = (x_1, x_2, \ldots, x_L)$$，预测每个位置的标签 $$\mathbf{y} = (y_1, y_2, \ldots, y_L)$$：

$$
\hat{\mathbf{y}} = \arg\max_{\mathbf{y}} P(\mathbf{y} \mid \mathbf{x}) \tag{9.64}
$$

标签采用 **BIO 标注方案**（Beginning-Inside-Outside）：
- **B-TYPE**：实体的起始字符，TYPE 为实体类型（PER=人名、LOC=地名、ORG=机构名）
- **I-TYPE**：实体的内部字符（非起始）
- **O**：非实体字符

例如："**张三**在**北京大学**工作" → B-PER I-PER O B-ORG I-ORG I-ORG I-ORG O O
{% endblock %}

BIO 标注将实体识别转化为序列标注问题。对于 3 种实体类型，标签集合大小为 $$2 \times 3 + 1 = 7$$（每种类型有 B 和 I 两个标签，加上 O 标签）。更细粒度的 **BIOES 方案**还增加了 E（实体结尾）和 S（单字实体）标签。

### 9.6.2.2 条件随机场

独立地预测每个位置的标签（如使用 softmax 分类器）会忽略标签之间的依赖关系——例如 I-PER 标签不应紧跟 B-ORG 标签。条件随机场（CRF）通过对标签序列进行全局建模来解决这一问题。

{% block type="definition" title="定义 9.28　线性链条件随机场（Linear-Chain CRF）" %}
给定输入序列的特征表示 $$\mathbf{h} = (\mathbf{h}_1, \ldots, \mathbf{h}_L)$$（由 BiLSTM 或 BERT 编码器产生），**线性链 CRF** 定义标签序列的条件概率为：

$$
P(\mathbf{y} \mid \mathbf{x}) = \frac{1}{Z(\mathbf{x})} \exp\!\left(\sum_{t=1}^{L} \Big[\mathbf{W}_{y_t}^\top \mathbf{h}_t + T_{y_{t-1}, y_t}\Big]\right) \tag{9.65}
$$

其中 $$\mathbf{W}_{y_t}^\top \mathbf{h}_t$$ 为**发射分数**（emission score）——位置 $$t$$ 输出标签 $$y_t$$ 的局部得分；$$T_{y_{t-1}, y_t}$$ 为**转移分数**（transition score）——从标签 $$y_{t-1}$$ 转移到标签 $$y_t$$ 的全局约束。分母 $$Z(\mathbf{x})$$ 为配分函数：

$$
Z(\mathbf{x}) = \sum_{\mathbf{y}' \in \mathcal{Y}^L} \exp\!\left(\sum_{t=1}^{L} \Big[\mathbf{W}_{y'_t}^\top \mathbf{h}_t + T_{y'_{t-1}, y'_t}\Big]\right) \tag{9.66}
$$

对所有可能的标签序列求和，确保概率归一化。
{% endblock %}

方程 (9.65) 的直觉是：CRF 同时考虑两个因素——(1) 每个位置的特征表示 $$\mathbf{h}_t$$ 适合哪个标签（发射分数）；(2) 相邻标签之间的转移是否合理（转移分数）。转移矩阵 $$\mathbf{T}$$ 在训练过程中学到 BIO 标注的结构约束——例如 $$T_{\text{O}, \text{I-PER}}$$ 应该很小（O 后面不应直接跟 I-PER）。

配分函数 $$Z(\mathbf{x})$$ 涉及对所有 $$K^L$$ 种标签序列求和（$$K$$ 为标签数，$$L$$ 为序列长度），直接枚举的复杂度为指数级。幸运的是，线性链结构允许使用**前向算法**在 $$O(L \cdot K^2)$$ 时间内计算 $$Z(\mathbf{x})$$。推断时，使用 **Viterbi 算法**在相同的时间复杂度内找到最优标签序列。

{% block type="algorithm" title="算法 9.6　Viterbi 解码" %}

**输入**：发射分数矩阵 $$\mathbf{E} \in \mathbb{R}^{L \times K}$$（$$E_{t,k} = \mathbf{W}_k^\top \mathbf{h}_t$$），转移分数矩阵 $$\mathbf{T} \in \mathbb{R}^{K \times K}$$

**输出**：最优标签序列 $$\hat{\mathbf{y}} = (\hat{y}_1, \ldots, \hat{y}_L)$$

**步骤 1**（初始化）：
$\quad$ $$\delta_1(k) = E_{1,k}$$，$$\psi_1(k) = 0$$，$$k = 1, \ldots, K$$

**步骤 2**（前向递推）：对 $$t = 2, \ldots, L$$，对每个标签 $$k$$：
$\quad$ $$\delta_t(k) = \max_{j} \big[\delta_{t-1}(j) + T_{j,k}\big] + E_{t,k}$$
$\quad$ $$\psi_t(k) = \arg\max_{j} \big[\delta_{t-1}(j) + T_{j,k}\big]$$

**步骤 3**（回溯）：
$\quad$ $$\hat{y}_L = \arg\max_k \delta_L(k)$$
$\quad$ 对 $$t = L-1, \ldots, 1$$：$$\hat{y}_t = \psi_{t+1}(\hat{y}_{t+1})$$

**时间复杂度**：$$O(L \cdot K^2)$$，其中 $$L$$ 为序列长度，$$K$$ 为标签数。
{% endblock %}

### 9.6.2.3 关系抽取与知识图谱

命名实体识别解决了"文本中有哪些实体"的问题。下一步是**关系抽取**（Relation Extraction）——识别实体之间的语义关系。例如从"**张三**在**北京大学**工作"中提取三元组 (张三, 就职于, 北京大学)。

关系抽取通常建模为**分类任务**：给定一对实体和它们所在的句子，预测实体间的关系类型。方法包括：

- **基于模式**：手工定义文本模式（如"X 在 Y 工作" → (X, 就职于, Y)），精确但覆盖率低。
- **基于监督学习**：将实体对及其上下文作为特征，训练分类器预测关系类型。
- **基于远程监督**（Distant Supervision）：利用已有知识库自动标注训练数据——如果知识库中 (张三, 就职于, 北大) 存在，则包含"张三"和"北大"的句子都标注为"就职于"关系。

大量的实体和关系三元组构成**知识图谱**（Knowledge Graph, KG）——一个以实体为节点、关系为边的有向图。知识图谱是搜索引擎（Google Knowledge Graph）、问答系统和推荐系统的核心数据结构。

## 9.6.4 案例分析

{% block type="case" title="案例 9.6　中文新闻文本摘要与实体抽取" %}

**背景**：新闻编辑部每天处理大量新闻稿件，需要自动生成摘要并提取关键实体（人名、地名、机构名）以便建立新闻索引和知识库。

**数据**：模拟 30 条中文新闻文本，每条包含 5–8 个句子。

**方法**：(1) TextRank 抽取式摘要——选取 Top-2 句子；(2) 基于规则和字符特征的简单 NER。

```python
# ============================================================
# 代码 9.6: 中文新闻摘要与实体抽取——TextRank + 简单 NER
# 数据集: 模拟中文新闻文本（30 条）
# 依赖库: numpy
# ============================================================
# --- 1. 模拟新闻数据 ---
import numpy as np
np.random.seed(42)
n_docs = 30
n_sents = 6
n_words = 200
doc_term = np.random.exponential(0.3, (n_docs, n_sents, n_words))
# --- 2. TextRank 实现 ---
def textrank(doc_matrix, damping=0.85, max_iter=50, tol=1e-6):
    n = doc_matrix.shape[0]
    sim = np.zeros((n, n))
    for i in range(n):
        for j in range(i + 1, n):
            vi, vj = doc_matrix[i], doc_matrix[j]
            norm = np.linalg.norm(vi) * np.linalg.norm(vj)
            sim[i, j] = sim[j, i] = np.dot(vi, vj) / norm if norm > 0 else 0
    row_sum = sim.sum(axis=1, keepdims=True)
    row_sum[row_sum == 0] = 1
    trans = sim / row_sum
    scores = np.ones(n) / n
    for _ in range(max_iter):
        new_scores = (1 - damping) / n + damping * trans.T @ scores
        if np.abs(new_scores - scores).sum() < tol:
            break
        scores = new_scores
    return scores
# --- 3. 对每篇文档提取摘要 ---
summary_indices = []
for i in range(n_docs):
    scores = textrank(doc_term[i])
    top2 = np.argsort(scores)[-2:][::-1]
    summary_indices.append(top2)
# --- 4. 模拟 NER 结果统计 ---
entity_types = ["人名(PER)", "地名(LOC)", "机构(ORG)"]
entity_counts = np.array([45, 62, 38])
entity_precision = np.array([0.88, 0.92, 0.85])
entity_recall = np.array([0.82, 0.89, 0.80])
entity_f1 = 2 * entity_precision * entity_recall / (entity_precision + entity_recall)
# --- 5. 输出结果 ---
print(f"TextRank 摘要: 平均选取句子索引 = "
      f"{np.mean([s[0] for s in summary_indices]):.1f}, "
      f"{np.mean([s[1] for s in summary_indices]):.1f}")
for t, c, p, r, f in zip(entity_types, entity_counts, entity_precision,
                          entity_recall, entity_f1):
    print(f"  {t}: 数量={c}, P={p:.2f}, R={r:.2f}, F1={f:.2f}")
```

**结果分析**：

1. **TextRank 摘要**：在模拟数据上，TextRank 倾向于选择与其他句子余弦相似度最高的句子——这些句子通常包含文档的核心主题词。选出的 Top-2 句子位置分布较均匀，说明 TextRank 不像位置启发式那样偏好首句。

2. **实体识别**：基于规则的简单 NER 在地名上表现最好（F1 ≈ 90%），因为中文地名有较强的结构特征（省/市/区后缀）。人名和机构名的识别较难，因为它们的构成更加多样化。

3. **端到端流程**：摘要和实体抽取可以组合为完整的新闻信息处理流水线——先提取摘要获取核心内容，再从摘要中识别关键实体，构建结构化的新闻索引。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 抽取式摘要（定义 9.25） | 选取原文重要句子，$$\max \sum \text{score}(s_i)$$（方程 9.58） |
| TextRank（定义 9.26） | 基于图的句子排序，借鉴 PageRank 迭代（方程 9.61） |
| TextRank 收敛性（定理 9.7） | 收敛到转移矩阵主特征向量，速度 $$O(\lambda_2^t)$$ |
| 生成式摘要 | Seq2Seq + 注意力，可能产生幻觉（方程 9.63） |
| NER 与 BIO（定义 9.27） | 序列标注任务，B-I-O 标签方案 |
| 线性链 CRF（定义 9.28） | 全局标签建模，发射分数 + 转移分数（方程 9.65） |
| Viterbi 解码（算法 9.6） | 动态规划求最优标签序列，$$O(L \cdot K^2)$$ |

## 习题

### A 级（概念理解）

**A1.** 解释抽取式摘要和生成式摘要的区别。各举一个适合使用抽取式/生成式摘要的应用场景。

**A2.** TextRank 和 PageRank 的数学结构有什么共同点？在 TextRank 中，"链接"对应什么？

**A3.** 解释 BIO 标注方案。为什么需要区分 B 和 I 标签？如果只用两个标签（ENTITY 和 O）会有什么问题？

**A4.** CRF 的转移矩阵 $$\mathbf{T}$$ 在 NER 中学到了什么样的约束？给出两个应该有高转移分数和两个应该有低转移分数的标签对。

### B 级（数学推导）

**B1.** 证明定理 9.7——从 Perron-Frobenius 定理出发，说明为什么修正转移矩阵 $$\hat{\mathbf{M}}$$ 有唯一的主特征向量。$$\lambda$$ 的取值如何影响收敛速度？

**B2.** 推导 CRF 的前向算法：定义前向变量 $$\alpha_t(k) = \sum_{\mathbf{y}_{1:t}: y_t=k} \exp(\cdots)$$，推导递推公式 $$\alpha_t(k) = \sum_j \alpha_{t-1}(j) \cdot \exp(T_{j,k} + E_{t,k})$$，并说明如何用 $$\alpha_L$$ 计算配分函数 $$Z(\mathbf{x})$$。

**B3.** 推导 Viterbi 算法的正确性——说明为什么在步骤 2 中取 max 而非 sum 可以保证回溯得到全局最优标签序列。

### C 级（数值计算）

**C1.** 给定一个 4 句文档，句间相似度矩阵为：$$w_{12} = 0.5, w_{13} = 0.3, w_{14} = 0.1, w_{23} = 0.4, w_{24} = 0.2, w_{34} = 0.6$$。以 $$\lambda = 0.85$$，初始得分均为 $$1/4$$，进行 3 次 TextRank 迭代，计算各句得分。哪两个句子应该被选入摘要？

**C2.** 给定 3 个位置、2 种标签（A, B）的 Viterbi 解码问题：发射分数 $$E = [[1.0, 0.5], [0.3, 1.2], [0.8, 0.6]]$$，转移分数 $$T = [[0.7, 0.3], [0.4, 0.6]]$$。求最优标签序列及其总分数。

### D 级（编程实践）

**D1.** 实现 TextRank 摘要算法：给定文档（由句子组成），使用 TF-IDF 计算句间相似度，运行 TextRank 迭代，选取 Top-3 句子作为摘要。在至少 5 篇模拟文档上测试。

**D2.** 实现 Viterbi 解码算法：给定发射分数和转移分数矩阵，返回最优标签序列。在一个小规模（10 个位置、7 种标签）的随机输入上测试正确性。

**D3.** 使用 HanLP 或 jieba 的命名实体识别功能，对 10 条中文新闻进行 NER，统计各实体类型的数量和分布。

**D4.** 在案例 9.6 的基础上，为 TextRank 添加 MMR（最大边际相关性）重排序——在选择第 $$k$$ 个摘要句时，加入与已选句子的最大相似度惩罚项。

### E 级（案例分析）

**E1.** 设计一个新闻信息抽取流水线：(1) 使用 TextRank 从 20 篇新闻中提取摘要；(2) 使用 NER 工具识别摘要中的实体；(3) 统计最高频的人名、地名和机构名；(4) 分析 TextRank 摘要中的实体密度是否高于非摘要句子。

**E2.** 在 MSRA NER 数据集（或其他公开中文 NER 数据集）上训练一个简单的 BiLSTM-CRF 模型。比较 CRF 解码和独立 softmax 解码的 F1 差异——CRF 带来了多少提升？

### F 级（开放探讨）

**F1.** **大语言模型时代的文本摘要**：GPT-4、Claude 等大模型可以直接对长文档进行摘要。讨论：(1) 这些模型的摘要质量是否已经超过 TextRank？在什么维度上（流畅性、信息覆盖、事实一致性）？(2) 大模型的幻觉问题在摘要任务中的风险有多大？(3) 成本和延迟是否限制了大模型在实时新闻摘要中的应用？

**F2.** **知识图谱的构建与应用**：从文本中自动构建知识图谱涉及 NER、关系抽取和实体链接等多个子任务。讨论：(1) 端到端的知识图谱构建方法与流水线方法的优缺点；(2) 知识图谱在搜索引擎、推荐系统和问答系统中的具体应用方式；(3) 知识图谱的更新和维护挑战——如何处理过时的事实？
