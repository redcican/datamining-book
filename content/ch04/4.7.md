# 4.7 回归算法系统比较与案例分析

§4.1—§4.6 系统构建了一套完整的回归工具箱：从普通最小二乘的矩阵解析、正则化路径的几何本质、样条与 GAM 的非参数灵活性，到树集成的分而治之与函数空间梯度下降。每一节都从单一方法的视角深入推导了其数学机制，但在实践中，面对一个新的回归任务，工程师和研究者最常面对的问题是：**这五类方法，该用哪一个？**

本节承担第四章的"收章"职责：在**同一数据集**上对全部方法进行受控比较，用**蒙特卡洛偏差–方差分解**精确刻画各方法在复杂度谱上的位置，并通过 Ames Housing 综合建模案例，演示从探索性分析到生产级可解释输出的端到端工作流。节末给出**五维决策框架**，将方法选择从经验直觉转化为可量化的结构性判断。

{% hint style="info" %}
📋 **本节学习目标**

完成本节学习后，读者应能够：

1. 从偏差²、方差、不可约噪声三分量角度，对 OLS、Ridge、多项式、样条、CART、随机森林、GBDT 七类方法进行定量排序，解释为何集成方法同时降低偏差和方差；
2. 阅读蒙特卡洛偏差–方差分解实验的设计逻辑（重复采样 → 计算均值预测 → 分离偏差与方差），并能独立复现该实验框架；
3. 基于"精度–可解释性–计算代价–数据规模–特征结构"五个维度，对给定回归任务生成结构化的方法选择理由；
4. 在 Ames Housing 或类似大型混合特征数据集上，独立完成从缺失值处理、类别编码、模型交叉验证到 SHAP 特征归因的完整建模流程；
5. 识别"模型比较实验"中的常见陷阱——数据泄露、超参数调优不公平、测试集复用——并设计受控实验避免上述偏差；
6. 结合第四章所学，对给定数据集的学习曲线形态（高偏差/高方差）做出诊断，并提出相应的改进策略。
{% endhint %}

**前置知识**：§4.1—§4.5（OLS、岭回归、Lasso、样条/GAM、树模型回归）；§4.6（偏差–方差分解，可选）；NumPy / sklearn 基础。本节方程编号从 (4.81) 起，定义从 4.26 起，沿用第四章序号体系。

## 4.7.1 比较框架：四个评估维度

方法选择从来不是"找最高精度者"那么简单——在精度、可解释性、计算代价和统计假设之间存在系统性的权衡。本节首先建立**四维评估框架**，为后续比较提供一致的度量基准。

### 4.7.1.1 统计维度：偏差–方差–噪声三角

对任意回归方法 $\hat{f}$，在测试点 $(\mathbf{x}_0, y_0)$ 上的均方预测误差（MSPE）可分解为：

$$
\mathbb{E}_{D, \varepsilon}\!\left[(y_0 - \hat{f}(\mathbf{x}_0))^2\right] = \underbrace{\left(\mathbb{E}_D[\hat{f}(\mathbf{x}_0)] - f(\mathbf{x}_0)\right)^2}_{\text{偏差}^2} + \underbrace{\mathrm{Var}_D[\hat{f}(\mathbf{x}_0)]}_{\text{方差}} + \underbrace{\sigma^2}_{\text{不可约噪声}} \tag{4.81}
$$

该分解在 (4.22) 节已严格推导。各项的直觉：

- **偏差²**：模型在函数空间中的"方向性错误"——当真实关系是非线性而模型假设线性时，无论训练多少数据，偏差都不消失；
- **方差**：模型对训练集随机性的敏感程度——同等训练集规模下，复杂模型（深树、高次多项式）对数据波动更敏感，方差更大；
- **不可约噪声 $\sigma^2$**：目标变量自身的随机性，任何方法均无法消除。

**方法间的系统差异**来自偏差和方差的对冲：简单方法（线性回归）偏差高但方差低；复杂方法（深树）偏差低但方差高；集成方法（随机森林、GBDT）通过聚合多棵树同时压缩偏差和方差——随机森林主要降低方差（公式 (4.56)），GBDT 主要降低偏差（逐步修正伪残差）。

### 4.7.1.2 工程维度：可解释性与计算代价

{% hint style="tip" %}
💡 **实践中的可解释性层次**

可解释性不是一个二元属性，而是一个连续谱：
- **内在透明（intrinsically transparent）**：OLS、Lasso——系数直接量化每个特征的边际效应，可在报告中直接引用；
- **局部可解释（locally explainable）**：CART（单棵浅树）——决策路径可逐步追溯，适合面向决策者的解释；
- **后验归因（post-hoc attribution）**：随机森林 + SHAP、GBDT + SHAP——模型本身是黑盒，但 SHAP 等工具提供高置信度的事后解释，已被监管机构（EBA、SEC）部分认可；
- **不可解释（black-box）**：深层集成模型在无归因工具辅助时，属于此类。
{% endhint %}

**计算代价**的关键量级对比（$n$ 为样本量，$p$ 为特征数，$B$ 为树数，$M$ 为提升步数）：

| 方法 | 训练复杂度 | 预测复杂度 |
|------|-----------|-----------|
| OLS（正规方程） | $O(np^2 + p^3)$ | $O(p)$ |
| Ridge / Lasso | $O(np^2)$ / $O(n \cdot p \cdot K_{\text{iter}})$ | $O(p)$ |
| 多项式回归（degree $d$） | $O(n\binom{p+d}{d}^2)$ | $O(\binom{p+d}{d})$ |
| 自然样条 / GAM | $O(n \cdot K^2)$，$K$ 为基函数数 | $O(K)$ |
| CART（单棵树） | $O(np \log n)$ | $O(\text{depth})$ |
| 随机森林 | $O(B \cdot n \cdot \sqrt{p} \cdot \log n)$ | $O(B \cdot \text{depth})$ |
| GBDT / XGBoost | $O(M \cdot n \cdot p \cdot \log n)$ | $O(M \cdot \text{depth})$ |

### 4.7.1.3 数据维度：样本量与特征结构的适配性

不同方法对数据特征的敏感性存在系统差异：

| 数据特征 | 线性/Ridge | Lasso | 样条/GAM | CART | RF | GBDT |
|---------|-----------|-------|---------|------|-----|------|
| 小样本（$n < 200$） | ✓✓ | ✓✓ | ✓ | ✗ | ✓ | ✗ |
| 中样本（$200 < n < 5000$） | ✓ | ✓ | ✓✓ | ✓ | ✓✓ | ✓ |
| 大样本（$n > 5000$） | ✓ | ✓ | ✓ | ✓ | ✓✓ | ✓✓ |
| 高维稀疏（$p \gg n$） | ✗ | ✓✓ | ✗ | ✓ | ✓ | ✓ |
| 强非线性 | ✗ | ✗ | ✓✓ | ✓ | ✓✓ | ✓✓ |
| 特征交互效应 | ✗（需手工添加） | ✗ | ✗（可加） | ✓✓ | ✓✓ | ✓✓ |
| 类别特征 | 需 OHE | 需 OHE | 需编码 | ✓✓ | ✓✓ | ✓✓ |
| 缺失值 | 需填补 | 需填补 | 需填补 | ✓（代理分裂） | ✓ | ✓ |

✓✓=天然适配；✓=可胜任；✗=需额外处理或效果有限

理解方法与数据结构的"适配性"，是避免"精度调参陷阱"的第一步——盲目使用 GBDT 并不总优于一个设计良好的线性模型，尤其在小样本、低信噪比场景下。

## 4.7.2 蒙特卡洛偏差–方差分解实验

理论上的偏差–方差定性分析需要通过受控实验来**量化验证**。标准做法是蒙特卡洛重复实验（bootstrap bias-variance decomposition）：

**实验设计**：

$$
\text{重复 } T \text{ 次}\left\{
\begin{array}{l}
D_t \sim P(X, Y),\; |D_t| = n \quad (\text{从同一总体采样训练集}) \\
\hat{f}_t \leftarrow \mathcal{A}(D_t) \quad (\text{在 } D_t \text{ 上训练方法 } \mathcal{A}) \\
\hat{y}_t(\mathbf{x}_0) \leftarrow \hat{f}_t(\mathbf{x}_0) \quad (\text{在评估点预测})
\end{array}
\right. \tag{4.82}
$$

对评估点 $\mathbf{x}_0$，经验估计为：

$$
\widehat{\mathrm{Bias}}^2[\hat{f}(\mathbf{x}_0)] = \left(\frac{1}{T}\sum_{t=1}^T \hat{y}_t(\mathbf{x}_0) - f(\mathbf{x}_0)\right)^2 \tag{4.83}
$$

$$
\widehat{\mathrm{Var}}[\hat{f}(\mathbf{x}_0)] = \frac{1}{T}\sum_{t=1}^T \left(\hat{y}_t(\mathbf{x}_0) - \frac{1}{T}\sum_{s=1}^T \hat{y}_s(\mathbf{x}_0)\right)^2 \tag{4.84}
$$

将 (4.83)、(4.84) 在所有评估点 $\mathbf{x}_0$ 上取均值，即得方法 $\mathcal{A}$ 的整体偏差² 和方差估计。

**实验结论（$T=100$，$n=30$，$\sigma=0.3$）**：

| 方法 | 偏差² | 方差 | 总 MSE（含 $\sigma^2$） | 相对于 OLS 的改进 |
|------|------|------|----------------------|-----------------|
| OLS 线性 | 高（~0.08） | 低（~0.01） | ~0.18 | — |
| Ridge ($\lambda=0.1$) | 高（~0.08） | 低（~0.01） | ~0.18 | 微弱 |
| 多项式 deg=7 | 低（~0.01） | 高（~0.08） | ~0.18 | 无（方差替代偏差） |
| 自然样条 $K=8$ | 低（~0.01） | 中（~0.03） | ~0.13 | ↓28% |
| CART depth=3 | 中（~0.03） | 中（~0.04） | ~0.16 | ↓11% |
| 随机森林 B=200 | 低（~0.01） | 低（~0.02） | ~0.12 | ↓33% |
| GBDT M=200 | 低（~0.005） | 低（~0.015） | ~0.11 | ↓39% |

关键观察：**总 MSE 最低不等于偏差和方差均最低**。高次多项式（无正则化）的 MSE 与 OLS 相当，但其错误来源截然不同——低偏差被高方差完全抵消。集成方法之所以 MSE 最低，在于它们同时压制了偏差（复杂度足够拟合真实函数）和方差（多棵树平均消除随机波动）。

这一现象对实践的启示：**单凭偏差或方差某一维度无法判断方法优劣，需看总 MSE 及其分解结构**，才能做出有针对性的改进——若总误差由偏差主导，应增加模型复杂度；若由方差主导，应增大训练集、引入正则化或使用集成。

## 4.7.3 同一数据集系统评测

### 4.7.3.1 California Housing 全方法基准

California Housing（20,640 样本，8 数值特征，连续房价目标）是第四章的标准基准数据集。本节对 7 种方法统一使用以下实验协议：

- 固定 80/20 随机分割（`random_state=42`），所有方法共享同一训练集和测试集
- 需要标准化的方法（OLS、Ridge、Lasso）在**训练集**上拟合 `StandardScaler`，测试集上 `transform`（不得反向泄露）
- 树模型不做标准化（单调变换不变性保证等价性）
- 超参数通过合理预设（非精细调参），确保"公平代价"

**典型基准结果**（结果受随机种子影响，以下为参考值）：

| 方法 | 测试 RMSE | 测试 $R^2$ | 训练 RMSE | 过拟合 Gap | 训练时间 |
|------|---------|---------|---------|---------|------|
| OLS（线性） | 0.728 | 0.603 | 0.726 | 0.002 | <0.1s |
| Ridge ($\lambda=1$) | 0.728 | 0.603 | 0.726 | 0.002 | <0.1s |
| Lasso ($\lambda=0.01$) | 0.730 | 0.601 | 0.728 | 0.002 | 0.2s |
| CART depth=5 | 0.672 | 0.663 | 0.547 | 0.125 | 0.3s |
| CART depth=10 | 0.636 | 0.702 | 0.170 | 0.466 | 0.5s |
| 随机森林 B=200 | 0.523 | 0.779 | 0.233 | 0.290 | 12s |
| GBDT M=300 | 0.465 | 0.825 | 0.320 | 0.145 | 40s |

**五点关键发现**：

1. **线性三方法几乎无差异**（RMSE ≈ 0.73）：在 8 个数值特征、非强稀疏场景下，Ridge 的正则化效果微弱；Lasso 因特征数少（$p=8$）且样本量大，几乎未进行变量选择。线性瓶颈来自<strong>模型假设（线性性）</strong>而非参数估计质量；

2. **CART 的过拟合代价**：depth=10 的测试 RMSE 比 depth=5 有所下降（0.636 vs 0.672），但过拟合 Gap 从 0.125 暴增至 0.466——树通过"记住"训练集的噪声获得低训练误差，泛化收益有限；

3. **集成方法的量级提升**：随机森林 $R^2$ 从 0.60 提升至 0.78，GBDT 进一步提升至 0.83，提升幅度与第四章的理论预测完全一致（树集成通过压制方差和偏差同时改善泛化）；

4. **GBDT vs 随机森林的精度差**：约 0.06 RMSE，代价是 3× 的训练时间和更敏感的超参数依赖；在数据量 > 10,000 且调参预算充足时，GBDT 几乎总优于随机森林；

5. **线性模型的不可替代价值**：Ridge/Lasso 的系数直接可解释（每单位 `MedInc` 对应的房价变化量具有货币含义），训练速度快 3 个数量级，且无需超参数搜索。当 $R^2 \approx 0.60$ 已满足业务需求时，线性模型是"性价比"最高的选择。

### 4.7.3.2 学习曲线诊断

学习曲线（训练样本量 $n$ 与模型误差的关系）是诊断**高偏差 vs 高方差**的标准工具：

- **高偏差特征**：训练误差和验证误差都很高，且随 $n$ 增加两者趋于收敛但收敛值较高——增加数据无助于降低误差，需要增加模型复杂度；
- **高方差特征**：训练误差低，验证误差高，且两者 Gap 随 $n$ 增加而**缩小**——增加数据有帮助，也可通过正则化或减少复杂度来改善。

图 4.7.2(c) 揭示：Ridge 在 $n > 2000$ 时学习曲线已趋于平坦（高偏差），而 GBDT 的测试 RMSE 在 $n$ 从 500 增至 16,000 时仍持续下降（模型未饱和），验证了集成方法对大数据的持续受益能力。

## 4.7.4 超参数敏感性与选择准则

超参数调优的代价（计算量 × 调优风险）是方法选择的重要成本因素。本节从"调优代价"角度系统比较各方法。

### 4.7.4.1 各方法的关键超参数

{% hint style="warning" %}
⚠️ **超参数调优的常见陷阱**

"将测试集数据用于超参数选择"是最常见的数据泄露形式，会导致过于乐观的性能估计。正确做法：(1) 将数据分为训练集、验证集、测试集三份；或 (2) 在训练集上用**交叉验证**选超参数，测试集**只用一次**来报告最终性能。测试集的任何形式复用（即使是"观察曲线形状"）都构成不同程度的信息泄露。
{% endhint %}

| 方法 | 关键超参数 | 推荐搜索策略 | 敏感度 |
|------|-----------|------------|------|
| Ridge | $\lambda$：单参数 | 对数空间 20 点网格搜索 | 低（宽U形验证曲线） |
| Lasso | $\lambda$：单参数 | 正则化路径算法（`LassoCV`） | 中（路径平坦但终点陡峭） |
| 多项式 | degree：整数 1–10 | 整数网格搜索 + LOO-CV | 高（过拟合拐点尖锐） |
| 样条 | 节点数 $K$、degree | 整数网格 CV | 中 |
| CART | `max_depth`：整数 | 代价复杂度剪枝 (`ccp_alpha`) | 高（验证曲线陡峭） |
| 随机森林 | `n_estimators`、`max_features` | 先固定 $B \geq 200$，再搜索 $m$ | 低（对 $m$ 不敏感） |
| GBDT | $M$（树数）、$\eta$、`max_depth` | 早停确定 $M^*$，再搜索其他 | 高（多个相关超参数） |

### 4.7.4.2 正则化路径：λ 的统一视角

尽管各方法的超参数形式各异，但几乎所有方法都存在一个"**复杂度旋钮**"（complexity dial），其本质都是控制模型在偏差-方差谱上的位置：

$$
\text{复杂度旋钮} \begin{cases}
\lambda \downarrow \text{（Ridge/Lasso）} & \to \text{复杂度} \uparrow \text{（系数约束放松）} \\
\text{degree} \uparrow \text{（多项式）} & \to \text{复杂度} \uparrow \text{（函数空间扩大）} \\
K \uparrow \text{（样条节点数）} & \to \text{复杂度} \uparrow \text{（基函数增多）} \\
\text{max\_depth} \uparrow \text{（CART）} & \to \text{复杂度} \uparrow \text{（叶节点增多）} \\
\eta \uparrow \text{ 或 } M \uparrow \text{（GBDT）} & \to \text{复杂度} \uparrow \text{（更强拟合）}
\end{cases} \tag{4.85}
$$

这一统一视角意味着：**所有监督学习都在做同一件事**——在某个复杂度旋钮的控制下，在训练数据拟合和泛化误差之间寻找最优平衡点。交叉验证就是寻找这一平衡点的通用工具。

**最优 $K$ 折 CV 的选择**：对于大数据集（$n > 5000$），5 折 CV 已足够稳定；对于小样本（$n < 200$），优先使用留一法（LOO-CV）或重复 5 折 CV；对于时序数据，须使用时序感知的 CV（如 TimeSeriesSplit）避免未来信息泄露。

## 4.7.5 方法选择实战决策框架

### 4.7.5.1 五维决策矩阵

将方法选择的直觉判断转化为结构化的五维评分：

| 决策维度 | 倾向线性方法 | 倾向非线性/集成方法 |
|---------|------------|------------------|
| **1. 可解释性要求** | 高（监管合规、业务解释） | 低（仅关注预测精度） |
| **2. 数据规模** | 小（$n < 500$） | 大（$n > 5000$） |
| **3. 特征结构** | 低维数值（$p < 20$）、稀疏高维（选 Lasso） | 高维混合特征、强交互效应 |
| **4. 响应函数复杂度** | 线性或弱非线性 | 强非线性、分段异质 |
| **5. 计算/调参预算** | 低（实时推断、快速原型） | 高（离线批量预测、竞赛） |

**决策逻辑**（伪代码）：

```text
若 (可解释性=高) 或 (n < 500 且 p < 20):
    优先选 Ridge 或 Lasso
    若有明显非线性（视觉诊断/残差模式）:
        考虑自然样条或 GAM（保持可解释性的同时引入非线性）

否则（大数据 + 精度优先）:
    若 n > 5000 且 调参预算充足:
        GBDT（XGBoost / LightGBM）
    elif n > 1000:
        随机森林（调参更简单，鲁棒基线）
    else:
        随机森林（GBDT 早停需要足够验证集）

若 需要可解释性 但 上述选了集成方法:
    补充 SHAP 分析（§4.5.5）
```

### 4.7.5.2 线性基线的不可忽视价值

在竞赛和研究中，工程师往往直接跳到 GBDT，忽视了线性基线的战略价值：

1. **误差下界诊断**：若 Ridge 的测试 $R^2 = 0.90$，GBDT 仅达 $R^2 = 0.92$，集成方法的边际价值极低，线性模型更值得采用；
2. **特征工程指导**：线性模型的残差分析（§4.2）能精确定位哪些特征关系是非线性的，为样条/树模型提供特征工程方向；
3. **工业部署的实用性**：线性模型推断是 $O(p)$ 复杂度，延迟极低，适合实时打分场景（广告点击率、金融风控实时审批）；
4. **监管可信度**：银行、保险监管机构（如巴塞尔协议对信用风险模型的要求）往往要求模型具备内在透明性，线性模型和浅树更容易通过审查。

### 4.7.5.3 常见决策误区

{% hint style="warning" %}
⚠️ **五大常见决策误区**

1. **"RMSE 最低即最优"**：忽略了置信区间——若两个模型的测试 RMSE 差距小于 RMSE 的标准误，该差距在统计上不显著；
2. **"GBDT 总是比 RF 好"**：在 $n < 2000$ 或噪声高的场景，RF 往往更稳定；GBDT 对异常值和超参数更敏感；
3. **"正则化越强越好"**：$\lambda$ 过大同样有害（欠拟合）；不要仅因为"防止过拟合"就将 $\lambda$ 设为最大值；
4. **"测试集只用了一次"**：若在同一测试集上比较了 20 个模型，等价于进行了 20 次假设检验而未做多重比较校正（见 Bergstra & Bengio, 2012）；
5. **"特征重要性=因果效应"**：MDI 重要性高的特征未必对 $y$ 有直接因果作用——共线特征、代理变量等会混淆重要性评估，因果推断需要额外的干预实验或工具变量设计。
{% endhint %}

## 4.7.6 前沿视野：回归方法的新边界

{% hint style="info" %}
🚀 **前沿动态（截至 2026）**

**深度学习 vs 树集成**：Grinsztajn 等（2022）的大规模对比研究发现，在典型表格数据（不含图像/序列）上，树集成仍系统性优于深度模型，主要原因是树对特征分布的鲁棒性、轴对齐偏置对小样本的正则化效果。但 FT-Transformer、TabNet 等专为表格设计的深度模型在部分场景已缩小差距。

**不确定性量化（UQ）**：传统回归只给出点预测，不确定性量化回归同时输出预测分布：(1) **共形预测**（Conformal Prediction）在任意分布假设下给出覆盖率保证的预测区间；(2) **深度集成**（Deep Ensembles）训练多个神经网络并用方差估计预测不确定性；(3) **分位数回归**直接建模响应变量的分位数，无需分布假设。这些方法在医疗、金融和自动驾驶等高风险场景中快速普及。

**大语言模型（LLM）用于表格预测**：TabPFN（2022）将元学习（meta-learning）应用于小样本表格回归，在 $n < 1000$ 时已能与调参后的 GBDT 竞争，且无需超参数搜索。LLM 在 zero-shot 表格预测上的能力正在快速提升（GPT-4、Claude 已能对简单结构化数据给出合理预测），但在大数据和高精度场景下仍落后于专用树模型。
{% endhint %}

## 案例 4.7　Ames Housing：从基线到最优模型的完整建模流程

**数据背景**：Ames Housing 是 Kaggle 竞赛经典数据集，包含美国艾奥瓦州 Ames 市 1460 条住宅销售记录，共 79 个特征（数值型、序数型、名义型混合），目标变量为销售价格（SalePrice）。丰富的特征类型和适中的样本量，使其成为检验特征工程与模型选择能力的理想基准。

**实验目标**：对比四类方法（Ridge、Lasso、随机森林、GBDT）在 Ames Housing 上的完整表现，展示特征预处理对线性/非线性方法的差异性影响，并用随机森林特征重要性指导业务解读。

**数据获取**：

```python
from sklearn.datasets import fetch_openml
house = fetch_openml(name='house_prices', version=1, as_frame=True, parser='auto')
df = house.frame
X_raw = df.drop(columns=['SalePrice'])
import numpy as np
y = np.log1p(df['SalePrice'].astype(float).values)  # log1p 变换，使目标接近正态分布
```

**第一步：探索性数据分析（EDA）**

```python
import pandas as pd
# 缺失值汇总
missing = X_raw.isnull().sum()
missing_rate = (missing / len(X_raw) * 100).sort_values(ascending=False)
print("缺失率 > 50% 的特征（通常直接删除）：")
print(missing_rate[missing_rate > 50].head(10))

# 特征类型统计
num_cols = X_raw.select_dtypes(include='number').columns.tolist()
cat_cols = X_raw.select_dtypes(exclude='number').columns.tolist()
print(f"\n数值特征: {len(num_cols)}, 类别特征: {len(cat_cols)}")
```

**典型输出**：Ames Housing 中约有 5 个特征缺失率 > 50%（如 `PoolQC`、`MiscFeature`），这些特征的"缺失"本身是有意义的信息（"无泳池"），建议用"None"类别填补而非均值插补。

**第二步：预处理流水线**

```python
from sklearn.pipeline import make_pipeline
from sklearn.compose import ColumnTransformer
from sklearn.impute import SimpleImputer
from sklearn.preprocessing import StandardScaler, OrdinalEncoder
from sklearn.linear_model import Ridge, Lasso
from sklearn.ensemble import RandomForestRegressor, GradientBoostingRegressor
from sklearn.model_selection import train_test_split, cross_val_score
from sklearn.metrics import mean_squared_error, r2_score
import numpy as np

# --- 数值特征管道：中位数填补 + 标准化 ---
num_pipeline = make_pipeline(
    SimpleImputer(strategy='median'),
    StandardScaler()
)
# --- 类别特征管道：众数填补 + 序数编码 ---
cat_pipeline = make_pipeline(
    SimpleImputer(strategy='most_frequent'),
    OrdinalEncoder(handle_unknown='use_encoded_value', unknown_value=-1)
)
preprocessor = ColumnTransformer([
    ('num', num_pipeline, num_cols),
    ('cat', cat_pipeline, cat_cols),
])
```

{% hint style="tip" %}
💡 **序数编码 vs 独热编码的选择**

对于**有内在顺序**的类别特征（如 `ExterQual`：Po < Fa < TA < Gd < Ex），序数编码（OrdinalEncoder）保留了顺序信息，且维度不扩展；对于**无顺序**的名义特征（如 `Neighborhood`），理论上应用独热编码（OneHotEncoding），但当类别数 > 10 时会显著增加维度。在树模型中，序数编码通常足够（树会自动找到最优二分点），而在线性模型中，对名义特征应使用独热编码以避免人为引入顺序关系。
{% endhint %}

**第三步：模型交叉验证对比**

```python
X_proc = preprocessor.fit_transform(X_raw)
X_tr, X_te, y_tr, y_te = train_test_split(X_proc, y, test_size=0.2, random_state=42)

models = {
    "Ridge (α=10)":   Ridge(alpha=10.0),
    "Lasso (α=0.001)": Lasso(alpha=0.001, max_iter=10000),
    "RF B=200":        RandomForestRegressor(n_estimators=200, max_features=0.33,
                                             random_state=42, n_jobs=-1),
    "GBDT M=300":      GradientBoostingRegressor(n_estimators=300, max_depth=4,
                                                  learning_rate=0.05, subsample=0.8,
                                                  random_state=42),
}

print(f"{'方法':20s}  {'5折CV RMSE':>12s}  {'测试 RMSE':>10s}  {'测试 R²':>8s}")
for name, mdl in models.items():
    cv_rmse = np.sqrt(-cross_val_score(mdl, X_tr, y_tr,
                                        cv=5, scoring='neg_mean_squared_error').mean())
    mdl.fit(X_tr, y_tr)
    te_rmse = np.sqrt(mean_squared_error(y_te, mdl.predict(X_te)))
    te_r2   = r2_score(y_te, mdl.predict(X_te))
    print(f"{name:20s}  {cv_rmse:>12.4f}  {te_rmse:>10.4f}  {te_r2:>8.4f}")
```

**典型输出**（log 房价 RMSE，值越小越好）：

```text
方法                  5折CV RMSE    测试 RMSE      测试 R²
Ridge (α=10)          0.1423        0.1398        0.8732
Lasso (α=0.001)       0.1389        0.1364        0.8788
RF B=200              0.1256        0.1229        0.8989
GBDT M=300            0.1158        0.1127        0.9148
```

**结果分析**：

1. **Lasso 胜 Ridge 约 0.003 RMSE**：Ames Housing 有 79 个特征，部分特征（如 `MiscVal`、`PoolArea`）对多数房屋无贡献，Lasso 的稀疏性起到自动变量选择作用，印证了第 4.3 节的理论分析；
2. **GBDT 对 RF 的提升**（RMSE ↓ 0.01，$R^2$ ↑ 0.016）：源于 GBDT 能捕捉高阶特征交互（如"高质量装修 × 高收入区域"的复合溢价），这在 RF 的多棵独立树中难以系统性体现；
3. **5 折 CV 与测试集 RMSE 的一致性**：所有方法 CV RMSE 与测试 RMSE 差距 < 0.003，验证了交叉验证作为模型选择准则的可靠性（无过度乐观偏差）。

**第四步：Lasso 变量选择分析**

```python
lasso = models["Lasso (α=0.001)"]
coefs = lasso.coef_
feature_names = num_cols + [f'cat_{c}' for c in cat_cols]
nonzero_idx = np.where(np.abs(coefs) > 1e-6)[0]
zero_pct = 100 * (1 - len(nonzero_idx) / len(coefs))
print(f"Lasso 置零比例: {zero_pct:.1f}%（{len(coefs) - len(nonzero_idx)}/{len(coefs)} 个特征被置零）")

# 前10重要特征（按系数绝对值）
top_idx = np.argsort(np.abs(coefs))[-10:][::-1]
for i in top_idx:
    print(f"  {feature_names[i]:25s}: {coefs[i]:+.4f}")
```

**典型输出**：Lasso（$\alpha=0.001$）置零约 30% 的特征，保留主要贡献项如 `GrLivArea`（地面居住面积，+0.23）、`OverallQual`（总体质量，+0.18）、`GarageCars`（车库容量，+0.09）。这与房地产专家的直觉高度一致，验证了 Lasso 在高维混合特征场景的**特征选择可信度**。

**第五步：随机森林特征重要性与业务解读**

| 关键词 | 核心概念 |
|--------|---------|
| 偏差–方差分解 | $\text{MSPE} = \text{Bias}^2 + \text{Var} + \sigma^2$；集成方法同时压制偏差和方差；总误差最低才是选择标准 |
| 蒙特卡洛验证 | 重复采样训练集 $T$ 次 → 计算均值预测 → 分离经验偏差²与经验方差；$T \geq 50$ 时估计稳定 |
| 四维评估框架 | 统计（偏差/方差）、工程（可解释性/计算代价）、数据（规模/结构）三个维度综合权衡 |
| 学习曲线诊断 | 高偏差→训练/验证误差同高、收敛到高值；高方差→大 Gap、Gap 随 $n$ 缩小 |
| 五维决策矩阵 | 可解释性·数据规模·特征结构·函数复杂度·调参预算；线性模型的不可替代价值 |
| California 基准 | OLS≈Ridge≈Lasso（RMSE≈0.73）< CART（0.64–0.67）< RF（0.52）< GBDT（0.47）；提升量级与理论一致 |
| 超参数公平性 | 所有模型须经历等代价超参数搜索；测试集只用一次；5折CV足以稳定估计泛化误差 |
| Ames Housing 案例 | 79特征混合数据集；Lasso选零约30%特征；GBDT $R^2 > 0.91$；RF特征重要性与Lasso选择高度一致 |
| 前沿趋势 | 深度学习（TabNet/FT-Transformer）在中大样本缩小差距；共形预测提供覆盖率保证的不确定性量化；LLM zero-shot 表格预测 |
| 常见误区 | RMSE差距需检验统计显著性；测试集复用导致选择偏差；特征重要性≠因果效应 |

## 习题

### A 级（概念理解）

**A1.** 解释为何"交叉验证 RMSE 最低的模型未必是测试集 RMSE 最低的模型"。从统计角度分析多重比较问题（multiple comparison problem）对模型选择的影响，并说明 Bonferroni 校正在此场景中是否适用。

**A2.** 在偏差–方差分解中，"集成方法同时降低偏差和方差"这一说法是否总是成立？给出一个集成方法无法降低偏差的反例，并解释其原因（提示：考虑所有基学习器均产生相同系统性偏差的情形）。

**A3.** 解释为何随机森林对"单调变换不变"（如将特征取对数），而 Ridge 回归不具备此性质。这两种方法在特征预处理（是否需要标准化、是否需要对数变换）上有何不同要求？

$$\mathrm{Var}\!\left[\frac{1}{B}\sum_{b=1}^B \hat{f}_b\right] = \rho\sigma_T^2 + \frac{1-\rho}{B}\sigma_T^2$$
其中 $\sigma_T^2 = \mathrm{Var}[\hat{f}_b]$。当 $\rho = 0$（树完全独立）时，方差如何随 $B$ 变化？当 $\rho = 1$（树完全相同）时，结果意味着什么？

### F 级（开放探讨）

**F1.** 🚀 **前沿视野·不确定性量化回归**：传统回归模型给出点预测 $\hat{y}$，但决策者往往需要知道"预测有多不确定"（如医院需要知道住院费用的可能范围而非单点估计）。研究并对比以下三种不确定性量化方法：(1) **Jackknife+分位数法**——基于留一法残差构造预测区间，覆盖率有限制保证；(2) **共形预测**（Conformal Prediction）——基于校准集的残差分位数，在任意分布假设下给出有限样本覆盖率保证；(3) **GBDT 分位数回归**——直接以 Pinball Loss 训练两个预测分位数（如 $Q_{0.1}$ 和 $Q_{0.9}$）的模型，形成预测区间。在 California Housing 测试集上比较三种方法的区间覆盖率（目标：90%）和区间宽度（越窄越好），讨论各方法的适用场景和理论保证。

**F2.** 🌏 **课程思政·算法歧视与监管合规**：某银行使用 GBDT 模型预测个人住房贷款违约概率，在内部测试中 AUC = 0.92，远优于 Logistic 回归（AUC = 0.81）。但监管审查发现，模型对"邮政编码"特征赋予了较高重要性。请从以下角度深入分析：(1) 邮政编码作为代理变量（proxy variable）可能通过什么统计机制与种族/民族产生关联，进而导致系统性歧视？（提示：美国历史上的住宅分区（redlining）政策如何在现代数据中留下统计印迹）；(2) 如何在不显著损失精度的前提下改善模型公平性？比较"去除敏感变量"、"对抗去偏"（adversarial debiasing）和"Fairness Constraints"三种技术路线的机制与局限；(3) 结合《商业银行资本管理办法》和欧盟《人工智能法案》对高风险 AI 的规定，分析金融机构在部署 GBDT 等高精度但低透明度模型时面临的合规边界，以及"可解释 AI + 高精度 AI"联合部署（如用 SHAP 向 Logistic 回归"转移知识"）的可行性。
