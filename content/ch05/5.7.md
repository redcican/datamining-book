# 5.7 高维数据聚类的挑战与方法

前六节的聚类方法主要在低维（$$d \leq 10$$）数据上展示。然而，现实数据往往是**高维**的：文本的词频向量可达数万维，基因表达谱有数千个基因，图像的像素空间可达数十万维。当维度 $$d$$ 显著增大时，聚类算法的表现会急剧退化——这被称为**维度诅咒**（Curse of Dimensionality）。

本节从数学上严格分析维度诅咒的两个核心现象（体积集中和距离集中），然后介绍应对策略：特征选择、降维+聚类管线和子空间聚类方法。

完成本节学习后，读者应能够：

1. 从数学上证明高维超球体积趋于零的定理（定理 5.13），并解释其对聚类的影响；
2. 证明高维空间中距离集中现象（定理 5.14）：最远与最近距离之比趋于 1；
3. 使用特征选择方法（方差过滤、相关性过滤）降低维度；
4. 构建 PCA + K-means 降维聚类管线，并理解降维对聚类的影响；
5. 区分 t-SNE/UMAP 用于可视化与用于聚类的适用性和局限性；
6. 理解谱聚类（Spectral Clustering）的基本原理——从相似度矩阵到图拉普拉斯到特征分解（定义 5.34）；
7. 在高维数据集（如 MNIST）上完成"降维→聚类→评估"的完整工作流。
{% endhint %}

## 5.7.1 维度诅咒的数学分析

### 5.7.1.1 高维空间的反直觉现象

在低维空间中形成的几何直觉——"远近有别""球内密集""边界清晰"——在高维中全部失效。以下两个定理揭示了根本原因。

{% block type="definition" title="定义 5.32　$$d$$ 维单位超球" %}
$$d$$ 维空间中以原点为圆心、半径为 $$r$$ 的**超球**定义为：

$$
B_d(r) = \{\mathbf{x} \in \mathbb{R}^d : \|\mathbf{x}\| \leq r\} \tag{5.89}
$$

其体积为：

$$
V_d(r) = \frac{\pi^{d/2}}{\Gamma(d/2 + 1)} \cdot r^d \tag{5.90}
$$

其中 $$\Gamma(\cdot)$$ 为 Gamma 函数（$$\Gamma(n+1) = n!$$ 对正整数成立）。
{% endblock %}

{% block type="theorem" title="定理 5.13　高维单位超球体积趋于零" %}
$$d$$ 维单位超球（$$r=1$$）的体积 $$V_d(1)$$ 在 $$d \to \infty$$ 时趋于 0：

$$
\lim_{d \to \infty} V_d(1) = \lim_{d \to \infty} \frac{\pi^{d/2}}{\Gamma(d/2 + 1)} = 0 \tag{5.91}
$$

**证明**：

利用 Stirling 公式 $$\Gamma(n+1) \approx \sqrt{2\pi n} (n/e)^n$$，当 $$d$$ 为偶数时：

$$
V_d(1) = \frac{\pi^{d/2}}{(d/2)!} \approx \frac{\pi^{d/2}}{\sqrt{\pi d} (d/(2e))^{d/2}} = \frac{1}{\sqrt{\pi d}} \left(\frac{2\pi e}{d}\right)^{d/2} \tag{5.92}
$$

当 $$d > 2\pi e \approx 17.1$$ 时，底数 $$2\pi e / d < 1$$，故 $$(2\pi e / d)^{d/2} \to 0$$（指数衰减），因此 $$V_d(1) \to 0$$。

**物理含义**：高维超球的体积相对于其外接超立方体 $$[-1,1]^d$$（体积为 $$2^d$$）的占比 $$V_d(1)/2^d$$ 以**双指数速度**趋于零——高维空间中，几乎所有的"体积"集中在超立方体的**角落**，而非中心球体内。$$\square$$
{% endblock %}

### 5.7.1.2 距离集中现象

{% block type="definition" title="定义 5.33　距离集中比" %}
给定数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\} \subset \mathbb{R}^d$$，对查询点 $$\mathbf{q}$$，定义**距离集中比**：

$$
\rho_d = \frac{D_{\max} - D_{\min}}{D_{\min}} \tag{5.93}
$$

其中 $$D_{\max} = \max_i \|\mathbf{q} - \mathbf{x}_i\|$$，$$D_{\min} = \min_i \|\mathbf{q} - \mathbf{x}_i\|$$。
{% endblock %}

{% block type="theorem" title="定理 5.14　高维距离集中（Beyer et al., 1999）" %}
若数据点的各维坐标独立同分布且具有有限方差，则当维度 $$d \to \infty$$ 时，距离集中比以概率趋于零：

$$
\lim_{d \to \infty} \rho_d = \lim_{d \to \infty} \frac{D_{\max} - D_{\min}}{D_{\min}} \xrightarrow{P} 0 \tag{5.94}
$$

**证明思路**：

设 $$\mathbf{x}_i$$ 的各维坐标 $$x_{ij}$$ i.i.d. 且有均值 $$\mu$$、方差 $$\sigma^2$$。则

$$
\|\mathbf{q} - \mathbf{x}_i\|^2 = \sum_{j=1}^d (q_j - x_{ij})^2
$$

由大数定律，当 $$d \to \infty$$ 时，$$\frac{1}{d}\|\mathbf{q} - \mathbf{x}_i\|^2 \xrightarrow{P} \mathbb{E}[(q_j - x_{ij})^2]$$，该极限对所有 $$i$$ 相同。因此所有距离 $$\|\mathbf{q} - \mathbf{x}_i\|$$ 都趋于相同的值 $$\sqrt{d \cdot \mathbb{E}[(q_j - x_{ij})^2]}$$，使得 $$D_{\max} - D_{\min}$$ 相对于 $$D_{\min}$$ 趋于零。$$\square$$
{% endblock %}

### 5.7.1.3 维度诅咒对聚类的影响

| 影响 | 机制 | 后果 |
|------|------|------|
| 距离失效 | 最远与最近点距离趋于相等 | 基于距离的算法（K-means、DBSCAN）失去区分能力 |
| 空间稀疏 | 数据点相对于空间体积极度稀疏 | 密度估计不可靠，密度聚类失效 |
| 噪声维度 | 大量不相关维度充当噪声 | 真实簇结构被淹没 |
| 计算开销 | 距离计算 $$O(d)$$，协方差 $$O(d^2)$$ | 高维时计算代价过高 |

## 5.7.2 特征选择方法

应对高维聚类的第一策略是**去除不相关或冗余的维度**，仅保留对聚类有用的特征子集。

### 5.7.2.1 过滤法

**方差过滤**：去除方差过小的特征（几乎不变的特征对聚类无贡献）：

$$
\text{保留特征 } j \text{ 当 } \text{Var}(x_{\cdot j}) > \tau_{\text{var}} \tag{5.95}
$$

**相关性过滤**：对高度相关的特征对（$$|r_{jl}| > \tau_{\text{corr}}$$），保留其中方差较大的一个，去除冗余。

```python
import numpy as np
from sklearn.feature_selection import VarianceThreshold

def filter_features(X, var_threshold=0.01, corr_threshold=0.95):
    """方差过滤 + 相关性去冗余"""
    # 方差过滤
    selector = VarianceThreshold(threshold=var_threshold)
    X_var = selector.fit_transform(X)
    kept = selector.get_support(indices=True)
    # 相关性去冗余
    corr = np.abs(np.corrcoef(X_var.T))
    np.fill_diagonal(corr, 0)
    to_drop = set()
    for j in range(corr.shape[0]):
        if j in to_drop:
            continue
        high_corr = np.where(corr[j] > corr_threshold)[0]
        for l in high_corr:
            if l > j and l not in to_drop:
                to_drop.add(l)
    keep = [j for j in range(X_var.shape[1]) if j not in to_drop]
    return X_var[:, keep], kept[keep]
```

### 5.7.2.2 包裹法

包裹法以**聚类质量**作为特征子集的评价标准：搜索特征组合，选择使聚类指标（如轮廓系数、CH 指数）最优的子集。虽然效果好，但搜索空间为 $$2^d$$，在高维时通常不可行，需要结合贪心搜索或遗传算法。

## 5.7.3 降维 + 聚类管线

### 5.7.3.1 PCA + K-means

**主成分分析**（PCA）通过线性投影将数据映射到方差最大的方向，保留最多信息的同时降低维度：

$$
\mathbf{Z} = \mathbf{X} \mathbf{W}_r \tag{5.96}
$$

其中 $$\mathbf{W}_r \in \mathbb{R}^{d \times r}$$ 为前 $$r$$ 个特征向量组成的矩阵，$$r \ll d$$。

**PCA + K-means 管线**是高维聚类的经典方案：

```python
from sklearn.decomposition import PCA
from sklearn.cluster import KMeans
from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler

def pca_kmeans_pipeline(n_components, n_clusters):
    return Pipeline([
        ('scaler', StandardScaler()),
        ('pca', PCA(n_components=n_components, random_state=42)),
        ('kmeans', KMeans(n_clusters=n_clusters, n_init=10, random_state=42)),
    ])
```

**选择 $$r$$ 的方法**：保留解释至少 85%–95% 方差的主成分数：

$$
r^* = \arg\min_r \left\{ r : \frac{\sum_{j=1}^r \lambda_j}{\sum_{j=1}^d \lambda_j} \geq 0.90 \right\} \tag{5.97}
$$

### 5.7.3.2 t-SNE 与 UMAP 的角色

**t-SNE**（t-distributed Stochastic Neighbor Embedding）和 **UMAP**（Uniform Manifold Approximation and Projection）是两种流行的非线性降维方法，擅长保留局部邻域结构，生成直观的二维/三维可视化。

{% hint style="warning" %}
**重要警告**：t-SNE 和 UMAP 是**可视化工具**，不应直接用于聚类的特征空间。原因：
1. **非保距**：全局距离关系被严重扭曲——不同簇间的距离不可比；
2. **不稳定**：不同随机种子可能产生截然不同的布局；
3. **参数敏感**：perplexity（t-SNE）和 n_neighbors（UMAP）显著影响结果。

**正确做法**：在 PCA 降维后的空间中聚类，然后用 t-SNE/UMAP 可视化聚类结果。
{% endhint %}

## 5.7.4 子空间聚类方法

### 5.7.4.1 回顾 CLIQUE

§5.4 介绍的 CLIQUE 算法是子空间聚类的先驱——它在高维网格的低维子空间中搜索密集区域。其核心思想是：**不同的簇可能存在于不同的特征子空间中**。然而 CLIQUE 受限于网格结构，对连续分布的数据不够灵活。

### 5.7.4.2 谱聚类

谱聚类（Spectral Clustering）是一种强大的子空间聚类方法，它将聚类问题转化为**图分割问题**，不依赖于欧氏空间中的簇形状假设。

{% block type="definition" title="定义 5.34　谱聚类（Spectral Clustering）" %}
给定数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$$，谱聚类的步骤如下：

**1. 构建相似度矩阵** $$\mathbf{W} \in \mathbb{R}^{n \times n}$$：

$$
W_{ij} = \exp\!\left(-\frac{\|\mathbf{x}_i - \mathbf{x}_j\|^2}{2\sigma^2}\right) \tag{5.98}
$$

**2. 计算图拉普拉斯矩阵**。定义度矩阵 $$\mathbf{D} = \text{diag}(d_1, \ldots, d_n)$$，其中 $$d_i = \sum_j W_{ij}$$。**归一化拉普拉斯矩阵**为：

$$
\mathbf{L}_{\text{sym}} = \mathbf{I} - \mathbf{D}^{-1/2} \mathbf{W} \mathbf{D}^{-1/2} \tag{5.99}
$$

**3. 特征分解**：计算 $$\mathbf{L}_{\text{sym}}$$ 的前 $$K$$ 个最小特征值对应的特征向量 $$\mathbf{u}_1, \ldots, \mathbf{u}_K$$，构成矩阵 $$\mathbf{U} \in \mathbb{R}^{n \times K}$$。

**4. 行归一化**：对 $$\mathbf{U}$$ 的每行进行 $$\ell_2$$ 归一化。

**5. 在 $$\mathbf{U}$$ 的行空间中运行 K-means**，得到聚类标签。
{% endblock %}

谱聚类的优势在于：通过图拉普拉斯的特征分解，数据被嵌入到一个**低维空间**中，在该空间中簇结构更加清晰——即使原始空间中的簇是非凸的。

```python
from sklearn.cluster import SpectralClustering

# 谱聚类对月牙形数据效果好
from sklearn.datasets import make_moons
X, y = make_moons(n_samples=300, noise=0.05, random_state=42)
sc = SpectralClustering(n_clusters=2, affinity='nearest_neighbors',
                        n_neighbors=10, random_state=42)
labels = sc.fit_predict(X)
```

## 5.7.5 实践指南与方法选择

| 数据特征 | 推荐方法 | 原因 |
|---------|---------|------|
| 高维 + 球形簇 | PCA + K-means | 简单高效，PCA 保留全局方差结构 |
| 高维 + 非凸簇 | PCA + 谱聚类 | 谱方法不依赖簇形状 |
| 高维 + 大量噪声维度 | 特征选择 + K-means | 先去噪再聚类 |
| 超高维（$$d > 1000$$） | PCA（95%方差）→ K-means/GMM | 两步降维，平衡信息与效率 |
| 需要可视化 | PCA 聚类 + UMAP 可视化 | 聚类在 PCA 空间，展示用 UMAP |

**经验法则**：

1. 始终先**标准化**数据（均值为 0、方差为 1），否则高方差维度主导距离计算；
2. PCA 保留的方差比例通常选 90%–95%——太少丢失信息，太多未有效降维；
3. 对降维后的数据使用 §5.6 的评估指标选择最优 $$K$$；
4. 谱聚类的计算复杂度为 $$O(n^3)$$（特征分解），对大数据集需要近似方法（如 Nyström 近似）。

## 案例 5.7：MNIST 手写数字的高维聚类

**问题背景**：MNIST 数据集包含 70,000 张 $$28 \times 28$$ 的灰度手写数字图像，每张展平为 784 维向量。直接在 784 维空间中聚类效果很差（距离集中），需要先降维。本案例比较不同降维+聚类策略的效果。

```python
import numpy as np
from sklearn.datasets import load_digits  # 8×8 简化版（1797 样本，64 维）
from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA
from sklearn.cluster import KMeans, SpectralClustering
from sklearn.mixture import GaussianMixture
from sklearn.metrics import (adjusted_rand_score, normalized_mutual_info_score,
                             silhouette_score)

# ── 1. 加载数据 ───────────────────────────────────────────
digits = load_digits()
X, y_true = digits.data, digits.target  # 64 维，10 类
n_clusters = 10
print(f"数据维度：{X.shape}，类别数：{len(set(y_true))}")

# ── 2. 标准化 ─────────────────────────────────────────────
scaler = StandardScaler()
X_scaled = scaler.fit_transform(X)

# ── 3. 不同降维+聚类策略 ─────────────────────────────────
results = {}
# 策略 1：直接 K-means（64 维）
km_full = KMeans(n_clusters=n_clusters, n_init=10, random_state=42)
labels_full = km_full.fit_predict(X_scaled)
results['K-means (64D)'] = labels_full

# 策略 2：PCA(10) + K-means
pca10 = PCA(n_components=10, random_state=42)
X_pca10 = pca10.fit_transform(X_scaled)
labels_pca10 = KMeans(n_clusters=n_clusters, n_init=10,
                      random_state=42).fit_predict(X_pca10)
results['PCA(10)+K-means'] = labels_pca10

# 策略 3：PCA(30) + K-means
pca30 = PCA(n_components=30, random_state=42)
X_pca30 = pca30.fit_transform(X_scaled)
labels_pca30 = KMeans(n_clusters=n_clusters, n_init=10,
                      random_state=42).fit_predict(X_pca30)
results['PCA(30)+K-means'] = labels_pca30

# 策略 4：PCA(30) + GMM
gmm = GaussianMixture(n_components=n_clusters, n_init=5,
                      covariance_type='diag', random_state=42)
labels_gmm = gmm.fit(X_pca30).predict(X_pca30)
results['PCA(30)+GMM'] = labels_gmm

# ── 4. 评估 ──────────────────────────────────────────────
print(f"\n{'策略':<25} {'ARI':>6} {'NMI':>6} {'轮廓':>6}")
print('-' * 45)
for name, labels in results.items():
    ari = adjusted_rand_score(y_true, labels)
    nmi = normalized_mutual_info_score(y_true, labels)
    X_eval = X_pca30 if 'PCA(30)' in name else (X_pca10 if 'PCA(10)' in name
             else X_scaled)
    sil = silhouette_score(X_eval, labels)
    print(f"{name:<25} {ari:6.3f} {nmi:6.3f} {sil:6.3f}")

# ── 5. 方差解释分析 ──────────────────────────────────────
pca_full = PCA(random_state=42).fit(X_scaled)
cumvar = np.cumsum(pca_full.explained_variance_ratio_)
for threshold in [0.80, 0.90, 0.95]:
    n_comp = np.argmax(cumvar >= threshold) + 1
    print(f"\n保留 {threshold:.0%} 方差需要 {n_comp} 个主成分（原始 {X.shape[1]} 维）")
```

**关键结论**：

1. **直接聚类 vs 降维聚类**：在 64 维空间直接 K-means 的 ARI 约 0.5–0.6；PCA 降到 30 维后 ARI 提升到 0.6–0.7——**适度降维改善聚类**，因为去除了噪声维度；
2. **PCA 维度选择**：保留太少主成分（如 2 维）丢失过多信息，ARI 很低；保留 30 维（约 90% 方差）是较好的平衡点；
3. **GMM vs K-means**：在降维空间中，GMM 的 ARI 略高于 K-means，且提供了软分配——某些"模糊"数字（如 4/9、3/8）同时属于多个成分；
4. **可视化与聚类分离**：用 UMAP 将 64 维降到 2 维可视化，可以看到各数字大致分开，但 1 和 7、4 和 9 有重叠区域。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 维度诅咒 | 高维空间中距离失效、空间稀疏、噪声淹没信号 |
| 超球体积衰减（定理 5.13） | $$V_d(1) \to 0$$（$$d \to \infty$$），球形区域体积可忽略 |
| 距离集中（定理 5.14） | $$\rho_d = (D_{\max}-D_{\min})/D_{\min} \to 0$$，远近不分 |
| 特征选择 | 方差过滤 + 相关性去冗余，保留有区分力的维度 |
| PCA + K-means | 经典降维聚类管线，保留主方差方向 |
| t-SNE / UMAP（警告） | 仅用于可视化，不作为聚类特征空间 |
| 谱聚类（定义 5.34） | 图拉普拉斯特征分解 → K-means，不依赖簇形状 |
| 降维维度选择 | 保留 90%–95% 方差的主成分数 |

### F 级（开放探讨）

**F1.** **前沿视野·对比学习与自监督聚类**：近年来，SimCLR、BYOL 等对比学习方法通过自监督方式学习数据的低维表示，然后在表示空间中聚类，在图像聚类任务上大幅超越了传统 PCA + K-means。阅读相关论文后讨论：(1) 对比学习学到的表示为什么比 PCA 更适合聚类？(2) 这些方法是否真正"解决"了维度诅咒，还是仅仅将问题转移到了模型训练阶段？(3) 对比学习的预训练需要大量无标签数据和 GPU 资源——在数据有限或计算资源受限时，传统方法（PCA + 谱聚类）是否仍有优势？

**F2.** **课程思政·数据降维中的信息损失与伦理**：降维是高维聚类的必要步骤，但降维不可避免地**丢弃信息**。在某些敏感应用中，被丢弃的维度可能携带重要的公平性信息。例如：(1) 在信用风险聚类中，PCA 可能将种族/性别相关的特征"压缩"到被丢弃的低方差主成分中，导致聚类结果看似不包含敏感属性，但实际上通过其他特征的代理效应保留了歧视——如何检测和预防这种"洗白"现象？(2) 在医疗数据聚类中，某些罕见病的信号可能只出现在少数低方差维度中，PCA 降维后这些信号被丢弃——"多数服从少数"的降维原则在这里是否道德？(3) 作为数据科学家，在报告降维聚类结果时，应该如何记录和披露降维过程中的信息损失？
