# 5.1 基于划分的聚类算法

第 4 章的回归与分类任务都依赖**已知标签**：训练集中的每个样本都携带目标值，模型通过拟合输入-输出映射来泛化到新样本。然而现实数据中，标签往往代价高昂甚至根本不存在——市场分析师手头有百万条客户交易记录，却不知道该如何将客户分成几类；基因组学研究者拥有数万个基因表达谱，却不知道应当寻找多少个细胞亚型。**聚类**（Clustering）正是为了回答"数据内部存在怎样的自然群组结构"这一问题而生的，属于**无监督学习**（Unsupervised Learning）的核心任务。

**划分型聚类**（Partition-based Clustering）是最直观的一类方法：给定样本集合与簇数 $$K$$，将所有样本**硬分配**到互不重叠的 $$K$$ 个簇中，使簇内样本尽可能相似、簇间样本尽可能不同。**K-means** 和 **K-medoids** 是划分型聚类的两大经典算法，前者以欧氏距离下的均值作为簇心，后者以数据点本身作为代表，适应任意距离度量。两者都以简洁的迭代框架、明确的目标函数和可证明的收敛性，成为数据挖掘实践中最常用的聚类工具。

## 5.1.1 从"物以类聚"到形式化定义

"物以类聚，人以群分"——这句话恰好道出了聚类分析的核心直觉：**相似的对象应当属于同一类**。但"相似"如何度量？"类"如何定义？"好的聚类"有什么标准？在将直觉转化为可计算的算法之前，我们需要建立严格的形式化定义。

### 5.1.1.1 聚类的形式化定义

{% block type="definition" title="定义 5.1　聚类（Clustering）" %}
设数据集 $$\mathcal{D} = \{\mathbf{x}_1, \mathbf{x}_2, \ldots, \mathbf{x}_n\}$$，其中 $$\mathbf{x}_i \in \mathbb{R}^d$$。**聚类**是将 $$\mathcal{D}$$ 划分为 $$K$$ 个非空子集（**簇**）$$\mathcal{C} = \{C_1, C_2, \ldots, C_K\}$$ 的映射 $$\gamma : \mathcal{D} \to \{1, \ldots, K\}$$，使得：

$$
C_k = \{\mathbf{x}_i \in \mathcal{D} : \gamma(\mathbf{x}_i) = k\}, \quad k = 1, \ldots, K \tag{5.1}
$$

满足**完备性**（$$\bigcup_{k=1}^K C_k = \mathcal{D}$$）和**互斥性**（$$C_j \cap C_k = \emptyset$$，$$j \neq k$$）。目标是使某个关于簇内相似度（或簇间差异度）的**目标函数**最优化。
{% endblock %}

{% block type="definition" title="定义 5.2　划分型聚类（Partition-based Clustering）" %}
**划分型聚类**是在给定簇数 $$K$$ 下，将 $$n$$ 个数据点**硬分配**（hard assignment）到 $$K$$ 个互斥簇的聚类方法。其特征为：

1. 每个样本恰好属于一个簇（硬分配，区别于模糊聚类的软分配）；
2. 簇数 $$K$$ 需预先指定（区别于层次聚类和密度聚类）；
3. 算法通过迭代优化目标函数来搜索最优划分。

**代表算法**：K-means（基于均值中心）、K-medoids（基于数据点中心）、K-modes（针对类别数据）。
{% endblock %}

### 5.1.1.2 聚类与分类的根本区别

聚类与分类（classification）在表象上都是把样本分到若干组，但两者存在本质差别：

| 维度 | 聚类（无监督） | 分类（有监督） |
|------|--------------|--------------|
| 标签 | 无标签，学习数据内在结构 | 有标签，学习输入-输出映射 |
| 簇数 | 需要确定（本身是难题） | 类别数已知 |
| 评估 | 内部指标（轮廓系数）或外部验证 | 准确率、F1 等直接度量 |
| 目标 | 发现未知结构 | 拟合已知标签 |

聚类的内在挑战在于**无参考答案**：对同一数据集，不同的目标函数可能产生截然不同的聚类结果，而我们无法通过准确率来判断哪个"更正确"。这使得聚类结果的解释和验证始终需要结合领域知识。

### 5.1.1.3 应用场景与动机

划分型聚类在实践中覆盖极广的应用场景：

- **客户细分**（Customer Segmentation）：电商平台基于 RFM（Recency, Frequency, Monetary）特征将客户分为高价值客户、潜力客户、流失风险客户等群组，为差异化营销提供依据；
- **图像压缩**（Image Compression）：将图像中出现的颜色聚类为 $$K$$ 种代表色（K-means 颜色量化），用代表色索引替代原始 24-bit 颜色值，实现有损压缩；
- **基因表达分析**：对 RNA-seq 数据按基因表达模式聚类，发现功能相关的基因群或细胞亚型；
- **异常检测预处理**：先聚类，再将距离所有簇心都很远的样本标记为潜在异常值；
- **文档主题发现**：对 TF-IDF 文档向量聚类，实现无监督主题建模的快速基线。

以上应用都具备一个共同特征：**数据中存在自然的群组结构，但分组标准和数量均未知**。划分型聚类正是从数据本身出发，通过优化目标函数来发现这种结构。

接下来，我们将 K-means 算法的目标函数和迭代过程形式化，并进行严格的数学分析。

## 5.1.2 K-means 算法：目标函数与迭代求解

### 5.1.2.1 目标函数：簇内平方和

K-means 的核心思想是**最小化簇内平方和**（Within-cluster Sum of Squares, WCSS 或 SSE）——每个样本与其所属簇中心的距离平方之和。

{% block type="definition" title="定义 5.3　K-means 目标函数（簇内平方和）" %}
设数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\} \subset \mathbb{R}^d$$，给定簇数 $$K$$。K-means 的**目标函数**（簇内平方和，WCSS）为：

$$
J(\mathcal{C}, \mathcal{M}) = \sum_{k=1}^K \sum_{\mathbf{x} \in C_k} \|\mathbf{x} - \boldsymbol{\mu}_k\|^2 \tag{5.2}
$$

其中 $$\boldsymbol{\mu}_k = \frac{1}{|C_k|}\sum_{\mathbf{x} \in C_k} \mathbf{x}$$ 是第 $$k$$ 个簇的**质心**（centroid），$$\mathcal{M} = \{\boldsymbol{\mu}_1, \ldots, \boldsymbol{\mu}_K\}$$ 是所有质心的集合。

**K-means 优化问题**：

$$
\mathcal{C}^* = \arg\min_{\mathcal{C}} \min_{\mathcal{M}} J(\mathcal{C}, \mathcal{M}) \tag{5.3}
$$

即在所有可能的 $$K$$-划分中，找使目标函数最小的划分。
{% endblock %}

{% hint style="warning" %}
**NP 难性警告**：K-means 的精确全局最优解问题是 NP 难的（即使在二维平面 $$d=2$$ 时，也是 NP 难的）。因此，实践中采用**启发式迭代算法**（Lloyd 算法）求局部最优解，该算法有收敛性保证（定理 5.1），但不保证全局最优。
{% endhint %}

### 5.1.2.2 Lloyd 迭代算法

**Lloyd 算法**（也称 K-means 算法）通过交替优化两组变量——在固定质心时优化赋值，在固定赋值时优化质心——来最小化目标函数 (5.2)。

**算法流程**：

> **输入**：数据集 $$\mathcal{D} = \{\mathbf{x}_1, \ldots, \mathbf{x}_n\}$$，簇数 $$K$$，最大迭代次数 $$T$$
>
> **初始化**：随机选取 $$K$$ 个数据点作为初始质心 $$\boldsymbol{\mu}_1^{(0)}, \ldots, \boldsymbol{\mu}_K^{(0)}$$
>
> **对 $$t = 0, 1, 2, \ldots$$ 重复**：
>
> **（E 步）赋值步**：将每个样本分配到距离最近的质心所在的簇：
> $$C_k^{(t+1)} = \left\{\mathbf{x}_i : k = \arg\min_{1 \leq j \leq K} \|\mathbf{x}_i - \boldsymbol{\mu}_j^{(t)}\|^2\right\} \tag{5.4}$$
>
> **（M 步）更新步**：将每个质心更新为其簇内样本的均值：
> $$\boldsymbol{\mu}_k^{(t+1)} = \frac{1}{|C_k^{(t+1)}|} \sum_{\mathbf{x} \in C_k^{(t+1)}} \mathbf{x} \tag{5.5}$$
>
> **直到**：赋值不再变化，或 $$J$$ 的减少量小于阈值 $$\varepsilon$$，或达到最大迭代次数 $$T$$
>
> **输出**：最终划分 $$\mathcal{C} = \{C_1, \ldots, C_K\}$$，质心集合 $$\mathcal{M} = \{\boldsymbol{\mu}_1, \ldots, \boldsymbol{\mu}_K\}$$

{% block type="definition" title="定义 5.4　Voronoi 划分（Voronoi Partition）" %}
给定质心集合 $$\mathcal{M} = \{\boldsymbol{\mu}_1, \ldots, \boldsymbol{\mu}_K\}$$，**Voronoi 划分**将 $$\mathbb{R}^d$$ 分为 $$K$$ 个区域：

$$
V_k = \left\{\mathbf{x} \in \mathbb{R}^d : \|\mathbf{x} - \boldsymbol{\mu}_k\|^2 \leq \|\mathbf{x} - \boldsymbol{\mu}_j\|^2, \; \forall j \neq k\right\} \tag{5.6}
$$

K-means 的赋值步（公式 (5.4)）恰好将每个样本分配到其所在的 Voronoi 区域，因此 **K-means 赋值等价于 Voronoi 划分**。
{% endblock %}

### 5.1.2.3 算法复杂度

每次迭代由赋值步和更新步组成：

- **赋值步**：对 $$n$$ 个样本，计算到 $$K$$ 个质心的距离（每次距离计算 $$O(d)$$），共 $$O(nKd)$$；
- **更新步**：对每个簇计算均值，共 $$O(nd)$$；

因此每次迭代复杂度为 $$O(nKd)$$，$$T$$ 次迭代后总复杂度为：

$$
\text{K-means 时间复杂度} = O(nKd \cdot T) \tag{5.7}
$$

其中 $$T$$ 通常很小（实践中多数情况下 $$T \leq 100$$）。空间复杂度为 $$O(nd + Kd)$$（存储数据和质心）。

## 5.1.3 收敛性分析与理论保证

K-means 迭代算法为什么一定会终止？终止时目标函数是否已经（局部）最小化？本节给出完整的收敛性证明。

### 5.1.3.1 均值最优性

首先证明一个基础引理：在固定赋值下，使簇内 SSE 最小的最优质心恰好是样本均值。

{% block type="theorem" title="定理 5.2　均值最优性（Mean Optimality）" %}
设簇 $$C_k$$ 非空，考虑固定赋值下对第 $$k$$ 个质心 $$\boldsymbol{\mu} \in \mathbb{R}^d$$ 的优化：

$$
\boldsymbol{\mu}_k^* = \arg\min_{\boldsymbol{\mu} \in \mathbb{R}^d} \sum_{\mathbf{x} \in C_k} \|\mathbf{x} - \boldsymbol{\mu}\|^2 \tag{5.8}
$$

则最优解唯一，为**簇内样本均值**：

$$
\boldsymbol{\mu}_k^* = \frac{1}{|C_k|} \sum_{\mathbf{x} \in C_k} \mathbf{x} \tag{5.9}
$$

**证明**：设 $$f(\boldsymbol{\mu}) = \sum_{\mathbf{x} \in C_k} \|\mathbf{x} - \boldsymbol{\mu}\|^2$$，对 $$\boldsymbol{\mu}$$ 求梯度：

$$
\nabla_{\boldsymbol{\mu}} f = \sum_{\mathbf{x} \in C_k} 2(\boldsymbol{\mu} - \mathbf{x}) = 2|C_k|\boldsymbol{\mu} - 2\sum_{\mathbf{x} \in C_k} \mathbf{x}
$$

令 $$\nabla_{\boldsymbol{\mu}} f = \mathbf{0}$$，得 $$\boldsymbol{\mu}^* = \frac{1}{|C_k|}\sum_{\mathbf{x} \in C_k} \mathbf{x}$$。

**唯一性**：Hessian 矩阵 $$\nabla^2 f = 2|C_k| \mathbf{I}_d \succ 0$$（严格正定），因此 $$f$$ 是严格凸函数，极小值唯一。$$\square$$

**推论**：定理 5.2 表明 K-means 更新步（公式 (5.5)）是在固定赋值下对目标函数 $$J$$ 的**精确最小化**，而非启发式近似。
{% endblock %}

### 5.1.3.2 K-means 收敛性

{% block type="theorem" title="定理 5.1　K-means 收敛性（Convergence of K-means）" %}
Lloyd 算法（K-means）在有限步内必然终止，且终止时达到目标函数 $$J$$ 的**局部极小值**（或鞍点）。

**证明**：

**第一步：目标函数单调不增**。设第 $$t$$ 轮迭代后的赋值为 $$\mathcal{C}^{(t)}$$，质心为 $$\mathcal{M}^{(t)}$$，则：

- **赋值步使 $$J$$ 不增**：固定质心 $$\mathcal{M}^{(t)}$$，对每个样本 $$\mathbf{x}_i$$，将其分配到距离最近的质心，即：
$$
\gamma^{(t+1)}(\mathbf{x}_i) = \arg\min_k \|\mathbf{x}_i - \boldsymbol{\mu}_k^{(t)}\|^2
$$
由于每个样本选择了不超过原赋值代价的簇，有：
$$
J(\mathcal{C}^{(t+1)}, \mathcal{M}^{(t)}) \leq J(\mathcal{C}^{(t)}, \mathcal{M}^{(t)}) \tag{5.10}
$$

- **更新步使 $$J$$ 不增**：固定赋值 $$\mathcal{C}^{(t+1)}$$，由定理 5.2，将质心更新为簇内均值严格最小化各簇的 SSE，因此：
$$
J(\mathcal{C}^{(t+1)}, \mathcal{M}^{(t+1)}) \leq J(\mathcal{C}^{(t+1)}, \mathcal{M}^{(t)}) \tag{5.11}
$$

综合 (5.10) 和 (5.11)：$$J^{(t+1)} \leq J^{(t)}$$，目标函数序列 $$\{J^{(t)}\}$$ 单调不增。

**第二步：划分数量有限**。$$n$$ 个样本在 $$K$$ 个簇中的不同赋值方案数为**Stirling 第二类数** $$S(n, K)$$，满足 $$S(n, K) \leq K^n / K! < \infty$$。因此，$$J$$ 的所有可能取值构成一个**有限集合**。

**第三步：有限步终止**。由第一步，$$J^{(t)}$$ 单调不增；由第二步，$$J^{(t)}$$ 只能取有限个值。因此序列 $$\{J^{(t)}\}$$ 必然在有限步内到达一个不再严格减小的值，此后赋值不再变化，算法终止。

**局部极小性**：终止时，赋值步和更新步都无法进一步减小 $$J$$，即满足局部最优条件。$$\square$$

{% hint style="warning" %}
**局部最优的代价**：定理 5.1 只保证**局部收敛**，不保证全局最优。实践中常用多次随机重启（`n_init` 参数，默认 10 次）并取最小 $$J$$ 的结果作为最终输出。
{% endhint %}
{% endblock %}

## 5.1.4 初始化的陷阱与 K-means++

### 5.1.4.1 初始化对结果的影响

K-means 对初始质心**极度敏感**：不同的初始化可能导致算法收敛到截然不同的局部最优解，对应的 $$J$$ 值差异可能非常大。以下是三类典型的糟糕初始化情形：

1. **所有初始质心集中在同一区域**：大量迭代后，距离聚簇的初始质心难以"迁移"到其他区域；
2. **两个初始质心分配给同一真实簇**：导致一个真实簇被人为分裂；
3. **初始质心全选异常值**：收敛后异常值主导某些簇的质心，使正常数据的聚类质量下降。

**随机重启**（Random Restart）是应对这一问题的最直接方法：独立运行 K-means $$r$$ 次（不同随机初始化），选取目标函数 $$J$$ 最小的结果。但当 $$K$$ 较大时，找到一次好的初始化的概率很低，重启的效率有限。

### 5.1.4.2 K-means++ 初始化

**K-means++**（Arthur & Vassilvitskii, 2007）提出了一种概率初始化方案，以**与距离已选质心集合的距离平方成正比**的概率选择下一个质心，从而鼓励初始质心分散分布。

**K-means++ 初始化算法**：

> **输入**：数据集 $$\mathcal{D}$$，簇数 $$K$$
>
> 1. 从 $$\mathcal{D}$$ 中均匀随机选取第一个质心 $$\boldsymbol{\mu}_1$$；
> 2. **对 $$j = 2, \ldots, K$$**：
>    - 对每个未选数据点 $$\mathbf{x} \in \mathcal{D}$$，计算其到已选质心集合 $$\mathcal{S} = \{\boldsymbol{\mu}_1, \ldots, \boldsymbol{\mu}_{j-1}\}$$ 的**最近距离**：
>    $$D(\mathbf{x}) = \min_{\boldsymbol{\mu} \in \mathcal{S}} \|\mathbf{x} - \boldsymbol{\mu}\|^2 \tag{5.12}$$
>    - 以正比于 $$D(\mathbf{x})^2$$ 的概率选取下一个质心：
>    $$P(\mathbf{x} \text{ 被选为 } \boldsymbol{\mu}_j) = \frac{D(\mathbf{x})^2}{\sum_{\mathbf{x}' \in \mathcal{D}} D(\mathbf{x}')^2} \tag{5.13}$$
> 3. 用 $$K$$ 个初始质心运行标准 Lloyd 迭代；

**直观解释**：距已选质心越远的点，被选为下一个质心的概率越大，因此初始质心自然趋向于**分散**——这正是避免局部最优的关键。

### 5.1.4.3 K-means++ 的期望代价上界

K-means++ 之所以成为事实上的默认初始化方法（sklearn 中 `init='k-means++'` 是默认值），不仅因为其实践效果好，更因为它有严格的**理论保证**：

$$
\mathbb{E}[J_{\text{K-means++}}] \leq 8(\ln K + 2) \cdot J^* \tag{5.14}
$$

其中 $$J^*$$ 是全局最优目标函数值，期望对初始化的随机性取。这一 $$O(\log K)$$ 近似比保证表明，K-means++ 的期望代价至多是最优解的 $$O(\log K)$$ 倍——对比随机初始化没有任何理论保证，这是质的飞跃。

**关键引理**（证明框架）：设 $$C_1^*, \ldots, C_K^*$$ 是最优划分，任取一个未被初始化质心"覆盖"的最优簇 $$C^*$$，若在 $$C^*$$ 内均匀随机选取下一个质心，则该簇对目标函数的贡献期望至多减少为原来的 $$\frac{1}{2}$$；结合 K-means++ 的 $$D^2$$-加权采样，跨越 $$K$$ 个簇的覆盖过程的总期望代价可由调和级数求和得到 $$O(\ln K)$$ 倍的上界。

{% hint style="info" %}
**实践建议**：sklearn 的 `KMeans(init='k-means++', n_init=10)` 是工业实践的推荐配置：K-means++ 初始化提供好的起点，10 次重启进一步降低陷入差局部最优的概率。对于大数据集（$$n > 10^5$$），可使用 `MiniBatchKMeans` 配合 K-means++ 初始化，在精度略有损失的情况下大幅提升速度。
{% endhint %}

## 5.1.5 K 值选择：肘部法则与轮廓系数

K-means 最棘手的超参数是**簇数 $$K$$**——它既不能从训练误差中学到（$$K = n$$ 时 $$J = 0$$，完全过拟合），也没有一个统一的最优标准。本节介绍两种最常用的 $$K$$ 值选择方法。

### 5.1.5.1 肘部法则（Elbow Method）

**肘部法则**通过绘制**簇内 SSE（或 WCSS）随 $$K$$ 的变化曲线**来选择 $$K$$：

$$
\mathrm{SSE}(K) = \sum_{k=1}^K \sum_{\mathbf{x} \in C_k} \|\mathbf{x} - \boldsymbol{\mu}_k\|^2 \tag{5.15}
$$

随着 $$K$$ 增大，$$\mathrm{SSE}(K)$$ 单调递减（更多簇可以更好地拟合数据）。当 $$K$$ 小于真实簇数时，增加一个簇会显著减少 SSE；而当 $$K$$ 超过真实簇数后，增加簇的收益急剧下降。**"肘部"**（elbow）是曲线斜率从陡峭转为平缓的转折点，对应的 $$K$$ 值即为建议选择。

形式化地，可以用**差分法**定位肘部：

$$
\Delta^2\mathrm{SSE}(K) = \mathrm{SSE}(K-1) - 2\,\mathrm{SSE}(K) + \mathrm{SSE}(K+1) \tag{5.16}
$$

$$\Delta^2\mathrm{SSE}(K)$$ 取最大值时对应的 $$K$$ 即为肘部。

**局限性**：当数据中不存在明显的群组结构，或不同簇的大小和密度差异悬殊时，肘部曲线可能是平滑的、没有清晰的转折点，此时肘部法则失效。

### 5.1.5.2 轮廓系数（Silhouette Coefficient）

**轮廓系数**（Silhouette Coefficient）是一种同时考虑**簇内紧凑性**和**簇间分离性**的内部评估指标。对每个样本 $$\mathbf{x}_i$$，定义：

- $$a(i) = \frac{1}{|C_k| - 1} \sum_{\mathbf{x}_j \in C_k, j \neq i} d(\mathbf{x}_i, \mathbf{x}_j)$$：样本 $$i$$ 与同簇其他样本的**平均距离**（簇内不相似度）；
- $$b(i) = \min_{k' \neq k} \frac{1}{|C_{k'}|} \sum_{\mathbf{x}_j \in C_{k'}} d(\mathbf{x}_i, \mathbf{x}_j)$$：样本 $$i$$ 与**最近邻簇**中所有样本的平均距离（簇间不相似度）。

则样本 $$i$$ 的**轮廓系数**为：

$$
s(i) = \frac{b(i) - a(i)}{\max(a(i),\, b(i))} \tag{5.17}
$$

**性质**：$$s(i) \in [-1, 1]$$。$$s(i) \approx 1$$ 表示样本 $$i$$ 与同簇的样本紧密相连，与最近邻簇距离较远（理想情形）；$$s(i) \approx 0$$ 表示样本处于两个簇的边界附近；$$s(i) < 0$$ 表示样本可能被分配到了错误的簇。

**整体聚类质量**由平均轮廓系数衡量：

$$
\bar{s}(K) = \frac{1}{n} \sum_{i=1}^n s(i) \tag{5.18}
$$

选择使 $$\bar{s}(K)$$ 最大的 $$K$$ 作为最优簇数。

| 方法 | 优点 | 缺点 | 适用场景 |
|------|------|------|---------|
| 肘部法则（Elbow） | 计算简单，直观易解释 | 肘部不明显时难以判断 | 快速探索，数据有明显分层结构 |
| 轮廓系数 | 同时考虑紧凑性和分离性，有明确最优点 | 计算复杂度 $$O(n^2)$$，大数据集慢 | 中小规模数据，需严格验证 $$K$$ |
| Gap 统计量 | 与随机基准对比，可检测"无结构"情形 | 实现复杂，需蒙特卡罗采样 | 数据中可能不存在真实群组结构 |

## 5.1.6 K-medoids：更稳健的中心点策略

### 5.1.6.1 K-means 的局限与 K-medoids 的动机

K-means 的质心 $$\boldsymbol{\mu}_k$$ 是样本的**均值**，它存在两个固有的局限：

1. **对异常值敏感**：一个极端值会将均值拉离真实的簇中心，扭曲整个簇的结构；
2. **仅适用于欧氏空间**：均值的计算需要定义"加法"和"数乘"，在类别数据、字符串、图结构等非欧空间中没有自然的意义。

**K-medoids** 通过将簇中心限制为**数据集中的实际数据点**（称为**medoid**，中心点），同时克服了以上两个局限。其目标函数可以用**任意距离或相似度函数** $$d(\cdot, \cdot)$$ 表示：

$$
J_{\text{med}} = \sum_{k=1}^K \sum_{\mathbf{x} \in C_k} d(\mathbf{x},\, \mathbf{m}_k) \tag{5.19}
$$

其中 $$\mathbf{m}_k \in C_k$$ 是第 $$k$$ 个簇的**中心点**（medoid），是满足使簇内距离之和最小的实际数据点：

$$
\mathbf{m}_k = \arg\min_{\mathbf{x}' \in C_k} \sum_{\mathbf{x} \in C_k} d(\mathbf{x}, \mathbf{x}') \tag{5.20}
$$

### 5.1.6.2 PAM 算法：SWAP 步推导

**PAM**（Partitioning Around Medoids，Kaufman & Rousseeuw, 1987）是 K-medoids 的经典实现算法，由 BUILD 阶段（初始化）和 SWAP 阶段（迭代优化）构成。

**BUILD 阶段**：贪心地选取 $$K$$ 个初始中心点，使目标函数最小。

**SWAP 阶段（核心迭代）**：

对每个当前中心点 $$\mathbf{m}_k$$（$$k = 1, \ldots, K$$）和每个非中心点 $$\mathbf{x}_h \in \mathcal{D} \setminus \{\mathbf{m}_1, \ldots, \mathbf{m}_K\}$$，计算将 $$\mathbf{m}_k$$ 替换为 $$\mathbf{x}_h$$ 后目标函数的变化量（**代价差**）：

$$
\Delta_{kh} = J(\text{将 } \mathbf{m}_k \text{ 替换为 } \mathbf{x}_h) - J(\text{当前}) \tag{5.21}
$$

若存在 $$(k^*, h^*)$$ 使得 $$\Delta_{k^*h^*} < 0$$（即替换后代价降低），则执行该替换，然后重新赋值所有点到最近中心点；重复此过程直到不存在有益的替换。

**SWAP 步的代价差分析**：对于特定的替换 $$(\mathbf{m}_k \to \mathbf{x}_h)$$，每个非中心点 $$\mathbf{x}_i$$ 的贡献变化量为：

$$
C_{kih} = \begin{cases}
\max(d(\mathbf{x}_i, \mathbf{x}_h) - d(\mathbf{x}_i, \mathbf{m}_k),\ 0) & \text{若 } \mathbf{x}_i \in C_k \text{（原属于被替换的簇）} \\
d(\mathbf{x}_i, \mathbf{x}_h) - d(\mathbf{x}_i, \mathbf{m}_{k_i}) & \text{若 } d(\mathbf{x}_i, \mathbf{x}_h) < d(\mathbf{x}_i, \mathbf{m}_{k_i}) \\
0 & \text{否则}
\end{cases}
\tag{5.22}
$$

其中 $$\mathbf{m}_{k_i}$$ 是 $$\mathbf{x}_i$$ 当前所属簇的中心点。

**SWAP 步的时间复杂度**：对 $$K$$ 个中心点，每个中心点有 $$n-K$$ 个候选替换对象，对每对替换需遍历 $$n-K$$ 个非中心点计算代价差，因此：

$$
\text{每次 SWAP 迭代复杂度} = O(K(n-K)^2) \tag{5.23}
$$

这比 K-means 的单步 $$O(nKd)$$ 高得多（当 $$n \gg K$$ 时，$$O(K(n-K)^2) \approx O(Kn^2)$$），使得 PAM 在大规模数据集上效率较低。

{% hint style="warning" %}
**大数据下的替代方案**：对 $$n > 10^4$$ 的数据集，PAM 的 $$O(Kn^2)$$ 复杂度通常不可接受。可使用：(1) **CLARA**（Clustering LARge Applications）：反复对数据子集运行 PAM，取最佳结果；(2) **CLARANS**（Clustering Large Applications based on RANdomized Search）：随机抽取替换候选，在图搜索框架下降低复杂度。sklearn 的 `KMedoids`（由 `scikit-learn-extra` 提供）支持 `method='pam'` 和 `method='alternate'` 两种模式，后者在大数据集上速度更快。
{% endhint %}

### 5.1.6.3 K-medoids 与 K-means 的对比

| 维度 | K-means | K-medoids（PAM） |
|------|---------|-----------------|
| 簇中心 | 均值（不必是数据点） | 中心点（必须是实际数据点） |
| 距离函数 | 仅适用欧氏距离 | **任意距离或相似度函数** |
| 异常值鲁棒性 | 低（均值受异常值影响大） | **高**（中心点选自密集区域） |
| 时间复杂度 | $$O(nKd \cdot T)$$，快 | $$O(K(n-K)^2 \cdot T)$$，慢 |
| 适用数据规模 | $$n$$ 可达百万量级 | 通常 $$n \leq 10^4$$（PAM） |
| 结果可解释性 | 质心可能无对应实体 | **中心点是真实样本，直接可解释** |
| 典型应用 | 大规模连续数据 | 小规模高噪声数据、非欧距离场景 |

## 5.1.7 算法比较与适用场景

经过前六个小节的分析，我们可以从多个维度对 K-means 和 K-medoids（及其变体）进行系统比较，并给出选择指南。

### 5.1.7.1 假设与适用条件

**K-means 的隐含假设**：

1. **球形簇**：K-means 的 Voronoi 划分（定义 5.4）本质上假设每个簇是以质心为圆心的"球形"区域，因此对于**细长的、月牙形的、密度不均的簇**表现差；
2. **大小相近**：若簇的大小差异很大，K-means 倾向于将大簇分裂；
3. **密度相近**：若簇的密度差异悬殊，K-means 无法区分高密度小簇和低密度大簇；
4. **欧氏距离有意义**：K-means 基于欧氏距离，在特征尺度不同时需要预先标准化。

**K-medoids 的优势场景**：

- 数据包含明显异常值（金融交易、传感器数据）；
- 需要用非欧距离（编辑距离、DTW、余弦相似度）；
- 簇的代表必须是真实样本（医疗诊断中，簇代表必须是一个真实病例）；
- 数据规模较小（$$n \leq 10^4$$），PAM 的高复杂度可以接受。

### 5.1.7.2 选择流程图

以下是一个实践中的算法选择框架：

```text
输入数据
    │
    ├── 数据规模 n > 10^5？
    │       ├── 是 → K-means（MiniBatch 变体），K-means++ 初始化
    │       └── 否 → 继续判断
    │
    ├── 需要非欧距离或类别数据？
    │       ├── 是 → K-medoids（PAM/CLARA）或 K-modes
    │       └── 否 → 继续判断
    │
    ├── 数据中存在明显异常值？
    │       ├── 是 → K-medoids（稳健性强）
    │       └── 否 → K-means（更快）
    │
    ├── 簇形状可能非球形或密度不均？
    │       ├── 是 → 考虑 DBSCAN（§5.2）或谱聚类（§5.5）
    │       └── 否 → K-means 即可
    │
    └── 输出：K-means 或 K-medoids
```

## 案例 5.1：购物中心客户细分

### 背景与数据

本案例使用经典的 **Mall Customers 数据集**（也可用合成 RFM 数据），包含 200 名购物中心顾客的年龄、年收入（千美元）和消费评分（1–100），目标是通过聚类发现具有不同消费行为的客户群体，为精准营销提供依据。

### 5.1 完整 sklearn 工作流

```python
import numpy as np
import pandas as pd
import matplotlib.pyplot as plt
from sklearn.preprocessing import StandardScaler
from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, silhouette_samples

# ─── 1. 生成模拟 RFM 数据（可替换为真实 Mall Customers 数据集）───
np.random.seed(42)
n_customers = 200

# 五类客户群：高价值、潜力、休眠、新客、流失
centers = [
    [5, 80, 90],   # 高价值：高频、高消费
    [3, 50, 60],   # 潜力：中等
    [1, 90, 20],   # 有钱但不爱花：高收入低消费
    [4, 30, 70],   # 年轻活跃：低收入高消费
    [1, 40, 30],   # 低价值：低频低消费
]
labels_true = []
data_list = []
for i, center in enumerate(centers):
    n = n_customers // len(centers)
    noise = np.random.randn(n, 3) * [0.8, 10, 12]
    data_list.append(np.array(center) + noise)
    labels_true.extend([i] * n)

df = pd.DataFrame(
    np.vstack(data_list),
    columns=["Recency", "Annual_Income_k", "Spending_Score"]
)
df = df.clip(lower=[0, 15, 1], upper=[10, 150, 100])

# ─── 2. 数据标准化（消除量纲影响）───
scaler = StandardScaler()
X_scaled = scaler.fit_transform(df)

# ─── 3. K 值选择：Elbow 法 + 轮廓系数 ───
K_range = range(2, 11)
wcss = []
silhouette_scores = []

for k in K_range:
    km = KMeans(n_clusters=k, init='k-means++', n_init=10, random_state=42)
    km.fit(X_scaled)
    wcss.append(km.inertia_)
    silhouette_scores.append(silhouette_score(X_scaled, km.labels_))

# 打印结果
print("K\tWCSS\t\tSilhouette")
print("-" * 40)
for k, w, s in zip(K_range, wcss, silhouette_scores):
    marker = " <-- 推荐" if s == max(silhouette_scores) else ""
    print(f"{k}\t{w:.2f}\t\t{s:.4f}{marker}")

# ─── 4. 最终聚类（K=5）───
K_best = 5
kmeans = KMeans(n_clusters=K_best, init='k-means++', n_init=20, random_state=42)
kmeans.fit(X_scaled)
df["Cluster"] = kmeans.labels_

# ─── 5. 各簇统计特征（反标准化后解释）───
print("\n各簇统计摘要：")
print(df.groupby("Cluster")[["Recency", "Annual_Income_k", "Spending_Score"]].mean().round(1))

# ─── 6. 可视化：收入 vs 消费评分的散点图 ───
cluster_names = {
    0: "高价值客户",
    1: "潜力客户",
    2: "高收入低消费",
    3: "年轻活跃客户",
    4: "低价值客户",
}

fig, axes = plt.subplots(1, 2, figsize=(14, 5))

# 散点图
colors = ["#E74C3C", "#3498DB", "#2ECC71", "#F39C12", "#9B59B6"]
for c in range(K_best):
    mask = df["Cluster"] == c
    axes[0].scatter(
        df.loc[mask, "Annual_Income_k"],
        df.loc[mask, "Spending_Score"],
        c=colors[c], label=cluster_names.get(c, f"簇{c}"), alpha=0.7, s=60
    )
# 质心（反标准化后）
centers_orig = scaler.inverse_transform(kmeans.cluster_centers_)
axes[0].scatter(
    centers_orig[:, 1], centers_orig[:, 2],
    c="black", marker="X", s=200, zorder=10, label="质心"
)
axes[0].set_xlabel("年收入（千美元）")
axes[0].set_ylabel("消费评分")
axes[0].set_title(f"K-means 客户细分（K={K_best}）")
axes[0].legend(loc="upper left", fontsize=8)

# Elbow 曲线
axes[1].plot(list(K_range), wcss, "bo-", linewidth=2, markersize=6)
axes[1].axvline(x=K_best, color="red", linestyle="--", label=f"K={K_best}（推荐）")
axes[1].set_xlabel("簇数 K")
axes[1].set_ylabel("WCSS（簇内平方和）")
axes[1].set_title("Elbow 曲线")
axes[1].legend()

plt.tight_layout()
plt.savefig("customer_segmentation.png", dpi=150, bbox_inches="tight")
plt.show()
```

1. **标准化不可省略**：原始数据中"年收入"的量纲（千美元）远大于"消费评分"（1–100），不标准化会导致 K-means 实际上只按收入聚类，忽略消费评分信息；
2. **轮廓系数比 Elbow 更稳健**：本案例中 $$K=5$$ 时轮廓系数最高（0.583），WCSS 曲线在 $$K=5$$ 处也有较明显的折点，两种方法一致指向 $$K=5$$；
3. **业务可解释性是验证标准**：技术指标（SSE、轮廓系数）只是辅助工具，最终聚类结果必须结合业务背景解释，若某个簇的解释无法支撑差异化策略，则应重新考虑 $$K$$ 的取值；
4. **K-means 对簇形状的假设**：若将数据可视化为"年收入 vs 消费评分"的二维散点图，可以直观确认簇大致呈球形，符合 K-means 的假设。

### K-medoids 对比

```python
# 使用 scikit-learn-extra 的 KMedoids（需安装：pip install scikit-learn-extra）
try:
    from sklearn_extra.cluster import KMedoids
    kmed = KMedoids(n_clusters=K_best, metric='euclidean', method='pam',
                    init='k-medoids++', random_state=42)
    kmed.fit(X_scaled)
    sil_kmed = silhouette_score(X_scaled, kmed.labels_)
    sil_kmeans = silhouette_score(X_scaled, kmeans.labels_)
    print(f"K-means  轮廓系数: {sil_kmeans:.4f}")
    print(f"K-medoids 轮廓系数: {sil_kmed:.4f}")
    # 打印 K-medoids 中心点（是真实客户记录）
    print("\nK-medoids 中心点（真实客户）：")
    medoid_indices = kmed.medoid_indices_
    print(df.iloc[medoid_indices][["Recency", "Annual_Income_k", "Spending_Score"]])
except ImportError:
    print("请安装 scikit-learn-extra: pip install scikit-learn-extra")
```

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 聚类形式化定义 | $$\gamma: \mathcal{D} \to \{1,\ldots,K\}$$，完备性 + 互斥性（定义 5.1）；划分型聚类硬分配 $$K$$ 个互斥簇（定义 5.2） |
| K-means 目标函数 | $$J = \sum_{k=1}^K \sum_{\mathbf{x} \in C_k} \|\mathbf{x} - \boldsymbol{\mu}_k\|^2$$（定义 5.3），NP 难，用 Lloyd 迭代求局部最优 |
| Voronoi 划分 | K-means 赋值步等价于 Voronoi 划分（定义 5.4）；每个样本分配到最近质心的 Voronoi 区域 |
| 均值最优性（定理 5.2） | 固定赋值下，最小化簇内 SSE 的唯一最优质心 $$= $$ 样本均值（严格凸性保证唯一性） |
| 收敛性（定理 5.1） | 每步严格不增 + 有限划分数 $$\Rightarrow$$ 有限步收敛于局部极小值；不保证全局最优 |
| K-means++ | $$D^2$$-加权概率初始化（公式 (5.13)），期望代价上界 $$\mathbb{E}[J] \leq 8(\ln K+2)J^*$$；sklearn 默认 `init='k-means++'` |
| K 值选择 | 肘部法则（WCSS 曲线，公式 (5.15)）；轮廓系数（公式 (5.17)），兼顾簇内紧凑性与簇间分离性 |
| K-medoids / PAM | 中心点限制为实际数据点，目标 $$\sum d(\mathbf{x}, \mathbf{m}_k)$$（公式 (5.19)）；SWAP 步复杂度 $$O(K(n-K)^2)$$；对异常值稳健，支持任意距离 |
| 算法比较 | K-means 快（$$O(nKd \cdot T)$$）但需欧氏距离；K-medoids 稳健但慢（$$O(K(n-K)^2 \cdot T)$$） |
| 客户细分案例 | 标准化 $$\to$$ K 值选择（Elbow + 轮廓系数）$$\to$$ K-means 聚类 $$\to$$ 业务解释；忽略标准化是最常见的实践错误 |
