# 5.3 基于层次的聚类算法

§5.1 的 K-means 需要预先指定簇数 $$K$$，§5.2 的 DBSCAN 虽不需要 $$K$$，但需要密度参数 $$(\varepsilon, \text{MinPts})$$。两者都输出一个**扁平**的聚类结果——数据被划分为互不重叠的若干组，且这种划分是**单一粒度**的。然而，许多真实数据具有**嵌套的层次结构**：生物界从"界→门→纲→目→科→属→种"逐级细分；企业组织从"集团→事业部→部门→团队→个人"层层分解；文档从"主题→子主题→段落→句子"递进细化。

**层次聚类**（Hierarchical Clustering）不输出单一的划分结果，而是构建一棵**树状图**（Dendrogram），完整地记录数据从"每个点自成一簇"到"所有点合为一簇"的全部合并（或分裂）过程。用户可以在树的任意高度"切一刀"，得到对应粒度的聚类结果——一次运行，多粒度输出。

## 5.3.1 层次聚类的两种策略

### 5.3.1.1 凝聚式与分裂式

{% block type="definition" title="定义 5.13　凝聚式层次聚类（Agglomerative Hierarchical Clustering）" %}
**凝聚式层次聚类**（又称 AGNES，AGglomerative NESting）是一种**自底向上**（bottom-up）策略：

1. **初始化**：将每个数据点视为一个独立的簇，共 $$n$$ 个簇；
2. **重复**：在当前所有簇对中，找到"最近"的一对，将其合并为一个簇；
3. **终止**：当所有数据点合并为单一簇时停止。

经过 $$n - 1$$ 次合并后，得到一棵完整的**合并树**（merge tree），每个内部节点记录两个子簇的合并及其距离。
{% endblock %}

{% block type="definition" title="定义 5.14　分裂式层次聚类（Divisive Hierarchical Clustering）" %}
**分裂式层次聚类**（又称 DIANA，DIvisive ANAlysis）是一种**自顶向下**（top-down）策略：

1. **初始化**：将所有数据点视为一个簇；
2. **重复**：选择一个"最不紧凑"的簇，将其分裂为两个子簇；
3. **终止**：当每个簇仅含一个数据点时停止。

分裂步通常通过在选定簇上运行 2-means 或最大直径启发式实现。
{% endblock %}

| 维度 | 凝聚式（AGNES） | 分裂式（DIANA） |
|------|----------------|----------------|
| 方向 | 自底向上（$$n$$ 个点 → 1 个簇） | 自顶向下（1 个簇 → $$n$$ 个点） |
| 每步决策 | 选择**最近**的一对簇合并 | 选择**最散**的一个簇分裂 |
| 分裂方法 | 不涉及 | 需要子算法（2-means 或启发式） |
| 时间复杂度 | $$O(n^2 \log n)$$（优先队列） | $$O(2^n)$$（精确最优分裂是 NP 难的） |
| 实际使用 | **广泛使用**（scipy、sklearn 默认） | 较少使用，因计算代价高 |

由于分裂式的精确最优分裂是 NP 难问题（从 $$n$$ 个点中选最优二分需枚举 $$2^{n-1} - 1$$ 种划分），实践中**凝聚式层次聚类**是绝对主流。以下内容以凝聚式为主。

### 5.3.1.2 层次聚类的核心问题：如何度量簇间距离？

凝聚式层次聚类的每一步都需要回答："哪一对簇最近？"这就要求定义**簇间距离**（cluster distance 或 linkage）。不同的簇间距离定义导致完全不同的合并顺序和最终树结构——这正是链接准则（linkage criterion）的作用。

明确了层次聚类的基本框架后，下一节将形式化四种经典链接准则，并分析它们各自的性质。

---

## 5.3.2 链接准则：四种经典方法

设 $$C_A$$ 和 $$C_B$$ 是两个待比较的簇，$$d(\mathbf{x}, \mathbf{y})$$ 是点间距离（默认欧氏距离）。四种经典链接准则定义如下：

{% block type="definition" title="定义 5.15　单链接（Single Linkage）" %}
单链接取两簇中**最近**的一对点之间的距离：

$$
d_{\text{single}}(C_A, C_B) = \min_{\mathbf{x} \in C_A,\, \mathbf{y} \in C_B} d(\mathbf{x}, \mathbf{y}) \tag{5.35}
$$

单链接倾向于合并**空间上相邻**的簇，能识别细长的、非凸的簇形状，但容易受噪声点"桥接"影响，产生**链式效应**（chaining effect）——一串噪声点可能将两个原本独立的簇串联起来。
{% endblock %}

{% block type="definition" title="定义 5.16　全链接（Complete Linkage）" %}
全链接取两簇中**最远**的一对点之间的距离：

$$
d_{\text{complete}}(C_A, C_B) = \max_{\mathbf{x} \in C_A,\, \mathbf{y} \in C_B} d(\mathbf{x}, \mathbf{y}) \tag{5.36}
$$

全链接倾向于产生**紧凑的球形**簇，对链式效应免疫，但对异常值敏感（一个异常值会使簇间最大距离膨胀）。
{% endblock %}

{% block type="definition" title="定义 5.17　均链接（Average Linkage，UPGMA）" %}
均链接取两簇中所有点对的**平均距离**：

$$
d_{\text{average}}(C_A, C_B) = \frac{1}{|C_A|\cdot|C_B|} \sum_{\mathbf{x} \in C_A} \sum_{\mathbf{y} \in C_B} d(\mathbf{x}, \mathbf{y}) \tag{5.37}
$$

均链接是单链接和全链接的折中：既不过于敏感于噪声桥接，也不过于保守于极端距离，在实践中使用广泛。UPGMA（Unweighted Pair Group Method with Arithmetic Mean）是其在系统发育学中的经典名称。
{% endblock %}

{% block type="definition" title="定义 5.18　Ward 链接（Ward's Method）" %}
Ward 链接将簇间距离定义为合并后**簇内平方和（SSE）的增量**：

$$
d_{\text{Ward}}(C_A, C_B) = \text{SSE}(C_A \cup C_B) - \text{SSE}(C_A) - \text{SSE}(C_B) \tag{5.38}
$$

其中 $$\text{SSE}(C) = \sum_{\mathbf{x} \in C} \|\mathbf{x} - \boldsymbol{\mu}_C\|^2$$。Ward 准则等价于选择使合并后**总方差增量最小**的那对簇进行合并。
{% endblock %}

Ward 准则的合并代价有一个简洁的封闭形式：

$$
d_{\text{Ward}}(C_A, C_B) = \frac{|C_A| \cdot |C_B|}{|C_A| + |C_B|} \|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2 \tag{5.39}
$$

其中 $$\boldsymbol{\mu}_A$$、$$\boldsymbol{\mu}_B$$ 分别是 $$C_A$$、$$C_B$$ 的质心。公式 (5.39) 表明：Ward 准则倾向于合并**质心接近**且**大小较小**的簇，天然偏好大小相近的球形簇——与 K-means 的行为类似。

| 链接准则 | 簇形状偏好 | 噪声敏感性 | 链式效应 | 计算效率 |
|---------|-----------|-----------|---------|---------|
| 单链接 | 任意形状（细长、非凸） | 高 | **严重** | 高（可用 MST） |
| 全链接 | 紧凑球形 | 高（异常值） | 无 | 中 |
| 均链接 | 中等紧凑 | 中 | 轻微 | 中 |
| Ward | 球形、等大 | 低 | 无 | 中 |

**实践建议**：当不确定数据结构时，**Ward 链接**是最稳健的默认选择（scipy 和 sklearn 的默认值）；当簇形状可能为细长或非凸时，考虑**均链接**或**单链接**。

四种链接准则定义了"簇间距离"的不同语义，下一节将给出凝聚式层次聚类的完整算法描述和高效实现。

## 5.3.3 凝聚式层次聚类算法

### 5.3.3.1 算法流程

```text
算法：Agglomerative Hierarchical Clustering(D, linkage)
─────────────────────────────────────────────────────
输入：数据集 D = {x_1, ..., x_n}，链接准则 linkage
输出：合并历史 Z（n-1 × 4 矩阵：[簇A, 簇B, 距离, 新簇大小]）

1. 初始化：n 个簇 C_1={x_1}, ..., C_n={x_n}
2. 计算 n×n 距离矩阵 dist[i][j] = d(C_i, C_j)
3. for step = 1 to n-1:
4.     (i*, j*) ← argmin_{i<j} dist[i][j]   # 找最近簇对
5.     记录合并：Z[step] = [i*, j*, dist[i*][j*], |C_i*|+|C_j*|]
6.     C_new ← C_i* ∪ C_j*
7.     # Lance-Williams 更新：对所有剩余簇 C_k，用公式更新 dist[new][k]
8.     删除 C_i* 和 C_j* 的行列，插入 C_new
9. return Z
```

### 5.3.3.2 Lance-Williams 更新公式

合并 $$C_A$$ 和 $$C_B$$ 为 $$C_{AB}$$ 后，新簇与任意未合并簇 $$C_K$$ 的距离可以统一用 **Lance-Williams 公式**表示：

$$
d(C_{AB}, C_K) = \alpha_A \, d(C_A, C_K) + \alpha_B \, d(C_B, C_K) + \beta \, d(C_A, C_B) + \gamma \, |d(C_A, C_K) - d(C_B, C_K)| \tag{5.40}
$$

不同链接准则对应不同的系数：

| 链接准则 | $$\alpha_A$$ | $$\alpha_B$$ | $$\beta$$ | $$\gamma$$ |
|---------|-----------|-----------|---------|---------|
| 单链接 | $$\frac{1}{2}$$ | $$\frac{1}{2}$$ | $$0$$ | $$-\frac{1}{2}$$ |
| 全链接 | $$\frac{1}{2}$$ | $$\frac{1}{2}$$ | $$0$$ | $$\frac{1}{2}$$ |
| 均链接 | $$\frac{n_A}{n_A+n_B}$$ | $$\frac{n_B}{n_A+n_B}$$ | $$0$$ | $$0$$ |
| Ward | $$\frac{n_A+n_K}{n_{AB}+n_K}$$ | $$\frac{n_B+n_K}{n_{AB}+n_K}$$ | $$-\frac{n_K}{n_{AB}+n_K}$$ | $$0$$ |

其中 $$n_A = |C_A|$$、$$n_B = |C_B|$$、$$n_K = |C_K|$$、$$n_{AB} = n_A + n_B$$。

Lance-Williams 公式的意义在于：合并后**无需重新遍历所有数据点**计算新簇与其他簇的距离，只需根据旧的簇间距离矩阵通过 $$O(1)$$ 运算更新——这是使算法复杂度降至 $$O(n^2 \log n)$$ 的关键。

### 5.3.3.3 时间与空间复杂度

{% block type="theorem" title="定理 5.7　凝聚式层次聚类复杂度" %}
凝聚式层次聚类的复杂度取决于最近簇对的查找方式：

- **朴素实现**（每步线性扫描）：$$n - 1$$ 步，每步扫描 $$O(n^2)$$ 的距离矩阵，共：

$$
T_{\text{naive}} = O(n^3) \tag{5.41}
$$

- **优先队列优化**（最小堆）：初始化 $$O(n^2)$$，每步提取最小值 $$O(\log n)$$，更新 $$O(n \log n)$$，共：

$$
T_{\text{heap}} = O(n^2 \log n) \tag{5.42}
$$

- **单链接特例**：等价于最小生成树（定理 5.5），可用 Prim 算法在 $$O(n^2)$$ 内完成。

空间复杂度：$$O(n^2)$$（存储距离矩阵）。
{% endblock %}

{% hint style="warning" %}
**数据规模限制**：$$O(n^2)$$ 的空间需求是层次聚类的主要瓶颈。$$n = 10^4$$ 时距离矩阵约 800 MB（双精度），$$n = 10^5$$ 时约 80 GB。因此，层次聚类适用于**中小规模数据集**（$$n \leq 10^4$$）。对于大规模数据，可使用 **BIRCH**（§5.4）或先用 K-means 缩减为少量代表点再做层次聚类。
{% endhint %}

### 5.3.3.4 Python 实现（从零实现）

```python
import numpy as np
from scipy.spatial.distance import pdist, squareform

def agglomerative_scratch(X, linkage='ward'):
    """凝聚式层次聚类从零实现（朴素 O(n³) 版本）"""
    n = len(X)
    # 初始化：每个点为一个簇
    clusters = {i: [i] for i in range(n)}
    centroids = {i: X[i].copy() for i in range(n)}
    # 计算初始点间距离矩阵
    dist_matrix = squareform(pdist(X, 'euclidean'))
    np.fill_diagonal(dist_matrix, np.inf)
    # 簇间距离矩阵（初始时等于点间距离）
    active = set(range(n))
    Z = np.zeros((n - 1, 4))  # [id_A, id_B, dist, size]
    next_id = n
    cluster_map = {i: i for i in range(n)}  # 活跃簇 → 行索引
    for step in range(n - 1):
        # 找最近簇对
        min_dist = np.inf
        best_i, best_j = -1, -1
        active_list = sorted(active)
        for idx_a, i in enumerate(active_list):
            for j in active_list[idx_a + 1:]:
                if dist_matrix[i, j] < min_dist:
                    min_dist = dist_matrix[i, j]
                    best_i, best_j = i, j
        # 记录合并
        size_new = len(clusters[best_i]) + len(clusters[best_j])
        Z[step] = [best_i, best_j, min_dist, size_new]
        # 合并簇
        clusters[best_i] = clusters[best_i] + clusters[best_j]
        pts = X[clusters[best_i]]
        centroids[best_i] = pts.mean(axis=0)
        del clusters[best_j]
        active.discard(best_j)
        # 更新距离（Ward 准则）
        for k in active:
            if k == best_i:
                continue
            if linkage == 'ward':
                n_i = len(clusters[best_i])
                n_k = len(clusters[k])
                d_new = (n_i * n_k / (n_i + n_k)) * np.sum(
                    (centroids[best_i] - centroids[k]) ** 2)
                dist_matrix[best_i, k] = d_new
                dist_matrix[k, best_i] = d_new
            elif linkage == 'single':
                dist_matrix[best_i, k] = min(
                    dist_matrix[best_i, k], dist_matrix[best_j, k])
                dist_matrix[k, best_i] = dist_matrix[best_i, k]
            elif linkage == 'complete':
                dist_matrix[best_i, k] = max(
                    dist_matrix[best_i, k], dist_matrix[best_j, k])
                dist_matrix[k, best_i] = dist_matrix[best_i, k]
            elif linkage == 'average':
                n_a = Z[step, 3] - len(clusters[k])
                n_a_old = Z[step, 3] - len(clusters.get(best_j, []))
                dist_matrix[best_i, k] = (
                    len(clusters[best_i]) * dist_matrix[best_i, k]
                    + len(clusters.get(best_j, [])) * dist_matrix[best_j, k]
                ) / (len(clusters[best_i]))
                dist_matrix[k, best_i] = dist_matrix[best_i, k]
        # 将被合并簇的距离设为 inf
        dist_matrix[best_j, :] = np.inf
        dist_matrix[:, best_j] = np.inf
    return Z
```

## 5.3.4 单链接与最小生成树的等价性

单链接层次聚类与图论中的**最小生成树**（Minimum Spanning Tree, MST）之间存在深刻的联系。

{% block type="theorem" title="定理 5.5　单链接等价于 MST 截断（Single Linkage ≡ MST Cut）" %}
设完全图 $$G = (\mathcal{D}, E)$$，其中每条边 $$(i, j)$$ 的权重为 $$d(\mathbf{x}_i, \mathbf{x}_j)$$，$$T$$ 是 $$G$$ 的**最小生成树**。则单链接层次聚类的合并顺序等价于：将 $$T$$ 的边按权重从小到大排序，依次加入对应边连接的两个连通分量。

**证明**：

**步骤 1**：单链接每步合并距离最近的两个簇 $$C_i$$、$$C_j$$，合并距离为 $$d_{\text{single}}(C_i, C_j) = \min_{\mathbf{x} \in C_i, \mathbf{y} \in C_j} d(\mathbf{x}, \mathbf{y})$$。

**步骤 2**：MST 的 Kruskal 算法每步选取全局最短的、连接不同连通分量的边 $$(u, v)$$，将 $$u$$ 和 $$v$$ 所在的连通分量合并。

**步骤 3**：在合并之前，$$C_i$$ 和 $$C_j$$ 是两个不同的连通分量。它们之间的单链接距离恰好是连接这两个连通分量的最短边——这正是 Kruskal 算法下一步选择的边。

**步骤 4**：由于每步的决策规则完全相同（选全局最短跨簇边），且初始状态相同（$$n$$ 个单点簇/$$n$$ 个单点连通分量），两个过程产生相同的合并历史。$$\square$$

**推论**：单链接层次聚类可在 $$O(n^2)$$ 时间内完成（Prim 算法构建 MST 后排序），而无需一般的 $$O(n^2 \log n)$$ 凝聚式框架。
{% endblock %}

这一等价性解释了单链接的**链式效应**：MST 沿最短路径连接点，若两个真实簇之间恰好有一串稀疏的"桥接"点，MST 会沿这串点建立连接，导致两个簇被过早合并。

## 5.3.5 Ward 准则的方差分解性质

{% block type="theorem" title="定理 5.6　Ward 合并代价的封闭形式" %}
合并簇 $$C_A$$（大小 $$n_A$$，质心 $$\boldsymbol{\mu}_A$$）和 $$C_B$$（大小 $$n_B$$，质心 $$\boldsymbol{\mu}_B$$）后，SSE 的增量为：

$$
\Delta\text{SSE} = \text{SSE}(C_A \cup C_B) - \text{SSE}(C_A) - \text{SSE}(C_B) = \frac{n_A n_B}{n_A + n_B} \|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2 \tag{5.43}
$$

**证明**：设合并后质心 $$\boldsymbol{\mu}_{AB} = \frac{n_A \boldsymbol{\mu}_A + n_B \boldsymbol{\mu}_B}{n_A + n_B}$$。对 $$C_A$$ 中的点：

$$
\sum_{\mathbf{x} \in C_A} \|\mathbf{x} - \boldsymbol{\mu}_{AB}\|^2 = \sum_{\mathbf{x} \in C_A} \|\mathbf{x} - \boldsymbol{\mu}_A + \boldsymbol{\mu}_A - \boldsymbol{\mu}_{AB}\|^2
$$

展开并利用 $$\sum_{\mathbf{x} \in C_A}(\mathbf{x} - \boldsymbol{\mu}_A) = \mathbf{0}$$（质心定义），交叉项消失：

$$
= \text{SSE}(C_A) + n_A \|\boldsymbol{\mu}_A - \boldsymbol{\mu}_{AB}\|^2
$$

代入 $$\boldsymbol{\mu}_A - \boldsymbol{\mu}_{AB} = \frac{n_B}{n_A + n_B}(\boldsymbol{\mu}_A - \boldsymbol{\mu}_B)$$：

$$
= \text{SSE}(C_A) + \frac{n_A n_B^2}{(n_A + n_B)^2} \|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2 \tag{5.44}
$$

同理对 $$C_B$$：

$$
\sum_{\mathbf{x} \in C_B} \|\mathbf{x} - \boldsymbol{\mu}_{AB}\|^2 = \text{SSE}(C_B) + \frac{n_B n_A^2}{(n_A + n_B)^2} \|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2 \tag{5.45}
$$

两式相加得：

$$
\text{SSE}(C_{AB}) = \text{SSE}(C_A) + \text{SSE}(C_B) + \frac{n_A n_B(n_A + n_B)}{(n_A + n_B)^2} \|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2
$$

化简即得公式 (5.43)。$$\square$$
{% endblock %}

**直觉解读**：公式 (5.43) 有优美的结构——$$\frac{n_A n_B}{n_A + n_B}$$ 是调和大小因子，$$\|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2$$ 是质心距离的平方。Ward 准则倾向于合并**质心接近**（$$\|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|$$ 小）且**大小相近**（调和因子不会太大）的簇。当两个簇大小相等（$$n_A = n_B = m$$）时，$$\Delta\text{SSE} = \frac{m}{2}\|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2$$，合并代价正比于质心距离平方。

Ward 准则与 K-means 有深刻的联系：K-means 最小化全局 SSE，而 Ward 层次聚类在每步贪心地最小化 SSE 增量——两者在球形等大簇上表现一致，且 Ward 层次聚类的结果常被用作 K-means 的初始化。

## 5.3.6 树状图（Dendrogram）：多粒度可视化

### 5.3.6.1 树状图的结构与解读

**树状图**（Dendrogram）是层次聚类结果的标准可视化，它将合并历史表示为一棵倒置的树：

- **叶节点**（底部）：每个原始数据点；
- **内部节点**：两个子簇的合并事件；
- **节点高度**（y 轴）：合并时的簇间距离（链接距离）；
- **水平线段**：连接被合并的两个子树。

<img src="/figures/ch05/fig5_3_01_dendrogram.png" alt="树状图与层次聚类过程" />
<figcaption>
<strong>图 5.3.1</strong>　凝聚式层次聚类的树状图（Ward 链接，$$n=30$$，三个真实簇）。<em>左</em>：完整树状图，y 轴为合并距离；红色虚线为在高度 $$h=8$$ 处的水平截断，与 3 条竖枝相交，得到 3 个簇。<em>右</em>：与左图对应的二维散点图，颜色标记截断后的簇归属。树状图中两簇合并的高度越大，表示这两簇之间的差异越显著——最后一次合并（树顶）的高度跳变最大，提示"从 2 簇到 1 簇"的合并代价远大于之前的合并。
</figcaption>
</figure>

### 5.3.6.2 从树状图中提取扁平聚类

从树状图中获取 $$K$$ 个簇有两种方式：

**方式一：指定高度 $$h$$**：在 y 轴高度 $$h$$ 处画一条水平线，切断树状图中高于 $$h$$ 的所有合并，得到若干棵子树，每棵子树即为一个簇。簇数 $$K$$ 等于水平线与竖枝的交点数。

**方式二：指定簇数 $$K$$**：等价于在树状图中找到第 $$K$$ 大的合并距离间隙，在该间隙上方截断。

```python
from scipy.cluster.hierarchy import linkage, fcluster, dendrogram

# 凝聚式层次聚类（Ward 链接）
Z = linkage(X, method='ward', metric='euclidean')

# 方式一：按高度截断
labels_h = fcluster(Z, t=8.0, criterion='distance')

# 方式二：按簇数截断
labels_k = fcluster(Z, t=3, criterion='maxclust')
```

### 5.3.6.3 树状图的"最优 K"选择

与 K-means 的 Elbow 法类似，层次聚类的簇数选择可通过观察树状图中**合并距离的跳变**来实现。设第 $$i$$ 步的合并距离为 $$d_i$$（$$i = 1, \ldots, n-1$$，从小到大排序），则**最大间隙**：

$$
K^* = n - \arg\max_{i \in \{1,\ldots,n-2\}} (d_{i+1} - d_i) \tag{5.46}
$$

对应于树状图中最高的"纵向空白区域"。实践中，scipy 的 `dendrogram` 函数会自动压缩小的合并（`truncate_mode='lastp'`），使大规模数据的树状图仍然可读。

多粒度可视化是层次聚类区别于扁平聚类方法的核心优势。下一节通过对比实验展示四种链接准则在不同数据分布上的表现差异。

## 5.3.7 链接准则对比实验

**实验结论**：

1. **Ward 是最稳健的默认选择**：在球形簇、等大小簇上表现最好，树状图结构最清晰；
2. **单链接适合非凸形状**：但必须确认数据中无噪声桥接，否则链式效应会破坏结果；
3. **均链接是折中方案**：对噪声和簇形状都有中等的容忍度；
4. **全链接偏好等直径簇**：对大小差异悬殊的簇不友好。

层次聚类的核心优势——树状图的多粒度结构和不需要预设 $$K$$——使其在探索性数据分析和具有自然层次结构的领域（生物分类学、文档主题分析、组织架构挖掘）中不可替代。下面的案例将展示这一优势在生物物种分类中的典型应用。

## 案例 5.3：基于形态特征的鸢尾花层次分类

**问题背景**：Fisher 的 Iris 数据集包含 150 朵鸢尾花的 4 个形态特征（花萼长宽、花瓣长宽）和 3 个物种标签（Setosa、Versicolor、Virginica）。我们假设物种标签未知，使用层次聚类发现数据中的自然分组，并通过树状图揭示物种之间的层次关系。

```python
import numpy as np
from sklearn.datasets import load_iris
from sklearn.preprocessing import StandardScaler
from sklearn.metrics import adjusted_rand_score
from scipy.cluster.hierarchy import linkage, fcluster, dendrogram
import matplotlib.pyplot as plt

# ── 1. 加载数据与标准化 ────────────────────────────────────
iris = load_iris()
X = StandardScaler().fit_transform(iris.data)
y_true = iris.target
species = iris.target_names

# ── 2. 四种链接准则对比 ────────────────────────────────────
methods = ['ward', 'complete', 'average', 'single']
for m in methods:
    Z = linkage(X, method=m)
    labels = fcluster(Z, t=3, criterion='maxclust')
    ari = adjusted_rand_score(y_true, labels)
    print(f"{m:10s}  ARI = {ari:.4f}")

# ── 3. Ward 链接详细分析 ──────────────────────────────────
Z_ward = linkage(X, method='ward')
labels_ward = fcluster(Z_ward, t=3, criterion='maxclust')

# ── 4. 树状图可视化 ───────────────────────────────────────
fig, axes = plt.subplots(1, 2, figsize=(16, 6))
# 树状图
dn = dendrogram(Z_ward, ax=axes[0], truncate_mode='lastp', p=30,
                leaf_rotation=90, leaf_font_size=8,
                color_threshold=Z_ward[-3, 2])
axes[0].set_title('Ward 链接树状图')
axes[0].set_xlabel('样本（合并后）')
axes[0].set_ylabel('合并距离')
axes[0].axhline(y=Z_ward[-3, 2], color='r', linestyle='--', alpha=0.7)
# 散点图（前两个主成分方向）
from sklearn.decomposition import PCA
X_pca = PCA(n_components=2).fit_transform(X)
colors = ['#2563eb', '#dc2626', '#16a34a']
for k in range(3):
    mask = labels_ward == k + 1
    axes[1].scatter(X_pca[mask, 0], X_pca[mask, 1],
                    c=colors[k], label=f'簇 {k+1}', alpha=0.7, s=50)
axes[1].set_title('层次聚类结果（PCA 投影）')
axes[1].set_xlabel('PC1')
axes[1].set_ylabel('PC2')
axes[1].legend()
plt.tight_layout()
plt.show()

# ── 5. 层次结构解读 ──────────────────────────────────────
print(f"\n最后 5 次合并距离：{Z_ward[-5:, 2].round(2)}")
print(f"最大间隙位于倒数第 {np.argmax(np.diff(Z_ward[-5:, 2]))+1} 次合并")
```

**业务解读**：

- **Ward 链接 ARI ≈ 0.73**：在无标签条件下，层次聚类仅凭形态特征就将 150 朵花分为三组，与真实物种高度一致；
- **Setosa 完美分离**：树状图中，Setosa 与其余两种在很低的高度就分离开来（最早的大间隙），这与生物学事实一致——Setosa 花瓣显著更小；
- **Versicolor 与 Virginica 难分**：两者在特征空间中有重叠，树状图中它们的合并距离较小，分离较晚；
- **层次结构的生物学意义**：树状图的嵌套结构暗示 Versicolor 和 Virginica 在进化上更近（花瓣形态相似），而 Setosa 是外群——这与植物分类学的结论吻合。

## 本节小结

| 关键词 | 核心概念 |
|--------|---------|
| 凝聚式层次聚类（定义 5.13） | 自底向上，$$n-1$$ 步合并，输出合并树/树状图 |
| 分裂式层次聚类（定义 5.14） | 自顶向下，精确最优分裂是 NP 难的，实践中较少使用 |
| 单链接（定义 5.15） | $$\min$$ 距离，识别非凸形状，但有链式效应 |
| 全链接（定义 5.16） | $$\max$$ 距离，紧凑球形簇，对异常值敏感 |
| 均链接（定义 5.17） | 平均距离，折中方案，UPGMA |
| Ward 链接（定义 5.18） | 最小化 SSE 增量，偏好球形等大簇，最稳健的默认选择 |
| Ward 封闭形式（定理 5.6） | $$\Delta\text{SSE} = \frac{n_A n_B}{n_A+n_B}\|\boldsymbol{\mu}_A - \boldsymbol{\mu}_B\|^2$$（公式 (5.43)） |
| MST 等价性（定理 5.5） | 单链接 ≡ MST 按边权排序依次合并，$$O(n^2)$$ |
| Lance-Williams 公式 | 统一的簇间距离更新公式，$$O(1)$$ 更新代替 $$O(n)$$ 重算 |
| 复杂度（定理 5.7） | 朴素 $$O(n^3)$$，优先队列 $$O(n^2 \log n)$$，空间 $$O(n^2)$$ |
| 树状图（Dendrogram） | 多粒度可视化，按高度或簇数截断得到扁平聚类 |
| 最优 K 选择 | 树状图中最大间隙（公式 (5.46)） |
