# 5.2 基于密度的聚类算法（DBSCAN, OPTICS）

§5.1 的 K-means 与 K-medoids 依赖"距最近质心的距离"来划分样本，这一机制天然偏爱**球形、凸性、大小相近**的簇。然而真实数据中的类别往往呈现月牙形、同心圆或不规则延伸的拓扑结构，且常混有游离于任何群体之外的**噪声点**。K-means 无法识别这类形状，更无法将噪声与有效数据区分开来。

**密度聚类**从一个截然不同的视角出发：不预设簇的形状，而是以"**局部密度**"来定义社区结构——样本密集的区域构成簇的核心，稀疏区域形成簇的边界或充当噪声。本节系统介绍两种最重要的密度聚类方法：**DBSCAN**（基于密度的带噪声空间聚类）和 **OPTICS**（利用排序识别聚类结构）。

## 5.2.1 密度视角：从球形簇到任意形状

### 5.2.1.1 K-means 的结构性局限

设想将两个月牙形点云交错排列（`make_moons`），无论如何选择 $$K=2$$ 个质心，K-means 的赋值步都会沿**垂直平分线**划分空间（Voronoi 划分），将每个月牙横腰切断，把本属于同一月牙的点分给不同的簇。这不是参数问题，而是 K-means **目标函数的结构性局限**——最小化球形距离无法捕获流形状结构。

| 局限 | K-means | 密度聚类 |
|------|---------|---------|
| 簇形状假设 | 凸/球形 | 任意形状 |
| 簇大小 | 偏好近似等大 | 可高度不均 |
| 噪声处理 | 所有点必须归簇 | 显式识别噪声点 |
| 需要预设 K | 是 | 否（自动确定簇数） |
| 对异常值 | 质心被拉偏 | 噪声点不影响簇 |

### 5.2.1.2 密度的直觉定义

在欧氏空间中，"密度"直觉上是指**单位体积内的点数**。给定半径 ε，点 $$\mathbf{x}$$ 处的局部密度可用其 **ε-邻域内的点数** $$|N_\varepsilon(\mathbf{x})|$$ 来衡量。密度高的区域（$$|N_\varepsilon| \geq \text{MinPts}$$）构成簇的核心，密度低的区域要么是边界，要么是噪声。

明确了密度的直觉之后，下一节将把这一直觉精确化为六个可操作的形式化定义，从而构建 DBSCAN 的完整概念体系。

## 5.2.2 DBSCAN 的形式化概念体系

DBSCAN（Density-Based Spatial Clustering of Applications with Noise）由 Ester 等人于 1996 年提出，其核心贡献是用**六个递进定义**将"密度相连的区域"形式化为可计算的数学对象。

{% block type="definition" title="定义 5.5　ε-邻域（ε-Neighborhood）" %}
给定数据集 $$\mathcal{D} \subset \mathbb{R}^d$$、参数 $$\varepsilon > 0$$ 和距离函数 $$d(\cdot,\cdot)$$，点 $$\mathbf{p}$$ 的 **ε-邻域**定义为：

$$
N_\varepsilon(\mathbf{p}) = \{\mathbf{q} \in \mathcal{D} : d(\mathbf{p}, \mathbf{q}) \leq \varepsilon\} \tag{5.24}
$$

即以 $$\mathbf{p}$$ 为圆心、半径为 $$\varepsilon$$ 的超球内的所有数据点（含 $$\mathbf{p}$$ 本身）。
{% endblock %}

{% block type="definition" title="定义 5.6　核心点（Core Point）" %}
给定 MinPts $$\in \mathbb{N}^+$$，若点 $$\mathbf{p}$$ 的 ε-邻域内至少包含 MinPts 个点，则称 $$\mathbf{p}$$ 为**核心点**：

$$
\mathbf{p} \text{ 是核心点} \iff |N_\varepsilon(\mathbf{p})| \geq \text{MinPts} \tag{5.25}
$$

核心点位于密集区域的内部，是 DBSCAN 建立连通性的基础。非核心点分为**边界点**（位于某核心点邻域内）和**噪声点**（不属于任何核心点邻域）。
{% endblock %}

{% block type="definition" title="定义 5.7　直接密度可达（Directly Density-Reachable）" %}
若 $$\mathbf{p}$$ 是核心点，且 $$\mathbf{q} \in N_\varepsilon(\mathbf{p})$$，则称 $$\mathbf{q}$$ 从 $$\mathbf{p}$$ **直接密度可达**（记 $$\mathbf{p} \xrightarrow{\text{dir}} \mathbf{q}$$）：

$$
\mathbf{p} \xrightarrow{\text{dir}} \mathbf{q} \iff \mathbf{q} \in N_\varepsilon(\mathbf{p}) \;\text{ 且 }\; |N_\varepsilon(\mathbf{p})| \geq \text{MinPts} \tag{5.26}
$$

注意直接密度可达是**不对称**关系：若 $$\mathbf{q}$$ 不是核心点，则 $$\mathbf{q} \xrightarrow{\text{dir}} \mathbf{p}$$ 不一定成立，即使 $$d(\mathbf{p},\mathbf{q}) \leq \varepsilon$$。
{% endblock %}

{% block type="definition" title="定义 5.8　密度可达（Density-Reachable）" %}
若存在点链 $$\mathbf{p} = \mathbf{o}_1, \mathbf{o}_2, \ldots, \mathbf{o}_\ell = \mathbf{q}$$，使得对所有 $$i = 1,\ldots,\ell-1$$ 均有 $$\mathbf{o}_i \xrightarrow{\text{dir}} \mathbf{o}_{i+1}$$，则称 $$\mathbf{q}$$ 从 $$\mathbf{p}$$ **密度可达**：

$$
\mathbf{p} \xrightarrow{\text{reach}} \mathbf{q} \iff \exists\, \mathbf{o}_1, \ldots, \mathbf{o}_\ell :\; \mathbf{p} = \mathbf{o}_1 \xrightarrow{\text{dir}} \mathbf{o}_2 \xrightarrow{\text{dir}} \cdots \xrightarrow{\text{dir}} \mathbf{o}_\ell = \mathbf{q} \tag{5.27}
$$

密度可达是直接密度可达的**传递闭包**，同样是不对称关系。链中除最后一个点外，每个点都必须是核心点。
{% endblock %}

{% block type="definition" title="定义 5.9　密度连接（Density-Connected）" %}
若存在核心点 $$\mathbf{o}$$，使得 $$\mathbf{p}$$ 和 $$\mathbf{q}$$ 都从 $$\mathbf{o}$$ 密度可达，则称 $$\mathbf{p}$$ 与 $$\mathbf{q}$$ **密度连接**：

$$
\mathbf{p} \leftrightarrow \mathbf{q} \iff \exists\, \mathbf{o} \in \mathcal{D} :\; \mathbf{o} \xrightarrow{\text{reach}} \mathbf{p} \;\text{ 且 }\; \mathbf{o} \xrightarrow{\text{reach}} \mathbf{q} \tag{5.28}
$$

密度连接是**对称**关系（不同于密度可达），是定义簇的基础。
{% endblock %}

{% block type="definition" title="定义 5.10　DBSCAN 簇与噪声点" %}
给定参数 $$(\varepsilon, \text{MinPts})$$，数据集 $$\mathcal{D}$$ 的一个 **DBSCAN 簇** $$C \subseteq \mathcal{D}$$ 是满足以下两个条件的非空集合：

**（极大性）** 若 $$\mathbf{p} \in C$$、$$\mathbf{p}$$ 是核心点，且 $$\mathbf{p} \xrightarrow{\text{reach}} \mathbf{q}$$，则 $$\mathbf{q} \in C$$；

**（连通性）** 对任意 $$\mathbf{p}, \mathbf{q} \in C$$，有 $$\mathbf{p} \leftrightarrow \mathbf{q}$$。

$$
C = \bigl\{\mathbf{q} \in \mathcal{D} : \exists\, \text{核心点 } \mathbf{p} \in C,\; \mathbf{p} \xrightarrow{\text{reach}} \mathbf{q}\bigr\} \tag{5.29}
$$

不属于任何簇的点称为**噪声点**，记为 $$C_{\text{noise}} = \mathcal{D} \setminus \bigcup_i C_i$$。
{% endblock %}

## 5.2.3 DBSCAN 算法：从定义到实现

### 5.2.3.1 算法流程

DBSCAN 的算法设计直接遵循定义 5.10：从未访问的核心点出发，沿密度可达链扩张，直到无法再添加新点为止。

```text
算法：DBSCAN(D, ε, MinPts)
─────────────────────────────────────────────
输入：数据集 D，邻域半径 ε，最小点数 MinPts
输出：簇集合 {C_1, C_2, ...}，噪声集合 C_noise

1. 将所有点标记为 UNVISITED
2. cluster_id ← 0
3. for each UNVISITED point p in D:
4.     将 p 标记为 VISITED
5.     N ← RangeQuery(D, p, ε)
6.     if |N| < MinPts:
7.         标记 p 为 NOISE（暂时，可能后续加入邻近簇）
8.     else:
9.         cluster_id ← cluster_id + 1
10.        C ← new cluster(cluster_id)
11.        ExpandCluster(D, p, N, C, ε, MinPts)
12. return 所有簇 {C} 和噪声点

ExpandCluster(D, p, N, C, ε, MinPts):
─────────────────────────────────────
1. 将 p 加入 C；Seeds ← N
2. while Seeds 非空:
3.     q ← Seeds.pop()
4.     if q 未 VISITED:
5.         将 q 标记为 VISITED
6.         M ← RangeQuery(D, q, ε)
7.         if |M| ≥ MinPts: Seeds ← Seeds ∪ M
8.     if q 尚未归属任何簇: 将 q 加入 C
```

### 5.2.3.2 Python 实现（kd-Tree 加速）

```python
import numpy as np
from scipy.spatial import cKDTree

def dbscan_scratch(X, eps, min_samples):
    """DBSCAN 从零实现，kd-Tree 加速邻域查询"""
    n = len(X)
    labels = np.full(n, -2)   # -2=未访问, -1=噪声, >=0=簇ID
    tree = cKDTree(X)

    cluster_id = 0
    for i in range(n):
        if labels[i] != -2:
            continue
        neighbors = tree.query_ball_point(X[i], eps)
        if len(neighbors) < min_samples:
            labels[i] = -1
            continue
        labels[i] = cluster_id
        seeds = list(neighbors)
        j = 0
        while j < len(seeds):
            q = seeds[j]; j += 1
            if labels[q] == -1:
                labels[q] = cluster_id       # 噪声→边界点
            if labels[q] != -2:
                continue
            labels[q] = cluster_id
            q_nbrs = tree.query_ball_point(X[q], eps)
            if len(q_nbrs) >= min_samples:
                seeds.extend(q_nbrs)         # 核心点：继续扩张
        cluster_id += 1
    return labels
```

### 5.2.3.3 时间复杂度分析

{% block type="theorem" title="定理 5.4　DBSCAN 时间复杂度" %}
DBSCAN 的时间复杂度取决于邻域查询实现方式：

- **暴力法**：每次查询 $$O(nd)$$，$$n$$ 次查询共 $$O(n^2 d)$$；
- **kd-Tree**（低维 $$d \lesssim 20$$）：构建 $$O(nd\log n)$$，每次范围查询期望 $$O(\log n + \bar{k})$$，$$n$$ 次共：

$$
T_{\text{kd-Tree}} = O(nd\log n) \tag{5.30}
$$

其中 $$\bar{k}$$ 为平均邻域大小。空间复杂度 $$O(nd)$$。

高维时（$$d > 20$$）kd-Tree 退化为暴力法，可改用**球树**（Ball Tree）或**局部敏感哈希**（LSH）。
{% endblock %}

## 5.2.4 唯一性定理与算法正确性

{% block type="theorem" title="定理 5.3　DBSCAN 簇唯一性（Cluster Uniqueness）" %}
给定数据集 $$\mathcal{D}$$、参数 $$(\varepsilon, \text{MinPts})$$，DBSCAN 输出的**核心点归属**与**噪声点判定**由数据唯一确定，与算法扫描点的顺序无关。

**证明**：

**步骤 1（核心点集唯一）**：$$N_\varepsilon(\mathbf{p})$$ 仅依赖 $$\mathbf{p}$$ 的坐标，与处理顺序无关。故核心点集 $$\mathcal{P}_{\text{core}}$$ 唯一确定。

**步骤 2（密度可达关系唯一）**：密度可达 $$\xrightarrow{\text{reach}}$$ 是由 $$\mathcal{P}_{\text{core}}$$ 上的有向图定义的传递闭包，与处理顺序无关。

**步骤 3（核心点等价类唯一）**：密度连接在 $$\mathcal{P}_{\text{core}}$$ 上诱导等价关系（自反、对称、传递，见习题 B1 证明）。每个等价类对应唯一一个簇的核心集。

**步骤 4（边界点）**：边界点可能同时位于多个簇的边界，其归属依赖扫描顺序，**不保证唯一**（原始 DBSCAN 的已知歧义）。DBSCAN*（确定性变体）通过"最近核心点"规则消除此歧义。$$\square$$
{% endblock %}

{% hint style="warning" %}
**边界点的歧义性**：定理 5.3 仅保证核心点归属的唯一性。处于多个簇边界的点可能因扫描顺序不同而归属不同簇。若需严格可重现性，应固定随机种子或使用 DBSCAN*。
{% endhint %}

## 5.2.5 参数选择：k-距离图方法

### 5.2.5.1 MinPts 的经验规则

对 $$d$$ 维数据，通常取：

$$
\text{MinPts} \geq d + 1 \tag{5.31}
$$

更常用的规则：$$\text{MinPts} = 2d$$（低维）；对于 $$n \leq 1000$$ 取 4，$$n > 10000$$ 取 10～20。

### 5.2.5.2 ε 的 k-距离图方法

固定 MinPts 后，通过 **k-距离图**选择 ε：

对每个点，计算到第 $$k = \text{MinPts} - 1$$ 近邻的距离：

$$
k\text{-dist}(\mathbf{p}) = \text{第 } k \text{ 小的 } d(\mathbf{p}, \mathbf{q}),\quad \mathbf{q} \in \mathcal{D} \setminus \{\mathbf{p}\} \tag{5.32}
$$

将所有 $$n$$ 个点的 $$k\text{-dist}$$ 值从大到小排序，绘制折线图；图中**肘部**（曲率最大处）即为推荐 ε 值。

```python
from sklearn.neighbors import NearestNeighbors
import numpy as np

def k_distance_plot(X, min_samples=5):
    k = min_samples - 1
    nbrs = NearestNeighbors(n_neighbors=k).fit(X)
    distances, _ = nbrs.kneighbors(X)
    k_dist_sorted = np.sort(distances[:, -1])[::-1]
    # 自动检测肘部
    d2 = np.diff(np.diff(k_dist_sorted))
    elbow_idx = np.argmax(np.abs(d2)) + 1
    eps_suggest = k_dist_sorted[elbow_idx]
    return eps_suggest, k_dist_sorted
```

## 5.2.6 OPTICS：连续密度层次的可视化

**OPTICS**（Ankerst 等 1999）不直接输出簇划分，而是输出数据点的有序序列和对应的**可达距离图**，以单次运行揭示数据在多个密度层次上的结构。

{% block type="definition" title="定义 5.11　核心距离（Core Distance）" %}
点 $$\mathbf{p}$$ 的**核心距离**为使其成为核心点所需的最小邻域半径：

$$
\text{cd}(\mathbf{p}) =
\begin{cases}
\text{UNDEFINED} & \text{若 } |N_\varepsilon(\mathbf{p})| < \text{MinPts} \\[4pt]
\text{到第 MinPts 近邻的距离} & \text{否则}
\end{cases} \tag{5.33}
$$
{% endblock %}

{% block type="definition" title="定义 5.12　可达距离（Reachability Distance）" %}
从核心点 $$\mathbf{o}$$ 到任意点 $$\mathbf{p}$$ 的**可达距离**为：

$$
\text{rd}(\mathbf{p}, \mathbf{o}) =
\begin{cases}
\text{UNDEFINED} & \text{若 } \mathbf{o} \text{ 不是核心点} \\[4pt]
\max\!\bigl(\text{cd}(\mathbf{o}),\; d(\mathbf{p}, \mathbf{o})\bigr) & \text{否则}
\end{cases} \tag{5.34}
$$

"截断"效果：当 $$\mathbf{p}$$ 在 $$\mathbf{o}$$ 的核心邻域内时，可达距离不低于 $$\text{cd}(\mathbf{o})$$，消除高密度区内的距离波动。
{% endblock %}

OPTICS 算法维护一个按可达距离排序的优先队列，优先处理可达距离最小的点，生成有序点列表。**可达距离图**（x=有序点序列，y=可达距离）的形态直接反映密度结构：

- **深谷（低可达距离区域）** → 密集簇内部
- **高峰（高可达距离）** → 簇间稀疏区域或噪声

从可达距离图提取簇的标准方法是 **ξ-陡峭区域**算法（sklearn 默认），找到曲线斜率变化超过阈值 ξ 的陡降-陡升区域对。

```python
from sklearn.cluster import OPTICS

op = OPTICS(min_samples=5, xi=0.05, min_cluster_size=0.05)
op.fit(X)

# 有序序列的可达距离
reach = op.reachability_[op.ordering_]
labels = op.labels_[op.ordering_]
# op.labels_[i]: 点 i 的簇标签（-1 为噪声）
```

## 5.2.7 DBSCAN 的局限性与改进方向

| 局限 | 原因 | 改进方法 |
|------|------|---------|
| 密度差异大时失效 | 单一 ε 无法同时适应高/低密度簇 | OPTICS、HDBSCAN |
| 高维数据退化 | 维度诅咒使 ε-邻域意义模糊 | 降维（PCA/UMAP）后再聚类 |
| 参数敏感 | ε、MinPts 微小变化可引起结果剧变 | k-距离图 + 多参数扫描 |
| 边界点不确定 | 同一边界点可归属多簇 | DBSCAN*（确定性变体） |

**HDBSCAN**（McInnes 等 2017）是目前最广泛使用的改进版：构建密度层次树，自动确定稳定簇，仅需 `min_cluster_size` 一个直觉参数，无需指定 ε，已成为现代密度聚类的首选工具。

## 案例 5.2：城市停车热点检测（GPS 轨迹密度聚类）

**问题背景**：共享单车运营商收集了某区域 3000 个 GPS 停车坐标，需识别**停车热点**（高密度停车区）和**异常停车点**（孤立停在非常规位置的单车）。这是密度聚类的典型场景：热点形状不规则，异常点应识别为噪声。

```python
import numpy as np
from sklearn.cluster import DBSCAN
from sklearn.neighbors import NearestNeighbors

np.random.seed(42)

# ── 1. 模拟 GPS 数据 ────────────────────────────────────────
# 4 个规则热点 + 1 个月牙形热点 + 噪声
hotspots = [
    ([116.38, 39.92], [[0.003, 0.001], [0.001, 0.002]], 600),
    ([116.41, 39.90], [[0.002,-0.001],[-0.001, 0.003]], 800),
    ([116.35, 39.88], [[0.005, 0.002], [0.002, 0.001]], 500),
    ([116.44, 39.93], [[0.001, 0.000], [0.000, 0.004]], 400),
]
X_parts = [np.random.multivariate_normal(c, cov, n)
           for c, cov, n in hotspots]
# 月牙形热点（沿商业街分布）
t = np.linspace(0, np.pi, 400)
moon = np.column_stack([
    116.39 + 0.03*np.cos(t) + np.random.normal(0, 0.002, 400),
    39.91  + 0.02*np.sin(t) + np.random.normal(0, 0.002, 400)
])
X_parts.append(moon)
# 异常停车（噪声）
noise = np.column_stack([
    np.random.uniform(116.33, 116.47, 100),
    np.random.uniform(39.86,  39.95,  100)
])
X_parts.append(noise)
X = np.vstack(X_parts)

# ── 2. k-距离图选 ε ─────────────────────────────────────────
MIN_SAMPLES = 10
nbrs = NearestNeighbors(n_neighbors=MIN_SAMPLES-1).fit(X)
distances, _ = nbrs.kneighbors(X)
k_dist = np.sort(distances[:, -1])[::-1]
# 从图中读取肘部，推荐 ε ≈ 0.008（经纬度单位）

# ── 3. DBSCAN 聚类 ──────────────────────────────────────────
EPS = 0.008
db = DBSCAN(eps=EPS, min_samples=MIN_SAMPLES, algorithm='kd_tree')
labels = db.fit_predict(X)

n_clusters = len(set(labels)) - (1 if -1 in labels else 0)
n_noise    = np.sum(labels == -1)
print(f"识别出 {n_clusters} 个停车热点，{n_noise} 个异常停车点")

# ── 4. 热点特征分析 ─────────────────────────────────────────
for k in range(n_clusters):
    pts = X[labels == k]
    center = pts.mean(axis=0)
    print(f"热点 {k+1}: 中心=({center[0]:.4f}, {center[1]:.4f}), "
          f"点数={len(pts)}")
```

**业务解读**：

- **停车热点（5 簇）**：对应地铁站出口、商场门口等高需求节点，作为调度补货的优先区域；月牙形热点沿商业街分布，密度聚类正确识别其形状，而 K-means 会将其横切为两段；
- **异常停车点（约 97 个）**：散落在居民区深处、立交桥旁，可安排定向回收；
- **工程提示**：真实 GPS 数据应使用 `DBSCAN(metric='haversine')` 并将经纬度转为弧度；百万级数据建议改用 HDBSCAN。

## 本节小结

| 核心概念 | 定义 | 作用 |
|----------|------|------|
| ε-邻域（定义 5.5） | $$N_\varepsilon(\mathbf{p}) = \{\mathbf{q}: d \leq \varepsilon\}$$ | 度量局部密度 |
| 核心点（定义 5.6） | $$\|N_\varepsilon(\mathbf{p})\| \geq \text{MinPts}$$ | 簇的种子 |
| 直接密度可达（定义 5.7） | $$\mathbf{q} \in N_\varepsilon(\mathbf{p})$$ 且 $$\mathbf{p}$$ 是核心点 | 基础可达关系 |
| 密度可达（定义 5.8） | 直接密度可达的传递闭包（不对称） | 定义簇连通性 |
| 密度连接（定义 5.9） | 存在公共密度可达核心点（对称） | 簇的最终定义 |
| 核心距离（定义 5.11） | 使 $$\mathbf{p}$$ 成为核心点的最小半径 | OPTICS 基础 |
| 可达距离（定义 5.12） | $$\max(\text{cd}(\mathbf{o}), d(\mathbf{p},\mathbf{o}))$$ | 平滑高密度区距离 |
| 唯一性（定理 5.3） | 核心点归属由数据和参数唯一确定 | 算法正确性保证 |
| 时间复杂度（定理 5.4） | kd-Tree 下 $$O(nd\log n)$$ | 工程实现指导 |
