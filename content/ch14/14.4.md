# 14.4 模型开发与评估流程

质量关口 G2 通过后，项目进入 CRISP-DM 的阶段 4（建模）和阶段 5（评估）——这也是数据科学家最熟悉的"技术核心区"。然而，"写代码训练模型"和"系统化的模型开发"之间仍有很大差距。后者要求**可重复、可比较、可追溯**的实验管理，而非在 Notebook 中随手调参、覆盖结果、最终连"最好的模型是哪一版"都说不清。

§3–§4 和 §13 的实践案例已经展示了各种模型的训练方法；本节聚焦于**过程管理**——如何组织实验使之高效、可靠、可审计。

{% hint style="info" %}
📋 **本节学习目标**

完成本节学习后，读者应能够：

1. 建立基线模型作为改进的参照锚点，理解"无基线则无进步"的原则；
2. 设计规范的实验方案：固定评估协议、控制单一变量、记录全量元数据；
3. 使用统计检验方法（而非直觉）判断模型改进是否显著；
4. 运用实验追踪工具（如 MLflow）实现实验的可重复性和可追溯性；
5. 编写实验报告，为质量关口 G4 做好准备。
{% endhint %}

**前置知识**：§3.7 分类算法评估（交叉验证、ROC 曲线），§4.6 偏差-方差分解与超参数选择。

## 14.4.1 基线优先原则

模型开发的第一步不是寻找最优模型，而是建立**基线**（baseline）。基线有两个不可替代的作用：

1. **验证数据管道**：如果最简单的模型都无法运行或产出合理结果，说明数据管道存在错误——越早发现越容易修复；
2. **提供参照锚点**：没有基线，就无法回答"这个复杂模型到底比简单方案好了多少"。

### 基线选择指南

| 任务类型 | 推荐基线 | 预期效果 |
|---------|---------|---------|
| 二分类 | 逻辑回归（默认参数） | 如果数据中存在线性可分信号，LR 通常能达到最终模型 80–90% 的效果 |
| 多分类 | 逻辑回归（one-vs-rest） | 同上 |
| 回归 | 线性回归 / Ridge（默认 α） | 如 §13.7 所示，线性基线有时就是最优方案 |
| 聚类 | K-means（k 由肘部法确定） | 为后续层次/密度方法提供对比基准 |
| 推荐 | 用户均值 / 全局均值 | 如 §13.5 所示，均值基线的 RMSE 是最低可接受标准 |

{% hint style="tip" %}
💡 **基线不是浪费时间，而是节省时间**

一个常见的反对意见是"为什么要花时间在一个注定会被替代的简单模型上"。答案是：(a) 基线通常只需 1–2 小时即可完成（§14.1 定义 14.2 的 MVM）；(b) 如果基线已经接近目标指标，说明问题比预想的简单，可以节省后续大量调参时间；(c) 如果基线远低于预期，可以在投入更多时间之前及时调整目标或数据策略。
{% endhint %}

## 14.4.2 实验设计规范

建立基线后，模型改进以**实验**为基本单元。规范化的实验设计确保每次改进都是可验证的、可归因的。

{% block type="definition" %}
**定义 14.11**（模型实验规范）　每一次模型实验须满足以下五项规范：

1. **固定评估协议**：数据划分方式（随机分割 / 时间分割 / 分层抽样）在整个项目中保持一致。不允许因为"这种分割方式效果更好"而中途更换。

2. **单一变量控制**：每次实验只改变一个因素（特征集 / 模型类型 / 超参数），其他因素保持不变。这样才能准确归因"性能变化是由什么导致的"。

3. **全量元数据记录**：每次实验记录——实验 ID、时间戳、数据版本、特征列表、模型类型、超参数、评估指标、训练时长、随机种子。

4. **可重复性保证**：固定随机种子 + 锁定依赖版本（`requirements.txt` / `conda.yml`），确保同样的代码和数据能产出完全相同的结果。

5. **结果不删除**：即使实验效果很差，也应保留记录——"什么不 work"与"什么 work"同样有价值。
{% endblock %}

### 渐进式改进路径

模型开发通常遵循"先易后难"的渐进路径，每一步都建立在前一步的基础上：

| 阶段 | 改进方向 | 典型实验数 | 预期提升 |
|------|---------|----------|---------|
| ① 基线建立 | MVM（默认参数简单模型） | 1–2 | — |
| ② 特征迭代 | 增删特征、特征变换、交互项 | 5–15 | 最大（通常 50–80% 的提升来自这一步） |
| ③ 模型迭代 | 更换模型类型（LR→RF→GBDT） | 3–5 | 中等 |
| ④ 超参数调优 | 网格搜索 / 贝叶斯优化 | 10–50 | 较小（通常 < 5%） |
| ⑤ 集成 / 融合 | Stacking、Blending | 2–5 | 最小但可能关键 |

{% hint style="warning" %}
⚠️ **收益递减法则**

当连续 3 次实验的主指标提升幅度 < 0.5% 时，应认真考虑是否继续迭代。此时通常处于收益递减区——继续调参的边际收益可能不如投入时间在数据获取、特征工程或部署优化上。
{% endhint %}

## 14.4.3 模型比较的统计检验

"模型 A 的 AUC 是 0.912，模型 B 是 0.908，所以 A 更好"——这个推理可能是错的。在有限的测试集上，0.004 的差异可能完全由随机波动解释。需要**统计检验**来判断差异是否显著。

### 常用检验方法

| 方法 | 适用场景 | 原假设 $$H_0$$ | 优势 |
|------|---------|------------|------|
| DeLong 检验 | AUC 比较 | 两个 AUC 相等 | 专为 ROC 曲线设计，考虑了预测相关性 |
| McNemar 检验 | 分类准确率 | 两个分类器错误率相同 | 只需预测标签，不需概率 |
| 配对 t 检验 | K 折交叉验证 | K 折评分均值相等 | 考虑折间方差 |
| Wilcoxon 符号秩 | 非正态分布 | 两组中位数相等 | 非参数方法，更稳健 |

### 多模型比较的修正

当同时比较 $$m$$ 个模型时，多重比较使 I 类错误率膨胀。使用 **Bonferroni 修正**将显著性阈值调整为 $$\alpha / m$$，或使用 **Friedman 检验 + Nemenyi 后检验**进行多模型排名比较。

{% hint style="warning" %}
⚠️ **"在测试集上调模型"是最隐蔽的数据泄漏**

如果反复在测试集上评估不同模型，并基于测试集结果选择"最好的"——实质上已经将测试集变成了验证集。正确做法：用**验证集**（或交叉验证）指导模型选择和调参，**测试集只用一次**——在最终模型确定后进行唯一一次评估。
{% endhint %}

## 14.4.4 实验报告与质量关口 G4

{% block type="definition" %}
**定义 14.12**（实验报告模板）　实验报告应包含以下要素：

| 编号 | 要素 | 内容 |
|------|------|------|
| 1 | 实验概述 | 目标、数据版本、评估协议 |
| 2 | 基线结果 | MVM 的性能指标 |
| 3 | 实验日志 | 全部实验的记录表（含失败实验） |
| 4 | 最终模型 | 选定模型的类型、超参数、特征列表 |
| 5 | 性能对比 | 最终模型 vs 基线的统计检验结果 |
| 6 | 误差分析 | 模型在哪些子群体/场景上表现不佳 |
| 7 | 公平性检查 | 模型在受保护属性上是否存在偏差 |
| 8 | 部署建议 | 推荐的部署方式、资源需求、已知局限 |
{% endblock %}

## 本节小结

| 关键概念 | 说明 |
|---------|------|
| 基线优先 | 先建立最简单的端到端模型，作为改进的参照锚点 |
| 实验规范 | 固定评估协议、单一变量控制、全量元数据记录 |
| 渐进式改进 | 基线 → 特征 → 模型 → 调参 → 集成，先易后难 |
| 统计检验 | 用 DeLong/McNemar/配对 t 检验判断模型差异是否显著 |
| 测试集纪律 | 测试集只用一次，不参与模型选择和调参 |
| 实验报告 | 质量关口 G4 的交付物，含基线、日志、检验、误差分析 |

## 习题

### A 级（概念理解）

**A1.** 解释"无基线则无进步"的含义。如果一个团队跳过基线直接训练 XGBoost 并获得 AUC=0.90，为什么仍然不知道这个结果"好不好"？

**A2.** 为什么"在测试集上调模型"是一种数据泄漏？它与 §1.3 中讨论的特征层面的数据泄漏有什么共同点？

### B 级（分析推理）

**B1.** 模型 A 的 5 折交叉验证 AUC 为 [0.88, 0.91, 0.87, 0.90, 0.89]，模型 B 为 [0.89, 0.90, 0.88, 0.91, 0.90]。仅从这些数字能否判断 B 显著优于 A？需要什么统计方法来回答？

### C 级（方法应用）

**C1.** 为 §13.2 的信用卡欺诈检测项目设计一个完整的实验方案：(a) 选择基线模型和评估协议；(b) 规划 5 轮实验的变量控制策略；(c) 确定使用哪种统计检验来比较最终候选模型。

### D 级（编程实践）

**D1.** 使用 `scipy.stats` 实现 McNemar 检验：输入两个分类器在同一测试集上的预测标签，输出检验统计量和 p 值。在 §13.6 的数据集上比较 Random Forest 和 Gradient Boosting 的预测差异是否显著。

### E 级（案例分析）

**E1.** 回顾 §13.7 的模型对比（LR/Ridge/RF/GB），按定义 14.12 的模板编写一份实验报告。重点分析为什么线性模型优于集成模型。

### F 级（开放探讨）

**F1.** 讨论"可重复性危机"在数据挖掘领域的表现。如果一篇论文声称其模型比基线提升了 2%，但没有公开代码、数据和随机种子，你如何评估这个结论的可信度？（不少于 300 字。）
